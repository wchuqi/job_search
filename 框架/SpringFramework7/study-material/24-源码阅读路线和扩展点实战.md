# Spring Framework 7 学习资料：源码阅读路线和扩展点实战

[返回索引](../SpringFramework7学习资料.md)

## 学习目标

- 建立 Spring 源码阅读顺序，避免一上来陷入细节。
- 掌握常用扩展点的使用时机和边界。
- 能通过小实验验证源码机制。

## 源码阅读顺序

建议按主线读，不按包名硬翻。

```text
1. ApplicationContext.refresh()
2. BeanDefinition 解析和注册
3. getBean() / doCreateBean()
4. BeanPostProcessor / BeanFactoryPostProcessor
5. 自动装配 resolveDependency()
6. AOP 自动代理创建器
7. TransactionInterceptor
8. DispatcherServlet.doDispatch()
9. RequestMappingHandlerAdapter
10. Spring TestContext
```

## 第一条主线：容器启动

重点类：

- `AbstractApplicationContext`
- `DefaultListableBeanFactory`
- `ConfigurationClassPostProcessor`
- `ClassPathBeanDefinitionScanner`
- `AutowiredAnnotationBeanPostProcessor`

阅读目标：

- 配置如何变成 BeanDefinition。
- BeanDefinition 何时被修改。
- 普通 Bean 何时开始实例化。

## 第二条主线：Bean 创建

重点方法方向：

- `getBean`
- `doGetBean`
- `createBean`
- `doCreateBean`
- `populateBean`
- `initializeBean`

阅读目标：

- 缓存在哪里查。
- 什么时候实例化。
- 什么时候依赖注入。
- 什么时候执行后置处理器。
- 什么时候生成代理。

## 第三条主线：AOP 和事务

重点类方向：

- 自动代理创建器。
- Advisor 查找和匹配。
- 代理工厂。
- 方法拦截器链。
- `TransactionInterceptor`。
- `PlatformTransactionManager` 实现。

阅读目标：

- 为什么目标对象会变成代理。
- 哪些方法会被拦截。
- 事务如何加入拦截器链。
- 事务资源如何绑定到线程。

## 第四条主线：Web MVC

重点类方向：

- `DispatcherServlet`
- `RequestMappingHandlerMapping`
- `RequestMappingHandlerAdapter`
- `HandlerMethodArgumentResolver`
- `HandlerMethodReturnValueHandler`
- `HttpMessageConverter`
- `HandlerExceptionResolver`

阅读目标：

- 请求如何找到 Controller 方法。
- 参数如何从请求转成 Java 对象。
- 返回值如何写成 JSON。
- 异常如何变成响应。

## 扩展点实战清单

| 扩展点 | 适合做什么 | 风险 |
| --- | --- | --- |
| `BeanFactoryPostProcessor` | 修改 BeanDefinition | 过度修改会让配置难懂 |
| `BeanPostProcessor` | 包装或检查 Bean 实例 | 可能导致过早创建 |
| `ImportBeanDefinitionRegistrar` | 框架式批量注册 Bean | 调试复杂 |
| `FactoryBean` | 封装复杂对象创建 | Bean 本体和产品易混淆 |
| `HandlerMethodArgumentResolver` | 自定义 Controller 参数 | 注册顺序影响解析 |
| `ResponseBodyAdvice` | 统一响应增强 | 不要破坏文件下载和流式响应 |
| `HandlerInterceptor` | MVC 层审计和耗时 | 不适合替代安全过滤链 |

## 实验：自定义参数解析器

```java
@Target(ElementType.PARAMETER)
@Retention(RetentionPolicy.RUNTIME)
public @interface CurrentUser {
}

class CurrentUserArgumentResolver implements HandlerMethodArgumentResolver {
    @Override
    public boolean supportsParameter(MethodParameter parameter) {
        return parameter.hasParameterAnnotation(CurrentUser.class)
            && parameter.getParameterType().equals(UserPrincipal.class);
    }

    @Override
    public Object resolveArgument(
        MethodParameter parameter,
        ModelAndViewContainer mavContainer,
        NativeWebRequest webRequest,
        WebDataBinderFactory binderFactory
    ) {
        String userId = webRequest.getHeader("X-User-Id");
        return new UserPrincipal(userId);
    }
}
```

## 实验：BeanDefinition 注册

```java
class FeatureRegistrar implements ImportBeanDefinitionRegistrar {
    @Override
    public void registerBeanDefinitions(
        AnnotationMetadata metadata,
        BeanDefinitionRegistry registry
    ) {
        var bd = BeanDefinitionBuilder
            .genericBeanDefinition(FeatureService.class)
            .getBeanDefinition();
        registry.registerBeanDefinition("featureService", bd);
    }
}
```
