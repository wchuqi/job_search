# Spring Framework 7 学习资料：环境版本和项目结构

[返回索引](../SpringFramework7学习资料.md)

## 学习目标

- 建立 Spring Framework 7.x 的环境基准。
- 会用 Maven 创建非 Spring Boot 的 Spring 项目。
- 理解模块依赖和包命名迁移风险。

## 版本基准

截至 2026-06-06，Maven Central 显示 `org.springframework:spring-core` 最新发布版本为 `7.0.7`。学习资料采用 Spring Framework 7.x 作为主线。

> **重点：** 学习 Spring Framework 时，先锁定 `spring-context`、`spring-webmvc`、`spring-jdbc`、`spring-tx` 等框架模块，再根据需要选择 Spring Boot、Spring Data、Spring Security 等生态项目。

## 推荐前置知识

- Java 17+ 语法和运行模型。
- Maven 坐标、依赖范围、传递依赖和版本管理。
- Servlet、HTTP、JSON、JDBC、事务基本概念。
- JUnit 5 和 Mockito 基础。

## 最小 POM

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>spring-framework-demo</artifactId>
    <version>1.0.0</version>

    <properties>
        <maven.compiler.release>17</maven.compiler.release>
        <spring.version>7.0.7</spring.version>
        <junit.version>5.13.0</junit.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-context</artifactId>
            <version>${spring.version}</version>
        </dependency>
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-test</artifactId>
            <version>${spring.version}</version>
            <scope>test</scope>
        </dependency>
        <dependency>
            <groupId>org.junit.jupiter</groupId>
            <artifactId>junit-jupiter</artifactId>
            <version>${junit.version}</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>
```

## 模块选择

| 需求 | 常用模块 |
| --- | --- |
| 容器和配置 | `spring-core`、`spring-beans`、`spring-context` |
| AOP | `spring-aop`、`spring-aspects` |
| 事务 | `spring-tx` |
| JDBC | `spring-jdbc` |
| Web MVC | `spring-web`、`spring-webmvc` |
| WebFlux | `spring-web`、`spring-webflux` |
| 测试 | `spring-test` |

## Jakarta 命名空间

现代 Spring Framework 使用 Jakarta EE 命名空间，很多旧代码里的 `javax.servlet.*`、`javax.validation.*`、`javax.persistence.*` 已迁移到 `jakarta.*`。

> **易错：** 升级项目时只改 Spring 版本，但依赖仍停留在旧 `javax.*` API，会导致编译或运行期类找不到。

## 练习

1. 创建一个 Maven 项目，只引入 `spring-context`。
2. 编写 `AppConfig` 和 `Main`，启动 `AnnotationConfigApplicationContext`。
3. 添加 `spring-webmvc` 后观察 Maven 依赖树变化。

## 验收

- 能说出 Spring Framework 学习最小依赖是什么。
- 能解释 Spring Framework、Boot Starter、Jakarta API 三类依赖的区别。
- 能通过 `mvn dependency:tree` 判断依赖冲突。
