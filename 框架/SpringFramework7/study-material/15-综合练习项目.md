# Spring Framework 7 学习资料：综合练习项目

[返回索引](../SpringFramework7学习资料.md)

## 项目目标

实现一个不依赖 Spring Boot 的订单管理 API，用 Spring Framework 7.x 覆盖 IoC、AOP、事务、JDBC、MVC、校验、异常处理和测试。

## 功能范围

- 创建订单。
- 查询订单详情。
- 分页查询订单。
- 取消订单。
- 记录审计日志。
- 统一错误响应。
- 参数校验。
- MockMvc 测试和事务集成测试。

## 必做任务

1. 使用 `@Configuration` 和 `@ComponentScan` 启动容器。
2. 使用构造器注入完成分层依赖。
3. 使用 `JdbcTemplate` 访问数据库。
4. 在 `OrderService` 上声明事务。
5. `AuditService` 使用 `REQUIRES_NEW` 记录独立审计日志。
6. 编写 `@Around` 切面记录 Service 方法耗时。
7. Controller 使用 `@Valid @RequestBody`。
8. 使用 `@RestControllerAdvice` 统一异常响应。
9. 使用 MockMvc 覆盖正常和异常请求。
10. 故意制造事务失败并验证回滚。

## 数据表

```sql
create table orders (
    id bigint generated always as identity primary key,
    order_no varchar(64) not null unique,
    amount decimal(18,2) not null,
    status varchar(32) not null,
    created_at timestamp not null
);

create table audit_log (
    id bigint generated always as identity primary key,
    action varchar(64) not null,
    detail varchar(512) not null,
    created_at timestamp not null
);
```

## 验收标准

- 容器能启动，所有 Bean 使用构造器注入。
- 创建订单成功时订单和审计日志都写入。
- 创建订单失败时订单回滚；使用 `REQUIRES_NEW` 的审计日志按设计独立提交或明确不记录。
- 参数非法返回统一 400 JSON。
- 未找到订单返回统一 404 JSON。
- Controller 测试覆盖参数绑定、校验、异常处理。
- Service 集成测试覆盖事务回滚。
- 能解释每个 Spring 注解背后的机制。

## 扩展任务

- 增加 Profile：dev 使用 H2，prod 使用 PostgreSQL。
- 增加自定义 Converter，把字符串转换为订单号值对象。
- 增加 Interceptor 记录 requestId 和耗时。
- 增加 WebFlux SSE 接口推送订单状态变更。

## 重点

这个项目的目标不是做复杂业务，而是把 Spring 的核心机制连成一条真实链路：请求进入 MVC，参数完成绑定和校验，Service 事务保护业务，Repository 写数据库，AOP 记录耗时，测试验证行为。
