# Spring Framework 7 学习资料：深度实验手册和能力验收

[返回索引](../SpringFramework7学习资料.md)

## 学习目标

用可运行实验验证 Spring 核心机制。每个实验都要求记录：实验目的、代码、现象、解释、易错点、回归测试。

## 实验 1：refresh 主线和 BeanDefinition

### 任务

- 创建非 Boot Spring 项目。
- 注册 `BeanFactoryPostProcessor` 打印所有 BeanDefinition。
- 使用 `@Conditional` 控制一个 Bean 是否注册。

### 验收

- 能说明 BeanDefinition 出现的时机。
- 能证明条件不满足时 BeanDefinition 不存在。

## 实验 2：Bean 生命周期和三级缓存

### 任务

- 创建 A、B setter 循环依赖。
- 添加生命周期回调和 BeanPostProcessor。
- 给 A 添加切面，观察 B 中注入的 A 是否代理。

### 验收

- 能解释日志顺序。
- 能说明早期引用为什么可能是代理。
- 能解释构造器循环依赖为什么失败。

## 实验 3：自动装配候选算法

### 任务

- 定义接口 `PaymentClient` 和 3 个实现。
- 分别使用 `@Qualifier`、`@Primary`、参数名、`@Order`。
- 注入单值、List、Map、ObjectProvider。

### 验收

- 能预测每种注入结果。
- 能故意制造并解决 `NoUniqueBeanDefinitionException`。

## 实验 4：AOP 调用链

### 任务

- 写两个 `@Around` 切面。
- 设置不同 `@Order`。
- 打印代理类型和调用顺序。
- 制造同类内部调用失效。

### 验收

- 能画出拦截器链。
- 能说明自调用为什么绕过代理。

## 实验 5：事务传播和资源绑定

### 任务

- 使用 JDBC 和本地事务管理器。
- 写 `REQUIRED`、`REQUIRES_NEW`、`NESTED` 实验。
- 测试 RuntimeException、checked exception、异常被 catch 的回滚差异。
- 在事务中启动新线程写数据库。

### 验收

- 能解释每条数据最终存在或不存在的原因。
- 能说明事务和线程绑定关系。

## 实验 6：MVC 分发链路

### 任务

- 写 REST API，覆盖 `@PathVariable`、`@RequestParam`、`@RequestBody`、校验。
- 自定义 `HandlerMethodArgumentResolver`。
- 自定义 `ResponseBodyAdvice`。
- 构造 404、405、415、406、400。

### 验收

- 能按状态码定位失败阶段。
- 能说明参数解析器和消息转换器的作用。

## 实验 7：WebFlux 阻塞边界

### 任务

- 写 SSE 接口。
- 用 WebClient 调用慢接口。
- 在链路中分别使用阻塞调用、`boundedElastic` 隔离、纯非阻塞调用。
- 做简单并发压测。

### 验收

- 能解释延迟和线程占用差异。
- 能说明什么时候不该用 WebFlux。

## 实验 8：生产排障复盘

### 任务

对以下故障各写一份复盘：

- Bean 找不到。
- Bean 多候选。
- 事务未回滚。
- AOP 不生效。
- MVC 404。
- JSON 415。
- 参数校验 500。
- 慢 SQL 导致接口慢。

### 验收

每份复盘必须包含：

- 现象。
- 复现步骤。
- 根因。
- 修复。
- 回归测试。
- 预防措施。

## 最终能力验收

能做到以下水平，才算真正掌握 Spring Framework 核心：

- 不看资料能讲出 `refresh()` 主线。
- 不看资料能讲出 `getBean()` 创建 Bean 的主线。
- 能解释三级缓存和循环依赖边界。
- 能预测自动装配多候选结果。
- 能画出 AOP 拦截器链。
- 能解释事务传播的资源语义。
- 能从 HTTP 状态码定位 MVC 处理阶段。
- 能判断 MVC 和 WebFlux 的选型。
- 能写容器扩展点和 MVC 扩展点。
- 能为生产故障写回归测试。
