# Spring Framework 7 学习资料：校验、类型转换和国际化

[返回索引](../SpringFramework7学习资料.md)

## 学习目标

- 掌握 Bean Validation、数据绑定、类型转换。
- 理解 `ConversionService`、Formatter、Validator 的职责。
- 会设计统一错误响应和国际化消息。

## 理论导读

Web 应用接收的都是字符串、JSON 或表单数据，而业务代码需要 Java 类型和合法对象。Spring 在 Controller 调用前完成数据绑定、类型转换和校验，避免业务层到处手写参数检查。

校验不是“前端已经校验过就不用后端做”。后端校验是安全边界，必须独立存在。

## Bean Validation 示例

```java
record CreateOrderRequest(
    @NotBlank String orderNo,
    @NotNull @DecimalMin("0.01") BigDecimal amount,
    @NotEmpty List<@NotBlank String> itemCodes
) {
}
```

Controller 中启用校验：

```java
@PostMapping("/orders")
OrderResponse create(@Valid @RequestBody CreateOrderRequest request) {
    return orderService.create(request);
}
```

## 分组校验

```java
interface CreateGroup {}
interface UpdateGroup {}

record UserRequest(
    @Null(groups = CreateGroup.class)
    @NotNull(groups = UpdateGroup.class)
    Long id,

    @NotBlank
    String name
) {
}
```

分组校验适合创建和更新规则不同的场景，但不要滥用到难以阅读。

## 类型转换

Spring 使用 `ConversionService` 统一处理字符串到目标类型的转换。

```java
@Component
class StringToMoneyConverter implements Converter<String, Money> {
    @Override
    public Money convert(String source) {
        return Money.parse(source);
    }
}
```

常见转换场景：

- `String` 到 `Long`、`Integer`、`Enum`。
- ISO 日期字符串到 `LocalDate`、`LocalDateTime`。
- 请求参数到领域值对象。

## 国际化消息

```java
@Bean
MessageSource messageSource() {
    var source = new ResourceBundleMessageSource();
    source.setBasenames("messages");
    source.setDefaultEncoding("UTF-8");
    return source;
}
```

`messages_zh_CN.properties`：

```properties
order.amount.min=订单金额必须大于 0
```

## 统一错误响应

```java
@RestControllerAdvice
class ValidationExceptionHandler {
    @ExceptionHandler(MethodArgumentNotValidException.class)
    ResponseEntity<ApiError> handle(MethodArgumentNotValidException ex) {
        var fields = ex.getBindingResult().getFieldErrors().stream()
            .map(error -> error.getField() + ": " + error.getDefaultMessage())
            .toList();
        return ResponseEntity.badRequest()
            .body(new ApiError("VALIDATION_FAILED", fields));
    }
}
```

## 易错

> **易错：** 只在 DTO 上写注解，但 Controller 参数没有 `@Valid` 或 `@Validated`。
>
> 正确做法：需要校验的入参必须触发校验入口。

> **易错：** 把所有校验塞进 Controller。
>
> 正确做法：格式和基础合法性放 DTO 校验，业务规则放 Service 或领域对象。
