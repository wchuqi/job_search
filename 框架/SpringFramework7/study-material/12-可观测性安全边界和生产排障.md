# Spring Framework 7 学习资料：可观测性、安全边界和生产排障

[返回索引](../SpringFramework7学习资料.md)

## 学习目标

- 掌握 Spring 应用常见故障定位路径。
- 理解日志、指标、追踪、健康检查在生产中的作用。
- 明确 Spring Framework 层面的安全边界。

## 理论导读

生产问题很少只来自一行代码。Spring 应用故障常由容器装配、代理边界、事务配置、线程池、连接池、序列化、HTTP 网关、数据库状态共同造成。排障需要沿框架链路定位，而不是只看业务异常。

## 常见故障矩阵

| 现象 | 可能原因 | 排查入口 |
| --- | --- | --- |
| 启动失败 | Bean 缺失、冲突、循环依赖、配置缺失 | cause 链、条件报告、Bean 名称 |
| 接口 404 | 路径不匹配、Controller 未扫描、Servlet 映射错误 | HandlerMapping 日志 |
| 415 Unsupported Media Type | Content-Type 不匹配、缺消息转换器 | 请求头、HttpMessageConverter |
| 400 参数错误 | JSON 格式、类型转换、校验失败 | BindingResult、异常处理器 |
| 事务未回滚 | 异常类型、代理失效、异常被吞 | AOP 代理、rollbackFor |
| 请求慢 | DB、远程调用、锁、线程池、序列化 | trace、慢 SQL、线程栈 |

## 日志建议

- 为每个请求生成 requestId 或 traceId。
- 入口记录方法、路径、状态码、耗时。
- 业务异常记录可读错误码，系统异常保留堆栈。
- 不在日志中打印密码、Token、身份证、银行卡等敏感信息。

## 可观测性层次

| 层次 | 看什么 |
| --- | --- |
| 日志 | 单次请求发生了什么 |
| 指标 | 系统整体趋势，如 QPS、错误率、延迟 |
| 链路追踪 | 一次请求跨服务耗时分布 |
| 线程栈 | 卡在哪里 |
| 堆转储 | 内存对象占用 |

## 安全边界

Spring Framework 提供 Web、数据绑定、资源加载等基础能力，但认证授权通常由 Spring Security 承担。即便不讲 Security，也必须注意：

- 不信任任何客户端输入。
- JSON 反序列化和数据绑定不能开放敏感字段。
- 文件上传要限制类型、大小、存储路径。
- 错误响应不要泄漏堆栈、SQL、内部路径。
- SpEL、模板、反射调用要谨慎处理外部输入。

## 排障步骤

1. 确定故障层级：启动、请求路由、参数绑定、业务逻辑、事务、数据库、外部服务。
2. 找第一处异常，而不是最后一层包装异常。
3. 复现最小案例。
4. 打开相关包 DEBUG 日志，例如 MVC 映射或事务拦截。
5. 验证配置是否实际生效。
6. 写回归测试。

## 练习

1. 制造 Bean 多候选冲突，写排查记录。
2. 制造事务未回滚，写根因分析。
3. 制造 Controller 415 和 400，写 HTTP 请求和响应对照。

## 验收

- 能根据异常判断大致层级。
- 能通过日志定位请求链路问题。
- 能列出 Spring 应用至少 8 类安全风险。

## 重点

排障最重要的是还原链路：容器链路、代理链路、请求链路、事务链路。链路清楚后，错误位置通常会变得明确。
