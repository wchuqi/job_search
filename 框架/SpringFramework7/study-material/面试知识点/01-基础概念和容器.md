## 一、基础概念和容器

### 1. Spring Framework 和 Spring Boot 有什么区别？

**参考答案：**

Spring Framework 是基础框架，提供 IoC、AOP、事务、Web MVC、WebFlux、数据访问、测试等能力。Spring Boot 是基于 Spring Framework 的应用快速开发框架，提供自动配置、Starter、嵌入式服务器、Actuator、外部化配置等能力。

Spring Boot 依赖 Spring Framework，但 Spring Framework 可以不依赖 Boot 单独使用。

> **重点：** Framework 讲机制，Boot 讲约定和自动装配。
> **易错：** 把 `@SpringBootApplication` 当成 Spring Framework 的核心入口。

### 2. BeanFactory 和 ApplicationContext 有什么区别？

**参考答案：**

`BeanFactory` 是基础 IoC 容器，负责 Bean 创建和依赖管理。`ApplicationContext` 是更完整的应用上下文，在 BeanFactory 基础上提供国际化、事件发布、资源加载、环境抽象、自动注册后置处理器等能力。

实际应用通常直接使用 `ApplicationContext`。

> **重点：** ApplicationContext 是 BeanFactory 的增强版，不只是另一个名字。

### 3. 什么是 BeanDefinition？

**参考答案：**

BeanDefinition 是 Bean 的元数据，记录 Bean class、作用域、构造参数、属性、初始化方法、销毁方法、懒加载等信息。Spring 容器启动时先注册 BeanDefinition，再根据它创建 Bean 实例。

> **难点：** BeanDefinition 是“施工图”，Bean 实例是“建出来的对象”。

### 4. `@Component`、`@Service`、`@Repository`、`@Controller` 有什么区别？

**参考答案：**

它们都可以把类注册为 Spring Bean。`@Service` 表示业务服务，`@Repository` 表示持久层并参与异常转换，`@Controller` 表示 MVC 控制器，`@Component` 是通用组件标识。

> **重点：** 语义化注解有助于分层和框架处理。

### 5. Spring 容器启动大致做了什么？

**参考答案：**

读取配置和扫描类，注册 BeanDefinition，准备 BeanFactory，调用 BeanFactoryPostProcessor，注册 BeanPostProcessor，实例化非懒加载单例 Bean，完成依赖注入、初始化和代理创建，发布上下文刷新完成事件。

> **易错：** 以为扫描到类就等于对象已经创建。扫描阶段主要注册定义。
