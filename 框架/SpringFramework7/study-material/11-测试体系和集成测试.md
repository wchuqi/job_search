## 理论导读

Spring 应用的测试分为两类：不启动容器的普通单元测试，以及启动 Spring 上下文的集成测试。前者快，适合纯业务逻辑；后者慢，但能验证真实装配、事务、MVC 参数绑定、异常处理等框架行为。

好的测试不是全部启动完整应用，而是根据风险选择最小必要环境。

## Spring TestContext

```java
@ExtendWith(SpringExtension.class)
@ContextConfiguration(classes = AppConfig.class)
class OrderServiceIntegrationTest {
    @Autowired
    OrderService orderService;

    @Test
    void createOrder() {
        OrderResponse response = orderService.create(new CreateOrderRequest("A1001"));
        assertNotNull(response.id());
    }
}
```

## 事务测试

```java
@Transactional
@ExtendWith(SpringExtension.class)
@ContextConfiguration(classes = TestDataConfig.class)
class OrderRepositoryTest {
    @Autowired
    OrderRepository repository;

    @Test
    void insertAndFind() {
        long id = repository.insert("A1001");
        assertNotNull(repository.findById(id));
    }
}
```

测试方法上的事务通常在测试结束后回滚，避免污染数据。

## MockMvc

```java
@ExtendWith(SpringExtension.class)
@WebAppConfiguration
@ContextConfiguration(classes = WebConfig.class)
class OrderControllerTest {
    @Autowired
    WebApplicationContext context;

    MockMvc mockMvc;

    @BeforeEach
    void setUp() {
        mockMvc = MockMvcBuilders.webAppContextSetup(context).build();
    }

    @Test
    void createOrder() throws Exception {
        mockMvc.perform(post("/api/orders")
                .contentType(MediaType.APPLICATION_JSON)
                .content("{\"orderNo\":\"A1001\"}"))
            .andExpect(status().isCreated());
    }
}
```

## 测试粒度

| 测试类型 | 是否启动 Spring | 适合验证 |
| --- | --- | --- |
| 单元测试 | 否 | 纯业务逻辑、算法、边界条件 |
| 容器测试 | 是 | Bean 装配、Profile、条件配置 |
| Repository 测试 | 是 | SQL、事务、异常转换 |
| MVC 测试 | 是 | 路由、参数绑定、校验、异常响应 |
| 端到端测试 | 是，且更完整 | 用户关键流程 |

## 上下文缓存

Spring Test 会缓存 ApplicationContext。测试配置相同可以复用上下文，配置不同会重新启动，影响速度。

> **易错：** 每个测试类都随意改配置，导致上下文缓存失效，测试变慢。

## 练习

1. 为 Service 写不启动 Spring 的单元测试。
2. 为 Repository 写事务回滚集成测试。
3. 为 Controller 写 MockMvc 测试，覆盖校验失败和正常创建。

- 能解释 Spring 测试上下文缓存。
