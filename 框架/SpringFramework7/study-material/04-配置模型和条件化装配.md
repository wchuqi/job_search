# Spring Framework 7 学习资料：配置模型和条件化装配

[返回索引](../SpringFramework7学习资料.md)

## 学习目标

- 掌握 `@Configuration`、`@Bean`、`@ComponentScan`、`@Import`。
- 理解 Environment、PropertySource、Profile 的作用。
- 区分 Spring Framework 条件装配和 Spring Boot 自动配置。

## 理论导读

配置的本质是告诉容器“有哪些对象、它们如何创建、在哪些环境下生效”。Spring 支持 Java 配置、注解扫描、XML、环境属性和条件化装配。现代项目主要使用 Java 配置和组件扫描。

`@Configuration` 类会被 Spring 增强，以保证同一个配置类中调用 `@Bean` 方法时仍能拿到容器管理的单例对象。这一点和普通 Java 方法调用不同。

## 配置方式对比

| 方式 | 适用场景 | 注意点 |
| --- | --- | --- |
| `@Bean` | 第三方类、复杂创建逻辑 | 方法名默认是 Bean 名 |
| `@Component` | 自有业务类 | 需要组件扫描 |
| `@Import` | 模块化导入配置 | 适合显式组装 |
| XML | 老项目或外部可编辑配置 | 新项目少用 |
| Profile | 环境差异 | 不适合承载复杂业务分支 |

## 例子：显式配置

```java
@Configuration
@ComponentScan("com.example.app")
@PropertySource("classpath:application.properties")
class AppConfig {
    @Bean
    Clock clock() {
        return Clock.systemUTC();
    }
}
```

## Environment 和属性

```java
@Component
class MailClient {
    private final String host;

    MailClient(Environment environment) {
        this.host = environment.getRequiredProperty("mail.host");
    }
}
```

也可以使用 `@Value`：

```java
@Component
class MailClient {
    MailClient(@Value("${mail.host}") String host) {
        this.host = host;
    }
}
```

> **易错：** 在 Spring Framework 中，类型安全配置绑定不像 Spring Boot 那样开箱即用。Boot 的 `@ConfigurationProperties` 是 Boot 体系提供的强能力。

## Profile

```java
@Configuration
class DataSourceConfig {
    @Bean
    @Profile("dev")
    DataSource devDataSource() {
        return new HikariDataSource();
    }

    @Bean
    @Profile("prod")
    DataSource prodDataSource() {
        return new HikariDataSource();
    }
}
```

Profile 适合环境切换，例如 dev、test、prod。不要把大量业务开关放进 Profile，否则运行组合会难以测试。

## 条件化装配

Spring Framework 提供 `@Conditional` 扩展点：

```java
class OnWindowsCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        return System.getProperty("os.name").toLowerCase().contains("win");
    }
}

@Bean
@Conditional(OnWindowsCondition.class)
FileStorage fileStorage() {
    return new WindowsFileStorage();
}
```

Spring Boot 的 `@ConditionalOnClass`、`@ConditionalOnMissingBean` 等是 Boot 自动配置体系的增强，不属于 Spring Framework 基础注解。

## 练习

1. 用 `@Import` 拆分 `WebConfig`、`ServiceConfig`、`RepositoryConfig`。
2. 用 `@Profile` 实现 dev 和 prod 两套数据源配置。
3. 自定义 `@Conditional`，根据系统属性启用某个 Bean。

## 验收

- 能解释 `@Configuration` 和普通 `@Component` 配置类的差异。
- 能说明配置属性来源和覆盖关系。
- 能区分 Framework 条件装配和 Boot 自动配置。

## 重点

配置越隐式，启动越方便；配置越显式，排障越直接。学习 Spring Framework 阶段建议先用显式配置理解机制，再学习 Boot 自动配置。
