# Spring Cloud 学习资料：综合练习项目

[返回索引](../SpringCloud学习资料.md)

## 项目目标

构建一个最小但接近真实的电商微服务系统，用来串联 Spring Cloud 核心能力。

## 项目结构

```text
spring-cloud-shop/
  gateway-service/
  config-server/
  order-service/
  inventory-service/
  notification-service/
  common-api/
  docker-compose.yml
```

## 功能要求

### 1. Gateway

- `/api/orders/**` 转发到 order-service。
- `/api/inventory/**` 转发到 inventory-service。
- 增加 traceId。
- 健康检查公开，其他 Actuator 端点保护。
- 对创建订单接口限流。

### 2. Config Server

- 使用 Git 或本地文件作为配置仓库。
- order-service 读取订单限制配置。
- inventory-service 读取库存预占超时配置。
- 配置非法值应启动失败。

### 3. order-service

- 提供创建订单接口。
- 使用 Feign 调 inventory-service 预占库存。
- 创建订单时写 outbox 事件。
- 对库存调用设置超时、熔断。
- 支持幂等键。

### 4. inventory-service

- 提供查询库存、预占库存接口。
- 预占库存接口使用唯一业务键防重复。
- 模拟延迟和错误，用于验证熔断。

### 5. notification-service

- 使用 Spring Cloud Stream 消费订单创建事件。
- 消费者幂等。
- 失败消息进入 DLQ。

## 必做实验

1. 停掉 inventory-service，观察 order-service 熔断。
2. 启动两个 inventory-service 实例，观察负载均衡。
3. 写错 Gateway StripPrefix，定位 404。
4. 修改配置中心库存阈值，刷新并验证。
5. 制造 notification-service 消费失败，观察 DLQ。
6. 给 Feign 调用去掉超时，观察线程堆积风险。

## 验收标准

- Gateway、Feign、Config、CircuitBreaker、Stream 都被实际使用。
- 每个服务有 Actuator health。
- 日志包含 traceId。
- 创建订单有幂等保护。
- 库存服务故障不会拖垮订单服务。
- 消息消费者重复消费不会产生重复通知。
- 有 README 记录启动、测试、故障实验和排查过程。

## 重点

- 项目不是为了堆组件，而是验证分布式失败路径。
- 每个远程调用必须有超时、观测和保护。
- 事件驱动必须有幂等和 DLQ。

## 易错

> **易错：** 只实现正常流程，不验证故障。
>
> 正确做法：每个组件都要设计并测试失败场景。
