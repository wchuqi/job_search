## 一、基础架构

### 1. Spring Cloud 和 Spring Boot 有什么关系？

**参考答案：**

Spring Boot 解决单个应用的快速开发、自动配置、运行和生产就绪；Spring Cloud 基于 Spring Boot，为多个服务组成的分布式系统提供通用模式，例如服务发现、配置中心、网关、服务间调用、熔断限流、消息驱动和可观测性。Spring Cloud 不是替代 Boot，而是建立在 Boot 应用之上的分布式协作工具集。

> **重点：** Boot 是单服务底座，Cloud 是多服务协作能力。

### 2. Spring Cloud 2025.1.1 的版本基准是什么？

**参考答案：**

截至 2026-06-06，官方 Spring Cloud Release 文档显示最新稳定 Release Train 是 `2025.1.1`，支持 Spring Boot `4.0.2`。同页还列出 `2025.0.2`、`2024.0.3`、`2023.0.6` 稳定维护线。生产项目应使用稳定线，不使用 SNAPSHOT。

> **重点：** Cloud 和 Boot 版本必须按兼容矩阵成套选择。

### 3. 什么是 Release Train？为什么 Spring Cloud 要用 BOM？

**参考答案：**

Spring Cloud 包含多个子项目，Release Train 把这些子项目组合成一组经过验证的版本。BOM 用于统一依赖版本，业务项目导入 `spring-cloud-dependencies` 后，不需要逐个指定 Gateway、OpenFeign、Config、Stream 等子项目版本，避免版本不一致。

> **易错：** 为解决冲突手工覆盖核心依赖版本，可能引入 `NoSuchMethodError`。

### 4. 微服务项目一定要用 Spring Cloud 吗？

**参考答案：**

不一定。如果团队使用 Kubernetes DNS、Ingress、Service Mesh、云平台配置和托管消息服务，也可以只用部分 Spring Cloud 组件，甚至不用 Spring Cloud。选型取决于服务规模、团队能力、平台能力和治理需求。

> **重点：** Spring Cloud 是工具集，不是微服务充分条件。
