# Vue3学习资料：表单处理、校验和异步状态

[返回索引](../Vue3学习资料.md)

## 学习目标

- 掌握 `v-model`、表单草稿、校验、提交、防重复提交。
- 能处理加载、成功、失败、空状态和请求竞态。
- 理解表单状态和后端数据的边界。

## 理论导读

真实表单不是几个 input 的集合，而是一套状态机：初始化、编辑、脏数据、校验失败、提交中、提交成功、提交失败。Vue 只负责状态到视图的映射，状态机设计仍然需要开发者明确。

## 核心心智模型

把表单看成“草稿箱”。后端返回的数据是原件，用户编辑的是草稿。保存成功后，草稿才同步为新的原件。

## 知识点详解

| 状态 | 含义 | 处理方式 |
| --- | --- | --- |
| `initial` | 初始数据 | 从接口或默认值创建 |
| `draft` | 用户编辑数据 | 用本地响应式对象维护 |
| `dirty` | 草稿与原件不同 | 离开页面前提示 |
| `errors` | 校验错误 | 字段级错误映射 |
| `submitting` | 正在提交 | 禁用按钮，防重复 |

## 例子

```ts
import { computed, reactive, ref } from 'vue'

const form = reactive({ name: '', email: '' })
const submitting = ref(false)
const errors = reactive<Record<string, string>>({})

const canSubmit = computed(() => form.name.trim() && form.email.includes('@') && !submitting.value)

async function submit() {
  errors.name = form.name.trim() ? '' : 'Name is required'
  errors.email = form.email.includes('@') ? '' : 'Email is invalid'
  if (!canSubmit.value) return

  submitting.value = true
  try {
    await saveUser({ ...form })
  } finally {
    submitting.value = false
  }
}
```

## 练习

实现一个用户编辑页：加载用户详情、编辑草稿、字段校验、保存、保存失败提示、离开页面前确认。

## 验收

- 保存时不会重复提交。
- 校验错误能定位到字段。
- 加载失败和保存失败有不同提示。

## 重点

- 表单草稿不要直接绑定全局 store 中的原始对象。
- 异步状态至少要有 loading、error、empty、success。

## 难点

- 请求竞态、取消请求、乐观更新和回滚需要配套设计。

## 易错

> **易错：** 后端详情对象直接传给表单编辑，用户输入立刻污染列表缓存。
>
> 正确做法：进入编辑页时复制为草稿，保存成功后再更新缓存。
