## 一、基础概念和项目结构

### 1. Vue3 的核心思想是什么？

**参考答案：**

Vue3 的核心思想是声明式渲染、响应式状态和组件化。开发者描述状态如何映射成界面，Vue 负责追踪状态依赖，在状态变化时调度组件更新并 patch DOM。

Vue3 相比 Vue2 更强调组合式 API、Proxy 响应式、TypeScript 友好、编译优化和更好的 Tree-shaking。

> **重点：** 状态驱动视图，组件承载 UI 边界，响应式系统承载更新通知。
>
> **易错：** 只回答“Vue 是 MVVM 框架”不够，面试官更关注你能否说清状态到 DOM 的更新链路。

### 2. `script setup` 是什么？和普通 `setup` 有什么区别？

**参考答案：**

`script setup` 是 Vue SFC 的编译期语法糖。里面的顶层变量、函数、import 可以直接在模板中使用，`defineProps`、`defineEmits`、`defineExpose` 等宏在编译期处理。

普通 `setup` 需要显式 return 暴露给模板；`script setup` 更简洁，类型推导更自然。

```vue
<script setup lang="ts">
const props = defineProps<{ title: string }>()
</script>
```

> **重点：** `script setup` 不是运行时 API，而是编译期能力。

### 3. Vue3 项目中 `views`、`components`、`composables`、`stores` 如何划分？

**参考答案：**

`views` 放路由页面组件，承载页面级编排；`components` 放可复用 UI 组件；`composables` 放组合式逻辑复用；`stores` 放 Pinia 全局状态；`api` 放请求封装；`types` 放共享类型。

边界标准是职责和复用范围，不是文件大小。页面组件可以组合多个业务组件和 composable，但不应塞满所有请求、校验、状态和渲染细节。

> **易错：** 把所有组件都放 `components`，导致页面级业务组件和基础组件混在一起。

### 4. Vue3 与 React 在思路上有什么差异？

**参考答案：**

两者都是组件化 UI 框架。Vue 使用响应式依赖追踪，模板编译器可做静态提升和 patch flag 优化；React 更强调不可变状态、函数组件重新执行和显式 memo。Vue 模板更接近 HTML，React JSX 更接近 JavaScript 表达能力。

工程取舍上，Vue 对常规业务表单和后台系统上手更快；React 生态更大、函数式模型更统一。大型项目里，两者都需要清晰状态边界和组件设计。

> **难点：** 不要把差异简化成“Vue 双向绑定，React 单向数据流”。Vue 组件通信也强调单向数据流。
