## 一、组件通信和渲染机制

### 1. Vue 组件通信有哪些方式？如何选择？

**参考答案：**

父子组件用 props 和 emits；内容定制用 slots；跨多层但范围明确的依赖用 provide/inject；跨页面共享状态用 Pinia；路由参数和 URL query 用于表达页面位置和可分享筛选条件。

选择标准是状态所有权和通信距离。能用 props/emits 表达的，不要过早放到全局 store。

> **重点：** props down，events up。
>
> **易错：** 子组件直接修改 props 或全局 store，绕过父组件业务规则。

### 2. Vue 中 `key` 的作用是什么？

**参考答案：**

`key` 用来标识 vnode 身份。列表 diff 时，Vue 根据 `key` 判断节点能否复用、移动或需要创建销毁。稳定唯一的 key 可以避免 DOM 和组件实例状态错配。

数组下标在列表插入、删除、排序时不稳定，会导致输入框值、组件本地状态、动画状态串位。

> **重点：** `key` 不是为了消除 warning，而是为了稳定 vnode 身份。

### 3. 状态变化后 Vue 如何更新 DOM？

**参考答案：**

状态写入触发响应式 trigger，相关组件渲染 effect 被放入更新队列。Vue 会批量刷新队列，执行 render 生成新 vnode，再和旧 vnode patch，最后把差异应用到真实 DOM。

如果修改状态后立刻读 DOM，可能读到旧值；需要等待 `nextTick`。

> **难点：** 状态更新、组件 render、DOM patch、浏览器绘制是不同阶段。

### 4. `v-if` 和 `v-show` 怎么选？

**参考答案：**

`v-if` 会创建和销毁 DOM，适合条件不频繁变化、初始不需要渲染的内容。`v-show` 初始会渲染 DOM，只用 CSS 控制显示隐藏，适合频繁切换。

> **易错：** 权限敏感内容不能只靠 `v-show` 隐藏，因为 DOM 仍然存在。
