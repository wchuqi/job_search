## 一、Router、Pinia 和工程实践

### 1. Vue Router 的导航守卫常用于什么场景？

**参考答案：**

常用于登录校验、权限拦截、页面标题、埋点、离开页面前确认、动态路由加载。登录和权限这类横切逻辑通常放全局前置守卫；页面内部未保存表单可用组件内离开守卫。

守卫返回路由地址会重定向，返回 `false` 可取消导航。

> **易错：** 守卫中不排除登录页或错误页，造成无限重定向。

### 2. 动态路由参数变化时为什么组件没有重新创建？

**参考答案：**

当同一个路由记录匹配不同参数时，Vue Router 可能复用同一个组件实例，以提高效率。此时 `onMounted` 不会重新执行，需要监听 `route.params`，或使用 `onBeforeRouteUpdate`。

```ts
watch(() => route.params.id, id => {
  fetchDetail(String(id))
})
```

> **重点：** 参数变了不代表组件一定销毁重建。

### 3. Pinia 适合存什么？不适合存什么？

**参考答案：**

适合存跨组件、跨页面共享的应用状态，例如当前用户、权限、主题、语言、购物车。表单草稿、弹窗开关、局部 loading、页面临时筛选不一定适合放 Pinia。

持久化时要考虑安全、过期、版本迁移和退出清理。

> **易错：** 把 Pinia 当作所有状态的默认归宿，最终导致状态来源混乱。

### 4. Vue3 项目如何做权限菜单？

**参考答案：**

常见做法是路由表带 `meta` 权限标识，登录后获取用户角色或权限点，根据后端权限过滤菜单和可访问路由。前端负责体验和入口控制，后端仍必须做接口级鉴权。

> **重点：** 前端权限不能替代后端权限。
>
> **难点：** 动态路由、刷新恢复、退出清理、无权限页和 404 顺序都要处理。
