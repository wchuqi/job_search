## 一、性能、测试、SSR 和生产排障

### 1. Vue 应用卡顿你怎么排查？

**参考答案：**

先复现并用 Performance 面板录制，判断瓶颈是脚本执行、组件更新、布局、绘制、网络还是内存。再结合 Vue DevTools 看组件更新时间和 Pinia 状态变化。常见修复包括路由懒加载、虚拟列表、减少深度响应式、拆分组件、缓存派生值、取消重复请求。

> **重点：** 先测量再优化。
>
> **易错：** 没定位瓶颈就加缓存或 memo，可能让状态一致性更差。

### 2. 首屏白屏如何排查？

**参考答案：**

查看 Network 是否资源 404、JS 是否加载失败、控制台是否运行时报错、路由 history fallback 是否配置、接口是否阻塞首屏、构建 `base` 是否正确。还要检查浏览器兼容性、环境变量、CSP 和 CDN 缓存。

> **难点：** 白屏可能是构建路径、运行时异常、接口等待、权限重定向、浏览器兼容等多类问题。

### 3. SSR 和 hydration 是什么？

**参考答案：**

SSR 是服务端先把组件渲染成 HTML 返回给浏览器。hydration 是客户端加载 JS 后把这份 HTML 接管成可交互应用。服务端渲染结果必须和客户端首次渲染一致，否则会出现 hydration mismatch。

常见不一致来源：时间、随机数、客户端专属 API、用户本地状态、条件渲染差异。

> **易错：** 在服务端渲染阶段直接访问 `window` 或 `localStorage`。

### 4. Vue 项目测试怎么分层？

**参考答案：**

composable 和工具函数用 Vitest 做单元测试；组件交互用 Vue Test Utils 做组件测试；登录、下单、权限跳转等关键路径用 Playwright 或 Cypress 做 E2E。测试重点是行为和契约，不是内部实现细节。

> **重点：** 测试金字塔要分层，不能所有场景都靠 E2E。
