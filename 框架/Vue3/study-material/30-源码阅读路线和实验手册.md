# Vue3学习资料：源码阅读路线和实验手册

[返回索引](../Vue3学习资料.md)

## 学习目标

- 建立 Vue3 源码阅读顺序，避免一上来陷入细节。
- 通过实验理解响应式、编译器、渲染器、组件、Router、Pinia。
- 能把源码机制转换成面试回答和工程判断。

## 阅读原则

- 先跑实验，再读源码。
- 先读主链路，再读边界分支。
- 先画数据流，再背函数名。
- 读源码的目标是解释行为和设计取舍，不是记住每个文件。

## 推荐顺序

| 阶段 | 内容 | 目标 |
| --- | --- | --- |
| 1 | 响应式核心 | track、trigger、effect、computed |
| 2 | scheduler | job queue、nextTick、flush 时机 |
| 3 | 编译器产物 | template 到 render，patch flag |
| 4 | 渲染器 | mount、patch、keyed diff |
| 5 | 组件实例 | setup、props、slots、生命周期 |
| 6 | 内置组件 | KeepAlive、Teleport、Suspense |
| 7 | Router | matcher、导航守卫、动态路由 |
| 8 | Pinia | defineStore、插件、订阅 |

## 实验 1：手写最小响应式

目标：

- 实现 `reactive`。
- 实现 `effect`。
- 实现 `computed`。
- 加入 scheduler，支持批处理。

验收：

- 读取时收集依赖。
- 写入时触发依赖。
- computed 依赖不变时不重复计算。
- 同步多次写入只执行一次调度任务。

## 实验 2：观察响应式调试钩子

目标：

- 在组件中使用 `onRenderTracked` 和 `onRenderTriggered`。
- 找出组件依赖了哪些字段。
- 改造大对象 props，减少无意义更新。

验收：

- 能说明某次 render 是由哪个 key 触发。
- 能减少一次不必要的组件更新。

## 实验 3：分析模板编译产物

目标：

- 准备静态节点、动态 class、动态文本、v-for、slot。
- 查看 render 产物。
- 标出 patch flag 和静态提升。

验收：

- 能解释哪些节点被提升。
- 能解释动态节点如何被收集。
- 能解释复杂 slot 为什么可能影响更新边界。

## 实验 4：模拟 keyed diff

目标：

- 用纸面或代码模拟 `[A, B, C, D] -> [B, A, D, C]`。
- 标出可复用节点、需移动节点、需卸载节点。
- 理解最长递增子序列减少移动。

验收：

- 能解释 index key 为什么会复用错实例。
- 能说明稳定 key 和组件本地状态的关系。

## 实验 5：Router 权限状态机

目标：

- 画出未登录、已登录未加载权限、已加载权限、无权限、404 的状态迁移。
- 实现动态路由 add/remove。
- 刷新恢复和退出重置。

验收：

- 不出现无限重定向。
- 刷新后能进入授权页面。
- 退出后旧动态路由不可访问。

## 实验 6：Pinia 持久化插件

目标：

- 写一个 Pinia plugin。
- 支持字段白名单、版本号、过期时间。
- action hook 统计耗时。

验收：

- 敏感字段不持久化。
- 版本不匹配能迁移或清理。
- action 错误能上报。

## 实验 7：SSR 水合不一致复现

目标：

- 在首轮 render 使用 `Date.now()`。
- 观察 hydration mismatch。
- 改成服务端注入固定值或 mounted 后渲染。

验收：

- 能复现 warning。
- 能说明服务端和客户端首轮 render 差异。
- 能给出修复方案。

## 实验 8：性能与内存排查

目标：

- 构造大列表和第三方实例。
- 使用 Performance 录制卡顿。
- 使用 Memory 找未清理实例。
- 改成虚拟列表、`shallowRef`、卸载清理。

验收：

- 有优化前后指标。
- 进入离开页面多次后 heap 不持续增长。

## 重点

- 源码学习要形成“行为 -> 机制 -> 边界 -> 工程取舍”的闭环。

## 易错

> **易错：** 一上来逐文件读源码，读了很多但解释不了真实问题。
>
> 正确做法：先拿一个行为做最小复现，再沿主链路读源码，最后回到工程场景。
