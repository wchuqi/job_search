# Vue3学习资料：环境工具链和项目结构

[返回索引](../Vue3学习资料.md)

## 学习目标

- 会创建 Vue3 + Vite 项目。
- 理解 SFC、入口文件、路由、状态、组件目录的职责。
- 建立 TypeScript、ESLint、Prettier、测试工具的基础环境意识。

## 理论导读

现代 Vue3 项目通常基于 Vite。Vite 在开发期利用浏览器原生 ES Module，启动快；生产构建再交给 Rollup 做打包优化。项目结构不是固定规则，但要让入口、页面、组件、状态、API、工具函数、类型定义各有边界。

## 核心心智模型

项目结构像城市规划：入口文件是大门，路由是道路，页面是街区，组件是建筑单元，API 层是对外窗口，store 是公共仓库。规划混乱时，小项目也会很快变成难维护系统。

## 知识点详解

推荐结构：

```text
src/
  main.ts
  App.vue
  router/
  stores/
  views/
  components/
  composables/
  api/
  types/
  utils/
  assets/
```

常用命令：

```bash
npm create vue@latest
npm install
npm run dev
npm run build
npm run preview
```

`main.ts` 创建应用并挂载插件：

```ts
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import router from './router'
import App from './App.vue'

createApp(App).use(createPinia()).use(router).mount('#app')
```

## 例子

页面组件放 `views/UserList.vue`，通用按钮放 `components/BaseButton.vue`，用户接口放 `api/user.ts`，分页逻辑放 `composables/usePagination.ts`。

## 练习

创建一个 Vue3 项目，添加 `views`、`components`、`api`、`composables`、`stores` 目录，并实现首页和用户列表页。

## 验收

- `npm run dev` 能启动。
- `npm run build` 能通过。
- 页面级组件和通用组件没有混放。

## 重点

- Vite 负责开发服务器和构建。
- SFC 把模板、逻辑、样式组织在一个组件文件中，但不等于所有逻辑都要写进组件。

## 难点

- 路径别名、环境变量、构建产物路径、静态资源路径在开发和生产可能表现不同。

## 易错

> **易错：** 在代码里硬编码接口地址和环境配置。
>
> 正确做法：使用 `.env.development`、`.env.production` 和 `import.meta.env`。
