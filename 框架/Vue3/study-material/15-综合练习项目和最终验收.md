## 项目：Vue3 用户权限管理后台

目标是完成一个小型但完整的 Vue3 SPA，覆盖组件、组合式 API、Router、Pinia、请求、权限、测试、性能和部署。

## 功能范围

- 登录、退出、刷新后恢复登录态。
- 用户列表：搜索、分页、排序、加载、空状态、错误状态。
- 用户详情：动态路由、参数变化重新请求。
- 用户编辑：表单草稿、校验、保存、防重复提交、离开确认。
- 权限菜单：按角色显示菜单，未授权页面跳转 403。
- 全局布局：侧边栏、顶部用户信息、面包屑。
- 错误页：403、404、500。

## 技术要求

- Vue3 + Vite + TypeScript。
- Vue Router 管理页面。
- Pinia 管理登录态和权限。
- API 层统一封装请求、错误和 token。
- 至少 5 个可复用组件。
- 至少 3 个 composable。
- 单元测试覆盖核心 composable。
- 组件测试覆盖表单和弹窗。
- E2E 覆盖登录到用户列表的主流程。

## 实操任务

1. 搭建项目并配置路径别名。
2. 创建路由和布局。
3. 实现登录和 Pinia auth store。
4. 实现用户列表和分页 composable。
5. 实现用户编辑表单和校验。
6. 实现权限菜单和路由守卫。
7. 增加测试和错误页。
8. 构建并用本地静态服务器预览。

## 验收标准

- `npm run build` 通过。
- 刷新任意二级路由不 404。
- 未登录访问受保护页面会跳登录，登录后回原页面。
- 用户列表搜索不会重复触发失控请求。
- 编辑页未保存离开有确认。
- 退出登录后清理 token、用户信息和权限菜单。
- 测试覆盖核心逻辑。

## 重点

- 项目验收看完整链路，不只看页面能显示。
- 权限、请求、表单、路由、状态要形成闭环。

## 难点

- 刷新恢复、动态路由、权限菜单和后端权限边界经常混在一起，需要拆清职责。

## 易错

> **易错：** 只在前端隐藏按钮，以为接口就安全。
>
> 正确做法：前端做体验控制，后端做真正鉴权。
