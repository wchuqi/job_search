# Vue3学习资料：模板语法、指令和事件

[返回索引](../Vue3学习资料.md)

## 学习目标

- 掌握插值、属性绑定、条件渲染、列表渲染、事件绑定、表单绑定。
- 理解模板表达式的限制和安全边界。
- 能写出稳定、不抖动、不串状态的列表。

## 理论导读

Vue 模板是声明式 UI 描述。它最终会被编译成渲染函数，所以模板不是任意 JavaScript 执行环境。模板表达式应保持简单，复杂逻辑放到 `computed`、方法或 composable。

## 核心心智模型

模板像表格公式：单元格显示依赖数据，数据变了公式自动重新计算。不要在公式里塞复杂业务流程，否则难读、难测、难定位。

## 知识点详解

| 指令 | 用途 | 注意点 |
| --- | --- | --- |
| `v-bind` / `:` | 绑定属性 | class/style 支持对象和数组 |
| `v-on` / `@` | 绑定事件 | 修饰符如 `.stop`、`.prevent`、`.once` |
| `v-if` | 条件创建或销毁 DOM | 切换成本较高 |
| `v-show` | 用 CSS 控制显示 | 初次渲染仍会创建 DOM |
| `v-for` | 列表渲染 | 必须使用稳定唯一 `key` |
| `v-model` | 双向绑定表单或组件值 | 组件中实际是 prop + emit |

## 例子

```vue
<script setup lang="ts">
import { computed, ref } from 'vue'

const keyword = ref('')
const users = ref([
  { id: 1, name: 'Alice', active: true },
  { id: 2, name: 'Bob', active: false }
])

const visibleUsers = computed(() =>
  users.value.filter(user => user.name.toLowerCase().includes(keyword.value.toLowerCase()))
)
</script>

<template>
  <input v-model.trim="keyword" placeholder="Search user" />
  <ul>
    <li v-for="user in visibleUsers" :key="user.id" :class="{ muted: !user.active }">
      {{ user.name }}
    </li>
  </ul>
  <p v-if="visibleUsers.length === 0">No users</p>
</template>
```

## 练习

实现一个商品列表，支持关键字搜索、库存状态过滤、价格排序和空状态展示。

## 验收

- 列表 `key` 使用业务唯一 ID。
- 搜索过滤使用 `computed`。
- 空状态、加载状态、错误状态都能展示。

## 重点

- `v-if` 是创建销毁，`v-show` 是显示隐藏。
- `v-for` 和 `key` 决定列表 diff 的稳定性。

## 难点

- 事件修饰符改变事件传播和默认行为，写错会导致父组件收不到事件或表单意外提交。

## 易错

> **易错：** 使用数组下标作为 `key`。
>
> 正确做法：数据会排序、插入、删除时必须使用稳定业务 ID，否则子组件内部状态可能串位。
