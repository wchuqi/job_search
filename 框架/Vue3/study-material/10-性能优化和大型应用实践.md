# Vue3学习资料：性能优化和大型应用实践

[返回索引](../Vue3学习资料.md)

## 学习目标

- 掌握 Vue 应用常见性能瓶颈和优化手段。
- 能区分首屏性能、交互性能、渲染性能、网络性能。
- 能用数据定位问题，而不是凭感觉优化。

## 理论导读

Vue3 性能优化分两层：一层是框架内的响应式和渲染优化，另一层是应用架构、网络、资源、浏览器渲染和后端接口。真实项目里，最大瓶颈往往不在 Vue 本身，而在过大的首屏包、无缓存接口、大列表、重复请求和不合理状态设计。

## 核心心智模型

性能优化像问诊：先测量症状，再定位器官，最后开药。没有指标的优化容易引入复杂度却没有收益。

## 知识点详解

| 问题 | 常用手段 |
| --- | --- |
| 首屏包过大 | 路由懒加载、组件懒加载、依赖分析、CDN、预加载策略 |
| 大列表卡顿 | 虚拟列表、分页、减少响应式深度、稳定 key |
| 重复渲染 | 拆分组件、`computed` 缓存、避免无意义全局状态 |
| 频繁输入请求 | debounce、请求取消、结果缓存 |
| 静态内容反复计算 | `v-once`、`v-memo`、缓存组件 |
| 图片资源过大 | 响应式图片、懒加载、现代格式、尺寸约束 |

## 例子

```vue
<template>
  <ExpensiveRow
    v-for="item in items"
    :key="item.id"
    v-memo="[item.id, item.updatedAt]"
    :item="item"
  />
</template>
```

`v-memo` 适合明确知道依赖边界的局部跳过更新，不应作为默认优化手段。

## 练习

构造 5000 条数据列表，先直接渲染，再改成分页或虚拟列表，并记录 FPS、脚本执行时间和内存变化。

## 验收

- 能使用浏览器 Performance 面板定位瓶颈。
- 能用构建分析工具找到大依赖。
- 优化前后有对比指标。

## 重点

- 先测量，再优化。
- 优先减少不必要工作，再考虑缓存和跳过更新。

## 难点

- 过度使用缓存会带来状态一致性问题。

## 易错

> **易错：** 一遇到卡顿就加 `computed` 或 `memo`。
>
> 正确做法：先判断瓶颈是网络、脚本、布局、绘制、内存还是组件更新。
