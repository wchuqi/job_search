# Vue3学习资料：生产故障案例和排障剧本

[返回索引](../Vue3学习资料.md)

## 学习目标

- 掌握 Vue3 项目常见生产故障的定位路径。
- 建立白屏、路由、权限、接口、性能、水合问题的排障清单。
- 能把问题复盘成可预防的工程规范。

## 故障 1：上线后白屏

### 现象

首页打开空白，控制台报资源 404 或 JS 运行时错误。

### 排查路径

1. 看 Network：JS、CSS、图片是否 404。
2. 看 Console：是否有语法错误、运行时异常、环境变量 undefined。
3. 检查构建 `base` 是否匹配部署子路径。
4. 检查 CDN 缓存是否混用旧 HTML 和新 JS。
5. 检查浏览器兼容和 polyfill。

### 预防

- 构建产物使用 hash。
- HTML 缓存短，静态资源缓存长。
- 上线前执行预览和二级路由刷新测试。

## 故障 2：刷新二级路由 404

### 原因

History 模式下，浏览器请求 `/users/1` 会到服务器。如果服务器没有 fallback 到 `index.html`，就会返回 404。

### 修复

Nginx 示例：

```nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

> **易错：** 本地路由跳转正常，不代表刷新 URL 正常。

## 故障 3：权限守卫无限重定向

### 原因

守卫中未排除登录页、403、404，或者每次进入都重新添加动态路由。

### 修复策略

- 明确白名单。
- 动态路由只添加一次。
- token 失效时清理状态再跳登录。
- 记录 `redirect` 时避免把登录页作为 redirect。

## 故障 4：列表输入框排序后串值

### 原因

列表使用 index 作为 key，排序后组件实例按位置复用，内部输入状态错配。

### 修复

使用业务唯一 ID 作为 key，并检查子组件本地状态是否应该跟随 props 重置。

## 故障 5：页面越用越卡

### 排查路径

- Performance 看长任务。
- Memory 看 detached DOM 和持续增长对象。
- 检查定时器、全局事件、WebSocket、第三方图表实例是否卸载清理。
- 检查 KeepAlive 是否缓存过多页面。
- 检查 watch 是否重复注册。

## 故障 6：SSR 水合不一致

### 常见原因

- 服务端和客户端使用不同时间、随机数。
- 服务端渲染时没有用户本地状态，客户端首次渲染有。
- 首次渲染访问浏览器 API。
- 条件渲染依赖客户端宽度。

### 修复

- 把客户端专属逻辑放到 mounted 后。
- 服务端和客户端使用同一份初始数据。
- 对不可避免差异做客户端延迟渲染。

## 最终排障清单

- 是否能稳定复现。
- 是否有错误日志和版本号。
- 是否是资源、代码、接口、路由、权限、浏览器、缓存其中一类。
- 是否能用最小回滚或开关止血。
- 是否补充了监控、测试和上线检查。

## 重点

- 生产排障先止血，再定位，再复盘。

## 难点

- 前端故障常跨构建、CDN、浏览器、接口和权限，不能只盯组件代码。

## 易错

> **易错：** 只在本地 dev server 验证。
>
> 正确做法：必须验证生产构建产物和实际部署路径。
