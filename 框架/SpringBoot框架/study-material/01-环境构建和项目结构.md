# Spring Boot 学习资料：环境构建和项目结构

[返回索引](../SpringBoot学习资料.md)

## 学习目标

- 能选择 JDK、构建工具、starter 和项目结构。
- 理解父 POM、BOM、插件、可执行 Jar 的作用。
- 能建立适合团队维护的分层结构。

## 理论导读

Spring Boot 项目不是只由 Java 代码构成。构建文件决定依赖版本和打包方式，目录结构决定组件扫描范围，配置文件决定运行行为，测试目录决定质量保障方式。

Boot 官方建议通过 Spring Initializr 启动项目，因为它能生成与当前 Boot 版本匹配的构建文件、插件和基础目录。手工搭建也可以，但要明确 BOM、插件和 starter 的关系。

## 核心心智模型

项目结构像一栋楼：

- `pom.xml` 或 `build.gradle` 是施工图，决定材料版本和打包方式。
- `src/main/java` 是业务代码。
- `src/main/resources` 是运行配置和静态资源。
- `src/test/java` 是质量验证。
- 主类所在包是组件扫描的根。

## 知识点详解

### Maven 依赖管理

典型 Maven 项目使用 `spring-boot-starter-parent`：

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>4.0.6</version>
    <relativePath/>
</parent>
```

它提供依赖版本管理、插件配置、资源过滤和默认编译设置。公司多模块项目也常用 `spring-boot-dependencies` BOM：

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>4.0.6</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

### Starter

Starter 是依赖聚合，不是运行时代码的主体。比如 Web MVC starter 会带来 Spring MVC、JSON、验证相关支持和内嵌服务器相关依赖。你引入 starter 后，自动配置才有足够类路径条件创建默认组件。

### 推荐项目结构

```text
com.example.order
  OrderApplication.java
  api/
    OrderController.java
    OrderRequest.java
    OrderResponse.java
  application/
    OrderService.java
  domain/
    Order.java
    OrderStatus.java
  infrastructure/
    OrderRepository.java
    JpaOrderRepository.java
  config/
    SecurityConfig.java
```

> **重点：** 主启动类放在根包，避免组件扫描漏掉兄弟包。

### 可执行 Jar

Boot Maven/Gradle 插件会把应用代码、依赖和启动器打包成可用 `java -jar` 运行的 Jar。Boot 的嵌套 Jar 机制允许依赖 Jar 保持 Jar 形态，而不是完全解压混合。

## 例子

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-webmvc</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

## 练习

1. 创建一个 `order-service` 项目。
2. 主类放在 `com.example.order`。
3. Controller、Service、Repository 分包。
4. 使用 Maven 打包并运行生成的 Jar。

## 验收

- `mvn test` 通过。
- `mvn package` 生成可执行 Jar。
- `java -jar target/*.jar` 能启动。
- 能说明 starter、BOM、plugin 各自解决什么问题。

## 重点

- 构建文件是 Boot 项目稳定性的源头。
- 不要随意手工覆盖 Boot 管理的依赖版本。
- 主类位置影响组件扫描边界。

## 难点

- 多模块项目中，父 POM 管理版本和聚合模块是两个概念。
- 第三方 starter 可能不支持最新 Boot 主版本，升级前必须检查。

## 易错

> **易错：** 每个依赖都手写版本号。
>
> 正确做法：优先使用 Boot BOM 管理版本，只有有明确兼容性理由时才覆盖。
