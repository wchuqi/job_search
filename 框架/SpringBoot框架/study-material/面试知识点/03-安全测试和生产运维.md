## 一、安全

### 1. 引入 Spring Security 后默认会发生什么？

**参考答案：**

Boot 会启用默认安全配置，默认保护接口，并生成一个默认用户密码。真实项目通常需要自定义 `SecurityFilterChain`，明确公开路径、认证方式、授权规则、异常处理和 Actuator 端点保护。

> **重点：** 默认安全只能用于开发认知，生产必须显式配置。

### 2. 认证和授权有什么区别？

**参考答案：**

认证解决“你是谁”，授权解决“你能访问什么”。认证失败通常返回 401；认证成功但权限不足通常返回 403。

在 Spring Security 中，请求先经过 Filter Chain，认证结果放入 SecurityContext，再由授权规则决定是否放行。

> **易错：** 把 401 和 403 混用。

### 3. CSRF 和 CORS 有什么区别？

**参考答案：**

CORS 是浏览器跨域访问控制机制，决定浏览器是否允许前端读取跨域响应。CSRF 是防止恶意站点借用户登录态发起伪造请求。无状态 JWT API 常禁用 CSRF，但基于 Cookie Session 的应用通常需要 CSRF 防护。

> **重点：** CORS 不是认证授权，CSRF 也不是跨域开关。

## 二、测试

### 4. `@WebMvcTest` 和 `@SpringBootTest` 有什么区别？

**参考答案：**

`@WebMvcTest` 只加载 Web 层相关组件，适合测试 Controller、参数绑定、校验、异常处理，速度较快。`@SpringBootTest` 加载完整应用上下文，适合跨层集成测试，但更慢。

测试应按目标选择最小上下文，不要所有测试都用完整上下文。

> **重点：** 切片测试提升速度和定位能力。

### 5. 为什么使用 Testcontainers？

**参考答案：**

Testcontainers 用容器启动真实数据库、消息队列等依赖，避免测试依赖本地安装或共享环境。它能让集成测试更接近生产，并在 CI 上可重复执行。

> **难点：** Testcontainers 更真实，但启动成本更高，应放在关键集成测试中。

## 三、生产运维

### 6. Actuator 常用来解决什么问题？

**参考答案：**

Actuator 提供健康检查、指标、Prometheus 抓取、日志级别、线程转储、堆转储、自动配置报告、配置属性等运行时信息。它帮助开发和运维判断应用是否健康、性能是否异常、配置是否正确。

敏感端点必须限制访问。

> **重点：** Actuator 是生产诊断入口，不是随便公开的调试接口。

### 7. readiness 和 liveness 有什么区别？

**参考答案：**

readiness 表示实例是否可以接收流量；liveness 表示实例是否需要重启。数据库短暂不可用可能影响 readiness，但不一定应该影响 liveness，否则会导致实例反复重启。

> **易错：** 把所有依赖检查都放进 liveness。

### 8. 如何设计 Spring Boot 应用的可观测性？

**参考答案：**

至少包括结构化日志、metrics、tracing、健康检查和业务指标。日志用于解释发生了什么，metrics 用于发现异常趋势，trace 用于定位跨服务慢调用，health 用于流量调度，业务指标用于判断业务是否异常。

> **重点：** 日志、指标、trace 要能通过 traceId 或标签关联。
