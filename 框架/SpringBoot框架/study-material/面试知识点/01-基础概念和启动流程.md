## 一、基础概念和启动流程

### 1. Spring Boot 和 Spring Framework 有什么区别？

**参考答案：**

Spring Framework 提供 IoC、AOP、事务、MVC 等基础能力；Spring Boot 基于 Framework，提供自动配置、Starter、外部化配置、嵌入式服务器、Actuator、打包部署和测试整合。Boot 不是替代 Framework，而是把 Framework 应用装配和生产运行标准化。

> **重点：** Framework 是基础能力，Boot 是应用装配和生产化。

### 2. `@SpringBootApplication` 包含哪些能力？

**参考答案：**

它组合了 `@SpringBootConfiguration`、`@EnableAutoConfiguration`、`@ComponentScan`。因此它既标记配置类，又启用自动配置，还从当前包向下扫描组件。

> **易错：** 主启动类包位置不对会导致组件扫描不完整。

### 3. `SpringApplication.run()` 大致做了什么？

**参考答案：**

它会推断应用类型，准备监听器和初始化器，准备 Environment，创建 ApplicationContext，加载 BeanDefinition，刷新容器，启动嵌入式 Web 服务器，执行 Runner，发布 started 和 ready 事件。

> **难点：** Boot 的 run 包住了 Spring Framework 的 refresh，并在前后扩展了环境、Web、事件和 Runner。

### 4. ApplicationRunner 和 CommandLineRunner 有什么区别？

**参考答案：**

两者都在应用启动完成后执行。`ApplicationRunner` 接收结构化的 `ApplicationArguments`，更适合解析选项；`CommandLineRunner` 接收原始字符串数组。

> **易错：** Runner 不适合执行长时间阻塞任务，否则应用迟迟不能 ready。

### 5. Boot 如何判断是 Web MVC、WebFlux 还是非 Web 应用？

**参考答案：**

Boot 根据 classpath 推断 `WebApplicationType`。存在 Servlet Web 相关类时通常是 SERVLET；存在响应式相关类且没有 Servlet 条件时可能是 REACTIVE；都没有则是 NONE。

> **重点：** 同时引入 MVC 和 WebFlux 时要明确期望的应用类型。
