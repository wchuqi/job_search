# Spring Boot 4 学习资料：自动配置源码、导入选择和条件评估深度解析

[返回索引](../SpringBoot4学习资料.md)

## 学习目标

- 能从源码主线解释自动配置如何导入。
- 掌握候选自动配置、过滤、排序、条件评估、报告生成。
- 能独立排查“Starter 引了但 Bean 没出现”的问题。

## 自动配置深度主线

```text
@SpringBootApplication
  -> @EnableAutoConfiguration
  -> AutoConfigurationImportSelector
      -> 读取 AutoConfiguration.imports
      -> 去重
      -> 排除 excludes
      -> 应用 AutoConfigurationImportFilter
      -> 排序
      -> 返回配置类名
  -> Spring Framework 注册配置类 BeanDefinition
  -> 条件注解评估
  -> 满足条件的 @Bean 方法注册 BeanDefinition
```

> **重点：** 自动配置导入和条件评估不是同一件事。类进入候选列表，不代表内部 Bean 一定创建。

## 候选列表

Boot 4 使用：

```text
META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

文件内容是自动配置类全限定名，一行一个。

## 排除自动配置

```java
@SpringBootApplication(exclude = DataSourceAutoConfiguration.class)
class App {
}
```

或配置：

```yaml
spring:
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
```

排除适合明确不用某个能力，不适合作为长期掩盖依赖冲突的手段。

## 条件评估分层

条件可以放在类上，也可以放在 `@Bean` 方法上：

```java
@AutoConfiguration
@ConditionalOnClass(DataSource.class)
class MyDataAutoConfiguration {
    @Bean
    @ConditionalOnMissingBean
    MyRepository myRepository(DataSource dataSource) {
        return new MyRepository(dataSource);
    }
}
```

类级条件不满足，整个自动配置跳过；方法级条件不满足，只跳过该 Bean。

## OnBeanCondition 的难点

`@ConditionalOnMissingBean` 判断的是当前 BeanFactory 中是否已经有匹配 Bean。顺序会影响结果，所以自动配置通常要排序，并且用户配置通常先于自动配置生效。

> **难点：** 如果两个自动配置都依赖“缺少某 Bean”，排序错误会导致默认 Bean 抢先注册。

## ConditionEvaluationReport

条件报告回答：

- 哪些自动配置 matched。
- 哪些 did not match。
- 每个条件的原因。
- 哪些被排除。

查看方式：

```bash
java -jar app.jar --debug
```

或 Actuator：

```text
/actuator/conditions
```

## 自定义 Starter 最小闭环

```java
@AutoConfiguration
@ConditionalOnProperty(prefix = "demo.audit", name = "enabled", havingValue = "true")
@EnableConfigurationProperties(AuditProperties.class)
class AuditAutoConfiguration {
    @Bean
    @ConditionalOnMissingBean
    AuditClient auditClient(AuditProperties properties) {
        return new AuditClient(properties.endpoint());
    }
}
```

`AutoConfiguration.imports`：

```text
com.example.audit.AuditAutoConfiguration
```

## 排障剧本

1. 确认 starter 是否真的在 dependency tree。
2. 确认自动配置类是否在 imports 文件。
3. 确认自动配置是否被 exclude。
4. 看 conditions 报告。
5. 检查 `@ConditionalOnClass` 所需类是否存在。
6. 检查属性条件是否拼写正确。
7. 检查用户是否已定义同类型 Bean。
8. 检查自动配置排序。

## 练习

1. 写一个自动配置类，分别加 class、property、missing bean 条件。
2. 做 4 组实验：缺类、属性关闭、用户自定义 Bean、显式 exclude。
3. 为每组保存 conditions 报告。

## 验收

- 能说明自动配置导入和 Bean 创建之间的关系。
- 能解释条件报告中的 match/not match。
- 能设计可被用户覆盖的企业 Starter。
