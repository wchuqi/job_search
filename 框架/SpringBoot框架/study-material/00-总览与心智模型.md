# Spring Boot 学习资料：总览与心智模型

[返回索引](../SpringBoot学习资料.md)

## 学习目标

- 明确 Spring Boot 与 Spring Framework、Spring MVC、Spring Security、Spring Data 的关系。
- 建立“依赖决定能力、配置决定细节、条件决定装配、Actuator 暴露状态”的心智模型。
- 知道学习 Spring Boot 时必须同时关注开发体验和生产运行。

## 理论导读

Spring Boot 的核心价值是把一个 Spring 应用需要的通用装配工作标准化：依赖版本由 BOM 管理，常见能力由 starter 聚合，组件由自动配置按条件创建，配置由外部化配置体系注入，生产能力由 Actuator 提供。

它不是绕开 Spring，而是在 Spring 容器之上提供一套“默认能跑、需要时可覆盖”的工程体系。默认配置的意义是让大多数项目先站起来；覆盖点的意义是当业务需要不同数据库、序列化策略、安全规则、线程池、缓存、消息系统时，不必推翻整个框架。

## 核心心智模型

把 Spring Boot 想成一个应用装配工厂：

- `starter` 是材料包：引入某类能力需要的一组依赖。
- `auto-configuration` 是装配说明：根据类路径、配置属性、已有 Bean 和应用类型决定是否创建组件。
- `Environment` 是参数表：端口、数据库、日志、Profile、密钥等都从不同来源合并而来。
- `ApplicationContext` 是成品仓库：最终可注入的 Bean 都在容器中。
- `Actuator` 是仪表盘：暴露健康、指标、配置、映射、线程等运行信息。

## 知识点详解

### Spring Boot 和 Spring Framework

Spring Framework 提供 IoC、AOP、事务、Web、资源抽象等基础能力；Spring Boot 提供启动器、自动配置、配置绑定、生产就绪特性、构建插件和运行时约定。

> **重点：** 面试中说“Spring Boot 简化配置”不够，要说明它通过 starter、BOM、自动配置、外部化配置和 Actuator 简化工程落地。

### 约定优于配置

Boot 给出默认目录结构、默认 Bean、默认端口、默认日志、默认 JSON、默认错误处理。只要业务没有特殊要求，就使用默认值；业务需要差异化时，通过声明自己的 Bean 或属性覆盖。

> **易错：** 以为自动配置无法控制。正确做法：优先通过属性调整，其次声明自定义 Bean，最后才排除某个自动配置类。

### 生产就绪

生产就绪包括健康检查、指标、日志、链路追踪、优雅停机、配置安全、镜像分层、故障诊断。Boot 把这些非功能需求作为一等能力，而不是项目上线前临时补。

## 例子

```java
@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

@RestController
class HelloController {
    @GetMapping("/hello")
    String hello() {
        return "hello Spring Boot";
    }
}
```

只要项目引入 Web MVC starter，Boot 会根据类路径自动创建内嵌 Web 服务器、Spring MVC 基础组件、JSON 转换器和错误处理基础设施。

## 练习

1. 用 Spring Initializr 创建一个 Web MVC 项目。
2. 添加 `/hello` 接口。
3. 修改 `server.port=8081`。
4. 启动后访问接口，并观察启动日志中 Tomcat、DispatcherServlet、Profile 的信息。

## 验收

- 能解释 `@SpringBootApplication` 包含哪些常见语义。
- 能说明 starter 和 auto-configuration 的区别。
- 能说出 Actuator 至少 5 类生产能力。

## 重点

- Spring Boot 的核心是“可覆盖的默认装配”。
- 类路径、配置属性、已有 Bean、Profile 决定自动配置结果。
- 生产能力必须在开发期就纳入设计。

## 难点

- 自动配置不是魔法，它有明确条件和顺序。
- 同一个功能可能同时受依赖、属性、Bean、Profile 和测试切片影响。

## 易错

> **易错：** 只会写接口，不会看自动配置报告。
>
> 正确做法：启动加 `--debug`，或使用 Actuator `conditions`、`beans`、`configprops`、`env` 定位装配和配置问题。
