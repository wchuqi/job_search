# Spring Boot 4 学习资料：数据访问、事务和数据库迁移

[返回索引](../SpringBoot4学习资料.md)

## 学习目标

- 掌握 Boot 对 DataSource、JDBC、JPA、事务的自动配置。
- 理解连接池、事务管理器、迁移工具的关系。
- 会用 Flyway 或 Liquibase 管理数据库结构。

## 理论导读

Boot 的数据访问自动配置会根据 classpath 和配置创建 DataSource、JdbcTemplate、EntityManagerFactory、TransactionManager 等 Bean。它降低了接入成本，但真正的生产稳定性仍取决于连接池参数、事务边界、SQL 质量、迁移流程。

## JDBC 示例

```java
@Repository
class OrderRepository {
    private final JdbcTemplate jdbcTemplate;

    OrderRepository(JdbcTemplate jdbcTemplate) {
        this.jdbcTemplate = jdbcTemplate;
    }

    long insert(String orderNo, BigDecimal amount) {
        KeyHolder keyHolder = new GeneratedKeyHolder();
        jdbcTemplate.update(connection -> {
            PreparedStatement ps = connection.prepareStatement(
                "insert into orders(order_no, amount) values (?, ?)",
                Statement.RETURN_GENERATED_KEYS
            );
            ps.setString(1, orderNo);
            ps.setBigDecimal(2, amount);
            return ps;
        }, keyHolder);
        return keyHolder.getKey().longValue();
    }
}
```

## 事务边界

```java
@Service
class OrderService {
    private final OrderRepository orderRepository;

    OrderService(OrderRepository orderRepository) {
        this.orderRepository = orderRepository;
    }

    @Transactional
    public OrderResponse create(CreateOrderCommand command) {
        long id = orderRepository.insert(command.orderNo(), command.amount());
        return new OrderResponse(id);
    }
}
```

事务应放在业务用例边界，而不是每个 DAO 方法上。

## Flyway 迁移

```text
src/main/resources/db/migration/
  V1__create_orders.sql
  V2__add_order_status.sql
```

`V1__create_orders.sql`：

```sql
create table orders (
    id bigint generated always as identity primary key,
    order_no varchar(64) not null unique,
    amount decimal(18,2) not null,
    created_at timestamp not null
);
```

## 连接池配置

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/orderdb
    username: order
    password: ${DB_PASSWORD}
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 2s
```

> **难点：** 连接池大小不是越大越好。它受数据库最大连接数、应用实例数、SQL 耗时、并发模型共同约束。

## 易错

> **易错：** 生产环境使用 `spring.jpa.hibernate.ddl-auto=update` 自动改表。
>
> 正确做法：生产结构变更使用 Flyway/Liquibase，并经过评审和回滚预案。

> **易错：** 只配置 DataSource，却不知道事务管理器是否匹配。
>
> 正确做法：确认 Repository 使用的连接来自同一个受事务管理的数据源。

## 练习

1. 用 Flyway 创建订单表。
2. 用 JdbcTemplate 写订单创建和查询。
3. 写事务测试，验证抛异常后回滚。
4. 调整 Hikari 最大连接数，做简单压测观察等待。

- 能使用数据库迁移工具管理结构。
