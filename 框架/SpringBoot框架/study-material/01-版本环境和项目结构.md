# Spring Boot 4 学习资料：版本、环境和项目结构

[返回索引](../SpringBoot4学习资料.md)

## 学习目标

- 明确 Spring Boot 4.x 的版本基准和生产选择。
- 掌握 Maven/Gradle 项目结构和依赖管理入口。
- 理解 Boot 4 与 Java、Spring Framework、Jakarta 的兼容边界。

## 版本基准

截至 2026-06-06：

- 稳定学习基准：Spring Boot `4.0.6`。
- Preview 版本：Maven Central 显示 `4.1.0-RC1`，适合了解新功能，不建议作为生产默认基准。

> **重点：** 学习和生产默认选稳定 GA 版本；RC 版本用于验证和预研。

## 最小 Maven 项目

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>4.0.6</version>
    </parent>

    <groupId>com.example</groupId>
    <artifactId>boot-order-service</artifactId>
    <version>1.0.0</version>

    <properties>
        <java.version>17</java.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-webmvc</artifactId>
        </dependency>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-actuator</artifactId>
        </dependency>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>
```

## Starter 选择

| 需求 | Starter |
| --- | --- |
| Spring MVC REST | `spring-boot-starter-webmvc` |
| WebFlux | `spring-boot-starter-webflux` |
| Actuator | `spring-boot-starter-actuator` |
| JDBC | `spring-boot-starter-jdbc` |
| JPA | `spring-boot-starter-data-jpa` |
| Validation | `spring-boot-starter-validation` |
| Security | `spring-boot-starter-security` |
| Test | `spring-boot-starter-test` |

## Maven 命令

```bash
mvn spring-boot:run
mvn test
mvn package
java -jar target/boot-order-service-1.0.0.jar
mvn dependency:tree
```

## 易错

> **易错：** 手工指定一堆 Spring 依赖版本，绕过 Boot BOM。
>
> 正确做法：优先让 Boot 管理版本，确需覆盖时记录原因和兼容验证。

> **易错：** 主启动类放在业务包旁边或更深层，导致组件扫描范围不完整。
>
> 正确做法：主启动类放在应用根包。

## 练习

1. 创建一个 Boot 4.0.6 Web MVC 项目。
2. 运行 `mvn dependency:tree`，观察 Starter 引入的依赖。
3. 删除 `starter-parent` 后改用 BOM 导入，比较差异。
