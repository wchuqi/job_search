# Spring Boot 4 学习资料：自动配置原理和条件装配

[返回索引](../SpringBoot4学习资料.md)

## 学习目标

- 理解自动配置类如何被发现和导入。
- 掌握 `@ConditionalOnClass`、`@ConditionalOnMissingBean`、`@ConditionalOnProperty` 等条件。
- 能用 condition evaluation report 排查自动配置。

## 理论导读

自动配置是 Spring Boot 的核心。它不是“扫描所有 jar 里的配置类”，而是从约定位置读取自动配置类列表，再结合条件判断是否生效。

自动配置的设计原则是：当应用具备某种能力的依赖和配置，并且用户没有提供自己的 Bean 时，Boot 提供一套合理默认值。

## 自动配置心智模型

```text
Starter 引入依赖
  -> classpath 出现某些类
  -> Boot 导入候选自动配置类
  -> 条件注解逐个评估
  -> 满足条件的配置类注册 BeanDefinition
  -> 用户自定义 Bean 通常优先于默认 Bean
```

## 常见条件

| 条件 | 含义 |
| --- | --- |
| `@ConditionalOnClass` | classpath 有某个类 |
| `@ConditionalOnMissingClass` | classpath 没有某个类 |
| `@ConditionalOnBean` | 容器已有某个 Bean |
| `@ConditionalOnMissingBean` | 容器没有某个 Bean |
| `@ConditionalOnProperty` | 配置属性满足条件 |
| `@ConditionalOnResource` | 资源存在 |
| `@ConditionalOnWebApplication` | 当前是 Web 应用 |
| `@ConditionalOnSingleCandidate` | 容器有唯一候选 Bean |

## 示例：自定义自动配置

```java
@AutoConfiguration
@ConditionalOnClass(AuditClient.class)
@EnableConfigurationProperties(AuditProperties.class)
public class AuditAutoConfiguration {
    @Bean
    @ConditionalOnMissingBean
    AuditClient auditClient(AuditProperties properties) {
        return new HttpAuditClient(properties.endpoint());
    }
}
```

属性类：

```java
@ConfigurationProperties(prefix = "app.audit")
public record AuditProperties(
    boolean enabled,
    String endpoint
) {
}
```

自动配置导入文件：

```text
META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

```text
com.example.audit.AuditAutoConfiguration
```

## 排查自动配置

常用方式：

```bash
java -jar app.jar --debug
```

或查看 Actuator 条件报告端点。

需要回答三个问题：

1. 自动配置类有没有进入候选列表。
2. 条件为什么 match 或 not match。
3. 用户 Bean 是否覆盖了默认 Bean。

## 易错

> **易错：** 看到自动配置类存在，就以为一定会生效。
>
> 正确做法：看条件评估报告，确认每个条件是否满足。

> **易错：** 自定义 Starter 中把业务 Bean 强行注册，导致用户无法覆盖。
>
> 正确做法：默认 Bean 通常加 `@ConditionalOnMissingBean`。

## 练习

1. 写一个 `audit-spring-boot-starter`。
2. 通过 `app.audit.enabled=true` 控制 Bean 是否创建。
3. 用户应用自定义 `AuditClient`，验证自动配置让路。
4. 打开 debug 报告，记录 match/not match 原因。

## 验收

- 能解释自动配置导入路径。
- 能说明条件装配的决策规则。
- 能排查某个自动配置为什么没有生效。
