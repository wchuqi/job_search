# Spring Boot 4 学习资料：生产故障案例和排障剧本深度版

[返回索引](../SpringBoot4学习资料.md)

## 学习目标

- 掌握 Boot 常见故障的链路化排查方法。
- 能用条件报告、配置来源、Actuator、依赖树、日志定位问题。
- 为每类故障写回归测试和预防措施。

## 总排障路径

```text
启动失败 / 运行异常
  -> 判断阶段：配置加载、自动配置、Bean 创建、Web 容器、业务运行
  -> 找 cause 链根因
  -> 看 conditions / env / dependency tree / actuator
  -> 最小复现
  -> 修复并加测试
```

## 案例 1：Starter 引了但功能没生效

排查：

1. `mvn dependency:tree` 确认依赖存在。
2. 查看 `AutoConfiguration.imports`。
3. 打开 `--debug` 看条件报告。
4. 检查属性开关。
5. 检查是否已有用户 Bean 导致默认 Bean 让路。

## 案例 2：配置在生产不生效

排查：

1. 是否 profile 未激活。
2. 环境变量名称是否正确。
3. 是否命令行参数覆盖。
4. `spring.config.import` 是否加载。
5. Actuator env 中最终值来自哪个 PropertySource。

## 案例 3：端口占用或路径不通

排查：

- `server.port` 最终值。
- 容器是否真的启动。
- `server.servlet.context-path`。
- 网关 rewrite。
- Controller 映射。

## 案例 4：健康检查失败导致 Pod 反复重启

排查：

1. liveness 是否依赖 DB 或外部服务。
2. health indicator 是否无超时。
3. readiness 和 liveness 是否混用。
4. K8s 探针 initialDelay 是否过短。

修复：

- liveness 只检查进程活性。
- readiness 检查关键依赖。
- 下游健康检查加超时和缓存。

## 案例 5：启动变慢

排查：

- Bean 数量。
- 组件扫描范围。
- 初始化阶段访问外部资源。
- 数据库迁移耗时。
- JPA 初始化。
- 日志过多。
- AOT 是否可用。

## 案例 6：测试很慢

排查：

- 是否所有测试都用 `@SpringBootTest`。
- 是否频繁 `@DirtiesContext`。
- 是否每个测试动态属性都不同。
- Testcontainers 是否重复启动。

## 练习

1. 制造自动配置不生效，写条件报告分析。
2. 制造 profile 配置覆盖，写来源分析。
3. 制造健康检查误杀，写修复方案。
4. 制造测试上下文缓存失效，比较耗时。

## 验收

- 能判断故障发生在 Boot 启动哪个阶段。
- 能用至少 4 种工具定位问题：conditions、env、dependency tree、Actuator。
- 能写出复现、根因、修复、预防。
