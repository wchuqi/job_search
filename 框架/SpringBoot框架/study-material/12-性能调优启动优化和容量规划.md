# Spring Boot 4 学习资料：性能调优、启动优化和容量规划

[返回索引](../SpringBoot4学习资料.md)

## 学习目标

- 掌握 Boot 应用常见性能瓶颈。
- 会从指标、线程、连接池、GC、SQL、启动耗时定位问题。
- 能做基本容量估算。

## 性能观察维度

| 维度 | 指标 |
| --- | --- |
| 请求 | QPS、P95/P99、错误率 |
| JVM | heap、GC、线程数、类加载 |
| Web 容器 | 工作线程、队列、连接 |
| 数据库 | 连接池 active/pending、慢 SQL、锁 |
| 外部调用 | 超时、重试、熔断 |
| 启动 | Bean 数、自动配置、扫描范围 |

## 启动优化

- 减少不必要 Starter。
- 控制组件扫描范围。
- 避免 Bean 初始化中访问慢资源。
- 使用 lazy initialization 时谨慎评估首个请求延迟。
- 使用 AOT 预处理部分运行时工作。

## 连接池估算

假设单实例目标 200 QPS，数据库平均响应 50ms：

```text
并发连接需求 ≈ QPS * 平均 DB 耗时
             ≈ 200 * 0.05
             ≈ 10
```

再考虑抖动、慢查询、实例数和数据库最大连接数设置池大小。

> **易错：** 每个应用实例都把连接池开到 100，10 个实例就是 1000 个数据库连接，可能直接压垮数据库。

## 超时和重试

生产中所有远程调用都应有：

- 连接超时。
- 读取超时。
- 最大重试次数。
- 幂等保护。
- 失败降级或隔离策略。

## 练习

1. 使用 Actuator metrics 查看 HTTP 延迟。
2. 制造慢 SQL，观察连接池指标。
3. 增加一个启动慢 Bean，记录启动耗时。
4. 调整 Web 容器线程和 Hikari 连接池，比较压测结果。

## 验收

- 能从指标判断瓶颈层。
- 能估算连接池合理范围。
- 能列出启动慢的常见原因。
