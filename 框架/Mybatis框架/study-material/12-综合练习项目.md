# MyBatis 学习资料：综合练习项目

[返回索引](../Mybatis学习资料.md)

## 项目目标

构建一个订单管理模块，覆盖 MyBatis 常见生产能力。

## 表结构

- users。
- orders。
- order_items。
- products。

## 功能要求

1. 用户查询：简单 resultType。
2. 订单详情：ResultMap 映射用户和明细。
3. 订单列表：动态 SQL 条件查询。
4. 批量插入订单明细：BatchExecutor 或 foreach。
5. 订单分页：限制最大页大小。
6. 安全排序：白名单字段。
7. 慢 SQL 日志插件：记录 statementId、耗时、traceId。
8. 枚举 TypeHandler：订单状态 code 映射。
9. Spring 事务：创建订单和明细同事务提交。
10. 集成测试：使用真实数据库或 Testcontainers。

## 必做故障实验

1. 写错 namespace，触发 `Invalid bound statement`。
2. 去掉 `@Param`，触发参数找不到。
3. 使用 `${keyword}`，说明注入风险。
4. 嵌套查询制造 N+1。
5. 大 offset 分页观察性能下降。
6. 开启二级缓存后模拟脏数据风险。

## 验收标准

- 所有 Mapper 有集成测试。
- 复杂 ResultMap 能正确映射一对多。
- SQL 日志能定位 statementId。
- 动态 SQL 不产生非法 where。
- 用户输入不直接进入 `${}`。
- 订单创建异常时事务回滚。
- 能写出项目排障报告。

## 重点

- 项目必须覆盖正常流程和失败流程。
- MyBatis 学习不能只写 CRUD，要验证映射、性能、安全和事务。
