# MyBatis 学习资料：版本基准和环境集成

[返回索引](../Mybatis学习资料.md)

## 学习目标

- 明确 MyBatis Core、MyBatis Spring、MyBatis Spring Boot Starter 的版本关系。
- 能在 Spring Boot 4 项目中正确引入 MyBatis。
- 能区分核心框架版本、starter 版本和第三方增强工具版本。
- 能排查版本冲突、Mapper 扫描失败和 XML 找不到问题。

## 版本基准

截至 2026-06-06 核对：

| 组件 | 版本基准 | 说明 |
| --- | --- | --- |
| MyBatis Core | `3.5.19` | 官方 MyBatis 3 文档当前版本 |
| MyBatis Spring Boot Starter | `4.0.x` | 官方文档显示 `4.0.0` 支持 Boot 4.0+ 和 Java 17+，Maven Central 已有 `4.0.1` |
| Spring Boot | `4.0+` | 使用 starter 4.x 时 |
| Java | `17+` | starter 4.x 要求 |
| MyBatis-Plus | 非 MyBatis 官方核心 | 可作为增强工具单独学习 |

> **重点：** MyBatis Core 和 Spring Boot starter 不是同一个版本号体系。不要看到 core 是 3.5.x，就以为 starter 也应该是 3.5.x。

## Maven 示例

Spring Boot 4 项目：

```xml
<dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>4.0.1</version>
</dependency>
```

如果团队严格按官方 starter 文档版本，也可以使用 `4.0.0`。生产项目应统一由依赖管理控制版本，不要各模块手写不同版本。

## Spring Boot 配置

```yaml
mybatis:
  mapper-locations: classpath*:mapper/**/*.xml
  type-aliases-package: com.example.demo.domain
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

生产环境不建议长期使用 `StdOutImpl` 输出 SQL，应接入规范日志框架并控制级别。

## Mapper 扫描

方式一：启动类上使用 `@MapperScan`：

```java
@SpringBootApplication
@MapperScan("com.example.demo.mapper")
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

方式二：每个接口加 `@Mapper`。大型项目更推荐集中 `@MapperScan`，避免漏标注。

## XML 路径

常见结构：

```text
src/main/java/com/example/demo/mapper/UserMapper.java
src/main/resources/mapper/UserMapper.xml
```

如果 XML 放在 `src/main/java` 下，Maven 默认可能不打包资源，导致运行时找不到 statement。推荐 XML 放 `resources`。

## 常见启动问题

| 现象 | 常见原因 |
| --- | --- |
| `Invalid bound statement` | namespace/id 不匹配，XML 没加载，mapper-locations 错 |
| Mapper Bean 找不到 | 没有 `@MapperScan` 或扫描包不对 |
| 参数名找不到 | 多参数未使用 `@Param`，或编译参数名不可用 |
| 字段映射为空 | 列名和属性名不匹配，驼峰映射未开 |
| `NoSuchMethodError` | 依赖版本冲突 |

## 练习

1. 创建 Spring Boot 4 项目，引入 starter 4.x。
2. 配置 `mapper-locations`。
3. 写 `UserMapper` 和 XML。
4. 故意写错 namespace，观察 `Invalid bound statement`。
5. 修复后运行集成测试。

## 验收

- 能解释 core 与 starter 版本区别。
- 能正确配置 Mapper 扫描和 XML 路径。
- 能定位 XML 未加载和 statement 绑定失败。
- 能说明 starter 4.x 与 Boot 4/Java 17 的关系。

## 重点

- Spring Boot 集成的关键是 Mapper 扫描、XML 加载、SqlSessionFactory 自动配置。
- XML 路径和 namespace/id 是最常见错误源。

## 难点

- 多模块项目中 XML 打包路径容易出错。
- 版本冲突常在运行时表现为方法找不到或自动配置失败。

## 易错

> **易错：** Mapper 接口路径和 XML namespace 不一致。
>
> 正确做法：namespace 写 Mapper 接口全限定名，SQL id 写接口方法名。
