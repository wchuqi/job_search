# MyBatis 学习资料：迁移、版本兼容和生产规范

[返回索引](../Mybatis学习资料.md)

## 学习目标

- 能制定 MyBatis 和 Spring Boot starter 版本升级计划。
- 能排查升级后的依赖冲突、映射变化和运行时错误。
- 能建立团队 MyBatis 编码规范。

## 迁移检查清单

- MyBatis Core 版本。
- mybatis-spring 版本。
- mybatis-spring-boot-starter 版本。
- Spring Boot 版本。
- Java 版本。
- 数据库驱动版本。
- 分页插件版本。
- MyBatis-Plus 或其他增强工具兼容性。
- 自定义 TypeHandler 和 Interceptor。
- XML 资源打包。
- 集成测试覆盖。

## 升级策略

1. 先升级到当前旧主线最新 patch。
2. 跑 Mapper 集成测试。
3. 检查依赖树。
4. 升级 Spring Boot 和 starter。
5. 验证所有动态 SQL、ResultMap、插件。
6. 压测关键 SQL。
7. 灰度发布。

## 生产规范

- XML namespace 必须等于 Mapper 接口全限定名。
- 多参数必须使用 `@Param`。
- 禁止用户输入直接进入 `${}`。
- join 查询必须显式列名和别名。
- 列表接口必须限制分页大小。
- 复杂 ResultMap 必须写测试。
- 禁止默认开启二级缓存。
- 所有慢 SQL 必须能定位 statementId。
- Mapper 不写业务逻辑，事务放 Service。

## 练习

1. 检查一个项目所有 `${}` 使用点。
2. 统计没有 `@Param` 的多参数 Mapper。
3. 给所有列表接口加分页上限。
4. 给慢 SQL 日志增加 statementId。

## 验收

- 能写版本升级清单。
- 能排查 starter 和 Boot 版本不兼容。
- 能制定 MyBatis 团队规范。

## 重点

- 版本升级要包含插件和增强工具。
- 生产规范重点是 SQL 安全、映射明确、性能可观测。

## 易错

> **易错：** 只升级 MyBatis Core，不检查 starter、Spring、分页插件和增强工具。
>
> 正确做法：把 MyBatis 生态依赖作为整体升级。
