## 一、性能

### 1. MyBatis 接口变慢，你怎么排查？

**参考答案：**

先看 SQL 次数和单条 SQL 耗时，记录 statementId、BoundSql、参数、耗时和 traceId。再用数据库 EXPLAIN 看执行计划、索引、扫描行数、排序和临时表。然后检查是否有 N+1、分页 offset 过大、动态 SQL 条件导致索引失效、连接池等待或锁等待。

### 2. 大 offset 分页为什么慢，怎么优化？

**参考答案：**

`limit 20 offset 1000000` 需要数据库跳过大量记录，越往后越慢。优化方式包括基于游标/seek 分页、覆盖索引、先查 id 再回表、限制最大页数。

### 3. 批量插入有哪些方式？

**参考答案：**

可以循环单条 insert、foreach 拼多 values、BatchExecutor、数据库导入工具。小批量可以 foreach，多批量可用 BatchExecutor，但要注意 flush、事务、错误定位和 SQL 长度限制。

## 二、安全

### 4. MyBatis 中 SQL 注入最常见来源是什么？

**参考答案：**

最常见来源是把用户输入放进 `${}`，例如搜索关键字、排序字段、表名、列名。普通值必须用 `#{}`。排序字段这类不能用 `?` 的位置，要使用服务端白名单映射。

### 5. 如何安全实现动态排序？

**参考答案：**

不能直接 `order by ${sort}` 接收用户输入。应在 Java 代码中把允许的排序字段做白名单映射，例如 `createdAt -> created_at`，非法值使用默认字段，然后 XML 中只拼接白名单结果。

## 三、场景题

### 6. 订单列表查询触发 N+1，你怎么改？

**参考答案：**

先通过 SQL 日志确认查询次数。如果列表页不需要明细，就去掉嵌套查询；如果需要明细，可以用 join + collection resultMap，或先查订单列表，再用 `where order_id in (...)` 批量查明细并在 Java 组装。还要限制分页大小。

### 7. `Parameter not found` 怎么排查？

**参考答案：**

看 Mapper 方法是否多参数且缺少 `@Param`，XML 中参数名是否与 `@Param` 一致，动态 SQL 的 OGNL 表达式是否引用了错误层级。如果参数是对象，看属性 getter 是否存在。

### 8. join 查询结果字段总是 null，怎么排查？

**参考答案：**

检查 SQL 是否查出了对应列，列名是否与 Java 属性匹配，是否开启驼峰映射，join 多表是否列名冲突，resultMap 的 column 是否写别名。复杂 join 建议所有列显式别名。

### 9. 为什么不建议默认开启二级缓存？

**参考答案：**

二级缓存按 namespace 管理，多表查询和跨 namespace 更新很难保证一致性。多实例部署下本地缓存还会不一致。除非非常清楚失效策略，否则不建议默认开启。
