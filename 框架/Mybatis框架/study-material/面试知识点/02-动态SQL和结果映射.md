## 一、动态 SQL

### 1. `#{}` 和 `${}` 有什么区别？

**参考答案：**

`#{}` 会生成 `?`，通过 PreparedStatement 绑定参数，能防 SQL 注入。`${}` 是字符串直接替换，会把内容拼进 SQL，存在注入风险。普通用户输入必须使用 `#{}`。表名、列名、排序字段这类无法用 `?` 绑定的位置，如果必须用 `${}`，必须做服务端白名单。

> **重点：** `#{}` 是参数绑定，`${}` 是文本替换。

### 2. MyBatis 动态 SQL 是怎么生成最终 SQL 的？

**参考答案：**

MyBatis 会根据参数对象和 OGNL 表达式计算 `<if>`、`<choose>`、`<foreach>`、`<where>`、`<set>` 等动态标签，生成最终 SQL 文本和参数映射，也就是 BoundSql。排查动态 SQL 必须看运行时 BoundSql，而不是只看 XML。

### 3. foreach 空集合有什么问题？

**参考答案：**

如果 `ids` 为空，`where id in (...)` 可能生成语法错误或不符合预期。应在业务层拦截空集合，直接返回空结果，或在 SQL 中使用安全条件，例如 `1 = 0`。

### 4. 多参数为什么建议使用 `@Param`？

**参考答案：**

多参数如果不使用 `@Param`，XML 中可能需要用 `param1`、`param2` 或依赖编译参数名，维护性差且容易错。`@Param` 能明确参数名，提升可读性和稳定性。

## 二、结果映射

### 5. resultType 和 resultMap 有什么区别？

**参考答案：**

resultType 适合简单映射，列名和属性名能自动对应。resultMap 适合复杂映射，可以明确 id、result、association、collection、构造器、嵌套查询和嵌套结果。复杂 join 和一对多映射应使用 resultMap。

### 6. association 和 collection 分别用于什么？

**参考答案：**

association 表示一对一或多对一关联，例如订单关联用户。collection 表示一对多集合，例如订单包含多个订单明细。它们可以用嵌套查询或嵌套结果实现。

### 7. MyBatis 中 N+1 问题如何产生，怎么解决？

**参考答案：**

N+1 常由嵌套查询产生：先查 N 条主记录，再为每条记录查一次子记录。解决方式包括 join + 嵌套结果、分两次批量查询后在 Java 组装、限制分页大小、避免列表接口延迟加载明细。

> **重点：** 嵌套查询直观但容易隐藏查询次数。
