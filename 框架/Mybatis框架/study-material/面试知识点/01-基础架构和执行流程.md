## 一、基础概念

### 1. MyBatis 和 JDBC 有什么区别？

**参考答案：**

JDBC 需要手写连接、PreparedStatement、参数绑定、ResultSet 遍历和对象封装。MyBatis 保留手写 SQL 的可控性，但把参数绑定、SQL 映射、结果映射、缓存和扩展点标准化。它本质是 SQL Mapper，不是完整 ORM。

> **重点：** MyBatis 没有消灭 SQL，而是让 SQL 执行和映射工程化。

### 2. MyBatis 和 Hibernate/JPA 有什么区别？

**参考答案：**

MyBatis 以 SQL 为中心，开发者明确写 SQL 和映射；Hibernate/JPA 以对象和持久化上下文为中心，有脏检查、对象状态管理和自动 SQL 生成。MyBatis 更适合复杂 SQL、报表和强调 SQL 调优的项目；JPA 更适合对象关系模型稳定、希望通过领域对象驱动持久化的项目。

### 3. MyBatis 最新版本基准是什么？

**参考答案：**

截至 2026-06-06，MyBatis Core 官方文档显示版本为 `3.5.19`。Spring Boot 集成使用 MyBatis Spring Boot Starter `4.0.x`，starter 4.x 面向 Spring Boot 4.0+ 和 Java 17+。Core 和 starter 是不同版本体系。

## 二、执行流程

### 4. Mapper 接口为什么不用写实现类？

**参考答案：**

MyBatis 会为 Mapper 接口创建动态代理。调用接口方法时，代理根据接口全限定名和方法名找到对应的 MappedStatement，然后通过 SqlSession 执行 SQL，再把结果映射为返回值。

> **重点：** Mapper 方法调用本质是动态代理分派，不是普通 Java 方法实现。

### 5. MyBatis 一次查询的核心执行链路是什么？

**参考答案：**

MapperProxy 接收方法调用，MapperMethod 找到 statementId，SqlSession 根据 statementId 获取 MappedStatement，SqlSource 生成 BoundSql，Executor 执行查询，StatementHandler 创建 PreparedStatement，ParameterHandler 绑定参数，JDBC 执行 SQL，ResultSetHandler 映射结果，TypeHandler 做类型转换。

> **难点：** 动态 SQL、参数绑定和结果映射分别发生在链路不同阶段。

### 6. `Invalid bound statement` 怎么排查？

**参考答案：**

先看 Mapper 接口全限定名是否等于 XML namespace，再看 SQL 标签 id 是否等于方法名，然后确认 XML 是否被 `mapper-locations` 加载，资源是否打包到 classpath，多模块项目尤其要检查 XML 是否放在 resources 下。

> **易错：** 以为 XML 文件名一样就能绑定，真正绑定依据是 namespace 和 id。
