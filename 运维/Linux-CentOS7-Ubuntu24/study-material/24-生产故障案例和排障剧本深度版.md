# Linux 学习资料：生产故障案例和排障剧本深度版

[返回索引](../Linux学习资料.md)

## 学习目标

- 用标准剧本处理 Linux 生产事故，覆盖取证、止血、根因和长期修复。
- 掌握 CPU、内存、I/O、网络、DNS、磁盘、启动、权限、安全模块和包管理故障的深度排查。
- 能把 CentOS 7 与 Ubuntu 24.04 的工具和路径差异纳入排障流程。

## 排障总原则

1. 先确认影响面：单机、单服务、单网段、单磁盘、单用户还是全站。
2. 先保存现场：时间、资源、进程、日志、网络、挂载、服务状态。
3. 先止血：限流、切流、释放空间、回滚配置、恢复旧版本。
4. 再根因：结合变更、日志、指标、系统调用和内核日志。
5. 最后固化：监控、告警、自动化、演练和文档。

## 案例一：服务启动失败

### 现象

- `systemctl start service` 失败。
- 应用端口无监听。
- 日志显示权限、配置、端口或依赖错误。

### 证据

```bash
systemctl status service
journalctl -u service -b --no-pager
systemctl cat service
systemctl show service -p User -p Group -p WorkingDirectory -p Environment -p ExecStart
ss -lntp
```

### 深度判断

- `ENOENT`：ExecStart 路径、解释器、动态库或配置文件缺失。
- `EACCES`：文件权限、目录 x、ACL、SELinux/AppArmor。
- `EADDRINUSE`：端口占用。
- timeout：Type 配错、启动脚本未退出、notify 未发送、依赖等待。

### 进一步定位

```bash
sudo -u service_user strace -f -o /tmp/service.strace /path/to/app args
grep -E 'ENOENT|EACCES|EPERM|EADDRINUSE' /tmp/service.strace | tail -50
```

### 止血

- 回滚最近配置。
- 临时切回旧版本。
- 修复端口冲突或权限。
- 不要直接给 777 或关闭安全模块作为长期方案。

## 案例二：机器很慢但 CPU 不高

### 现象

- 登录慢、命令慢、应用超时。
- top 中 CPU idle 仍高。

### 证据

```bash
uptime
vmstat 1 5
iostat -xz 1 5 2>/dev/null
ps -eo state,pid,ppid,cmd | awk '$1=="D"{print}'
```

### 判断

- `wa` 高、D 状态多：I/O 等待。
- `si/so` 高：swap 压力。
- DNS 慢：命令卡在解析。
- NFS/网络盘卡住：D 状态和挂载相关。

### 止血

- 暂停备份、压缩、大扫描。
- 切走流量。
- 扩容或提升磁盘规格。
- 修复 DNS 或挂载。

## 案例三：磁盘满但找不到大文件

### 证据

```bash
df -hT
du -xh --max-depth=1 / | sort -h
df -i
sudo lsof +L1
journalctl --disk-usage
```

### 判断

- `df` 满、`du` 对不上：被删除但仍打开文件、挂载覆盖、权限导致 du 看不到。
- inode 满：大量小文件。
- journal 大：journald 未限制。
- 容器 overlay：Docker/containerd 层和日志。

### 止血

- 重启或通知进程 reopen 日志。
- 清理可确认的临时文件。
- journal vacuum。
- 扩容。
- 不要删除未知系统目录。

## 案例四：外部访问端口超时

### 证据

```bash
ss -lntp | grep ':PORT'
ip route get CLIENT_IP
sudo firewall-cmd --list-all 2>/dev/null || sudo ufw status verbose
sudo nft list ruleset 2>/dev/null | head -80
sudo tcpdump -ni any host CLIENT_IP and port PORT
```

### 判断

- 没有监听：应用问题。
- 只监听 127.0.0.1：监听地址问题。
- tcpdump 看不到 SYN：上游路由、安全组或客户端没发到。
- 看到 SYN 不回：本机防火墙或内核拒绝。
- 回 SYN-ACK 客户端收不到：回程路由或中间网络。

### 止血

- 修复监听地址。
- 临时放行端口。
- 切换负载均衡后端。
- 回滚防火墙变更。

## 案例五：DNS 间歇失败

### 证据

```bash
getent hosts domain
dig domain
cat /etc/nsswitch.conf
cat /etc/resolv.conf
resolvectl status 2>/dev/null || true
tcpdump -ni any port 53
```

### 判断

- `dig` 正常、`getent` 异常：NSS、hosts、systemd-resolved。
- DNS 请求无响应：DNS 服务器或网络。
- Ubuntu 24.04 resolv.conf 被覆盖：应改 netplan/resolved。
- search domain 导致解析慢：`ndots` 和 search 配置。

## 案例六：SSH 突然无法登录

### 证据

CentOS 7：

```bash
tail -100 /var/log/secure
journalctl -u sshd -b
```

Ubuntu 24.04：

```bash
tail -100 /var/log/auth.log
journalctl -u ssh -b
```

通用：

```bash
sudo sshd -t
ls -ld /home/user /home/user/.ssh
ls -l /home/user/.ssh/authorized_keys
```

### 判断

- sshd_config 语法错误。
- 用户锁定或过期。
- home 或 authorized_keys 权限错误。
- PAM 拒绝。
- 防火墙或 fail2ban。
- SELinux/AppArmor。

### 止血

- 使用控制台修复配置。
- 回滚 sshd_config。
- 恢复用户和密钥权限。
- 不要在未验证时关闭当前 SSH 会话。

## 案例七：CentOS 7 yum 失败

### 证据

```bash
yum repolist -v
cat /etc/yum.repos.d/*.repo
yum clean all
yum makecache
```

### 判断

- CentOS 7 EOL，mirrorlist 不可用。
- DNS/代理问题。
- GPG key 问题。
- 第三方源冲突。

### 止血

- 切企业内部镜像或 vault 源维持安装能力。
- 不关闭 gpgcheck 作为长期方案。
- 制定迁移到受支持系统计划。

## 案例八：Ubuntu 24.04 网络配置 apply 后断联

### 原因

- netplan YAML 缩进错误。
- renderer 不匹配。
- 网卡名写错。
- 默认路由错误。
- DNS 配置位置错误。

### 防护

远程机器优先：

```bash
sudo netplan try
```

而不是直接 apply。保留控制台或带外管理。

## 案例九：SELinux/AppArmor 拒绝导致应用异常

### CentOS 7

```bash
getenforce
sudo ausearch -m avc -ts recent
ls -Z path
ps -eZ | grep process
```

修复顺序：

1. 检查 DAC 权限。
2. restorecon 修复上下文。
3. semanage fcontext 添加合理上下文。
4. setsebool 开启明确业务布尔值。
5. 最后才考虑自定义策略。

### Ubuntu 24.04

```bash
sudo aa-status
sudo journalctl -k | grep -i apparmor
```

修复顺序：

1. 判断 profile。
2. 临时 complain 验证。
3. 调整 profile。
4. reload profile。
5. 恢复 enforce。

## 案例十：load 高、连接也多

综合判断：

```bash
uptime
vmstat 1 5
ss -ant | awk '{print $1}' | sort | uniq -c | sort -nr
top -H -p <pid>
journalctl -u app -b --since "30 min ago"
```

可能链路：

```text
下游超时 -> 连接堆积 -> 线程增多 -> 上下文切换增加 -> 内存增加 -> GC/OOM -> 服务更慢
```

止血：

- 限流。
- 熔断下游。
- 扩容实例。
- 降低连接超时。
- 回滚发布。

长期：

- 连接池。
- 超时和重试预算。
- 线程数限制。
- 指标和告警。

## 验收

- 能按故障类别保存现场。
- 能区分临时止血和长期修复。
- 能明确 CentOS 7 与 Ubuntu 24.04 的日志、服务名和工具差异。
- 能写出可复盘的事故记录。

## 重点

- 排障要先分层，后定位。
- 工具输出要形成证据链，不是堆命令。
- 远程网络、SSH、fstab、GRUB 变更必须有回滚通道。

## 易错

> **易错：** 只凭单个命令结果下结论。
>
> 正确做法：用资源指标、日志、进程状态、系统调用、网络包和变更记录交叉验证。
