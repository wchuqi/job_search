# Linux 学习资料：实验手册和自测题

[返回索引](../Linux学习资料.md)

## 学习目标

- 通过实验验证 Linux 机制，而不是只背命令。
- 对比 CentOS 7 与 Ubuntu 24.04 在包管理、网络、防火墙、日志和安全模块上的差异。
- 建立可复用排障记录模板。

## 实验 1：系统信息和发行版差异

```bash
cat /etc/os-release
uname -a
systemctl --version
command -v yum apt rpm dpkg || true
getenforce 2>/dev/null || true
aa-status 2>/dev/null || true
```

验收：

- 能说明当前系统是 CentOS 7 还是 Ubuntu 24.04。
- 能记录内核、systemd、包管理器、安全模块。

## 实验 2：权限路径排查

```bash
sudo mkdir -p /srv/lab/private
sudo bash -c 'echo secret > /srv/lab/private/secret.txt'
sudo chmod 700 /srv/lab/private
namei -l /srv/lab/private/secret.txt
```

创建普通用户尝试读取，观察失败原因。

验收：

- 能解释目录 `x` 权限。
- 能用 `namei -l` 找到阻断点。

## 实验 3：sudo 最小权限

创建 `deploy` 用户，只允许查看 nginx 状态。

```text
deploy ALL=(root) NOPASSWD: /usr/bin/systemctl status nginx
```

验收：

- `deploy` 能执行指定命令。
- 不能执行其他 root 命令。
- 能解释命令路径风险。

## 实验 4：包管理查询

CentOS 7：

```bash
rpm -qf /usr/bin/bash
rpm -ql bash | head
yum info bash
```

Ubuntu 24.04：

```bash
dpkg -S /usr/bin/bash
dpkg -L bash | head
apt show bash
```

验收：

- 能查询文件属于哪个包。
- 能解释包数据库的价值。

## 实验 5：systemd 服务失败排查

创建一个故意失败的 service，ExecStart 指向不存在的路径。

验收命令：

```bash
systemctl status bad.service
journalctl -u bad.service -b
systemctl cat bad.service
```

验收：

- 能从日志定位失败原因。
- 修改 unit 后执行 `daemon-reload`。

## 实验 6：网络监听和防火墙

启动测试服务：

```bash
python3 -m http.server 8080 --bind 127.0.0.1
ss -lntp | grep 8080
```

验收：

- 能解释为什么外部无法访问。
- 改为监听 `0.0.0.0` 后，再检查防火墙。

## 实验 7：DNS 排查

```bash
getent hosts example.com
dig example.com
cat /etc/resolv.conf
resolvectl status 2>/dev/null || true
```

验收：

- 能说明系统使用哪个 DNS。
- Ubuntu 24.04 能说明 systemd-resolved 的作用。

## 实验 8：LVM 扩容设计

不一定真实执行，写出流程：

```text
新盘 -> 分区 -> pvcreate -> vgextend -> lvextend -> xfs_growfs/resize2fs -> df 验证 -> fstab 检查
```

验收：

- 能区分块设备、PV、VG、LV、文件系统和挂载点。
- 能说明 XFS 与 ext4 扩容命令差异。

## 实验 9：删除文件但空间不释放

测试环境：

```bash
mkdir -p /tmp/lab
python3 -c 'f=open("/tmp/lab/big.log","w"); f.write("x"*1024*1024*100); input()'
```

另一个终端删除文件：

```bash
rm /tmp/lab/big.log
sudo lsof +L1 | grep big.log
```

验收：

- 能解释进程仍打开文件导致空间不释放。

## 实验 10：cron 环境差异

写一个脚本依赖 PATH，然后放到 cron 中执行，观察失败。修复为绝对路径。

验收：

- 能说明 cron 环境和交互 shell 不同。

## 实验 11：SELinux 或 AppArmor 拒绝

CentOS 7：

```bash
getenforce
sudo ausearch -m avc -ts recent
```

Ubuntu 24.04：

```bash
sudo aa-status
sudo journalctl -k | grep -i apparmor
```

验收：

- 能区分传统权限拒绝和 MAC 拒绝。

## 实验 12：性能现场快照

执行：

```bash
date
uptime
free -h
df -hT
df -i
top -b -n1 | head -40
vmstat 1 5
ss -s
journalctl -p err -b --no-pager | tail -50
```

验收：

- 能写一份现场快照说明。
- 能判断下一步该查 CPU、内存、磁盘、网络还是服务日志。

## 综合自测题

1. CentOS 7 与 Ubuntu 24.04 的包管理、网络配置、防火墙、安全模块分别是什么？
2. 为什么文件权限 644 仍可能无法读取？
3. `systemctl enable` 和 `systemctl start` 有什么区别？
4. 修改 systemd unit 后为什么要 `daemon-reload`？
5. 为什么 Ubuntu 24.04 不建议直接改 `/etc/resolv.conf`？
6. 删除大日志后磁盘空间为什么可能不释放？
7. load average 高有哪些原因？
8. free 很低是否一定表示内存不足？
9. SELinux 和 AppArmor 的排查入口分别是什么？
10. cron 中脚本为什么手动执行成功但定时失败？

## 最终验收任务

提交一份 Linux 服务器初始化和排障报告，包含：

- 系统版本、内核、systemd、包管理器、安全模块。
- 用户和 sudo 设计。
- SSH 加固配置。
- 防火墙规则。
- 磁盘和挂载设计。
- 日志和时间同步配置。
- systemd 服务部署和日志查看。
- Shell 巡检脚本。
- CPU、内存、磁盘、网络故障排查剧本。
- CentOS 7 到 Ubuntu 24.04 迁移风险清单。

## 重点

- 实验要记录命令、输出、判断和结论。
- 所有破坏性实验必须在测试环境执行。
- 排障报告比单条命令更能证明能力。

## 易错

> **易错：** 实验只记录命令，不记录现象和解释。
>
> 正确做法：每个实验都要写出预期、实际输出、机制解释和风险。
