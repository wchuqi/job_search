## 一、网络、磁盘和日志

### 1. 端口监听了，但外部访问不通，你怎么排查？

**参考答案：**

按层排查：服务是否监听正确地址、本机能否访问、服务器 IP 是否可达、本机防火墙是否放行、上游安全组是否放行、路由是否正确、DNS 是否解析正确、包是否到达服务器。

```bash
ss -lntp
curl -v http://127.0.0.1:80/
curl -v http://server-ip:80/
ip addr
ip route
sudo firewall-cmd --list-all 2>/dev/null || sudo ufw status verbose
sudo tcpdump -ni any port 80
```

> **重点：** 监听 `127.0.0.1` 和 `0.0.0.0` 的外部访问行为不同。

### 2. CentOS 7 和 Ubuntu 24.04 网络配置有什么差异？

**参考答案：**

CentOS 7 常见 `/etc/sysconfig/network-scripts/ifcfg-*`、NetworkManager 和 `nmcli`。Ubuntu 24.04 常见 netplan，配置在 `/etc/netplan/*.yaml`，由 netplan 生成 systemd-networkd 或 NetworkManager 配置；DNS 常由 systemd-resolved 管理。

> **易错：** 在 Ubuntu 24.04 上照搬 ifcfg 配置文件。

### 3. Ubuntu 24.04 为什么不建议直接编辑 `/etc/resolv.conf`？

**参考答案：**

Ubuntu 24.04 中 `/etc/resolv.conf` 通常是 systemd-resolved 管理的链接或生成文件。直接编辑可能被重启网络、netplan apply 或 resolved 覆盖。应通过 netplan、NetworkManager 或 systemd-resolved 配置 DNS。

```bash
resolvectl status
ls -l /etc/resolv.conf
```

### 4. 删除大文件后磁盘空间为什么没有释放？

**参考答案：**

如果进程仍打开该文件，即使目录项已删除，inode 和数据块仍被进程持有，空间不会释放。需要让进程关闭文件，通常重启相关服务或让应用重新打开日志。

```bash
sudo lsof +L1
```

> **重点：** 删除文件只是删除目录项，打开的文件描述符仍可持有数据。

### 5. fstab 写错会有什么后果？如何降低风险？

**参考答案：**

fstab 写错可能导致系统重启进入 emergency mode，或挂载失败导致服务数据目录不可用。降低风险：使用 UUID，修改后执行 `mount -a` 和 `findmnt` 验证，远程机器保留控制台，非关键挂载可加 `nofail`。

```bash
blkid
sudo mount -a
findmnt /data
```

### 6. journalctl 和 `/var/log/*` 有什么关系？

**参考答案：**

journald 收集 systemd 服务 stdout/stderr、内核和系统日志，可用 `journalctl` 查询。rsyslog 可能把部分日志写入 `/var/log/*`。不同发行版传统日志路径不同：CentOS 7 认证日志常在 `/var/log/secure`，Ubuntu 24.04 常在 `/var/log/auth.log`。

```bash
journalctl -u sshd   # CentOS
journalctl -u ssh    # Ubuntu
```

### 7. cron 中脚本手动执行成功，定时执行失败，常见原因是什么？

**参考答案：**

cron 环境变量少，PATH 和工作目录与交互 shell 不同。常见原因包括命令未写绝对路径、脚本无执行权限、依赖环境变量、相对路径、用户权限不同、输出未重定向、时间理解错误。

> **重点：** cron 脚本中显式设置 PATH、工作目录和日志输出。
