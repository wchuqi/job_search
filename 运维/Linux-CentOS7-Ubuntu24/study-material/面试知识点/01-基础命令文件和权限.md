## 一、基础命令、文件和权限

### 1. Linux 中硬链接和软链接有什么区别？

**参考答案：**

硬链接是多个目录项指向同一个 inode，删除其中一个文件名不会删除实际数据，只有链接计数归零后数据才会释放。硬链接通常不能跨文件系统，也不能普通地链接目录。软链接是一个特殊文件，内容是目标路径，可以跨文件系统，目标不存在时会变成失效链接。

```bash
ln file hardlink
ln -s /path/to/file symlink
ls -li file hardlink
```

> **重点：** 硬链接共享 inode，软链接保存路径。
>
> **易错：** 以为删除原文件后硬链接也失效。硬链接不依赖“原文件名”。

### 2. 为什么文件权限是 644，用户仍然不能读取？

**参考答案：**

读取文件不仅需要文件本身有读权限，还需要路径上每一级目录有执行权限。还要检查 ACL、SELinux/AppArmor、挂载选项、文件所属用户组以及访问主体是否真的是当前用户。

```bash
id
ls -l /path/to/file
namei -l /path/to/file
getfacl /path/to/file
```

> **重点：** 目录 `x` 权限表示能进入或穿过目录。

### 3. `chmod 777` 为什么危险？

**参考答案：**

`777` 允许所有用户读、写、执行。它会破坏最小权限原则，可能让普通用户修改配置、替换脚本、写入 Web 目录、植入恶意文件或破坏审计边界。生产中应先判断访问主体和所需最小权限，再用 owner、group、ACL 或 sudo 精确授权。

> **易错：** 把 `chmod -R 777` 当成权限问题通用修复。

### 4. umask 是什么？

**参考答案：**

umask 决定新建文件或目录默认权限中要扣除哪些权限。普通文件默认最高 666，目录默认最高 777。比如 umask 027 时，新文件通常是 640，新目录通常是 750。

```bash
umask
umask 027
touch a
mkdir d
ls -ld a d
```

> **重点：** umask 影响新建对象，不会自动修改已有文件。

### 5. inode 耗尽和磁盘空间耗尽有什么区别？

**参考答案：**

磁盘空间耗尽是数据块用完，`df -h` 可见；inode 耗尽是文件元数据数量用完，`df -i` 可见。大量小文件可能导致空间还剩很多但无法创建新文件。

```bash
df -h
df -i
```

> **重点：** inode 满常见于缓存、session、小日志文件、临时文件。

### 6. `grep`、`sed`、`awk` 各自适合什么？

**参考答案：**

`grep` 适合过滤匹配行；`sed` 适合流式替换、抽取和简单编辑；`awk` 适合按字段处理、统计和格式化输出。

```bash
grep -E 'ERROR|WARN' app.log
sed -n '1,20p' app.log
awk '$9 >= 500 {count++} END {print count}' access.log
```

> **重点：** 日志统计常用 awk，简单匹配优先 grep。
