## 一、安全、性能和排障

### 1. SELinux 和 AppArmor 有什么区别？

**参考答案：**

SELinux 常见于 CentOS/RHEL 系，基于标签、类型和策略控制访问；AppArmor 常见于 Ubuntu，基于程序 profile 和路径规则控制访问。两者都是强制访问控制，会在传统权限允许的情况下继续限制访问。

CentOS 7：

```bash
getenforce
ls -Z file
sudo ausearch -m avc -ts recent
```

Ubuntu 24.04：

```bash
sudo aa-status
sudo journalctl -k | grep -i apparmor
```

> **重点：** 权限正确但仍拒绝时，要检查 MAC。

### 2. load average 高一定是 CPU 不够吗？

**参考答案：**

不一定。load average 包含可运行队列和不可中断睡眠任务。CPU 忙、I/O wait、D 状态进程、线程风暴都可能导致 load 高。要结合 `top`、`vmstat`、`iostat` 判断。

```bash
uptime
vmstat 1 5
iostat -xz 1 5
```

> **易错：** load 高直接扩 CPU。

### 3. Linux free 很低是否代表内存不足？

**参考答案：**

不一定。Linux 会用空闲内存做 page cache。应看 `available`、swap、OOM 日志和应用内存。真正压力通常表现为 available 低、swap in/out 持续、OOM 或应用分配失败。

```bash
free -h
vmstat 1 5
dmesg -T | grep -i -E 'out of memory|killed process'
```

### 4. 如何排查 OOM？

**参考答案：**

先查内核日志确认是否 OOM killer 杀进程，再找被杀进程、内存使用趋势、cgroup/systemd 限制和应用日志。

```bash
journalctl -k | grep -i -E 'out of memory|killed process'
ps aux --sort=-%mem | head
systemctl status service
cat /proc/<pid>/limits
```

> **重点：** 区分系统级 OOM、容器/cgroup 限制和应用主动退出。

### 5. 如何排查 I/O wait 高？

**参考答案：**

先用 `vmstat` 看 `wa` 和 `b`，再用 `iostat -xz` 找设备延迟和利用率，必要时用 `iotop` 找进程。结合备份、日志、数据库、压缩、容器写入和磁盘限速判断。

```bash
vmstat 1 5
iostat -xz 1 5
```

### 6. SSH 加固的关键点有哪些？

**参考答案：**

常见措施：禁止 root 登录、禁用密码登录、使用 key、限制用户、保留审计、只开放必要来源、修改前用 `sshd -t` 验证配置并保留现有会话。

CentOS 7：

```bash
sudo sshd -t
sudo systemctl reload sshd
```

Ubuntu 24.04：

```bash
sudo sshd -t
sudo systemctl reload ssh
```

### 7. 生产排障时为什么不建议第一步重启？

**参考答案：**

重启可能清掉现场：进程状态、内存状态、临时文件、连接、journal 缓冲、错误上下文和复现条件。正确做法是先保存现场快照，再判断是否需要重启止血。

```bash
date
uptime
free -h
df -hT
top -b -n1 | head -40
journalctl -p err -b --no-pager | tail -100
```

> **重点：** 先取证，再止血，再根治。
