# Linux 学习资料：总览、心智模型和发行版差异

[返回索引](../Linux学习资料.md)

## 学习目标

- 理解 Linux 系统由内核、用户态工具、发行版、服务管理器、包管理器和配置约定共同组成。
- 建立排障心智模型：命令只是入口，真正改变系统的是文件、进程、内核接口和服务状态。
- 掌握 CentOS 7 与 Ubuntu 24.04 的核心差异，避免跨发行版套命令。

## 理论导读

Linux 不是一个单一产品。严格说，Linux 是内核；你日常登录进去看到的系统，是内核加 GNU 工具、systemd、包管理器、网络配置工具、安全模块、日志系统和发行版维护策略组合起来的结果。CentOS 7 和 Ubuntu 24.04 都是 Linux，但它们的生命周期、软件版本、配置路径、默认安全策略和运维工具差异很大。

学习 Linux 的关键不是记住 `ls`、`top`、`systemctl` 这些命令，而是理解它们读写了哪些文件、查询了哪些内核接口、操作了哪些进程和服务状态。这样遇到新发行版、新工具或异常输出时，仍能按层拆解。

## 核心心智模型

### 1. Linux 是分层系统

```text
用户和应用
  -> Shell / 命令 / 服务
  -> glibc / 系统库 / systemd / 包管理器
  -> 系统调用
  -> Linux kernel
  -> CPU / 内存 / 磁盘 / 网卡
```

命令行看到的是表层，内核状态和配置文件才是底层事实。例如 `systemctl status nginx` 展示服务状态，但服务能否启动还取决于 unit 文件、可执行文件、配置文件、权限、端口占用、SELinux/AppArmor、防火墙和依赖服务。

### 2. Linux 排障按层推进

| 层 | 典型问题 | 常用证据 |
| --- | --- | --- |
| 应用层 | 配置错、端口错、依赖缺失 | 应用日志、启动参数 |
| 服务层 | systemd unit、环境变量、权限 | `systemctl status`、`journalctl` |
| 用户权限层 | 文件权限、sudo、PAM、安全模块 | `id`、`namei -l`、audit 日志 |
| 网络层 | 监听、路由、防火墙、DNS | `ss`、`ip`、`dig`、`tcpdump` |
| 存储层 | 磁盘满、挂载错误、inode 满 | `df`、`lsblk`、`findmnt` |
| 内核资源层 | CPU、内存、I/O、OOM | `top`、`vmstat`、`iostat`、`dmesg` |

## CentOS 7 与 Ubuntu 24.04 总体差异

| 维度 | CentOS 7 | Ubuntu 24.04 |
| --- | --- | --- |
| 生命周期 | 已 EOL，生产应迁移或购买延长支持 | LTS，标准支持到 2029，Ubuntu Pro 可延长 |
| 上游体系 | RHEL 7 生态 | Debian/Ubuntu 生态 |
| 内核 | 3.10 系列加大量 backport | 6.8 系列 |
| init/service | systemd 219 | systemd 255 系列 |
| 包管理 | `rpm`、`yum` | `dpkg`、`apt`，并常见 `snap` |
| 网络配置 | NetworkManager、ifcfg、network-scripts | netplan、systemd-networkd 或 NetworkManager |
| 防火墙 | firewalld，底层多为 iptables | ufw 常见，底层 nftables/iptables-nft |
| 安全模块 | SELinux 默认常见 | AppArmor 默认常见 |
| 日志 | journald + rsyslog，认证日志 `/var/log/secure` | journald + rsyslog，认证日志 `/var/log/auth.log` |
| DNS | 常见直接 `/etc/resolv.conf` 或 NetworkManager 管理 | systemd-resolved，`/etc/resolv.conf` 通常是链接 |
| Python | Python 2 历史包多，Python 3 需额外关注 | Python 3 默认 |
| 容器生态 | 旧内核和旧用户态限制较多 | cgroup v2、较新容器运行时生态 |

## 为什么 CentOS 7 仍要学

CentOS 7 已 EOL，但很多存量企业系统仍运行在 CentOS 7 上。你需要掌握它，是为了维护、迁移和排障，而不是为了新项目继续选用。CentOS 7 的旧内核、旧 OpenSSL、旧 glibc、旧 systemd 和旧 Python 生态会影响安全、性能、软件兼容和容器能力。

## 为什么 Ubuntu 24.04 要单独学

Ubuntu 24.04 是现代 LTS 系统，默认工具链和系统组件明显更新。它的 netplan、systemd-resolved、AppArmor、snap、cloud-init、cgroup v2、nftables 生态与 CentOS 7 差异明显。新环境交付时，如果仍按 CentOS 7 的路径编辑网络和 DNS 配置，很容易被自动生成机制覆盖。

## 例子：同一个任务在两个发行版上的差异

### 查看系统版本

```bash
cat /etc/os-release
uname -a
systemctl --version
```

### 安装软件

CentOS 7：

```bash
sudo yum install -y nginx
rpm -q nginx
```

Ubuntu 24.04：

```bash
sudo apt update
sudo apt install -y nginx
dpkg -l nginx
```

### 查看认证日志

CentOS 7：

```bash
sudo tail -f /var/log/secure
```

Ubuntu 24.04：

```bash
sudo tail -f /var/log/auth.log
```

## 练习

1. 分别在 CentOS 7 和 Ubuntu 24.04 上执行：

```bash
cat /etc/os-release
uname -r
systemctl --version
which python python2 python3 || true
```

2. 记录两者的内核版本、systemd 版本、包管理器和默认安全模块。
3. 用 `systemctl list-unit-files --type=service | head` 观察服务列表。
4. 找到认证日志、系统日志和包管理日志的位置。

## 验收

- 能说明 CentOS 7 和 Ubuntu 24.04 的关键差异。
- 能解释为什么 CentOS 7 新项目风险高。
- 能说出系统排障时应按哪些层级收集证据。
- 能把“命令执行失败”拆成命令不存在、包未安装、权限不足、配置路径不同、安全模块拒绝、服务状态异常等可能原因。

## 重点

- 发行版差异不是表面命令差异，而是包生态、配置生成机制、安全模块和生命周期差异。
- Linux 排障要看证据链，不能只凭经验执行修复命令。
- CentOS 7 是存量维护对象，Ubuntu 24.04 更适合新环境交付。

## 难点

- 许多配置文件不是最终事实。例如 Ubuntu 24.04 的 DNS 可能由 systemd-resolved 和 netplan 管理，直接编辑 `/etc/resolv.conf` 往往不是持久配置。

## 易错

> **易错：** 在 Ubuntu 24.04 上照搬 CentOS 7 的 `/etc/sysconfig/network-scripts/ifcfg-*` 配置方式。
>
> 正确做法：Ubuntu 24.04 通常使用 netplan，配置在 `/etc/netplan/*.yaml`。

> **易错：** 在 CentOS 7 上看到内核是 3.10 就认为没有任何新特性或安全修复。
>
> 正确做法：RHEL/CentOS 会 backport 修复和部分功能，但 EOL 后官方更新停止，仍需要迁移规划。
