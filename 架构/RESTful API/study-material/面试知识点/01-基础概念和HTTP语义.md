## 一、基础概念和HTTP语义

### 1. 什么是 RESTful API？

**参考答案：**

RESTful API 是一种围绕资源设计接口的风格。它使用 URI 定位资源，使用 HTTP 方法表达操作语义，使用状态码表达通用处理结果，使用 JSON 等表示格式传输资源状态。它强调资源、表示、统一接口和无状态交互。

> **重点：** 不要只回答“URL 用名词不用动词”，这只是外在规范。

### 2. RESTful API 和 RPC 有什么区别？

**参考答案：**

RESTful API 以资源为中心，接口像 `GET /orders/1001`；RPC 以过程或方法为中心，接口像 `createOrder`、`cancelOrder`。REST 更容易利用 HTTP 语义、缓存、状态码和通用工具；RPC 在复杂动作、强命令式业务、内部服务调用中可能更直接。

> **易错：** REST 和 RPC 不是绝对优劣，关键看场景和团队治理能力。

### 3. HTTP 方法的安全性和幂等性是什么？

**参考答案：**

安全性指请求不应改变服务端资源状态，例如 `GET`。幂等性指执行一次和执行多次的最终效果相同，例如 `PUT`、`DELETE` 通常幂等。`POST` 通常不幂等，但可以通过幂等键实现业务幂等。

> **难点：** 安全和幂等不是一回事。安全一定幂等，幂等不一定安全。

### 4. `PUT` 和 `PATCH` 有什么区别？

**参考答案：**

`PUT` 通常表示对目标资源的整体替换，请求体应包含完整资源表示；`PATCH` 表示局部修改，只提交要变更的字段或补丁。`PUT` 通常幂等，`PATCH` 是否幂等取决于补丁语义。

```http
PATCH /users/u1
Content-Type: application/json

{
  "nickname": "alice"
}
```

> **易错：** 用 `PATCH` 做库存增减时，如果补丁表示“减少 1”，重复请求会重复扣减。
