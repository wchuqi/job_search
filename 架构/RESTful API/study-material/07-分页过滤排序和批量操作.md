# RESTful API学习资料：分页过滤排序和批量操作

[返回索引](../RESTful API学习资料.md)

## 学习目标

- 掌握集合查询 API 的常见能力。
- 能选择页码分页和游标分页。
- 能设计批量操作和局部失败响应。

## 理论导读

集合资源是 RESTful API 中最常用也最容易变复杂的部分。查询条件、排序、分页、字段投影和关联展开如果没有统一规范，会导致接口爆炸和性能问题。

## 核心心智模型

集合查询像在货架上找商品：过滤决定看哪类货架，排序决定查看顺序，分页决定每次搬多少，字段投影决定标签显示哪些信息。

## 知识点详解

### 页码分页

```http
GET /orders?page=1&pageSize=20
```

优点是简单，适合后台列表。缺点是大页码性能差，数据变化时可能重复或漏掉。

### 游标分页

```http
GET /orders?limit=20&cursor=eyJjcmVhdGVkQXQiOiIyMDI2...
```

适合信息流、时间线和大数据量列表。游标通常包含排序字段和上一次位置，不应让客户端依赖内部格式。

### 过滤和排序

```http
GET /orders?status=PAID&createdAtFrom=2026-06-01&sort=-createdAt,id
```

排序字段要有白名单，避免把任意数据库字段暴露给调用方。

### 批量操作

批量删除：

```http
POST /orders/batch-deletion-requests
Content-Type: application/json

{
  "orderIds": ["1001", "1002", "1003"]
}
```

局部失败响应：

```json
{
  "succeeded": ["1001", "1002"],
  "failed": [
    {
      "id": "1003",
      "code": "ORDER_NOT_CANCELLABLE",
      "message": "Order has been shipped."
    }
  ]
}
```

## 练习

为商品搜索设计分页、过滤、排序和字段投影参数，要求支持分类、价格区间、品牌、库存、销量排序和上架时间排序。

## 验收

- 能解释页码分页和游标分页适用场景。
- 能处理批量操作的全成功、全失败和部分失败。
- 能说明过滤排序参数的安全和性能边界。

> **重点：** 集合查询参数必须标准化，否则 API 会快速膨胀。
>
> **难点：** 总数统计、深分页、多字段排序和实时数据变化会带来性能与一致性问题。
>
> **易错：** 无限制允许 `pageSize` 或任意排序字段，容易造成慢查询和资源耗尽。
