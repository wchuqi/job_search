# RESTful API学习资料：RESTful API完整知识点清单

[返回索引](../RESTful API学习资料.md)

## 一、基础和术语

- REST、RESTful、HTTP API、RPC、GraphQL 的边界。
- 资源、表示、URI、统一接口、无状态。
- 客户端、服务端、代理、网关、缓存。
- 请求行、状态行、Header、Body、媒体类型。

## 二、HTTP 方法和语义

- `GET`、`HEAD`、`POST`、`PUT`、`PATCH`、`DELETE`、`OPTIONS`。
- 安全方法、幂等方法、可缓存方法。
- 方法选择规则和错误用法。
- 幂等键、防重复提交、重试边界。

## 三、资源建模

- 集合资源、单资源、子资源、关系资源。
- 动作资源、任务资源、状态资源。
- 路径参数和查询参数。
- 嵌套深度、资源生命周期、资源归属。

## 四、URI和查询设计

- 复数名词、小写连字符、稳定 ID。
- 分页：页码分页、游标分页。
- 过滤、排序、搜索、字段投影、关联展开。
- 批量操作和部分失败。

## 五、请求响应表示

- JSON 字段命名、金额、时间、枚举、可空字段。
- 响应包装与裸资源。
- `Content-Type`、`Accept`、内容协商。
- 创建响应 `201 + Location`。
- 空响应 `204`。

## 六、状态码和错误模型

- 2xx、3xx、4xx、5xx 分类。
- `400`、`401`、`403`、`404`、`409`、`412`、`422`、`429`、`503`。
- Problem Details。
- 业务错误码、字段错误、traceId。
- 错误信息安全。

## 七、安全

- TLS、认证、授权、资源归属。
- Bearer Token、Cookie Session、JWT、OAuth2。
- CORS、CSRF、XSS 对 API 的影响。
- IDOR、越权、敏感字段泄露、重放攻击。
- 输入校验、输出脱敏、审计日志。

## 八、缓存和并发

- `Cache-Control`、`ETag`、`Last-Modified`。
- `If-None-Match`、`If-Modified-Since`。
- `If-Match`、`If-Unmodified-Since`。
- `304`、`412`。
- 乐观锁和资源版本。

## 九、版本和兼容

- 兼容变更和破坏性变更。
- URI 版本、Header 版本、媒体类型版本。
- 字段新增、删除、改名、类型变化、枚举扩展。
- 废弃、迁移、灰度、调用方统计。

## 十、文档和测试

- OpenAPI 基础结构。
- Schema、参数、响应、错误示例、安全方案。
- Mock、SDK 生成、契约测试。
- 兼容性检查和文档漂移治理。

## 十一、生产治理

- API 网关、路由、认证、限流、灰度。
- 超时、重试、熔断、降级。
- 日志、指标、链路追踪。
- 慢接口、错误率、P95/P99、依赖耗时。
- 隐私日志和合规。

## 十二、面试和实践

- REST 与 RPC 对比。
- 方法语义和状态码辨析。
- 订单、支付、退款等高风险接口设计。
- 大列表分页和性能。
- 版本演进和兼容方案。
- 线上故障排查。

## 十三、REST架构约束和成熟度

- Client-Server、Stateless、Cacheable、Uniform Interface、Layered System、Code on Demand。
- Richardson 成熟度模型 Level 0 到 Level 3。
- HATEOAS 的理论价值和工程落地成本。
- RESTful API、HTTP API、RPC、GraphQL、Webhook、事件接口的取舍。
- 复杂命令、长任务、实时通信不适合强行 REST 的场景。

## 十四、资源建模深度规则

- 资源是否可命名、是否有独立生命周期、是否有独立权限、是否有独立查询入口。
- 关系资源：成员关系、关注关系、授权关系。
- 动作资源：取消申请、审批动作、状态转移记录。
- 任务资源：导入、导出、批处理、报表生成。
- 搜索资源：复杂搜索条件、保存搜索、一次性搜索。
- 资源建模反模式：动词路径、深度嵌套、表结构暴露、万能 `/api`。

## 十五、HTTP边界和错误决策

- `400`、`409`、`412`、`422` 的选择规则。
- `401`、`403`、`404` 在安全隐藏策略下的取舍。
- `201`、`202`、`204` 的成功响应边界。
- `405`、`406`、`415`、`429`、`503`、`504` 的生产语义。
- `Location`、`Retry-After`、`ETag`、`If-Match`、`Content-Type`、`Accept` 的治理作用。
- Problem Details、业务错误码、字段级错误、traceId。

## 十六、可靠性和一致性

- HTTP 方法幂等、业务幂等、消息幂等。
- 幂等键、请求摘要、处理状态、结果复用、过期策略。
- 唯一约束、状态机、乐观锁、分布式锁的优先级。
- 超时后结果未知的处理方式。
- 支付回调、退款回调、消息重复、乱序事件处理。
- 重试策略、指数退避、重试风暴。

## 十七、安全深度

- 威胁建模：身份伪造、越权、权限提升、重放、数据泄露、接口滥用。
- Bearer Token、Cookie Session、JWT、OAuth2、OIDC 的边界。
- JWT 撤销、短期 token、refresh token、密钥轮换、权限版本。
- CORS 和 CSRF 的区别。
- 多租户隔离、资源级授权、数据范围、客服临时授权。
- 审计日志、敏感字段脱敏、日志合规。

## 十八、性能和网关治理

- 浏览器缓存、CDN、网关缓存、应用缓存、数据库缓存。
- `public`、`private`、`max-age`、`no-cache`、`no-store`、`stale-while-revalidate`。
- 页码分页深分页问题、游标分页稳定排序。
- 过滤排序白名单、索引评估、字段投影、include 展开深度。
- 网关职责：TLS、路由、认证、限流、灰度、日志、请求体限制。
- 限流算法：固定窗口、滑动窗口、令牌桶、漏桶。

## 十九、契约和治理

- API 契约包含路径、方法、参数、响应、状态码、错误码、Header、权限语义、分页排序和限流。
- 兼容变更、破坏性变更、灰色风险变更。
- OpenAPI 设计评审、Mock、Schema 校验、SDK 生成、破坏性变更扫描。
- Provider Contract Test 和 Consumer-Driven Contract。
- Deprecated、Sunset、调用方统计、灰度拒绝、下线回滚。

## 完整性自检

- 能否把业务动作转化为资源、状态或任务？
- 能否解释每个状态码的选择理由？
- 能否说明接口是否安全、幂等、可缓存？
- 能否处理重复请求、并发更新和超时重试？
- 能否保护敏感数据并做资源级授权？
- 能否通过 OpenAPI 和契约测试保证调用方不被破坏？
- 能否解释复杂动作为什么建模为资源、任务或命令？
- 能否为支付、退款、发券设计完整幂等和并发方案？
- 能否根据状态码、延迟、traceId 和依赖耗时定位线上 API 问题？
