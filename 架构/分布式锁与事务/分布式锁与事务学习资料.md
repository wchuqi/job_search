# 分布式锁与事务学习资料

这是一套面向后端开发、架构设计和面试复习的分布式锁与事务学习资料。重点不是背 API，而是理解多个服务、多个数据库、消息队列、缓存和外部支付系统共同参与时，系统如何保证互斥、并发正确性、最终一致、可补偿和可恢复。

## 使用建议

- 入门：先读 00、01、02，建立“锁只解决互斥，事务解决原子性和隔离，一致性还要靠业务不变量”的底层认知。
- 做项目：重点读 03 到 09，能设计 Redis 锁、数据库约束、Saga、TCC、Outbox、幂等消费和对账任务。
- 面试复习：先读 14 完整清单，再读 13 和 `study-material/面试知识点/`；中高级面试重点读 16 到 23。
- 生产排障：重点读 04、05、08、09、10、15、22、23，训练从锁过期、旧请求覆盖、消息重复、补偿失败、对账差异中定位问题。
- 深入提升：读 16 到 23，重点训练“方案为什么正确、在哪些故障模型下不成立、如何用状态机和恢复日志兜底”。

## 环境假设

示例以 Java 后端、Redis、ZooKeeper/etcd、关系型数据库、消息队列、订单支付库存等常见系统为背景，但核心原则适用于多数后端语言和中间件。
