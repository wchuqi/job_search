## Redis 锁

```bash
SET lock:resource request-id NX PX 30000
```

释放锁：

```lua
if redis.call("GET", KEYS[1]) == ARGV[1] then
  return redis.call("DEL", KEYS[1])
else
  return 0
end
```

fencing token：

```bash
INCR lock:resource:token
```

## 数据库状态机

```sql
UPDATE orders
SET status = 'PAID',
    version = version + 1
WHERE id = 'o1001'
  AND status = 'CREATED'
  AND version = 3;
```

## 去重表

```sql
CREATE TABLE processed_message (
  message_id VARCHAR(64) NOT NULL,
  consumer VARCHAR(64) NOT NULL,
  processed_at TIMESTAMP NOT NULL,
  PRIMARY KEY (message_id, consumer)
);
```

## Outbox 最小字段

| 字段 | 作用 |
| --- | --- |
| id | 事件唯一 ID |
| aggregate_id | 聚合根 ID |
| event_type | 事件类型 |
| payload | 事件内容 |
| status | NEW/PUBLISHED/FAILED |
| retry_count | 重试次数 |
| next_retry_at | 下次重试时间 |

## 方案选择速查

| 需求 | 推荐方案 |
| --- | --- |
| 防重复插入 | 唯一索引 |
| 防止余额扣成负数 | 原子更新或行锁 |
| 多实例任务只跑一个 | Redis 锁 + 幂等任务 |
| 高可靠选主 | ZooKeeper/etcd |
| 短事务强一致 | XA/2PC |
| 长流程最终一致 | Saga |
| 资源预留 | TCC |
| 写库后可靠通知其他服务 | Outbox |
| 消费者防重复 | Inbox/去重表 |
| 外部支付一致性 | 流水 + 幂等回调 + 对账 |

## 快速自检

- 是否有数据库唯一约束保护核心不变量。
- 是否有状态机防止旧状态覆盖新状态。
- 锁是否设置 TTL，释放是否校验 value。
- 长任务是否考虑续期失败和旧请求继续执行。
- 关键写入是否有 fencing token 或版本校验。
- 消息消费者是否幂等。
- 补偿失败是否有告警和人工入口。
