# 算法学习资料：总览与心智模型

[返回索引](../算法学习资料.md)

## 学习目标

- 建立算法学习的整体地图。
- 明白为什么算法题不是背代码，而是训练建模、证明和实现。
- 学会用统一流程分析题目。

## 理论导读

算法解决的是“在约束条件下，用有限资源把输入变成正确输出”的问题。面试题通常不会只考某个语法 API，而是考你能否读懂输入规模、抽象状态、选择合适结构、证明不会漏解或错解，并把复杂度控制在可接受范围内。

一个有效的算法心智模型是“仓库管理员”：题目不断给你物品和规则，你要决定用什么货架存信息、按什么顺序处理、什么时候丢弃无用信息、什么时候合并状态。数组像连续货架，哈希表像贴标签的抽屉，堆像随时能拿出最小或最大元素的优先窗口，图像关系网络，DP 表像记录已经验证过答案的账本。

## 解题流程

1. 读清楚输入、输出、数据范围和边界条件。
2. 判断暴力解复杂度，确认是否会超时。
3. 识别题型：数组、哈希、窗口、二分、栈、树、图、DP、贪心等。
4. 定义状态：指针、集合、窗口、队列、递归参数、DP 数组。
5. 写出不变量：循环过程中什么始终成立。
6. 实现并检查边界：空数组、单元素、重复值、负数、越界、溢出。
7. 计算复杂度。
8. 复盘错因。

## 算法知识覆盖图

| 领域 | 核心能力 | 高频题型 |
| --- | --- | --- |
| 复杂度 | 判断算法可行性 | 数据规模估算、递归复杂度 |
| 线性结构 | 顺序扫描和局部状态 | 数组、字符串、链表 |
| 哈希和计数 | 快速查找和频率统计 | 两数之和、异位词、最长连续序列 |
| 指针和窗口 | 边界维护 | 三数之和、无重复最长子串、最小覆盖子串 |
| 单调结构 | 维护候选最优值 | 下一个更大元素、柱状图最大矩形 |
| 二分和排序 | 利用有序性 | 搜索旋转数组、二分答案 |
| 树图遍历 | 关系结构搜索 | 层序遍历、岛屿数量、课程表 |
| 回溯 | 枚举所有合法选择 | 全排列、组合、N 皇后 |
| 动态规划 | 复用子问题答案 | 背包、子序列、路径、区间 |
| 贪心 | 局部选择形成全局最优 | 跳跃游戏、区间调度 |
| 高级结构 | 复杂状态维护 | 并查集、Trie、线段树、树状数组 |
| 设计题 | 数据结构组合 | LRU、随机集合、数据流中位数 |

## 学习方法

- 每道题至少写一次暴力思路，再优化。
- 每个模板至少用 3 道题验证边界。
- 错题记录必须包含：错因、正确模型、边界条件、最终复杂度。
- Java 21 和 Python 3 都要能写核心题，但面试优先选自己最稳的语言。

## 练习

选择任意 5 道简单题，为每道题写出：

- 题目约束。
- 暴力解。
- 优化思路。
- 正确性理由。
- 时间和空间复杂度。

## 验收

- 能看到 `n = 10^5` 时主动排除多数 `O(n^2)` 解法。
- 能独立解释“为什么这个结构能减少重复计算”。
- 能用 Java 21 或 Python 3 在 20 分钟内完成一道常见简单题。

> **重点：** 算法题的核心不是“用了哪个模板”，而是“这个模板维护了什么不变量”。
>
> **难点：** 很多中等题难在状态定义，而不是代码长度。
>
> **易错：** 直接看题解背代码。正确做法是先写自己的暴力思路，再理解优化消除了哪部分重复。
