# 算法学习资料：复杂度和数据结构基础

[返回索引](../算法学习资料.md)

## 学习目标

- 掌握复杂度分析。
- 熟悉算法题中最常用的数据结构。
- 能根据操作需求选择结构。

## 理论导读

复杂度描述的是输入规模增长时，算法成本如何增长。它不是精确计时器，而是“增长趋势说明书”。在面试中，复杂度通常决定一个方案能不能进入下一轮讨论。

数据结构是算法的容器。相同逻辑换一个结构，复杂度可能完全不同。例如查找一个元素，数组线性扫描是 `O(n)`，哈希表平均是 `O(1)`；但如果需要按顺序不断取最小值，哈希表不合适，堆更合适。

## 复杂度速查

| 复杂度 | 常见场景 | `n = 10^5` 是否通常可行 |
| --- | --- | --- |
| `O(1)` | 哈希平均查找、数组下标访问 | 可行 |
| `O(log n)` | 二分、平衡树操作 | 可行 |
| `O(n)` | 一次扫描 | 可行 |
| `O(n log n)` | 排序、堆处理 | 通常可行 |
| `O(n^2)` | 双重循环、所有数对 | 多数不可行 |
| `O(2^n)` | 子集枚举 | 只适合小 `n` |
| `O(n!)` | 全排列 | 只适合很小 `n` |

## 数据结构选择

| 需求 | Java 21 | Python 3 | 复杂度 |
| --- | --- | --- | --- |
| 尾部增删 | `ArrayList` | `list` | 均摊 `O(1)` |
| 双端队列 | `ArrayDeque` | `collections.deque` | `O(1)` |
| 哈希映射 | `HashMap` | `dict` | 平均 `O(1)` |
| 去重集合 | `HashSet` | `set` | 平均 `O(1)` |
| 优先队列 | `PriorityQueue` | `heapq` | 插入/删除 `O(log n)` |
| 排序数组 | `Arrays.sort` | `list.sort` | `O(n log n)` |

## Java 21 示例：计数

```java
import java.util.*;

class Solution {
    public Map<Integer, Integer> count(int[] nums) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int x : nums) {
            freq.merge(x, 1, Integer::sum);
        }
        return freq;
    }
}
```

## Python 3 示例：计数

```python
from collections import Counter

def count(nums: list[int]) -> dict[int, int]:
    return dict(Counter(nums))
```

## 常见边界

- 空输入：`[]`、`""`、`null` 或 `None`。
- 单元素：很多双指针和递归题会漏。
- 重复元素：排序、去重、回溯必须特别处理。
- 负数：滑动窗口和前缀和题经常因为负数失效。
- 整数溢出：Java 中 `int` 可能溢出，必要时用 `long`。

## 验收

- 能把 `O(n^2)`、`O(n log n)`、`O(n)` 的差距讲清楚。
- 能说明堆、哈希表、双端队列分别适合什么操作。

> **重点：** 复杂度要按最坏或题目要求的主要场景分析，不能只看样例。
>
> **难点：** 递归复杂度通常要同时考虑递归树分支数和每层工作量。
>
> **易错：** Python 切片会复制，Java 字符串拼接在循环里可能产生额外成本。
