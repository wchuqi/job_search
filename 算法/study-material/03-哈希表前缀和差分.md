- 了解差分数组处理批量区间修改。

## 理论导读

哈希表用空间换时间，把“我之前见过什么”保存起来。前缀和把区间求和变成两个端点的差。差分数组则反过来，把大量区间修改延迟到最后一次扫描。

这三类方法的共同点是：不重复做已经做过的计算，而是把历史信息整理成可快速查询的状态。

## 前缀和公式

设 `prefix[0] = 0`，`prefix[i + 1] = nums[0] + ... + nums[i]`。

区间 `[l, r]` 的和：

```text
sum(l, r) = prefix[r + 1] - prefix[l]
```

如果要找和为 `k` 的子数组，当前前缀和为 `s`，需要历史上出现过 `s - k`。

## Java 21 示例：和为 K 的子数组

```java
import java.util.*;

class Solution {
    public int subarraySum(int[] nums, int k) {
        Map<Integer, Integer> count = new HashMap<>();
        count.put(0, 1);
        int prefix = 0, ans = 0;
        for (int x : nums) {
            prefix += x;
            ans += count.getOrDefault(prefix - k, 0);
            count.merge(prefix, 1, Integer::sum);
        }
        return ans;
    }
}
```

## Python 3 示例：和为 K 的子数组

```python
from collections import defaultdict

class Solution:
    def subarraySum(self, nums: list[int], k: int) -> int:
        count = defaultdict(int)
        count[0] = 1
        prefix = ans = 0
        for x in nums:
            prefix += x
            ans += count[prefix - k]
            count[prefix] += 1
        return ans
```

## 差分数组模型

对区间 `[l, r]` 增加 `v`：

```text
diff[l] += v
diff[r + 1] -= v
```

最后对 `diff` 做一次前缀和，就得到修改后的数组。

## 验收

- 能解释为什么 `count[0] = 1` 是必要初始化。
- 能手推一个差分数组恢复过程。

> **重点：** 前缀和保存的是“到当前位置之前”的累计状态。
>
> **难点：** 子数组计数题往往不是记录一个前缀和是否出现，而是记录出现次数。
>
> **易错：** 把子数组和子序列混淆。前缀和只直接适用于连续区间。
