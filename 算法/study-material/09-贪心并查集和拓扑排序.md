## 学习目标

- 掌握并查集维护连通关系。
- 掌握拓扑排序处理依赖关系。

## 理论导读

贪心算法每一步选择当前看来最优的方案，但它只有在局部最优能推出全局最优时才正确。不能因为代码短就相信贪心，必须能给出交换论证、反证或清晰的不变量。

并查集把多个元素划分成集合，支持快速查询两个元素是否属于同一集合。拓扑排序则处理有向无环图中的依赖顺序，例如课程先修关系和任务编排。

## 并查集模板

```java
class DSU {
    private final int[] parent;
    private final int[] rank;

    DSU(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }

    boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;
        if (rank[ra] < rank[rb]) {
            parent[ra] = rb;
        } else if (rank[ra] > rank[rb]) {
            parent[rb] = ra;
        } else {
            parent[rb] = ra;
            rank[ra]++;
        }
        return true;
    }
}
```

## Python 3 并查集

```python
class DSU:
    def __init__(self, n: int):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, a: int, b: int) -> bool:
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False
        if self.rank[ra] < self.rank[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        if self.rank[ra] == self.rank[rb]:
            self.rank[ra] += 1
        return True
```

## 验收

- 能说明贪心策略的正确性依据。
- 能手动模拟并查集路径压缩。
- 能用入度为 0 的队列写拓扑排序。

> **重点：** 贪心必须证明，不能只靠样例通过。
>
> **难点：** 并查集只能表达连通性，不适合直接表达最短路径。
>
> **易错：** 拓扑排序结束后没检查是否处理了所有节点，导致环检测失败。
