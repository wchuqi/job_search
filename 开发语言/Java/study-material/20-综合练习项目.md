# Java 学习资料：综合练习项目

[返回索引](../Java学习资料.md)

## 项目目标

构建一个基于 JDK 21 的任务管理系统，从命令行版本逐步演进到 Web API。通过一个项目串联 Java 语言、集合、IO、并发、JVM、测试、数据库和 Web 后端知识。

## 阶段 1：命令行任务管理器

功能：

- 创建任务。
- 修改任务状态。
- 查询任务列表。
- 按优先级排序。
- 保存到 JSON 或文本文件。

知识点：

- class / record
- enum
- List / Map
- Comparator
- Files
- 异常处理

示例模型：

```java
public record Task(
        long id,
        String title,
        TaskStatus status,
        int priority
) {}

public enum TaskStatus {
    TODO, DOING, DONE
}
```

## 阶段 2：函数式数据处理

功能：

- 按状态分组。
- 统计每种状态数量。
- 查询最高优先级任务。
- 导出报表。

知识点：

- Stream
- groupingBy
- Optional
- BigDecimal，若涉及耗时或金额
- java.time

## 阶段 3：并发任务执行器

功能：

- 批量执行任务。
- 支持超时。
- 支持失败重试。
- 用虚拟线程并发执行 IO 模拟任务。

知识点：

- ExecutorService
- virtual thread
- CompletableFuture
- timeout
- 并发集合
- 日志

## 阶段 4：JDBC 持久化

功能：

- 使用数据库保存任务。
- 支持事务。
- 使用连接池。
- 防止 SQL 注入。

表结构：

```sql
create table tasks (
    id bigint primary key,
    title varchar(200) not null,
    status varchar(20) not null,
    priority int not null,
    created_at timestamp not null
);
```

## 阶段 5：Spring Boot REST API

接口：

- `POST /tasks`
- `GET /tasks/{id}`
- `GET /tasks`
- `PUT /tasks/{id}/status`
- `DELETE /tasks/{id}`

要求：

- 参数校验。
- 统一异常处理。
- 分层结构。
- 单元测试和集成测试。
- 日志不输出敏感信息。

## 阶段 6：质量和排障

要求：

- JUnit 5 测试。
- Mockito 隔离外部依赖。
- JaCoCo 覆盖率。
- `mvn clean verify`。
- JFR 或 jcmd 观察运行状态。
- 模拟内存或线程问题并写排障笔记。

## 验收标准

- 使用 JDK 21 编译。
- 有清晰分层。
- 有不少于 20 个单元测试。
- 有至少 1 个集成测试。
- 有数据库事务场景。
- 有虚拟线程并发场景。
- 有统一异常处理。
- 有 README 说明运行方式。
- 能回答项目中使用了哪些 Java 语言特性。
