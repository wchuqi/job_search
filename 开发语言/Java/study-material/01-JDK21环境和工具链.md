# Java 学习资料：JDK 21 环境和工具链

[返回索引](../Java学习资料.md)

## 学习目标

完成本章后，你应该能：

- 安装并验证 JDK 21。
- 使用 `javac`、`java`、`jar`、`jshell`。
- 理解 classpath、module path、source / target / release。
- 知道常用 JDK 诊断工具的作用。

## 理论导读：工具链是在管理“编译、查找、运行、诊断”

JDK 工具链不是安装完就结束的环境配置，而是在管理 Java 程序的一生。`javac` 负责把源码变成字节码，`java` 负责启动 JVM 并找到入口类，`jar` 把一组 class 和资源打包成可分发单元，`jcmd`、`jstack`、`jmap`、`jfr` 等工具则负责在程序运行后观察它的状态。可以把它们理解为“建造工具、启动器、打包箱、监控仪表”。

classpath 和 module path 是工具链里最容易抽象的部分。它们本质上都是“类和依赖的搜索地图”：JVM 启动后需要知道到哪里找 `com.example.App`，也要知道到哪里找第三方库里的 class。classpath 像一条按顺序查找的路径列表，路径顺序和版本冲突会直接影响最终加载到哪个类；module path 则在此基础上加入模块边界，明确哪些包能被外部访问，哪些包只允许反射打开。

因此，环境问题不能只靠“重装 JDK”解决。真正的排查思路是：当前命令使用的是哪个 JDK、源码按哪个版本编译、运行时能不能找到 class、找到的是不是预期版本、JVM 启动后能否用诊断工具看到进程内部状态。

## 一、安装和验证

验证 JDK：

```bash
java --version
javac --version
```

期望看到 21.x：

```text
java 21.x.x ...
javac 21.x.x
```

> **易错：** 机器上可能安装多个 JDK。`java --version` 和 IDE 使用的 JDK 必须确认一致。

## 二、环境变量

常见变量：

| 变量 | 作用 |
| --- | --- |
| `JAVA_HOME` | 指向 JDK 安装目录 |
| `PATH` | 包含 JDK 的 `bin` 目录 |
| `CLASSPATH` | 老式类路径变量，不建议全局设置 |

建议：

- 设置 `JAVA_HOME`。
- 把 `%JAVA_HOME%\bin` 或 `$JAVA_HOME/bin` 加入 `PATH`。
- 不要随意设置全局 `CLASSPATH`，项目应由构建工具管理依赖。

## 三、编译和运行

`Hello.java`：

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, JDK 21");
    }
}
```

编译：

```bash
javac Hello.java
```

运行：

```bash
java Hello
```

> **重点：** 文件名必须和 `public class Hello` 的类名一致。

## 四、包和目录

`src/com/example/App.java`：

```java
package com.example;

public class App {
    public static void main(String[] args) {
        System.out.println("App");
    }
}
```

编译到 `out`：

```bash
javac -d out src/com/example/App.java
```

运行：

```bash
java -cp out com.example.App
```

> **易错：** 有 package 的类不能直接 `java App`，必须使用完整类名。

## 五、classpath

classpath 是 JVM 查找 class 和 jar 的路径。

```bash
java -cp out;lib/foo.jar com.example.App
```

Linux / macOS 使用 `:`：

```bash
java -cp out:lib/foo.jar com.example.App
```

Windows 使用 `;`：

```bash
java -cp out;lib/foo.jar com.example.App
```

> **难点：** classpath 冲突会导致 `ClassNotFoundException`、`NoClassDefFoundError` 或版本不一致问题。

## 六、jar

打包：

```bash
jar --create --file app.jar -C out .
```

查看：

```bash
jar --list --file app.jar
```

运行有 Main-Class 的 jar：

```bash
java -jar app.jar
```

创建 manifest：

```text
Main-Class: com.example.App
```

打包：

```bash
jar --create --file app.jar --manifest MANIFEST.MF -C out .
```

## 七、JShell

JShell 是 Java REPL，适合快速试验 API：

```bash
jshell
```

示例：

```java
var list = java.util.List.of("a", "b", "c");
list.stream().map(String::toUpperCase).toList();
```

退出：

```text
/exit
```

## 八、source、target、release

`javac` 常见参数：

```bash
javac --release 21 App.java
```

`--release` 同时控制语言级别和可用标准库 API。

> **重点：** 使用 `--release` 比单独设置 `-source` / `-target` 更安全。

## 九、模块路径

JDK 9 引入模块系统。传统依赖在 classpath，模块化应用可以使用 module path。

```bash
javac --module-path mods -d out --module com.example
java --module-path out -m com.example/com.example.App
```

普通业务项目不一定需要一开始就模块化，但要理解模块系统对 JDK 内部 API 访问的限制。

## 十、常用诊断工具

| 工具 | 作用 |
| --- | --- |
| `jps` | 查看 Java 进程 |
| `jcmd` | 多功能诊断入口 |
| `jstack` | 线程栈 |
| `jmap` | 堆信息 |
| `jstat` | GC / 类加载统计 |
| `jfr` | Java Flight Recorder |
| `jdeps` | 依赖分析 |
| `jlink` | 生成定制运行时 |
| `jpackage` | 打包原生安装包 |

例子：

```bash
jps
jcmd <pid> VM.version
jcmd <pid> Thread.print
jcmd <pid> GC.heap_info
```
