# Java 学习资料：基础语法、类型系统和运算

[返回索引](../Java学习资料.md)

## 学习目标

完成本章后，你应该能：

- 掌握 Java 基础语法、控制流和方法定义。
- 区分基本类型和引用类型。
- 理解自动装箱、拆箱、类型转换和数值溢出。
- 正确使用 `var`、数组、字符串和作用域。

## 理论导读：语法是表层，类型和内存位置才是骨架

Java 基础语法看起来像一组写法规则，但背后真正支撑程序的是“类型”和“值存放在哪里”。基本类型变量直接保存数值，例如 `int n = 10` 里的变量槽保存的是整数值；引用类型变量保存的是通往对象的引用，例如 `String name = "Alice"` 中，变量保存的不是整段字符串对象本身，而是能找到该对象的一份引用。理解这一点，`==`、`equals`、参数传递、数组、空指针都会变得清晰。

可以把 Java 程序想象成一张桌子和一间仓库：局部变量像桌面上的标签或小盒子，基本类型的小盒子里直接放值；引用类型的小盒子里放的是仓库货架编号，对象本体在堆这间仓库中。把一个引用赋给另一个变量，相当于复制货架编号，不是复制整个对象。方法调用传参也是复制一份值：基本类型复制数值，引用类型复制引用编号，所以方法能改对象内部状态，却不能让调用方变量自动指向另一个对象。

运算规则也是类型系统的一部分。`byte + byte` 会提升为 `int`，整数除法会丢掉小数部分，整数溢出默认绕回而不是抛异常。这些行为不是代码示例里的小细节，而是 Java 为了稳定、明确和高效执行而固定下来的语言规则。

## 一、程序结构

Java 程序结构像一份分层归档的文件：`package` 决定这份代码属于哪个命名空间，`class` 是程序的基本组织单位，方法是可执行动作，语句是动作里的具体步骤。JVM 启动命令行程序时，会寻找符合约定的 `main` 方法作为入口；没有入口方法，类仍然可以被其他代码使用，但不能直接作为命令行应用启动。

包名和目录结构不是纯粹的形式要求。它们帮助 JVM、构建工具和 IDE 定位类，也避免不同项目里同名类互相冲突。`com.example.App` 这个完整类名，才是 JVM 眼里的真实身份，文件名只是源码层面的载体。

```java
package com.example;

public class App {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
```

说明：

- `package` 声明包名。
- `public class App` 定义公共类。
- `main` 是命令行程序入口。
- Java 语句以分号结束。

## 二、基本类型

基本类型是 Java 里最直接的值，它们不需要对象包装就能参与计算。可以把它们理解成放在变量槽里的小纸条：`int` 纸条上直接写着整数，`boolean` 纸条上直接写着真假。它们的大小和取值范围固定，这让编译器和 JVM 能用稳定方式处理数值运算。

基本类型的代价是表达能力有限。`int` 只能表达固定范围的整数，溢出时不会自动变成更大的类型；`double` 适合科学计算，但不能精确表达十进制金额；`char` 是 UTF-16 代码单元，不等于所有人类字符。学习基本类型时，重点不是背 8 个名字，而是知道每种类型适合表达什么、不能表达什么。

Java 有 8 个基本类型：

| 类型 | 位数 | 示例 |
| --- | --- | --- |
| `byte` | 8 | `byte b = 1;` |
| `short` | 16 | `short s = 2;` |
| `int` | 32 | `int n = 10;` |
| `long` | 64 | `long x = 10L;` |
| `float` | 32 | `float f = 1.0f;` |
| `double` | 64 | `double d = 1.0;` |
| `char` | 16 | `char c = 'A';` |
| `boolean` | JVM 未规定具体位数 | `boolean ok = true;` |

> **重点：** `String` 不是基本类型，它是引用类型。

## 三、引用类型

引用类型表示“变量指向某个对象”。变量本身不是对象，只保存能找到对象的引用。把引用赋给另一个变量时，复制的是这个“指向关系”，不是复制对象本体。于是两个变量可能指向同一个对象，一个变量通过方法修改对象状态，另一个变量随后也能观察到变化。

这就是 `==` 和 `equals` 容易混淆的根源。`==` 问的是“两个引用是不是指向同一个对象”，`equals` 问的是“两个对象在业务意义上是否相等”。如果把两个问题混在一起，字符串、集合 key、值对象比较都会出错。

引用类型包括：

- 类
- 接口
- 数组
- enum
- record
- annotation

```java
String name = "Alice";
int[] numbers = {1, 2, 3};
```

引用变量保存的是对象引用，不是对象本身。

> **易错：** `==` 比较引用是否指向同一对象，字符串内容比较应使用 `equals`。

```java
String a = new String("hi");
String b = new String("hi");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true
```

## 四、变量和 var

变量是给某个值或引用起名字。Java 是静态类型语言，所以变量一旦声明类型，后续只能保存兼容类型的值。`var` 只是让编译器根据右侧初始化表达式替你写出局部变量类型，它不会让变量变成 JavaScript 那样的动态类型。

使用 `var` 的判断标准是“右侧是否足够清楚”。`var users = new ArrayList<User>()` 可读性通常还可以，因为右侧已经暴露类型；`var result = service.call()` 就可能隐藏太多信息，读者需要跳转方法定义才能知道它是什么。`var` 的目标是减少重复，不是牺牲语义。

JDK 10 引入局部变量类型推断：

```java
var name = "Alice";
var count = 10;
var users = new java.util.ArrayList<String>();
```

限制：

- 只能用于局部变量。
- 必须能从初始化表达式推断类型。
- 不能用于字段、方法参数、返回值。

> **易错：** `var` 不是动态类型。变量类型在编译期已经确定。

## 五、运算符和数值提升

运算符背后有一套固定的类型提升规则。Java 不会因为你把结果赋给 `byte`，就自动认为 `byte + byte` 仍然是 `byte`；小整数参与算术运算时会先提升到 `int`，这是为了让运算规则简单一致。强制类型转换像把大盒子里的东西硬塞回小盒子，能通过编译，但可能丢失信息。

整数溢出尤其需要警惕。`Integer.MAX_VALUE + 1` 不会变成更大的数，也不会抛异常，而是绕回最小值。金额、库存、计数器、分页偏移这些业务值如果可能接近边界，就不能只靠默认整数运算。浮点运算则要记住：`double` 表达的是二进制近似值，不适合精确金额。

整数运算默认至少提升到 `int`：

```java
byte a = 1;
byte b = 2;
// byte c = a + b; // 编译错误，a + b 是 int
byte c = (byte) (a + b);
```

整数除法：

```java
System.out.println(5 / 2);   // 2
System.out.println(5 / 2.0); // 2.5
```

溢出：

```java
int max = Integer.MAX_VALUE;
System.out.println(max + 1); // -2147483648
```

> **重点：** Java 整数溢出默认不会抛异常。

## 六、控制流

控制流决定程序的执行路线。`if` 像岔路口，根据条件选择一条路；`for` 和 `while` 像重复执行的任务清单，直到计数或条件结束；增强 `for` 更像“逐个处理集合中的元素”，适合不关心下标的遍历。控制流写得好，业务规则会像流程图一样清楚；控制流写得过深，代码会变成难以验证的嵌套迷宫。

选择循环结构时要看意图：已知次数通常用普通 `for`，遍历集合通常用增强 `for` 或 Stream，需要等待某个状态变化时用 `while`。不要为了炫技把简单判断写成复杂表达式，基础语法的第一目标是让条件和分支边界一眼可读。

if：

```java
if (score >= 90) {
    System.out.println("A");
} else if (score >= 60) {
    System.out.println("Pass");
} else {
    System.out.println("Fail");
}
```

for：

```java
for (int i = 0; i < 10; i++) {
    System.out.println(i);
}
```

增强 for：

```java
for (String item : java.util.List.of("a", "b")) {
    System.out.println(item);
}
```

while：

```java
while (running) {
    doWork();
}
```

## 七、switch 表达式

现代 `switch` 不只是传统分支语句，它也可以是“根据输入产生一个值”的表达式。表达式形式要求每条路径都有结果，这能减少忘记赋值、漏掉 `break`、分支不完整等老式 `switch` 常见错误。它更适合表达枚举、状态码、类型分类、命令分发这类“有限输入到结果”的映射。

理解 `switch` 表达式时，可以把它看成一张规则表：输入落在哪个 case，就产出对应结果。若某个 case 内需要多步逻辑，就用代码块并通过 `yield` 明确交出结果。清晰的 `switch` 应该让读者看到“覆盖了哪些情况，还有没有默认兜底”。

现代 switch：

```java
String type = switch (score / 10) {
    case 10, 9 -> "A";
    case 8 -> "B";
    case 7 -> "C";
    default -> "D";
};
```

代码块返回：

```java
String result = switch (code) {
    case 200 -> "OK";
    case 500 -> {
        logError();
        yield "SERVER_ERROR";
    }
    default -> "UNKNOWN";
};
```

> **重点：** switch 表达式必须覆盖所有可能路径，并产生值。

## 八、数组

数组是固定长度、同类型元素的连续容器。它像一排编号储物格，创建时格子数量就确定了，后续只能替换格子里的内容，不能让这排格子自动变长。按下标访问很快，但下标必须在 `0` 到 `length - 1` 之间，越界就是明确错误。

数组适合大小稳定、强调性能或和底层 API 交互的场景；日常业务里如果需要频繁增删，`ArrayList` 更自然。多维数组本质上是“数组里放数组”，每一行长度甚至可以不同，所以不要简单把它想成一定规整的数学矩阵。

```java
int[] numbers = {1, 2, 3};
System.out.println(numbers.length);
System.out.println(numbers[0]);
```

多维数组：

```java
int[][] matrix = {
    {1, 2},
    {3, 4}
};
```

> **易错：** 数组长度固定，集合 `ArrayList` 才适合动态增删。

## 九、方法

方法是把一段有名字、有输入、有输出的行为封装起来。参数是调用方交给方法的值，返回值是方法交还给调用方的结果。好的方法应该像一个清楚的小任务：名字说明做什么，参数说明需要什么，返回值说明产出什么，异常说明失败边界。

Java 参数传递永远是值传递。基本类型传的是数值副本，引用类型传的是引用副本。引用副本仍然指向同一个对象，所以方法内部能修改对象状态；但如果方法内部让参数变量指向新对象，外部变量不会跟着改。这个规则解释了大量“为什么方法里改了对象有效，但重新赋值无效”的问题。

```java
public static int add(int a, int b) {
    return a + b;
}
```

可变参数：

```java
public static int sum(int... numbers) {
    int total = 0;
    for (int n : numbers) {
        total += n;
    }
    return total;
}
```

> **易错：** Java 参数传递都是值传递。对象参数传递的是“引用的副本”。

```java
static void rename(User user) {
    user.setName("Bob");
}
```

这里能修改对象内部状态，但不能让调用方变量指向新对象。

## 十、作用域和生命周期

作用域决定名字在哪里可见，生命周期决定值或对象能存在多久。局部变量只在当前代码块内可见，方法结束后局部变量槽会被释放；对象如果仍然被其他引用可达，就可以继续存在于堆中，否则未来可能被 GC 回收。字段则属于对象或类，生命周期通常比局部变量更长。

作用域越大，变量越容易被误用。把变量限制在最小必要范围内，能减少状态混乱，也能让读者更快判断它在哪里被修改。`static` 字段尤其要谨慎，因为它属于类级共享状态，在测试、并发和多请求环境中都可能带来隐藏耦合。

局部变量只在代码块内有效：

```java
if (true) {
    int x = 1;
}
// System.out.println(x); // 编译错误
```

字段属于对象或类：

```java
class Counter {
    private int value;
    private static int total;
}
```

## 练习

写一个命令行成绩评级程序：

- 输入分数。
- 使用 switch 表达式输出等级。
- 检查非法分数。
- 使用方法封装评级逻辑。

示例：

```java
static String grade(int score) {
    if (score < 0 || score > 100) {
        return "INVALID";
    }
    return switch (score / 10) {
        case 10, 9 -> "A";
        case 8 -> "B";
        case 7 -> "C";
        case 6 -> "D";
        default -> "F";
    };
}
```
