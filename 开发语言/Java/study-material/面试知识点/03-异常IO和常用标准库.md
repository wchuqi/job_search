## 一、异常

### 22. checked exception 和 unchecked exception 有什么区别？

**参考答案：**

checked exception 必须捕获或声明抛出，例如 IOException。unchecked exception 继承 RuntimeException，不强制处理，通常表示编程错误或运行期状态问题。

### 23. try-with-resources 解决什么问题？

**参考答案：**

它自动关闭实现 AutoCloseable 的资源，避免文件、网络、数据库连接泄漏。

```java
try (var reader = Files.newBufferedReader(path)) {
    return reader.readLine();
}
```

### 24. 为什么不要吞异常？

**参考答案：**

吞异常会隐藏真实错误，导致数据不一致、排障困难。至少应补充上下文后抛出，或在边界层记录日志并返回明确错误。

## 二、IO 和标准库

### 25. 字节流和字符流有什么区别？

**参考答案：**

字节流处理原始字节，适合图片、二进制文件。字符流处理字符，适合文本，需要明确字符编码。

### 26. NIO 的 Path / Files 相比 File 有什么优势？

**参考答案：**

Path / Files API 更现代，支持更丰富的文件操作、符号链接、文件属性、遍历和异常信息，是新代码优先选择。

### 27. BigDecimal 为什么不能用 `new BigDecimal(0.1)`？

**参考答案：**

因为 0.1 作为 double 已经是不精确的二进制浮点值。应使用字符串或 `BigDecimal.valueOf`。

```java
new BigDecimal("0.1");
BigDecimal.valueOf(0.1);
```

### 28. java.time 相比 Date / Calendar 有什么优势？

**参考答案：**

java.time 类型不可变、线程安全、语义清晰，区分日期、时间、时区、时间段，API 更易用。

### 29. Java 原生序列化有什么风险？

**参考答案：**

反序列化不可信输入可能触发恶意对象链，造成远程代码执行等安全问题。生产中通常使用 JSON、Protobuf 等，并限制类型和输入大小。
