## 一、JVM 基础

### 40. JVM 运行时内存区域有哪些？

**参考答案：**

包括程序计数器、Java 虚拟机栈、本地方法栈、堆、方法区/元空间、直接内存。栈和程序计数器线程私有，堆和元空间线程共享。

### 41. 类加载过程是什么？

**参考答案：**

加载、验证、准备、解析、初始化。类加载器通常遵循双亲委派模型。

### 42. 双亲委派模型有什么作用？

**参考答案：**

类加载器先委托父加载器加载，避免核心类被篡改，并保证类加载一致性。

## 二、GC

### 43. Java 如何判断对象可以回收？

**参考答案：**

HotSpot 主要使用可达性分析。从 GC Roots 出发不可达的对象可以被回收。

### 44. GC Roots 包括哪些？

**参考答案：**

包括栈中的引用、静态字段引用、常量引用、JNI 引用等。

### 45. G1 和 ZGC 怎么理解？

**参考答案：**

G1 是平衡吞吐和停顿的通用 GC，JDK 21 默认常用。ZGC 关注低延迟，适合大堆和低停顿要求场景，但需要结合业务和资源评估。

## 三、排障

### 46. CPU 飙高怎么排查？

**参考答案：**

找 Java 进程，定位高 CPU 线程，把线程 id 转为十六进制，在线程 dump 中找到对应线程栈，分析热点代码。

### 47. 内存泄漏怎么排查？

**参考答案：**

观察 GC 日志和堆使用趋势，导出 heap dump，分析对象保留路径和大对象来源，定位未释放的集合、缓存、监听器、ThreadLocal 等。

### 48. 死锁怎么排查？

**参考答案：**

使用 `jcmd <pid> Thread.print` 或 `jstack <pid>` 查看线程 dump，JVM 通常会标记 deadlock 信息。

### 49. JFR 有什么作用？

**参考答案：**

Java Flight Recorder 是低开销运行时事件记录工具，可分析 CPU、内存、锁、线程、IO、GC 等问题。
