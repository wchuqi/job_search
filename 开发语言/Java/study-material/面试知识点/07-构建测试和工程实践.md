## 一、构建和依赖

### 57. Maven 的标准目录结构是什么？

**参考答案：**

`src/main/java` 放主代码，`src/main/resources` 放资源，`src/test/java` 放测试，`src/test/resources` 放测试资源，`pom.xml` 管理项目信息和依赖。

### 58. Maven dependency management 有什么作用？

**参考答案：**

它用于集中管理依赖版本，子模块引用依赖时可不重复声明版本，保证多模块版本一致。

### 59. 如何排查依赖冲突？

**参考答案：**

使用 `mvn dependency:tree` 查看依赖树，确认版本来源，通过 dependencyManagement、exclusion 或升级依赖解决冲突。

## 二、测试

### 60. 单元测试和集成测试有什么区别？

**参考答案：**

单元测试聚焦小范围逻辑，速度快，外部依赖通常 mock。集成测试验证多个组件协作，可能访问数据库、容器或真实框架。

### 61. Mockito 适合 mock 什么？

**参考答案：**

适合 mock 外部依赖、仓储接口、远程客户端等。不要过度 mock 领域对象，否则测试只验证 mock 行为而非真实业务。

## 三、日志和质量

### 62. 日志为什么用占位符而不是字符串拼接？

**参考答案：**

占位符可避免在日志级别未开启时提前构造字符串，也能保持结构清晰。

```java
log.info("order created, orderId={}", orderId);
```

### 63. CI 中 Java 项目通常跑哪些步骤？

**参考答案：**

检出代码、设置 JDK 21、缓存依赖、编译、单元测试、静态检查、覆盖率、安全扫描、打包和发布制品。
