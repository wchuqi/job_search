## 一、并发基础

### 30. 什么是线程安全？

**参考答案：**

多个线程同时访问同一对象或数据时，如果不需要额外同步也能得到正确结果，就称为线程安全。关键问题通常来自共享可变状态。

### 31. volatile 和 synchronized 有什么区别？

**参考答案：**

volatile 保证可见性和有序性，不保证复合操作原子性。synchronized 同时提供互斥和可见性，适合保护临界区。

> **易错：** `volatile int count; count++` 不是线程安全的。

### 32. CAS 是什么？

**参考答案：**

CAS 是 Compare-And-Swap，比较内存中的旧值是否符合预期，符合则更新。Atomic 类常基于 CAS 实现无锁原子操作。

### 33. happens-before 是什么？

**参考答案：**

happens-before 是 JMM 中的可见性规则。如果 A happens-before B，那么 A 的结果对 B 可见。常见规则包括 volatile 写先于后续读、解锁先于后续加锁、start 先于新线程动作。

## 二、线程池

### 34. ThreadPoolExecutor 核心参数有哪些？

**参考答案：**

核心线程数、最大线程数、空闲线程存活时间、任务队列、线程工厂、拒绝策略。

### 35. 为什么不建议盲目使用 Executors 默认工厂方法？

**参考答案：**

某些默认方法可能使用无界队列或无界线程数，极端情况下导致 OOM 或资源耗尽。生产中应明确线程数、队列大小和拒绝策略。

### 36. CompletableFuture 适合什么场景？

**参考答案：**

适合异步任务编排、结果转换、并行任务组合和异常处理。但阻塞任务应使用明确 Executor，避免污染 common pool。

## 三、虚拟线程

### 37. JDK 21 虚拟线程解决什么问题？

**参考答案：**

虚拟线程降低线程创建和阻塞成本，让阻塞式 IO 代码也能承载大量并发。它适合 IO 密集任务，不是 CPU 加速器。

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> callRemoteApi());
}
```

### 38. 虚拟线程不适合什么场景？

**参考答案：**

不适合 CPU 密集计算加速，也不适合用线程数限制外部资源的旧模型。数据库连接、下游服务并发仍需要用连接池、信号量或限流控制。

### 39. ThreadLocal 在虚拟线程中要注意什么？

**参考答案：**

虚拟线程数量可能非常多，滥用 ThreadLocal 会增加内存压力。上下文传递应谨慎，必要时关注 Scoped Values 等现代机制。
