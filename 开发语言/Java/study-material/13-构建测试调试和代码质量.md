# Java 学习资料：构建、测试、调试和代码质量

[返回索引](../Java学习资料.md)

## 学习目标

完成本章后，你应该能：

- 使用 Maven / Gradle 管理 Java 项目。
- 编写 JUnit 5 单元测试。
- 使用 Mockito 构造隔离测试。
- 掌握日志、调试、远程调试和代码质量工具。
- 理解 CI 中的 Java 构建流程。

## 理论导读：工程化是在保证代码能被持续交付和验证

写出一段能运行的 Java 代码，只是工程的起点。真实项目还要保证它能稳定编译、依赖版本可控、测试能自动执行、问题能被定位、质量规则能被持续检查、构建产物能在不同环境中一致运行。Maven、Gradle、JUnit、Mockito、日志、调试器、静态检查、CI 共同构成了 Java 工程的质量护栏。

构建工具像项目的装配说明书：源码在哪里、依赖从哪里来、用哪个 JDK 编译、测试怎么跑、最终打成什么包，都应该由构建脚本明确表达。测试像可重复执行的验收清单，单元测试验证小块逻辑，集成测试验证组件协作，端到端测试验证完整流程。Mockito 这类 mock 工具不是为了让测试“看起来隔离”，而是为了切断数据库、网络、消息队列等不稳定外部依赖。

调试和日志则负责运行期理解。断点适合开发阶段观察变量和调用栈，日志适合线上留下关键事件轨迹，JFR 和诊断工具适合分析性能。代码质量工具的目标不是追求形式完美，而是提前发现重复、空指针风险、未覆盖分支、危险依赖和不可维护结构。

## 一、Maven 项目结构

```text
project/
  pom.xml
  src/main/java/
  src/main/resources/
  src/test/java/
  src/test/resources/
```

常用命令：

```bash
mvn test
mvn package
mvn clean verify
```

JDK 21 编译配置：

```xml
<properties>
    <maven.compiler.release>21</maven.compiler.release>
</properties>
```

## 二、Gradle 简介

`build.gradle.kts`：

```kotlin
plugins {
    java
}

java {
    toolchain {
        languageVersion.set(JavaLanguageVersion.of(21))
    }
}
```

常用命令：

```bash
./gradlew test
./gradlew build
```

## 三、JUnit 5

```java
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.assertEquals;

class CalculatorTest {
    @Test
    void addShouldReturnSum() {
        assertEquals(3, new Calculator().add(1, 2));
    }
}
```

生命周期：

```java
@BeforeEach
void setUp() {}

@AfterEach
void tearDown() {}
```

参数化测试：

```java
@ParameterizedTest
@CsvSource({"1,2,3", "2,3,5"})
void add(int a, int b, int expected) {
    assertEquals(expected, calculator.add(a, b));
}
```

## 四、Mockito

```java
UserRepository repository = mock(UserRepository.class);
when(repository.findName(1L)).thenReturn("Alice");

UserService service = new UserService(repository);
assertEquals("Alice", service.findName(1L));

verify(repository).findName(1L);
```

> **易错：** 不要过度 mock。领域逻辑优先用真实对象测试，外部依赖再 mock。

## 五、测试分层

| 类型 | 目标 |
| --- | --- |
| 单元测试 | 小范围逻辑 |
| 集成测试 | 多组件协作 |
| 契约测试 | 服务接口约定 |
| 端到端测试 | 完整流程 |
| 性能测试 | 响应时间和吞吐 |

## 六、日志

常用组合：

- SLF4J API
- Logback / Log4j2 实现

示例：

```java
private static final Logger log = LoggerFactory.getLogger(OrderService.class);

log.info("order created, orderId={}", orderId);
log.error("create order failed, userId={}", userId, e);
```

> **重点：** 日志中不要输出密码、token、身份证号等敏感信息。

## 七、调试

IDE 调试：

- 断点
- 条件断点
- 表达式求值
- 调用栈
- 线程视图

远程调试：

```bash
java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005 -jar app.jar
```

> **易错：** 生产环境开放远程调试端口有安全风险。

## 八、代码质量工具

常见工具：

- Checkstyle：代码风格。
- SpotBugs：静态缺陷。
- PMD：代码规则。
- Error Prone：编译期缺陷。
- JaCoCo：测试覆盖率。
- SonarQube：综合质量平台。

## 九、依赖管理

注意：

- 依赖版本冲突。
- 传递依赖。
- scope。
- dependency management。
- 依赖漏洞。

Maven 查看依赖树：

```bash
mvn dependency:tree
```

## 十、CI 流程

典型流程：

```text
checkout -> setup JDK 21 -> cache dependencies -> test -> package -> static analysis -> publish artifact
```

CI 必须执行：

- 编译
- 单元测试
- 静态检查
- 安全扫描，若需要
- 打包
