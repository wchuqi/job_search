## 理论导读：把 Java 想成一条完整流水线

Java 的学习入口不是某个关键字，而是一条“设计图到工厂生产”的流水线。`.java` 源码像人能读懂的设计图，`javac` 把它翻译成 `.class` 字节码，字节码像一份平台无关的生产指令，JVM 才是真正执行指令的工厂。不同操作系统上的 JVM 负责把同一份字节码解释或编译成本机机器码，所以 Java 的跨平台本质是“统一字节码 + 各平台 JVM 适配”，不是源码天然能在任何机器上直接运行。

这条流水线里有几条主线会贯穿所有章节。类型系统负责在编译期尽早拦住错误，就像施工前检查图纸尺寸；对象模型负责组织状态和行为，就像把零件封装成有规则的设备；JVM 运行时负责类加载、内存分配、方法调用、线程协作和垃圾回收，就像工厂里的仓库、生产线、调度系统和清理系统。后面学泛型、集合、并发、JVM、Spring 时，很多问题都能回到这几条主线上解释。

初学时最容易犯的错，是把 Java 看成“语法清单”：会写 `class`、`if`、`for`、`new`，却不知道对象到底在哪里、引用保存了什么、方法调用时栈帧如何变化、为什么集合依赖 `equals` 和 `hashCode`。本资料后续章节会不断把代码放回这张运行图里理解。

## Java 是什么

Java 是一门静态类型、面向对象、支持泛型、函数式风格和并发编程的通用编程语言。Java 程序通常编译为字节码，在 JVM 上运行。

```text
Java 源码 (.java)
  -> javac 编译
字节码 (.class)
  -> JVM 加载、验证、解释执行或 JIT 编译
机器码
```

> **重点：** Java 的跨平台不是源码直接跨平台，而是字节码运行在不同平台的 JVM 上。

## JDK、JRE、JVM

| 名称 | 含义 | 作用 |
| --- | --- | --- |
| JVM | Java Virtual Machine | 运行字节码 |
| JRE | Java Runtime Environment | 运行 Java 程序所需环境 |
| JDK | Java Development Kit | 开发工具包，包含编译器、运行工具和标准库 |

JDK 21 中，开发时通常只安装 JDK。传统单独安装 JRE 的场景已经不再是主流。

## JDK 21 的定位

JDK 21 是 LTS 版本，适合作为长期学习和生产基准。它包含许多现代 Java 能力：

- var 局部变量类型推断
- switch 表达式
- text blocks
- records
- sealed classes
- pattern matching for instanceof
- pattern matching for switch
- virtual threads
- sequenced collections

> **易错：** JDK 21 不是 Java 8 的简单升级。现代 Java 写法在数据建模、并发和模式匹配上已经明显不同。

## Java 学习的核心主线

### 1. 类型系统

Java 是静态类型语言，变量、表达式、方法参数和返回值都有类型。类型系统决定了编译期能发现多少错误。

必须理解：

- 基本类型和引用类型
- 自动装箱和拆箱
- 泛型和类型擦除
- 继承、接口、多态
- record、sealed、模式匹配

### 2. 对象模型

Java 使用类描述对象结构和行为。对象通常分配在堆上，引用保存在变量、字段、数组或栈帧中。

必须理解：

- 类和对象
- 字段和方法
- 构造器
- this / super
- static
- equals / hashCode

### 3. 运行时模型

Java 程序运行在 JVM 中。JVM 负责：

- 类加载
- 字节码验证
- 解释执行
- JIT 编译
- 内存管理
- 垃圾回收
- 线程调度协作

### 4. 标准库和生态

Java 的工程能力来自语言本身 + 标准库 + 生态工具：

- 集合框架
- 并发库
- IO / NIO
- java.time
- JDBC
- Maven / Gradle
- JUnit
- Spring / Spring Boot

## 学习建议

- 不要跳过基础类型和对象模型，否则后续泛型、集合、JVM 都会不稳。
- 不要只学语法，要写可运行的小程序。
- 学并发前先理解对象共享和可变状态。
- 学 JVM 前先理解类、对象、方法调用和异常。
- 学 Spring 前先掌握 Java 语言、集合、反射、注解和异常。

> **重点：** Java 后端开发不是只会 Spring Boot。语言基础、JVM、并发、数据库、测试和工程规范同样关键。

## 练习

创建一个最小 Java 文件：

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, JDK 21");
    }
}
```

运行：

```bash
javac Hello.java
java Hello
```

回答：

- `javac` 产生了什么文件？
- JVM 运行的是 `.java` 还是 `.class`？
- `main` 方法为什么必须是 `public static void`？
