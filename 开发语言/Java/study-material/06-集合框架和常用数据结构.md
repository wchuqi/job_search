## 学习目标

完成本章后，你应该能：

- 根据场景选择 List、Set、Map、Queue、Deque。
- 理解 ArrayList、LinkedList、HashMap、TreeMap、HashSet 的核心特性。
- 正确实现对象的 `equals` 和 `hashCode`。
- 理解不可变集合、迭代器、fail-fast、排序和比较器。
- 掌握 JDK 21 Sequenced Collections。

## 理论导读：集合是在不同访问需求之间做取舍

集合框架不是“背 API 名称”，而是在不同数据组织方式之间选择合适的容器。`List` 像一排有编号的座位，适合按位置访问和保持顺序；`Set` 像一个不允许重复入场的名单，重点是唯一性；`Map` 像字典或索引表，用 key 快速找到 value；`Queue` 和 `Deque` 则像排队通道，强调先进先出或两端进出。每种集合都在访问速度、插入删除、排序、去重、线程安全和内存成本之间做不同取舍。

底层结构决定使用边界。`ArrayList` 背后是连续数组，按下标访问快，但中间插入删除可能搬动大量元素；`LinkedList` 背后是节点串起来的链表，不需要搬动数组，但查找第 N 个元素要一路走过去；`HashMap` 依赖哈希定位桶，所以 key 的 `equals` 和 `hashCode` 错了，整个索引系统就会失灵；`TreeMap` 维护排序结构，所以能按顺序遍历，但单次操作通常比哈希表更贵。

学习集合时要把问题翻译成访问模式：是否需要顺序、是否允许重复、是否频繁按 key 查找、是否需要排序、是否会并发修改、数据量是否大到需要预估容量。选集合就是回答这些问题，而不是默认 `ArrayList` 或 `HashMap`。


| --- | --- |

## 二、List

### ArrayList

底层是动态数组。

适合：

- 频繁按索引访问。
- 追加较多。
- 中间插入删除较少。

```java
List<String> names = new ArrayList<>();
names.add("Alice");
names.add("Bob");
System.out.println(names.get(0));
```

### LinkedList

双向链表，同时实现 List 和 Deque。

> **易错：** LinkedList 不是“插入删除一定更快”。如果需要先查找位置，链表遍历成本很高。

## 三、Set

```java
Set<String> tags = new HashSet<>();
tags.add("java");
tags.add("java");
System.out.println(tags.size()); // 1
```

常见实现：

| 实现 | 特点 |
| --- | --- |
| `HashSet` | 基于 hash，不保证顺序 |
| `LinkedHashSet` | 保留插入顺序 |
| `TreeSet` | 排序集合 |

> **重点：** HashSet 判断重复依赖 `equals` 和 `hashCode`。

## 四、Map

```java
Map<String, Integer> score = new HashMap<>();
score.put("Alice", 90);
score.put("Bob", 80);
```

常见实现：

| 实现 | 特点 |
| --- | --- |
| `HashMap` | 常用，平均 O(1) |
| `LinkedHashMap` | 保留插入顺序或访问顺序 |
| `TreeMap` | 按 key 排序 |
| `ConcurrentHashMap` | 并发场景 |
| `EnumMap` | enum key 高效实现 |

常用 API：

```java
map.getOrDefault("missing", 0);
map.computeIfAbsent("key", k -> new ArrayList<>());
map.merge("count", 1, Integer::sum);
```

> **易错：** `HashMap` 不是线程安全的。并发写入应使用 `ConcurrentHashMap` 或外部同步。

## 五、Queue 和 Deque

```java
Deque<String> deque = new ArrayDeque<>();
deque.addLast("a");
deque.addLast("b");
System.out.println(deque.removeFirst());
```

建议：

- 栈：优先 `ArrayDeque`，不要用老的 `Stack`。
- 队列：`ArrayDeque` 或并发队列。

## 六、排序和比较

自然排序：

```java
record User(String name, int age) implements Comparable<User> {
    @Override
    public int compareTo(User other) {
        return Integer.compare(this.age, other.age);
    }
}
```

Comparator：

```java
users.sort(Comparator.comparing(User::age).thenComparing(User::name));
```

> **易错：** Comparator 必须满足传递性，否则排序结果可能异常。

## 七、不可变集合

```java
List<String> names = List.of("Alice", "Bob");
Set<String> tags = Set.of("java", "jdk21");
Map<String, Integer> ages = Map.of("Alice", 18);
```

复制为不可变集合：

```java
List<String> copy = List.copyOf(names);
```

> **重点：** `List.of()` 返回不可变集合，不能 add/remove。

## 八、迭代器和 fail-fast

错误示例：

```java
for (String item : list) {
    if (item.isBlank()) {
        list.remove(item); // 可能 ConcurrentModificationException
    }
}
```

正确方式：

```java
list.removeIf(String::isBlank);
```

或：

```java
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    if (iterator.next().isBlank()) {
        iterator.remove();
    }
}
```

## 九、JDK 21 Sequenced Collections

JDK 21 引入 Sequenced Collections，用统一 API 表达有首尾顺序的集合。

常见方法：

```java
sequenced.getFirst();
sequenced.getLast();
sequenced.reversed();
```

适用类型包括 `List`、`LinkedHashSet`、`LinkedHashMap` 等相关有序集合。

> **重点：** Sequenced Collections 解决了过去不同集合取首尾 API 不统一的问题。

## 十、容量和性能

HashMap 初始容量：

```java
Map<String, Integer> map = new HashMap<>(128);
```

ArrayList 初始容量：

```java
List<String> list = new ArrayList<>(1000);
```

> **易错：** 数据量大时不设置合理容量，可能导致多次扩容和 rehash。

## 练习

实现一个词频统计器：

- 读取字符串列表。
- 使用 `Map<String, Integer>` 统计出现次数。
- 输出按次数降序、单词升序排序的结果。

示例：

```java
Map<String, Integer> counts = new HashMap<>();
for (String word : words) {
    counts.merge(word, 1, Integer::sum);
}
```
