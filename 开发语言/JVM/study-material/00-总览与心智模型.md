# JVM 学习资料：总览与心智模型

[返回索引](../JVM学习资料.md)

## 学习目标

- 建立 JVM 的整体运行图景。
- 区分 JVM 规范、HotSpot 实现、JDK 工具和 Java 语言。
- 明白 JVM 学习的主线：加载、执行、内存、回收、优化、诊断。

## 理论导读

JVM 可以想象成 Java 程序的运行工厂。`.class` 文件是标准化的半成品，类加载器负责把它们搬进工厂，字节码验证器检查安全和格式，解释器先让程序启动，JIT 把热点路径加工成机器码，堆和栈承载运行状态，GC 清理不再可达的对象，诊断工具把内部状态暴露出来。

学习 JVM 不应该只背“堆、栈、GC”。真正有用的能力是看到一个现象能定位到运行链路中的位置：启动慢可能和类加载、JIT、IO、框架初始化有关；请求延迟抖动可能和 GC、锁竞争、线程池饱和有关；内存涨可能来自 Java 堆、直接内存、线程栈、元空间或 Native 内存。

## 核心心智模型

```text
源码 .java
  -> javac 编译
  -> 字节码 .class / jar
  -> ClassLoader 加载
  -> 验证、准备、解析、初始化
  -> 解释执行 + JIT 编译
  -> 堆/栈/元空间/直接内存承载状态
  -> GC 回收对象
  -> JFR、jcmd、GC 日志、dump 诊断问题
```

## 知识覆盖地图

| 领域 | 必学内容 | 容易漏掉的内容 |
| --- | --- | --- |
| 运行时结构 | JVM、JRE、JDK、HotSpot、字节码 | JVM 规范和 HotSpot 实现差异 |
| 内存模型 | 堆、栈、元空间、程序计数器 | 直接内存、Code Cache、线程栈、Native Memory |
| 类加载 | 双亲委派、初始化时机 | 类加载器命名空间、SPI、容器隔离、LinkageError |
| 字节码 | 局部变量表、操作数栈、调用指令 | `invokedynamic`、异常表、栈图帧 |
| GC | 可达性、分代、STW、G1 | 写屏障、记忆集、SATB、低延迟 GC |
| JIT | 热点探测、内联、逃逸分析 | 去优化、OSR、JMH、Profile 污染 |
| JMM | happens-before、volatile、锁 | 发布安全、final 语义、伪共享 |
| 工具 | jcmd、jstack、jmap、jstat、JFR | NMT、async-profiler、容器指标 |
| 生产 | 参数、容器、日志、监控 | cgroup 限制、OOM killer、dump 风险 |
| 面试 | 概念解释、场景分析 | 用证据链回答线上问题 |

## 例子

查看 JVM 基础信息：

```bash
java -version
java -XshowSettings:vm -version
jcmd <pid> VM.version
jcmd <pid> VM.flags
```

查看类加载：

```bash
java -Xlog:class+load=info -cp target/classes com.example.App
```

## 练习

1. 写一个 `HelloJvm`，编译后用 `javap -c HelloJvm` 查看字节码。
2. 启动程序时加 `-Xlog:class+load=info`，观察加载了哪些类。
3. 用 `jps -l` 找到进程，再用 `jcmd <pid> VM.command_line` 查看启动参数。

## 验收

- 能不用资料解释 Java 程序从源码到运行的路径。
- 能说清 JDK 工具、JVM 运行时、Java 语言语法之间的边界。
- 能把 CPU、内存、GC、类加载、死锁问题映射到对应章节。

## 重点

- JVM 是规范，HotSpot 是实现，JDK 是工具和类库集合。
- `.class` 字节码是 JVM 的输入，不只 Java 能生成字节码。
- 排障时先定位问题属于哪类资源，再选择工具。

## 难点

- 很多概念跨层：`volatile` 同时涉及 Java 语义、JIT 重排序、CPU 内存屏障。
- JVM 参数存在版本差异，线上参数必须结合目标 JDK 验证。

## 易错

> **易错：** 把“JVM 调优”理解为背参数。
>
> 正确做法：先定义目标和指标，再看证据，最后才调参数。

> **易错：** 把 Java 堆内存等同于进程全部内存。
>
> 正确做法：进程内存还包括线程栈、直接内存、元空间、Code Cache、JNI 和 libc 分配。
