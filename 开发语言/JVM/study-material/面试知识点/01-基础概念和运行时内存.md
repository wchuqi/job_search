## 一、基础概念和运行时内存

### 1. JDK、JRE、JVM 有什么区别？

**参考答案：**

JVM 是运行 Java 字节码的虚拟机规范和实现载体；JRE 是运行环境，包含 JVM 和运行时类库；JDK 是开发工具包，包含 JRE 以及 `javac`、`jar`、`jcmd`、`jlink` 等开发和诊断工具。现代 JDK 发布中，JRE 常不再作为单独发行重点，但概念边界仍然重要。

> **重点：** JVM 运行字节码，JDK 提供开发、编译和诊断工具。

### 2. JVM 运行时数据区有哪些？

**参考答案：**

主要包括程序计数器、Java 虚拟机栈、本地方法栈、Java 堆、方法区或元空间，以及实现层面的直接内存、Code Cache、Native Memory 等。线程私有的是程序计数器、Java 虚拟机栈、本地方法栈；线程共享的是堆和类元数据相关区域。

> **易错：** 面试只背“堆、栈、方法区”不够，生产排障必须补上直接内存、线程栈、Code Cache 和容器 RSS。

### 3. 栈帧里有什么？

**参考答案：**

每次方法调用都会创建栈帧，栈帧通常包含局部变量表、操作数栈、动态链接、方法返回地址等信息。局部变量表存放基本类型值和对象引用，操作数栈用于字节码指令计算。

> **重点：** 栈帧是方法调用现场，不是对象主要存放位置。

### 4. 堆 OOM、栈溢出、元空间 OOM 如何区分？

**参考答案：**

堆 OOM 常见为 `Java heap space`，通常是对象过多或泄漏；栈溢出常见为 `StackOverflowError`，多由递归过深或调用链过深导致；元空间 OOM 常见为 `Metaspace`，多和动态生成类、ClassLoader 泄漏或类过多有关。

排查入口：

```bash
jcmd <pid> GC.heap_info
jcmd <pid> GC.class_histogram
jcmd <pid> Thread.print
```

### 5. Java 对象一定在堆上吗？

**参考答案：**

从 Java 语义上可以把对象理解为在堆上分配；HotSpot JIT 可能通过逃逸分析、标量替换等优化消除实际对象分配。面试中不要简单说“对象都在栈上分配”，更准确的表达是：对象语义上是对象，具体机器层面分配可能被 JIT 优化掉。

> **难点：** 区分 Java 语义模型和 HotSpot 优化实现。

### 6. 直接内存是什么，为什么会 OOM？

**参考答案：**

直接内存是堆外内存，常由 NIO、Netty、`ByteBuffer.allocateDirect` 使用。它不计入 Java 堆 used，但计入进程 RSS 和容器内存。直接内存泄漏或上限过高可能导致 `Direct buffer memory` 或容器 OOMKilled。

> **易错：** heap dump 不一定能直接看到完整的直接内存占用，需要结合 NMT、RSS、框架池化指标判断。
