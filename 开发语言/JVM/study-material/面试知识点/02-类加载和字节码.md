## 一、类加载和字节码

### 1. 类加载过程有哪些阶段？

**参考答案：**

类加载生命周期包括加载、验证、准备、解析、初始化、使用、卸载。加载把字节流转换为运行时结构并生成 `Class` 对象；验证保证字节码安全；准备给静态变量分配内存并设置零值；解析把符号引用转为直接引用；初始化执行 `<clinit>`。

> **重点：** 准备阶段是零值，初始化阶段才执行 Java 静态赋值和 static 块。

### 2. 什么情况下会触发类初始化？

**参考答案：**

常见主动使用会触发初始化：创建实例、访问或设置非编译期常量静态字段、调用静态方法、反射要求初始化、初始化子类前初始化父类、JVM 启动主类。访问编译期常量、创建类数组通常不触发目标类初始化。

### 3. 双亲委派模型是什么？

**参考答案：**

类加载器收到加载请求后，先检查是否已加载，再委托父加载器加载，父加载器无法加载时才由自己加载。它保证核心类优先由高层加载器加载，降低核心类被篡改和重复加载风险。

> **易错：** SPI、Tomcat、插件系统可能调整委派模型，不能说所有场景都严格 parent-first。

### 4. 为什么同一个类名可能不能互相强转？

**参考答案：**

JVM 中类身份由“类加载器实例 + 全限定类名”共同决定。两个不同 ClassLoader 加载的同名类，在 JVM 看来是两个类型，因此可能出现 `ClassCastException: User cannot be cast to User`。

### 5. `ClassNotFoundException` 和 `NoClassDefFoundError` 有什么区别？

**参考答案：**

`ClassNotFoundException` 通常是显式按名称加载类时找不到，例如 `Class.forName`；`NoClassDefFoundError` 通常表示编译时存在，但运行时缺失、链接失败或初始化失败后再次使用。一个偏受检异常，一个偏错误。

### 6. `NoSuchMethodError` 常见原因是什么？

**参考答案：**

常见原因是编译期和运行期依赖版本不一致。编译时调用的方法在新版本存在，但运行时实际加载了旧版本 jar。排查应查看依赖树、启动 classpath、fat jar 内容和类加载日志。

```bash
java -Xlog:class+load=info -jar app.jar
mvn dependency:tree
```

### 7. `invokevirtual`、`invokestatic`、`invokespecial` 有什么区别？

**参考答案：**

`invokestatic` 调用静态方法；`invokespecial` 常用于构造器、私有方法、父类方法；`invokevirtual` 用于普通虚方法调用，需要根据接收者实际类型动态分派。接口调用使用 `invokeinterface`，动态语言和 Lambda 等常涉及 `invokedynamic`。

> **重点：** 方法调用指令体现了 JVM 层面的分派规则。
