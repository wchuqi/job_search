## 一、深度机制和高阶场景题

### 1. 为什么 Minor GC 需要写屏障和卡表？

**参考答案：**

年轻代 GC 需要知道是否有老年代对象引用年轻代对象。如果每次都扫描整个老年代，成本太高。写屏障在引用写入时把对应卡片标脏，Minor GC 只扫描脏卡片，降低扫描范围。代价是引用写入路径多一点记录成本。

> **重点：** 屏障和卡表是用写入开销换回收扫描效率。

### 2. 三色标记在并发 GC 中有什么问题？

**参考答案：**

并发标记时应用线程还在修改引用。如果黑色对象新增对白色对象的引用，而 GC 没记录到，白色对象可能被误回收。解决方式包括增量更新、SATB 等屏障策略，用来维持并发标记正确性。

### 3. G1 的 Mixed GC 为什么不是 Full GC？

**参考答案：**

Mixed GC 会回收年轻代 Region 和一部分经过收益评估的老年代 Region，目标是在停顿预算内回收价值高的区域。Full GC 通常是退化路径，可能对整个堆做更重的 STW 处理。二者停顿、触发原因和回收范围都不同。

### 4. 为什么低延迟 GC 需要更多内存余量？

**参考答案：**

低延迟 GC 大量工作与应用线程并发执行。在并发回收尚未完成时，应用仍会继续分配对象，因此需要足够余量避免分配速度超过回收速度。否则可能触发退化、分配停顿或 OOM。

### 5. 为什么类存在还会报 `NoSuchMethodError`？

**参考答案：**

这是典型二进制版本冲突。编译时依赖的类有该方法，运行时实际加载的同名类版本没有该方法。应检查依赖树、fat jar、容器公共库、类加载日志和 Maven/Gradle 版本冲突。

### 6. 线上内存涨但 heap dump 看不出大对象，怎么办？

**参考答案：**

怀疑 Native 内存或堆外内存。检查 RSS、容器 memory working set、NMT、直接内存、线程数量、元空间、Code Cache、JNI、本地库和 mmap。heap dump 只能说明 Java 堆对象关系，不能覆盖全部进程内存。

```bash
jcmd <pid> VM.native_memory summary
jcmd <pid> Thread.print
jcmd <pid> VM.metaspace
```

### 7. 如何回答一个 JVM 生产调优问题？

**参考答案：**

先问目标和约束：吞吐、P99 延迟、堆大小、CPU、容器限制、流量模型、对象分配速率、可接受停顿。再看证据：GC 日志、JFR、堆曲线、线程数、RSS、业务指标。最后给参数或代码调整，并说明验证方式。不要上来就背一组固定参数。

> **易错：** JVM 调优不是参数背诵题，而是目标、证据、取舍和验证。

### 8. 虚拟线程对 JVM 内存和排障有什么影响？

**参考答案：**

虚拟线程让大量阻塞式任务以更低线程成本运行，但不代表没有资源限制。平台线程、连接池、同步块 pinning、ThreadLocal 使用、栈片段、调度器和外部依赖仍可能成为瓶颈。排障时要关注虚拟线程 dump、JFR、连接池和阻塞点。

### 9. JDK 升级时 JVM 层面要关注什么？

**参考答案：**

关注 GC 默认值变化、废弃或移除参数、类加载和非法反射限制、JIT 行为变化、容器感知策略、默认编码、TLS/安全算法、依赖库兼容性和性能基线。升级必须做压测、GC 日志对比、启动参数校验和回滚方案。
