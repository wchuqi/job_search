# JVM 学习资料：深度实验手册和能力验收

[返回索引](../JVM学习资料.md)

## 学习目标

- 用可运行实验验证 JVM 核心机制。
- 建立类加载、内存、GC、JIT、JMM、排障的动手能力。
- 完成最终能力验收，证明不是只会背概念。

## 环境假设

- JDK 21。
- Maven 或 Gradle 任一构建工具。
- 推荐 Linux 或容器环境；Windows 可使用等价 JDK 命令。
- 可选工具：JMC、JFR、VisualVM、MAT、async-profiler。

## 实验 1：类初始化触发条件

### 任务

写父类、子类、编译期常量、运行期常量和数组创建代码，记录输出顺序。

### 验收

- 能说明访问父类静态字段为什么不初始化子类。
- 能说明编译期常量为什么可能不触发初始化。

## 实验 2：自定义 ClassLoader 和类型身份

### 任务

用两个 ClassLoader 加载同一个 `User.class`，比较：

```java
clazz1 == clazz2
clazz1.getClassLoader()
clazz2.getClassLoader()
```

再尝试强转实例，观察异常。

### 验收

- 能解释 `ClassCastException: User cannot be cast to User`。

## 实验 3：OOM 分类

### 任务

分别制造：

- 堆 OOM。
- 栈溢出。
- 直接内存 OOM。
- native thread OOM。
- Metaspace 上涨，需谨慎控制规模。

### 验收

- 能根据异常文案判断区域。
- 能说明哪些 OOM 不一定出现在 heap dump 中。

## 实验 4：GC 日志阅读

### 任务

用一个持续分配对象的程序，分别使用 G1 和 ZGC：

```bash
java -Xms512m -Xmx512m -XX:+UseG1GC -Xlog:gc* -jar alloc.jar
java -Xms512m -Xmx512m -XX:+UseZGC -Xlog:gc* -jar alloc.jar
```

### 验收

- 能指出停顿时间、回收前后容量、并发阶段和 Full GC 迹象。
- 能解释为什么同样堆大小下不同 GC 的日志结构不同。

## 实验 5：JIT 和 JMH

### 任务

写一个手写循环 benchmark，再用 JMH 改写。

### 验收

- 能说明预热、fork、死代码消除、常量折叠的影响。
- 能解释为什么两种测试结果不应直接等价。

## 实验 6：JMM 可见性

### 任务

写一个非 volatile 停止标志实验，再改成 volatile 或 AtomicBoolean。

### 验收

- 能用 happens-before 解释修复原因。

## 实验 7：CPU 高和死锁排查

### 任务

制造死循环和死锁，采集：

```bash
top -Hp <pid>
jcmd <pid> Thread.print -l
jcmd <pid> JFR.start name=profile settings=profile duration=60s filename=profile.jfr
```

### 验收

- 能定位高 CPU 线程。
- 能从 thread dump 中找出死锁环。

## 最终能力验收

| 能力 | 验收标准 |
| --- | --- |
| 运行时模型 | 能画出 JVM 执行链路和运行时内存结构 |
| 类加载 | 能解释初始化触发、双亲委派、类冲突 |
| GC | 能读一段 GC 日志并判断主要压力 |
| JIT | 能解释预热、内联、逃逸分析和微基准陷阱 |
| JMM | 能用 happens-before 分析可见性问题 |
| 工具 | 能用 `jcmd`、JFR、heap dump、thread dump 定位本地故障 |
| 生产 | 能设计容器 JVM 参数和 OOM 证据保留策略 |
| 面试 | 能按“结论、机制、例子、边界、排查”回答 JVM 问题 |

## 自测问题

1. 为什么同名类可能无法强转？
2. 为什么 `static final` 字段有时不触发类初始化？
3. Minor GC 为什么需要关注老年代引用？
4. G1 的 Mixed GC 和 Full GC 有什么区别？
5. ZGC 为什么能做到低停顿，它付出了什么代价？
6. 为什么 Java 服务刚启动时性能可能不稳定？
7. `volatile` 能保证什么，不能保证什么？
8. 容器 OOMKilled 和 Java heap OOM 如何区分？
9. CPU 高时如何把 OS 线程映射到 Java 线程栈？
10. 2C4G 容器如何做 JVM 内存预算？

## 重点

- 每个核心概念至少用一个实验验证。
- 面试回答要能落到命令、日志、dump 或代码。

## 易错

> **易错：** 只看书面概念，不做故障实验。
>
> 正确做法：用实验把 JVM 行为变成可观察证据，再回到理论解释。
