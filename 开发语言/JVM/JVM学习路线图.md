# JVM 学习路线图

这份路线图面向 Java 后端、性能排查、架构面试和生产调优场景。版本基准以 HotSpot JVM 和 JDK 21 LTS 为主，兼顾现代 JDK 的低延迟 GC、容器感知、JFR、虚拟线程等运行时特征。

## 阶段 1：运行时基础认知

- 目标：知道 Java 程序从 `.java` 到机器执行的大致链路。
- 需要掌握：JDK、JRE、JVM、HotSpot、字节码、解释执行、JIT、GC、线程。
- 例子：用 `javac` 编译类，用 `javap -c` 查看字节码，用 `java -XshowSettings:vm -version` 看虚拟机配置。
- 练习：写一个 `HelloJvm`，分别观察编译产物、类加载日志和进程 PID。
- 验收：能画出 `源码 -> 字节码 -> 类加载 -> 解释/JIT -> GC/线程/内存` 的链路图。
- 重点：JVM 是规范，HotSpot 是最常见实现，不要把实现细节都说成规范。
- 易错：把 JDK、JRE、JVM 混为一谈；把“Java 跨平台”误解为源码直接跨平台。

## 阶段 2：运行时内存和对象模型

- 目标：理解堆、栈、元空间、程序计数器、本地方法栈、直接内存的职责边界。
- 需要掌握：栈帧、局部变量表、操作数栈、对象头、引用、TLAB、逃逸分析、OOM 分类。
- 例子：递归导致 `StackOverflowError`，大量对象导致 Java heap space，动态类生成导致 Metaspace。
- 练习：分别构造栈溢出、堆 OOM、直接内存 OOM，并用 `jcmd` 或 heap dump 验证。
- 验收：看到 OOM 文案能判断大致区域和第一排查入口。
- 重点：引用变量可能在栈上，对象通常在堆上；直接内存不在 Java 堆里，但仍占进程内存。
- 易错：只背“堆和栈”，忽略元空间、Code Cache、直接内存、线程栈和 Native 内存。

## 阶段 3：类加载、链接和字节码

- 目标：掌握类从磁盘、网络或内存进入 JVM 的过程。
- 需要掌握：加载、验证、准备、解析、初始化、类加载器层级、双亲委派、命名空间、SPI 破坏委派、字节码指令。
- 例子：`static final` 编译期常量和运行期常量触发初始化规则不同。
- 练习：用自定义 ClassLoader 加载同名类，验证“类名相同但加载器不同就是不同类型”。
- 验收：能解释 `ClassNotFoundException`、`NoClassDefFoundError`、`LinkageError` 的区别。
- 重点：类身份由“类加载器 + 全限定类名”共同决定。
- 难点：初始化触发条件、解析时机、父子加载器可见性、框架隔离模型。

## 阶段 4：GC 和内存回收

- 目标：从“会背垃圾回收算法”升级为能读 GC 日志和判断内存压力。
- 需要掌握：可达性分析、GC Roots、分代假说、Minor/Major/Full GC、STW、三色标记、写屏障、记忆集、G1、ZGC、Shenandoah。
- 例子：缓存没有淘汰导致老年代增长；大量短生命周期对象导致年轻代频繁回收。
- 练习：开启 `-Xlog:gc*`，观察吞吐优先和低延迟 GC 的日志差异。
- 验收：能基于 GC 日志判断是分配速率高、老年代压力、元空间压力还是引用保留。
- 重点：调 GC 不是先改参数，而是先明确目标：吞吐、延迟、内存占用、成本。
- 易错：把 Full GC 次数当成唯一指标；忽略业务请求延迟和对象分配模式。

## 阶段 5：JIT、性能模型和并发内存模型

- 目标：理解 JVM 为什么运行一段时间后会变快，以及并发可见性如何被约束。
- 需要掌握：解释器、C1/C2、分层编译、热点探测、内联、逃逸分析、锁消除、去优化、JMM、happens-before、volatile、synchronized。
- 例子：微基准不预热导致结果错误；错误双重检查锁因为缺少 `volatile` 出现发布问题。
- 练习：用 JMH 写一个字符串拼接或集合遍历 benchmark，观察预热和 JIT 的影响。
- 验收：能解释“JIT 优化不是 Java 语义之外的魔法，必须遵守 JMM 和可观察行为”。
- 难点：硬件内存模型、编译器重排序、CPU 缓存一致性和 Java happens-before 的关系。

## 阶段 6：工具、排障和生产调优

- 目标：能把线上“CPU 高、内存涨、卡顿、OOM、死锁、启动慢”变成可定位证据。
- 需要掌握：`jps`、`jcmd`、`jstack`、`jmap`、`jstat`、JFR、JMC、GC 日志、heap dump、thread dump、NMT、async-profiler。
- 例子：CPU 100% 用线程 dump 定位热点线程；内存泄漏用 heap dump 找保留链；停顿长用 GC 日志定位。
- 练习：搭建一个故障实验程序，分别制造 CPU 空转、死锁、内存泄漏和频繁 GC。
- 验收：能写出一份排障报告：现象、证据、定位过程、根因、修复、预防。
- 重点：生产排障先保现场，避免盲目重启丢失 dump 和日志。
- 易错：只看平均值，不看 P95/P99、STW 停顿、容器内存限制和线程数量。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 总览、内存区域、对象模型 | JVM 运行链路图、OOM 分类表 |
| 第 2 周 | 类加载、字节码、执行引擎 | `javap` 笔记、自定义 ClassLoader 实验 |
| 第 3 周 | GC 算法和常见收集器 | GC 日志阅读报告 |
| 第 4 周 | JIT、JMM、锁和线程 | JMH benchmark、并发可见性实验 |
| 第 5 周 | JDK 工具和生产排障 | CPU/内存/死锁/GC 故障剧本 |
| 第 6 周 | 容器、参数、面试复盘 | JVM 参数基线、面试题复述 |
| 第 7 周 | 深度机制补强 | 类加载冲突、GC 屏障、JIT 去优化、JMM 实验 |
| 第 8 周 | 综合生产演练 | 一份完整 JVM 故障复盘报告 |
| 第 9 周 | 深水区专题 | safepoint、class 文件、G1/ZGC 日志深读 |
| 第 10 周 | 容量和面试闭环 | 参数决策记录、高阶追问口述、压测报告 |

## 最终能力清单

- 能解释 JVM 规范和 HotSpot 实现的关系。
- 能区分运行时数据区、Native 内存、Code Cache、线程栈和容器内存。
- 能解释类加载生命周期、双亲委派和类冲突。
- 能读懂常见字节码指令和方法调用指令。
- 能解释对象创建、对象头、锁状态和逃逸分析。
- 能比较 Serial、Parallel、G1、ZGC、Shenandoah 的适用场景。
- 能用 GC 日志判断吞吐、停顿、分配速率和老年代压力。
- 能理解 JIT 热点编译、内联、去优化和微基准陷阱。
- 能用 JDK 工具定位 CPU 高、死锁、内存泄漏、OOM、频繁 GC。
- 能给 Spring Boot 或普通 Java 服务设计合理 JVM 启动参数和观测策略。
- 能解释屏障、记忆集、safepoint、去优化、容器 RSS 等深水区问题。
- 能把 JVM 面试答案组织成“结论、机制、例子、边界、排查证据”。
