# NumPy学习资料：性能优化、内存诊断和大数组处理

[返回索引](../Numpy学习资料.md)

## 学习目标

- 能定位 Python 循环、临时数组、非连续内存和算法复杂度问题。
- 会估算数组内存并减少拷贝。
- 掌握分块处理和 memmap 的适用场景。

## 理论导读

NumPy 快的前提是计算能落到底层批量操作。若代码在 Python 层逐元素循环，优势会大幅下降。另一方面，向量化表达式可能创建多个临时数组，导致内存峰值过高。性能优化要同时看运行时间、峰值内存和数值正确性。

## 示例：减少临时数组

```python
import numpy as np

x = np.random.default_rng(0).normal(size=(1_000_000,))

# 可能创建多个临时数组
y = (x - x.mean()) / x.std()

# 可通过分步和 out 控制部分内存
mean = x.mean()
std = x.std()
y = x.copy()
np.subtract(y, mean, out=y)
np.divide(y, std, out=y)
```

## 大数组策略

| 问题 | 策略 |
| --- | --- |
| 内存不足 | 降 dtype、分块、memmap |
| Python 循环慢 | 向量化、Numba、Cython |
| 中间数组过大 | 使用 `out`、分步计算、分块 |
| 非连续访问慢 | `ascontiguousarray` 或调整算法 |
| IO 慢 | 二进制格式、批量读取 |

## 练习

对一个 10000x10000 数组做列标准化，先估算内存，再设计分块方案。

## 验收

- 能根据 shape 和 dtype 估算 `nbytes`。
- 能解释为什么广播距离矩阵可能爆内存。
- 能说出何时考虑 Numba、Dask 或更换算法。

## 重点

- 性能优化前先测量，优化时同时关注内存。

## 难点

- 最快的 NumPy 表达式未必是最省内存的表达式。

## 易错

> **易错：** 所有慢代码都用向量化硬改。
>
> 正确做法：先看数据规模、内存峰值和算法复杂度，必要时分块或换算法。
