# NumPy学习资料：总览与心智模型

[返回索引](../Numpy学习资料.md)

## 学习目标

- 理解 NumPy 在 Python 科学生态中的位置。
- 建立 `ndarray = 数据缓冲区 + dtype + shape + strides` 的心智模型。
- 知道 NumPy 为什么在数值计算中通常快于 Python list。

## 理论导读

NumPy 是 Python 数值计算的基础库。Pandas、SciPy、scikit-learn、PyTorch、JAX、OpenCV 等生态都直接或间接围绕数组计算展开。NumPy 的核心不是“更多函数”，而是 `ndarray` 这种高效数组对象。

Python list 保存的是对象引用，每个元素可以是不同类型；NumPy 数组保存同类型数据，能用紧凑内存布局和底层 C/BLAS 例程批量处理。它牺牲了一部分动态灵活性，换来更好的内存局部性、向量化执行和生态统一接口。

## 核心心智模型

一个 NumPy 数组由四个关键信息解释：

```text
data buffer + dtype + shape + strides
```

- `data buffer`：真实数据所在内存。
- `dtype`：每个元素如何解释，例如 `float64`、`int32`。
- `shape`：每个维度有多少元素。
- `strides`：每个维度移动一步需要跨过多少字节。

同一块数据可以通过不同 shape 和 strides 被解释成不同视图，这也是 reshape、transpose、切片能很快的原因之一。

## 例子

```python
import numpy as np

a = np.arange(12, dtype=np.int32).reshape(3, 4)
print(a.shape)    # (3, 4)
print(a.dtype)    # int32
print(a.strides)  # 常见为 (16, 4)
print(a.ndim)     # 2
```

`strides=(16, 4)` 表示沿第 0 维移动一行跨 16 字节，沿第 1 维移动一列跨 4 字节。

## 练习

1. 创建一个 `float64` 的 3x4 数组。
2. 打印 `shape`、`dtype`、`strides`、`nbytes`。
3. 转置它并观察 `strides` 如何变化。

## 验收

- 能解释 NumPy 数组和 Python list 的差异。
- 能说出 shape、dtype、strides 的含义。
- 能说明为什么切片经常不复制数据。

## 重点

- NumPy 的底层模型比 API 名称更重要。
- 学 NumPy 要围绕 shape、axis、dtype、broadcasting、view/copy 建立体系。

## 难点

- `shape` 决定逻辑结构，`strides` 决定访问方式，二者不一定对应连续内存。

## 易错

> **易错：** 只把 NumPy 当成“数学函数集合”。
>
> 正确做法：先理解 `ndarray` 的内存和形状模型，再学习函数，很多性能和 bug 都来自这个模型。
