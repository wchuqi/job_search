# NumPy学习资料：测试、调试和工程化规范

[返回索引](../Numpy学习资料.md)

## 学习目标

- 会为数值代码编写测试。
- 掌握容差断言、shape 断言和随机性控制。
- 建立 NumPy 工程化规范。

## 理论导读

数值代码不能只用 `==` 断言。浮点误差、随机性、dtype 转换、边界 shape、空数组和缺失值都会影响结果。好的 NumPy 测试应该同时验证 shape、dtype、数值范围、容差和异常输入。

## 示例

```python
import numpy as np
from numpy.testing import assert_allclose


def normalize(x: np.ndarray) -> np.ndarray:
    return (x - x.mean(axis=0, keepdims=True)) / x.std(axis=0, keepdims=True)


def test_normalize():
    x = np.array([[1.0, 2.0], [3.0, 4.0]])
    y = normalize(x)
    assert y.shape == x.shape
    assert_allclose(y.mean(axis=0), np.array([0.0, 0.0]), atol=1e-12)
```

## 测试清单

- 正常输入。
- 空数组。
- 单元素数组。
- 包含 `nan` 或 `inf`。
- 不同 dtype。
- 非连续数组。
- 随机输入但固定 seed。
- 极大或极小数值。

## 练习

为标准化函数、距离矩阵函数和异常值裁剪函数各写 3 个测试。

## 验收

- 能使用 `numpy.testing.assert_allclose`。
- 能在函数入口断言 shape 或给出清晰错误。
- 能固定随机数生成器保证测试可复现。

## 重点

- 数值测试要使用容差，不能盲目使用精确相等。

## 难点

- 算法正确但数值不稳定时，测试可能在不同平台上表现不同。

## 易错

> **易错：** 测试只覆盖一个漂亮的二维数组。
>
> 正确做法：覆盖边界 shape、异常值、缺失值和 dtype 差异。
