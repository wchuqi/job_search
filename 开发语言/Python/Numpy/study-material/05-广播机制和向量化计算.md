# NumPy学习资料：广播机制和向量化计算

[返回索引](../Numpy学习资料.md)

## 学习目标

- 掌握广播规则。
- 能用向量化替代低效 Python 循环。
- 能识别广播带来的巨大中间数组风险。

## 理论导读

广播允许不同 shape 的数组在满足规则时参与逐元素运算。规则是从右向左对齐维度：两个维度相等，或其中一个为 1，则兼容；缺失的左侧维度按 1 处理。

向量化通常让底层 C 循环处理数据，减少 Python 解释器循环开销。但向量化表达式可能产生临时数组，内存峰值不一定低。

## 示例

```python
import numpy as np

x = np.array([[1, 2, 3], [4, 5, 6]])
mean = x.mean(axis=0, keepdims=True)
std = x.std(axis=0, keepdims=True)
z = (x - mean) / std
```

`x.shape=(2, 3)`，`mean.shape=(1, 3)`，可以沿第 0 维广播。

## 广播规则例子

| A shape | B shape | 是否兼容 | 结果 |
| --- | --- | --- | --- |
| `(3, 4)` | `(4,)` | 是 | `(3, 4)` |
| `(3, 1)` | `(1, 4)` | 是 | `(3, 4)` |
| `(2, 3)` | `(2,)` | 否 | 无 |
| `(5, 1, 4)` | `(3, 4)` | 是 | `(5, 3, 4)` |

## 练习

实现两个矩阵 `X` 和 `Y` 之间的成对欧氏距离：

```python
dist = np.sqrt(((X[:, None, :] - Y[None, :, :]) ** 2).sum(axis=2))
```

然后估算中间数组的 shape 和内存。

## 验收

- 能从右向左推导广播兼容性。
- 能用 `keepdims=True` 保持维度便于广播。
- 能发现成对计算可能产生大中间数组。

## 重点

- 广播是 NumPy 最重要的形状推导规则。

## 难点

- 写法短不代表内存少。广播表达式可能隐式扩展逻辑形状。

## 易错

> **易错：** 看到没有 for 循环就认为性能一定好。
>
> 正确做法：同时估算时间复杂度和中间数组内存。
