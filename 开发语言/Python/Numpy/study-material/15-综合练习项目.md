# NumPy学习资料：综合练习项目

[返回索引](../Numpy学习资料.md)

## 项目目标

实现一个“纯 NumPy 数据特征处理工具包”，覆盖数组读取、缺失值处理、标准化、异常值裁剪、距离矩阵、简单线性回归、测试和性能优化。

## 功能要求

- 读取 `.npy` 或 `.csv` 数据。
- 检查 shape、dtype、缺失值和无穷值。
- 支持按列标准化。
- 支持异常值分位数裁剪。
- 支持计算样本间距离矩阵。
- 支持最小二乘线性回归。
- 为每个函数提供测试。

## 推荐结构

```text
numpy_toolkit/
  __init__.py
  io.py
  cleaning.py
  scaling.py
  distance.py
  regression.py
tests/
  test_cleaning.py
  test_scaling.py
  test_distance.py
  test_regression.py
```

## 技术要求

- 所有函数入口检查 shape 和 dtype。
- 浮点测试使用容差断言。
- 随机测试使用 `np.random.default_rng(seed)`。
- 对距离矩阵实现内存估算，大数据时给出分块方案。
- 文档中说明 view/copy 行为。

## 验收标准

- 测试覆盖正常输入、缺失值、空数组、非连续数组和不同 dtype。
- 标准化后每列均值接近 0，标准差接近 1。
- 距离矩阵 shape 正确且对角线接近 0。
- 线性回归结果与 `np.linalg.lstsq` 对齐。
- 能解释每个函数的时间复杂度和主要内存消耗。

## 扩展任务

- 支持 memmap 分块读取。
- 添加 PCA 降维。
- 添加图像数组归一化。
- 用 Numba 优化一个无法自然向量化的循环。

## 重点

- 项目重点不是包装 API，而是练 shape、dtype、广播、数值稳定性和测试。

## 难点

- 距离矩阵和大数组标准化很容易创建巨大中间数组，要学会估算内存并分块。

## 易错

> **易错：** 只验证输出值，不验证 shape、dtype 和边界输入。
>
> 正确做法：数值工具包的测试必须覆盖形状、类型、容差和异常路径。
