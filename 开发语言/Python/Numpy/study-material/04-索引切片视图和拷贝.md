# NumPy学习资料：索引、切片、视图和拷贝

[返回索引](../Numpy学习资料.md)

## 学习目标

- 掌握基础索引、切片、布尔索引和高级索引。
- 理解 view 和 copy 的差异。
- 避免链式索引和意外修改。

## 理论导读

基础切片通常返回视图，共享原数组数据；布尔索引和整数数组索引通常返回拷贝。视图修改可能影响原数组，拷贝修改不会影响原数组。这一规则是 NumPy 高频 bug 来源。

## 示例

```python
import numpy as np

a = np.arange(10)
b = a[2:5]
b[0] = 99
print(a)  # 原数组被修改

c = a[[2, 3, 4]]
c[0] = -1
print(a)  # 通常不受 c 修改影响

print(np.shares_memory(a, b))
print(np.shares_memory(a, c))
```

## 索引类型比较

| 类型 | 示例 | 通常是否共享内存 |
| --- | --- | --- |
| 基础切片 | `a[1:5]` | 是 |
| 步长切片 | `a[::2]` | 是 |
| 布尔索引 | `a[a > 0]` | 否 |
| 整数数组索引 | `a[[1, 3]]` | 否 |
| 转置 | `a.T` | 通常是 |

## 练习

对一个 5x5 矩阵完成：

- 取第 2 行。
- 取第 2 到第 4 列。
- 取所有大于 10 的元素。
- 修改一个切片并观察原数组。

## 验收

- 能判断常见索引返回 view 还是 copy。
- 能用 `copy()` 主动切断共享。
- 能避免 `a[mask][0] = x` 这类链式写入。

## 重点

- 视图和拷贝不是语法细节，而是正确性和性能问题。

## 难点

- 混合基础索引和高级索引时，结果形状和拷贝行为更复杂，应通过测试确认。

## 易错

> **易错：** 以为所有切出来的数组都是独立副本。
>
> 正确做法：需要隔离修改时显式调用 `.copy()`。
