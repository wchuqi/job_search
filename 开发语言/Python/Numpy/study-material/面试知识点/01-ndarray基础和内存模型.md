## 一、ndarray 基础和内存模型

### 1. NumPy 数组和 Python list 有什么区别？

**参考答案：**

Python list 存储对象引用，元素类型可以不同；NumPy 数组通常存储同类型、紧凑排列的数据，并用 dtype、shape 和 strides 解释内存。NumPy 能把批量计算交给底层 C/BLAS，减少 Python 循环开销。

> **重点：** 同类型、紧凑内存、向量化和底层实现是关键词。

### 2. `ndarray` 由哪些核心信息组成？

**参考答案：**

可以理解为数据缓冲区、dtype、shape 和 strides 的组合。数据缓冲区保存原始数据，dtype 决定如何解释单个元素，shape 决定逻辑维度，strides 决定每个维度移动一步跨多少字节。

> **难点：** 同一块数据可以通过不同 shape 和 strides 形成不同视图。

### 3. `axis=0` 和 `axis=1` 怎么理解？

**参考答案：**

axis 表示沿哪个维度操作或折叠。对二维数组 `(行, 列)`，`axis=0` 通常沿行方向聚合，结果按列保留；`axis=1` 沿列方向聚合，结果按行保留。

> **易错：** 背“0 是列、1 是行”容易混乱。更准确是看哪个维度被折叠。

### 4. `dtype` 为什么重要？

**参考答案：**

dtype 决定元素字节数、数值范围、精度和底层计算方式。它会影响内存占用、溢出风险、浮点误差和下游库兼容性。

> **重点：** dtype 是性能和正确性问题，不只是类型标记。
