## 一、性能、工程化和生态互操作

### 1. NumPy 代码慢，你会怎么排查？

**参考答案：**

先确认是否存在 Python 层逐元素循环，再看数组是否连续、是否产生巨大临时数组、算法复杂度是否过高、BLAS 是否参与、IO 是否成为瓶颈。排查时同时看运行时间和内存峰值。

> **重点：** 不要只说“向量化”，要具体分析瓶颈。

### 2. 如何估算数组内存？

**参考答案：**

数组内存大致等于元素个数乘以每个元素字节数，可以用 `a.nbytes` 查看。例如 shape 为 `(10000, 1000)` 的 `float64` 数组约为 `10000 * 1000 * 8 = 80MB`。

> **易错：** 忽略中间数组和拷贝，实际峰值可能是结果数组的数倍。

### 3. 浮点数为什么不能直接用 `==` 比较？

**参考答案：**

浮点数采用有限二进制表示，很多十进制小数无法精确表示，计算会产生舍入误差。应使用 `np.isclose`、`np.allclose` 或 `numpy.testing.assert_allclose`。

> **重点：** 数值测试要使用容差。

### 4. NumPy 和 Pandas、SciPy、scikit-learn 的关系是什么？

**参考答案：**

NumPy 提供基础数组和数值计算能力；Pandas 面向表格和索引数据；SciPy 提供更丰富的科学计算；scikit-learn 通常以 NumPy 数组或兼容数组作为特征矩阵输入。

> **难点：** 跨库转换时要检查 shape、dtype、copy 和缺失值。
