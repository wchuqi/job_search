# NumPy学习路线图

## 阶段 1：数组基础和 NumPy 定位

- 目标：理解 NumPy 为什么比 Python list 更适合数值计算。
- 需要掌握：`ndarray`、shape、axis、dtype、基础创建、数组属性、基本运算。
- 例子：把一组学生成绩从 list 改成 NumPy 数组并计算均值、标准差和排名。
- 练习：创建一维、二维、三维数组，打印 `shape`、`ndim`、`dtype`、`size`。
- 验收：能解释 `ndarray` 是“同类型、连续或可步进访问的数据块视图”。
- 重点：shape、dtype、axis 是 NumPy 的基本语言。
- 易错：把 NumPy 数组当成任意嵌套 list 使用。

## 阶段 2：内存模型、索引和视图拷贝

- 目标：掌握 dtype、strides、连续性、切片、布尔索引和高级索引。
- 需要掌握：C/F order、view、copy、reshape、transpose、boolean mask、fancy indexing。
- 例子：解释为什么 `a[1:3]` 修改会影响原数组，而 `a[[1, 2]]` 通常不会。
- 练习：构造一个二维数组，分别用切片、布尔索引和整数数组索引取子集，并判断是否共享内存。
- 验收：能用 `np.shares_memory` 辅助判断视图和拷贝。
- 难点：strides 决定数组如何在同一块内存上解释数据。
- 易错：链式索引导致修改没有写回原数组。

## 阶段 3：广播、ufunc 和向量化

- 目标：写出没有 Python for 循环的批量数值计算。
- 需要掌握：广播规则、ufunc、聚合函数、axis、keepdims、where、out。
- 例子：对一个样本矩阵做每列标准化 `(x - mean) / std`。
- 练习：实现欧氏距离矩阵和 min-max 归一化。
- 验收：能解释两个 shape 从右往左对齐的广播规则。
- 重点：向量化不是魔法，本质是把循环交给底层 C/BLAS。
- 易错：广播生成巨大中间数组导致内存爆炸。

## 阶段 4：统计、随机、线性代数和生态互操作

- 目标：能完成常见数据科学和算法基础任务。
- 需要掌握：`np.random.default_rng`、统计聚合、协方差、矩阵乘法、求解线性方程、SVD、FFT、与 Pandas/SciPy/scikit-learn 互操作。
- 例子：用 NumPy 实现线性回归闭式解。
- 练习：模拟随机样本，计算置信区间，并用线性代数求最小二乘解。
- 验收：能区分逐元素乘法、矩阵乘法和广播乘法。
- 难点：数值稳定性、矩阵条件数和 dtype 精度会影响结果。
- 易错：用 `np.linalg.inv(A) @ b` 代替更稳定的 `np.linalg.solve(A, b)`。

## 阶段 5：性能、测试和工程化

- 目标：能定位慢计算、内存过大、精度误差和边界输入。
- 需要掌握：内存占用估算、连续性、临时数组、分块处理、memmap、profiling、`numpy.testing`。
- 例子：优化一个大数组归一化过程，减少临时数组和拷贝。
- 练习：为数值函数写容差断言测试。
- 验收：能说明性能瓶颈是 Python 循环、内存带宽、BLAS、IO 还是算法复杂度。
- 重点：NumPy 快，但不等于所有表达式都省内存。
- 易错：只看运行时间，不看峰值内存和数值误差。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | ndarray、dtype、shape、基础创建 | 数据清洗小脚本 |
| 第 2 周 | 索引、视图拷贝、广播、ufunc | 向量化指标计算模块 |
| 第 3 周 | 随机、统计、线性代数 | 线性回归或 PCA 小实验 |
| 第 4 周 | 性能、测试、生态互操作 | 可测试的 NumPy 工程模板 |

## 最终能力清单

- 能从 shape、dtype、strides 判断数组行为。
- 能正确使用索引、切片、布尔 mask 和高级索引。
- 能用广播和 ufunc 替代低效 Python 循环。
- 能处理缺失值、异常值和数值稳定性问题。
- 能写出可靠的随机数、统计和线性代数代码。
- 能估算内存、减少拷贝并定位性能瓶颈。
- 能回答 NumPy 高频面试题并解释底层原因。
