# Paramiko学习路线图

## 阶段 1：SSH 基础和 Paramiko 定位

- 目标：理解 Paramiko 是 Python 中实现 SSHv2 客户端和 SFTP 自动化的库。
- 需要掌握：SSH、认证、Host Key、`SSHClient`、`connect`、`exec_command`。
- 例子：连接一台测试主机并执行 `hostname`、`uptime`。
- 练习：写一个脚本读取远程命令输出和退出码。
- 验收：能解释 SSH 连接建立时服务器身份校验和用户认证的区别。
- 重点：Host Key 校验是安全边界，不是可有可无的警告。
- 易错：为了省事使用 `AutoAddPolicy` 到处接受未知主机。

## 阶段 2：命令执行、Channel 和 SFTP

- 目标：掌握远程命令执行的输入输出、退出码、超时和文件传输。
- 需要掌握：stdout、stderr、`recv_exit_status`、`open_sftp`、上传下载、目录操作。
- 例子：上传配置文件，执行远程校验命令，再下载日志。
- 练习：实现一个远程文件存在性检查和安全上传函数。
- 验收：能处理命令失败、stderr、超时和文件不存在。
- 难点：远程 shell 的环境和交互式登录环境不一定相同。
- 易错：只读取 stdout，不检查 stderr 和退出码。

## 阶段 3：认证、跳板机和网络边界

- 目标：能处理密钥、密码、Agent、跳板机、代理命令和端口转发。
- 需要掌握：私钥加载、passphrase、SSH Agent、ProxyCommand、direct-tcpip、超时。
- 例子：通过堡垒机连接内网主机并执行命令。
- 练习：封装一个支持跳板机的连接工厂。
- 验收：能说清楚认证失败、网络不可达、Host Key 不匹配的差异。
- 重点：凭据不要写死在代码里。
- 易错：把私钥、密码或主机清单提交到仓库。

## 阶段 4：批量执行、可靠性和排障

- 目标：构建可用于生产的 SSH 自动化脚本。
- 需要掌握：线程池、连接复用边界、重试、幂等、结构化日志、结果汇总。
- 例子：并发检查 100 台机器磁盘使用率，输出成功、失败和超时列表。
- 练习：为批量执行器加入失败重试和最大并发控制。
- 验收：能避免无限等待、雪崩并发和重复执行危险命令。
- 难点：重试只适合幂等操作，非幂等命令要有确认和补偿。
- 易错：批量执行 `rm`、重启、改配置前没有 dry-run 和主机分批。

## 阶段 5：测试、安全规范和替代方案

- 目标：把脚本变成可测试、可审计、可维护的运维工具。
- 需要掌握：Mock、测试 SSH Server、日志脱敏、最小权限、审计、Fabric/AsyncSSH/Ansible 边界。
- 例子：用 fake client 测试命令拼接和结果解析。
- 练习：为远程执行器写单元测试和集成测试。
- 验收：能说明什么时候不应该使用 Paramiko。
- 重点：Paramiko 是底层库，大规模编排可能更适合 Ansible、Fabric 或平台化方案。
- 易错：用 Paramiko 重造完整配置管理系统。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | SSH 安全模型、连接、命令执行 | 单主机执行工具 |
| 第 2 周 | SFTP、密钥、跳板机 | 文件同步工具 |
| 第 3 周 | 超时、重试、并发、日志 | 批量执行器 |
| 第 4 周 | 测试、安全规范、项目化 | 可审计运维工具 |

## 最终能力清单

- 能安全建立 SSH 连接并校验 Host Key。
- 能执行远程命令并处理 stdout、stderr、退出码和超时。
- 能使用 SFTP 上传、下载、列目录和处理远程文件。
- 能管理密钥、密码、Agent 和跳板机连接。
- 能设计并发批量执行、重试和幂等策略。
- 能记录日志、定位认证、网络和远程命令错误。
- 能回答 Paramiko 高频面试题，并说明生产风险。
