# Paramiko学习资料：超时、重试、幂等和错误处理

[返回索引](../Paramiko学习资料.md)

## 学习目标

- 掌握连接、认证、命令和文件传输超时。
- 区分可重试和不可重试错误。
- 设计幂等远程操作。

## 理论导读

SSH 自动化如果没有超时，脚本可能永久挂住。重试可以缓解临时网络问题，但不能盲目用于非幂等命令。幂等意味着重复执行不会造成额外破坏，例如检查文件存在再上传、检查服务状态再重启。

## 常见异常分类

| 类型 | 示例 | 是否适合重试 |
| --- | --- | --- |
| 网络临时失败 | 连接超时、连接重置 | 可有限重试 |
| 认证失败 | 密码错、密钥不匹配 | 不应重试太多 |
| Host Key 失败 | 指纹不匹配 | 不自动重试 |
| 命令失败 | exit code 非 0 | 看业务语义 |
| 权限不足 | permission denied | 修配置，不盲重试 |

## 结果结构

```python
from dataclasses import dataclass


@dataclass
class RemoteResult:
    host: str
    command: str
    exit_code: int | None
    stdout: str
    stderr: str
    error: str | None
```

## 练习

为 `run_command` 添加最多 2 次重试，只对连接超时重试，不对认证失败和非 0 退出码重试。

## 验收

- 能为连接和命令设置超时。
- 能区分异常类别。
- 能解释为什么重试需要幂等性。

## 重点

- 批量运维中，重试策略必须和命令副作用一起设计。

## 难点

- 远程命令可能已经执行成功但连接中断，客户端无法简单判断最终状态。

## 易错

> **易错：** 所有失败都自动重试。
>
> 正确做法：只对明确可重试的临时错误重试，对非幂等命令保守处理。
