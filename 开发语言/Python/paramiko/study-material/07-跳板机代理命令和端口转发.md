# Paramiko学习资料：跳板机、代理命令和端口转发

## 学习目标

- 理解跳板机和内网主机连接链路。
- 掌握 ProxyCommand 和 direct-tcpip 思路。
- 了解本地和远程端口转发的边界。

## 理论导读

很多生产环境不能直接连接目标主机，需要通过堡垒机或跳板机。Paramiko 可以通过代理命令或在一个 SSH Transport 上打开到目标主机的 channel，再把这个 channel 作为 socket 给第二个 SSHClient。

## ProxyCommand 示例

```python
import paramiko

proxy = paramiko.ProxyCommand("ssh -W target.internal:22 bastion.example.com")
client = paramiko.SSHClient()
client.load_system_host_keys()
client.connect("target.internal", username="deploy", sock=proxy)
```

## direct-tcpip 思路

```text
本机 -> bastion SSH Transport -> direct-tcpip channel -> target:22 -> target SSHClient
```

## 练习

封装一个连接工厂，支持直连和通过跳板机连接两种模式。

## 验收

- 能解释跳板机链路。
- 能区分代理命令和二次 SSHClient。
- 能说明端口转发的安全风险。

## 重点

- 跳板机连接要记录审计链路和目标主机，避免绕过堡垒机策略。

## 难点

- 多跳链路错误可能来自本机到跳板、跳板到目标、目标认证三个不同阶段。

## 易错

> **易错：** 为了方便在脚本里绕过堡垒机审计。
>
> 正确做法：遵守组织的 SSH 审计和访问控制流程。
