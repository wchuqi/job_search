# Paramiko学习资料：并发批量执行和连接池边界

[返回索引](../Paramiko学习资料.md)

## 学习目标

- 掌握批量执行的并发控制。
- 理解连接复用和线程安全边界。
- 设计结果汇总和失败隔离。

## 理论导读

Paramiko 常用于批量运维，但 SSH 连接成本高，远程主机和堡垒机也有连接限制。并发过高会导致认证失败、连接拒绝、堡垒机压力或远程命令雪崩。应使用线程池、限流、分批和超时。

## 示例

```python
from concurrent.futures import ThreadPoolExecutor, as_completed


def run_one(host: str) -> dict:
    # 创建独立 SSHClient，执行命令，关闭连接
    return {"host": host, "ok": True}


with ThreadPoolExecutor(max_workers=10) as pool:
    futures = [pool.submit(run_one, host) for host in hosts]
    for future in as_completed(futures):
        print(future.result())
```

## 设计原则

- 每个线程使用自己的 client 或明确同步访问。
- 设置最大并发。
- 结果包含 host、命令、耗时、退出码和错误。
- 支持主机分批和失败停止策略。
- 危险操作提供 dry-run。

## 练习

实现一个批量执行器：最大并发 5，超时 30 秒，输出成功列表、失败列表和超时列表。

## 验收

- 能限制并发。
- 能聚合每台主机结果。
- 能说明批量执行危险命令的保护措施。

## 重点

- 批量执行不是把单机脚本套一层 for 循环。

## 难点

- 大规模场景下，瓶颈可能在堡垒机、网络、DNS、远程主机或本机线程资源。

## 易错

> **易错：** 对几百台机器同时发起 SSH 连接。
>
> 正确做法：限流、分批、超时、失败隔离，并记录完整结果。
