# Paramiko学习资料：日志、调试和生产排障

[返回索引](../Paramiko学习资料.md)

## 学习目标

- 启用 Paramiko 日志。
- 分层排查连接、认证、命令和 SFTP 问题。
- 设计结构化日志和脱敏策略。

## 理论导读

SSH 问题要分层定位：DNS 是否解析、TCP 是否连通、SSH banner 是否返回、Host Key 是否匹配、认证是否成功、远程命令是否存在、权限是否足够。Paramiko 日志能帮助定位协议和认证阶段问题。

## 启用日志

```python
import logging

logging.basicConfig(level=logging.INFO)
logging.getLogger("paramiko").setLevel(logging.DEBUG)
```

## 排障清单

| 现象 | 排查方向 |
| --- | --- |
| 连接超时 | 网络、端口、防火墙、堡垒机 |
| banner timeout | 服务端 SSH 响应慢或非 SSH 服务 |
| Host Key 失败 | known_hosts、主机重装、中间人风险 |
| 认证失败 | 用户、密码、私钥、权限、Agent |
| 命令找不到 | PATH、非交互 shell、绝对路径 |
| SFTP 权限失败 | 目录权限、用户权限、磁盘 |

## 练习

分别制造错误端口、错误密码、错误命令三种失败，并记录结构化日志。

## 验收

- 能开启 Paramiko debug 日志。
- 能根据异常阶段判断排查方向。
- 日志不泄露密码和私钥。

## 重点

- 日志要能定位 host、阶段、命令、退出码、耗时和错误摘要。

## 难点

- Debug 日志可能包含敏感上下文，生产采集要控制级别和脱敏。

## 易错

> **易错：** 出错只打印 `failed`。
>
> 正确做法：记录主机、阶段、异常类型、退出码、stderr 摘要和耗时。
