# Paramiko学习资料：密钥、密码、代理和凭据管理

[返回索引](../Paramiko学习资料.md)

## 学习目标

- 掌握密码、私钥、passphrase、SSH Agent 的使用。
- 理解凭据存储和日志脱敏。
- 避免凭据泄露。

## 理论导读

SSH 自动化的最大风险之一是凭据泄露。密码、私钥路径、私钥内容、passphrase、临时 token 都应通过安全配置系统、环境变量或密钥管理服务传递，不应硬编码或记录到日志。

## 私钥示例

```python
client.connect(
    hostname="example.com",
    username="deploy",
    key_filename=r"C:\Users\me\.ssh\id_ed25519",
    passphrase=None,
    timeout=10,
)
```

## 凭据管理建议

| 凭据 | 建议 |
| --- | --- |
| 密码 | 尽量避免自动化使用，必须使用时放入密钥管理系统 |
| 私钥文件 | 权限最小化，不提交仓库 |
| passphrase | 不写日志，不写配置文件明文 |
| SSH Agent | 控制 Agent 转发和生命周期 |
| 主机清单 | 避免包含敏感内网结构时随意外发 |

## 练习

把脚本中的密码参数改成从环境变量读取，并确保日志中不打印密码。

## 验收

- 能使用私钥连接。
- 能说明 Agent 的作用和风险。
- 日志中不出现密码、私钥和 passphrase。

## 重点

- 凭据泄露通常比脚本 bug 更严重。

## 难点

- Agent 转发很方便，但在不可信跳板机上可能扩大攻击面。

## 易错

> **易错：** 把私钥文件、密码或完整连接 URL 提交到 Git。
>
> 正确做法：使用密钥管理、环境变量和 `.gitignore`，并做日志脱敏。
