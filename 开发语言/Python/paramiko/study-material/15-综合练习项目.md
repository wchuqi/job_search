# Paramiko学习资料：综合练习项目

[返回索引](../Paramiko学习资料.md)

## 项目目标

实现一个“批量 SSH 运维执行器”，覆盖安全连接、远程命令、SFTP、跳板机、超时、并发、日志、测试和审计。

## 功能要求

- 从 YAML 或 JSON 读取主机清单。
- 支持直连和跳板机连接。
- 支持私钥认证。
- 校验 Host Key。
- 批量执行只读命令，例如 `df -h`、`uptime`。
- 支持上传文件到临时路径并 rename。
- 设置连接、认证、命令超时。
- 支持最大并发和分批执行。
- 输出 JSON 结果报告。
- 日志脱敏。

## 推荐结构

```text
ssh_runner/
  __init__.py
  config.py
  client.py
  command.py
  sftp.py
  runner.py
  logging.py
tests/
  test_command.py
  test_runner.py
  test_sftp.py
```

## 验收标准

- 认证失败、连接超时、命令失败都有清晰结果。
- stdout、stderr、exit code 都被记录。
- 批量执行不会因单台失败中断全部任务。
- 危险命令默认禁止或需要显式确认。
- 测试覆盖成功、失败、超时和日志脱敏。
- 主机清单和凭据不写死在代码里。

## 扩展任务

- 添加 dry-run。
- 添加主机分组。
- 添加跳板机链路审计。
- 添加结果导出为 CSV。
- 添加命令白名单和变更审批字段。

## 重点

- 这个项目重点练安全、可靠性和审计，而不是简单跑通 SSH。

## 难点

- 批量远程操作要同时处理连接失败、部分成功、命令副作用和结果可追溯。

## 易错

> **易错：** 先做高并发批量执行，再补安全和日志。
>
> 正确做法：从第一版就设计 Host Key、超时、结果结构、日志脱敏和 dry-run。
