# Paramiko学习资料：安全规范、运维集成和替代方案

[返回索引](../Paramiko学习资料.md)

## 学习目标

- 建立生产 SSH 自动化安全规范。
- 理解 Paramiko 与 Fabric、Ansible、AsyncSSH 的选择边界。
- 能把 Paramiko 工具接入运维流程。

## 理论导读

Paramiko 给了你 SSH 能力，但不会自动提供审批、审计、权限、回滚、变更窗口、灰度和配置管理。生产工具要围绕操作风险设计，而不是只围绕代码能否执行。

## 安全规范

- Host Key 必须校验。
- 凭据不得硬编码。
- 日志必须脱敏。
- 危险命令支持 dry-run 和确认。
- 批量操作支持分批和暂停。
- 每次执行记录操作者、主机、命令、结果和时间。
- 最小权限账号执行。
- 提供回滚策略。

## 工具选择

| 场景 | 工具 |
| --- | --- |
| 定制 SSH/SFTP 自动化 | Paramiko |
| 更高层 Python 运维任务 | Fabric |
| 大规模配置管理 | Ansible |
| 异步 SSH 高并发 | AsyncSSH |
| 数据库式本地分析 | 不适用，应使用数据工具 |

## 练习

为批量执行工具添加 dry-run、审计日志、主机白名单和危险命令拦截。

## 验收

- 能说明什么时候不用 Paramiko。
- 能列出生产 SSH 自动化安全要求。
- 能把执行结果转成审计记录。

## 重点

- 运维自动化的安全边界比 API 使用更重要。

## 难点

- 权限、审计和回滚往往需要组织流程配合，不是单个脚本能完全解决。

## 易错

> **易错：** 用 Paramiko 脚本替代正式变更流程。
>
> 正确做法：脚本接入审批、审计、灰度和回滚机制。
