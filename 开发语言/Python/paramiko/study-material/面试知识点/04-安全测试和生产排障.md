## 一、安全、测试和生产排障

### 1. Paramiko 脚本如何避免凭据泄露？

**参考答案：**

密码、私钥和 passphrase 不硬编码、不提交仓库、不打印日志。使用环境变量、密钥管理系统或 SSH Agent，并对日志做脱敏。

> **重点：** 日志泄露是常见凭据泄露路径。

### 2. 如何测试 Paramiko 封装代码？

**参考答案：**

单元测试用 Mock 或 fake SSHClient 测试命令构造、异常处理和结果解析；集成测试使用隔离的本地 SSH Server 或容器环境，不连接生产主机。

> **易错：** 单元测试直接依赖真实服务器。

### 3. 连接失败时如何分层排查？

**参考答案：**

先查 DNS、端口和网络，再查 SSH banner、Host Key、认证方式、用户权限，最后查远程命令、PATH、sudo 和文件权限。Paramiko debug 日志可辅助定位协议和认证阶段。

> **重点：** 分层排查比只看异常字符串更可靠。

### 4. 什么时候不应该用 Paramiko？

**参考答案：**

如果需求是大规模配置管理、声明式编排、复杂回滚、强审计和权限治理，应优先考虑 Ansible、Fabric、运维平台或云原生工具。Paramiko 更适合定制底层 SSH/SFTP 自动化。

> **难点：** 选型要看组织流程和规模，不是只看能否写代码实现。
