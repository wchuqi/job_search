## 一、SSH 基础和安全模型

### 1. Paramiko 是什么，适合什么场景？

**参考答案：**

Paramiko 是 Python 实现的 SSHv2 协议库，适合远程命令执行、SFTP 文件传输、跳板机连接、端口转发和定制 SSH 自动化。不适合直接替代完整配置管理和编排系统。

> **重点：** 它是底层 SSH 库，不是 Ansible 这类完整运维平台。

### 2. Host Key 校验解决什么问题？

**参考答案：**

Host Key 校验用于确认连接的是预期服务器，防止中间人攻击。它不同于用户认证，用户认证是服务器确认客户端用户身份。

> **易错：** 以为密码或私钥认证成功就能证明服务器可信。

### 3. `AutoAddPolicy` 为什么不适合生产默认使用？

**参考答案：**

它会自动信任未知服务器 Host Key，降低中间人攻击防护。生产环境应维护 known_hosts 或主机指纹白名单，未知主机需要显式审批。

> **重点：** 安全策略不能为了方便默认放开。

### 4. 密码认证和私钥认证有什么区别？

**参考答案：**

密码认证简单但泄露风险高；私钥认证更适合自动化，但需要保护私钥文件、passphrase 和权限。两者都不应硬编码到脚本或日志。

> **难点：** 私钥泄露同样严重，不能因为不用密码就忽略凭据管理。
