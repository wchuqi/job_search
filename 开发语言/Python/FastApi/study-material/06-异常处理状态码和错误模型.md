# FastAPI学习资料：异常处理、状态码和错误模型

[返回索引](../FastApi学习资料.md)

## 学习目标

- 会使用 `HTTPException`、自定义异常处理器和统一错误响应。
- 能区分校验错误、业务错误、权限错误和系统错误。
- 能设计稳定的错误模型。

## 理论导读

API 错误不是简单返回字符串。客户端需要稳定的状态码和错误结构，便于前端展示、重试、监控和排障。FastAPI 默认会为请求校验失败返回 422，为未捕获异常返回 500。业务错误应显式转换成合适的 HTTP 状态码。

## 核心心智模型

错误响应是 API 契约的一部分。成功响应告诉客户端“拿到了什么”，错误响应告诉客户端“为什么失败、是否能修复、是否能重试”。

## 示例

```python
from fastapi import FastAPI, HTTPException, Request
from fastapi.responses import JSONResponse
from pydantic import BaseModel

app = FastAPI()


class ErrorResponse(BaseModel):
    code: str
    message: str


class DomainError(Exception):
    def __init__(self, code: str, message: str):
        self.code = code
        self.message = message


@app.exception_handler(DomainError)
async def handle_domain_error(request: Request, exc: DomainError):
    return JSONResponse(
        status_code=400,
        content=ErrorResponse(code=exc.code, message=exc.message).model_dump(),
    )


@app.get("/orders/{order_id}")
async def get_order(order_id: int):
    if order_id <= 0:
        raise HTTPException(status_code=404, detail="order not found")
    return {"id": order_id}
```

## 状态码参考

| 状态码 | 场景 |
| --- | --- |
| 400 | 请求语义错误，客户端可修正 |
| 401 | 未认证 |
| 403 | 已认证但无权限 |
| 404 | 资源不存在或不应暴露 |
| 409 | 状态冲突，例如重复创建 |
| 422 | 请求格式或字段校验失败 |
| 500 | 未预期服务端错误 |

## 练习

为订单服务设计错误模型，覆盖库存不足、重复支付、未登录、无权限、订单不存在五类错误。

## 验收

- 能给业务错误选择合适状态码。
- 能写自定义异常处理器。
- 错误响应中包含稳定的 `code`，而不是只靠自然语言 `message`。

## 重点

- 错误结构要稳定，错误文案可以调整。
- 不要把内部异常堆栈直接返回给客户端。

## 难点

- 404 有时用于隐藏资源是否存在，403 有时会泄露资源存在性，需要结合安全策略判断。

## 易错

> **易错：** 所有业务失败都返回 200，然后在 body 里写 `success=false`。
>
> 正确做法：使用 HTTP 状态码表达协议层结果，用业务错误码表达领域失败原因。
