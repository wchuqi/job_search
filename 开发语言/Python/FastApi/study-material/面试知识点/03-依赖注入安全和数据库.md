## 一、依赖注入、安全和数据库

### 1. `Depends` 的作用是什么？

**参考答案：**

`Depends` 用来声明请求处理所需的依赖。FastAPI 会根据函数签名解析依赖树，执行子依赖，把结果注入路由函数。它常用于数据库会话、当前用户、权限检查、配置和外部客户端。

> **重点：** `Depends` 是请求级依赖解析机制，不只是减少重复代码。

### 2. `yield` 依赖适合什么场景？

**参考答案：**

`yield` 依赖适合管理需要清理的请求级资源。例如创建数据库 Session，在 `yield` 前提供资源，在 `finally` 中关闭或回滚资源。

```python
async def get_session():
    async with SessionLocal() as session:
        yield session
```

> **易错：** 在异常处理中吞掉异常，导致请求看似成功但资源状态不一致。

### 3. FastAPI 中如何做认证和授权？

**参考答案：**

认证通常通过安全依赖读取 token 并解析当前用户；授权在当前用户基础上判断角色、权限或资源归属。认证失败常返回 401，已认证但无权限常返回 403。

> **重点：** 认证是“你是谁”，授权是“你能做什么”。

### 4. 数据库 Session 应该如何管理？

**参考答案：**

连接池是应用级资源，Session 是请求级工作单元。通常用依赖为每个请求提供 Session，请求结束后关闭。事务边界应围绕业务一致性设计，而不是随意在路由中提交。

> **难点：** 多 worker 下每个进程都有自己的连接池，部署时要合并估算数据库连接数。

### 5. CORS 是认证机制吗？

**参考答案：**

不是。CORS 是浏览器同源策略相关的跨域访问控制，只限制浏览器环境下的跨域请求。它不能替代 token、cookie、权限校验等后端认证授权机制。

> **易错：** 以为配置了 CORS 就能保护 API。真正的保护必须在服务端做认证授权。
