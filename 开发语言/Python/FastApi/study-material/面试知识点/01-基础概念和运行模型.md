## 一、基础概念和运行模型

### 1. FastAPI 的核心特点是什么？

**参考答案：**

FastAPI 的核心特点是基于标准 Python 类型标注声明 API 契约，并自动完成参数解析、数据校验、序列化和 OpenAPI 文档生成。它建立在 Starlette 和 Pydantic 之上，运行在 ASGI 协议生态中。

> **重点：** 类型标注、Pydantic、OpenAPI、ASGI 是回答关键词。

### 2. FastAPI、Starlette、Pydantic、Uvicorn 分别负责什么？

**参考答案：**

FastAPI 负责 API 层封装、路由声明、依赖注入和文档集成；Starlette 提供底层 ASGI Web 能力；Pydantic 负责数据校验和序列化；Uvicorn 是 ASGI Server，负责监听端口并把请求交给应用。

> **易错：** 说 FastAPI 自己处理所有网络协议和数据校验。更准确的说法是它集成了底层组件。

### 3. ASGI 和 WSGI 有什么区别？

**参考答案：**

WSGI 主要面向同步 HTTP 请求响应模型；ASGI 支持异步、WebSocket、生命周期事件和更通用的连接模型。FastAPI 基于 ASGI，因此能自然支持 async 路由和 WebSocket。

> **难点：** ASGI 支持异步能力，但具体性能还取决于数据库、外部调用和部署配置。

### 4. 一次 FastAPI 请求的处理链路是什么？

**参考答案：**

请求先到 Uvicorn，再进入 FastAPI/Starlette 应用，经过中间件、路由匹配、依赖解析、参数校验，然后执行 path operation，最后进行响应序列化并返回客户端。

> **重点：** 回答中要包含路由匹配、依赖解析和响应序列化。
