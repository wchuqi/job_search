## 一、异步、测试、部署和排障

### 1. `async def` 一定比 `def` 快吗？

**参考答案：**

不一定。`async def` 的优势是等待异步 IO 时能释放事件循环，让同一 worker 处理其他任务。如果函数内部执行阻塞 IO 或 CPU 密集计算，可能反而阻塞事件循环。

> **重点：** async 优化的是等待，不是让 CPU 计算变快。

### 2. `BackgroundTasks` 适合做什么？不适合做什么？

**参考答案：**

适合响应后执行短小、失败影响较低的任务，例如写审计日志。不适合支付、扣库存、可靠消息投递、长任务和需要重试的流程，这些应使用任务队列或事件系统。

> **易错：** 把后台任务当成可靠任务队列。

### 3. FastAPI 如何做接口测试？

**参考答案：**

可以使用 `TestClient` 发起测试请求，用 pytest 组织用例，并通过 `app.dependency_overrides` 覆盖认证、数据库或外部服务依赖。测试应断言状态码、响应结构和副作用。

> **重点：** 依赖覆盖是 FastAPI 测试的关键能力。

### 4. 生产部署 FastAPI 要关注哪些点？

**参考答案：**

要关注 ASGI Server、worker 数、容器资源、反向代理、HTTPS、健康检查、优雅关闭、日志、指标、追踪、数据库连接池和迁移流程。开发 reload 模式不能直接用于生产。

> **难点：** worker 数增加会复制内存和连接池，不是越多越好。

### 5. 一个 FastAPI 接口变慢，你怎么排查？

**参考答案：**

先看整体延迟、错误率和请求量，再分段看路由处理、依赖解析、数据库查询、外部 HTTP 调用、响应序列化和日志耗时。结合请求 ID、结构化日志、指标、tracing、SQL 日志和压测复现。

> **重点：** 先观测再优化，不要直接改 worker 或盲目加缓存。
