# FastAPI学习资料：版本环境和项目结构

[返回索引](../FastApi学习资料.md)

## 学习目标

- 能搭建可维护的 FastAPI 项目。
- 理解 `fastapi dev`、`fastapi run`、`uvicorn` 的使用边界。
- 掌握依赖管理、配置分层和目录拆分原则。

## 理论导读

FastAPI 项目的长期质量，很大程度取决于一开始是否把接口层、业务层、数据访问层、配置和测试拆开。小 demo 可以写在 `main.py`，真实项目应让路由只负责 HTTP 契约，业务规则放在 service，数据库访问放在 repository 或 DAO。

环境建议使用 Python 3.10+，因为现代 FastAPI 文档和示例大量使用 `str | None`、`Annotated` 等写法。安装时优先使用 `fastapi[standard]`，它包含常用运行和开发依赖。

## 核心心智模型

项目结构不是为了“看起来专业”，而是为了控制依赖方向：HTTP 层依赖业务层，业务层依赖数据访问接口，底层不要反向依赖路由。

## 推荐结构

```text
app/
  main.py
  core/
    config.py
    logging.py
  api/
    router.py
    routes/
      users.py
      items.py
  schemas/
    user.py
    item.py
  services/
    user_service.py
  repositories/
    user_repository.py
  db/
    session.py
    models.py
tests/
  test_users.py
pyproject.toml
```

## 常用命令

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install "fastapi[standard]"
fastapi dev app/main.py
fastapi run app/main.py
```

## 最小入口

```python
from fastapi import FastAPI

app = FastAPI(title="Example API", version="0.1.0")


@app.get("/health", tags=["system"])
async def health_check():
    return {"status": "ok"}
```

## 练习

创建一个 `app` 目录，把用户相关接口放入 `api/routes/users.py`，再通过 `api/router.py` 汇总到 `main.py`。

## 验收

- 能本地启动服务并访问 `/docs`。
- 新增一个业务模块时，不需要修改无关模块。
- 配置、路由、Schema、业务逻辑有明确边界。

## 重点

- `main.py` 应该薄，主要负责创建应用、注册路由、中间件和生命周期。
- `pyproject.toml` 中要固定关键依赖版本，生产项目不要无约束地追最新。

## 难点

- 目录结构要服务业务复杂度。过早拆得太细会增加跳转成本，完全不拆会让测试和维护困难。

## 易错

> **易错：** 在路由函数里直接写 SQL、权限判断、业务计算和响应拼装。
>
> 正确做法：路由层处理 HTTP 输入输出，业务层表达规则，数据访问层处理存储细节。
