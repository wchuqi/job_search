# FastAPI学习资料：路由、参数、请求体和响应模型

[返回索引](../FastApi学习资料.md)

## 学习目标

- 掌握路径参数、查询参数、Header、Cookie、Body、Form、File 的声明方式。
- 理解路由匹配顺序和冲突规则。
- 会设计响应模型和状态码。

## 理论导读

FastAPI 通过函数参数判断数据来源。路径模板里的变量来自路径；简单类型默认来自查询参数；Pydantic 模型默认来自请求体；`Header`、`Cookie`、`Body`、`Path`、`Query` 可以显式改变参数元数据和校验约束。

路由按注册顺序匹配。固定路径要放在动态路径前面，例如 `/users/me` 要先于 `/users/{user_id}`，否则 `me` 会被当成 `user_id`。

## 核心心智模型

路由函数签名就是 HTTP 契约。参数名、类型、默认值和额外约束共同决定客户端应该怎么调用。

## 示例

```python
from typing import Annotated

from fastapi import APIRouter, Header, Path, Query, status
from pydantic import BaseModel, Field

router = APIRouter(prefix="/items", tags=["items"])


class ItemCreate(BaseModel):
    name: str = Field(min_length=1, max_length=80)
    price: float = Field(gt=0)


class ItemRead(BaseModel):
    id: int
    name: str
    price: float


@router.get("/featured")
async def featured_items():
    return [{"id": 1, "name": "Keyboard", "price": 299}]


@router.get("/{item_id}", response_model=ItemRead)
async def get_item(
    item_id: Annotated[int, Path(gt=0)],
    trace_id: Annotated[str | None, Header(alias="X-Trace-Id")] = None,
):
    return {"id": item_id, "name": "Mouse", "price": 99, "internal": "hidden"}


@router.post("", response_model=ItemRead, status_code=status.HTTP_201_CREATED)
async def create_item(
    payload: ItemCreate,
    page: Annotated[int, Query(ge=1)] = 1,
):
    return {"id": 10, **payload.model_dump()}
```

## 响应模型的作用

`response_model` 不只是文档。它会参与响应序列化和字段过滤。返回对象中多余字段通常不会暴露给客户端，这能降低敏感字段泄露风险，但不能替代明确的安全设计。

## 练习

设计一个 `POST /orders` 接口，要求：

- 请求体包含商品 ID、数量、收货地址。
- 数量必须大于 0。
- 响应中返回订单 ID、状态和创建时间。
- 内部成本价不能出现在响应里。

## 验收

- 能解释 404、422、400 的区别。
- 能正确安排固定路径和动态路径顺序。
- 能用 `response_model` 控制输出字段。

## 重点

- 路由匹配顺序会影响请求落到哪个处理函数。
- 输入和输出模型要分开设计。

## 难点

- 部分参数默认来源取决于类型和声明方式，复杂接口建议显式使用 `Query`、`Path`、`Body`。

## 易错

> **易错：** 把 `/users/{user_id}` 写在 `/users/me` 前面。
>
> 正确做法：固定路径优先注册，动态路径后注册，并用测试覆盖关键路由。
