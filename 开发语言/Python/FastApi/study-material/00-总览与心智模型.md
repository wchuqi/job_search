# FastAPI学习资料：总览与心智模型

[返回索引](../FastApi学习资料.md)

## 学习目标

- 理解 FastAPI 在 Python Web 技术栈中的位置。
- 建立“类型标注驱动 API 契约”的心智模型。
- 知道 FastAPI、Starlette、Pydantic、Uvicorn、ASGI 各自负责什么。

## 理论导读

FastAPI 是一个基于标准 Python 类型标注构建 API 的框架。它把函数签名、类型标注、Pydantic 模型和路由声明组合起来，用同一份声明完成参数解析、数据校验、序列化、错误响应和 OpenAPI 文档生成。

FastAPI 的底层不是传统 WSGI，而是 ASGI。ASGI 支持异步请求处理、WebSocket、生命周期事件和中间件链。Uvicorn 是常见 ASGI Server，负责监听端口、接收请求并把请求交给应用。Starlette 提供路由、中间件、请求响应对象、后台任务等 Web 基础能力。Pydantic 负责数据校验和序列化。

## 核心心智模型

把 FastAPI 想成一个“API 契约编译器”：你写的路由函数签名是契约源代码，FastAPI 在启动和请求进入时读取这些类型信息，决定参数来自路径、查询、Header、Cookie、Body 还是依赖注入，然后把外部数据转换成 Python 对象，最后把返回值转换成符合响应模型的 JSON。

一次请求的大致路径：

```text
Client -> Uvicorn -> ASGI app -> Middleware -> Router -> Dependencies -> Path operation -> Response serialization -> Client
```

## 知识点详解

### FastAPI 适合什么

适合构建 HTTP API、微服务、后台管理 API、AI 服务包装层、内部工具 API、异步 IO 密集型服务。不适合直接承担复杂前端渲染、重型任务调度、长耗时 CPU 计算或完整企业架构本身。

### 关键组件责任边界

| 组件 | 主要职责 | 常见误解 |
| --- | --- | --- |
| FastAPI | 路由声明、依赖注入、OpenAPI 集成 | 以为它包含 ORM 和任务队列 |
| Starlette | ASGI Web 基础设施 | 忽略中间件和响应对象来自 Starlette |
| Pydantic | 数据校验和序列化 | 把 Pydantic 模型当数据库模型长期复用 |
| Uvicorn | ASGI Server | 把开发 reload 配置用于生产 |
| SQLAlchemy | 数据访问和事务 | 以为 FastAPI 自动管理事务 |

## 例子

```python
from typing import Annotated

from fastapi import FastAPI, Query
from pydantic import BaseModel

app = FastAPI(title="Book API")


class BookCreate(BaseModel):
    title: str
    price: float


@app.post("/books")
async def create_book(
    payload: BookCreate,
    source: Annotated[str | None, Query(max_length=20)] = None,
):
    return {"title": payload.title, "price": payload.price, "source": source}
```

这个例子中，`payload` 来自请求体，`source` 来自查询参数，`BookCreate` 决定请求体校验规则，返回值会被序列化成 JSON。

## 重点

- FastAPI 的核心优势来自 Python 类型标注、Pydantic 校验和 OpenAPI 文档的合并。
- FastAPI 不是“只写装饰器”的框架，真正关键是请求解析、依赖解析和响应序列化。

## 难点

- 同一个类型标注既服务编辑器、运行时校验、文档生成，也影响响应输出。
- async 性能取决于整个调用链是否非阻塞，不是把函数改成 `async def` 就一定更快。

## 易错

> **易错：** 把 Pydantic 模型、数据库 ORM 模型、业务领域对象混为一体。
>
> 正确做法：对外 API Schema、数据库实体和业务对象可以互相转换，但边界要清晰，尤其不要在响应中泄露密码哈希、内部状态或审计字段。
