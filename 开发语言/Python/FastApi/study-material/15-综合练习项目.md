# FastAPI学习资料：综合练习项目

[返回索引](../FastApi学习资料.md)

## 项目目标

实现一个“任务管理 API”，覆盖 FastAPI 的核心生产能力：用户认证、任务 CRUD、分页查询、权限控制、数据库迁移、测试、容器化、健康检查和日志。

## 功能要求

- 用户注册、登录、获取当前用户。
- 任务创建、查询、更新、删除。
- 普通用户只能访问自己的任务。
- 管理员可以查看所有任务。
- 任务列表支持分页和状态筛选。
- 所有错误返回统一错误模型。

## 推荐结构

```text
app/
  main.py
  core/config.py
  core/security.py
  api/router.py
  api/routes/auth.py
  api/routes/tasks.py
  schemas/user.py
  schemas/task.py
  services/auth_service.py
  services/task_service.py
  repositories/task_repository.py
  db/session.py
  db/models.py
tests/
  test_auth.py
  test_tasks.py
migrations/
```

## 技术要求

- 使用 Pydantic v2 模型区分创建、更新和响应。
- 使用 `Depends` 注入当前用户和数据库 Session。
- 使用 Alembic 管理表结构。
- 使用 pytest 覆盖成功和失败路径。
- 提供 Dockerfile 和 `/health`、`/ready`。

## 验收标准

- `pytest` 全部通过。
- 未登录访问任务接口返回 401。
- 访问他人任务返回 403 或 404，并说明选择理由。
- 请求体字段错误返回 422。
- 响应中不包含密码哈希。
- 容器启动后能访问 `/docs`。

## 扩展任务

- 添加刷新 Token。
- 添加请求 ID 日志。
- 添加 Prometheus 指标。
- 添加软删除和审计字段。
- 添加角色和权限表。

## 重点

- 这个项目要练的是“接口契约 + 依赖注入 + 事务边界 + 测试验证”，不是堆功能。

## 难点

- 权限判断既依赖当前用户，也依赖资源归属，最好放在服务层或专门的权限依赖中统一处理。

## 易错

> **易错：** 测试只覆盖创建成功，不覆盖未登录、无权限、字段错误和不存在。
>
> 正确做法：每个核心接口至少覆盖成功路径、认证失败、授权失败、校验失败和资源不存在。
