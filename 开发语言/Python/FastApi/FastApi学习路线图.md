# FastAPI学习路线图

## 阶段 1：Web API 和 FastAPI 基础认知

- 目标：理解 FastAPI 解决什么问题，以及它和 Python 类型标注、Pydantic、Starlette、Uvicorn 的关系。
- 需要掌握：HTTP 基础、REST API、ASGI、自动文档、路径参数、查询参数、请求体、响应模型。
- 例子：实现 `/health`、`/users/{user_id}`、`POST /users` 三个接口，并在 `/docs` 中调试。
- 练习：把一个字典列表改造成带 Pydantic 模型校验的接口。
- 验收：能解释一次请求从客户端到路由函数再到 JSON 响应的过程。
- 重点：FastAPI 是 API 框架，不是 ORM、任务队列或完整业务架构。
- 易错：只会写装饰器，不理解参数来源和校验规则。

## 阶段 2：请求响应、Pydantic v2 和 OpenAPI

- 目标：掌握输入校验、输出裁剪、错误响应、文档生成和 Schema 设计。
- 需要掌握：`BaseModel`、`Field`、`Annotated`、请求体嵌套、响应模型、状态码、`HTTPException`、OpenAPI 元数据。
- 例子：为商品接口添加创建、更新、分页查询和错误模型。
- 练习：把内部数据库模型和外部 API 模型拆开，避免泄露敏感字段。
- 验收：能独立设计一个稳定的 API 契约，并用自动文档验证。
- 重点：输入模型、输出模型、领域模型不要长期混用。
- 易错：以为 `response_model` 只是文档，忽略它还会做序列化过滤。

## 阶段 3：依赖注入、资源生命周期和模块化

- 目标：用 `Depends` 管理认证、数据库会话、配置、权限和可复用业务上下文。
- 需要掌握：依赖树、子依赖、缓存、`yield` 依赖、`APIRouter`、生命周期事件、配置注入。
- 例子：实现 `get_current_user`、`get_db`、`require_admin` 依赖。
- 练习：把一个单文件 API 拆成 `api/`、`schemas/`、`services/`、`repositories/`。
- 验收：能解释依赖解析顺序、每请求缓存和资源释放顺序。
- 重点：依赖注入是 FastAPI 的核心扩展机制。
- 难点：依赖既能表达参数来源，也能表达资源边界和权限规则。
- 易错：在依赖中吞掉异常，导致事务、认证或日志链路失真。

## 阶段 4：数据库、安全、测试和生产化

- 目标：完成一个可测试、可部署、可观察的真实服务。
- 需要掌握：SQLAlchemy 2.x、Alembic、事务边界、OAuth2/JWT、CORS、pytest、TestClient、容器镜像、健康检查、结构化日志。
- 例子：实现用户注册登录、资源 CRUD、分页查询、权限控制和迁移脚本。
- 练习：写单元测试、接口测试和数据库回滚测试。
- 验收：本地能跑测试，容器能启动，健康检查可用，异常日志可定位请求。
- 重点：生产服务要关注超时、连接池、限流、日志、指标和升级兼容。
- 易错：把开发服务器配置直接用于生产，或在 async 接口中调用阻塞 IO。

## 阶段 5：性能、排障和源码机制

- 目标：能定位慢接口、连接池耗尽、事件循环阻塞、依赖解析错误和部署故障。
- 需要掌握：事件循环、线程池、worker 进程、连接池、OpenAPI 生成、Starlette 中间件、Pydantic 序列化成本。
- 例子：为慢查询接口添加指标和超时，比较 sync/async 数据库调用差异。
- 练习：制造并修复路由顺序冲突、依赖资源泄露、CORS 错误和 422 校验错误。
- 验收：能用日志、请求 ID、指标和测试复现问题，并写出修复方案。
- 难点：区分框架问题、业务问题、数据库问题和部署问题。
- 易错：看到 FastAPI 性能高，就忽略数据库、网络和序列化才是常见瓶颈。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 路由、请求响应、Pydantic | 一个内存版 CRUD API |
| 第 2 周 | 依赖注入、模块化、异常处理 | 一个分层 API 项目 |
| 第 3 周 | 数据库、认证、安全 | 一个带登录和权限的服务 |
| 第 4 周 | 测试、部署、排障 | 一个可容器化的生产模板 |

## 最终能力清单

- 能独立搭建 FastAPI 项目结构。
- 能设计稳定的请求、响应、错误和 OpenAPI 契约。
- 能使用依赖注入管理认证、权限、数据库和配置。
- 能处理 async/sync 边界，避免阻塞事件循环。
- 能写接口测试、依赖覆盖测试和数据库测试。
- 能完成容器化、健康检查、日志、指标和基础部署。
- 能回答常见面试题，并能结合项目经验解释取舍。
