# Python学习路线图

本路线图面向 Python 3.x 学习者，目标是从能写脚本逐步达到能写可维护项目、能排查问题、能解释核心机制的水平。

## 阶段 1：基础认知

- 目标：能安装 Python、运行脚本、理解解释器、缩进、变量、表达式和基本输入输出。
- 需要掌握：Python 3.x 版本差异、REPL、脚本执行、`python -m`、缩进语法、注释、基础类型、`print`、`input`。
- 例子：写一个根据用户输入计算 BMI 的脚本。
- 练习：把摄氏温度转换为华氏温度，并处理非法输入。
- 验收：能独立解释“源代码、解释器、字节码、运行时对象”之间的关系。
- 重点：Python 是动态类型语言，变量名绑定对象，不是固定内存盒子。
- 易错：把 Python 2 教程里的 `print x`、默认字符串编码和整数除法规则套到 Python 3。

## 阶段 2：核心语法和数据结构

- 目标：能熟练使用 `list`、`tuple`、`dict`、`set`、字符串、控制流和函数组织程序。
- 需要掌握：切片、解包、可变与不可变、浅拷贝与深拷贝、`if`、`for`、`while`、函数参数、作用域、异常基础。
- 例子：读取一组订单记录，按用户统计总金额。
- 练习：实现一个去重、排序、分组的小工具。
- 验收：能说明列表和字典的适用场景，能避免默认可变参数陷阱。
- 重点：数据结构选择决定代码复杂度和性能边界。
- 易错：用列表做大量成员判断，导致本该 O(1) 的查找变成 O(n)。

## 阶段 3：抽象能力

- 目标：能用函数、类、模块和包拆分复杂问题。
- 需要掌握：闭包、装饰器、上下文管理器、类、继承、组合、特殊方法、模块导入、包结构。
- 例子：把一个脚本拆成 `parser.py`、`service.py`、`cli.py` 三个模块。
- 练习：实现一个支持命令行参数和配置文件的任务清单工具。
- 验收：能解释 `import` 的查找顺序，能判断何时用类、何时用普通函数。
- 重点：抽象的目的不是显得高级，而是隔离变化和降低重复。
- 易错：过早继承，导致类层级比业务本身还复杂。

## 阶段 4：工程化和标准库

- 目标：能创建虚拟环境、管理依赖、写测试、记录日志、排查异常和维护项目结构。
- 需要掌握：`venv`、`pip`、`pyproject.toml`、`pathlib`、`json`、`csv`、`argparse`、`logging`、`unittest` 或 `pytest`、格式化和静态检查。
- 例子：写一个读取 CSV、清洗数据并输出 JSON 报告的 CLI 项目。
- 练习：为项目补充单元测试、错误日志和 README。
- 验收：能在新机器上根据说明复现运行环境。
- 重点：可维护 Python 项目依赖明确、入口清晰、测试可重复。
- 易错：把绝对路径、密钥、临时文件名硬编码进业务代码。

## 阶段 5：进阶运行机制

- 目标：理解迭代协议、生成器、异步 IO、并发模型、GIL、内存和性能分析。
- 需要掌握：迭代器协议、生成器暂停恢复、`async`/`await`、线程、进程、协程、`cProfile`、缓存、惰性计算。
- 例子：用生成器流式处理大文件，用 `asyncio` 并发请求多个接口。
- 练习：比较同步、线程池、进程池和异步版本的耗时差异。
- 验收：能为 IO 密集和 CPU 密集任务选择合适模型。
- 重点：并发不是自动变快，瓶颈位置决定方案。
- 易错：在协程里调用阻塞函数，导致整个事件循环卡住。

## 阶段 6：生产实践和面试准备

- 目标：能写可靠、可诊断、可交付的 Python 代码，并能在面试中清楚解释机制。
- 需要掌握：配置管理、异常边界、日志上下文、安全输入、依赖锁定、版本兼容、部署脚本、常见面试题。
- 例子：为 CLI 工具加入配置文件、日志等级、退出码和测试数据。
- 练习：完成一个数据处理或自动化综合项目，并写出设计说明。
- 验收：别人能根据文档安装、运行、测试、定位错误。
- 重点：生产代码不仅要能跑，还要能被复现、观察、升级和回滚。
- 易错：只在本地“刚好能跑”，没有记录依赖版本和运行入口。

## 阶段 7：深入机制和源码级理解

- 目标：能解释 Python 高级机制背后的运行规则，并能把这些规则用于排障、性能优化和框架源码阅读。
- 需要掌握：CPython 字节码、code object、frame、引用计数、循环 GC、描述符、元类、类型系统、导入 finder/loader、依赖解析、事件循环、取消、背压、可观测性。
- 例子：用 `dis` 分析函数，用自定义描述符实现字段校验，用 `tracemalloc` 定位内存分配热点。
- 练习：实现一个插件注册系统、一个类型友好的报告生成模块、一个带并发限制和超时的异步任务调度器。
- 验收：能回答“为什么这样运行”，而不只是“代码怎么写”。
- 重点：高级机制要服务实际问题：框架理解、排障、性能、可靠性。
- 易错：为了炫技使用元类、魔法方法和异步，反而让业务代码更难维护。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 环境、基础语法、类型、控制流 | 3 到 5 个小脚本 |
| 第 2 周 | 函数、数据结构、异常、文件 IO | 一个文本或 CSV 处理工具 |
| 第 3 周 | OOP、模块包、迭代器、装饰器 | 一个拆分模块的小项目 |
| 第 4 周 | 标准库、虚拟环境、测试、日志 | 一个可安装、可测试的 CLI 项目 |
| 第 5 周 | 并发、异步、性能、排障 | 一个带性能对比的实验报告 |
| 第 6 周 | 综合项目和面试复盘 | 项目说明、测试报告、面试题答案 |
| 第 7 周 | CPython 运行时、描述符、类型系统、导入系统 | 深入机制笔记和最小验证代码 |
| 第 8 周 | 并发异步、性能、安全和生产可靠性 | 性能报告、排障清单、生产化改造 |

## 最终能力清单

- 能解释 Python 3.x 运行代码的基本路径：源码、编译、字节码、解释执行、对象模型。
- 能正确选择和使用内置数据结构。
- 能写清晰的函数、类、模块和包。
- 能用异常、日志、测试和类型提示提高可维护性。
- 能处理文件、命令行、JSON、CSV、时间、路径和网络请求等常见任务。
- 能理解导入机制、作用域、迭代协议、上下文管理器和装饰器。
- 能为 IO 密集、CPU 密集、大文件处理选择合适方案。
- 能解释 CPython 对象生命周期、描述符优先级、事件循环调度和依赖解析。
- 能做基础性能剖析、内存排查、日志设计和生产配置管理。
- 能回答 Python 基础、工程化、运行机制和性能相关面试题。
