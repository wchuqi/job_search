## 理论导读

数据管道的风险不只在代码异常，更在“代码正常运行但结果错误”。Pandas 项目应检查输入字段、dtype、主键唯一性、行数变化、缺失率、金额汇总和输出 Schema。

## 示例

```python
import pandas as pd
from pandas.testing import assert_frame_equal


def clean_orders(df: pd.DataFrame) -> pd.DataFrame:
    out = df.copy()
    out["amount"] = pd.to_numeric(out["amount"], errors="coerce")
    return out.dropna(subset=["amount"])


def test_clean_orders():
    raw = pd.DataFrame({"amount": ["10", "bad"]})
    expected = pd.DataFrame({"amount": [10.0]}, index=[0])
    result = clean_orders(raw)
    assert_frame_equal(result, expected)
```

## 工程化清单

- 读取函数、清洗函数、聚合函数、导出函数拆开。
- 每步输入输出有字段契约。
- merge 前后检查行数和主键。
- groupby 指标有口径说明。
- 测试覆盖缺失、重复、异常类型和空输入。
- 记录数据日期、来源和版本。

## 重点

## 难点

>
