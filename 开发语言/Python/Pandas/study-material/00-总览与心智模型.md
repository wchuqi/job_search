## 学习目标

- 理解 Pandas 在 Python 数据生态中的位置。
- 建立 DataFrame、Series、Index 和对齐机制的心智模型。
- 知道 Pandas 与 NumPy、SQL、Excel 的边界。

## 理论导读

Pandas 是面向表格数据的分析库。它在 NumPy 数组之上增加了行列标签、索引对齐、缺失值处理、分组聚合、关联、时间序列和丰富 IO 能力。它非常适合中小规模单机内存数据处理、探索分析、报表加工和特征工程。

Pandas 不是数据库，也不是分布式计算引擎。数据量超过单机内存、需要复杂事务、多人并发写入或强一致查询时，应考虑数据库、Spark、DuckDB、Polars、Dask 等工具。

## 核心心智模型

Pandas 的核心不是二维数组，而是“带标签的表格对象”：

```text
DataFrame = 多个带 dtype 的列 + 行 Index + 列 Index + 对齐规则
Series = 一列数据 + Index
```

当两个 Series 或 DataFrame 运算时，Pandas 通常按标签对齐，而不是只按位置对齐。这是它比 NumPy 更适合业务数据的关键，也是不少 bug 的来源。

## 例子

```python
import pandas as pd

a = pd.Series([10, 20], index=["x", "y"])
b = pd.Series([1, 2], index=["y", "z"])

print(a + b)
```

结果中只有 `y` 能对齐，`x` 和 `z` 会得到缺失值。

2. 设置 `order_id` 为索引。
3. 创建一个折扣 Series，观察按索引相加的结果。

## 验收

- 能解释 Series、DataFrame、Index 的关系。
- 能说明 Pandas 和 NumPy 的主要差异。
- 能说出索引对齐的好处和风险。

## 重点

- Pandas 的很多行为都围绕 Index 和 dtype 展开。
- 做真实数据处理前，先看 `shape`、`columns`、`dtypes`、缺失值和唯一键。

## 难点

- Pandas 自动对齐很强大，但当索引不符合预期时，会悄悄产生缺失值或行数变化。

## 易错

> **易错：** 把 Pandas 当 Excel 的代码版，只按视觉表格理解。
>
> 正确做法：始终关注索引、字段类型、主键、缺失值和数据规模。
