# Pandas学习资料：综合练习项目

[返回索引](../Pandas学习资料.md)

## 项目目标

实现一个“电商销售分析数据管道”，覆盖数据读取、清洗、关联、聚合、时间序列指标、报表导出、测试和性能优化。

## 数据表

- `orders.csv`：订单 ID、用户 ID、商品 ID、金额、状态、下单时间。
- `users.csv`：用户 ID、城市、注册时间、会员等级。
- `products.csv`：商品 ID、品类、成本价、售价。

## 功能要求

- 读取并显式指定关键 dtype。
- 清洗金额、状态、日期和缺失值。
- 检查订单 ID 唯一。
- 关联用户和商品信息。
- 按月、城市、品类统计销售额、订单数、客单价和毛利。
- 计算 7 日滚动销售额。
- 导出 Excel 和 Parquet。
- 编写测试覆盖核心逻辑。

## 验收标准

- 所有输入表字段存在且 dtype 符合契约。
- merge 前后订单行数不异常。
- 销售额汇总与原始订单金额一致。
- 缺失值处理有明确口径。
- 测试覆盖重复订单、非法金额、缺失用户、重复商品 ID。
- 输出报表无业务无关索引列。

## 扩展任务

- 支持分块读取大订单表。
- 添加数据质量报告。
- 添加同比、环比和留存指标。
- 使用 DuckDB 或 Polars 改写大数据版本。

## 重点

- 这个项目的核心是数据口径和质量校验，不是简单调用 Pandas API。

## 难点

- merge、groupby 和时间序列指标都可能改变粒度，要持续检查行数、主键和汇总金额。

## 易错

> **易错：** 报表数值看起来合理就认为正确。
>
> 正确做法：用自动化断言校验行数、唯一键、缺失率、金额守恒和输出字段。
