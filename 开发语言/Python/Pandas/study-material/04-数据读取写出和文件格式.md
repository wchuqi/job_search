# Pandas学习资料：数据读取、写出和文件格式

[返回索引](../Pandas学习资料.md)

## 学习目标

- 掌握 CSV、Excel、Parquet、JSON、SQL 的读写。
- 会控制 dtype、日期解析、列选择和分块读取。
- 能选择合适文件格式。

## 理论导读

IO 是 Pandas 项目最容易出错的环节。编码、分隔符、表头、日期、金额、空字符串、类型推断都会影响后续结果。生产数据管道应尽量显式声明关键字段类型和解析规则。

## 示例

```python
import pandas as pd

df = pd.read_csv(
    "orders.csv",
    usecols=["order_id", "user_id", "amount", "created_at"],
    dtype={"order_id": "string", "user_id": "Int64"},
    parse_dates=["created_at"],
)

df.to_parquet("orders.parquet", index=False)
```

## 格式比较

| 格式 | 优点 | 风险 |
| --- | --- | --- |
| CSV | 通用、可读 | dtype 易丢、体积大、慢 |
| Excel | 业务友好 | 多表头、格式混乱、慢 |
| Parquet | 列式、压缩、保留类型 | 需要额外依赖和生态约定 |
| JSON | 半结构化友好 | 结构不统一时处理复杂 |
| SQL | 适合查询和过滤 | 需要连接、权限、SQL 设计 |

## 分块读取

```python
total = 0
for chunk in pd.read_csv("orders.csv", chunksize=100_000):
    total += chunk["amount"].sum()
```

## 练习

读取一个百万行 CSV，只保留必要列，指定 dtype，按分块统计总金额。

## 验收

- 能读写常见格式。
- 能解释 CSV 和 Parquet 的差异。
- 能用 `chunksize` 降低内存压力。

## 重点

- 读取阶段就要控制 dtype 和日期解析，不要把所有问题留到后面。

## 难点

- Excel 里视觉上相同的单元格，底层可能是字符串、数字、日期或空值。

## 易错

> **易错：** 订单号、手机号等 ID 被读成数字后丢失前导 0。
>
> 正确做法：ID 类字段通常按字符串读取。
