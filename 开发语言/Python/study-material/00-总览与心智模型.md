# Python学习资料：总览与心智模型

[返回索引](../Python学习资料.md)

## 学习目标

- 明确 Python 3.x 的定位、适用场景和边界。
- 建立“变量绑定对象”“解释器执行字节码”“标准库优先”的核心心智模型。
- 配好可复现的本地环境，能运行脚本、模块和交互式代码。

## 理论导读

Python 是一门强调可读性和开发效率的通用编程语言。它常用于自动化脚本、Web 后端、数据处理、测试工具、运维工具、机器学习周边工程和胶水代码。Python 的优势不是每一行都最快，而是能用较少代码把业务流程、数据处理和系统能力连接起来。

学习 Python 3.x 要先把它当成一个运行时系统，而不只是语法集合。你写的 `.py` 文件会被解释器读取，编译成字节码，再由虚拟机执行。运行过程中，变量名指向对象，对象有类型和值，很多行为由对象自己的特殊方法决定。

## 核心心智模型

把 Python 程序想成一个仓库管理系统：变量名是标签，对象是货架上的货物，解释器是调度员。给变量赋值不是把货物复制进标签，而是把标签贴到某个对象上。多个标签可以贴在同一个对象上，所以可变对象会带来“我改了 A，B 也变了”的现象。

## 知识点详解

| 主题 | 需要理解的内容 |
| --- | --- |
| Python 3.x | 现代 Python 主线版本，字符串默认是 Unicode，`print` 是函数，整数除法规则和 Python 2 不同。 |
| CPython | 最常用的 Python 实现，很多性能、GIL、对象引用计数讨论默认指 CPython。 |
| 解释执行 | Python 会先编译为字节码，再由虚拟机执行，不是简单逐字符解释。 |
| REPL | 交互式环境，适合快速验证表达式和 API 行为。 |
| 标准库 | Python 自带的大量模块，优先用标准库解决路径、文件、JSON、日志、并发等常见问题。 |

## 例子

```python
name = "Ada"
other = name
name = "Grace"

print(other)  # Ada，other 仍然绑定原来的字符串对象
```


## 练习

1. 安装 Python 3.10 或更高版本。
2. 创建一个虚拟环境并激活。
3. 写 `hello.py`，输出当前日期、Python 版本和当前工作目录。
4. 分别用 `python hello.py` 和 `python -m hello` 运行，观察差异。

## 验收

- 能说明 Python 3.x 与 Python 2 的关键差异。
- 能解释变量名和对象之间的绑定关系。
- 能创建、激活、退出虚拟环境。
- 能区分脚本执行、模块执行和 REPL 验证。

## 重点

- 以 Python 3.x 为准，不混用 Python 2 教程。
- 变量保存的是对象引用，赋值通常不是复制对象。
- 学习标准库能显著减少重复造轮子。

## 难点

- 运行时对象模型会影响可变对象、参数传递、作用域、拷贝和缓存。
- 不同 Python 实现可能有差异，面试或性能分析里默认要说明是否讨论 CPython。

## 易错

> **易错：** 把虚拟环境当成可选项，所有项目都装进全局 Python。
>
> 正确做法：每个项目创建独立虚拟环境，并记录依赖来源。

> **易错：** 认为 Python “完全不编译”。
>
> 正确做法：理解 Python 会编译为字节码，只是编译过程通常由解释器自动完成。
