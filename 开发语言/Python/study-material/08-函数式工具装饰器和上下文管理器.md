# Python学习资料：函数式工具、装饰器和上下文管理器

[返回索引](../Python学习资料.md)

## 学习目标

- 理解函数是一等对象，可以赋值、传参和返回。
- 掌握闭包、装饰器、`functools` 常用工具。
- 理解上下文管理器协议和 `contextlib`。

## 理论导读

Python 中函数也是对象。这意味着函数可以作为参数传给另一个函数，也可以作为返回值。装饰器利用这一点，在不修改原函数主体的情况下添加日志、鉴权、缓存、计时、重试等横切逻辑。

上下文管理器用于管理“进入和退出”成对出现的资源或状态，例如文件、锁、数据库事务、临时目录。`with` 语句会调用对象的 `__enter__` 和 `__exit__` 方法，确保退出逻辑被执行。

## 核心心智模型

装饰器像给函数外面套一层透明外壳：调用者仍然调用同一个名字，但真正执行时会先经过外壳。上下文管理器像有门禁的房间：进入时登记，离开时清理。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 一等函数 | 函数可赋值、传参、放入容器、作为返回值。 |
| 闭包 | 内层函数引用外层函数的变量。 |
| 装饰器 | 接收函数并返回新函数的可调用对象。 |
| `functools.wraps` | 保留原函数元数据，避免调试和文档混乱。 |
| `lru_cache` | 对纯函数结果做缓存。 |
| 上下文管理器 | 实现 `__enter__` 和 `__exit__`，或用 `contextlib.contextmanager`。 |

## 例子

装饰器计时：

```python
from functools import wraps
from time import perf_counter

def timing(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = perf_counter()
        try:
            return func(*args, **kwargs)
        finally:
            cost = perf_counter() - start
            print(f"{func.__name__} cost {cost:.4f}s")
    return wrapper

@timing
def work():
    return sum(range(1_000_000))
```

上下文管理器：

```python
from contextlib import contextmanager

@contextmanager
def section(name):
    print(f"enter {name}")
    try:
        yield
    finally:
        print(f"leave {name}")

with section("load"):
    print("loading data")
```

## 练习

写一个 `@retry(times=3)` 装饰器，当被装饰函数抛出指定异常时最多重试 3 次。要求保留原函数名称，并能配置要捕获的异常类型。

## 验收

- 能解释装饰器调用顺序。
- 能写带参数的装饰器。
- 能用 `functools.wraps` 保留元数据。
- 能用上下文管理器封装资源清理。

## 重点

- 装饰器适合横切逻辑，不适合隐藏核心业务流程。
- 闭包捕获变量绑定，循环中使用时要小心。
- `with` 是异常情况下仍执行清理逻辑的重要工具。

## 难点

- 带参数装饰器有三层函数：接收参数、接收函数、执行包装逻辑。
- `__exit__` 返回 `True` 会吞掉异常，除非明确需要，否则不要这样做。

## 易错

> **易错：** 写装饰器不使用 `functools.wraps`，导致函数名、文档、签名丢失。
>
> 正确做法：包装函数上加 `@wraps(func)`。

> **易错：** 用装饰器隐藏过多控制流，让调用者看不出函数会重试、缓存或吞异常。
>
> 正确做法：装饰器命名清楚，副作用写进文档和测试。
