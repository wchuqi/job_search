# Python学习资料：描述符、属性查找和元类

[返回索引](../Python学习资料.md)

## 学习目标

- 理解普通属性、类属性、实例属性、描述符和 `property` 的查找规则。
- 掌握数据描述符和非数据描述符的优先级差异。
- 理解方法绑定、`staticmethod`、`classmethod` 的底层原因。
- 知道元类解决什么问题，以及什么时候不应该使用元类。

## 理论导读

Python 的“点号访问”不是简单地从字典里取值。`obj.attr` 背后有一套明确的属性解析规则：数据描述符、实例字典、非数据描述符、类属性、父类链、`__getattr__`。ORM 字段、`property`、方法绑定、缓存属性、依赖注入框架，都大量依赖这套机制。

元类则是“创建类对象的类”。普通对象由类创建，类对象由元类创建。大多数业务代码不需要元类，但理解它能帮助你读懂框架源码，尤其是 ORM、序列化、配置声明、插件注册系统。

## 核心心智模型

把 `obj.attr` 想成一次带优先级的查询：

1. 类型上是否有数据描述符。
2. 实例自己有没有这个属性。
3. 类型上是否有非数据描述符或普通类属性。
4. 父类是否有。
5. 找不到时是否定义了 `__getattr__` 兜底。

这个顺序解释了为什么 `property` 能拦截实例属性，为什么普通方法访问后会变成绑定方法。

## 属性查找优先级

简化规则如下：

| 优先级 | 位置 | 说明 |
| --- | --- | --- |
| 1 | 数据描述符 | 定义了 `__get__` 且定义了 `__set__` 或 `__delete__`。 |
| 2 | 实例 `__dict__` | 实例自己的属性。 |
| 3 | 非数据描述符 | 只定义 `__get__`，普通函数方法属于这一类。 |
| 4 | 类属性和父类属性 | 按 MRO 查找。 |
| 5 | `__getattr__` | 前面都失败后调用。 |

## 描述符协议

描述符是实现了以下任一方法的对象：

- `__get__(self, instance, owner)`
- `__set__(self, instance, value)`
- `__delete__(self, instance)`

```python
class Positive:
    def __set_name__(self, owner, name):
        self.name = "_" + name

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.name)

    def __set__(self, instance, value):
        if value <= 0:
            raise ValueError("must be positive")
        setattr(instance, self.name, value)

class Product:
    price = Positive()

    def __init__(self, price):
        self.price = price
```

这里 `Product.price` 是描述符。给 `self.price` 赋值时，会调用 `Positive.__set__`，从而统一校验。

> **重点：** 描述符让“属性访问”可以变成“带逻辑的访问”。

## `property` 的本质

`property` 是内置数据描述符。它把方法包装成属性访问接口，常用于只读属性、延迟计算、校验和兼容旧接口。

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    @property
    def area(self):
        return self.width * self.height
```

适合使用 `property`：

- 计算成本低或结果应实时反映状态。
- 需要在不改变调用方写法的情况下添加校验。
- 想把概念暴露为属性，而不是动作。

不适合使用 `property`：

- 计算很慢或有网络 IO。
- 有明显副作用。
- 失败模式复杂，调用者需要明确知道这是一次操作。

## 方法绑定

类中定义的普通函数是非数据描述符。通过实例访问时，函数的 `__get__` 会返回绑定方法，把实例自动绑定为第一个参数 `self`。

```python
class User:
    def hello(self):
        return "hi"

u = User()
print(User.hello)  # function
print(u.hello)     # bound method
```

`staticmethod` 关闭自动绑定；`classmethod` 绑定类对象为第一个参数。

```python
class User:
    @staticmethod
    def validate_name(name):
        return bool(name)

    @classmethod
    def anonymous(cls):
        return cls("anonymous")
```

## `__getattribute__` 和 `__getattr__`

- `__getattribute__`：每次属性访问都会调用，极其底层，写错容易无限递归。
- `__getattr__`：只有正常查找失败时才调用，适合懒加载、代理、兼容别名。

```python
class Config:
    def __init__(self, data):
        self._data = data

    def __getattr__(self, name):
        try:
            return self._data[name]
        except KeyError as exc:
            raise AttributeError(name) from exc
```

> **易错：** 在 `__getattribute__` 里直接访问 `self.x`，触发无限递归。应使用 `object.__getattribute__(self, "x")`。

## 元类基础

类也是对象。创建一个类时，Python 会调用元类创建类对象。默认元类是 `type`。

```python
class User:
    pass

print(type(User))  # <class 'type'>
print(type(User()))  # <class '__main__.User'>
```

元类常见用途：

- 创建类时校验类定义。
- 自动注册子类。
- 自动收集字段声明。
- 修改类命名空间。

```python
class RegistryMeta(type):
    registry = {}

    def __new__(mcls, name, bases, namespace):
        cls = super().__new__(mcls, name, bases, namespace)
        if name != "Plugin":
            mcls.registry[name] = cls
        return cls

class Plugin(metaclass=RegistryMeta):
    pass

class CsvPlugin(Plugin):
    pass
```

## `__init_subclass__`：元类的轻量替代

很多“子类注册”场景不需要元类，可以用 `__init_subclass__`。

```python
class Plugin:
    registry = {}

    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        Plugin.registry[cls.__name__] = cls

class JsonPlugin(Plugin):
    pass
```

> **重点：** 能不用元类就不用元类。`__init_subclass__`、类装饰器、描述符往往更简单。

## 类创建顺序

简化流程：

1. 解析类体。
2. 准备类命名空间。
3. 执行类体代码，把方法和属性放入命名空间。
4. 调用元类创建类对象。
5. 描述符收到 `__set_name__` 通知。
6. 调用父类的 `__init_subclass__`。
7. 类名绑定到模块命名空间。

这个顺序解释了为什么类体里的代码会在导入时执行，也解释了声明式框架如何在类创建时收集字段。

## 验收

- 能说清数据描述符和非数据描述符的优先级。
- 能解释普通方法为什么访问后会自动绑定 `self`。
- 能判断何时使用 `property`、描述符、类装饰器、`__init_subclass__` 或元类。
- 能读懂 ORM 字段声明背后的基本机制。

## 重点

- 属性访问是 Python 对象模型的核心机制，不是简单字典查询。
- 描述符是 `property`、方法绑定、ORM 字段、缓存属性的基础。
- 元类强大但成本高，优先考虑更简单的替代方案。

## 难点

- 描述符优先级容易和实例属性混淆。
- `__getattribute__` 太底层，容易导致递归和难排查问题。
- 元类冲突在多继承框架中可能很复杂。

## 易错

> **易错：** 用元类解决普通装饰器、注册表、工厂函数就能解决的问题。
>
> 正确做法：优先选择最简单、调用者最容易理解的机制。

> **易错：** 把 `property` 写成有网络请求或数据库查询的重操作。
>
> 正确做法：重操作用显式方法，让调用者知道成本和失败模式。
