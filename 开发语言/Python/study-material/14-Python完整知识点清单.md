## 三、查找、选择和调度规则

| 机制 | 决策规则 | 容易忽略的边界 |
| --- | --- | --- |
| 名字查找 | 按 LEGB 查找：Local、Enclosing、Global、Built-in。 | 闭包捕获变量绑定，循环闭包常出错。 |
| 属性查找 | 先实例，再类，再按 MRO 查找父类。 | 类属性可变对象会被实例共享。 |
| 模块查找 | 按 `sys.path` 顺序查找，已加载模块缓存在 `sys.modules`。 | 本地文件可能遮蔽标准库或第三方库。 |
| 运算派发 | 内置语法调用对象特殊方法，如 `len` 调 `__len__`。 | 特殊方法通常从类型上查找，不等同普通属性访问。 |
| 异常传播 | 沿调用栈向上传播，直到被匹配的 `except` 捕获。 | 裸 `except` 会隐藏根因。 |
| 迭代调度 | `for` 调用 `iter()` 和 `next()`，直到 `StopIteration`。 | 迭代器被消费后不能自动重置。 |
| 异步调度 | 事件循环在 `await` 点切换任务。 | 阻塞调用会卡住整个事件循环。 |

## 四、日常操作和工作流

- 创建虚拟环境：`python -m venv .venv`。
- 安装依赖：`python -m pip install 包名`。
- 运行模块：`python -m 包名.模块名`。
- 检查语法：`python -m py_compile 文件.py`。
- 读写文件：优先 `pathlib.Path`，文本显式 `encoding="utf-8"`。
- 处理数据：优先标准库 `json`、`csv`、`collections`、`itertools`。
- 写 CLI：优先 `argparse`。
- 记录日志：使用 `logging.getLogger(__name__)`。
- 写测试：用 `unittest` 或 `pytest` 覆盖正常、异常、边界路径。

## 六、排障和恢复

| 问题 | 排查方向 |
| --- | --- |
| `ModuleNotFoundError` | 检查虚拟环境、`sys.path`、包是否安装、运行方式是否为 `python -m`。 |
| 导入了错误模块 | 打印 `module.__file__`，排查同名文件遮蔽。 |
| 编码错误 | 明确文件编码，检查输入源，避免依赖系统默认编码。 |
| 路径错误 | 使用 `pathlib.Path.cwd()`、绝对路径和配置项定位。 |
| 依赖冲突 | 新建干净虚拟环境，检查版本约束和 Python 版本。 |
| 性能差 | 先测量，再检查算法、IO 次数、数据结构和并发模型。 |
| 内存暴涨 | 避免一次性读入大文件，使用生成器和流式处理。 |
| 异步任务卡住 | 检查协程内是否调用同步阻塞函数，检查任务是否缺少超时。 |
| 进程池失败 | 检查函数是否可 pickle，Windows 是否有 `__main__` 保护。 |
| 日志泄密 | 检查异常日志、请求头、配置对象和 Token 是否脱敏。 |

## 七、配置、环境和版本兼容

- 明确支持的 Python 版本，例如 `requires-python = ">=3.10"`。
- 不在代码中硬编码密钥、绝对路径、临时目录和环境专属配置。
- 使用环境变量或配置文件传入部署差异。
- 第三方依赖要记录版本，生产项目要有升级和回滚策略。
- 注意不同系统路径分隔符、换行符、默认编码和 shell 行为差异。

## 八、安全、风险和生产实践

- 不对不可信输入使用 `eval` 或 `exec`。
- 不对不可信数据使用 `pickle.loads`。
- 处理路径输入时防止目录穿越。
- 日志不要输出密码、Token、身份证号等敏感信息。
- 临时文件使用 `tempfile`。
- 命令执行使用结构化参数，避免拼接 shell 字符串。
- 外部请求要设置超时、重试和错误边界。
- 生产入口要有退出码、日志、配置、健康检查或可观测信息。

## 九、面试和评估标准

- 能用代码验证基础结论，例如可变默认参数、闭包捕获、浅拷贝。
- 能解释导入、作用域、属性查找、迭代协议、上下文协议的规则。
- 能解释描述符优先级、普通方法绑定、`property` 的本质。
- 能解释 CPython 引用计数、循环 GC 和 GIL 的适用边界。
- 能判断并发模型适用场景。
- 能描述一个 Python 项目的目录、依赖、测试和运行入口。
- 能从异常栈、日志和最小复现出发定位问题。
