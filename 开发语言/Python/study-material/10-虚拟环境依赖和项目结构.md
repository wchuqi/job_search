# Python学习资料：虚拟环境、依赖和项目结构

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握虚拟环境、依赖安装、依赖记录和项目布局。
- 理解包管理、版本约束、可复现环境和入口点。
- 能搭建一个清晰、可测试、可交付的 Python 项目。

## 理论导读

工程化的第一步是让别人能复现你的环境。Python 项目如果只依赖“我电脑上装过”，很快会遇到版本冲突、路径不一致和部署失败。虚拟环境把项目依赖隔离起来，依赖文件或 `pyproject.toml` 负责描述项目需要什么。

现代 Python 项目越来越多使用 `pyproject.toml` 作为构建和工具配置入口。简单脚本可以用 `requirements.txt`，可发布包或团队项目建议采用标准项目结构。

## 核心心智模型

把项目环境看成厨房。虚拟环境是独立厨房，依赖文件是采购清单，项目结构是厨房动线。没有清单和动线，短期能做饭，长期很难协作和复现。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| `venv` | Python 标准库虚拟环境工具。 |
| `pip` | 安装、卸载、查看 Python 包。 |
| `requirements.txt` | 常见依赖清单，适合应用部署或简单项目。 |
| `pyproject.toml` | 构建系统和工具配置标准入口。 |
| `src` 布局 | 把源码放到 `src/包名`，减少本地路径误导。 |
| 入口点 | 通过命令行脚本或 `python -m 包.模块` 启动。 |
| 版本约束 | 使用范围约束和锁定文件控制兼容性与复现性。 |

## 例子

```text
my_tool/
  pyproject.toml
  README.md
  src/
    my_tool/
      __init__.py
      cli.py
      service.py
  tests/
    test_service.py
```

基础命令：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
python -m pip install requests
python -m pip freeze > requirements.txt
python -m pip install -r requirements.txt
```

## 练习

创建一个 `file_report` 项目，使用 `src` 布局。提供 `cli.py`、`service.py` 和 `tests/`，实现统计目录文件信息的命令行工具。

## 验收

- 能创建和激活虚拟环境。
- 能记录项目依赖并在新环境安装。
- 能解释 `src` 布局的优点。
- 能把入口、业务逻辑和测试分离。

## 重点

- 每个项目使用独立虚拟环境。
- 用 `python -m pip` 明确当前解释器对应的 pip。
- 依赖版本要记录，生产项目要考虑锁定和升级策略。

## 难点

- 依赖解析会受到版本约束、平台、Python 版本和已安装包影响。
- 本地可导入不代表打包后可导入，项目结构和测试方式要覆盖安装场景。

## 易错

> **易错：** 运行 `pip install` 时不知道装到了哪个 Python 环境。
>
> 正确做法：使用 `python -m pip`，并用 `python -c "import sys; print(sys.executable)"` 检查解释器。

> **易错：** 不记录依赖版本，导致几个月后无法复现。
>
> 正确做法：至少维护 `requirements.txt`；团队项目使用更严格的锁定策略。
