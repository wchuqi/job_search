# Python学习资料：标准库常用能力

[返回索引](../Python学习资料.md)

## 学习目标

- 熟悉 Python 标准库中高频、稳定、工程价值高的模块。
- 能优先使用标准库完成路径、时间、序列化、命令行、日志、并发和数据结构任务。
- 能查阅官方文档并识别模块适用边界。

## 理论导读

Python 常被称为“自带电池”，意思是标准库覆盖了大量日常工程能力。掌握标准库的价值不只是少安装依赖，还能减少供应链风险、降低部署复杂度、提高跨项目可迁移性。

标准库并不是越多越好地死记硬背，而是建立分类地图：路径文件、数据格式、时间日期、命令行、日志、集合工具、迭代工具、并发、网络、调试、测试。遇到任务先判断是否已有标准模块。

## 核心心智模型

把标准库当成工具箱。不要记住每个螺丝刀的尺寸，但要知道工具箱里有哪些类别，什么时候该打开哪一层。

## 知识点详解

| 类别 | 模块 | 用途 |
| --- | --- | --- |
| 路径和文件 | `pathlib`、`shutil`、`tempfile` | 路径处理、复制删除、临时文件。 |
| 数据格式 | `json`、`csv`、`configparser`、`tomllib` | 读写常见文本数据格式。 |
| 时间日期 | `datetime`、`time`、`zoneinfo` | 日期计算、时间戳、时区。 |
| 命令行 | `argparse` | 构建 CLI 参数解析。 |
| 日志 | `logging` | 分级日志、格式化、处理器。 |
| 数据结构 | `collections`、`heapq`、`bisect` | 计数、队列、堆、二分查找。 |
| 迭代工具 | `itertools` | 高效组合迭代。 |
| 调试测试 | `pdb`、`unittest`、`doctest` | 调试和测试。 |
| 并发 | `threading`、`multiprocessing`、`concurrent.futures`、`asyncio` | 不同并发模型。 |

## 例子

命令行参数和 JSON 输出：

```python
import argparse
import json
from pathlib import Path

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("path")
    args = parser.parse_args()

    path = Path(args.path)
    result = {"file": str(path), "size": path.stat().st_size}
    print(json.dumps(result, ensure_ascii=False, indent=2))

if __name__ == "__main__":
    main()
```

计数：

```python
from collections import Counter

words = "to be or not to be".split()
print(Counter(words).most_common(2))
```

## 练习

写一个 CLI 工具，接收目录路径，递归统计不同扩展名的文件数量和总大小，输出 JSON 报告。要求使用 `argparse`、`pathlib`、`collections.Counter` 和 `json`。

## 验收

- 能为常见任务快速找到对应标准库模块。
- 能写一个带参数解析的命令行脚本。
- 能使用 `pathlib` 替代手写路径拼接。
- 能用 `logging` 替代随处 `print` 调试。

## 重点

- 标准库优先，第三方库用于标准库无法高效覆盖的领域。
- 标准库模块也有边界，例如 `argparse` 适合基础 CLI，复杂交互 CLI 可考虑第三方库。
- 官方文档是标准库学习的主资料。

## 难点

- 时间和时区处理容易出错，生产代码应尽量使用带时区的时间对象。
- 日志配置要统一，否则库代码和应用代码的日志会互相干扰。

## 易错

> **易错：** 为简单 JSON、CSV、路径处理安装不必要依赖。
>
> 正确做法：先检查标准库是否已经覆盖。

> **易错：** 用字符串拼接路径。
>
> 正确做法：使用 `pathlib.Path`，提高跨平台可靠性。
