# Python学习资料：模块、包和导入机制

[返回索引](../Python学习资料.md)

## 学习目标

- 理解模块、包、脚本入口和导入路径。
- 掌握 `import`、`from ... import ...`、相对导入、`__name__ == "__main__"`。
- 能解释 Python 如何查找模块，以及常见导入错误如何排查。

## 理论导读

模块是一个 `.py` 文件，包是包含模块的目录。Python 用模块和包把大型程序拆分为多个文件，避免所有代码堆在一个脚本里。导入并不是简单复制代码，而是加载模块对象、执行模块顶层代码、把结果缓存到 `sys.modules`。

导入机制的核心是查找路径和缓存。解释器会按 `sys.path` 顺序查找目标模块，找到后执行一次并缓存。后续再次导入同一模块时，通常直接复用缓存对象。

## 核心心智模型

把导入想成“按路径找文件，执行一次，登记到模块缓存”。如果路径里有同名文件，靠前路径会先匹配；如果模块顶层代码有副作用，导入时就会执行。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 模块 | 单个 `.py` 文件，导入后是模块对象。 |
| 包 | 包含 `__init__.py` 或命名空间包目录的一组模块。 |
| `sys.path` | 模块搜索路径，包含脚本目录、环境变量、站点包等。 |
| `sys.modules` | 已加载模块缓存。 |
| 绝对导入 | 从项目顶层包名开始导入，更清晰稳定。 |
| 相对导入 | 使用 `.` 表示当前包，适合包内部模块互相引用。 |
| 入口保护 | `if __name__ == "__main__":` 避免导入时执行脚本逻辑。 |

## 例子

推荐结构：

```text
task_tool/
  pyproject.toml
  src/
    task_tool/
      __init__.py
      cli.py
      parser.py
      service.py
```

入口保护：

```python
def main() -> int:
    print("run task tool")
    return 0

if __name__ == "__main__":
    raise SystemExit(main())
```

观察导入路径：

```python
import sys
from pprint import pprint

pprint(sys.path)
```

## 练习

把一个单文件脚本拆成三个模块：`reader.py` 负责读取输入，`service.py` 负责业务处理，`cli.py` 负责命令行入口。要求可以通过 `python -m 包名.cli` 运行。

## 验收

- 能解释模块第一次导入时会执行顶层代码。
- 能解释 `sys.path` 的搜索顺序对同名模块的影响。
- 能处理 `ModuleNotFoundError` 和循环导入。
- 能写出干净的 `main()` 入口。

## 重点

- 导入路径和包结构要从项目一开始就保持清晰。
- 模块顶层尽量只放定义，不做耗时或有副作用的业务操作。
- 使用 `python -m 包.模块` 可以减少脚本相对路径问题。

## 难点

- 循环导入常发生在两个模块顶层互相导入并立即使用对方对象。
- 相对导入依赖包上下文，直接运行包内文件可能失败。

## 易错

> **易错：** 文件命名为 `json.py`、`typing.py` 等标准库同名文件，导致导入被本地文件遮蔽。
>
> 正确做法：避免和标准库、第三方库重名，出错时打印 `模块.__file__` 检查实际导入来源。

> **易错：** 在被导入模块顶层直接执行网络请求、数据库连接或文件写入。
>
> 正确做法：顶层只定义函数、类、常量；执行逻辑放入 `main()` 或显式函数。
