# Python学习资料：语法基础和运行模型

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握缩进、表达式、赋值、注释、输入输出和基础运算符。
- 理解 Python 代码从源码到字节码再到执行的基本过程。
- 能读懂简单错误信息并定位语法错误。

## 理论导读

Python 用缩进表达代码块。缩进不是格式偏好，而是语法的一部分。`if`、`for`、`while`、函数、类、异常处理等结构都依靠缩进确定边界。这使代码天然可读，但也要求团队保持一致的格式。

Python 的运行过程可以分为三步：读取源文件、编译成字节码、在虚拟机中执行。字节码通常会缓存在 `__pycache__` 目录下。语法错误发生在编译前后，运行时错误发生在执行阶段。

## 核心心智模型

把一段 Python 代码看成“先检查语法，再生成执行计划，最后按对象行为执行”。语法决定程序能不能被解释器接受，对象模型决定运行时每一步具体如何反应。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 缩进 | 推荐 4 个空格；同一代码块缩进必须一致。 |
| 变量赋值 | 名字绑定到对象，可以重新绑定到不同类型对象。 |
| 表达式 | 会产生值，如 `1 + 2`、`name.upper()`。 |
| 语句 | 执行动作，如 `if`、`return`、`import`。 |
| 注释 | 用 `#` 写单行说明；文档字符串用于模块、函数和类说明。 |
| 运算符 | 算术、比较、逻辑、成员、身份、位运算。 |

## 例子

```python
price = 19.9
count = 3
total = price * count

if total >= 50:
    print("free shipping")
else:
    print("shipping fee required")
```

查看字节码：

```python
import dis

def add(a, b):
    return a + b

dis.dis(add)
```

## 练习

写一个命令行脚本，输入商品单价和数量，输出总价。如果总价大于等于 100，输出折后金额。

## 验收

- 能用 4 空格缩进写出稳定可读的代码块。
- 能解释 `SyntaxError` 和 `NameError` 的差别。
- 能用 `python -m py_compile 文件名.py` 检查语法。
- 能用 `dis` 粗略观察函数字节码。

## 重点

- 缩进是语法，不只是风格。
- Python 变量是动态绑定，类型属于对象而不是变量名。
- 运行错误要结合异常类型、调用栈和出错行定位。

## 难点

- 表达式和语句的边界会影响可读性，例如列表推导式适合短逻辑，不适合塞入复杂流程。
- `is` 比较对象身份，`==` 比较值，二者机制不同。

## 易错

> **易错：** 用 `is` 判断字符串或数字值是否相等。
>
> 正确做法：值比较使用 `==`；只有判断是否同一个对象或是否为 `None` 时用 `is`。

> **易错：** 混用 Tab 和空格。
>
> 正确做法：编辑器统一设置为 4 个空格，并启用格式化工具。
