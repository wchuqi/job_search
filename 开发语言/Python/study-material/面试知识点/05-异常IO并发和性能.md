## 一、异常、IO、并发和性能

### 1. Python 异常处理的基本原则是什么？

**参考答案：**

只捕获自己能处理的异常，捕获要具体，错误信息要有上下文。底层可以抛出具体异常，上层入口负责转换为用户可读信息、日志和退出码。不要用裸 `except` 静默吞掉错误。

```python
try:
    value = int(text)
except ValueError as exc:
    raise RuntimeError(f"invalid number: {text}") from exc
```

> **重点：** 异常链能保留根因。
> **易错：** `except Exception: pass` 会让问题消失在表面，排障成本极高。

### 2. `with` 语句解决什么问题？

**参考答案：**

`with` 管理进入和退出成对出现的资源或状态。它会调用上下文管理器的 `__enter__` 和 `__exit__`，即使代码块中发生异常，也能执行清理逻辑。

```python
with open("data.txt", "r", encoding="utf-8") as f:
    data = f.read()
```

> **重点：** 文件、锁、连接、事务都适合上下文管理。
> **易错：** 手动打开文件后忘记关闭，异常时资源泄漏。

### 3. 文件读写为什么要指定编码？

**参考答案：**

如果不指定编码，Python 会使用平台默认编码，不同系统可能不同。为了跨平台一致性，文本文件通常显式使用 `encoding="utf-8"`。CSV 读写还应使用 `newline=""` 避免换行处理问题。

```python
from pathlib import Path
text = Path("README.md").read_text(encoding="utf-8")
```

> **重点：** 编码是数据契约，不应依赖系统默认值。

### 4. 线程、进程、协程分别适合什么场景？

**参考答案：**

线程适合许多阻塞 IO 场景，如文件、网络、数据库等待；进程适合 CPU 密集计算，因为可以绕开 CPython GIL 的 CPU 并行限制；协程适合大量可等待 IO，但要求调用链使用异步库。

| 模型 | 适合场景 | 风险 |
| --- | --- | --- |
| 线程 | 阻塞 IO | 共享状态和锁 |
| 进程 | CPU 密集 | 序列化和通信成本 |
| 协程 | 大量异步 IO | 阻塞调用卡住事件循环 |

> **重点：** 先判断瓶颈，再选择并发模型。
> **易错：** 认为并发一定更快。

### 5. GIL 是什么？有什么影响？

**参考答案：**

在 CPython 中，GIL 是全局解释器锁，同一时刻通常只允许一个线程执行 Python 字节码。它限制多线程在 CPU 密集 Python 代码上的并行能力，但线程在 IO 等待时仍可能带来吞吐提升。

> **重点：** GIL 主要影响 CPython 多线程 CPU 并行。
> **难点：** 不同 Python 实现、C 扩展释放 GIL、IO 等待场景要分开讨论。

### 6. 如何优化 Python 程序性能？

**参考答案：**

先测量再优化。先用 `time.perf_counter`、`cProfile`、日志耗时定位瓶颈，再考虑算法复杂度、数据结构、减少 IO、缓存、流式处理、并发或更合适的底层库。不要先重写代码。

```python
from time import perf_counter

start = perf_counter()
do_work()
print(f"cost: {perf_counter() - start:.3f}s")
```

> **重点：** 算法和 IO 次数通常比微优化更重要。
> **易错：** 没有基准数据就宣布优化有效。

### 7. 处理大文件时应该注意什么？

**参考答案：**

大文件应流式读取，逐行或分块处理，避免一次性 `read()` 全部内容。中间处理尽量使用生成器，输出也可以分批写入。还要处理编码错误、异常行和磁盘 IO 瓶颈。

```python
def iter_errors(path):
    with open(path, "r", encoding="utf-8") as f:
        for line in f:
            if "ERROR" in line:
                yield line
```

> **重点：** 大文件处理的核心是控制内存峰值。
> **易错：** 为了写法简单把所有数据读入列表。

### 8. 异步代码中最常见的问题是什么？

**参考答案：**

最常见问题是在协程中调用阻塞函数，例如同步网络请求、同步数据库客户端或耗时 CPU 计算。这会阻塞事件循环，导致其他任务无法调度。

```python
import asyncio

async def task():
    await asyncio.sleep(1)  # 非阻塞等待
```

> **重点：** 异步收益要求等待点可让出控制权。
> **易错：** 给函数加 `async` 不会自动让内部阻塞代码变成非阻塞。
