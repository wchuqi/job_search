## 一、函数、对象和作用域

### 1. Python 函数参数是值传递还是引用传递？

**参考答案：**

更准确地说，Python 传递的是对象引用的赋值。调用函数时，形参名字绑定到实参对象。函数内部如果修改可变对象本身，外部能看到；如果只是让形参重新绑定到新对象，外部名字不受影响。

```python
def add_item(items):
    items.append(1)

def rebind(items):
    items = [9]

data = []
add_item(data)
rebind(data)
print(data)  # [1]
```

> **重点：** 结合“名字绑定对象”解释，比简单说值传递或引用传递更准确。

### 2. 为什么默认可变参数危险？

**参考答案：**

默认参数在函数定义时求值一次，不是在每次调用时重新创建。如果默认值是列表、字典等可变对象，多次调用会共享同一个对象，导致状态泄漏。

```python
def append_item(item, items=[]):
    items.append(item)
    return items

print(append_item(1))  # [1]
print(append_item(2))  # [1, 2]
```

正确写法：

```python
def append_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

> **重点：** 默认参数求值时机是定义阶段。
> **易错：** 把这个问题误解成“列表不能作为参数”。

### 3. LEGB 作用域规则是什么？

**参考答案：**

LEGB 是名字查找顺序：Local 当前局部作用域、Enclosing 外层嵌套函数作用域、Global 模块全局作用域、Built-in 内置作用域。解释器按这个顺序查找名字。

```python
x = "global"

def outer():
    x = "enclosing"
    def inner():
        return x
    return inner()
```

> **重点：** LEGB 是名字读取的查找规则。
> **难点：** 给变量赋值会在当前局部作用域创建或修改绑定，除非使用 `global` 或 `nonlocal`。

### 4. 闭包是什么？常见陷阱是什么？

**参考答案：**

闭包是内层函数引用外层函数变量，并在外层函数返回后仍能访问这些变量的机制。常见陷阱是循环中创建闭包时，闭包捕获变量绑定，而不是捕获每轮循环的即时值。

```python
funcs = [lambda: i for i in range(3)]
print([f() for f in funcs])  # [2, 2, 2]

fixed = [lambda i=i: i for i in range(3)]
print([f() for f in fixed])  # [0, 1, 2]
```

> **重点：** 闭包保存对变量的引用关系。
> **易错：** 以为 lambda 会自动保存循环当时的值。

### 5. 装饰器的本质是什么？

**参考答案：**

装饰器本质上是接收函数并返回新函数的可调用对象。它常用于日志、计时、鉴权、缓存、重试等横切逻辑。写装饰器时应使用 `functools.wraps` 保留原函数元数据。

```python
from functools import wraps

def log_call(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"call {func.__name__}")
        return func(*args, **kwargs)
    return wrapper
```

> **重点：** `@decorator` 等价于 `func = decorator(func)`。
> **易错：** 不用 `wraps` 导致函数名、文档和调试信息丢失。

### 6. 生成器函数和普通函数有什么区别？

**参考答案：**

包含 `yield` 的函数是生成器函数，调用后返回生成器对象，不会立即执行完整函数体。每次迭代执行到下一个 `yield` 暂停，并在下一次迭代时恢复。

```python
def numbers():
    yield 1
    yield 2

gen = numbers()
print(next(gen))
print(next(gen))
```

> **重点：** 生成器适合惰性计算和流式处理。
> **易错：** 认为生成器可以像列表一样重复遍历。

### 7. `*args` 和 `**kwargs` 的作用是什么？

**参考答案：**

`*args` 收集额外位置参数为元组，`**kwargs` 收集额外关键字参数为字典。它们常用于包装函数、装饰器、转发参数和兼容扩展接口。

```python
def wrapper(*args, **kwargs):
    print(args, kwargs)
```

> **重点：** 参数收集和参数展开是相反操作。
> **易错：** 滥用 `**kwargs` 会让函数接口不清晰，调用者不知道支持哪些参数。
