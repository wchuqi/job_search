## 一、面向对象和数据模型

### 1. 类属性和实例属性有什么区别？

**参考答案：**

类属性定义在类对象上，所有实例通过属性查找都能访问；实例属性保存在具体实例上，通常在 `__init__` 中设置。如果实例上有同名属性，会优先使用实例属性。

```python
class User:
    role = "member"

    def __init__(self, name):
        self.name = name
```

> **重点：** 属性查找通常先实例后类。
> **易错：** 把可变列表作为类属性保存实例数据，导致所有实例共享。

### 2. Python 的属性查找顺序是什么？

**参考答案：**

一般实例属性查找会先查实例字典，再查类，再按 MRO 查找父类。描述符会影响查找优先级，数据描述符通常优先于实例字典。

```python
class A:
    value = 1

class B(A):
    pass

obj = B()
print(obj.value)
```

> **重点：** 普通属性查找和描述符优先级是理解 `property`、ORM、框架行为的基础。
> **难点：** 深入框架源码时，需要补充描述符和元类知识。

### 3. 什么是 MRO？

**参考答案：**

MRO 是 Method Resolution Order，即方法解析顺序。多继承中，Python 使用 C3 线性化算法确定查找父类方法的顺序，可以通过 `类名.__mro__` 查看。

```python
class A: ...
class B(A): ...
class C(A): ...
class D(B, C): ...

print(D.__mro__)
```

> **重点：** MRO 决定多继承下属性和方法的查找顺序。
> **易错：** 以为多继承简单按深度优先或写法直觉查找。

### 4. 特殊方法有什么作用？

**参考答案：**

特殊方法让自定义对象参与 Python 语法和内置函数。例如 `len(obj)` 调用 `__len__`，迭代调用 `__iter__`，上下文管理调用 `__enter__` 和 `__exit__`，字符串展示调用 `__str__` 或 `__repr__`。

```python
class Bag:
    def __init__(self, items):
        self.items = list(items)

    def __len__(self):
        return len(self.items)

print(len(Bag([1, 2])))
```

> **重点：** Python 的很多“语法糖”背后是协议。
> **易错：** 只实现普通方法，导致对象不能自然融入内置操作。

### 5. `__str__` 和 `__repr__` 有什么区别？

**参考答案：**

`__str__` 面向用户可读展示，`__repr__` 面向开发者调试，理想情况下 `__repr__` 应尽量清楚表达对象状态。`print(obj)` 优先用 `__str__`，交互式显示和容器展示常用 `__repr__`。

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"
```

> **重点：** 调试对象时优先实现有信息量的 `__repr__`。

### 6. 继承和组合如何选择？

**参考答案：**

继承适合稳定的“是一种”关系，组合适合“拥有一个”或“使用一个”关系。实际工程中组合通常更灵活，因为它降低父类变更对派生类的影响。

```python
class ReportService:
    def __init__(self, writer):
        self.writer = writer
```

> **重点：** 优先组合，谨慎继承。
> **易错：** 为了复用几行代码建立深层继承结构。

### 7. `dataclass` 适合什么场景？

**参考答案：**

`dataclass` 适合以数据为主的类，自动生成 `__init__`、`__repr__`、比较方法等样板代码。需要不可变数据对象时可以使用 `frozen=True`，可变默认值应使用 `field(default_factory=...)`。

```python
from dataclasses import dataclass, field

@dataclass
class Team:
    name: str
    members: list[str] = field(default_factory=list)
```

> **重点：** `dataclass` 提高数据类可读性。
> **易错：** 直接写 `members: list[str] = []`。
