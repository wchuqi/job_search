## 一、深入运行机制和生产实践

### 1. CPython 执行一段 Python 函数大致经历哪些阶段？

**参考答案：**

源码先被解析为 AST，再编译成 code object 和字节码。函数定义会创建函数对象，函数对象引用 code object。每次调用函数时会创建 frame，保存本次调用的局部变量、执行位置和调用上下文。解释器循环执行字节码指令，操作运行时对象。

```python
import dis

def add(a, b):
    return a + b

dis.dis(add)
print(add.__code__.co_varnames)
```

> **重点：** Python 不是简单逐字符解释源码，CPython 会先编译字节码。
> **难点：** code object 是静态代码信息，frame 是每次调用的运行时状态。

### 2. CPython 如何管理对象生命周期？

**参考答案：**

CPython 主要使用引用计数。对象被变量、容器、栈帧等引用时计数增加，引用消失时计数减少，降到 0 通常立即释放。循环引用无法仅靠引用计数解决，因此 CPython 还有循环垃圾回收器检测容器对象之间的引用环。

```python
import gc

a = []
b = [a]
a.append(b)
del a, b
gc.collect()
```

> **重点：** 引用计数解决大多数对象释放，循环 GC 解决引用环。
> **易错：** 认为离开作用域后内存一定立刻还给操作系统。对象可能释放给 Python 分配器，但 RSS 不一定马上下降。

### 3. 小整数缓存和字符串驻留能否作为业务判断依据？

**参考答案：**

不能。这些是 CPython 的实现优化，不是语言层面的业务语义。它们可能导致某些整数或字符串用 `is` 比较时返回 `True`，但值比较必须使用 `==`。

```python
a = 256
b = 256
print(a is b)  # 不能依赖
print(a == b)  # 正确值比较
```

> **重点：** `is` 比较身份，`==` 比较值。

### 4. 描述符是什么？`property` 为什么能拦截属性访问？

**参考答案：**

描述符是实现了 `__get__`、`__set__` 或 `__delete__` 的对象。`property` 是数据描述符，定义在类上后，实例访问该属性时会优先触发描述符逻辑，而不是直接从实例字典取值。

```python
class User:
    def __init__(self, age):
        self._age = age

    @property
    def age(self):
        return self._age
```

> **重点：** 描述符是 `property`、方法绑定、ORM 字段的基础。
> **难点：** 数据描述符优先级高于实例 `__dict__`。

### 5. 普通方法为什么会自动绑定 `self`？

**参考答案：**

类中定义的普通函数是非数据描述符。通过实例访问时，函数对象的 `__get__` 会返回绑定方法，把实例绑定为第一个参数。通过类访问时得到的是原始函数，需要手动传实例。

```python
class A:
    def f(self):
        return 1

a = A()
print(a.f)   # bound method
print(A.f)   # function
```

> **重点：** 自动传 `self` 来自描述符协议，不是特殊语法硬编码。

### 6. 元类解决什么问题？为什么不建议滥用？

**参考答案：**

元类控制类对象的创建过程，适合类定义期校验、字段收集、自动注册、声明式框架等场景。但元类会增加理解成本，并可能在多继承中产生元类冲突。很多场景可以用类装饰器、描述符或 `__init_subclass__` 更简单地解决。

```python
class Plugin:
    registry = {}

    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        Plugin.registry[cls.__name__] = cls
```

> **重点：** 能不用元类时优先不用。

### 7. 类型提示会不会在运行时强制类型？

**参考答案：**

默认不会。类型提示主要服务静态检查、编辑器提示和接口文档。外部输入仍然需要运行时校验。`Any` 会让类型检查放弃约束，应该限制在边界层。

```python
def double(x: int) -> int:
    return x * 2

print(double("a"))  # 运行时不会因类型提示自动报错，结果是 'aa'
```

> **重点：** 类型提示不是运行时校验。
> **易错：** 标注了 `int` 就不校验 JSON 或用户输入。

### 8. `Protocol` 和抽象基类怎么选择？

**参考答案：**

`Protocol` 表达结构化子类型，适合“只要有这些方法就能用”的鸭子类型；抽象基类表达显式继承关系，适合需要注册、共享实现或强约束层级的场景。Python 工程中用 `Protocol` 可以减少不必要继承。

```python
from typing import Protocol

class Writer(Protocol):
    def write(self, text: str) -> int:
        ...
```

> **重点：** `Protocol` 关注对象能做什么，不关注它继承自谁。

### 9. 为什么循环导入会出现“部分初始化模块”？

**参考答案：**

模块导入时会创建模块对象并放入 `sys.modules`，然后执行模块顶层代码。循环导入中，另一个模块可能在第一个模块尚未执行完时就从 `sys.modules` 拿到它，此时需要的变量或类还没定义。

> **重点：** 循环导入本质是模块依赖方向不清晰。
> **正确做法：** 拆出共享模块、延迟导入、反转依赖或重构边界。

### 10. 应用项目和库项目的依赖版本策略有什么不同？

**参考答案：**

库项目通常声明兼容范围，避免过度锁死依赖，方便使用者和其他库共同解析；应用项目追求部署复现，通常需要锁定具体版本，并记录 Python 版本和构建环境。

> **重点：** 库关注兼容性，应用关注可复现。
> **易错：** 应用生产部署只写宽泛依赖范围，导致每次部署安装到不同版本。

### 11. `asyncio` 为什么不能直接调用阻塞函数？

**参考答案：**

事件循环依赖协程在 `await` 点让出控制权。如果协程内调用 `time.sleep`、同步 HTTP 请求或 CPU 密集计算，会阻塞事件循环，其他任务无法运行。阻塞 IO 应使用异步库或 `asyncio.to_thread`，CPU 密集任务应考虑进程池。

```python
import asyncio
import time

async def bad():
    time.sleep(1)

async def good():
    await asyncio.sleep(1)
```

> **重点：** `async def` 不会自动让阻塞代码变非阻塞。

### 12. 如何设计一个生产级 Python CLI 的错误处理？

**参考答案：**

底层函数抛出具体异常并保留异常链，中间层添加业务上下文，入口层捕获已知异常，记录日志，输出用户可读错误，返回明确退出码。未知异常要记录完整栈，避免静默失败。

```python
def main() -> int:
    try:
        run()
        return 0
    except ConfigError as exc:
        print(f"config error: {exc}")
        return 2
    except Exception:
        logger.exception("unexpected_error")
        return 1
```

> **重点：** 入口层处理展示和退出码，业务层保留根因。

### 13. 如何排查 Python 内存持续增长？

**参考答案：**

先确认增长指标：对象数量、Python 分配、RSS 还是外部资源。用 `tracemalloc` 找分配热点，检查全局缓存、无界队列、闭包、日志 handler、长生命周期容器和循环引用。注意对象释放给 Python 分配器后，操作系统 RSS 不一定立刻下降。

```python
import tracemalloc

tracemalloc.start()
run()
snapshot = tracemalloc.take_snapshot()
print(snapshot.statistics("lineno")[:10])
```

> **重点：** 内存排查要区分“对象还活着”和“内存未归还 OS”。

### 14. Python 生产日志应该包含什么，避免什么？

**参考答案：**

日志应包含事件名、时间、任务或请求标识、关键业务上下文、耗时、结果和错误根因。避免输出密码、Token、Cookie、身份证、完整连接串等敏感信息。库代码只创建 logger，不应随意配置全局日志。

> **重点：** 日志要能排障，同时必须脱敏。
> **易错：** `logger.exception` 时把完整请求头或配置对象直接打出来。

### 15. 为什么外部请求必须设置超时和重试边界？

**参考答案：**

没有超时的外部调用可能无限等待，占住线程、连接或任务槽。重试可以缓解瞬时故障，但无限重试会放大故障；非幂等操作还可能造成重复写入。生产中要设置超时、重试次数、退避策略，并区分可重试和不可重试错误。

> **重点：** 超时是可靠性边界，重试必须有上限。
> **难点：** 非幂等操作的重试要非常谨慎。
