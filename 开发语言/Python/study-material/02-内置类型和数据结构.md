# Python学习资料：内置类型和数据结构

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握数字、字符串、列表、元组、字典、集合、布尔值和 `None`。
- 理解可变与不可变、哈希、切片、解包、浅拷贝和深拷贝。
- 能根据场景选择合适数据结构。

## 理论导读

数据结构是程序组织信息的方式。Python 内置类型覆盖了日常开发的大多数基础场景：列表适合有序可变序列，元组适合固定结构，字典适合键值映射，集合适合去重和成员判断，字符串适合文本处理。

可变性是 Python 数据结构学习的主线。不可变对象如 `int`、`str`、`tuple` 通常不能原地修改；可变对象如 `list`、`dict`、`set` 可以在原对象上变化。函数参数、默认值、拷贝和缓存问题都和可变性有关。

## 核心心智模型

把数据结构当成不同形状的容器：列表像可调整顺序的队列，字典像带索引的柜子，集合像只关心“有没有”的门禁表，元组像固定格式的记录单。先问数据的访问方式，再选容器。

## 知识点详解

| 类型 | 典型用途 | 关键点 |
| --- | --- | --- |
| `str` | 文本、路径片段、消息 | 不可变，支持切片、格式化、编码转换。 |
| `list` | 有序集合、批量处理 | 可变，追加快，头部插入慢。 |
| `tuple` | 固定记录、多返回值 | 不可变，可作为字典键的前提是元素也可哈希。 |
| `dict` | 映射、索引、配置 | 平均 O(1) 查找，键必须可哈希。 |
| `set` | 去重、集合运算 | 平均 O(1) 成员判断，无重复元素。 |
| `bool` / `None` | 条件和空值 | 空容器、0、空字符串在布尔上下文中为假。 |

## 例子

```python
orders = [
    {"user": "alice", "amount": 30},
    {"user": "bob", "amount": 20},
    {"user": "alice", "amount": 15},
]

total_by_user = {}
for order in orders:
    user = order["user"]
    total_by_user[user] = total_by_user.get(user, 0) + order["amount"]

print(total_by_user)  # {'alice': 45, 'bob': 20}
```

浅拷贝陷阱：

```python
rows = [[0] * 3] * 2
rows[0][0] = 1
print(rows)  # [[1, 0, 0], [1, 0, 0]]

safe_rows = [[0] * 3 for _ in range(2)]
```

## 练习

给定一组日志行，提取 IP 地址，统计每个 IP 出现次数，并输出出现次数最多的前 3 个 IP。

## 验收

- 能说明 `list`、`tuple`、`dict`、`set` 的适用场景和基本复杂度。
- 能正确使用切片、解包和字典的 `get`、`items`。
- 能解释为什么列表不能作为字典键。
- 能避免嵌套列表浅拷贝陷阱。

## 重点

- 字典和集合依赖哈希，适合快速查找和去重。
- 可变对象共享引用时，一个地方修改会影响所有引用者。
- 字符串不可变，大量拼接优先使用 `''.join(...)`。

## 难点

- “不可变容器”和“容器内对象可变”要分开理解。元组本身不可变，但元组里如果放列表，列表仍可修改。
- 拷贝要看层级：浅拷贝复制外层容器，深拷贝递归复制内部对象。

## 易错

> **易错：** 用列表做大量成员判断。
>
> 正确做法：如果只关心是否存在，使用 `set`。

> **易错：** 以为 `a = b` 会复制对象。
>
> 正确做法：赋值只是绑定名字，需要复制时使用切片、`copy.copy` 或 `copy.deepcopy`。
