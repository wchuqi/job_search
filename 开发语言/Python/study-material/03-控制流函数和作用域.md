# Python学习资料：控制流、函数和作用域

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握 `if`、`for`、`while`、`break`、`continue`、`match` 等控制流。
- 理解函数定义、参数传递、返回值、默认参数、闭包和 LEGB 作用域规则。
- 能把脚本逻辑拆成可测试的小函数。

## 理论导读

控制流决定程序执行路径，函数决定代码组织边界。好的 Python 代码通常由一组短小、职责清晰、输入输出明确的函数组成。函数不是只为复用而存在，也用于命名一段业务意图，让调用者不用关心内部细节。

Python 的作用域查找遵循 LEGB：Local、Enclosing、Global、Built-in。解释器解析一个名字时，会从当前局部作用域开始，逐层向外查找。闭包、装饰器和默认参数行为都和这个规则相关。

## 核心心智模型

把函数想成带入口和出口的小机器。参数是输入，返回值是输出，局部变量是机器内部零件。全局变量像共享仓库，能用但要谨慎，因为它会让函数行为依赖外部状态。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 条件分支 | `if` / `elif` / `else` 按条件选择路径。 |
| 循环 | `for` 遍历可迭代对象，`while` 按条件重复执行。 |
| `match` | Python 3.10 引入的结构化模式匹配，适合清晰表达多形状数据分支。 |
| 函数参数 | 位置参数、关键字参数、默认参数、`*args`、`**kwargs`。 |
| 返回值 | 可以返回一个对象，也可以用元组返回多个值。 |
| 作用域 | LEGB 查找规则，`global` 和 `nonlocal` 用于修改外层绑定。 |

## 例子

```python
def normalize_name(name: str) -> str:
    return " ".join(part.capitalize() for part in name.strip().split())

def classify_score(score: int) -> str:
    if score >= 90:
        return "A"
    if score >= 80:
        return "B"
    if score >= 60:
        return "C"
    return "D"
```

默认可变参数陷阱：

```python
def append_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

## 练习

写一个 `parse_duration(text)` 函数，把 `"2h30m"`、`"45m"`、`"1h"` 转为分钟数。要求非法格式抛出 `ValueError`。

## 验收

- 能把 50 行以上的脚本拆成多个函数。
- 能解释默认参数只在函数定义时求值一次。
- 能解释 LEGB 查找顺序。
- 能使用类型提示表达函数输入输出。

## 重点

- 函数应尽量少依赖全局可变状态。
- 默认参数适合不可变值；可变默认值通常用 `None` 占位。
- `match` 是表达结构分支的工具，不是所有 `if` 的替代品。

## 难点

- 闭包捕获的是变量绑定，不是每次循环的即时值。
- `global` 和 `nonlocal` 修改的是名字绑定，滥用会降低可维护性。

## 易错

> **易错：** 在循环里创建闭包时以为会保存每次循环的值。
>
> 正确做法：用默认参数或工厂函数固定当前值。

```python
funcs = [lambda x, i=i: x + i for i in range(3)]
```

> **易错：** 函数既计算结果又打印又修改外部状态。
>
> 正确做法：优先让函数返回结果，把 IO 和状态修改放在清晰边界上。
