# Python学习资料：面向对象和数据模型

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握类、对象、属性、方法、继承、组合和 `dataclass`。
- 理解 Python 数据模型和特殊方法，如 `__str__`、`__len__`、`__iter__`、`__enter__`。
- 能判断何时用类，何时用函数或简单数据结构。

## 理论导读

面向对象用于把数据和相关行为放在一起。Python 的对象模型很开放：类本身也是对象，属性可以动态查找，很多内置语法会转化为特殊方法调用。例如 `len(x)` 会尝试调用 `x.__len__()`，`for x in items` 会使用迭代协议。

继承表达“是一种”的关系，组合表达“拥有一个”的关系。实际项目中，组合通常更容易维护，因为它让对象职责更小，替换也更方便。

## 核心心智模型

把类看成对象的说明书，对象是按说明书创建出来的实例。特殊方法像协议接口，告诉 Python 这个对象如何参与语言内置操作。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 类和实例 | 类定义属性和行为，实例保存具体状态。 |
| 实例方法 | 第一个参数通常是 `self`，表示当前实例。 |
| 类方法 | 使用 `@classmethod`，第一个参数是 `cls`，常用于替代构造器。 |
| 静态方法 | 使用 `@staticmethod`，不依赖实例或类状态。 |
| 属性查找 | 先查实例，再查类，再沿 MRO 查找父类。 |
| `dataclass` | 适合声明以数据为主的类，减少样板代码。 |
| 特殊方法 | 让对象融入 Python 语法和内置函数。 |

## 例子

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class Money:
    amount: int
    currency: str = "CNY"

    def __str__(self) -> str:
        return f"{self.amount / 100:.2f} {self.currency}"

price = Money(1990)
print(price)  # 19.90 CNY
```

属性查找和特殊方法：

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, name):
        self.items.append(name)

    def __len__(self):
        return len(self.items)

cart = Cart()
cart.add("book")
print(len(cart))
```

## 练习

实现一个 `Task` 和 `TaskList`。`Task` 包含标题、完成状态和创建时间；`TaskList` 支持添加、完成、删除、按状态筛选，并实现 `__len__` 和 `__iter__`。

## 验收

- 能解释类属性和实例属性的区别。
- 能说明 MRO 的基本查找顺序。
- 能用 `dataclass` 简化数据类。
- 能实现至少三个特殊方法改善对象可用性。

## 重点

- 类适合封装状态和行为，函数适合表达无状态转换。
- 优先组合，谨慎继承。
- 特殊方法是 Python 对象融入语言协议的入口。

## 难点

- Python 属性查找不是只在对象自身查找，还会沿类和父类链查找。
- 多继承需要理解 MRO，否则方法调用顺序容易失控。

## 易错

> **易错：** 把所有逻辑都写成类，导致无状态工具也变成复杂对象。
>
> 正确做法：没有持久状态时优先使用函数。

> **易错：** 在类属性上保存可变列表作为实例状态。
>
> 正确做法：实例状态放到 `__init__` 中，或用 `dataclasses.field(default_factory=list)`。
