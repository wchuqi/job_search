# Python学习资料：迭代器、生成器和推导式

[返回索引](../Python学习资料.md)

## 学习目标

- 理解可迭代对象、迭代器、生成器和迭代协议。
- 掌握列表、字典、集合推导式以及生成器表达式。
- 能用惰性迭代处理大文件或流式数据。

## 理论导读

迭代是 Python 最核心的抽象之一。`for` 循环并不要求对象是列表，只要求对象可迭代。可迭代对象能通过 `iter(obj)` 得到迭代器；迭代器通过 `next()` 逐个产出值，直到抛出 `StopIteration`。

生成器是写迭代器的简洁方式。包含 `yield` 的函数调用后不会立刻执行完，而是返回一个生成器对象；每次 `next()` 会执行到下一个 `yield` 暂停。它特别适合大文件、分页接口、流水线处理。

## 核心心智模型

列表像一次性把所有数据装进箱子，生成器像传送带，数据一件一件过来。数据量小时二者都能用，数据量大时传送带更省内存。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 可迭代对象 | 能被 `iter()` 处理，如列表、字符串、文件对象。 |
| 迭代器 | 实现 `__iter__` 和 `__next__`，会记住当前位置。 |
| 生成器函数 | 使用 `yield`，天然返回迭代器。 |
| 生成器表达式 | 类似列表推导式，但惰性产生值。 |
| 推导式 | 用简洁语法构建列表、字典、集合。 |
| 惰性计算 | 不提前构建所有结果，需要时才计算。 |

## 例子

```python
def read_non_empty_lines(path):
    with open(path, "r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if line:
                yield line

for line in read_non_empty_lines("app.log"):
    print(line)
```

推导式：

```python
names = [" Ada ", "grace", "", " Linus "]
clean_names = [name.strip().title() for name in names if name.strip()]
index = {name.lower(): name for name in clean_names}
```

## 练习

实现一个日志处理流水线：逐行读取大日志文件，过滤包含 `"ERROR"` 的行，提取时间和错误码，最后统计每种错误码的数量。要求中间步骤使用生成器。

## 验收

- 能解释可迭代对象和迭代器的区别。
- 能写一个带 `yield` 的生成器函数。
- 能说明生成器为什么适合大文件处理。
- 能在可读性允许的范围内使用推导式。

## 重点

- `for` 循环背后是迭代协议。
- 生成器节省内存，但只能按顺序消费，很多生成器不能重复遍历。
- 推导式适合短表达式，复杂逻辑应拆成函数。

## 难点

- 迭代器有状态，被消费后位置会前进。
- 生成器暂停和恢复的执行模型需要结合 `yield` 位置理解。

## 易错

> **易错：** 反复遍历同一个生成器，以为每次都会从头开始。
>
> 正确做法：需要重复遍历时重新创建生成器，或明确转成列表。

> **易错：** 为了“一行写完”写出难读的多层推导式。
>
> 正确做法：复杂逻辑拆成命名函数或普通循环。
