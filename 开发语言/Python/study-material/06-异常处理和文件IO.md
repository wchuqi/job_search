# Python学习资料：异常处理和文件 IO

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握 `try`、`except`、`else`、`finally`、`raise` 和自定义异常。
- 能安全读写文本、JSON、CSV 和二进制文件。
- 理解编码、路径、上下文管理器和错误边界。

## 理论导读

异常是 Python 表达失败路径的主要机制。它让正常流程和错误处理分开：正常代码只描述应该发生什么，异常处理描述发生问题时如何恢复、报告或终止。好的异常处理不是吞掉错误，而是在正确边界把错误转化为对调用者有意义的信息。

文件 IO 的核心问题是资源生命周期和数据格式。文件打开后必须关闭，文本读写必须指定编码，路径处理要跨平台，外部输入要当作不可信数据处理。

## 核心心智模型

把异常看成运行时的“紧急信号”。信号会沿调用栈向上传递，直到有人能处理它。如果没人处理，程序终止并打印调用栈。文件句柄则像租来的工具，用完必须归还，`with` 语句负责可靠归还。

## 知识点详解

| 知识点 | 说明 |
| --- | --- |
| 捕获异常 | 精确捕获能处理的异常，不要裸 `except`。 |
| 抛出异常 | 用 `raise` 主动报告非法状态。 |
| 异常链 | `raise NewError(...) from exc` 保留根因。 |
| `finally` | 无论是否异常都会执行，适合清理资源。 |
| `with` | 进入和退出上下文时自动管理资源。 |
| 编码 | 文本文件优先显式指定 `encoding="utf-8"`。 |
| `pathlib` | 面向对象路径 API，优于手写字符串拼接路径。 |

## 例子

```python
from pathlib import Path
import json

def load_config(path: Path) -> dict:
    try:
        text = path.read_text(encoding="utf-8")
        return json.loads(text)
    except FileNotFoundError as exc:
        raise RuntimeError(f"config not found: {path}") from exc
    except json.JSONDecodeError as exc:
        raise RuntimeError(f"invalid json config: {path}") from exc
```

CSV 读取：

```python
import csv
from pathlib import Path

with Path("orders.csv").open("r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["order_id"], row["amount"])
```

## 练习

写一个 `load_users(path)` 函数，读取 JSON 文件并返回用户列表。要求处理文件不存在、JSON 格式错误、字段缺失三类错误，并保留原始异常链。

## 验收

- 能解释异常传播和调用栈。
- 能区分应该捕获、应该继续抛出、应该包装的错误。
- 能用 `with` 和 `pathlib` 安全处理文件。
- 能为文本读写显式指定编码。

## 重点

- 捕获异常要具体，错误信息要能帮助定位。
- 文件和网络连接这类资源要用上下文管理器管理生命周期。
- 外部数据必须校验，不要假设字段和格式一定正确。

## 难点

- 异常边界设计需要结合业务：底层函数通常抛出具体异常，上层入口负责转换为用户可读信息和退出码。
- 异常链能保留根因，排障时比只抛新错误更有价值。

## 易错

> **易错：** 使用 `except Exception: pass`。
>
> 正确做法：只捕获能处理的异常；至少记录日志或重新抛出。

> **易错：** 不指定编码，导致不同系统上读写结果不一致。
>
> 正确做法：文本文件统一使用 `encoding="utf-8"`，CSV 读写加 `newline=""`。
