# Python学习资料：测试、调试、日志和质量工具

[返回索引](../Python学习资料.md)

## 学习目标

- 掌握单元测试、断言、测试边界和回归测试。
- 能使用调试器、日志和异常栈定位问题。
- 了解格式化、静态检查、类型检查和持续集成的基本价值。

## 理论导读

测试是对代码行为的可执行说明。好的测试不是追求数量，而是覆盖关键输入、边界条件、异常路径和曾经出错的回归场景。调试帮助你理解当前发生了什么，日志帮助你在运行后追溯发生过什么。

质量工具把团队约定自动化。格式化工具减少风格争论，静态检查提前发现明显错误，类型检查帮助维护接口契约，CI 确保每次提交都经过同样的验证。

## 核心心智模型

把测试看成护栏，调试看成显微镜，日志看成行车记录仪，质量工具看成自动门禁。它们解决的是不同阶段的问题，不能互相替代。

## 知识点详解

| 主题 | 内容 |
| --- | --- |
| 单元测试 | 测单个函数或类的行为，输入输出明确。 |
| 集成测试 | 测多个模块协作，覆盖文件、网络、数据库等边界。 |
| 调试 | 使用 `breakpoint()`、`pdb`、IDE 调试器。 |
| 日志 | `logging` 分级、格式、处理器和上下文。 |
| 格式化 | 常见工具如 Black、Ruff format。 |
| 静态检查 | 常见工具如 Ruff、flake8。 |
| 类型检查 | 常见工具如 mypy、pyright。 |

## 例子

被测函数：

```python
def parse_amount(text: str) -> int:
    value = int(text)
    if value < 0:
        raise ValueError("amount must be non-negative")
    return value
```

`unittest` 示例：

```python
import unittest

class ParseAmountTest(unittest.TestCase):
    def test_parse_valid_amount(self):
        self.assertEqual(parse_amount("12"), 12)

    def test_reject_negative_amount(self):
        with self.assertRaises(ValueError):
            parse_amount("-1")
```

日志示例：

```python
import logging

logger = logging.getLogger(__name__)

def process(order_id):
    logger.info("processing order", extra={"order_id": order_id})
```

## 练习

为 `parse_duration(text)` 补充测试：正常格式、空字符串、非法单位、负数、边界值。再加入日志，记录每次解析失败的输入。

## 验收

- 能为纯函数编写单元测试。
- 能覆盖正常路径、异常路径和边界条件。
- 能使用 `breakpoint()` 进入调试。
- 能配置基础日志格式和日志等级。

## 重点

- 测试要验证行为，不要复制实现细节。
- 日志适合长期观测，`print` 只适合临时验证。
- 工具链应服务项目，不要让配置复杂度超过项目收益。

## 难点

- 测试外部依赖时要决定使用真实资源、临时资源还是 mock。
- 日志字段要兼顾可读性和机器检索，生产环境尤其重要。

## 易错

> **易错：** 只测试正常输入，不测试失败路径。
>
> 正确做法：关键函数至少覆盖正常、边界、非法三类输入。

> **易错：** 在库代码里直接配置全局日志。
>
> 正确做法：库代码创建 `logger = logging.getLogger(__name__)`，由应用入口统一配置日志。
