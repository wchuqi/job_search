# Python学习资料：综合练习项目

[返回索引](../Python学习资料.md)

## 项目目标

实现一个 Python 3.x 命令行工具 `log_report`，用于扫描日志目录，过滤错误日志，统计错误类型，输出 JSON 或 CSV 报告，并提供测试、日志、配置和性能说明。

## 需求范围

- 支持命令行参数：输入目录、输出文件、输出格式、日志等级。
- 使用 `pathlib` 遍历目录。
- 流式读取日志文件，不能一次性把所有文件读入内存。
- 提取时间、级别、错误码、消息。
- 支持 JSON 和 CSV 输出。
- 对输入目录不存在、文件编码错误、输出路径不可写给出明确错误。
- 使用 `logging` 记录运行过程。
- 为解析函数、统计函数和输出函数编写测试。

## 实现阶段

## 阶段 1：最小可运行版本

- 读取单个日志文件。
- 过滤包含 `ERROR` 的行。
- 统计错误码出现次数。
- 在终端打印统计结果。

> **验收：** 能用样例日志跑出稳定结果；解析逻辑有单元测试。

## 阶段 2：命令行和项目结构

- 引入 `argparse`。
- 拆分 `cli.py`、`parser.py`、`service.py`。
- 支持目录递归扫描。

> **验收：** 可以通过 `python -m log_report.cli examples` 运行。

## 阶段 3：输出和错误处理

- 支持 JSON 和 CSV 输出。
- 用自定义异常包装文件读取和格式错误。
- 命令行入口根据异常返回非 0 退出码。

> **验收：** 非法输入不会打印难懂调用栈，而是输出清晰错误信息。

## 阶段 4：日志、测试和质量

- 用 `logging` 记录扫描文件数、错误行数、耗时。
- 补齐正常、边界、异常测试。
- 添加格式化和静态检查说明。

> **验收：** 测试能在干净虚拟环境中运行，通过率 100%。

## 阶段 5：性能和扩展

- 对大文件使用生成器。
- 记录同步版本耗时。
- 可选：用线程池并发读取多个文件，并比较收益。

> **验收：** 能给出优化前后耗时数据，并解释为什么变快或没有变快。

## 重点

- 项目要可运行、可测试、可复现。
- 解析、统计、输出要分层，便于测试。
- 大文件处理必须流式读取。

## 难点

- 错误边界设计：底层保留根因，上层输出用户可读信息。
- CSV、JSON、终端输出格式要和业务数据模型解耦。
- 并发读取文件时要考虑磁盘瓶颈，不一定线程越多越快。

## 易错

> **易错：** 为了快速完成把所有逻辑写在 `cli.py`。
>
> 正确做法：入口只解析参数和处理退出，业务逻辑放到可测试函数中。

> **易错：** 性能优化只凭感觉。
>
> 正确做法：用同一批数据重复测试，记录耗时和资源变化。
