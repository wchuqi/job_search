# Shell学习资料：综合练习项目

[返回索引](../Shell学习资料.md)

## 项目目标

编写一个生产可用的 `log-maintainer.sh`，用于扫描应用日志、统计错误、压缩旧日志、清理过期文件，并支持 dry-run、日志、锁、配置校验和安全删除。

## 功能要求

- 参数：`--log-dir DIR`、`--days N`、`--dry-run`、`--report FILE`、`--help`。
- 统计：输出状态码或错误关键字 Top N。
- 压缩：压缩 N 天前未压缩日志。
- 清理：删除 N 天前 `.gz` 文件。
- 安全：只允许操作指定根目录下文件。
- 工程：ShellCheck 通过，失败有日志，临时文件自动清理。

## 推荐结构

```text
shell-project/
  log-maintainer.sh
  tests/
    test-log-maintainer.sh
  fixtures/
    access.log
  README.md
```

## 阶段 1：参数解析

### 练习

- 支持 `--help`。
- 检查 `--log-dir` 必填。
- 检查 `--days` 是正整数。

### 验收

- 未传必填参数时退出码为 2。
- 错误信息写 stderr。

## 阶段 2：日志统计

### 练习

使用 awk 统计 5xx 或 ERROR：

```bash
awk '$9 ~ /^5/ {count[$1]++} END {for (ip in count) print count[ip], ip}' access.log \
  | sort -nr \
  | head
```

### 验收

- 能处理空文件。
- 能处理日志格式不符合预期的行。

## 阶段 3：安全批处理

### 练习

- 用 `find -print0` 查找旧日志。
- dry-run 时只打印操作，不执行。
- 删除前校验 realpath 在允许根目录下。

### 验收

- 文件名包含空格、换行时仍能正确处理。
- `--dry-run` 不改变文件系统。

## 阶段 4：工程化

### 练习

- 增加 `flock` 防重入。
- 增加 trap 清理临时目录。
- 增加 ShellCheck 和测试脚本。

### 验收

- ShellCheck 无关键告警。
- 连续运行两次不会出错。
- 故意制造权限错误时，脚本能输出明确原因。

## 最终验收

- `bash -n log-maintainer.sh` 通过。
- `shellcheck log-maintainer.sh` 无高危问题。
- `./log-maintainer.sh --dry-run --log-dir fixtures --days 7` 不修改文件。
- 删除逻辑有路径范围保护。
- README 说明依赖、参数、退出码和风险。

## 重点

- 这个项目重点不是命令多，而是把 Shell 写成可靠程序。
- 所有危险操作都应先可观察，再可执行。
