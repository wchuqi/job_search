# Shell学习资料：解析器、展开和执行模型深度解析

[返回索引](../Shell学习资料.md)

## 学习目标

- 从解析层面理解 Shell 为什么会产生“看起来没问题”的 bug。
- 掌握命令查找、内建命令、函数、外部命令的执行顺序。
- 理解分词、glob 和 quote removal 的边界。

## 理论导读

Shell 脚本的最终执行单位不是一行字符串，而是解析后的命令结构和参数数组。展开发生在命令执行前，重定向也由 Shell 先处理，然后才执行命令。理解这个过程，才能解释为什么变量加不加引号会改变参数数量，为什么 glob 会在 grep 前先展开，为什么 alias 在脚本里不可靠。

## 命令查找顺序

Bash 执行一个命令名时，大致查找：

1. alias，默认脚本非交互不展开 alias。
2. reserved word，如 `if`、`for`、`case`。
3. function。
4. builtin。
5. hash table 中缓存的外部命令路径。
6. 按 `PATH` 查找外部命令。

检查命令来源：

```bash
type -a cd
type -a test
type -a grep
command -v awk
```

## assignment command

```bash
FOO=bar cmd arg
```

这不是普通变量赋值后再执行，而是给单个命令的环境设置变量。若没有命令：

```bash
FOO=bar
```

则设置当前 Shell 变量。

## 分词和 glob 的真实影响

```bash
var='a b *.txt'
printf '<%s>\n' $var
printf '<%s>\n' "$var"
```

第一条会先把变量展开为 `a b *.txt`，再按 IFS 分词为 `a`、`b`、`*.txt`，最后 `*.txt` 可能进行路径名展开。第二条保留为一个参数。

## 命令替换细节

```bash
out=$(printf 'a\n\n')
printf '<%s>\n' "$out"
```

命令替换会删除所有尾随换行。不要用命令替换保存需要精确保留尾随换行或二进制的数据。

## pipeline subshell

很多 Shell 中，管道各段在子 Shell 中执行：

```bash
count=0
printf '%s\n' a b | while read -r line; do
  count=$((count + 1))
done
echo "$count"
```

在 Bash 默认行为下，`count` 可能仍是 0，因为 `while` 在管道子进程中修改变量。改法：

```bash
count=0
while read -r line; do
  count=$((count + 1))
done < <(printf '%s\n' a b)
echo "$count"
```

## 练习

1. 用 `type -a` 比较 `echo`、`test`、`[`、`time`。
2. 写 `show_args` 函数观察分词和 glob。
3. 验证命令替换删除尾随换行。
4. 验证管道 while 中变量修改是否保留。

## 验收

- 能解释一条命令从文本到执行的步骤。
- 能说明命令查找顺序。
- 能解释 pipeline subshell 导致变量丢失的原因。

## 重点

- 传给命令的是参数数组，不是原始字符串。
- quote removal 在展开之后发生。
- shell option 会改变解析或展开行为，脚本中应谨慎依赖。

## 易错

> **易错：** 以为 `grep *.log` 是 grep 处理通配符。
>
> 正确做法：知道 `*.log` 先由 Shell 展开，grep 收到的是展开后的文件名列表。
