# Shell学习资料：生产故障深度复盘和事故预防

[返回索引](../Shell学习资料.md)

## 学习目标

- 用事故视角审视 Shell 脚本风险。
- 建立生产脚本上线前检查和事故复盘机制。
- 把常见故障映射到可验证的预防措施。

## 事故模型

Shell 事故通常不是单点错误，而是多个弱点叠加：

```text
输入未校验
  + 变量未引用
  + 权限过大
  + 无 dry-run
  + 无日志
  + 无测试环境
  = 生产事故
```

预防要在链路上加多道阻断，而不是只修一行代码。

## 案例 1：空变量导致删除范围扩大

### 触发

```bash
rm -rf "$ROOT/$subdir"
```

`ROOT` 或 `subdir` 来自配置，加载失败后为空或错误。

### 根因

- 配置未校验。
- 删除路径没有 realpath 白名单。
- 无 dry-run。
- 脚本以高权限运行。

### 预防

- `${ROOT:?}` 必填校验。
- `realpath` 后确认在允许根目录。
- dry-run 默认开启，生产需要显式 `--execute`。
- 删除前输出计划和数量。

## 案例 2：cron PATH 导致调用错误命令

### 触发

脚本手动运行正常，cron 中找不到命令或执行了不同版本命令。

### 根因

- 依赖交互 Shell PATH。
- 未记录运行环境。
- 未检查依赖命令。

### 预防

- 脚本开头固定 PATH。
- 启动时打印版本和 `command -v`。
- 用 systemd timer 或 cron 时重定向日志。

## 案例 3：管道吞掉部署失败

### 触发

```bash
deploy | tee deploy.log
```

deploy 失败，tee 成功，脚本继续。

### 预防

- `set -o pipefail`。
- 检查 `PIPESTATUS`。
- 部署关键步骤显式判断。

## 案例 4：source 配置执行恶意命令

### 触发

低权限用户可写配置文件，root 脚本 `source config`。

### 预防

- 配置文件权限检查。
- 不可信配置按白名单解析。
- root 脚本不 source 可写路径文件。

## 上线检查清单

- 解释器和版本明确。
- ShellCheck 通过。
- 参数和配置都有校验。
- dry-run 覆盖危险操作。
- 删除、覆盖、移动都有限定根目录。
- secret 不进日志和命令行。
- cron/CI/容器目标环境已测试。
- 失败退出码可被调度系统感知。
- 重要操作有锁或幂等性。
- 有回滚或恢复步骤。

## 复盘问题

```text
这次事故哪个输入没有被校验？
哪个隐式 Shell 行为参与了事故？
为什么测试没有覆盖？
为什么权限允许扩大影响？
为什么日志不能更早发现？
哪一道防线可以阻断同类事故？
```

## 练习

1. 对已有脚本做上线检查，列出风险。
2. 为删除脚本添加三层保护：配置校验、路径白名单、dry-run。
3. 写事故复盘，必须包含预防措施和验证方法。

## 验收

- 能把事故映射到 Shell 机制和工程治理两层。
- 能为高风险脚本设计上线 gate。
- 能写出可执行的复盘和防复发任务。

## 重点

- Shell 事故预防靠机制理解加工程约束。
- 高权限脚本默认危险。
- 没有 dry-run 和日志的破坏性脚本不应进生产。
