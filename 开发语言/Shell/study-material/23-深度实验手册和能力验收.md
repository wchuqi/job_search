# Shell学习资料：深度实验手册和能力验收

[返回索引](../Shell学习资料.md)

## 实验 1：展开顺序

### 目标

观察变量展开、分词和 glob。

### 步骤

```bash
show_args() { for a in "$@"; do printf '<%s>\n' "$a"; done; }
touch 'a b.txt' x.txt
v='a b *.txt'
show_args $v
show_args "$v"
```

### 验收

能解释两次参数数量不同的原因。

## 实验 2：`"$@"` 和参数转发

```bash
wrapper() {
  target "$@"
}
```

写 `target` 打印参数，传入空格、换行、`*`，验证安全转发。

## 实验 3：set -e 和 pipefail

```bash
set -e
false | true
echo still
set -o pipefail
false | true
echo never
```

### 验收

能解释为什么第一段继续执行。

## 实验 4：find 空字符批处理

创建特殊文件名：

```bash
touch 'a b.log'
touch $'line\nbreak.log'
find . -name '*.log' -print0 | xargs -0 -I{} printf '<%s>\n' '{}'
```

### 验收

能说明为什么普通按行读取不可靠。

## 实验 5：trap 清理

写脚本创建临时目录，运行中 Ctrl-C，确认临时目录被删除。

## 实验 6：while 管道变量丢失

复现并修复：

```bash
count=0
printf '%s\n' a b | while read -r x; do count=$((count+1)); done
echo "$count"
```

## 实验 7：ShellCheck 修复

准备一个包含未引用变量、`for f in $(ls)`、`read` 缺 `-r` 的脚本，用 ShellCheck 修复。

## 实验 8：生产脚本验收

写一个 `backup.sh`：

- 参数解析。
- dry-run。
- 日志。
- flock 防重入。
- trap 清理。
- safe path check。
- ShellCheck 通过。

## 最终能力验收

- 能画出 Shell 解析展开执行顺序。
- 能解释 Bash 和 POSIX sh 的关键差异。
- 能写安全参数解析和错误处理。
- 能用 awk/sed/find/xargs 处理真实文件和日志。
- 能排查 cron、管道、subshell、权限、路径问题。
- 能识别高风险删除和 secret 泄漏。
