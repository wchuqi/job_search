## 一、退出码和条件

### 1. Shell 中 if 判断的本质是什么？

**参考答案：**

`if` 判断的是命令退出状态。退出状态为 0 表示成功，进入 then；非 0 表示失败，进入 else 或跳过。`[` 是 `test` 命令，`[[ ]]` 是 Bash 关键字。

### 2. `[ ]` 和 `[[ ]]` 有什么区别？

**参考答案：**

`[ ]` 是 POSIX test 命令，兼容性好但需要更小心引用和转义；`[[ ]]` 是 Bash 关键字，变量展开更安全，支持 `&&`、`||`、模式匹配和正则匹配。Bash 脚本优先用 `[[ ]]`，POSIX sh 用 `[ ]`。

### 3. `set -euo pipefail` 分别是什么意思？

**参考答案：**

`-e` 让部分未处理失败导致脚本退出；`-u` 引用未设置变量时报错；`pipefail` 让管道整体状态反映管道中失败命令；常配合 `-E` 和 `ERR trap` 增强错误上下文。

> **易错：** `set -e` 有很多例外，不能替代显式错误处理。

### 4. 为什么 `cmd | tee log` 可能掩盖 cmd 失败？

**参考答案：**

管道默认退出状态是最后一个命令的退出状态。如果 `cmd` 失败但 `tee` 成功，管道整体仍可能是 0。应启用 `set -o pipefail` 或检查 `PIPESTATUS`。

### 5. grep 没匹配时返回 1，算错误吗？

**参考答案：**

要看场景。grep 返回 1 表示未匹配，可能是预期结果；返回 2 通常表示执行错误。严谨脚本应区分未匹配和执行失败，而不是简单 `grep ... || true`。

### 6. `return` 和 `exit` 有什么区别？

**参考答案：**

`return` 从函数或 source 的脚本返回，设置函数退出状态；`exit` 退出当前 Shell 脚本进程。函数中通常用 `return`，顶层入口决定是否 `exit`。
