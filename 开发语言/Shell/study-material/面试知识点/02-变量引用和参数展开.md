## 一、变量和引用

### 1. 为什么 Shell 变量通常要加双引号？

**参考答案：**

未引用变量展开后会发生 word splitting 和 pathname expansion，包含空格、换行、通配符的值会变成多个参数或匹配文件名。双引号允许变量展开，但阻止分词和 glob，能保留为一个参数。

```bash
cp -- "$src" "$dst"
```

### 2. 单引号和双引号有什么区别？

**参考答案：**

单引号强引用，内容几乎原样保留，不展开变量；双引号弱引用，变量展开、命令替换、算术展开仍发生，但展开结果不再按 IFS 分词和 glob。

### 3. `"$@"` 和 `"$*"` 有什么区别？

**参考答案：**

`"$@"` 会把每个位置参数作为独立参数保留，适合转发参数；`"$*"` 会把所有位置参数拼成一个字符串。脚本包装命令时几乎总是用 `"$@"`。

### 4. Shell 变量和环境变量有什么区别？

**参考答案：**

Shell 变量默认只在当前 Shell 中存在，子进程看不到。`export` 后才进入环境变量，外部命令作为子进程才能读取。`VAR=value cmd` 可以只给单次命令设置环境变量。

### 5. `${var:-default}` 和 `${var-default}` 有什么区别？

**参考答案：**

`${var:-default}` 在 var 未设置或为空时使用 default；`${var-default}` 只在 var 未设置时使用 default，如果 var 为空字符串则使用空字符串。

> **难点：** 冒号决定是否把空字符串也当作缺失。

### 6. Bash 数组解决什么问题？

**参考答案：**

数组能安全保存参数列表，避免把多个参数拼成一个字符串后再分词。构造命令参数时应使用数组：

```bash
args=(--name "$name" --path "$path")
cmd "${args[@]}"
```
