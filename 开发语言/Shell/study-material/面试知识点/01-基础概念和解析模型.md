## 一、基础概念

### 1. Shell 是什么？

**参考答案：**

Shell 是命令解释器，也是脚本语言和系统工具胶水层。它读取命令文本，完成语法解析、变量展开、命令替换、分词、glob、重定向，然后执行内建命令、函数或外部程序，并通过退出状态组织控制流。

> **重点：** Shell 的核心不是语法多，而是解析、展开和退出码。

### 2. Bash 和 sh 有什么区别？

**参考答案：**

`sh` 通常表示 POSIX Shell 接口，不同系统上可能是 dash、bash 的 POSIX 模式、ash 等。Bash 是功能更丰富的 Shell，支持数组、`[[ ]]`、`(( ))`、`pipefail`、process substitution、关联数组等扩展。写 Bash 特性时 shebang 应明确为 Bash。

> **易错：** `/bin/sh` 不等于 Bash。

### 3. Shell 执行一条命令的大致步骤是什么？

**参考答案：**

读取文本后先解析语法结构，再进行 brace、tilde、参数、命令、算术展开，然后执行 word splitting、pathname expansion 和 quote removal，随后处理重定向、查找命令并执行，最后得到退出状态。

### 4. Shell 中命令查找顺序是什么？

**参考答案：**

Bash 中大致包括 alias、关键字、函数、内建命令、hash 缓存、`PATH` 外部命令。脚本中默认不应依赖 alias。可用 `type -a cmd` 和 `command -v cmd` 检查。

### 5. 为什么 Shell 脚本常出现“本机可以，生产不行”？

**参考答案：**

常见原因是解释器不同、Bash/POSIX 差异、PATH 不同、GNU/BSD/BusyBox 工具差异、当前目录不同、权限不同、locale 不同、交互 Shell 配置未加载，以及文件名或输入数据在生产中更复杂。
