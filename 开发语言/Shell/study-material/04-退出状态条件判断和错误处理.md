# Shell学习资料：退出状态、条件判断和错误处理

[返回索引](../Shell学习资料.md)

## 学习目标

- 理解退出状态是 Shell 控制流的核心。
- 掌握 `if`、`test`、`[[ ]]`、`case`、`&&`、`||`。
- 正确使用 `set -euo pipefail`，知道它的边界。

## 理论导读

Shell 中命令的成功失败由退出状态表示，`0` 表示成功，非 `0` 表示失败。条件判断本质上也是执行命令并看退出状态。`[` 不是语法括号，而是 `test` 命令的另一个名字。Bash 的 `[[ ]]` 是关键字，语义更安全，支持模式和正则。

## 退出状态

```bash
grep -q 'error' app.log
status=$?
if [[ $status -eq 0 ]]; then
  echo found
fi
```

常见状态：

- `0`：成功。
- `1`：一般失败，grep 未匹配也返回 1。
- `2`：命令用法错误常见状态。
- `126`：找到命令但无法执行。
- `127`：命令不存在。
- `130`：被 Ctrl-C 中断。

## 条件判断

```bash
if [[ -f "$file" && -s "$file" ]]; then
  echo "regular non-empty file"
fi
```

`[[ ]]` 中变量展开不会发生 word splitting 和 glob，通常比 `[ ]` 更适合 Bash 脚本。

POSIX sh 中使用：

```sh
if [ -f "$file" ] && [ -s "$file" ]; then
  echo ok
fi
```

## 字符串、数字、文件判断

```bash
[[ -z "$s" ]]      # 空字符串
[[ -n "$s" ]]      # 非空
[[ "$a" == "$b" ]] # 字符串相等
[[ $n -gt 10 ]]    # 数字比较
[[ -e "$p" ]]      # 存在
[[ -d "$p" ]]      # 目录
[[ -x "$p" ]]      # 可执行
```

## `&&` 和 `||`

```bash
mkdir -p "$dir" && cp -- "$src" "$dir/"
```

`cmd1 && cmd2` 表示 cmd1 成功才执行 cmd2。`cmd1 || cmd2` 表示 cmd1 失败才执行 cmd2。不要把复杂错误处理写成难读的链式表达式。

## set 选项

```bash
set -Eeuo pipefail
```

- `-e`：简单命令失败时退出，但有很多例外。
- `-E`：让 ERR trap 在函数、命令替换、子 Shell 中继承得更符合预期。
- `-u`：引用未设置变量时报错。
- `pipefail`：管道退出状态为第一个失败命令，非默认的最后命令状态。

> **难点：** `set -e` 在 `if` 条件、`while` 条件、`&&`、`||`、管道非最后命令等上下文中不会按直觉退出。

## 错误处理模板

```bash
#!/usr/bin/env bash
set -Eeuo pipefail

trap 'echo "error: line=$LINENO status=$?" >&2' ERR

require_cmd() {
  command -v "$1" >/dev/null 2>&1 || {
    echo "missing command: $1" >&2
    return 127
  }
}

main() {
  require_cmd rsync
  rsync -a -- "$1" "$2"
}

main "$@"
```

## 练习

1. 写脚本分别触发退出码 1、126、127。
2. 比较 `false | true` 在有无 `pipefail` 下的退出状态。
3. 写一个函数，用 `return` 返回错误，由调用方处理。

## 验收

- 能解释 Shell 条件判断为什么看退出码。
- 能区别 `[ ]` 和 `[[ ]]`。
- 能说明 `set -e` 的至少三个例外场景。

## 重点

- Shell 的错误处理不能只靠 `set -e`。
- 预期失败要显式处理，比如 grep 未匹配。
- 函数用 `return`，脚本顶层用 `exit`。

## 易错

> **易错：** 看到 `set -euo pipefail` 就认为脚本错误处理可靠。
>
> 正确做法：理解例外上下文，对关键步骤显式检查并输出可诊断日志。
