# Shell学习资料：生产脚本排障案例和剧本

[返回索引](../Shell学习资料.md)

## 学习目标

- 掌握 Shell 生产故障的分层排查方法。
- 能快速定位环境、解析、权限、退出码、路径和数据风险。
- 为脚本事故建立复盘模板。

## 总体排障框架

先收集：

```bash
set -x
echo "shell=$SHELL bash=${BASH_VERSION-}"
id
pwd
printf 'PATH=%s\n' "$PATH"
type -a command_name
```

看四层：

1. 环境：解释器、PATH、工作目录、权限、locale、cron/CI。
2. 输入：参数、文件名、空格、换行、通配符、空变量。
3. 执行：退出码、管道、subshell、set 选项。
4. 数据：删除、覆盖、备份、临时文件、secret。

## 剧本 1：cron 中成功手动执行失败

### 常见原因

- PATH 不同。
- 当前目录不同。
- 没有交互环境变量。
- 权限或用户不同。
- 输出未记录。

### 修复

- 脚本内设置 PATH。
- 使用绝对路径或切到脚本目录。
- 显式记录日志。
- 用 `env -i` 模拟最小环境。

## 剧本 2：文件名包含空格导致失败

### 症状

`No such file`、参数数量异常、处理部分文件。

### 排查

添加参数打印：

```bash
printf 'arg=<%s>\n' "$@"
```

### 修复

- 变量加双引号。
- 列表用数组。
- find 用 `-print0`。

## 剧本 3：管道前面的命令失败但脚本成功

### 症状

日志为空，但脚本退出码 0。

### 原因

管道默认取最后命令退出码。

### 修复

```bash
set -o pipefail
cmd | tee log
```

或检查 `PIPESTATUS`。

## 剧本 4：while 管道中变量丢失

### 原因

管道中的 while 可能运行在子 Shell。

### 修复

```bash
while IFS= read -r line; do
  count=$((count + 1))
done < <(generate_lines)
```

## 剧本 5：误删或差点误删

### 排查

- 立刻停止脚本。
- 保留命令日志。
- 查看变量值和 realpath。
- 检查是否有备份。

### 预防

- dry-run。
- 路径白名单。
- 非空校验。
- `--`。
- 删除前打印计划。

## 复盘模板

```text
运行环境：
输入参数：
影响范围：
直接原因：
根本原因：
为什么测试没发现：
修复：
预防：
```

## 验收

- 能用最小环境复现 cron 问题。
- 能定位路径、引用和管道退出码问题。
- 能为删除类脚本写事故预防清单。

## 重点

- Shell 故障先看环境和参数。
- 生产脚本必须有日志和 dry-run。
- 数据风险优先于脚本修复速度。
