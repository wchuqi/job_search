# Shell学习资料：数组、字符串和参数展开

[返回索引](../Shell学习资料.md)

## 学习目标

- 掌握 Bash indexed array 和 associative array。
- 使用参数展开处理默认值、截取、替换、长度和间接引用。
- 避免数组展开中的分词错误。

## 理论导读

Shell 原生数据结构很少，Bash 数组是写复杂脚本时最重要的扩展之一。数组能安全保存多个参数，避免把列表拼成一个字符串后再拆分。字符串处理主要依赖参数展开，但复杂文本处理应交给 awk、sed、perl、python 等工具。

## indexed array

```bash
files=("a.txt" "b c.txt" "*.log")

for f in "${files[@]}"; do
  printf '<%s>\n' "$f"
done
```

`"${arr[@]}"` 保留每个元素为独立参数。`"${arr[*]}"` 会把所有元素拼成一个字符串。

## associative array

```bash
declare -A ports=(
  [web]=8080
  [db]=5432
)

printf '%s\n' "${ports[web]}"
```

关联数组需要 Bash 4+。macOS 老版本系统 Bash 可能是 3.2，不支持关联数组。

## 参数展开

```bash
path=/var/log/nginx/access.log
echo "${path##*/}"   # access.log
echo "${path%/*}"    # /var/log/nginx
echo "${path%.log}"  # /var/log/nginx/access
```

| 写法 | 含义 |
| --- | --- |
| `${#var}` | 字符串长度 |
| `${var:offset:length}` | 子串 |
| `${var#pattern}` | 从开头删除最短匹配 |
| `${var##pattern}` | 从开头删除最长匹配 |
| `${var%pattern}` | 从结尾删除最短匹配 |
| `${var%%pattern}` | 从结尾删除最长匹配 |
| `${var/pat/repl}` | 替换第一个匹配 |
| `${var//pat/repl}` | 替换全部匹配 |

## 安全拼接命令参数

不要这样：

```bash
args="--name $name --path $path"
cmd $args
```

如果路径含空格会坏。应该用数组：

```bash
args=(--name "$name" --path "$path")
cmd "${args[@]}"
```

## 练习

1. 用数组保存多个文件路径，其中至少一个路径包含空格。
2. 用参数展开提取文件名、目录名、扩展名。
3. 用关联数组统计不同服务端口。

## 验收

- 能解释 `"${arr[@]}"` 和 `"${arr[*]}"` 的区别。
- 能用数组安全构造命令参数。
- 能使用 `${var##*/}` 和 `${var%/*}`。

## 重点

- 列表用数组，不要用空格拼字符串。
- 关联数组不是 POSIX sh。
- 复杂数据结构不适合硬塞进 Shell。

## 易错

> **易错：** 把命令参数拼成字符串再执行。
>
> 正确做法：用 Bash 数组保存参数，再用 `"${args[@]}"` 展开。
