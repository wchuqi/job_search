# Shell学习资料：变量、参数、环境和作用域

[返回索引](../Shell学习资料.md)

## 学习目标

- 掌握变量赋值、环境变量、位置参数、特殊参数。
- 理解 Shell 变量和子进程环境的关系。
- 正确使用 `local`、`readonly`、`export`。

## 理论导读

Shell 变量默认只存在于当前 Shell 进程。外部命令作为子进程运行，只能看到被 `export` 的环境变量。函数中的变量默认也是全局的，Bash 中需要 `local` 才能限制在函数内。很多脚本污染全局状态，就是因为忘记 `local`。

## 变量赋值

```bash
name="alice"
count=3
empty=""
```

赋值时等号两边不能有空格。`name = alice` 会被解析为执行命令 `name`，参数是 `=` 和 `alice`。

## 环境变量

```bash
export APP_ENV=prod
APP_ENV=prod ./app
```

第一种把变量导出到当前 Shell 后续所有子进程；第二种只给单次命令设置环境变量。

## 位置参数

| 参数 | 含义 |
| --- | --- |
| `$0` | 脚本名或 Shell 名 |
| `$1`..`${10}` | 位置参数 |
| `$#` | 参数数量 |
| `"$@"` | 每个参数独立保留 |
| `$?` | 上一个命令退出状态 |
| `$$` | 当前 Shell PID |
| `$!` | 最近后台任务 PID |

## 函数作用域

```bash
parse() {
  local input=$1
  local result
  result=${input^^}
  printf '%s\n' "$result"
}
```

Bash 的 `local` 是动态作用域：函数调用链中的子函数可能看到上层函数的 local 变量。这和多数语言的词法作用域不同。

## 只读变量

```bash
readonly SCRIPT_DIR
SCRIPT_DIR=$(cd -- "$(dirname -- "${BASH_SOURCE[0]}")" && pwd)
```

只读适合常量，防止后续误改。

## 默认值展开

```bash
: "${APP_ENV:=dev}"
port=${PORT:-8080}
required=${DB_URL:?DB_URL is required}
```

| 写法 | 含义 |
| --- | --- |
| `${var:-word}` | var 未设置或为空时使用 word |
| `${var-word}` | var 未设置时使用 word，空字符串仍使用 var |
| `${var:=word}` | 未设置或为空时赋值为 word |
| `${var:?msg}` | 未设置或为空时报错退出 |
| `${var:+word}` | var 设置且非空时使用 word |

## 练习

1. 写函数时不加 `local`，观察全局变量被覆盖。
2. 用 `APP_ENV=prod env | grep APP_ENV` 和 `APP_ENV=prod; env | grep APP_ENV` 比较。
3. 使用 `${var:?}` 实现必填配置校验。

## 验收

- 能解释变量和环境变量的区别。
- 能正确使用 `local` 避免污染全局变量。
- 能用参数展开处理默认值和必填值。

## 重点

- Shell 变量不自动传给子进程，必须 `export`。
- 函数变量默认全局，Bash 中要用 `local`。
- 位置参数超过 9 要写 `${10}`。

## 易错

> **易错：** `export "$key=$value"` 处理不可信 key。
>
> 正确做法：先校验变量名格式，避免注入非法环境名或覆盖关键变量。
