# Shell学习资料：命令解析、展开和引用机制

[返回索引](../Shell学习资料.md)

## 学习目标

- 掌握 Shell 展开顺序。
- 理解单引号、双引号、反斜杠、命令替换和 glob。
- 能避免空格、换行、通配符导致的参数错误。

## 理论导读

Shell 最危险也最核心的机制是展开。你写的是一段文本，Shell 执行前会把变量、命令替换、算术、通配符等转换为最终参数列表。最终传给程序的不是你屏幕上看到的字符串，而是 Shell 展开后的参数数组。

## 展开顺序

Bash 中常见展开顺序可简化理解为：

1. brace expansion：`{a,b}`、`{1..3}`。
2. tilde expansion：`~`。
3. parameter expansion：`$var`、`${var}`。
4. command substitution：`$(cmd)`。
5. arithmetic expansion：`$((expr))`。
6. word splitting：按 `$IFS` 分词。
7. pathname expansion：`*`、`?`、`[]` glob。
8. quote removal：去掉语法引号。

> **重点：** 引号影响的是后续分词和 glob。`"$var"` 通常保留为一个参数。

## 引用规则

| 写法 | 含义 | 典型用途 |
| --- | --- | --- |
| `'text'` | 强引用，不展开变量 | 原样字符串 |
| `"text $var"` | 弱引用，变量和命令替换仍发生，不做分词和 glob | 绝大多数变量使用 |
| `\ ` | 转义单个字符 | 临时转义空格或特殊字符 |
| `$'a\nb'` | Bash ANSI-C quoting | 写转义字符 |

## 参数安全

错误：

```bash
cp $src $dst
```

如果 `src="a b.txt"`，会被拆成两个参数。

正确：

```bash
cp -- "$src" "$dst"
```

`--` 用于告诉命令后面是普通参数，避免文件名以 `-` 开头被当选项。

## `"$@"` 和 `$*`

```bash
for arg in "$@"; do
  printf '<%s>\n' "$arg"
done
```

`"$@"` 会把每个位置参数保留为独立参数，是转发参数的标准写法。

| 写法 | 行为 |
| --- | --- |
| `$@` | 未引用，可能被分词和 glob |
| `"$@"` | 每个参数独立保留 |
| `$*` | 未引用，类似拼接后再分词 |
| `"$*"` | 所有参数拼成一个字符串 |

## 例子：观察最终参数

```bash
show_args() {
  local i=0
  for arg in "$@"; do
    printf 'arg[%d]=<%s>\n' "$i" "$arg"
    i=$((i + 1))
  done
}

name='a b *.txt'
show_args $name
show_args "$name"
```

## 练习

1. 创建文件 `a b.txt` 和 `x.txt`，比较 `ls $var` 与 `ls "$var"`。
2. 写 `show_args` 函数观察不同引用方式的参数数量。
3. 创建名为 `-rf` 的文件，尝试用 `rm -- "$file"` 删除。

## 验收

- 能背出常见展开顺序。
- 能解释为什么变量通常要加双引号。
- 能正确使用 `"$@"` 转发参数。

## 难点

- 引号不会阻止变量展开，但会阻止展开结果继续被分词和 glob。
- 命令替换会删除末尾换行，处理二进制或保留换行时要谨慎。

## 易错

> **易错：** 使用 `for f in $(ls)` 遍历文件。
>
> 正确做法：用 glob、`find -print0` 加 `read -d ''`，或直接让命令接收文件参数。
