# Shell学习资料：控制流、函数和脚本结构

[返回索引](../Shell学习资料.md)

## 学习目标

- 掌握 `if`、`case`、`for`、`while`、`until`。
- 写出结构清晰的函数和 main 入口。
- 正确处理参数解析、返回值和全局变量。

## 理论导读

Shell 脚本可维护性的关键是控制流简单、函数职责清楚、全局状态少。函数不是子程序隔离环境，默认会修改当前 Shell 的变量和工作目录。脚本越接近生产，越应该有 `main "$@"`、日志函数、usage、参数解析和统一退出路径。

## if 和 case

```bash
if [[ -f "$path" ]]; then
  echo file
elif [[ -d "$path" ]]; then
  echo dir
else
  echo missing
fi
```

`case` 适合模式分支和参数解析：

```bash
case "$1" in
  start|stop|restart) action=$1 ;;
  -h|--help) usage; exit 0 ;;
  *) echo "unknown: $1" >&2; exit 2 ;;
esac
```

## 循环

```bash
for item in "$@"; do
  printf '%s\n' "$item"
done
```

按行读取文件：

```bash
while IFS= read -r line; do
  printf '<%s>\n' "$line"
done < input.txt
```

`IFS= read -r` 是为了保留前后空白并避免反斜杠转义。

## 函数设计

```bash
log() {
  printf '[%s] %s\n' "$(date '+%F %T')" "$*" >&2
}

die() {
  log "fatal: $*"
  exit 1
}

copy_file() {
  local src=$1
  local dst=$2
  [[ -f "$src" ]] || return 1
  cp -- "$src" "$dst"
}
```

函数中尽量用 `return` 表示失败，顶层决定是否 `exit`。

## 参数解析模板

```bash
usage() {
  cat <<'EOF'
usage: backup.sh [--dry-run] --src DIR --dst DIR
EOF
}

dry_run=false
src=
dst=

while (($#)); do
  case "$1" in
    --dry-run) dry_run=true; shift ;;
    --src) src=${2:?missing value for --src}; shift 2 ;;
    --dst) dst=${2:?missing value for --dst}; shift 2 ;;
    -h|--help) usage; exit 0 ;;
    --) shift; break ;;
    -*) echo "unknown option: $1" >&2; exit 2 ;;
    *) break ;;
  esac
done
```

## 练习

1. 写一个支持 `start|stop|status` 的服务管理脚本。
2. 写一个支持 `--dry-run` 的复制脚本。
3. 把重复逻辑抽成函数，并用 `local` 避免全局污染。

## 验收

- 能写 main 入口和 usage。
- 能用 `case` 解析参数。
- 能正确用 `while IFS= read -r` 读文件。

## 重点

- 参数解析必须处理未知参数和缺失值。
- 函数默认共享当前 Shell 状态。
- 读文件不要用 `for line in $(cat file)`。

## 易错

> **易错：** `while read line; do ...; done < file` 没有设置 `IFS=` 和 `-r`。
>
> 正确做法：`while IFS= read -r line; do ...; done < file`。
