# Shell学习资料：通配符、正则和模式匹配

[返回索引](../Shell学习资料.md)

## 学习目标

- 区分 glob、正则、Shell pattern。
- 掌握 `case`、`[[ == pattern ]]`、`[[ =~ regex ]]`。
- 避免通配符误匹配和未匹配行为。

## 理论导读

Shell 中有多种“模式”：glob 用于文件路径匹配，`case` 和 `[[ == ]]` 使用 Shell pattern，`grep`、`sed`、`awk` 使用正则。它们语法相似但不是一回事。把 glob 当正则、把正则当 glob，是 Shell 脚本常见错误。

## glob

| 模式 | 含义 |
| --- | --- |
| `*` | 匹配任意字符串 |
| `?` | 匹配任意单字符 |
| `[abc]` | 匹配字符集合 |
| `[!abc]` | 非集合 |

```bash
for f in ./*.log; do
  [[ -e "$f" ]] || continue
  gzip -- "$f"
done
```

如果没有匹配，Bash 默认保留原样 `./*.log`。所以要检查 `[[ -e "$f" ]]`，或启用 `nullglob`。

## case pattern

```bash
case "$file" in
  *.tar.gz) echo "tar gzip" ;;
  *.log) echo "log" ;;
  *) echo "other" ;;
esac
```

## Bash 正则

```bash
if [[ "$line" =~ ^([0-9]{3})[[:space:]]+(.+)$ ]]; then
  code=${BASH_REMATCH[1]}
  msg=${BASH_REMATCH[2]}
fi
```

`[[ string =~ regex ]]` 右侧正则通常不要加引号，否则不同 Bash 版本和写法下容易改变匹配语义。

## grep 正则

```bash
grep -E 'error|fatal' app.log
grep -F 'literal.*text' app.log
```

- `grep`：基础正则。
- `grep -E`：扩展正则。
- `grep -F`：固定字符串，不按正则解释。

## 练习

1. 比较 `*.log` 在 Shell 中展开和在 `grep '*.log'` 中的含义。
2. 写 `case` 判断文件类型。
3. 用 `[[ =~ ]]` 从日志行提取状态码。

## 验收

- 能区分 glob、Shell pattern、regex。
- 能处理 glob 没有匹配时的行为。
- 能用 `grep -F` 避免把用户输入当正则。

## 重点

- Shell 先展开 glob，再执行命令。
- 正则通常由 grep/sed/awk 解释，不由 Shell 解释。
- 用户输入默认不应直接当正则。

## 易错

> **易错：** `grep $pattern file`，当 pattern 含空格、`*`、`[` 时出错。
>
> 正确做法：`grep -- "$pattern" file`，固定字符串用 `grep -F -- "$pattern" file`。
