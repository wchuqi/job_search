# Shell学习路线图

## 阶段 1：基础认知

- 目标：理解 Shell 是命令解释器、交互环境和脚本语言的结合。
- 需要掌握：Bash、sh、zsh、命令、参数、变量、退出状态、标准输入输出错误。
- 例子：用 Shell 启动命令、组合命令、查看退出码。
- 练习：写一个检查文件是否存在并输出结果的小脚本。
- 验收：能解释 Shell 脚本不是“命令逐行文本”，而是经过解析、展开、执行的程序。
- 重点：Shell 的核心难点是解析和展开，不是命令数量。
- 易错：把所有 Shell 都当 Bash，用 Bash 特性写 `/bin/sh` 脚本。

## 阶段 2：核心语义

- 目标：掌握引用、变量、参数展开、条件判断、循环、函数、退出码。
- 需要掌握：单引号、双引号、命令替换、算术展开、`$?`、`test`、`[[ ]]`、`case`、函数返回。
- 例子：解析参数，检查依赖命令，按条件执行任务。
- 练习：写一个带 `--dry-run`、`--force` 参数的清理脚本。
- 验收：能解释 `"$@"`、`$*`、`"$var"`、`${var:-default}` 的差异。
- 重点：变量展开几乎总要加双引号。
- 易错：用未加引号变量处理包含空格、换行或通配符的路径。

## 阶段 3：数据流和系统操作

- 目标：能用重定向、管道、文件描述符和文本处理工具处理真实数据。
- 需要掌握：stdin/stdout/stderr、`2>&1`、管道、process substitution、grep、sed、awk、find、xargs。
- 例子：扫描日志、统计错误、批量处理文件。
- 练习：写一个日志分析脚本，输出接口错误率和 Top N IP。
- 验收：能解释管道中每个命令的退出状态和 `pipefail` 的作用。
- 重点：Shell 适合胶水脚本和流式文本处理。
- 易错：用 `for f in $(find ...)` 处理文件名，导致空格和换行出错。

## 阶段 4：工程化和安全

- 目标：写出可维护、可调试、可审计的脚本。
- 需要掌握：脚本结构、日志函数、错误处理、trap 清理、ShellCheck、测试、dry-run、安全删除。
- 例子：部署脚本、备份脚本、巡检脚本。
- 练习：为脚本增加 `--help`、日志、错误退出、临时目录清理和单元测试。
- 验收：ShellCheck 无高危问题，失败时有清晰日志，危险操作有确认或 dry-run。
- 重点：生产脚本必须先保护数据，再追求自动化。
- 易错：直接 `rm -rf "$dir"`，没有校验变量为空、路径范围和 dry-run。

## 阶段 5：深入机制和生产排障

- 目标：掌握 Shell 解析顺序、subshell、信号、异步任务、可移植性和性能边界。
- 需要掌握：brace、tilde、parameter、command、arithmetic、word splitting、pathname expansion 的顺序；subshell；`set -e` 陷阱；trap；locale；POSIX 差异。
- 例子：解释为什么 `while read` 管道中变量修改丢失，为什么 `set -e` 没有在某些位置退出。
- 练习：完成 23 的深度实验，写排障报告。
- 验收：能从解析、展开、执行、退出码四层定位 Shell bug。
- 重点：Shell 的“坑”多数来自隐式展开和特殊上下文。
- 易错：死记 `set -euo pipefail`，不知道它们在函数、条件、管道和命令替换中的边界。

## 阶段 6：机制级 Bash 和生产脚本设计

- 目标：能解释 Bash 的命令分类、查找顺序、simple command 展开和重定向顺序、special builtin 影响、IFS 分词规则、glob 选项、subshell 和 trap 继承。
- 需要掌握：alias/keyword/function/builtin/PATH 查找、assignment command、redirection-only command、`exec` 改 FD、process substitution、coproc、`lastpipe`、`ERR/DEBUG/RETURN` trap、getopts 状态机、配置优先级、TOCTOU。
- 例子：解释为什么 `VAR=1 cd /tmp` 不只是普通环境变量设置，为什么 `while read` 管道变量丢失，为什么 `cmd 2>&1 >file` 和 `cmd >file 2>&1` 不同。
- 练习：完成 24 到 34 的机制实验和故障复盘，把每个现象写成“解析规则 -> 运行结果 -> 修复方式”。
- 验收：能在不运行脚本的情况下预测参数数量、退出状态、变量作用域和文件描述符变化。
- 重点：深度 Shell 能力是可预测性，不是记更多一行命令。
- 易错：把 Bash 行为归因于“玄学”，没有回到解析、展开、查找、重定向、执行和退出状态。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 基础语法、引用、变量、退出码 | 10 个小脚本 |
| 第 2 周 | 重定向、管道、文本处理、find/xargs | 日志分析和批处理脚本 |
| 第 3 周 | 函数、参数解析、trap、调试、ShellCheck | 工程化脚本模板 |
| 第 4 周 | 安全、可移植性、生产排障 | 部署或备份脚本和排障剧本 |
| 第 5 周 | 解析模型、set 选项、awk/sed 深入 | 深度实验报告 |
| 第 6 周 | Bash 命令模型、IFS/glob、FD、subshell、trap | 机制级解释笔记 |
| 第 7 周 | 参数优先级、兼容性、安全攻防、CI、性能 | 生产脚本框架和事故预防清单 |

## 最终能力清单

- 能写 Bash 脚本，也知道哪些写法不适合 POSIX sh。
- 能解释 Shell 解析、展开、分词和 glob 的顺序。
- 能正确处理路径、空格、换行、退出码、管道和错误。
- 能用 grep、sed、awk、find、xargs 做可靠批处理。
- 能写具备日志、参数解析、dry-run、trap 清理和 ShellCheck 检查的生产脚本。
- 能识别高风险命令和 secret 泄漏路径。
- 能解释 Bash 查找命令、处理赋值、建立重定向、执行内建和外部命令的细节。
- 能设计具备配置优先级、安全边界、测试、发布和性能评估的 Shell 脚本体系。
