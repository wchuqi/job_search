# TypeScript学习资料：联合、交叉、字面量和收窄

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 掌握联合类型、交叉类型和字面量类型。
- 理解控制流分析和类型收窄。
- 能使用判别联合表达有限状态。
- 能用 `never` 做穷尽检查。

## 理论导读

联合类型表示“值可能是多种形态之一”，交叉类型表示“同时满足多个类型”。字面量类型把值限制为具体字符串、数字或布尔值。TypeScript 会根据控制流判断，在不同分支里自动缩小类型，这就是收窄。

判别联合是 TypeScript 建模业务状态最重要的技巧之一。它把状态名称和状态数据绑定在一起，让非法组合无法出现。

## 核心心智模型

联合类型像岔路口：

```text
当前只知道它可能是 A 或 B
  -> 通过 typeof / in / 判别字段检查
  -> 进入某个分支后确定为 A
```

## 知识点详解

### 联合类型

```ts
type Id = string | number;
```

使用联合类型时，只能直接访问所有成员共同拥有的能力。

### 交叉类型

```ts
type WithId = { id: string };
type WithTimestamp = { createdAt: Date };
type Entity = WithId & WithTimestamp;
```

交叉常用于组合对象能力，但不应滥用来拼出难以理解的类型。

### 字面量类型

```ts
type Level = "INFO" | "WARN" | "ERROR";
```

字面量类型适合表达枚举值、状态码、事件名、权限动作。

### 判别联合

```ts
type Result<T> =
  | { kind: "loading" }
  | { kind: "success"; data: T }
  | { kind: "failed"; error: string };

function render(result: Result<string>): string {
  switch (result.kind) {
    case "loading":
      return "loading";
    case "success":
      return result.data;
    case "failed":
      return result.error;
  }
}
```

### 穷尽检查

```ts
function assertNever(value: never): never {
  throw new Error(`unhandled: ${value}`);
}
```

当新增联合成员却没有处理时，`never` 能让编译器报错。

## 例子

```ts
type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rect"; width: number; height: number };

function area(shape: Shape): number {
  if (shape.kind === "circle") {
    return Math.PI * shape.radius ** 2;
  }
  return shape.width * shape.height;
}
```

## 练习

1. 用判别联合表达订单支付状态。
2. 写一个 `formatId(id: string | number)`。
3. 为 `Shape` 增加 `triangle`，用 `never` 发现未处理分支。
4. 比较 `type A = X | Y` 和 `type B = X & Y` 的差异。

## 验收

- 能解释联合和交叉的语义。
- 能用 `typeof`、`in`、判别字段做收窄。
- 能使用字面量类型限制可选值。
- 能写出穷尽检查。

## 重点

- 判别联合是表达有限状态机的首选。
- 收窄依赖控制流，随分支、返回、抛错而变化。
- `never` 能把“理论不可能”交给编译器验证。

## 难点

- 交叉类型不是“二选一”，而是“同时满足”。对象交叉常见，原始类型交叉可能变成 `never`。

## 易错

> **易错：** 用 `{ loading?: boolean; data?: T; error?: string }` 表达请求状态。
>
> 正确做法：用判别联合，让 `success` 必须有 `data`，`failed` 必须有 `error`。
