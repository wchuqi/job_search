# TypeScript学习资料：类型系统基础和结构化类型

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 理解 TypeScript 的结构化类型系统。
- 掌握对象类型、可选属性、只读属性、接口和类型别名。
- 理解赋值兼容性和多余属性检查。
- 能设计稳定的业务数据类型。

## 理论导读

TypeScript 采用结构化类型系统。一个值是否满足某个类型，主要看它是否拥有需要的结构，而不是看它显式声明“实现了某个类型”。这和 Java 等名义类型系统不同。

结构化类型让 JavaScript 生态更自然：只要对象有需要的属性，就能传给函数。但它也带来边界：类型兼容可能比你想象得宽，尤其是变量赋值和对象字面量的检查规则不同。

## 核心心智模型

TypeScript 问的是：

```text
这个值有没有我需要的字段和方法？
```

不是：

```text
这个值是不是某个类的实例？
```

## 知识点详解

### 对象类型

```ts
type User = {
  id: string;
  name: string;
  age?: number;
  readonly createdAt: Date;
};
```

- `age?` 表示属性可以不存在。
- `readonly` 表示通过该类型视角不能修改。

### 接口和类型别名

```ts
interface UserDto {
  id: string;
  name: string;
}

type UserId = string;
```

接口适合描述对象形状和可扩展 API；类型别名适合联合、交叉、条件类型、元组等更广泛组合。

### 结构化兼容

```ts
type Named = { name: string };

const user = { id: "u1", name: "Alice" };

function printName(value: Named) {
  console.log(value.name);
}

printName(user);
```

`user` 没有显式声明为 `Named`，但结构满足。

### 多余属性检查

对象字面量直接传参时会更严格：

```ts
function save(user: { id: string }) {}

save({ id: "u1", extra: true }); // 通常报错

const value = { id: "u1", extra: true };
save(value); // 通常允许
```

这是为了捕获拼写错误，同时保留结构化类型的灵活性。

## 例子

```ts
type ApiUser = {
  id: string;
  profile?: {
    nickname?: string;
  };
};

function displayName(user: ApiUser): string {
  return user.profile?.nickname ?? user.id;
}
```

## 练习

1. 为商品、订单、订单项建模。
2. 分别用 interface 和 type 表达一个对象结构。
3. 构造一个对象字面量多余属性检查的例子。
4. 给一个字段加 `readonly`，观察修改时报错。

## 验收

- 能解释结构化类型。
- 能区分接口和类型别名的常见使用场景。
- 能说明可选属性和 `undefined` 的关系。
- 能解释多余属性检查。

## 重点

- TypeScript 看结构，不看声明来源。
- 对象字面量直接赋值和变量间赋值的检查强度不同。
- 类型建模要表达业务不变量，而不是只让代码“能过编译”。

## 难点

- 结构化兼容会让“多字段对象”赋给“少字段类型”看起来过于宽松，但这符合 JS 鸭子类型生态。

## 易错

> **易错：** 用一个包含大量可选字段的类型表达多个业务状态。
>
> 正确做法：状态互斥时优先考虑判别联合，让非法组合无法表示。
