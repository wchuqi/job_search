# TypeScript学习资料：版本迁移和大型项目治理

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 能制定 TypeScript 版本升级和 strict 迁移策略。
- 理解大型项目中的类型性能、声明治理和配置分层。
- 能处理 monorepo、项目引用和公共类型包。
- 能建立团队可持续的类型质量规则。

## 理论导读

TypeScript 在小项目中是语法和类型问题，在大型项目中是治理问题。版本升级会引入更严格的检查或推断变化；第三方依赖类型会影响全局体验；复杂类型可能拖慢编辑器；多个 tsconfig 层叠后，团队成员可能并不知道实际生效配置。

## 核心心智模型

```text
类型质量 = 编译器规则 + 代码习惯 + 依赖声明 + CI 门禁 + 迁移节奏
```

单靠“开启 strict”不能自动得到高质量类型。需要计划、度量和分阶段落地。

## 知识点详解

### strict 迁移策略

1. 先跑 `tsc --noEmit` 获取错误基线。
2. 按目录、包或业务域分批处理。
3. 优先处理公共 API 和高频变更模块。
4. 用 `unknown`、类型保护替换高风险 `any`。
5. 对短期无法修复处使用带原因的 `@ts-expect-error`。
6. CI 阻止新增错误。

### 版本升级策略

- 阅读 release notes。
- 在分支中升级 TypeScript。
- 运行类型检查、测试和构建。
- 关注推断变化、标准库类型变化、模块解析变化。
- 对公共库升级后验证生成的 `.d.ts`。

### 类型性能

复杂条件类型、深层递归类型、巨大的联合类型、过度泛型组合都可能拖慢编辑器和 CI。大型项目要关注类型工具的复杂度，必要时显式落地中间类型。

### 配置分层

```text
tsconfig.base.json
tsconfig.json
tsconfig.build.json
packages/*/tsconfig.json
```

基础配置放通用规则，构建配置控制输出，测试配置包含测试环境类型。

## 例子

```json
{
  "extends": "../../tsconfig.base.json",
  "compilerOptions": {
    "composite": true,
    "declaration": true,
    "outDir": "dist"
  },
  "include": ["src"]
}
```

## 练习

1. 为一个老 JS/TS 混合项目写 strict 迁移计划。
2. 用 `@ts-expect-error` 替代 `@ts-ignore`，并写明原因。
3. 把一个 monorepo 拆成项目引用。
4. 找出一个复杂类型工具，给它补类型测试和文档。

## 验收

- 能设计渐进 strict 迁移。
- 能说明版本升级要验证哪些风险点。
- 能识别类型性能问题的常见来源。
- 能设计 tsconfig 分层和项目引用边界。

## 重点

- 大型项目里 TypeScript 是工程治理的一部分。
- CI 应阻止类型质量倒退。
- 复杂类型要可测试、可解释、可维护。

## 难点

- 类型升级问题经常来自第三方声明变化，不一定来自业务代码。

## 易错

> **易错：** 大规模迁移时用 `// @ts-ignore` 扫平所有错误。
>
> 正确做法：优先修复真实问题；短期抑制使用 `@ts-expect-error` 并写清原因，后续能被编译器发现是否已不再需要。
