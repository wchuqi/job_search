# TypeScript学习资料：条件类型、映射类型和模板字面量类型

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 掌握条件类型、`infer`、映射类型和模板字面量类型。
- 理解内置工具类型的机制。
- 能编写适度的类型工具，避免过度类型体操。
- 能判断类型运算复杂度和可维护性风险。

## 理论导读

TypeScript 的类型系统不仅能标注类型，还能在类型层做计算。条件类型类似类型层面的 `if`，映射类型类似遍历对象键，模板字面量类型能拼接字符串字面量类型。它们构成了 `Pick`、`Partial`、`Record`、`ReturnType` 等工具类型的基础。

这些能力很强，但也容易过度使用。类型工具应服务真实复用和约束，不应把业务逻辑全部搬到类型层。

## 核心心智模型

```text
条件类型：T 满足某约束吗？
映射类型：遍历对象的每个键并变换属性
模板字面量类型：在类型层拼接字符串模式
infer：在条件类型中声明待推断片段
```

## 知识点详解

### 条件类型

```ts
type IsString<T> = T extends string ? true : false;
```

### `infer`

```ts
type UnwrapPromise<T> = T extends Promise<infer R> ? R : T;
```

### 映射类型

```ts
type ReadonlyFields<T> = {
  readonly [K in keyof T]: T[K];
};
```

### 属性修饰符

```ts
type RequiredFields<T> = {
  [K in keyof T]-?: T[K];
};
```

### 模板字面量类型

```ts
type EventName<T extends string> = `${T}Changed`;

type UserEvent = EventName<"name" | "age">;
```

### 内置工具类型

| 工具类型 | 作用 |
| --- | --- |
| `Partial<T>` | 所有属性可选 |
| `Required<T>` | 所有属性必填 |
| `Readonly<T>` | 所有属性只读 |
| `Pick<T, K>` | 选取部分属性 |
| `Omit<T, K>` | 排除部分属性 |
| `Record<K, V>` | 构造键值映射 |
| `ReturnType<F>` | 获取函数返回类型 |
| `Parameters<F>` | 获取函数参数元组 |

## 例子

```ts
type ApiEvents<T> = {
  [K in keyof T as `${string & K}Changed`]: (value: T[K]) => void;
};

type User = {
  name: string;
  age: number;
};

type UserEvents = ApiEvents<User>;
```

## 练习

1. 实现 `Nullable<T>`，让所有字段都能为 `null`。
2. 实现 `ValueOf<T>`，得到对象值类型联合。
3. 实现 `UnwrapPromise<T>`。
4. 用模板字面量类型生成事件名。

## 验收

- 能解释条件类型和映射类型。
- 能写出一个使用 `infer` 的类型。
- 能理解常见工具类型的实现思路。
- 能说出过度类型运算的维护风险。

## 重点

- 类型运算应减少重复和增强约束。
- `infer` 只能出现在条件类型的 `extends` 分支模式中。
- 映射类型配合键重映射可以构造复杂 API 类型。

## 难点

- 条件类型遇到联合类型时通常会分发，很多高级类型行为都来自这一点。

## 易错

> **易错：** 为了追求“全自动类型推导”写出无人能维护的类型工具。
>
> 正确做法：类型工具要有清晰输入、输出、测试样例和文档，复杂到影响可读性时应退回显式类型。
