# TypeScript学习资料：函数、对象、类和接口

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 掌握函数签名、回调、重载和 `this` 类型。
- 理解对象类型、接口扩展和类的类型关系。
- 能区分类的运行时身份和接口的编译期结构。
- 能写出可维护的公共 API 类型。

## 理论导读

TypeScript 中函数是最重要的边界之一。参数类型约束调用方，返回值类型约束实现方和后续使用方。对象、接口和类则用于描述结构和行为。接口没有运行时实体，类既有类型侧也有运行时值侧。

## 核心心智模型

```text
函数类型：调用契约
对象类型：数据形状
接口：可扩展的结构协议
类：运行时构造器 + 实例类型
```

## 知识点详解

### 函数签名

```ts
type Mapper<T, R> = (value: T, index: number) => R;
```

### 可选参数和默认参数

```ts
function greet(name: string, prefix = "hello"): string {
  return `${prefix} ${name}`;
}
```

可选参数必须处理不存在的情况；默认参数通常能推断类型。

### 函数重载

```ts
function parse(value: string): string;
function parse(value: number): number;
function parse(value: string | number): string | number {
  return value;
}
```

重载用于让调用方看到更精确的签名，但实现必须覆盖所有重载。

### 接口扩展

```ts
interface Animal {
  name: string;
}

interface Dog extends Animal {
  bark(): void;
}
```

### 类

```ts
class UserService {
  constructor(private readonly baseUrl: string) {}

  async findUser(id: string): Promise<string> {
    return `${this.baseUrl}/users/${id}`;
  }
}
```

## 例子

```ts
interface Repository<T> {
  findById(id: string): Promise<T | null>;
  save(value: T): Promise<void>;
}
```

这个接口表达持久化能力，不关心实现来自内存、数据库还是 HTTP。

## 练习

1. 为事件监听器写函数类型。
2. 写一个接口 `Cache<K, V>`。
3. 用类实现 `Repository<User>`。
4. 写一个函数重载，并解释实现签名为什么更宽。

## 验收

- 能写函数类型和回调类型。
- 能解释接口和类的关系。
- 能使用访问修饰符和 `readonly`。
- 能判断何时需要函数重载。

## 重点

- 函数边界的类型越清晰，调用错误越早暴露。
- 接口描述结构，类提供运行时实现。
- 重载应服务调用体验，不应替代清晰的数据建模。

## 难点

- 类有值空间和类型空间两面：`typeof MyClass` 表示构造器类型，`MyClass` 表示实例类型。

## 易错

> **易错：** 把接口当成运行时可检查的东西。
>
> 正确做法：运行时检查用实际字段、类实例、schema 或自定义校验函数。
