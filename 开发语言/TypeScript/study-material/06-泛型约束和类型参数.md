# TypeScript学习资料：泛型、约束和类型参数

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 理解泛型解决“类型关系复用”问题。
- 掌握泛型函数、泛型接口、泛型类。
- 能使用 `extends` 约束、默认类型参数和 `keyof` 约束。
- 理解类型参数推断规则和常见失败场景。

## 理论导读

泛型不是“任意类型”。泛型表示一个由调用方决定、由实现方保持关系的类型变量。它的价值不只是避免 `any`，而是把输入和输出之间的类型关系保存下来。

例如 `identity<T>(value: T): T` 表示传入什么类型，就返回同一种类型。`T` 是关系，不是具体类型。

## 核心心智模型

```text
调用点提供具体类型
  -> 编译器推断或显式指定 T
  -> 函数内部只能使用约束允许的能力
  -> 返回值保留 T 的信息
```

## 知识点详解

### 泛型函数

```ts
function identity<T>(value: T): T {
  return value;
}
```

### 泛型约束

```ts
function lengthOf<T extends { length: number }>(value: T): number {
  return value.length;
}
```

约束告诉实现方：你可以安全使用哪些属性。

### `keyof` 约束

```ts
function get<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
```

这里 `K` 必须是 `T` 的键，返回值精确为该键对应的类型。

### 默认类型参数

```ts
type ApiResponse<T = unknown> = {
  data: T;
  requestId: string;
};
```

默认类型参数适合公共类型在简单场景下减少噪声。

## 例子

```ts
function pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K> {
  const result = {} as Pick<T, K>;
  for (const key of keys) {
    result[key] = obj[key];
  }
  return result;
}
```

## 练习

1. 实现 `first<T>(items: T[]): T | undefined`。
2. 实现 `get<T, K extends keyof T>`。
3. 为分页响应 `Page<T>` 建模。
4. 写一个默认类型参数的 API 响应类型。

## 验收

- 能解释泛型保存输入输出关系。
- 能写泛型约束。
- 能使用 `keyof` 约束对象键。
- 能判断什么时候不该滥用泛型。

## 重点

- 泛型的核心是类型关系。
- 泛型内部不能使用未被约束保证的能力。
- 公共工具函数常需要泛型，业务函数不一定需要。

## 难点

- 类型参数推断来自参数位置、上下文类型和默认值；推断失败时可能需要显式类型参数或重构签名。

## 易错

> **易错：** 写 `<T>(value: T) => any`，让泛型关系在返回值处丢失。
>
> 正确做法：尽量让返回类型保留输入类型关系，例如 `T`、`T[K]`、`Promise<T>`。
