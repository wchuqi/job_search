# TypeScript学习资料：异步、错误处理和安全边界

[返回索引](../TypeScript学习资料.md)

## 学习目标

- 掌握 `Promise<T>`、`async/await` 和异步返回类型。
- 理解异常、错误对象和结果类型的建模方式。
- 能用 TypeScript 表达安全边界和权限动作。
- 能避免类型断言掩盖安全问题。

## 理论导读

异步代码的类型不只是 `Promise<T>`。真实工程还要表达失败、超时、取消、重试、权限不足和数据不合法。JavaScript 的异常可以抛出任意值，TypeScript 也不能保证 `catch` 中一定是 `Error`。因此错误处理应有明确策略。

## 核心心智模型

```text
异步调用
  -> 可能成功返回 T
  -> 可能抛错或返回失败结果
  -> 外部输入需要校验
  -> 权限和安全边界不能只靠类型
```

## 知识点详解

### Promise 返回类型

```ts
async function fetchText(): Promise<string> {
  return "ok";
}
```

### 结果类型

```ts
type Result<T, E = string> =
  | { ok: true; value: T }
  | { ok: false; error: E };
```

### `catch` 中的 unknown

```ts
try {
  await fetchText();
} catch (err: unknown) {
  if (err instanceof Error) {
    console.error(err.message);
  }
}
```

### 权限动作建模

```ts
type Action = "read" | "write" | "delete";
type Resource = "user" | "order";

type Permission = `${Resource}:${Action}`;
```

这能限制权限字符串形态，但不能替代真实鉴权。

## 例子

```ts
async function safeJson<T>(
  request: Promise<Response>,
  validate: (value: unknown) => value is T
): Promise<Result<T>> {
  const response = await request;
  const value: unknown = await response.json();
  return validate(value)
    ? { ok: true, value }
    : { ok: false, error: "invalid payload" };
}
```

## 练习

1. 写一个 `Result<T>` 类型。
2. 把一个会抛错的异步函数改成返回 `Result<T>`。
3. 在 `catch` 中按 `unknown` 处理错误。
4. 用模板字面量类型表达权限字符串。

## 验收

- 能写正确的异步返回类型。
- 能解释 `catch` 中为什么应按 `unknown` 处理。
- 能用判别联合表达成功/失败。
- 能说明类型系统和真实鉴权的边界。

## 重点

- 异步错误要在 API 契约中明确。
- `catch` 里不要假设一定是 `Error`。
- 类型能表达权限字符串格式，不能证明请求有权限。

## 难点

- 在复杂系统中，错误可能来自网络、业务、校验、权限、超时和取消，应区分是否可重试、是否可展示、是否需要告警。

## 易错

> **易错：** 用 `as Error` 直接读取 `message`。
>
> 正确做法：先判断 `err instanceof Error`，或统一把错误规范化为内部错误类型。
