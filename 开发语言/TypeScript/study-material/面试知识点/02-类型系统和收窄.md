## 一、类型系统和收窄

### 1. 什么是结构化类型系统？

**参考答案：**

结构化类型系统按对象结构判断兼容性。只要一个值拥有目标类型要求的属性和方法，它就可以被当作该类型使用，不要求显式声明继承或实现。

```ts
type Named = { name: string };
const user = { id: "u1", name: "Alice" };
const named: Named = user;
```

> **重点：** TypeScript 看形状，不看名义身份。
>
> **易错：** 用 Java/C# 名义类型思维理解 TS。

### 2. 多余属性检查为什么有时生效，有时不生效？

**参考答案：**

对象字面量直接赋给目标类型或作为参数传入时，TypeScript 会额外检查多余属性，以捕获拼写错误。对象先赋给变量后再传递时，兼容性按结构判断，只要包含目标字段即可。

> **难点：** 这是“新鲜对象字面量”检查和结构化兼容共同作用的结果。

### 3. 联合类型和交叉类型有什么区别？

**参考答案：**

联合类型 `A | B` 表示值可能是 A 或 B；交叉类型 `A & B` 表示值同时满足 A 和 B。联合常用于多状态输入，交叉常用于组合对象能力。

> **易错：** 以为交叉类型是“取公共部分”。对象交叉更接近“合并要求”。

### 4. 什么是判别联合？为什么适合表达状态？

**参考答案：**

判别联合是每个联合成员都有一个共同的字面量字段，例如 `kind` 或 `type`。编译器可以根据该字段自动收窄到对应成员。

```ts
type State =
  | { kind: "loading" }
  | { kind: "success"; data: string }
  | { kind: "failed"; error: string };
```

它能让非法状态不可表示，例如 `success` 没有 `data` 会报错。

> **重点：** 判别联合比多个可选字段更能表达互斥状态。

### 5. 如何做穷尽检查？

**参考答案：**

常见做法是写一个接收 `never` 的函数，在 `switch` 的默认分支调用。新增联合成员但未处理时，默认分支的值不再是 `never`，编译器会报错。

```ts
function assertNever(value: never): never {
  throw new Error(`unhandled ${value}`);
}
```

> **重点：** 穷尽检查把“漏处理状态”变成编译期错误。
