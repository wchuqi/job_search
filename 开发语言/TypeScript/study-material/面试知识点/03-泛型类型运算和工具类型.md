## 一、泛型、类型运算和工具类型

### 1. 泛型解决什么问题？

**参考答案：**

泛型解决类型关系复用问题。它让函数、接口或类型在调用时保留具体类型信息，而不是退化成 `any`。

```ts
function first<T>(items: T[]): T | undefined {
  return items[0];
}
```

> **重点：** 泛型的价值是保存输入输出之间的类型关系。

### 2. `T extends X` 是什么意思？

**参考答案：**

它表示泛型参数 `T` 必须满足约束 `X`。在函数实现内部，只能使用约束保证存在的能力。

```ts
function len<T extends { length: number }>(value: T): number {
  return value.length;
}
```

> **易错：** 把 `extends` 理解成类继承。这里更多是类型约束。

### 3. `keyof` 和索引访问类型怎么配合？

**参考答案：**

`keyof T` 得到对象类型键的联合，`T[K]` 得到键对应的值类型。二者配合能写出类型安全的属性访问。

```ts
function get<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
```

> **重点：** 返回类型会随 key 精确变化。

### 4. 条件类型和 `infer` 用来做什么？

**参考答案：**

条件类型在类型层做判断，`infer` 可以在匹配某个类型模式时推断其中一部分。

```ts
type UnwrapPromise<T> = T extends Promise<infer R> ? R : T;
```

> **难点：** 条件类型遇到联合类型时通常会分发。

### 5. 映射类型是什么？

**参考答案：**

映射类型遍历对象类型的键，并对每个属性做变换。`Partial`、`Readonly`、`Pick` 等工具类型都基于这个机制。

```ts
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};
```

> **易错：** 为了炫技写过度复杂的类型工具。工程上要优先可读和可测试。
