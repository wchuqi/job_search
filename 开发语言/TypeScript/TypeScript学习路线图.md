# TypeScript学习路线图

## 阶段 1：基础认知

- 目标：理解 TypeScript 是 JavaScript 的类型化超集，类型主要在编译期工作。
- 需要掌握：类型标注、类型推断、`tsc`、`tsconfig.json`、`strict`。
- 例子：给函数参数和返回值加类型，编译后观察输出仍是 JavaScript。
- 练习：把一个普通 JS 工具函数改成 TS，并开启 `noImplicitAny`。
- 验收：能解释“类型会擦除”和“TS 不替代运行时校验”。
- 重点：TypeScript 解决的是开发期建模、协作和重构安全。
- 易错：以为通过 TS 编译就代表外部输入一定安全。

## 阶段 2：类型建模

- 目标：能用类型表达对象结构、函数契约、可选字段、只读字段和业务状态。
- 需要掌握：基础类型、对象类型、数组/元组、接口、类型别名、联合和交叉。
- 例子：用判别联合表达请求状态：`loading | success | failed`。
- 练习：为订单、用户、支付结果建模。
- 验收：能判断什么时候用联合，什么时候用可选字段。
- 难点：结构化类型系统下，“长得像”就可赋值。

## 阶段 3：类型收窄和控制流分析

- 目标：理解 TypeScript 如何根据 `if`、`typeof`、`in`、判别字段和自定义类型保护缩小类型。
- 需要掌握：字面量类型、判别联合、类型保护、`never` 穷尽检查。
- 例子：对 `Result<T>` 做 `switch (result.kind)`。
- 练习：给接口返回值写穷尽分支。
- 验收：能通过 `assertNever` 发现新增状态未处理。
- 易错：过度使用 `as` 绕过收窄。

## 阶段 4：泛型和类型复用

- 目标：能写类型安全的通用函数、容器、工具类型和 API 包装。
- 需要掌握：泛型函数、泛型接口、约束、默认类型参数、条件类型、映射类型。
- 例子：实现 `pick<T, K extends keyof T>(obj, keys)`。
- 练习：实现分页响应、表单字段、事件映射类型。
- 验收：能说明泛型参数如何从调用点推断。
- 难点：泛型不是“任意类型”，而是“调用者决定但受约束的类型变量”。

## 阶段 5：模块、配置和工程化

- 目标：能处理真实项目里的模块解析、声明文件、构建输出和工具链协作。
- 需要掌握：ESM/CJS、`module`、`moduleResolution`、`target`、`lib`、`paths`、项目引用、`.d.ts`。
- 例子：解释为什么编辑器能跳转，但运行时报 `Cannot find module`。
- 练习：搭建一个包含 `src`、`tests`、`types` 的小项目。
- 验收：能定位“类型能过但运行失败”的模块问题。
- 重点：TypeScript 的路径别名只影响编译期，运行期也要配套解析。

## 阶段 6：生产实践和迁移治理

- 目标：把 TypeScript 用在团队协作、CI、包发布和大型仓库中。
- 需要掌握：严格模式渐进迁移、类型覆盖率、API 边界校验、性能诊断、版本升级。
- 例子：从 JS 项目逐步开启 `allowJs`、`checkJs`、`strict`。
- 练习：为一个 API client 增加运行时校验和静态类型。
- 验收：能设计一套从宽松到严格的迁移计划。
- 易错：一次性开启所有严格选项导致迁移无法落地。

## 阶段 7：深度机制和生产排障

- 目标：能从 TypeScript 编译器、类型检查器、模块解析和运行时边界的角度解释复杂问题。
- 需要掌握：scanner/parser/binder/checker/emitter、结构化兼容、控制流分析、泛型推断、条件类型分发、`infer`、模块解析、声明入口、类型性能、排障剧本。
- 例子：解释为什么 `paths` 让编辑器能跳转，但 Node 运行时仍报 `Cannot find module`。
- 练习：用 `--traceResolution` 排查模块问题，用 `--extendedDiagnostics` 分析类型检查耗时。
- 验收：能把一个线上 TypeScript 工程问题拆成类型层、配置层、构建层和运行时层。
- 重点：中高级能力不只是会写高级类型，而是能定位边界、解释机制、制定迁移和治理方案。
- 难点：同一个问题可能跨 tsserver、tsc、Node、打包器、测试框架和第三方声明。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 基础语法、类型标注、tsconfig | 一个能编译运行的小项目 |
| 第 2 周 | 对象建模、联合、收窄、函数类型 | 一组业务类型和工具函数 |
| 第 3 周 | 泛型、类型运算、工具类型 | 类型安全的 API client 或表单模型 |
| 第 4 周 | 模块、声明文件、工程化 | 能发布或被其他项目引用的包 |
| 第 5 周 | 框架实践、运行时边界、测试 | 前端或 Node 小项目 |
| 第 6 周 | 面试复习、迁移治理、排障 | 知识清单、面试答案、迁移方案 |
| 第 7 周 | 编译器、类型推断、模块解析、类型性能 | 深度实验报告和排障手册 |

## 最终能力清单

- 能解释 TypeScript 类型擦除、结构化类型和运行时边界。
- 能用联合、交叉、泛型和类型运算建模真实业务。
- 能处理模块解析、声明文件、路径别名和包发布问题。
- 能为外部输入补运行时校验，而不是只依赖类型。
- 能设计 strict 迁移、版本升级和大型项目治理方案。
- 能解释 TypeScript 编译器主要阶段和类型检查问题层级。
- 能排查模块解析、声明文件、路径别名和 ESM/CJS 互操作问题。
- 能识别复杂类型导致的类型性能问题。
