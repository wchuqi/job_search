# Golang 学习路线图

## 阶段 1：基础认知

- 目标：理解 Go 的定位、工程哲学、工具链和最小项目结构。
- 需要掌握：`go run`、`go test`、`go mod`、包、模块、入口函数、格式化、基础类型、控制流。
- 例子：写一个命令行程序读取参数、校验输入、输出 JSON。
- 练习：创建 `hello-go` 模块，包含 `main.go`、一个业务包和一组单元测试。
- 验收：能解释 `package main`、`func main`、`go.mod`、`go.sum` 的职责。
- 重点：Go 把格式化、测试、依赖管理纳入官方工具链。
- 易错：把 Go 当成“语法更简单的 Java/C++”。Go 的核心不是继承层次，而是组合、接口和显式错误处理。

## 阶段 2：语言核心能力

- 目标：掌握类型系统、函数、方法、接口、错误处理和泛型。
- 需要掌握：值语义、指针、切片底层数组、map 哈希表、方法集、接口隐式实现、`errors.Is`、`errors.As`、类型参数。
- 例子：实现一个带接口抽象的缓存组件，并用泛型封装常用集合操作。
- 练习：写一个 `Repository[T]` 内存实现，支持增删改查、错误包装和单元测试。
- 验收：能说明切片扩容、接口 nil、值接收者和指针接收者的差异。
- 重点：接口描述行为，结构体承载状态，错误值描述失败。
- 难点：方法集、接口动态类型、逃逸分析和泛型约束边界。
- 易错：把 `panic` 当业务错误处理；在接口变量里保存 nil 指针后误判为 nil。

## 阶段 3：并发与系统能力

- 目标：能写出可取消、可限流、无数据竞争的并发程序。
- 需要掌握：goroutine、channel、select、context、WaitGroup、Mutex、atomic、竞态检测、调度器 G/M/P、Go 内存模型。
- 例子：实现一个带并发数限制、超时取消和错误聚合的批量任务执行器。
- 练习：用 `go test -race` 验证并发代码，使用 pprof 找到 goroutine 泄漏。
- 验收：能解释 happens-before、channel 关闭规则、锁保护边界和 goroutine 生命周期。
- 重点：并发不是只会 `go f()`，而是能控制生命周期、背压和共享状态。
- 难点：看似偶发的并发 bug 往往来自数据竞争、取消链断裂或 channel 所有权不清。
- 易错：无限制创建 goroutine；多个发送方关闭同一个 channel；忽略 `context.Context`。

## 阶段 4：工程化与生产实践

- 目标：能构建、测试、发布和排查 Go 服务。
- 需要掌握：模块依赖解析、workspace、构建标签、交叉编译、HTTP 服务、数据库访问、日志、指标、trace、pprof、安全配置、CI。
- 例子：实现一个 REST API，包含配置加载、健康检查、结构化日志、指标、数据库访问和容器镜像。
- 练习：对服务进行基准测试和压测，定位一次 CPU 或内存热点。
- 验收：能说明版本升级策略、依赖替换风险、超时设置、连接池和优雅关闭。
- 重点：生产 Go 程序的质量来自明确边界、可观测性和可靠的测试。
- 难点：性能问题通常需要从分配、逃逸、锁竞争、系统调用和 GC 多角度定位。
- 易错：HTTP 客户端不设置超时；日志泄露 token；把 `replace` 提交到共享模块导致环境漂移。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 工具链、语法、类型、函数 | 命令行小工具 |
| 第 2 周 | 结构体、接口、错误、模块 | 可测试业务包 |
| 第 3 周 | 并发、context、内存模型 | 并发任务执行器 |
| 第 4 周 | HTTP、测试、pprof、发布 | 小型后端服务 |
| 第 5 周 | GC、调度器、生产排障、面试 | 排障报告和面试题复盘 |

## 最终能力清单

- 能从零创建 Go 模块并组织包边界。
- 能设计小而稳定的接口，避免过度抽象。
- 能写出可取消、可测试、无竞态的并发代码。
- 能解释 Go modules 版本选择和依赖替换规则。
- 能用 `go test`、`go test -race`、benchmark、pprof、trace 定位问题。
- 能部署一个带超时、日志、指标、健康检查和优雅关闭的 Go 服务。
