# Golang 学习资料：性能剖析、内存分配和 GC 调优

[返回索引](../Golang学习资料.md)

## 学习目标

- 掌握 benchmark、pprof、trace、逃逸分析和 GC 基本调优。
- 理解分配、锁竞争、系统调用、序列化和 GC 对性能的影响。
- 能用数据驱动优化，而不是凭感觉改代码。

## 理论导读

Go 性能优化的顺序应是测量、定位、解释、修改、复测。CPU 高不一定是算法问题，内存高不一定是泄漏，延迟抖动也可能来自 GC、锁竞争、网络超时或下游阻塞。GC 调优不是第一步，减少无意义分配和控制对象生命周期通常更有效。

## 核心心智模型

性能问题像水管堵塞：pprof 告诉你堵在哪里，benchmark 告诉你改动是否有效，trace 告诉你 goroutine 为什么等待。

## 知识点详解

### 常用命令

```powershell
go test -bench=. -benchmem ./...
go test -run=^$ -bench=BenchmarkX -cpuprofile cpu.out
go tool pprof cpu.out
go build -gcflags=-m ./...
```

### GC 参数

- `GOGC` 控制 GC 目标百分比，值越小越频繁，内存更低但 CPU 成本更高。
- `GOMEMLIMIT` 给 runtime 一个软内存上限，适合容器环境辅助控制内存。

## 练习

对一个 JSON 批量解析函数做 benchmark，记录优化前后的 `ns/op`、`B/op`、`allocs/op`，解释每次优化的来源。

## 验收

- 能读懂 pprof top、list、web。
- 能判断对象是否逃逸。
- 能区分 CPU profile、heap profile、mutex profile、block profile 的用途。

## 重点

- 先定位热点，再优化。
- 减少分配、避免无界缓存、控制并发数比盲目调 GC 更重要。

## 难点

- 微基准与真实流量可能不同，必须结合生产指标和压测。

## 易错

> **易错：** 看到内存上涨就判断为内存泄漏。
>
> 正确做法：区分堆存活对象、缓存、GC 目标堆大小、RSS 和容器统计口径。
