# Golang 学习资料：总览与心智模型

[返回索引](../Golang学习资料.md)

## 学习目标

- 理解 Go 的设计取向：简单语法、快速编译、内置并发、静态类型、官方工具链。
- 建立源码、包、模块、运行时和操作系统之间的整体图景。
- 知道 Go 适合的场景和不适合用它硬套的场景。

## 理论导读

Go 的核心目标不是提供最多语言特性，而是让团队在大型代码库里稳定交付网络服务、命令行工具和基础设施程序。它用静态类型减少运行期错误，用垃圾回收降低内存管理成本，用 goroutine 和 channel 降低并发编程门槛，用 `go` 命令统一构建、测试、格式化、依赖和文档。

可以把一个 Go 项目想成一座工厂：包是车间，导出的标识符是车间对外窗口，`go.mod` 是供应商清单，`go test` 是质量检查线，runtime 是调度工人和回收废料的厂务系统。代码写得好不好，不只看函数能不能运行，还要看包边界是否清楚、错误是否可诊断、并发任务是否能收口、依赖是否可复现。

## 核心心智模型

- 源码层：`.go` 文件按目录组成 package，同一目录通常只属于一个包。
- 模块层：`go.mod` 定义模块路径、Go 版本和依赖要求。
- 编译层：Go 直接编译为机器码，编译器会做逃逸分析、内联和边界检查优化。
- 运行时层：runtime 管理 goroutine 调度、栈增长、GC、map、channel、panic 等机制。
- 操作系统层：网络、文件、进程、线程和系统调用仍由 OS 提供，Go runtime 做封装和调度。

## 知识点详解

### Go 的优势边界

Go 很适合 HTTP/RPC 服务、云原生组件、命令行工具、代理网关、数据管道和并发 IO 程序。它不以极致表达能力为目标，也不追求完全零成本抽象；需要极端手工内存控制的场景通常更偏向 C/C++/Rust。

### Go 的工程约束

Go 强制格式化和显式错误处理，减少风格争论。接口隐式实现让依赖方向更灵活，但也要求你在包边界上认真设计行为。并发语法很轻，但生命周期、取消、背压和共享状态仍然必须严肃处理。

## 练习

创建一个 `hello-go` 模块，包含 `cmd/hello/main.go` 和 `internal/greeting/greeting.go`，让 main 包只负责命令行入口，业务逻辑放到 internal 包。

## 验收

- 能解释 package、module、runtime、standard library 的关系。
- 能说出 Go 与 Java、Python、C++ 在类型、内存和并发模型上的主要差异。
- 能判断一个需求是否适合 Go。

## 重点

- Go 的学习重点是工程化语义，不只是语法。
- runtime 是理解调度、GC、栈、channel、map 和 panic 的关键。

## 难点

- Go 简单语法背后有复杂运行时机制，不能因为代码短就忽略生命周期和资源边界。

## 易错

> **易错：** 认为 goroutine 很轻就可以无限创建。
>
> 正确做法：任何后台任务都要有退出条件、取消信号、并发上限和观测方式。
