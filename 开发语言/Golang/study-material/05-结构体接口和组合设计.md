# Golang 学习资料：结构体、接口和组合设计

[返回索引](../Golang学习资料.md)

## 学习目标

- 掌握结构体、嵌入字段、方法、接口、类型断言和类型 switch。
- 理解隐式接口实现和组合优先的设计方式。
- 能设计小接口和清晰包边界。

## 理论导读

Go 没有类继承，结构体表达数据，方法表达行为，接口表达能力。接口由使用方定义更自然，因为使用方最清楚自己需要哪些行为。小接口让测试替身和模块替换更简单，大接口会把实现者锁死。

## 核心心智模型

接口像插座规格：只规定插头形状，不关心设备内部怎么供电。结构体只要拥有接口要求的方法，就自动满足接口。

## 知识点详解

### 小接口

```go
type UserStore interface {
	FindByID(ctx context.Context, id string) (User, error)
}
```

接口不需要提前放在实现包里。调用方只依赖自己用到的方法，测试时可以替换为 fake。

### 接口 nil

接口值包含动态类型和动态值。只有两者都为空时，接口才等于 nil。

```go
var p *bytes.Buffer = nil
var r io.Reader = p
fmt.Println(r == nil) // false
```

## 练习

为订单服务定义 `OrderRepository` 接口，并提供内存实现和测试 fake。要求服务层不依赖具体数据库。

## 验收

- 能解释隐式接口实现。
- 能说明 `interface{}` 与 `any` 的关系。
- 能判断接口应该放在调用方还是实现方。

## 重点

- “接收接口，返回结构体”是常见设计经验，但不是机械规则。
- 接口越小，越容易测试和演进。

## 难点

- 嵌入字段带来的方法提升容易让 API 表面变大。

## 易错

> **易错：** 为每个结构体都创建一个同名大接口。
>
> 正确做法：在真实需要替换或解耦的边界定义接口，且只包含调用方需要的方法。
