# Golang 学习资料：基础语法、变量、常量和控制流

[返回索引](../Golang学习资料.md)

## 学习目标

- 掌握变量声明、短变量声明、常量、`iota`、if、for、switch、defer。
- 理解零值、作用域、遮蔽和显式类型转换。
- 能写出可读的基础 Go 程序。

## 理论导读

Go 的语法有意克制：没有 while、三元表达式、隐式数字转换和复杂宏系统。它鼓励代码直接表达控制流程。零值是 Go 的重要设计：变量即使没有显式初始化，也有可用的默认值，例如数值为 0、bool 为 false、引用类型为 nil。

## 核心心智模型

变量是一块有类型的存储；短变量声明 `:=` 是“声明新名字并赋值”，不是普通赋值。控制流越简单，错误边界越容易看清。

## 知识点详解

### 变量和常量

```go
var port int = 8080
host := "127.0.0.1"
const timeoutSeconds = 3

const (
	StatusPending = iota
	StatusRunning
	StatusDone
)
```

短变量声明至少要引入一个新变量，否则编译失败。常量在编译期求值，可用于枚举式常量，但不要滥用 `iota` 造成可读性下降。

### 控制流

Go 只有 `for` 一种循环形式：

```go
for i := 0; i < 3; i++ {
	fmt.Println(i)
}

for key, value := range map[string]int{"a": 1} {
	fmt.Println(key, value)
}
```

`switch` 默认不贯穿到下一分支，只有显式 `fallthrough` 才会继续。

## 练习

实现一个命令行参数校验函数：端口必须在 1 到 65535 之间，环境必须是 `dev`、`test` 或 `prod`。

## 验收

- 能解释零值和 nil 的区别。
- 能指出变量遮蔽导致的 bug。
- 能使用 `defer` 做资源释放。

## 重点

- Go 依赖编译器保持语法和类型边界清晰。
- `defer` 按后进先出执行，适合紧跟资源获取之后声明释放动作。

## 难点

- `range` 变量复用、短变量声明遮蔽、defer 参数求值时机经常混在一起。

## 易错

> **易错：** 在内层用 `:=` 误创建新变量，外层错误变量没有被赋值。
>
> 正确做法：需要给已有变量赋值时使用 `=`，提交前关注编译器和 linter 提示。
