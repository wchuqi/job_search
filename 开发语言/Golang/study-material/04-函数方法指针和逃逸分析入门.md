# Golang 学习资料：函数、方法、指针和逃逸分析入门

[返回索引](../Golang学习资料.md)

## 学习目标

- 掌握函数、闭包、多返回值、命名返回值、方法和值/指针接收者。
- 理解指针不是手工内存管理工具，而是共享和修改同一对象的方式。
- 建立逃逸分析的基本直觉。

## 理论导读

Go 函数是一等值，可以作为参数、返回值和字段保存。方法只是带接收者的函数，接收者决定方法属于哪个类型的方法集。指针让函数能修改调用方可见的对象，也减少大对象复制，但并不意味着对象一定在堆上。对象放栈还是堆由编译器逃逸分析决定。

## 核心心智模型

值接收者像“拿一份复印件处理”，指针接收者像“拿原件地址处理”。逃逸分析像编译器问：这个值在函数返回后是否还可能被使用？如果可能，就不能只放在当前栈帧。

## 知识点详解

### 方法接收者

```go
type Counter struct {
	n int
}

func (c *Counter) Inc() {
	c.n++
}

func (c Counter) Value() int {
	return c.n
}
```

需要修改状态或避免复制时用指针接收者；不可变小值可以用值接收者。一个类型的方法接收者风格尽量保持一致。

### 闭包

闭包会捕获外部变量。捕获的是变量本身，不是创建闭包那一刻的值副本。

## 例子

```go
func makeAdder(base int) func(int) int {
	return func(x int) int {
		return base + x
	}
}
```

## 练习

实现一个 `RateLimiter`，提供 `Allow() bool` 方法。先用值接收者写一版，再改成指针接收者，观察行为差异。

## 验收

- 能解释函数多返回值为什么适合错误处理。
- 能说明方法集与接口实现的关系。
- 能用 `go build -gcflags=-m` 观察逃逸提示。

## 重点

- 指针接收者影响方法集，也影响接口实现。
- 逃逸分析是性能优化入口，但不要为躲避堆分配写晦涩代码。

## 难点

- 闭包捕获、循环变量、goroutine 启动组合时容易产生错误结果。

## 易错

> **易错：** 大量使用命名返回值，导致返回路径不清晰。
>
> 正确做法：简单函数可以用命名返回值，复杂函数优先显式返回。
