# Golang 学习资料：泛型、约束和类型参数

[返回索引](../Golang学习资料.md)

## 学习目标

- 掌握类型参数、约束接口、类型集合、近似类型 `~T`。
- 理解泛型适合抽象算法和容器，不适合替代所有接口。
- 能写出简单可读的泛型函数和类型。

## 理论导读

Go 泛型解决的是“同一算法需要支持多种静态类型”的问题，例如集合工具、排序、缓存、数值计算。泛型保留静态类型检查，减少重复代码。但 Go 的泛型设计刻意克制，不鼓励复杂类型体操。业务层如果只是需要行为多态，接口往往更简单。

## 核心心智模型

类型参数像函数参数，只是传入的是类型；约束像门禁规则，规定哪些类型可以进入，以及进入后能使用哪些操作。

## 知识点详解

### 泛型函数

```go
func Map[S ~[]E, E any, R any](s S, f func(E) R) []R {
	out := make([]R, 0, len(s))
	for _, v := range s {
		out = append(out, f(v))
	}
	return out
}
```

`~[]E` 表示底层类型是 `[]E` 的命名切片类型也可以使用。

### 约束

```go
type Ordered interface {
	~int | ~int64 | ~float64 | ~string
}
```

约束接口主要用于类型参数，不应随意作为普通值接口使用。

## 练习

实现 `Set[T comparable]`，支持 `Add`、`Has`、`Delete`、`Len`，并写测试覆盖 `string` 和自定义类型。

## 验收

- 能解释 `any` 和 `comparable`。
- 能说明接口多态与泛型多态的选择差异。
- 能使用 `~` 支持底层类型匹配。

## 重点

- 泛型适合类型安全的重复算法。
- 不要为了“高级”把简单业务逻辑泛型化。

## 难点

- 约束方法集和类型集合同时存在，初学者容易把它们与普通接口混淆。

## 易错

> **易错：** 用 `any` 后在函数内部大量类型断言。
>
> 正确做法：如果需要特定操作，就在约束中表达；如果需要运行期分派，考虑普通接口。
