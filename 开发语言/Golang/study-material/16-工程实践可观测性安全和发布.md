# Golang 学习资料：工程实践、可观测性、安全和发布

[返回索引](../Golang学习资料.md)

## 学习目标

- 掌握 Go 服务的生产基线：配置、日志、指标、trace、健康检查、优雅关闭。
- 理解安全编码、依赖治理、容器发布和 CI。
- 能为 Go 服务设计可维护的工程结构。

## 理论导读

生产程序不是“本地能跑”的程序，而是能在故障、升级、流量波动和依赖异常中被观察、被控制、被回滚的程序。Go 服务通常二进制简单，但这不等于生产治理简单。超时、取消、连接池、日志字段、错误码、指标标签和发布版本都需要明确设计。

## 核心心智模型

服务像一个长期运行的黑盒。可观测性是在盒子上安装仪表盘，安全是给入口和内部数据加边界，发布是让盒子能被稳定替换。

## 知识点详解

### 生产基线

- 所有外部调用必须有超时。
- 入口请求必须有 request id 或 trace id。
- 日志结构化，避免输出 secret。
- 指标包含请求量、错误率、延迟、队列长度和资源使用。
- 服务支持健康检查和优雅关闭。
- 构建产物带版本、提交号和构建时间。

### 安全风险

`os/exec` 必须避免拼接 shell 命令；文件路径要清理和限制根目录；HTTP 请求体要限制大小；TLS 证书校验不能随意关闭。

## 练习

为一个 HTTP 服务加入配置加载、结构化日志、Prometheus 指标、pprof 开关、优雅关闭和 Dockerfile。

## 验收

- 能写出 Go 服务的上线检查清单。
- 能解释日志、指标、trace、pprof 的定位边界。
- 能说明依赖升级和漏洞修复流程。

## 重点

- Go 二进制部署简单，但运行时治理不能省。
- 安全默认值和超时默认值要显式检查。

## 难点

- 指标标签设计不当会造成高基数，反过来拖垮监控系统。

## 易错

> **易错：** 把 token、密码、请求体原文直接打进日志。
>
> 正确做法：字段分级、脱敏输出，生产日志只保留排障必要信息。
