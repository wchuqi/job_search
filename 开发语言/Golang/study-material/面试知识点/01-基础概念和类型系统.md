## 一、基础概念和类型系统

### 1. Go 的主要设计目标是什么？

**参考答案：**

Go 主要面向工程化软件开发，强调简单语法、快速编译、静态类型、内置并发、垃圾回收和统一工具链。它适合网络服务、基础设施、命令行工具和云原生组件。

> **重点：** 回答时要提工程效率和团队协作，不要只说“语法简单”。

### 2. 切片和数组有什么区别？

**参考答案：**

数组是定长值，长度是类型的一部分；切片是对底层数组的窗口，包含指针、长度和容量。切片赋值会复制窗口，不复制底层数组。`append` 可能复用底层数组，也可能扩容分配新数组。

> **易错：** 认为切片赋值后两边完全独立。实际上它们可能共享底层数组。

### 3. map 是否线程安全？

**参考答案：**

普通 map 不是线程安全的。并发读写会导致数据竞争，严重时 runtime 会直接报错。需要使用 `sync.Mutex`、`sync.RWMutex`、`sync.Map` 或通过单 goroutine 串行管理。

> **重点：** 只要存在并发写，同一 map 的读写都需要同步。

### 4. 字符串的 `len` 返回什么？

**参考答案：**

`len(s)` 返回字节数，不是 Unicode 字符数。遍历字符应使用 `for _, r := range s`，此时得到的是 rune。

### 5. defer 的执行顺序和参数求值时机是什么？

**参考答案：**

defer 按后进先出执行。defer 调用的参数在 defer 语句执行时就会求值，而不是函数返回时才求值。

```go
x := 1
defer fmt.Println(x)
x = 2
// 输出 1
```

> **难点：** defer、命名返回值和 recover 组合时要特别小心返回值修改时机。
