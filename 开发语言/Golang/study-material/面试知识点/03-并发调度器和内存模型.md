## 一、并发、调度器和内存模型

### 1. goroutine 和线程有什么区别？

**参考答案：**

goroutine 是 Go runtime 管理的轻量执行单元，初始栈小且可增长，由调度器多路复用到 OS 线程。线程由操作系统调度，创建和切换成本更高。goroutine 轻量但不是免费，仍需要控制数量和生命周期。

### 2. channel 关闭规则是什么？

**参考答案：**

发送到已关闭 channel 会 panic；从已关闭 channel 接收会得到零值和 `ok=false`；关闭 nil channel 会 panic。通常由发送方关闭 channel，多发送方需要额外协调。

> **易错：** 把 close 当取消机制到处使用。通用取消应优先使用 context。

### 3. Go 调度器 G/M/P 是什么？

**参考答案：**

G 是 goroutine，M 是 OS 线程，P 是执行 Go 代码所需的调度资源。`GOMAXPROCS` 控制 P 的数量，也就是同时执行 Go 代码的并行度上限。调度器负责运行队列、抢占、系统调用阻塞处理和网络轮询协作。

### 4. 什么是数据竞争？

**参考答案：**

多个 goroutine 访问同一内存位置，至少一个是写，并且没有同步建立 happens-before 关系，就是数据竞争。可用 `go test -race ./...` 检测。

```go
var n int
go func() { n++ }()
go func() { n++ }()
```

> **重点：** 数据竞争不是概率问题，而是程序语义不可靠。

### 5. Mutex、channel 和 atomic 怎么选？

**参考答案：**

保护共享状态时优先考虑 Mutex；传递所有权、任务和事件时使用 channel；简单计数器或状态位可用 atomic。选择依据是数据所有权和同步语义，而不是哪个看起来更“Go 风格”。

> **难点：** channel 滥用会让控制流更难理解，锁滥用会造成死锁和竞争。
