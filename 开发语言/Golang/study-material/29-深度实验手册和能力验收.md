# Golang 学习资料：深度实验手册和能力验收

[返回索引](../Golang学习资料.md)

## 实验目标

这份实验手册用于把 Go 深度知识从“能背”变成“能证明”。每个实验都要求你写代码、运行命令、记录现象、解释机制，并给出修复或优化结果。

## 实验 1：初始化顺序

### 任务

创建三个包 `a`、`b`、`main`，让 `main` 导入 `a` 和 `b`，`a` 再导入 `b`。每个包包含包级变量初始化和 `init`。

### 验收

- 画出 import 图。
- 记录初始化输出。
- 解释为什么每个包只初始化一次。

## 实验 2：逃逸分析和分配

### 任务

写三组函数：

- 返回局部变量值。
- 返回局部变量指针。
- 把局部变量传给接口函数。

运行：

```powershell
go build -gcflags="-m -m" ./...
go test -bench=. -benchmem ./...
```

### 验收

- 标出哪些对象逃逸。
- 解释逃逸是否导致实际性能问题。

## 实验 3：切片保留大数组

### 任务

分配 100MB 字节切片，只保留前 10 字节，比较复制前后 heap profile。

### 验收

- 能在 profile 中看到大对象是否仍存活。
- 能解释 slice header 为什么保留底层数组。

## 实验 4：并发取消流水线

### 任务

实现生产者、worker、聚合器三阶段流水线。任何 worker 出错时取消全链路。

### 验收

- `go test -race ./...` 通过。
- 错误发生后 goroutine 数能回落。
- 所有发送点都监听 context。

## 实验 5：HTTP 超时矩阵

### 任务

写一个慢服务端，分别模拟慢连接、慢 header、慢 body 和慢响应。客户端分别配置 `Client.Timeout`、`ResponseHeaderTimeout` 和 context deadline。

### 验收

- 记录每种超时触发阶段。
- 解释哪个配置覆盖哪个阶段。

## 实验 6：GC 和 GOGC

### 任务

写一个持续分配对象的程序，分别设置：

```powershell
$env:GODEBUG="gctrace=1"
$env:GOGC="50"
$env:GOGC="300"
```

### 验收

- 比较 GC 频率、堆大小和 CPU 消耗。
- 解释为什么降低 GOGC 不等于解决泄漏。

## 实验 7：pprof 排障

### 任务

构造三个接口：

- CPU 热点。
- 内存增长。
- goroutine 阻塞。

分别抓 CPU、heap、goroutine profile。

### 验收

- 给出每个问题的 profile 证据。
- 写出修复方案和复测结果。

## 最终能力验收

你应该能完成以下任务：

- 从源码到二进制解释 Go 构建和启动流程。
- 画出 slice、map、interface 的底层结构。
- 用 escape analysis 和 benchmark 分析一次分配优化。
- 写出可取消、无竞态、无 goroutine 泄漏的并发流水线。
- 用 pprof 和 trace 定位 CPU、内存、阻塞或延迟问题。
- 为 HTTP 服务配置超时、重试、连接池和优雅关闭。
- 解释 Go modules MVS、v2 路径和私有依赖配置。

## 面试自测

1. Go 程序为什么不是直接从 `main.main` 开始执行？
2. `append` 后原切片和新切片什么时候共享底层数组？
3. 为什么接口里保存 nil 指针后接口不等于 nil？
4. `GOMAXPROCS` 控制什么，不控制什么？
5. channel close 和 context cancel 的语义区别是什么？
6. Race Detector 为什么不能证明业务状态机正确？
7. `GOGC` 调大和调小分别有什么代价？
8. HTTP response body 不关闭会造成什么后果？
9. MVS 为什么不会自动选择最新版本？
10. 线上 goroutine 数上涨，你如何定位？
