# Rust学习资料：集合、字符串、迭代器和闭包

[返回索引](../Rust学习资料.md)

## 学习目标

- 掌握 `Vec`、`HashMap`、`HashSet`、`String`、`&str` 的常见用法。
- 理解迭代器的惰性、适配器和消费器。
- 掌握闭包捕获、`Fn`、`FnMut`、`FnOnce` 的基本差异。

## 理论导读

Rust 的集合和迭代器设计大量体现所有权模型：遍历时到底是消费集合、借用元素，还是可变借用元素，会直接决定后续能否继续使用原集合。字符串处理也必须区分字节、Unicode 标量值和用户感知字符。

## 核心心智模型

迭代器像一条传送带。`map`、`filter` 是在传送带上安装加工工位，只有遇到 `collect`、`sum`、`for` 这类消费器时，传送带才真正启动。

## 知识点详解

### Vec 和 HashMap

```rust
use std::collections::HashMap;

let mut nums = vec![1, 2, 3];
nums.push(4);

let mut scores = HashMap::new();
scores.insert("alice", 90);
scores.entry("bob").or_insert(80);
```

### String 和 &str

`String` 拥有可增长的 UTF-8 字节缓冲区，`&str` 是对 UTF-8 字符串数据的借用视图。字符串索引不能直接用 `s[0]`，因为字节偏移不一定落在字符边界。

### 迭代器所有权

```rust
let values = vec![1, 2, 3];
let sum: i32 = values.iter().map(|x| x * 2).sum();
println!("{sum}");
println!("{}", values.len());
```

`iter()` 借用元素，`into_iter()` 消费集合，`iter_mut()` 可变借用元素。

### 闭包捕获

闭包会根据使用方式自动按不可变借用、可变借用或 move 捕获环境。需要跨线程或延长生命周期时常用 `move`。

```rust
let prefix = String::from("id");
let build = move |n: u32| format!("{prefix}-{n}");
println!("{}", build(7));
```

## 例子

```rust
use std::collections::HashMap;

fn word_count(input: &str) -> HashMap<&str, usize> {
    let mut counts = HashMap::new();
    for word in input.split_whitespace() {
        *counts.entry(word).or_insert(0) += 1;
    }
    counts
}
```

## 练习

1. 实现单词统计，并按出现次数排序输出。
2. 分别用 `iter`、`iter_mut`、`into_iter` 操作同一个 Vec，记录所有权变化。
3. 用闭包实现一个过滤器工厂。

## 验收

- 能判断遍历集合时是否会消费集合。
- 能解释为什么 Rust 字符串不能按整数直接索引。
- 能写出常见迭代器链并知道何时真正执行。

## 重点

- 迭代器是 Rust 高性能抽象的重要入口，通常可读性和性能都好。

## 难点

- 闭包捕获方式和 `Fn`/`FnMut`/`FnOnce` 会影响 API 设计和并发场景。

## 易错

> **易错：** 认为 `map` 调用后一定立即执行。
>
> 正确做法：记住迭代器适配器是惰性的，必须由消费器驱动。
