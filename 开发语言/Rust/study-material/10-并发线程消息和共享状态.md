# Rust学习资料：并发、线程、消息和共享状态

[返回索引](../Rust学习资料.md)

## 学习目标

- 掌握线程创建、join、channel、共享状态。
- 理解 `Send`、`Sync`、`Arc`、`Mutex`、`RwLock`。
- 能识别死锁、锁粒度、阻塞和数据竞争边界。

## 理论导读

Rust 的并发安全不是没有共享状态，而是共享状态必须通过类型系统表达清楚。能跨线程移动的类型需要满足 `Send`，能被多线程共享引用的类型需要满足 `Sync`。这些 marker trait 让编译器能提前阻止很多并发错误。

## 核心心智模型

跨线程传值像物流运输：`Send` 表示货物可以安全运到另一个仓库，`Sync` 表示多个仓库可以安全看同一份货物，`Mutex` 表示每次只能一个人拿钥匙进库房。

## 知识点详解

### 线程

```rust
use std::thread;

let handle = thread::spawn(|| {
    1 + 2
});

let result = handle.join().unwrap();
println!("{result}");
```

### Channel

```rust
use std::{sync::mpsc, thread};

let (tx, rx) = mpsc::channel();
thread::spawn(move || {
    tx.send("done").unwrap();
});
println!("{}", rx.recv().unwrap());
```

消息传递适合任务分发、流水线和降低共享状态复杂度。

### 共享状态

```rust
use std::sync::{Arc, Mutex};
use std::thread;

let counter = Arc::new(Mutex::new(0));
let mut handles = vec![];

for _ in 0..4 {
    let counter = Arc::clone(&counter);
    handles.push(thread::spawn(move || {
        let mut guard = counter.lock().unwrap();
        *guard += 1;
    }));
}

for handle in handles {
    handle.join().unwrap();
}
```

锁 guard 离开作用域时自动释放锁。实际项目要缩短持锁时间，避免在锁内做 IO 或调用外部回调。

## 练习

1. 用 channel 实现 worker 汇报结果。
2. 用 `Arc<Mutex<HashMap<_, _>>>` 实现共享计数器。
3. 故意制造锁顺序相反的死锁，并写出预防规则。

## 验收

- 能解释 `Rc` 为什么不是 `Send`。
- 能解释 `Arc<Mutex<T>>` 的两层含义。
- 能说清楚 channel 和共享状态各自适合什么。

## 重点

- Rust 能防数据竞争，不等于能自动防逻辑死锁和性能瓶颈。

## 难点

- `Send`/`Sync` 常在泛型库、异步 runtime 和 trait object 中以错误信息形式出现，需要能反推出哪个字段不满足约束。

## 易错

> **易错：** 在锁保护范围内执行慢 IO、网络请求或用户回调。
>
> 正确做法：只在锁内读写必要共享数据，尽快释放锁。
