## 一、基础语法和类型系统

### 1. Rust 的变量默认为什么不可变？

**参考答案：**

默认不可变让状态变化必须显式出现，降低误修改和并发共享风险。需要修改时使用 `mut`，需要转换时可以用 shadowing 创建新的绑定。

```rust
let input = "42";
let input: u32 = input.parse().unwrap();
```

> **重点：** 不可变默认是工程约束，不是语法负担。
> **易错：** 把 shadowing 当成原地修改；它是新绑定。

### 2. `String` 和 `&str` 有什么区别？

**参考答案：**

`String` 拥有堆上 UTF-8 字节缓冲区，可增长、可修改；`&str` 是对 UTF-8 字符串数据的借用视图，通常不可变。函数只读字符串时优先接收 `&str`，调用方可以传 `String` 或字符串字面量。

```rust
fn greet(name: &str) {
    println!("hello {name}");
}
```

> **难点：** `&String` 可以自动解引用为 `&str`，但 API 设计上 `&str` 更通用。

### 3. `match` 和普通 switch 有什么不同？

**参考答案：**

`match` 必须穷尽所有可能，能解构 enum、tuple、struct，也能绑定变量和加 guard。它是表达式，可以返回值。

```rust
let text = match Some(1) {
    Some(n) if n > 0 => "positive",
    Some(_) => "other",
    None => "none",
};
```

> **重点：** 穷尽性检查让新增枚举状态时编译器能提醒所有处理位置。

### 4. Rust 为什么不能直接用 `s[0]` 访问字符串字符？

**参考答案：**

Rust 字符串是 UTF-8，整数索引代表字节偏移，但一个字符可能占多个字节。直接索引容易切到非法字符边界，因此 Rust 要求显式使用 bytes、chars 或切片边界。

> **易错：** 把字节长度、Unicode 标量值数量和用户感知字符数量混为一谈。
