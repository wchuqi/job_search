## 一、所有权借用和生命周期

### 1. 所有权规则是什么？

**参考答案：**

每个值同一时刻有一个所有者；所有者离开作用域时值被释放；赋值、传参或返回可能发生 move。这个模型让 Rust 在没有 GC 的情况下避免重复释放和悬垂引用。

```rust
let a = String::from("hi");
let b = a;
// a 不再可用
println!("{b}");
```

> **重点：** 所有权负责资源释放责任。

### 2. Copy 和 Clone 有什么区别？

**参考答案：**

`Copy` 是隐式按位复制，适合简单栈值；`Clone` 是显式复制，可能分配内存或执行复杂逻辑。实现了 `Copy` 的类型赋值后旧绑定仍可用。

```rust
let x = 1;
let y = x;
println!("{x} {y}");

let s1 = String::from("a");
let s2 = s1.clone();
```

> **易错：** 为了绕过所有权问题盲目 clone，可能隐藏性能和设计问题。

### 3. 生命周期标注会延长引用寿命吗？

**参考答案：**

不会。生命周期标注只描述输入引用和输出引用之间的有效性关系，让编译器确认返回引用不会超过被引用数据。

```rust
fn longest<'a>(a: &'a str, b: &'a str) -> &'a str {
    if a.len() > b.len() { a } else { b }
}
```

> **难点：** `'a` 表示约束，不是分配或续命机制。

### 4. 什么时候函数参数用 `T`、`&T`、`&mut T`？

**参考答案：**

需要取得所有权并保存或消费时用 `T`；只读时用 `&T` 或更通用的切片类型；需要原地修改时用 `&mut T`。公共 API 还可用 `impl AsRef<str>`、`impl Into<String>` 提升调用灵活性。

> **重点：** 参数类型表达所有权意图。
