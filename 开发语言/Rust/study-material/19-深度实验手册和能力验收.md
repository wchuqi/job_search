# Rust学习资料：深度实验手册和能力验收

[返回索引](../Rust学习资料.md)

## 实验 1：借用检查器错误拆解

写出三个会失败的程序：

- move 后继续使用。
- 不可变借用存在时尝试可变借用。
- 返回指向局部变量的引用。

要求记录编译器错误，并写出两种修复方式：调整所有权或调整 API。

> **验收：** 能解释每个错误对应的真实内存风险。

## 实验 2：生命周期关系建模

实现一个从两个字符串切片中返回较长者的函数，再实现一个包含引用字段的结构体。

> **验收：** 能说明生命周期标注只描述关系，不延长任何值的生命周期。

## 实验 3：Cargo 依赖解析

创建 workspace，设置 `edition = "2024"`，添加多个依赖和 feature。使用：

```powershell
cargo tree
cargo tree -e features
cargo metadata
```

> **验收：** 能画出依赖图，能解释 feature 合并和 resolver 3 的影响。

## 实验 4：线程并发和死锁

用 `Arc<Mutex<T>>` 写一个共享计数器，再故意制造两个锁顺序相反的死锁。

> **验收：** 能给出锁顺序规范、持锁时间限制和排障方法。

## 实验 5：Tokio 异步和阻塞边界

写一个异步服务，在 handler 中故意执行阻塞 sleep，再改成 `tokio::time::sleep` 或 `spawn_blocking`。

> **验收：** 能用日志或压测说明阻塞对并发延迟的影响。

## 实验 6：unsafe 封装审查

封装一个裸指针和长度到 slice 的函数：

```rust
/// # Safety
///
/// caller must ensure pointer is valid for `len` elements,
/// properly aligned, and not mutated while the returned slice exists.
unsafe fn view<'a>(ptr: *const u8, len: usize) -> &'a [u8] {
    std::slice::from_raw_parts(ptr, len)
}
```

> **验收：** 能指出这个函数为什么 unsafe，以及安全 API 应如何约束调用方。

## 实验 7：性能剖析

实现两个版本的日志解析器：一个频繁分配字符串，一个使用切片和迭代器。用 criterion 和 flamegraph 比较。

> **验收：** 能用数据说明瓶颈来自分配、解析、IO 还是算法。

## 最终能力验收

- 能 30 分钟内创建一个带测试的 Rust workspace。
- 能解释所有权、借用、生命周期、Trait、Result、Send/Sync、Future。
- 能读懂常见编译器错误并给出设计级修复。
- 能写一个小型异步服务并加上 tracing。
- 能审查一个 unsafe 代码块并写出不变量。
- 能说明 Rust 项目发布前的质量、安全和可观测性检查项。
