# Rust学习资料：总览与心智模型

[返回索引](../Rust学习资料.md)

## 学习目标

- 理解 Rust 的定位：系统编程性能、内存安全、并发安全、可维护工程化。
- 建立 Rust 最核心的三张图：所有权图、类型约束图、错误边界图。
- 知道 Rust 适合和不适合的场景，避免只因为“性能高”而盲目引入。

## 理论导读

Rust 试图解决 C/C++ 常见的内存安全问题，同时保留接近底层的控制力。它不依赖 GC 统一回收对象，而是在编译期通过所有权、借用和生命周期检查多数内存访问错误。你可以把 Rust 程序想成一座仓库：每个值都有明确保管人，借出货物必须登记借期，修改货物时不能同时把它借给别人查看。

Rust 的“难”主要来自编译期约束提前暴露设计问题。其他语言中很多问题会在运行时变成空指针、数据竞争、悬垂引用、资源泄漏；Rust 会在编译期让你重新表达数据归属、共享方式和错误边界。

## 核心心智模型

| 模型 | 含义 | 典型问题 |
| --- | --- | --- |
| 值的所有权 | 一个值同一时刻有一个所有者，所有者离开作用域时释放值 | move 后继续使用、过度 clone |
| 借用规则 | 可以有多个不可变借用，或一个可变借用 | 读写混用、引用超过被引用值 |
| 类型即契约 | 函数签名说明输入输出、错误和约束 | API 没表达失败、泛型约束过宽 |
| 零成本抽象 | 高级抽象尽量编译成低层代码，无额外运行时成本 | 误以为所有抽象都免费 |
| 显式错误 | 可恢复错误用 Result，不可恢复错误才 panic | unwrap 滥用、错误上下文丢失 |

## 知识点详解

### Rust 适合什么

Rust 适合需要性能、可靠性和资源控制的场景，例如 CLI 工具、网络服务、存储系统、嵌入式、WebAssembly、数据库组件、代理网关、安全敏感工具和高性能库。

### Rust 不一定适合什么

如果团队没有系统编程经验、需求快速变化、生态库不成熟，或者主要工作是简单业务 CRUD，Rust 的学习成本可能压过收益。Rust 不是“更快的 Python”或“更安全的 Java”，它更像一门把资源管理显式建模的工程语言。

### 版本和 Edition

Rust 的 Edition 是语法和语言体验的兼容边界，不等同于编译器版本。Rust 2024 Edition 让新项目默认使用 `edition = "2024"`，Cargo resolver 3 会按 MSRV 兼容性参与依赖选择。版本升级时要分别看 rustc、edition、依赖 MSRV 和 CI 基线。

## 例子

```rust
fn main() {
    let name = String::from("rust");
    print_name(&name);
    println!("{}", name);
}

fn print_name(value: &str) {
    println!("hello {value}");
}
```

这里 `String` 的所有权仍在 `main`，`print_name` 只是借用字符串切片，因此调用后还能继续使用 `name`。

## 练习

1. 写出 Rust、C++、Go、Java 在内存管理上的差异表。
2. 用自己的话解释所有权、借用、生命周期分别解决什么问题。
3. 安装 Rust，创建一个 `hello-rust` 项目并运行测试。

## 验收

- 能解释 Rust 为什么不需要 GC 也能自动释放大多数资源。
- 能区分 Edition、rustc 版本、crate 版本和 MSRV。
- 能说明 Rust 的主要收益和引入成本。

## 重点

- Rust 把资源生命周期和共享方式放进类型系统。
- 学 Rust 要先接受“编译器是设计评审者”的心智模型。

## 难点

- Rust 的报错往往不是局部语法错误，而是数据归属、生命周期或抽象边界设计错误。

## 易错

> **易错：** 把 Rust 理解成“语法奇怪的 C++”。
>
> 正确做法：从所有权、借用、类型系统和错误处理开始理解，而不是从指针和类继承开始套模型。
