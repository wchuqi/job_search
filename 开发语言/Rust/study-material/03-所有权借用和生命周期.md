# Rust学习资料：所有权借用和生命周期

[返回索引](../Rust学习资料.md)

## 学习目标

- 掌握所有权、move、Copy、Clone、Drop。
- 掌握不可变借用、可变借用和借用规则。
- 理解生命周期标注表达的是引用关系约束，而不是延长对象寿命。

## 理论导读

所有权是 Rust 内存安全的中心机制。每个值有一个所有者，所有者离开作用域时值被释放。把值传给函数、赋给新变量或放进容器时，可能发生 move；如果只想临时访问值，就使用引用借用。

生命周期是编译器用于证明“引用不会活得比被引用值更久”的约束系统。多数生命周期可以由编译器推断；只有当函数签名中引用关系不够明确时，才需要显式标注。

## 核心心智模型

所有权像房产证，借用像临时钥匙。不可变钥匙可以发很多把，大家只能看；可变钥匙同一时间只能有一把，因为有人装修时不能让别人同时拿旧图纸读取。

## 知识点详解

### Move

```rust
let a = String::from("hello");
let b = a;
// println!("{a}"); // 编译失败，a 已被 move
println!("{b}");
```

`String` 管理堆内存，move 后所有权转移，旧绑定失效。这样可以避免两个所有者同时释放同一块内存。

### Copy 和 Clone

实现 `Copy` 的类型在赋值时按位复制，旧绑定仍可用，例如整数、布尔、部分简单组合。`Clone` 是显式复制，可能分配内存。

```rust
let a = 1;
let b = a;
println!("{a} {b}");

let s1 = String::from("hi");
let s2 = s1.clone();
println!("{s1} {s2}");
```

### 借用规则

```rust
fn len(s: &String) -> usize {
    s.len()
}

fn push_bang(s: &mut String) {
    s.push('!');
}
```

同一作用域中可以有多个 `&T`，或者一个 `&mut T`。这个规则阻止数据竞争和迭代过程中修改容器等危险行为。

### 生命周期

```rust
fn longest<'a>(left: &'a str, right: &'a str) -> &'a str {
    if left.len() >= right.len() { left } else { right }
}
```

`'a` 表示返回引用的有效期不能超过 `left` 和 `right` 中较短的那个。它不是让两个参数活得更久，而是告诉调用方返回值依赖输入引用。

## 例子

```rust
struct User {
    name: String,
}

fn rename(user: &mut User, name: String) {
    user.name = name;
}

fn main() {
    let mut user = User { name: "alice".to_string() };
    rename(&mut user, "bob".to_string());
    println!("{}", user.name);
}
```

## 练习

1. 故意写一个 move 后继续使用的例子，阅读编译器提示。
2. 把一个函数从接收 `String` 改为接收 `&str`，比较调用端变化。
3. 写一个返回较长字符串切片的函数，并解释生命周期标注。

## 验收

- 能解释为什么 `String` 赋值会 move，而 `i32` 不会。
- 能判断函数参数应该用 `T`、`&T`、`&mut T` 还是 `impl Into<T>`。
- 能看懂常见的 borrow checker 错误。

## 重点

- 所有权决定资源释放责任，借用决定临时访问权限，生命周期证明引用有效性。

## 难点

- 生命周期标注描述关系，不创建或延长生命周期。

## 易错

> **易错：** 遇到借用错误就到处加 `.clone()`。
>
> 正确做法：先判断数据到底应该归谁、函数是否只需要读、是否需要可变访问，再决定 clone 是否是业务上真正需要的复制。
