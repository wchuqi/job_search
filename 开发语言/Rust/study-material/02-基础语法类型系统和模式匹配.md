# Rust学习资料：基础语法类型系统和模式匹配

[返回索引](../Rust学习资料.md)

## 学习目标

- 掌握变量绑定、可变性、表达式、函数、控制流、基础类型。
- 理解 Rust 的静态强类型、类型推断和显式转换。
- 掌握 `match`、`if let`、`while let` 和解构。

## 理论导读

Rust 中大多数语句和块都是表达式，表达式可以产生值。变量默认不可变，这不是限制，而是让数据变化变得显式。类型系统帮助编译器在运行前确认值的形状、可用操作和错误边界。

## 核心心智模型

把 Rust 类型系统想成一套门禁卡。每个值拿着自己的卡，函数、操作符和方法只接受符合权限的卡。类型推断能帮你少写卡号，但不会让不匹配的卡混进去。

## 知识点详解

### 变量和可变性

```rust
let x = 1;
let mut y = 2;
y += 1;
let x = x + 10; // shadowing，创建新绑定
```

`mut` 表示这个绑定可以重新赋值。shadowing 是创建新的同名绑定，常用于类型转换或逐步加工。

### 基础类型

| 类型 | 说明 |
| --- | --- |
| `i32`、`u64` | 有符号和无符号整数 |
| `f32`、`f64` | 浮点数 |
| `bool` | 布尔值 |
| `char` | Unicode 标量值，不等于一个字节 |
| tuple | 固定长度异构组合 |
| array | 固定长度同类型数组 |
| slice | 对连续序列的视图 |

Rust 不做隐式数值转换，`u32` 和 `usize` 不能随意混用，需要显式 `as` 或 `try_into()`。

### 控制流是表达式

```rust
let score = 82;
let level = if score >= 60 { "pass" } else { "fail" };
```

`if` 分支返回类型必须一致，因为整个 `if` 表达式要产生一个确定类型。

### 模式匹配

```rust
fn describe(value: Option<i32>) -> &'static str {
    match value {
        Some(n) if n > 0 => "positive",
        Some(0) => "zero",
        Some(_) => "negative",
        None => "missing",
    }
}
```

`match` 必须穷尽所有可能。模式匹配不是简单 switch，它能解构、绑定变量、加守卫条件。

## 例子

```rust
fn parse_port(input: &str) -> u16 {
    match input.parse::<u16>() {
        Ok(port @ 1..=65535) => port,
        _ => 8080,
    }
}

fn main() {
    println!("{}", parse_port("3000"));
}
```

## 练习

1. 写一个函数，把分数转换成 A/B/C/D/F。
2. 用 `match` 解析命令：`add <name>`、`remove <id>`、`list`。
3. 用 tuple 和 struct 分别表示二维坐标，比较可读性。

## 验收

- 能解释 `let`、`let mut`、shadowing 的区别。
- 能写出穷尽的 `match`。
- 能说明 `String`、`&str`、`char`、`u8` 的区别。

## 重点

- Rust 的 `match` 是类型系统和控制流的核心工具。
- 默认不可变让数据变化变成显式决策。

## 难点

- `&str` 是字符串切片视图，`String` 是拥有堆内存的字符串；后续所有权章节会反复用到。

## 易错

> **易错：** 认为 `char` 就是一个字节。
>
> 正确做法：Rust 的 `char` 是 Unicode 标量值；处理文本长度、字节偏移和字符边界时要区分 bytes、chars 和 grapheme clusters。
