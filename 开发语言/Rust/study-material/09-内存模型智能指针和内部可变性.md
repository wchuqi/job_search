# Rust学习资料：内存模型、智能指针和内部可变性

[返回索引](../Rust学习资料.md)

## 学习目标

- 理解栈、堆、所有权释放和 Drop。
- 掌握 `Box`、`Rc`、`Arc`、`RefCell`、`Mutex` 的适用边界。
- 理解内部可变性和运行时借用检查。

## 理论导读

Rust 默认在编译期检查借用规则，但有些数据结构无法用静态规则直接表达，例如图结构、共享缓存、回调注册表。智能指针和内部可变性提供了受控逃生口：它们不会取消安全性，而是把部分约束转移到类型、引用计数或运行时检查中。

## 核心心智模型

智能指针像不同合同的托管协议。`Box` 是独占托管，`Rc` 是单线程多人共有，`Arc` 是多线程多人共有，`RefCell` 是运行时登记借用，`Mutex` 是加锁后才能修改。

## 知识点详解

### Box

`Box<T>` 把值放到堆上，并保持唯一所有权。常用于递归类型、大对象转移和 Trait object。

```rust
enum List {
    Cons(i32, Box<List>),
    Nil,
}
```

### Rc 和 Arc

`Rc<T>` 用于单线程引用计数共享，`Arc<T>` 用于多线程原子引用计数共享。二者默认只提供共享读，不允许直接可变修改内部值。

```rust
use std::sync::Arc;

let shared = Arc::new(String::from("config"));
let cloned = Arc::clone(&shared);
```

### RefCell

`RefCell<T>` 把借用检查从编译期移到运行时，违反规则会 panic。它适合单线程内部可变性，不适合跨线程共享。

```rust
use std::cell::RefCell;

let values = RefCell::new(vec![1]);
values.borrow_mut().push(2);
```

### Drop

`Drop` 定义值离开作用域时的清理逻辑。文件句柄、锁、网络连接和临时目录都依赖 RAII 风格释放。

## 例子

```rust
use std::{cell::RefCell, rc::Rc};

#[derive(Default)]
struct EventBus {
    handlers: RefCell<Vec<Rc<dyn Fn(&str)>>>,
}

impl EventBus {
    fn subscribe(&self, handler: Rc<dyn Fn(&str)>) {
        self.handlers.borrow_mut().push(handler);
    }
}
```

## 练习

1. 用 `Box` 实现递归链表。
2. 用 `Rc<RefCell<T>>` 实现一个简单树节点，并说明风险。
3. 把单线程共享结构改成 `Arc<Mutex<T>>`。

## 验收

- 能解释 `Rc` 和 `Arc` 为什么不同。
- 能说明 `RefCell` 的 panic 条件。
- 能识别引用循环导致的内存泄漏风险。

## 重点

- 智能指针改变所有权和共享模型，但不会让数据竞争变安全。

## 难点

- `Rc<RefCell<T>>` 很方便，但容易隐藏设计复杂度；复杂系统要谨慎使用。

## 易错

> **易错：** 认为 Rust 没有内存泄漏。
>
> 正确做法：Rust 防止多数悬垂引用和重复释放，但引用循环、故意 `mem::forget`、泄漏全局缓存仍可能造成泄漏。
