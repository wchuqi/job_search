# Rust学习资料：测试、文档、调试和质量工具

[返回索引](../Rust学习资料.md)

## 学习目标

- 掌握单元测试、集成测试、文档测试。
- 掌握 rustfmt、clippy、cargo check、cargo test 的日常流程。
- 理解调试、日志、错误上下文和 CI 基线。

## 理论导读

Rust 的质量工具和 Cargo 深度集成，测试不是独立步骤，而是 crate API 设计的一部分。文档测试尤其重要：公开 API 示例如果能被编译和执行，就能避免文档腐烂。

## 核心心智模型

把测试分成三层：模块内测试验证私有细节，集成测试从外部调用公开 API，文档测试保证用户复制示例能运行。

## 知识点详解

### 单元测试

```rust
pub fn add(left: i32, right: i32) -> i32 {
    left + right
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn add_two_numbers() {
        assert_eq!(add(1, 2), 3);
    }
}
```

### 集成测试

`tests/*.rs` 会作为独立 crate 编译，只能访问公开 API。这能提醒你 API 是否真的好用。

### 文档测试

````rust
/// Adds two numbers.
///
/// ```
/// assert_eq!(my_crate::add(1, 2), 3);
/// ```
pub fn add(left: i32, right: i32) -> i32 {
    left + right
}
````

### 质量命令

```powershell
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
cargo doc --no-deps
```

## 例子

```rust
#[derive(Debug, PartialEq, Eq)]
pub enum ParseError {
    Empty,
    Invalid,
}

pub fn parse_id(input: &str) -> Result<u64, ParseError> {
    if input.trim().is_empty() {
        return Err(ParseError::Empty);
    }
    input.parse().map_err(|_| ParseError::Invalid)
}
```

测试应覆盖成功、空输入、非法输入和边界值。

## 练习

1. 为一个解析函数写单元测试和集成测试。
2. 为公开函数写文档测试。
3. 配置 CI：fmt、clippy、test 三步必须通过。

## 验收

- 能解释 `#[cfg(test)]` 的作用。
- 能区分单元测试和集成测试的访问范围。
- 能把 clippy 警告当作代码质量反馈，而不是机械关闭。

## 重点

- Rust 项目质量基线至少包括 fmt、clippy、test。

## 难点

- 文档测试会以外部使用者视角编译，能暴露 API 可见性和依赖导出问题。

## 易错

> **易错：** 只写 happy path 测试。
>
> 正确做法：Rust 的错误类型很显式，测试也应覆盖错误分支、边界值和不变量。
