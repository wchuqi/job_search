# Rust学习资料：宏、派生宏和过程宏

[返回索引](../Rust学习资料.md)

## 学习目标

- 理解声明宏 `macro_rules!` 的基本匹配。
- 理解 derive 宏、属性宏、函数式过程宏的用途。
- 知道宏的适用边界和调试方法。

## 理论导读

宏是 Rust 的编译期代码生成能力。声明宏适合重复语法模板，过程宏适合解析 TokenStream 并生成更复杂代码。宏能提升表达力，但也会降低可读性和错误定位清晰度。

## 核心心智模型

宏像编译前的代码模具。你给它 token，它按规则展开成 Rust 代码，然后编译器再检查展开结果。

## 知识点详解

### macro_rules

```rust
macro_rules! say {
    ($name:expr) => {
        println!("hello {}", $name);
    };
}

fn main() {
    say!("rust");
}
```

### derive 宏

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
struct User {
    id: u64,
    name: String,
}
```

derive 宏为类型生成常见 Trait 实现。Serde、clap、thiserror 都大量使用过程宏。

### 宏调试

常用 `cargo expand` 查看展开结果。遇到复杂宏错误时，先缩小输入，再看展开代码是否符合预期。

## 例子

```rust
macro_rules! hashmap {
    ($($key:expr => $value:expr),* $(,)?) => {{
        let mut map = std::collections::HashMap::new();
        $(map.insert($key, $value);)*
        map
    }};
}
```

## 练习

1. 写一个 `vec_of_strings!` 宏，把多个表达式转成 `Vec<String>`。
2. 使用 `thiserror` derive 实现错误类型。
3. 用 `cargo expand` 查看 serde derive 生成的大致结构。

## 验收

- 能解释声明宏和过程宏的区别。
- 能判断什么时候该用函数，什么时候才值得用宏。
- 能用展开结果辅助排错。

## 重点

- 宏生成的是代码，不是运行时魔法。

## 难点

- 宏错误常出现在展开后的代码，定位要从宏输入、匹配规则和展开结果一起看。

## 易错

> **易错：** 为了减少几行重复代码过早写宏。
>
> 正确做法：优先用函数、泛型和 Trait；只有语法层重复或编译期生成确实必要时再用宏。
