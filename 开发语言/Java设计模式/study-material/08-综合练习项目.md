## 项目：订单处理与通知平台

目标：用 Java 实现一个小型订单处理平台，综合应用创建型、结构型和行为型模式。

## 业务需求

- 创建订单：支持普通订单、预售订单、虚拟商品订单。
- 支付：支持支付宝、微信、余额。
- 优惠：支持会员折扣、新人券、满减。
- 风控：黑名单、限额、频率检查。
- 状态：待支付、已支付、已取消、已发货。
- 通知：支付成功后发送短信、邮件、站内信。
- 存储：发票文件支持本地存储和云存储。

## 推荐模式

| 需求 | 推荐模式 | 说明 |
| --- | --- | --- |
| 创建不同订单 | 工厂方法 | 隔离具体订单类型 |
| 构造支付请求 | Builder | 参数多且需要校验 |
| 支付渠道 | 策略 | 不同渠道算法可替换 |
| 风控检查 | 责任链 | 多个检查器顺序执行 |
| 订单状态 | 状态 | 状态驱动行为 |
| 支付成功通知 | 观察者 | 发布者和订阅者解耦 |
| 文件存储接入 | 适配器 | 接入不同存储 API |
| 文件加密压缩 | 装饰器 | 动态叠加职责 |
| 发票访问权限 | 代理 | 控制读取权限 |
| 下单流程入口 | 外观 | 封装复杂子系统 |

## 核心接口草图

```java
interface PaymentStrategy {
    String channel();
    PaymentResult pay(PaymentRequest request);
}

interface RiskCheck {
    void check(Order order, RiskContext context);
}

interface OrderState {
    void pay(Order order);
    void cancel(Order order);
    void ship(Order order);
}

interface NotificationListener {
    void onOrderPaid(OrderPaidEvent event);
}

interface FileStorage {
    String save(String filename, byte[] bytes);
    byte[] read(String fileId);
}
```

## 实操任务

1. 实现支付策略注册表。
2. 实现风控责任链，并支持配置处理器顺序。
3. 实现订单状态迁移，禁止非法状态操作。
4. 实现事件发布器，支付成功后通知多个监听器。
5. 实现文件存储适配器，并用装饰器增加压缩。
6. 实现 `OrderFacade`，对外提供 `createAndPay` 方法。
7. 为每个模式写至少一个单元测试。

## 验收标准

- 新增一种支付渠道时，不修改 `OrderFacade`。
- 新增一个风控规则时，只新增一个 `RiskCheck` 实现。
- 非法状态迁移能抛出明确异常。
- 通知失败不会破坏支付结果，且能记录失败信息。
- 文件存储可替换实现，调用方不依赖具体 API。

## 重点

- 设计模式要落到可测试代码。
- 每个模式都必须有明确业务变化点。
- 对外接口要稳定，内部实现可替换。

## 难点

- 责任链和观察者都涉及顺序、异常和失败处理。
- 状态模式要避免状态对象反向依赖过多服务。
- 外观不能吞掉所有业务细节变成上帝类。

## 易错

> **易错：** 为了展示模式，把所有代码都拆成很多小类。
>
> 正确做法：只有变化点明确的地方才引入模式。

> **易错：** 只写成功路径，不处理异常和非法状态。
>
> 正确做法：练习项目必须覆盖失败路径和边界条件。
