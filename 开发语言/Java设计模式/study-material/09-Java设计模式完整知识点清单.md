# Java设计模式学习资料：Java设计模式完整知识点清单

[返回索引](../Java设计模式学习资料.md)

## 基础与术语

- 面向对象：封装、继承、多态、抽象。
- Java 基础：接口、抽象类、组合、泛型、枚举、Lambda、record、反射、动态代理。
- UML：类、接口、继承、实现、关联、聚合、组合、依赖。
- 设计原则：SOLID、DRY、KISS、YAGNI、组合优于继承、面向接口编程。

## 创建型模式

- 单例：懒汉、饿汉、双重检查、静态内部类、枚举单例、容器单例。
- 工厂方法：产品接口、具体产品、工厂接口、具体工厂。
- 抽象工厂：产品族、一致性约束、跨平台或跨厂商切换。
- 建造者：复杂构造、不可变对象、参数校验、链式调用。
- 原型：复制构造器、深拷贝、浅拷贝、模板对象。

## 结构型模式

- 适配器：类适配器、对象适配器、第三方接口接入。
- 桥接：抽象维度和实现维度解耦。
- 组合：树形结构、透明式和安全式组合。
- 装饰器：同接口包装、职责叠加、包装顺序。
- 外观：统一入口、子系统协调、边界控制。
- 享元：内部状态、外部状态、对象池、缓存。
- 代理：静态代理、JDK 动态代理、CGLIB、远程代理、保护代理、虚拟代理。

## 行为型模式

- 策略：算法替换、策略注册、默认策略、策略选择。
- 模板方法：流程骨架、钩子方法、回调。
- 观察者：发布订阅、同步事件、异步事件、事件一致性。
- 责任链：顺序、短路、共享上下文、异常传播。
- 命令：请求对象化、队列、撤销、重试、日志。
- 状态：状态对象、状态迁移、非法操作、状态机。
- 迭代器：遍历协议、隐藏集合结构。
- 中介者：交互集中协调、降低网状耦合。
- 备忘录：快照、撤销、封装状态。
- 解释器：表达式语法、规则解释、适用边界。
- 访问者：稳定对象结构、新增操作、双分派。

## Java 与框架应用

- JDK：`Runtime`、`Calendar`、`Iterator`、`InputStream`、`Proxy`、`Comparator`、`Executor`。
- Spring：`BeanFactory`、`FactoryBean`、AOP、`JdbcTemplate`、事件机制、MVC 拦截器。
- Servlet：FilterChain、Listener。
- MyBatis：Mapper 动态代理、`SqlSessionFactory`、插件拦截器。
- 日志框架：门面模式，如 SLF4J。

## 调度、选择和冲突规则

- 策略选择：通常由业务 key、类型字段、注解、配置或容器 Bean 名称决定。
- 责任链顺序：由列表顺序、优先级注解、配置文件或注册顺序决定。
- 观察者顺序：同步监听器可能有顺序，异步事件通常不保证全局顺序。
- 代理分派：调用必须经过代理对象，目标对象内部自调用通常不被代理拦截。
- 装饰器顺序：外层先接收调用，内层后执行；返回时顺序相反。
- 状态迁移：必须定义合法迁移表和非法操作处理方式。
- 工厂选择：要定义重复 key、默认实现、懒加载、失败回退和配置热更新规则。
- 模板扩展点：要定义钩子方法是否允许跳过、异常是否吞掉、资源是否由模板统一关闭。
- 访问者分派：依赖元素类型和访问者重载方法，适合对象结构稳定的场景。

## 深度机制

- 对象生命周期：类加载、实例化、依赖注入、初始化、代理包装、注册、销毁。
- DI 容器创建链：BeanDefinition、构造器选择、循环依赖、三级缓存、BeanPostProcessor。
- 代理链：方法匹配、增强排序、目标调用、异常传播、返回值处理。
- 责任链执行模型：固定链、动态链、短路链、过滤链、拦截链。
- 策略注册表：手动 Map、枚举路由、注解扫描、Spring `Map<String, Bean>` 注入。
- 状态机：状态、事件、动作、守卫条件、迁移表、并发版本控制。
- 事件模型：本地事件、事务事件、MQ 事件、Outbox、幂等消费。
- 缓存装饰器：缓存 key、穿透、击穿、雪崩、双写一致性、失效顺序。
- 线程安全：单例可变状态、策略无状态化、上下文隔离、ThreadLocal 清理。

## 性能、维护和风险

- 抽象层会增加跳转成本，过度设计会降低可读性。
- 动态代理和反射增加排查难度。
- 单例和共享缓存需要关注线程安全。
- 责任链过长会增加延迟和隐式依赖。
- 观察者异步化要处理幂等、重试、顺序和事务一致性。
- 享元要求共享对象不可变或受控修改。

## 排障清单

- 接口实现没有被调用：检查注册、Bean 扫描、key 映射和条件匹配。
- AOP 不生效：检查是否经过代理、方法是否 public、代理类型、切点表达式。
- 责任链顺序错误：检查配置顺序、优先级、短路条件。
- 策略选择错误：检查业务 key、默认策略、重复注册。
- 状态流转异常：检查状态迁移表和并发修改。
- 装饰器行为异常：检查包装顺序和是否重复包装。
