## 一、创建型模式

### 1. 单例模式有哪些实现？推荐哪种？

**参考答案：**

常见实现有饿汉式、懒汉式、双重检查锁、静态内部类和枚举。Java 中枚举单例最简洁，天然防反射和序列化破坏；静态内部类也常用，具备延迟加载和线程安全。

```java
public enum AppConfig {
    INSTANCE;
}
```

> **重点：** 单例最大风险是全局状态和测试困难。

### 2. 双重检查锁为什么要用 volatile？

**参考答案：**

因为对象创建不是单一步骤，可能发生指令重排。没有 `volatile` 时，其他线程可能看到一个引用非空但尚未初始化完成的对象。`volatile` 可以保证可见性并禁止相关重排序。

> **难点：** 这里考的是 Java 内存模型，不只是设计模式。

### 3. 工厂方法和抽象工厂有什么区别？

**参考答案：**

工厂方法关注一个产品等级结构的创建；抽象工厂关注一族相关产品的创建。比如创建不同支付客户端可以用工厂方法；按云厂商创建存储、队列、消息等一组组件，更适合抽象工厂。

> **易错：** 用“一个方法还是多个方法”区分，这是表面差异。

### 4. Builder 模式适合什么场景？

**参考答案：**

适合参数多、可选字段多、构造过程需要校验、对象希望不可变的场景。Builder 能避免构造器参数顺序错误，也能在 `build()` 中集中检查业务不变量。

### 5. 原型模式在 Java 中怎么实现？

**参考答案：**

可以用 `Cloneable`，但实践中更推荐复制构造器、静态工厂或 `copy` 方法，因为 `clone()` 语义复杂，深拷贝和浅拷贝容易出错。

> **易错：** 可变集合只浅拷贝，导致两个对象共享同一份内部状态。
