## 一、结构型模式

### 1. 代理、装饰器、适配器、外观有什么区别？

**参考答案：**

适配器解决接口不兼容；装饰器在保持同一接口的情况下叠加功能；代理控制对真实对象的访问；外观为复杂子系统提供简单入口。它们代码上都可能“包一层”，但意图不同。

> **重点：** 面试时用意图区分，不要只看类结构。

### 2. Java 动态代理和 CGLIB 有什么区别？

**参考答案：**

JDK 动态代理基于接口生成代理类；CGLIB 通过继承目标类生成子类代理，因此不能代理 final 类和 final 方法。Spring AOP 会根据目标对象是否有接口等条件选择代理方式。

> **易错：** 认为代理一定能拦截所有调用。内部自调用通常不会经过代理。

### 3. 装饰器模式和继承相比有什么优势？

**参考答案：**

装饰器可以运行时组合多个功能，避免继承层级爆炸。例如 IO 流可以叠加缓冲、压缩、加密，而不用为每种组合创建一个子类。

## 二、行为型模式

### 4. 策略模式适合解决什么问题？

**参考答案：**

策略模式适合一组可替换算法或业务规则，例如不同支付方式、不同折扣算法、不同路由规则。调用方依赖策略接口，通过 key、配置或容器选择具体策略。

```java
interface PayStrategy {
    String channel();
    void pay(Order order);
}
```

### 5. 策略模式和状态模式有什么区别？

**参考答案：**

策略模式关注算法替换，通常由外部选择策略；状态模式关注对象内部状态变化导致行为变化，状态迁移是核心。支付渠道是策略，订单从待支付到已支付再到已发货更像状态。

### 6. 责任链模式要注意什么？

**参考答案：**

要明确处理器顺序、是否短路、异常如何传播、是否共享上下文、处理器是否允许修改请求。否则链条越长，隐式依赖越多，排查越困难。

### 7. 观察者模式同步和异步有什么区别？

**参考答案：**

同步观察者简单直接，但监听器执行慢会拖慢发布者；异步观察者解耦更强，但要处理消息丢失、重试、幂等、顺序和事务一致性。

> **难点：** 业务项目中观察者演进到 MQ 后，问题从设计模式变成分布式一致性问题。
