# Java设计模式学习路线图

## 阶段 1：基础认知

- 目标：理解设计模式解决的是“变化带来的耦合问题”，不是为了让代码显得高级。
- 需要掌握：类、接口、组合、继承、多态、依赖、关联、聚合、组合、SOLID、UML 类图。
- 例子：订单支付系统中，支付方式从 `if-else` 扩展到支付宝、微信、银行卡、余额。
- 练习：画出支付模块重构前后的类关系。
- 验收：能解释“为什么优先组合而不是继承”。
- 重点：模式是复用设计经验，不是固定模板。
- 易错：把每个业务类都套模式，导致代码比问题本身更复杂。

## 阶段 2：创建型模式

- 目标：掌握对象创建过程的封装，避免调用方依赖具体类和复杂构造过程。
- 需要掌握：单例、简单工厂、工厂方法、抽象工厂、建造者、原型。
- 例子：根据渠道创建不同 `PaymentClient`，用 Builder 构造复杂请求对象。
- 练习：实现一个通知发送器工厂，支持短信、邮件、钉钉。
- 验收：能区分工厂方法和抽象工厂，能写线程安全单例。
- 重点：创建型模式处理“谁来创建、创建什么、创建过程是否复杂”。
- 易错：滥用单例隐藏全局状态，导致测试困难。

## 阶段 3：结构型模式

- 目标：掌握对象和类之间的组合方式，让接口适配、职责增强和复杂结构更稳定。
- 需要掌握：适配器、桥接、组合、装饰器、外观、享元、代理。
- 例子：用适配器接入第三方短信 API，用装饰器给查询服务增加缓存和日志。
- 练习：为文件存储服务增加本地、OSS、S3 三种实现，并用代理做权限校验。
- 验收：能区分代理、装饰器、适配器和外观。
- 重点：结构型模式处理“对象之间怎么连接才不拧巴”。
- 易错：用继承叠加功能，类数量迅速爆炸。

## 阶段 4：行为型模式

- 目标：掌握职责分配、流程编排、状态变化和算法替换。
- 需要掌握：策略、模板方法、观察者、责任链、命令、状态、迭代器、中介者、备忘录、解释器、访问者。
- 例子：优惠券计算使用策略模式，审批流程使用责任链，订单状态流转使用状态模式。
- 练习：实现一个订单风控责任链，包括黑名单、限额、频率和地域检查。
- 验收：能解释每种行为型模式的调度规则和适用边界。
- 重点：行为型模式处理“谁负责做、按什么顺序做、变化如何替换”。
- 易错：把策略模式写成一堆无命名规则的 Map，失去可读性。

## 阶段 5：框架源码与工程实践

- 目标：能在 JDK、Spring、MyBatis、Servlet、日志框架中识别模式，并能判断是否适合业务项目。
- 需要掌握：动态代理、AOP、模板回调、BeanFactory、FactoryBean、HandlerMapping、FilterChain、Executor。
- 例子：Spring AOP 使用代理，JdbcTemplate 使用模板方法和回调，Servlet Filter 使用责任链。
- 练习：阅读一个 Spring Bean 创建流程片段，标记工厂、单例、代理和模板回调。
- 验收：能把“模式名称”落到真实代码结构和调用链上。
- 重点：框架里的模式通常不是教科书式单独出现，而是组合出现。
- 易错：只记“Spring 用了工厂模式”，却说不清创建顺序、生命周期和扩展点。

## 阶段 6：深度机制和架构场景

- 目标：能解释模式在运行时如何分派、如何排序、如何冲突、如何和事务/并发/缓存组合。
- 需要掌握：Bean 生命周期、策略注册表、责任链排序、代理边界、装饰器包装顺序、状态迁移表、事件一致性。
- 例子：支付成功后要扣库存、写订单、发事件、清缓存、发通知，哪些同步做，哪些异步做，失败如何补偿。
- 练习：为订单系统画出创建链、调用链、事务边界和事件流。
- 验收：能说清一个请求从 Controller 到 Service、代理、策略、责任链、事件监听器的完整执行路径。
- 重点：设计模式进入生产系统后，核心问题会变成顺序、边界、一致性和可观测性。
- 易错：只关注类图，不关注运行时调用路径和失败路径。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | SOLID、UML、创建型模式 | 6 个小示例和模式对比表 |
| 第 2 周 | 结构型模式 | 一个可插拔文件存储模块 |
| 第 3 周 | 行为型模式 | 一个订单优惠、风控、状态流转 Demo |
| 第 4 周 | 框架源码和面试复习 | 一份源码模式清单和面试问答 |
| 第 5 周 | 深度机制、重构和生产场景 | 一份调用链图、失败处理矩阵和重构报告 |

## 最终能力清单

- 能根据变化点选择模式，而不是按模式找场景。
- 能画出模式的类关系和调用链。
- 能写出 Java 版本的最小实现和真实业务版本。
- 能识别 JDK/Spring 常见模式。
- 能指出模式的代价、适用边界和常见误用。
- 能解释运行时分派顺序、代理边界、事务边界、缓存一致性和异常传播。
