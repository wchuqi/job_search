## 使用建议

- 初学：按 0 到 4 学习，先理解“为什么要解耦”，再记模式名称。
- 写代码：重点看 2、3、4、5，每学一种模式都写一个 Java 小例子。
- 看框架源码：先看 6，再看 13，重点理解创建链、代理链、拦截链和模板回调。
- 面试复习：先看 9 的完整清单，再看 10 和 `study-material/面试知识点/`，深水区问题重点看 11 到 15。
- 项目实践：完成 8 和 16 的练习，用重构前后代码、测试和调用链证明模式是否真的降低复杂度。

## 环境假设

- Java 17+，示例尽量使用现代 Java，但不依赖复杂框架。
- 能读懂接口、抽象类、泛型、Lambda、异常、集合、反射和简单 Spring 概念。
- 设计模式不是背诵题，核心目标是识别变化点、隔离变化、降低耦合、保持可测试。
