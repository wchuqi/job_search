# Docker学习资料：日志、监控、排障和性能

[返回索引](../Docker学习资料.md)

## 学习目标

- 掌握 Docker 常见故障的分层排查方法。
- 能使用 logs、inspect、events、stats、exec、network inspect。
- 理解日志驱动、磁盘膨胀、资源限制和性能瓶颈。

## 理论导读

Docker 排障要先确认问题层级：镜像构建失败、容器启动失败、应用运行失败、网络不可达、存储权限错误、宿主机资源不足。不要一开始就重建或删除资源，先保留现场：状态、日志、退出码、inspect、events。

## 核心心智模型

排障顺序：

1. 看状态：`docker ps -a`。
2. 看日志：`docker logs`。
3. 看配置：`docker inspect`。
4. 进入现场：`docker exec`，若容器已退出则用同镜像临时容器复现。
5. 看资源：`docker stats`、宿主机磁盘和内存。
6. 看网络：`docker network inspect`、DNS、端口监听。

## 知识点详解

### 启动失败

```powershell
docker ps -a
docker logs app
docker inspect app --format "{{.State.ExitCode}} {{.State.Error}}"
```

常见原因包括命令不存在、环境变量缺失、配置文件挂载错、端口已占用、权限不足。

### 网络不通

检查点：

- 服务是否监听 `0.0.0.0`，而不是只监听 `127.0.0.1`。
- 容器是否在同一网络。
- 使用的是服务名还是错误 IP。
- 宿主机端口是否发布且未冲突。
- 防火墙或安全组是否放行。

### 日志膨胀

默认 json-file 日志如果不限制大小，可能撑满磁盘。

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
```

### 性能

- CPU：`--cpus`、线程数、GC、请求并发。
- 内存：`--memory`、OOM、缓存、语言运行时参数。
- IO：volume 性能、日志写入、数据库刷盘。
- 网络：端口转发、DNS、跨主机链路。

## 例子

```powershell
docker events --since 30m
docker stats --no-stream
docker system df
docker builder prune
```

## 练习

1. 制造端口冲突，记录 Docker 报错。
2. 制造缺失环境变量导致应用退出，写排查步骤。
3. 配置日志轮转，验证日志文件数量和大小。

## 验收

- 能独立定位容器启动失败原因。
- 能解释容器磁盘空间增长来自镜像、容器层、volume、日志还是构建缓存。
- 能根据 `stats` 判断是否受 CPU 或内存限制影响。

## 重点

- 先看日志和退出码，再改配置。
- `docker system prune` 有删除风险，执行前要知道会清理什么。
- 生产日志应输出到 stdout/stderr，再由平台采集。

## 难点

- Docker 的问题常跨应用、容器、宿主机、网络四层。
- Desktop 环境中真实磁盘和网络路径可能在内部 VM。

## 易错

> **易错：** 容器无法访问服务时直接怀疑 Docker 网络。
>
> 正确做法：先确认应用是否监听正确地址和端口，再查 Docker 网络。
