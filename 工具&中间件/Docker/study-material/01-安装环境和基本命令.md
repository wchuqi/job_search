# Docker学习资料：安装环境和基本命令

[返回索引](../Docker学习资料.md)

## 学习目标

- 安装并验证 Docker 环境。
- 掌握日常容器、镜像、日志、进入容器、清理资源命令。
- 知道 Windows、macOS、Linux 环境差异。

## 理论导读

Docker 命令看起来像是在本机直接创建容器，实际流程是 CLI 向 Docker daemon 发送请求。daemon 再负责镜像拉取、容器创建、网络配置和文件系统挂载。排障时要区分“客户端命令不可用”“daemon 未运行”“镜像拉取失败”“容器内应用失败”四类问题。

## 核心心智模型

每条 Docker 命令都可以问三个问题：

- 操作对象是什么：image、container、volume、network、builder、compose project。
- 操作阶段是什么：拉取、构建、创建、启动、停止、观察、清理。
- 状态在哪里：镜像层、容器可写层、volume、宿主机端口、日志驱动。

## 知识点详解

### 环境选择

- Windows/macOS：常用 Docker Desktop，容器实际运行在内部 Linux VM 中。
- Linux：安装 Docker Engine，更接近生产环境。
- WSL2：Windows 上适合开发，但文件性能和路径挂载需要注意。

### 常用命令分类

```powershell
docker version
docker info
docker pull nginx:alpine
docker images
docker run --name web -d -p 8080:80 nginx:alpine
docker ps
docker ps -a
docker logs web
docker exec -it web sh
docker stop web
docker start web
docker rm web
docker rmi nginx:alpine
```

### run 命令的关键参数

- `--name`：容器名，便于后续操作。
- `-d`：后台运行。
- `-p HOST:CONTAINER`：端口发布。
- `-e KEY=VALUE`：注入环境变量。
- `-v SOURCE:TARGET`：挂载 volume 或宿主机路径。
- `--rm`：容器退出后自动删除，适合一次性任务。
- `--restart`：重启策略，如 `unless-stopped`。

## 例子

```powershell
docker run --rm alpine:3.20 echo "hello docker"
docker run --name redis-demo -d -p 6379:6379 redis:7-alpine
docker logs redis-demo
docker exec -it redis-demo redis-cli ping
docker rm -f redis-demo
```

## 练习

1. 使用 `docker run --rm` 执行一次性命令。
2. 使用 `docker run -d` 启动后台服务。
3. 用 `docker inspect` 找到容器 IP、环境变量和挂载信息。

## 验收

- 能启动、停止、删除容器。
- 能查看镜像、容器、日志、详细元数据。
- 能解释 `docker run` 和 `docker start` 的区别：前者创建新容器，后者启动已有容器。

## 重点

- `docker run` 等于 `create` 加 `start`。
- `docker ps` 默认只看运行中容器，`docker ps -a` 才包含已退出容器。
- `docker logs` 看的是容器标准输出和标准错误，不等于所有应用日志文件。

## 难点

- Docker Desktop 下宿主机不一定等于真正运行容器的 Linux VM。
- 端口映射、路径挂载、文件权限在不同系统上表现可能不同。

## 易错

> **易错：** 容器退出后以为 Docker 坏了。
>
> 正确做法：先看 `docker ps -a`、`docker logs <container>` 和退出码。

> **易错：** 运行多个容器都映射同一个宿主机端口。
>
> 正确做法：同一个宿主机端口同一时间只能被一个进程或映射占用。
