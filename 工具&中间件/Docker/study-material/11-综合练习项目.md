# Docker学习资料：综合练习项目

[返回索引](../Docker学习资料.md)

## 项目目标

构建一个 `Web API + PostgreSQL + Redis + Nginx` 的 Docker 化开发环境，并补齐镜像构建、Compose 编排、数据持久化、健康检查、日志和安全基础。

## 项目结构

```text
docker-demo/
  app/
    Dockerfile
    src/
  nginx/
    default.conf
  compose.yml
  .dockerignore
  README.md
```

## 阶段 1：单容器运行

### 学习目标

- 能用 Dockerfile 构建 Web API 镜像。
- 能用环境变量配置端口和数据库连接。

### 练习

1. 编写 Dockerfile。
2. 构建镜像 `docker-demo-api:local`。
3. 使用 `docker run` 启动，并通过宿主机访问健康接口。

### 验收

- `docker run --rm docker-demo-api:local` 可以启动。
- 镜像内没有 `.git`、本地日志和密钥文件。

## 阶段 2：加入数据库和缓存

### 学习目标

- 使用 Compose 编排多容器。
- 使用服务名连接 PostgreSQL 和 Redis。

### 练习

1. 增加 `db` 和 `redis` service。
2. 为 PostgreSQL 配置 volume。
3. 给数据库添加 healthcheck。

### 验收

- `docker compose up -d` 后所有服务健康。
- 删除 Web 容器不影响数据库数据。

## 阶段 3：加入 Nginx

### 学习目标

- 理解反向代理容器和后端服务的网络关系。
- 只暴露 Nginx 端口，不直接暴露后端端口。

### 练习

1. Nginx 通过 `proxy_pass http://api:8080` 访问后端。
2. 只在 Nginx 上配置 `ports`。
3. 后端仅使用 `expose` 或不暴露宿主机端口。

### 验收

- 宿主机通过 `http://localhost:8080` 访问 Nginx。
- API 容器不发布宿主机端口。

## 阶段 4：安全和排障

### 学习目标

- 改为非 root 用户运行。
- 添加日志轮转、资源限制和只读文件系统尝试。

### 练习

1. Dockerfile 增加 `USER`。
2. Compose 中为 API 增加 `read_only: true`，必要目录用 tmpfs。
3. 设置资源限制并观察 `docker stats`。

### 验收

- 应用非 root 运行。
- `docker compose logs` 能看到结构化日志。
- 能写出 5 个常见故障和排查命令。

## 最终验收

- 一条命令启动完整环境：`docker compose up -d --build`。
- 一条命令清理开发环境：`docker compose down`。
- 数据卷清理必须显式执行：`docker compose down -v`。
- README 记录端口、环境变量、启动、停止、备份和排障命令。

## 重点

- 开发环境要便捷，生产镜像要干净。
- 数据持久化和服务健康检查必须明确。
- Compose 的服务名是容器通信的稳定入口。

## 易错

> **易错：** 为了本地调试把所有服务端口都暴露到宿主机。
>
> 正确做法：只暴露需要从宿主机访问的入口，内部服务走 Docker 网络。
