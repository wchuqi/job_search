## 一、基础概念

### 1. Docker 是什么？解决什么问题？

**参考答案：**

Docker 是容器化平台，用镜像打包应用及其运行依赖，再用容器在隔离环境中运行应用进程。它解决环境不一致、部署步骤不可复现、依赖冲突和交付效率低的问题。

> **重点：** Docker 的核心价值是可复现交付，不只是快速安装软件。

### 2. Docker 和虚拟机有什么区别？

**参考答案：**

虚拟机虚拟硬件并运行完整操作系统和独立内核；Docker 容器共享宿主机内核，通过 namespace 和 cgroup 隔离进程视图和资源。容器启动更快、资源开销更小，但隔离边界通常弱于虚拟机。

> **易错：** 说“Docker 是轻量虚拟机”不够准确。更准确是“基于操作系统内核能力的进程隔离和镜像分发机制”。

### 3. 镜像和容器有什么区别？

**参考答案：**

镜像是只读模板，包含文件系统和启动元数据；容器是镜像运行后的实例，包含运行进程、网络命名空间和可写层。同一个镜像可以创建多个容器。

### 4. Dockerfile、镜像、容器、仓库之间是什么关系？

**参考答案：**

Dockerfile 定义镜像如何构建；构建后得到镜像；镜像运行后成为容器；镜像可推送到仓库供其他机器拉取。

```text
Dockerfile -> docker build -> image -> docker run -> container -> docker push/pull -> registry
```

### 5. 容器为什么会退出？

**参考答案：**

容器由主进程决定生命周期。容器内 PID 1 进程退出，容器就退出。常见原因包括启动命令执行完毕、应用异常、配置缺失、端口冲突、权限错误或 OOM。

> **重点：** 排查先看 `docker ps -a`、`docker logs`、退出码和 `docker inspect`。
