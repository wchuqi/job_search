## 一、容器运行

### 1. namespace 和 cgroup 分别解决什么问题？

**参考答案：**

namespace 负责隔离视图，例如进程号、网络、挂载点、主机名、用户等；cgroup 负责资源限制和统计，例如 CPU、内存、IO 和进程数。容器隔离通常由两者配合完成。

> **重点：** namespace 解决“看见什么”，cgroup 解决“能用多少”。

### 2. Docker 的重启策略有哪些？

**参考答案：**

常见策略有 `no`、`on-failure`、`always`、`unless-stopped`。生产中经常使用 `unless-stopped`，但具体选择要结合部署平台。如果由 Kubernetes 管理，通常交给 Kubernetes 控制重启。

## 二、网络

### 3. 容器之间如何通信？

**参考答案：**

推荐把容器加入同一个用户自定义 bridge 网络，通过容器名或 Compose 服务名访问对方的容器端口。不要依赖容器 IP，因为 IP 会变化。

```powershell
docker network create appnet
docker run -d --name redis --network appnet redis:7-alpine
docker run --rm --network appnet redis:7-alpine redis-cli -h redis ping
```

### 4. ports 和 expose 有什么区别？

**参考答案：**

`ports` 把容器端口发布到宿主机端口，外部或宿主机可以访问；`expose` 只是声明容器端口，不做宿主机端口映射。容器间在同一网络中访问通常不需要 `ports`。

### 5. 为什么容器里访问 localhost 连接不到另一个容器？

**参考答案：**

容器内的 `localhost` 指容器自己的网络命名空间，不是宿主机，也不是其他容器。访问另一个容器应使用服务名或容器名。

## 三、存储

### 6. volume 和 bind mount 有什么区别？

**参考答案：**

volume 由 Docker 管理，适合持久化服务数据；bind mount 直接挂载宿主机路径，适合开发调试和挂载配置。bind mount 更依赖宿主机目录结构和权限。

### 7. 删除容器后数据一定会丢吗？

**参考答案：**

不一定。容器可写层会随容器删除而删除，但 volume 和 bind mount 中的数据独立于容器生命周期。是否丢失取决于数据放在哪里。

> **易错：** 数据库数据放在容器可写层是高风险做法。
