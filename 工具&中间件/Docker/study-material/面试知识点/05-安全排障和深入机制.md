## 一、安全

### 1. Docker 容器默认安全吗？

**参考答案：**

Docker 提供一定隔离，但不能视为绝对安全边界。容器共享宿主机内核，错误配置如 `--privileged`、挂载 Docker socket、挂载宿主机根目录都会显著扩大风险。

### 2. 如何提升容器安全性？

**参考答案：**

使用可信基础镜像，固定版本；以非 root 用户运行；减少 capability；避免 `--privileged`；使用只读文件系统；密钥外置；扫描镜像漏洞；记录 digest；限制资源；只暴露必要端口。

### 3. 为什么挂载 Docker socket 危险？

**参考答案：**

Docker socket 可以控制 Docker daemon。容器如果能访问 socket，就可能创建高权限容器、挂载宿主机目录，从而接近获得宿主机 root 控制能力。

> **重点：** 挂载 `/var/run/docker.sock` 不是普通文件挂载，它暴露了容器管理控制面。

## 二、排障

### 4. 容器启动后马上退出怎么排查？

**参考答案：**

先执行 `docker ps -a` 查看状态，再用 `docker logs <container>` 看日志，用 `docker inspect` 看退出码、启动命令、环境变量、挂载和错误信息。常见原因是主进程执行完、命令不存在、配置缺失、权限错误、端口冲突或 OOM。

### 5. 容器网络不通怎么排查？

**参考答案：**

检查服务是否监听正确地址和端口，容器是否在同一网络，DNS 是否能解析服务名，访问的是容器端口还是宿主机端口，端口是否发布，宿主机防火墙或安全组是否拦截。

### 6. Docker 占用磁盘很大怎么办？

**参考答案：**

先用 `docker system df` 判断占用来自镜像、容器、volume 还是构建缓存。无用镜像和缓存可以 prune，但 volume 可能包含业务数据，不能直接清理。

```powershell
docker system df
docker image prune
docker builder prune
```

> **易错：** 直接执行 `docker system prune -a --volumes` 可能删除重要数据。

## 三、深入机制

### 7. overlay2 大致如何工作？

**参考答案：**

overlay2 是 Docker 常见存储驱动之一，它把多个只读镜像层和一个容器可写层叠加成容器看到的统一文件系统。写入已有文件时通常会触发 copy-on-write，把文件复制到可写层再修改。

### 8. 容器内 root 等于宿主机 root 吗？

**参考答案：**

默认情况下不完全等价，因为有 namespace、capability、seccomp 等限制。但如果配置不当，例如 privileged、危险挂载、内核漏洞、Docker socket 暴露，容器内 root 可能对宿主机造成严重影响。
