## 速记清单

- Docker 是镜像分发加进程隔离，不是完整虚拟机。
- 镜像只读，容器运行时有可写层。
- 容器生命周期由 PID 1 主进程决定。
- namespace 隔离视图，cgroup 限制资源。
- 同网络容器用服务名访问，不依赖 IP。
- `ports` 发布宿主机端口，`expose` 只声明容器端口。
- volume 适合持久化数据，bind mount 适合开发挂载和配置。
- 构建上下文要用 `.dockerignore` 控制。
- `ENTRYPOINT` 是入口程序，`CMD` 是默认命令或参数。
- tag 可变，digest 锁定内容。
- 生产不要只用 `latest`。
- 密钥不能写进 Dockerfile、镜像层和 Git。
- `--privileged`、Docker socket、宿主机根目录挂载都是高风险配置。
- 排障顺序：ps、logs、inspect、exec、stats、network inspect。

## 自测题

### 1. 为什么同一个镜像可以启动多个互不影响的容器？

**参考答案：**

因为镜像层是共享只读层，每个容器有独立可写层、进程和网络等隔离视图，所以运行状态互不影响。

### 2. 为什么 Dockerfile 中先复制依赖描述文件再复制源码？

**参考答案：**

依赖描述文件变化频率通常低于源码。先安装依赖可以提高缓存命中率，源码变化时只重建后面的层。

### 3. Web 容器连接数据库容器应该写什么地址？

**参考答案：**

在同一 Docker 网络或 Compose 项目中，应写数据库服务名和容器端口，例如 `db:5432`，不是 `localhost:5432`。

### 4. 删除容器前如何判断数据是否安全？

**参考答案：**

用 `docker inspect` 查看 Mounts，确认业务数据是否在 volume 或 bind mount 中；再确认是否有备份。容器可写层数据删除容器后通常不可保留。

### 5. 容器 exited，第一步做什么？

**参考答案：**

执行 `docker ps -a` 看状态，再用 `docker logs` 看应用输出，用 `docker inspect` 看退出码和错误。

### 6. 如何回答“Docker 怎么做生产发布”？

**参考答案：**

源码提交触发 CI，测试通过后构建镜像，扫描漏洞和敏感信息，推送 registry，部署系统按明确 tag 或 digest 拉取。配置和密钥外置，日志指标由平台采集，发布策略支持回滚。

## 最终验收

- 能 3 分钟讲清 Docker 架构和虚拟机区别。
- 能现场写一个含 `FROM`、`WORKDIR`、`COPY`、`RUN`、`USER`、`ENTRYPOINT` 的 Dockerfile。
- 能现场写一个 Web + DB 的 Compose 文件。
- 能解释一次容器启动失败的排查链路。
