## 一、镜像和构建

### 1. Docker 镜像为什么是分层的？

**参考答案：**

分层可以复用公共层、加快构建和拉取、节省磁盘空间。Dockerfile 中多数指令会生成层，后续镜像可以共享相同内容层。构建缓存也依赖层和指令顺序。

> **难点：** 前面层变化会导致后续层缓存失效，所以依赖安装通常放在源码复制之前。

### 2. 什么是构建上下文？

**参考答案：**

构建上下文是 `docker build` 发送给构建器的文件集合。`docker build -t app .` 里的 `.` 就是上下文。Dockerfile 的 `COPY` 和 `ADD` 只能访问上下文内文件。

> **易错：** `.dockerignore` 不是忽略镜像内文件，而是过滤构建上下文。

### 3. COPY 和 ADD 有什么区别？

**参考答案：**

`COPY` 只复制文件或目录，语义清晰。`ADD` 额外支持自动解压本地 tar 和从 URL 添加文件。多数场景推荐 `COPY`，避免隐式行为。

### 4. ENTRYPOINT 和 CMD 有什么区别？

**参考答案：**

`ENTRYPOINT` 表示容器固定入口程序，`CMD` 表示默认命令或默认参数。两者同时存在时，通常组合成 `ENTRYPOINT + CMD`。

```dockerfile
ENTRYPOINT ["java", "-jar", "/app/app.jar"]
CMD ["--spring.profiles.active=prod"]
```

### 5. 如何减小镜像体积？

**参考答案：**

使用合适的基础镜像、多阶段构建、清理包管理缓存、只复制运行所需文件、减少不必要工具和依赖、正确编写 `.dockerignore`。

> **重点：** 体积小不是唯一目标，还要考虑可维护性、安全补丁和调试需求。

### 6. ARG 和 ENV 有什么区别？

**参考答案：**

`ARG` 是构建期变量，主要在 `docker build` 阶段使用；`ENV` 写入镜像元数据，容器运行时默认可见。敏感信息不应放入 `ARG` 或 `ENV`，因为可能出现在镜像历史或元数据中。
