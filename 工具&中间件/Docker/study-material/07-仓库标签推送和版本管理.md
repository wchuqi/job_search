# Docker学习资料：仓库、标签、推送和版本管理

[返回索引](../Docker学习资料.md)

## 学习目标

- 理解 registry、repository、tag、digest。
- 掌握镜像命名、打标签、登录、推送、拉取。
- 设计可追溯的镜像版本策略。

## 理论导读

镜像仓库负责分发镜像。镜像引用通常形如 `registry.example.com/team/app:1.2.3`。tag 是可变标签，digest 是内容哈希。生产系统如果只依赖 tag，可能出现同名 tag 指向不同镜像的情况；digest 更适合锁定不可变内容。

## 核心心智模型

tag 像便签，可以挪到新镜像；digest 像指纹，内容变了指纹就变。发布系统需要人类可读的版本，也需要机器可验证的不可变标识。

## 知识点详解

### 镜像命名

```text
[registry/][namespace/]repository[:tag]
```

示例：

```text
nginx:1.27-alpine
docker.io/library/redis:7-alpine
registry.example.com/backend/order-service:2026.06.06-abc1234
```

### 打标签和推送

```powershell
docker build -t order-service:local .
docker tag order-service:local registry.example.com/backend/order-service:1.0.0
docker login registry.example.com
docker push registry.example.com/backend/order-service:1.0.0
```

### 标签策略

- 语义版本：`1.4.2`，适合产品版本。
- Git commit：`sha-abc1234`，适合追溯源码。
- 日期构建：`2026.06.06.1`，适合流水线。
- 环境标签：`dev`、`staging`、`prod` 可以作为指针，但不能作为唯一追溯依据。

## 例子

```powershell
docker pull nginx:1.27-alpine
docker image inspect nginx:1.27-alpine --format "{{index .RepoDigests 0}}"
```

## 练习

1. 为同一个镜像打两个 tag，观察 image ID 是否相同。
2. 推送到本地 registry 或私有 registry。
3. 记录镜像 tag、digest、Git commit 的对应关系。

## 验收

- 能解释 tag 和 digest 区别。
- 能完成镜像 push/pull。
- 能设计至少一种可回滚的标签策略。

## 重点

- tag 可变，digest 不可变。
- 生产发布要能追溯源码、构建时间、依赖版本和镜像 digest。
- 私有仓库凭据不能写进镜像。

## 难点

- 多架构镜像可能是 manifest list，同一 tag 在不同平台拉取不同镜像内容。
- registry 权限、证书、代理和镜像加速会影响拉取推送。

## 易错

> **易错：** 生产部署固定写 `latest`。
>
> 正确做法：使用明确版本，关键环境记录 digest。
