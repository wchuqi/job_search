# Docker学习资料：命令速查

[返回索引](../Docker学习资料.md)

## 环境

```powershell
docker version
docker info
docker context ls
```

## 镜像

```powershell
docker pull nginx:alpine
docker images
docker image inspect nginx:alpine
docker image history nginx:alpine
docker rmi nginx:alpine
```

## 构建

```powershell
docker build -t app:local .
docker build --no-cache -t app:local .
docker build --progress=plain -t app:local .
docker builder prune
```

## 容器

```powershell
docker run --name app -d -p 8080:80 nginx:alpine
docker ps
docker ps -a
docker logs -f app
docker exec -it app sh
docker stop app
docker start app
docker restart app
docker rm -f app
docker inspect app
```

## 网络

```powershell
docker network ls
docker network create appnet
docker network inspect appnet
docker network connect appnet app
docker network disconnect appnet app
docker network rm appnet
```

## 卷

```powershell
docker volume ls
docker volume create data
docker volume inspect data
docker volume rm data
docker run --rm -v data:/data alpine:3.20 ls /data
```

## Compose

```powershell
docker compose config
docker compose up -d
docker compose up -d --build
docker compose ps
docker compose logs -f
docker compose exec app sh
docker compose restart app
docker compose down
docker compose down -v
```

## 排障

```powershell
docker events --since 1h
docker stats
docker system df
docker logs --tail 100 app
docker inspect app --format "{{json .State}}"
```

## 高风险清理命令

```powershell
docker container prune
docker image prune
docker image prune -a
docker volume prune
docker system prune
docker system prune -a --volumes
```

> **易错：** `prune` 类命令可能删除未使用资源，`--volumes` 会影响数据。生产环境执行前必须确认资源是否真的无用。

## 重点

- 先 inspect 再删除。
- 先 `down` 再考虑 `down -v`。
- 镜像和容器可重建，业务数据不能随意清理。
