# Maven 学习资料：构建插件、资源、测试和打包

[返回索引](../Maven学习资料.md)

## 学习目标

- 配置常用构建插件。
- 理解资源处理和过滤。
- 掌握 Surefire 和 Failsafe 的区别。
- 理解 jar、war、shade、assembly 打包方式。

## 理论导读

Maven 的构建能力来自插件。生命周期只是规定顺序，插件才是真正工作的组件。编译、复制资源、运行测试、生成 jar、生成源码包、打可执行包，都由不同插件完成。学习插件时要关注三个问题：它绑定到哪个阶段、执行哪个 goal、读取哪些配置。

资源和测试是构建中最常出错的环节。资源文件可能需要复制或变量替换，测试可能被跳过或分成单元测试和集成测试，打包可能只是普通 jar，也可能需要把依赖一起打进可执行包。不同插件解决的是不同层次的问题。

## 常用插件

| 插件 | 作用 |
| --- | --- |
| `maven-compiler-plugin` | 编译 Java |
| `maven-surefire-plugin` | 运行单元测试 |
| `maven-failsafe-plugin` | 运行集成测试 |
| `maven-jar-plugin` | 打 jar |
| `maven-war-plugin` | 打 war |
| `maven-shade-plugin` | 生成包含依赖的 fat jar |
| `maven-source-plugin` | 生成源码包 |
| `maven-javadoc-plugin` | 生成 Javadoc |

## 资源过滤

```xml
<build>
    <resources>
        <resource>
            <directory>src/main/resources</directory>
            <filtering>true</filtering>
        </resource>
    </resources>
</build>
```

`application.properties`：

```properties
app.version=${project.version}
```

> **易错：** 不要对二进制资源开启 filtering，可能破坏文件内容。

## 单元测试和集成测试

Surefire 默认运行：

```text
*Test.java
*Tests.java
*TestCase.java
```

Failsafe 常用于集成测试：

```text
*IT.java
*ITCase.java
```

## 跳过测试

```bash
mvn package -DskipTests
mvn package -Dmaven.test.skip=true
```

区别：

- `-DskipTests` 跳过执行测试，但仍可能编译测试。
- `-Dmaven.test.skip=true` 跳过测试编译和执行。

## Shade 打包

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-shade-plugin</artifactId>
    <version>3.5.3</version>
    <executions>
        <execution>
            <phase>package</phase>
            <goals>
                <goal>shade</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

## 重点

> **重点：** 插件配置要说明版本、执行阶段、goal 和 configuration，避免依赖 Maven 默认版本带来不可控行为。

## 难点

> **难点：** 单元测试和集成测试的生命周期位置不同。集成测试通常不应混进普通 `test` 阶段，否则会拖慢日常构建。
