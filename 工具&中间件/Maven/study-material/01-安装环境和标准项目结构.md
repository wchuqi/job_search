# Maven 学习资料：安装、环境和标准项目结构

[返回索引](../Maven学习资料.md)

## 学习目标

- 安装并验证 Maven。
- 理解 Maven 和 JDK 的关系。
- 掌握标准项目目录结构。
- 能创建并运行一个最小 Maven 项目。

## 理论导读

Maven 本身不编译 Java，真正编译的是 JDK 里的 `javac`。Maven 像构建调度员，读取 POM 后决定什么时候调用编译插件、测试插件和打包插件；JDK 像真正的工具箱，提供编译器、运行时和标准库。因此 Maven 环境排查必须同时检查 `mvn -version` 和 `java -version`。

标准目录结构是 Maven 的重要约定。`src/main/java` 放生产代码，`src/test/java` 放测试代码，`src/main/resources` 放运行时资源，`target/` 放构建产物。Maven 通过约定减少配置：只要目录符合规范，很多插件就知道去哪里找源码、资源和测试。

## 核心心智模型

```text
project/
  pom.xml                   项目说明书
  src/main/java/            生产 Java 源码
  src/main/resources/       生产资源文件
  src/test/java/            测试源码
  src/test/resources/       测试资源
  target/                   Maven 构建输出
```

## 安装和验证

```bash
mvn -version
java -version
javac -version
```

输出中要关注：

- Maven 版本。
- Java version。
- Java home。
- 操作系统信息。

> **易错：** IDE 使用的 JDK、命令行 `java` 使用的 JDK、Maven 编译插件配置的 release 可能不是同一个版本。

## 最小 Maven 项目

`pom.xml`：

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>maven-demo</artifactId>
    <version>1.0.0</version>

    <properties>
        <maven.compiler.release>21</maven.compiler.release>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>
</project>
```

`src/main/java/com/example/App.java`：

```java
package com.example;

public class App {
    public static void main(String[] args) {
        System.out.println("Hello Maven");
    }
}
```

构建：

```bash
mvn compile
mvn package
```

## 常用命令

```bash
mvn clean
mvn compile
mvn test
mvn package
mvn install
```

## 重点

> **重点：** Maven 借助标准目录结构和 POM 约定，让项目可以被命令行、IDE 和 CI 以同一种方式构建。

## 难点

> **难点：** 环境问题常常不是 Maven 单独的问题，而是 Maven、JDK、IDE、PATH、JAVA_HOME 和插件配置共同造成的。

## 练习

创建一个最小 Maven 项目，执行：

```bash
mvn clean package
```

观察：

- `target/classes` 里有什么？
- `target/*.jar` 是否生成？
- 删除 `target/` 后重新构建会发生什么？

## 验收

- 能解释 Maven 标准目录结构。
- 能独立创建最小 Maven 项目。
- 能说明 `target/` 为什么不应提交。
- 能检查 Maven 和 JDK 版本。
