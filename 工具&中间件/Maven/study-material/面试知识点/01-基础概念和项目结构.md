## 一、基础概念和项目结构

### 1. Maven 是什么？它解决了什么问题？

**参考答案：**

Maven 是 Java 项目的构建和依赖管理工具。它通过 POM 描述项目，通过坐标定位依赖，通过仓库获取构件，通过生命周期和插件执行编译、测试、打包、安装和部署。

它解决的问题包括：手工复制 jar 不可控、团队构建流程不一致、依赖版本难管理、CI 难复现、项目结构混乱。Maven 用标准目录结构、统一依赖坐标和生命周期，让项目可以用同一条命令在不同机器上构建。

> **重点：** Maven 不只是下载依赖，它还负责构建流程标准化。

### 2. Maven、JDK、IDE 的关系是什么？

**参考答案：**

JDK 提供 Java 编译器和运行环境，Maven 负责读取 POM 并调度构建流程，IDE 只是开发工具。Maven 编译 Java 时仍然依赖 JDK。IDE 中能运行不代表命令行 Maven 一定能构建成功，因为 IDE 使用的 JDK、Maven 使用的 JDK、compiler 插件配置可能不同。

> **易错：** 排查 Maven 编译问题时只看 IDE 配置，不看 `mvn -version` 输出里的 Java version。

### 3. Maven 标准目录结构有什么意义？

**参考答案：**

标准目录结构体现了“约定优于配置”。`src/main/java` 放生产代码，`src/main/resources` 放生产资源，`src/test/java` 放测试代码，`target/` 放构建产物。插件按这些默认路径工作，项目就不需要重复配置源码、资源和测试目录。

> **重点：** 标准目录结构让命令行、IDE 和 CI 可以用同一套规则构建项目。

### 4. `target/` 目录是什么？为什么不提交？

**参考答案：**

`target/` 是 Maven 构建输出目录，包含编译后的 class、测试结果、打包产物等。它可以通过源码和 POM 重新生成，不属于源代码。提交 `target/` 会污染仓库，增加冲突和无意义变更。
