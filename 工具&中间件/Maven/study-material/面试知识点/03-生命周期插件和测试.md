## 一、生命周期、插件和测试

### 1. 生命周期、阶段、插件、目标有什么区别？

**参考答案：**

生命周期是一组有顺序的构建流程，阶段是生命周期中的节点，插件是提供构建能力的组件，目标是插件中真正执行的任务。阶段本身不干活，绑定到阶段上的插件目标才执行具体工作。

> **重点：** phase 定义顺序，goal 执行任务。

### 2. `mvn package` 会做什么？

**参考答案：**

`mvn package` 会执行 default 生命周期从开始到 package 阶段的所有前置阶段，通常包括资源处理、编译、测试资源处理、测试编译、测试执行和打包。它不是只执行打包。

### 3. `mvn clean package` 是什么意思？

**参考答案：**

它先执行 clean 生命周期的 clean 阶段，删除旧的构建输出；然后执行 default 生命周期到 package 阶段，重新编译、测试并打包。

### 4. Surefire 和 Failsafe 有什么区别？

**参考答案：**

Surefire 通常运行单元测试，绑定到 test 阶段；Failsafe 通常运行集成测试，绑定到 integration-test 和 verify 阶段。单元测试强调快速反馈，集成测试可能依赖数据库、网络或外部服务，适合放在更靠后的验证阶段。

### 5. `-DskipTests` 和 `-Dmaven.test.skip=true` 有什么区别？

**参考答案：**

`-DskipTests` 跳过测试执行，但通常仍会编译测试代码；`-Dmaven.test.skip=true` 会跳过测试编译和执行。后者更激进，可能掩盖测试代码本身的编译问题。

> **易错：** 长期跳过测试会让构建质量失控。
