## 一、面试速记

- Maven 是 Java 构建和依赖管理工具。
- POM 是项目说明书。
- GAV 坐标定位构件。
- 仓库存放构件，本地仓库缓存依赖。
- 生命周期定义构建顺序。
- 插件目标真正执行任务。
- `mvn package` 会执行 package 前的阶段。
- `dependencyManagement` 管版本，不引入依赖。
- BOM 管理一组兼容依赖版本。
- 传递依赖会形成依赖树。
- 冲突通常最近路径优先，同级声明靠前优先。
- 聚合管构建范围，继承管配置复用。
- Reactor 管多模块构建顺序。
- Surefire 跑单元测试，Failsafe 跑集成测试。
- `install` 到本地仓库，`deploy` 到远程仓库。
- SNAPSHOT 可变，release 应稳定不可变。
- 排障先看阶段、插件、依赖树和 effective POM。

## 二、自测题

### 1. 为什么 `mvn package` 会运行测试？

因为 package 是 default 生命周期中的较后阶段，执行 package 会先执行前置阶段，其中包括 test 阶段。

### 2. 为什么 dependencyManagement 不会自动引入依赖？

因为它只提供版本和配置管理，相当于版本清单。真正进入 classpath 的依赖必须写在 dependencies 中。

### 3. 多模块项目为什么需要 Reactor？

因为模块之间可能有依赖关系，Maven 需要根据依赖图决定构建顺序，并支持裁剪构建范围。

### 4. 为什么不能把所有依赖放进父 POM 的 dependencies？

因为 dependencies 会被子模块继承，导致每个模块都获得无关依赖，增加 classpath 污染和冲突风险。

### 5. 为什么不建议生产依赖 SNAPSHOT？

SNAPSHOT 是可变版本，同一个版本号在不同时间可能解析到不同内容，导致构建不可复现。

### 6. Maven 排障第一步是什么？

先判断失败发生在哪个阶段和哪个插件目标，再分类为依赖、编译、测试、打包、部署、环境或网络问题。
