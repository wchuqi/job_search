## 一、私服、发布、CI 和排障

### 1. Maven 私服有什么作用？

**参考答案：**

私服是企业内部的构件仓库，常见如 Nexus、Artifactory。它可以代理中央仓库、缓存依赖、发布内部构件、管理权限、审计依赖和提升构建稳定性。团队和 CI 通常通过私服获取依赖，而不是直接访问所有外部仓库。

### 2. install 和 deploy 有什么区别？

**参考答案：**

`install` 把构件安装到本地仓库，供本机其他项目使用；`deploy` 把构件发布到远程仓库或私服，供团队共享。deploy 通常需要 `distributionManagement` 和 `settings.xml` 中的认证信息。

### 3. SNAPSHOT 和 release 有什么区别？

**参考答案：**

SNAPSHOT 是开发中的可变版本，可能被更新；release 是稳定发布版本，原则上应保持不可变。生产依赖应尽量使用 release，避免构建结果随 SNAPSHOT 变化。

### 4. Maven 构建失败如何排查？

**参考答案：**

先定位失败阶段和插件目标，再分类为依赖解析、插件解析、编译、测试、打包、部署、JDK、网络或认证问题。常用命令：

```bash
mvn -X package
mvn -e package
mvn dependency:tree
mvn help:effective-pom
mvn help:active-profiles
```

> **重点：** 不要一开始就删除整个 `.m2/repository`。

### 5. 为什么运行时报 `NoSuchMethodError` 可能和 Maven 依赖有关？

**参考答案：**

`NoSuchMethodError` 往往说明编译期和运行期加载的类版本不一致。Maven 依赖冲突、传递依赖覆盖、打包时依赖版本变化，都可能导致运行时加载了旧版本或不兼容版本。应使用 `mvn dependency:tree` 和运行包内容排查。

### 6. 私服账号密码应该放在哪里？

**参考答案：**

认证信息应放在用户级 `settings.xml`、CI secret 或密钥系统中，不应写进项目 POM。POM 会进入代码仓库，写入密码会造成敏感信息泄露。
