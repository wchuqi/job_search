# Maven 学习资料：生命周期、阶段、插件和目标

[返回索引](../Maven学习资料.md)

## 学习目标

- 理解 Maven 生命周期。
- 区分 lifecycle、phase、plugin、goal。
- 知道常用命令实际触发了什么。
- 能配置常用插件。

## 理论导读

Maven 生命周期像一条构建流水线，阶段规定顺序，但阶段本身不直接干活。真正执行任务的是插件目标，例如 compiler 插件负责编译，surefire 插件负责运行单元测试，jar 插件负责打 jar。生命周期告诉 Maven “走到哪一步”，插件目标告诉 Maven “这一步具体做什么”。

执行 `mvn package` 时，Maven 不只运行 package 阶段，而是从 default 生命周期的前置阶段一路执行到 package。也就是说，validate、compile、test、package 相关步骤都会按顺序发生。很多初学者以为 package 只是打包，忽略了它会先编译和测试。

## 核心心智模型

```text
lifecycle: default
  validate
  compile      -> maven-compiler-plugin:compile
  test         -> maven-surefire-plugin:test
  package      -> maven-jar-plugin:jar
  install
  deploy
```

## 三套生命周期

| 生命周期 | 作用 |
| --- | --- |
| clean | 清理构建输出 |
| default | 编译、测试、打包、安装、部署 |
| site | 生成项目站点文档 |

## 常见阶段

```bash
mvn validate
mvn compile
mvn test
mvn package
mvn install
mvn deploy
```

含义：

- `compile`：编译主代码。
- `test`：运行测试。
- `package`：打包。
- `install`：安装到本地仓库。
- `deploy`：发布到远程仓库。

## 插件目标

直接执行插件目标：

```bash
mvn dependency:tree
mvn help:effective-pom
mvn compiler:compile
```

`dependency:tree` 中：

- `dependency` 是插件前缀。
- `tree` 是 goal。

## 配置 compiler 插件

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.13.0</version>
            <configuration>
                <release>21</release>
            </configuration>
        </plugin>
    </plugins>
</build>
```

## 重点

> **重点：** phase 是生命周期中的阶段，goal 是插件中的任务。执行阶段时，Maven 会运行绑定到该阶段的插件目标。

## 难点

> **难点：** 同一个插件目标可以直接执行，也可以绑定到生命周期阶段中自动执行。

## 易错

> **易错：** `mvn clean package` 是先执行 clean 生命周期的 clean 阶段，再执行 default 生命周期到 package 阶段，不是一条单独生命周期。

## 练习

执行：

```bash
mvn -X package
```

观察日志中哪些插件目标被执行。

## 验收

- 能解释 lifecycle、phase、plugin、goal。
- 能说明 `mvn package` 会执行测试。
- 能配置 compiler 插件。
- 能直接执行 `dependency:tree`。
