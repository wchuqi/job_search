# Maven 学习资料：综合练习项目

[返回索引](../Maven学习资料.md)

## 学习目标

通过一个多模块订单系统练习 Maven 的核心能力：

- 父 POM 和多模块。
- dependencyManagement。
- 插件管理。
- 单元测试。
- 打包和安装。
- 依赖树排查。
- profile 和 CI 命令。

## 理论导读

练习 Maven 不能只跑 `mvn package`。真正有效的练习要让项目出现依赖、模块、测试、插件、资源和构建顺序。这样你才能看见 Maven 如何读取父 POM、如何构建 Reactor、如何解析依赖、如何执行生命周期。

这个练习项目故意设计成三层结构：api 提供接口和 DTO，service 实现业务，web 作为入口。它小到可以快速完成，大到足够覆盖企业项目中最常见的 Maven 场景。

## 项目结构

```text
maven-order-demo/
  pom.xml
  order-api/
    pom.xml
    src/main/java/
  order-service/
    pom.xml
    src/main/java/
    src/test/java/
  order-web/
    pom.xml
    src/main/java/
```

## 任务 1：创建父 POM

要求：

- `packaging` 为 `pom`。
- 声明三个模块。
- 管理 Java 版本。
- 管理 JUnit 版本。
- 管理 compiler 和 surefire 插件版本。

## 任务 2：创建 api 模块

定义：

```java
package com.example.order.api;

public record OrderRequest(String productId, int count) {
}
```

## 任务 3：创建 service 模块

依赖 api，并实现：

```java
package com.example.order.service;

import com.example.order.api.OrderRequest;

public class OrderService {
    public String create(OrderRequest request) {
        if (request.count() <= 0) {
            throw new IllegalArgumentException("count must be positive");
        }
        return "ORDER-" + request.productId();
    }
}
```

添加 JUnit 测试。

## 任务 4：创建 web 模块

依赖 service，创建入口类：

```java
package com.example.order.web;

import com.example.order.api.OrderRequest;
import com.example.order.service.OrderService;

public class App {
    public static void main(String[] args) {
        var service = new OrderService();
        System.out.println(service.create(new OrderRequest("book", 1)));
    }
}
```

## 任务 5：构建和排查

执行：

```bash
mvn clean test
mvn -pl order-web -am package
mvn dependency:tree
mvn help:effective-pom
```

## 扩展任务

- 给 service 添加一个外部依赖，并查看传递依赖。
- 用 dependencyManagement 管理版本。
- 添加 dev profile。
- 配置 shade 插件生成可执行 jar。
- 故意制造依赖冲突并排查。

## 重点

> **重点：** 综合项目要训练 Maven 的构建图思维，而不是只训练单个命令。

## 难点

> **难点：** 多模块项目中，模块依赖、父 POM 配置和 Reactor 顺序会同时影响构建结果。

## 易错

> **易错：** 只在 IDE 中运行成功，却没有验证命令行 `mvn clean test`。企业构建必须以命令行和 CI 可重复执行为准。

## 验收

- 能完成三模块项目。
- 能执行完整构建。
- 能解释父 POM 配置如何影响子模块。
- 能排查依赖树。
- 能用 `-pl -am` 构建指定模块。
