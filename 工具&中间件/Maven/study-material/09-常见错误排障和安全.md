# Maven 学习资料：常见错误、排障和安全

[返回索引](../Maven学习资料.md)

## 学习目标

- 按类别排查 Maven 构建失败。
- 使用常见诊断命令。
- 识别依赖安全风险。
- 避免破坏性排障方式。

## 理论导读

Maven 错误要先分类：依赖解析失败、插件解析失败、编译失败、测试失败、打包失败、部署失败、网络代理失败、JDK 不匹配、版本冲突。分类清楚后，命令选择才有意义。否则一上来删除整个本地仓库，可能只是掩盖问题，还会浪费大量时间重新下载依赖。

安全也是 Maven 学习的一部分。依赖会把第三方代码带进项目，传递依赖也可能带来漏洞。企业项目需要依赖锁定、漏洞扫描、私服治理、禁止不可信仓库、保护 deploy 凭据。

## 常见诊断命令

```bash
mvn -X package
mvn -e package
mvn help:effective-pom
mvn help:active-profiles
mvn dependency:tree
mvn dependency:analyze
mvn -U clean package
mvn -o test
```

| 命令 | 作用 |
| --- | --- |
| `-X` | debug 日志 |
| `-e` | 显示异常栈 |
| `-U` | 强制更新 SNAPSHOT |
| `-o` | 离线模式 |

## 常见错误

### 依赖下载失败

可能原因：

- 坐标或版本不存在。
- 仓库地址错误。
- 私服权限不足。
- 代理或网络问题。
- 本地缓存了损坏文件。

### 编译失败

可能原因：

- JDK 版本不匹配。
- `maven.compiler.release` 配置错误。
- 源码使用了更高版本 API。

### 测试失败

可能原因：

- 测试本身失败。
- Surefire 版本过旧。
- 测试依赖缺失。
- 环境依赖未准备。

### 运行时报错

构建成功但运行失败，常见原因：

- scope 错误。
- 传递依赖冲突。
- 打包方式没有包含运行依赖。
- 容器提供的依赖和项目依赖冲突。

## 安全检查

- 不使用不可信仓库。
- 不提交 settings.xml 中的凭据。
- 定期扫描依赖漏洞。
- 关注传递依赖。
- 避免长期使用过期插件。
- release 版本保持不可变。

## 重点

> **重点：** Maven 排障先定位阶段和类别，再选择命令。

## 难点

> **难点：** 构建成功不代表依赖无冲突。运行期错误往往来自 classpath 和依赖版本问题。

## 易错

> **易错：** 删除整个 `.m2/repository` 当作通用解决方案。更合理的是定位具体损坏构件或具体仓库配置问题。

## 练习

故意制造三个问题：

- 写错依赖版本。
- 设置过高 Java release。
- 添加冲突依赖。

分别用命令定位原因。

## 验收

- 能按类别排查 Maven 错误。
- 能使用 `-X`、`dependency:tree`、`effective-pom`。
- 能说明依赖安全风险。
- 能避免破坏性排障。
