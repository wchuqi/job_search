# Maven 学习资料：私服、发布、部署和 CI

[返回索引](../Maven学习资料.md)

## 学习目标

- 理解 Maven 私服的作用。
- 区分 install 和 deploy。
- 配置 distributionManagement。
- 理解 Maven 在 CI 中的常见流程。

## 理论导读

企业项目很少直接依赖公共中央仓库。私服像团队的构件中转站：代理外部依赖、缓存下载结果、保存内部 jar、控制权限、支持漏洞治理。开发者和 CI 从私服拿依赖，也把内部构件发布到私服。

`install` 和 `deploy` 经常被混淆。`install` 把当前项目产物安装到本机本地仓库，方便本机其他项目依赖；`deploy` 把产物发布到远程仓库，供团队或环境使用。一个影响本机，一个影响团队。

## distributionManagement

```xml
<distributionManagement>
    <repository>
        <id>releases</id>
        <url>https://repo.example.com/releases</url>
    </repository>
    <snapshotRepository>
        <id>snapshots</id>
        <url>https://repo.example.com/snapshots</url>
    </snapshotRepository>
</distributionManagement>
```

账号放在 `settings.xml`：

```xml
<servers>
    <server>
        <id>releases</id>
        <username>deploy-user</username>
        <password>***</password>
    </server>
</servers>
```

> **重点：** `server/id` 必须和 `distributionManagement` 中的 repository id 对应。

## SNAPSHOT 和 Release

`1.0.0-SNAPSHOT` 表示开发中的快照版本，可能被覆盖更新。`1.0.0` release 版本应保持不可变，适合作为稳定发布依赖。

## CI 常见流程

```text
checkout
  -> setup JDK
  -> cache ~/.m2/repository
  -> mvn clean verify
  -> security scan
  -> mvn deploy
```

## 常用命令

```bash
mvn clean verify
mvn install
mvn deploy
mvn -B clean deploy
```

`-B` 表示 batch mode，适合 CI。

## 重点

> **重点：** 私服是企业依赖治理和内部制品发布的核心基础设施。

## 难点

> **难点：** deploy 失败可能来自权限、仓库 URL、repository id、版本策略、网络代理或私服规则。

## 易错

> **易错：** 在项目 POM 中写明文账号密码。凭据应放在 settings.xml 或 CI secret 中。
