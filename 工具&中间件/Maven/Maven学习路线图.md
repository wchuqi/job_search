# Maven 学习路线图

这份路线图从 Maven 基础认知开始，逐步进入依赖管理、生命周期、插件、多模块、发布部署、排障和面试表达。

## 学习总原则

Maven 学习要建立四个核心画面：`pom.xml` 是项目说明书，坐标是构件地址，仓库是构件仓库，生命周期是构建流水线。每个命令都不是孤立动作，而是在这条流水线上触发某个阶段，并由绑定到阶段上的插件目标完成真实工作。

每学一个知识点，都按“它描述项目、解析依赖、执行构建、还是发布产物”来分类。能讲清分类，再记命令才稳定。

## 阶段 1：基础认知和环境

- 目标：安装 Maven，理解 Maven 解决什么问题。
- 需要掌握：Maven、JDK、`mvn`、本地仓库、标准目录结构。
- 练习：创建最小 Java 项目，执行 `mvn test` 和 `mvn package`。
- 验收：能解释 `pom.xml`、`target/`、本地仓库的作用。
- 重点：Maven 是构建系统，不只是依赖下载器。
- 易错：JDK 版本和 Maven 编译插件版本不匹配。

## 阶段 2：POM、依赖和仓库

- 目标：能读懂 POM，能添加依赖并排查下载问题。
- 需要掌握：GAV 坐标、packaging、dependencies、repositories、本地仓库、中央仓库。
- 练习：添加 JUnit 依赖，查看本地仓库缓存。
- 验收：能解释一个 jar 如何从远程仓库进入项目 classpath。
- 重点：坐标定位构件，仓库保存构件。
- 易错：把 repository、pluginRepository、distributionManagement 混为一谈。

## 阶段 3：生命周期和插件

- 目标：理解 `mvn compile`、`mvn test`、`mvn package` 为什么会执行一连串动作。
- 需要掌握：clean、default、site 生命周期；phase、goal、plugin 关系。
- 练习：配置 compiler、surefire、jar 插件。
- 验收：能说明 phase 不是插件，插件目标才是真正执行者。
- 重点：生命周期定义顺序，插件目标执行任务。
- 易错：以为 `package` 只执行打包，不知道它会先执行前置阶段。

## 阶段 4：依赖治理

- 目标：能处理依赖范围、传递依赖、版本冲突和 BOM。
- 需要掌握：scope、optional、exclusion、dependencyManagement、BOM、dependency tree。
- 练习：制造依赖冲突并用 `mvn dependency:tree` 排查。
- 验收：能解释 Maven 最近优先和声明优先规则。
- 重点：依赖管理是大型项目稳定性的基础。
- 易错：只升级直接依赖，忽略传递依赖和漏洞。

## 阶段 5：多模块和企业构建

- 目标：能维护父 POM、多模块项目和 CI 构建。
- 需要掌握：parent、modules、聚合、继承、Reactor、私服、deploy、release。
- 练习：创建 api、service、web 三模块项目。
- 验收：能解释聚合和继承的区别。
- 重点：多模块是构建编排，父 POM 是配置复用。
- 易错：模块间版本、构建顺序和相对路径混乱。

## 阶段 6：排障、安全和面试表达

- 目标：能排查 Maven 构建失败，并能在面试中讲清机制。
- 需要掌握：`-X`、`-U`、离线模式、镜像、代理、依赖安全、SNAPSHOT。
- 练习：排查依赖下载失败、测试失败、插件版本冲突。
- 验收：能从错误日志定位是依赖、插件、JDK、仓库还是网络问题。
- 重点：先分类问题，再执行命令。
- 易错：看到构建失败就删整个本地仓库，破坏排障现场。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 天 | 环境、项目结构、POM | 一个可运行的 Maven Java 项目 |
| 第 2 天 | 生命周期和插件 | 能解释 `mvn package` 执行链路 |
| 第 3 天 | 依赖范围和冲突 | 一份依赖树分析记录 |
| 第 4 天 | 多模块和父 POM | 一个三模块练习项目 |
| 第 5 天 | 私服、CI、排障 | 一份构建失败排查清单 |

## 最终能力清单

- 能从零创建 Maven 项目。
- 能读懂 POM 的核心元素。
- 能解释生命周期、阶段、插件、目标。
- 能处理依赖范围、传递依赖和版本冲突。
- 能维护多模块项目。
- 能配置常用插件。
- 能排查依赖下载、测试、编译、打包和部署失败。
- 能在面试中讲清 Maven 的底层模型和工程价值。
