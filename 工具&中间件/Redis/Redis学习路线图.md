# Redis学习路线图

## 阶段 1：基础认知

- 目标：理解 Redis 是内存数据结构服务器，不只是缓存。
- 需要掌握：key、value、database、TTL、String、Hash、List、Set、Sorted Set、redis-cli。
- 例子：用 `SET`、`GET`、`HSET`、`LPUSH`、`SADD`、`ZADD` 操作不同结构。
- 练习：用 Redis 实现验证码、计数器、排行榜和简单队列。
- 验收：能根据场景选择数据类型。
- 重点：Redis 命令通常是单线程事件循环中顺序执行，单条命令原子。
- 易错：把 Redis 当无限内存数据库，忽略 key 数量、value 大小、TTL 和淘汰策略。

## 阶段 2：缓存和应用模式

- 目标：能把 Redis 正确用于缓存、会话、限流、排行榜、分布式锁。
- 需要掌握：cache-aside、TTL、穿透、击穿、雪崩、双写一致性、锁过期、Lua 原子操作。
- 例子：查询缓存、热点 key 防击穿、令牌桶限流。
- 练习：设计商品详情缓存，包含空值缓存、互斥重建和随机 TTL。
- 验收：能解释缓存不一致的来源和可接受边界。
- 重点：缓存是性能和一致性的权衡，不是数据库的透明替代。
- 易错：删除缓存和更新数据库没有设计失败补偿。

## 阶段 3：持久化、高可用和分布式

- 目标：理解 RDB、AOF、复制、哨兵和 Cluster 的职责边界。
- 需要掌握：fork、COW、AOF fsync、AOF rewrite、PSYNC、replication backlog、Sentinel failover、hash slot、MOVED/ASK。
- 例子：搭建一主两从、Sentinel、三主三从 Cluster。
- 练习：模拟主节点宕机、从节点提升、Cluster 槽迁移。
- 验收：能说明 Redis 高可用不等于强一致。
- 重点：复制是异步为主，故障切换可能丢最近写入。
- 易错：把 Sentinel 或 Cluster 当成零丢失方案。

## 阶段 4：性能、内存和排障

- 目标：能定位慢命令、big key、hot key、内存碎片、阻塞、网络和持久化抖动。
- 需要掌握：SLOWLOG、LATENCY、INFO、MONITOR 风险、SCAN、MEMORY、CLIENT LIST、redis-benchmark。
- 例子：定位 `KEYS *` 阻塞、big hash、AOF rewrite 抖动。
- 练习：构造 big key，观察慢查询和内存占用。
- 验收：能写出 Redis 性能排障路径。
- 重点：Redis 快的前提是命令复杂度可控、数据结构合理、内存稳定。
- 易错：生产使用 `KEYS`、大范围 `LRANGE`、大集合 `SMEMBERS`。

## 阶段 5：机制级深入和生产治理

- 目标：能解释事件循环、对象编码、过期淘汰、复制、Cluster 和持久化的底层行为。
- 需要掌握：IO 多路复用、命令执行、SDS、dict、skiplist、listpack、过期字典、近似 LRU/LFU、fork COW、PSYNC、槽迁移、重定向。
- 例子：解释为什么 big key 会阻塞，为什么 fork 时内存峰值升高，为什么 Cluster 多 key 命令要求同槽。
- 练习：完成 28 的深度实验，写故障复盘。
- 验收：能从指标推断问题属于数据模型、命令复杂度、内存、持久化、复制、网络还是客户端。
- 重点：Redis 深度能力是把“快”拆解成事件循环、内存访问和命令复杂度。
- 易错：只背“Redis 单线程所以快”，不理解网络线程、后台线程、fork 和命令阻塞的边界。

## 阶段 6：架构级容量、客户端和灾备治理

- 目标：能设计 Redis 服务的容量、压测、客户端保护、灾备恢复和故障切换方案。
- 需要掌握：RESP、pipeline 背压、连接池、超时重试、key 模型容量估算、big key 在线拆分、hot key 治理、fencing token、Sentinel/Cluster 选举、RSS 和碎片、备份校验、SLA。
- 例子：解释一次客户端超时风暴如何从慢命令扩散为连接池耗尽、重试放大和数据库雪崩。
- 练习：完成 29 到 40 的容量规划、故障推演、客户端保护和恢复演练。
- 验收：能给出 Redis 上线前容量表、压测报告、降级策略、备份恢复 RTO/RPO 和客户端超时配置。
- 重点：生产 Redis 不是单实例命令题，而是客户端、网络、内存、数据模型、复制和业务降级共同组成的系统。
- 易错：只优化 Redis 服务器，不治理客户端重试、连接池和业务降级。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 数据类型、命令、TTL、事务 Lua | 基础命令实验 |
| 第 2 周 | 缓存模式、锁、限流、消息 | 业务场景设计 |
| 第 3 周 | RDB/AOF、复制、哨兵、Cluster | 高可用实验 |
| 第 4 周 | 性能、内存、安全、监控排障 | 生产排障手册 |
| 第 5 周 | 事件循环、对象编码、淘汰、复制协议 | 深度原理笔记 |
| 第 6 周 | 故障注入和容量规划 | 能力验收报告 |
| 第 7 周 | 客户端保护、big key 治理、锁和一致性工程 | 生产设计评审文档 |
| 第 8 周 | Sentinel/Cluster 选举、灾备、压测和 SLA | 上线验收和灾备演练报告 |

## 最终能力清单

- 能根据业务场景选择 Redis 数据结构和命令。
- 能设计缓存一致性、穿透、击穿、雪崩处理方案。
- 能说明 RDB、AOF、复制、哨兵、Cluster 的机制和边界。
- 能定位 big key、hot key、慢命令、内存碎片、阻塞和复制延迟。
- 能解释事件循环、对象编码、过期淘汰、fork COW 和 Cluster 槽迁移。
- 能写出生产配置、安全、备份、监控和故障恢复方案。
- 能设计 Redis 客户端超时、重试、连接池、限流和熔断策略。
- 能完成容量估算、压测验证、RTO/RPO 设计和灾备恢复演练。
