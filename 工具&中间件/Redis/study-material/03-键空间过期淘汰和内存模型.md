# Redis学习资料：键空间、过期、淘汰和内存模型

[返回索引](../Redis学习资料.md)

## 学习目标

- 理解 keyspace、TTL、过期删除和内存淘汰。
- 掌握 TTL 设计、maxmemory 和淘汰策略。
- 能解释缓存雪崩和内存打满的关系。

## 理论导读

Redis 的 key 存在于键空间字典中，TTL 信息存放在过期字典中。过期不是“到点立刻删除所有 key”，而是惰性删除和定期删除结合。内存达到 maxmemory 后，Redis 根据淘汰策略选择 key 淘汰。如果没有合适策略，写入可能失败。

## TTL 命令

```bash
SET session:1 value EX 3600
EXPIRE session:1 1800
TTL session:1
PERSIST session:1
```

TTL 返回：

- 正数：剩余秒数。
- `-1`：存在但无过期。
- `-2`：key 不存在。

## 过期删除

Redis 过期删除包含：

- 惰性删除：访问 key 时发现过期再删除。
- 定期删除：周期性抽样检查过期 key。

这意味着过期 key 可能在物理上短暂存在，但逻辑访问时会被视为不存在。

## 淘汰策略

常见策略：

- `noeviction`：不淘汰，写入报错。
- `allkeys-lru`：所有 key 中近似 LRU。
- `volatile-lru`：有 TTL 的 key 中近似 LRU。
- `allkeys-lfu`：所有 key 中近似 LFU。
- `volatile-ttl`：有 TTL 的 key 中优先淘汰 TTL 小的。
- `allkeys-random`、`volatile-random`：随机。

选择：

- 纯缓存：常用 `allkeys-lru` 或 `allkeys-lfu`。
- 混合缓存和重要数据：不建议混放，至少要用实例隔离。
- 不允许写失败：必须监控内存并设计淘汰。

## 内存模型

Redis 内存不仅包括 value：

- key 名。
- Redis object 元数据。
- 底层结构。
- allocator 开销。
- 复制 backlog。
- client output buffer。
- AOF/RDB rewrite COW 峰值。
- 内存碎片。

## 练习

1. 设置带 TTL 的 key，观察 `TTL`、过期后访问。
2. 设置 `maxmemory` 和淘汰策略，写入大量 key 观察淘汰。
3. 用 `INFO memory` 查看 used_memory、mem_fragmentation_ratio。

## 验收

- 能解释 TTL 返回 -1 和 -2。
- 能说明过期删除不是精确定时删除。
- 能根据业务选择淘汰策略。

## 重点

- TTL 是缓存治理核心。
- 淘汰策略不能替代容量规划。
- 内存峰值要考虑 fork COW 和客户端缓冲。

## 易错

> **易错：** 设置 TTL 后认为 key 到期瞬间一定释放内存。
>
> 正确做法：理解惰性删除和定期删除，并通过内存指标观察实际释放。
