# Redis学习资料：监控、排障、备份恢复和运维

[返回索引](../Redis学习资料.md)

## 学习目标

- 掌握 Redis 核心监控指标。
- 建立 Redis 排障路径。
- 设计备份、恢复、容量和变更流程。

## 理论导读

Redis 生产稳定性依赖持续观测。只看 CPU 和内存不够，还要看命令、延迟、连接、复制、持久化、淘汰、碎片、客户端缓冲和 keyspace。排障时先保留现场，再判断是客户端流量、命令复杂度、内存、持久化、复制还是网络。

## 核心指标

```bash
INFO server
INFO clients
INFO memory
INFO stats
INFO persistence
INFO replication
INFO commandstats
INFO keyspace
LATENCY LATEST
SLOWLOG GET 10
```

关注：

- used_memory、maxmemory、mem_fragmentation_ratio。
- connected_clients、blocked_clients。
- instantaneous_ops_per_sec。
- evicted_keys、expired_keys。
- keyspace_hits、keyspace_misses。
- rejected_connections。
- master_repl_offset、slave_repl_offset。
- aof_rewrite_in_progress、rdb_bgsave_in_progress。

## 排障路径

1. 是否所有命令慢，还是部分命令慢。
2. 是否有 big key/hot key。
3. 是否持久化或 fork 正在发生。
4. 是否内存接近上限或频繁淘汰。
5. 是否客户端连接数、输出缓冲异常。
6. 是否复制延迟、网络抖动。
7. 是否系统 CPU、磁盘、内存压力。

## 备份恢复

- RDB 文件适合备份。
- AOF 提供更小丢失窗口。
- 备份必须异地保存。
- 恢复必须定期演练。
- Cluster 恢复要考虑每个分片。

## 变更原则

- 配置变更先测试。
- 大规模删除使用异步或分批。
- 扩容迁移避开高峰。
- 持久化策略调整要评估 IO 和内存峰值。

## 练习

1. 写一份 Redis 巡检脚本，输出 memory、clients、slowlog、replication。
2. 备份 RDB 并在新实例恢复。
3. 构造慢命令并通过 SLOWLOG 找到。

## 验收

- 能列出核心 INFO 指标含义。
- 能写 Redis 排障顺序。
- 能设计备份恢复演练。

## 重点

- 监控要覆盖命令、内存、客户端、持久化和复制。
- 备份没有恢复演练就不算可靠。
- 排障命令不要制造更大压力。

## 易错

> **易错：** 只看 Redis 进程是否存活。
>
> 正确做法：看延迟、命令、内存、淘汰、复制、持久化和客户端行为。
