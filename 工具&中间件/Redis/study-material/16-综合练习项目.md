# Redis学习资料：综合练习项目

[返回索引](../Redis学习资料.md)

## 项目目标

设计一个“商品详情缓存 + 排行榜 + 限流 + Stream 订单事件”的 Redis 实战项目，覆盖缓存一致性、热点保护、Lua、Stream、监控和故障处理。

## 功能模块

- 商品详情缓存：cache-aside、空值缓存、随机 TTL。
- 热点商品：互斥重建或逻辑过期。
- 浏览量计数：`INCR`。
- 销量排行榜：ZSet。
- 接口限流：Lua + ZSet 滑动窗口。
- 订单事件：Stream 消费组。
- 监控：slowlog、memory、keyspace。

## 阶段 1：数据建模

### 练习

设计 key：

```text
product:{id}:detail
product:{id}:lock
rank:sales:daily
rate:{user_id}:{api}
stream:orders
```

### 验收

- 每类 key 有 TTL 策略。
- 说明哪些 key 可能成为 hot key。

## 阶段 2：缓存一致性

### 练习

实现读缓存流程：

1. 查 Redis。
2. 未命中查数据库。
3. 不存在写空值短 TTL。
4. 存在写缓存随机 TTL。

### 验收

- 说明穿透、击穿、雪崩防护。
- 更新商品时删除缓存并设计失败补偿。

## 阶段 3：Lua 限流

### 练习

用 Lua 实现滑动窗口：

- 删除窗口外请求。
- 添加当前请求。
- 统计窗口内请求。
- 超限返回拒绝。
- 设置过期。

### 验收

- 并发下结果正确。
- key 能自动清理。

## 阶段 4：Stream 消息

### 练习

- `XADD` 写订单事件。
- 消费组读取。
- 成功 `XACK`。
- 模拟消费者崩溃，处理 pending。

### 验收

- 能解释重复消费和幂等。
- 监控 pending 数量。

## 最终验收

- 有 key 设计文档。
- 有缓存一致性流程图。
- 有限流 Lua 脚本。
- 有 Stream 消费和重试。
- 有 big key/hot key 风险说明。
- 有 Redis 指标监控清单。

## 重点

- 实战项目要体现 Redis 的边界，而不是只堆命令。
- 每个 key 都要有生命周期和容量预估。
