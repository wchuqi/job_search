# Redis学习资料：持久化损坏恢复、备份校验和灾备演练深度解析

[返回索引](../Redis学习资料.md)

## 学习目标

- 设计 RDB/AOF 备份、校验、恢复和灾备演练。
- 掌握 AOF 损坏、RDB 损坏、误删和全实例故障的处理思路。
- 明确 RTO/RPO。

## 理论导读

备份不是把文件复制出来就结束。可靠备份必须可校验、可恢复、可演练、可追溯。Redis 的 RDB/AOF 恢复还要考虑版本兼容、数据时点、Cluster 分片、ACL/config、业务回源和缓存预热。

## RTO/RPO

- RTO：恢复服务需要多久。
- RPO：最多允许丢多少数据。

不同业务的 Redis 角色不同：

- 纯缓存：RPO 可较大，RTO 关注预热。
- 会话：RPO/RTO 中等。
- 准数据存储：需要更严格，但应考虑是否该用更强存储。

## 备份流程

```text
BGSAVE
  -> 复制 RDB 到备份目录
  -> 校验 checksum/大小/时间
  -> 上传异地
  -> 在隔离实例恢复验证
```

AOF 也应备份并验证能加载。

## AOF 损坏

故障断电可能导致 AOF 尾部不完整。处理思路：

- 停止实例。
- 备份原 AOF。
- 使用修复工具或截断尾部损坏部分。
- 在隔离环境验证。
- 再决定是否用于生产恢复。

## 误删恢复

误删缓存：

- 可从数据库回源和预热。

误删 Redis 中作为状态的数据：

- 从备份恢复到隔离实例。
- 按 key 或时间点筛选恢复。
- 评估覆盖风险。

## Cluster 灾备

Cluster 要按分片备份。恢复时要保证 slot 拓扑、节点配置、数据文件和客户端路由一致。灾备演练比单节点复杂。

## 练习

1. 备份 RDB 到新实例恢复。
2. 模拟 AOF 尾部损坏并在隔离环境修复。
3. 写 Redis 灾备演练记录，包括 RTO/RPO。

## 验收

- 能区分备份、校验、恢复演练。
- 能写出 AOF 损坏处理流程。
- 能设计缓存预热和回源保护。

## 易错

> **易错：** 从未恢复过的备份被认为可靠。
>
> 正确做法：定期在隔离环境恢复，并记录恢复耗时和数据校验结果。
