## 一、安全

### 1. Redis 可以暴露公网吗？

**参考答案：**

不应暴露公网。Redis 应放在内网或受控网络，通过安全组、防火墙、认证、ACL 和 TLS 保护。密码不能替代网络隔离。

### 2. ACL 有什么作用？

**参考答案：**

ACL 可限制用户可访问的 key pattern 和可执行命令。应用账号应最小权限，不应拥有 `FLUSHALL`、`CONFIG`、`DEBUG` 等管理权限。

### 3. 生产 Redis 要监控哪些指标？

**参考答案：**

内存、碎片、连接数、阻塞客户端、OPS、命中率、淘汰数、过期数、慢查询、延迟、复制延迟、持久化状态、客户端输出缓冲、keyspace。

### 4. Redis 备份怎么做？

**参考答案：**

根据持久化策略备份 RDB/AOF，备份应异地保存并定期恢复演练。Cluster 要按分片考虑。备份文件包含敏感数据，也需要访问控制。

### 5. 哪些命令生产要慎用？

**参考答案：**

`KEYS`、`MONITOR`、`FLUSHALL`、`FLUSHDB`、`CONFIG SET`、`DEBUG`、大集合 `SMEMBERS`、大范围 `LRANGE`、大范围 `ZRANGE` 等。
