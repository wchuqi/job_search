## 一、数据结构

### 1. Redis 有哪些常用数据类型？分别适合什么？

**参考答案：**

String 适合缓存值、计数器、token；Hash 适合对象字段；List 适合简单队列和列表；Set 适合去重和集合运算；Sorted Set 适合排行榜和延迟队列；Stream 适合带消费组的消息。

### 2. Hash 和 String JSON 存对象怎么选？

**参考答案：**

整体读取、整体更新、对象较小可用 String JSON；字段级更新、局部读取、计数累加更适合 Hash。但 Hash 字段过多也会成为 big key。

### 3. ZSet 为什么适合排行榜？

**参考答案：**

ZSet 每个 member 有 score，支持按 score 排序、排名查询和范围查询。底层大结构通常结合 dict 和 skiplist，既能快速查 member，又能按 score 排序。

### 4. Redis big key 是什么？有什么危害？

**参考答案：**

big key 指 value 很大或集合元素很多的 key。危害包括命令执行阻塞、网络传输大、复制和持久化压力、删除阻塞、迁移困难。

### 5. `KEYS` 和 `SCAN` 有什么区别？

**参考答案：**

`KEYS` 一次扫描整个 keyspace，可能阻塞 Redis；`SCAN` 是游标式增量扫描，每次返回部分结果，更适合生产排查，但可能重复返回，需要客户端处理。

> **易错：** SCAN 不是一致性快照。
