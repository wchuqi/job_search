## 一、缓存问题

### 1. 缓存穿透、击穿、雪崩有什么区别？

**参考答案：**

穿透是查询不存在数据导致请求绕过缓存打数据库；击穿是热点 key 过期瞬间大量请求打数据库；雪崩是大量 key 同时失效或 Redis 故障导致数据库承压。

### 2. 如何解决缓存穿透？

**参考答案：**

参数校验、空值缓存、Bloom Filter、限流和熔断。空值缓存 TTL 应较短，避免长期缓存不存在状态。

### 3. 如何解决缓存击穿？

**参考答案：**

互斥锁重建、逻辑过期异步刷新、热点 key 主动续期、本地缓存和请求合并。

### 4. 更新数据库和缓存的一致性怎么做？

**参考答案：**

常见是先更新数据库，再删除缓存，并通过重试、消息补偿、延迟双删、binlog 订阅等方式处理删除失败。目标通常是最终一致，不是强一致。

### 5. Redis 分布式锁的正确释放方式是什么？

**参考答案：**

加锁使用 `SET key value NX PX ttl`，value 是唯一 token；释放时用 Lua 校验 value 匹配后删除，避免误删其他客户端的新锁。

> **易错：** 直接 `DEL lock`。
