## 一、性能

### 1. Redis 慢了怎么排查？

**参考答案：**

先看 SLOWLOG、LATENCY、INFO commandstats，判断是否慢命令；再查 big key/hot key、客户端连接和输出缓冲、内存淘汰、持久化 rewrite、复制延迟、网络和系统资源。

### 2. SLOWLOG 没有慢命令，为什么客户端仍觉得慢？

**参考答案：**

SLOWLOG 记录命令执行时间，不包含网络传输、客户端排队、连接池等待和大结果返回耗时。还要看网络、客户端、输出缓冲和延迟指标。

### 3. big key 怎么发现和处理？

**参考答案：**

用 `redis-cli --bigkeys`、`MEMORY USAGE`、SCAN 抽样等发现。处理方式包括拆分 key、限制元素数量、分批读取、异步删除、避免全量返回。

### 4. hot key 怎么处理？

**参考答案：**

本地缓存、多副本读、key 拆分、请求合并、限流、热点探测和预热。核心是降低单节点单 key 的访问集中度。

### 5. pipeline 有什么风险？

**参考答案：**

pipeline 减少网络 RTT，但批量过大可能导致服务端输入/输出缓冲膨胀、单次延迟升高和内存压力。需要控制批量大小。
