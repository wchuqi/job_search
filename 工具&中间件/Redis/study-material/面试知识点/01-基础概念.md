## 一、基础概念

### 1. Redis 是什么？

**参考答案：**

Redis 是内存数据结构服务器，支持 String、Hash、List、Set、Sorted Set、Stream 等结构，常用于缓存、计数器、排行榜、限流、分布式锁、会话和消息。它不是简单的 key-value 字符串存储。

> **重点：** Redis 的核心是“内存 + 数据结构 + 事件循环 + 持久化/复制”。

### 2. Redis 为什么快？

**参考答案：**

主要原因包括数据在内存中、常用命令复杂度低、核心命令执行路径避免多线程锁竞争、IO 多路复用处理连接、协议简单、底层结构优化。不能只回答“单线程所以快”。

### 3. Redis 的单线程是什么意思？

**参考答案：**

通常指核心命令执行路径以单线程串行执行为主，单条命令执行期间不会被其他命令插入。但 Redis 仍可能有后台线程、后台子进程、网络 IO 线程等。单线程降低锁竞争，也意味着慢命令会阻塞其他请求。

### 4. Redis 和数据库有什么区别？

**参考答案：**

Redis 通常用于高性能内存访问和短生命周期数据，关系型数据库更适合强事务、复杂查询和长期持久化。Redis 可持久化，但不应替代核心强一致数据库，尤其是账务类数据。

### 5. Redis 单条命令为什么是原子的？

**参考答案：**

因为命令在服务端核心执行路径中串行执行，某条命令执行期间不会穿插执行其他客户端命令。但多个命令组合不是天然原子，需要事务、Lua 或业务幂等。
