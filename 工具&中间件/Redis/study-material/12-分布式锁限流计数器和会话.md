# Redis学习资料：分布式锁、限流、计数器和会话

[返回索引](../Redis学习资料.md)

## 学习目标

- 掌握 Redis 分布式锁的正确基本写法和边界。
- 使用 Redis 实现计数器、滑动窗口、令牌桶、会话。
- 理解锁过期、续期、误删和时钟问题。

## 理论导读

Redis 常用于分布式协调，但这类场景对原子性、过期、故障和时序要求高。一个看似简单的锁，如果没有唯一 value、过期时间和 Lua 校验删除，就可能误删别人的锁。限流和计数器也要考虑窗口边界、并发、内存增长和清理。

## 分布式锁

加锁：

```bash
SET lock:order:1 token-uuid NX PX 30000
```

释放锁必须校验 value：

```lua
if redis.call("GET", KEYS[1]) == ARGV[1] then
  return redis.call("DEL", KEYS[1])
else
  return 0
end
```

基本要求：

- `NX` 防止覆盖。
- 过期时间防止死锁。
- value 唯一，释放时校验。
- 业务耗时可能超过过期时间时要续期或缩短临界区。

## 锁边界

Redis 锁适合避免重复执行、降低并发重建，不适合作为金融级强一致锁。主从故障切换、网络分区、客户端 GC 暂停都可能造成锁语义问题。

## 计数器

```bash
INCR pv:2026-06-06
EXPIRE pv:2026-06-06 86400
```

首次设置 TTL 可用 Lua 保证 INCR 和 EXPIRE 组合逻辑。

## 固定窗口限流

```bash
INCR rate:user:1:202606061820
EXPIRE rate:user:1:202606061820 60
```

缺点是窗口边界可能突刺。

## 滑动窗口

用 ZSet 存请求时间戳：

```bash
ZREMRANGEBYSCORE rate:user:1 0 now-window
ZADD rate:user:1 now request-id
ZCARD rate:user:1
EXPIRE rate:user:1 60
```

需要 Lua 保证原子。

## 会话

```bash
HSET session:token user_id 1 login_at 1710000000
EXPIRE session:token 7200
```

会话要考虑续期、踢下线、并发登录和安全退出。

## 练习

1. 实现 value 校验释放锁。
2. 实现固定窗口限流。
3. 用 ZSet + Lua 实现滑动窗口限流。

## 验收

- 能说明 `SETNX` 后单独 `EXPIRE` 的问题。
- 能解释释放锁为什么要校验 value。
- 能比较固定窗口和滑动窗口。

## 重点

- Redis 锁是工程折中，不是绝对分布式一致性工具。
- 限流要考虑窗口边界和内存清理。
- 原子组合逻辑优先用 Lua。

## 易错

> **易错：** `DEL lock` 直接释放锁。
>
> 正确做法：用唯一 token 加 Lua 校验 value 后删除，避免删掉别人新获得的锁。
