# 正则表达式学习路线图

## 阶段 1：基础认知

- 目标：理解正则是“描述字符串模式”的语言，不是万能文本解析器。
- 需要掌握：普通字符、转义、字符类、预定义字符类、点号。
- 例子：用 `\d{4}-\d{2}-\d{2}` 匹配日期形态。
- 练习：匹配手机号、日期、简单邮箱。
- 验收：能说明 `.`、`\d`、`\w`、`\s`、`[]` 的含义和边界。
- 重点：先匹配形态，再考虑语义校验。
- 易错：把正则当成完整语义校验工具。

## 阶段 2：结构表达

- 目标：掌握量词、分组、选择、捕获和引用。
- 需要掌握：`*`、`+`、`?`、`{m,n}`、`()`、`(?:)`、`|`、反向引用。
- 例子：匹配 HTML 标签成对名称：`<([a-z]+)>.*?</\1>`。
- 练习：提取 URL 中协议、域名、路径。
- 验收：能区分捕获组和非捕获组。
- 难点：选择分支和量词组合会影响匹配路径。

## 阶段 3：位置和上下文

- 目标：掌握锚点、边界、断言和模式修饰符。
- 需要掌握：`^`、`$`、`\b`、`(?=)`、`(?!)`、`(?<=)`、`(?<!)`、多行、忽略大小写、dotAll。
- 例子：匹配后面跟 `元` 的数字，但不包含 `元`。
- 练习：提取日志中 level 后面的 message。
- 验收：能解释断言“匹配位置但不消耗字符”。

## 阶段 4：引擎机制和性能

- 目标：理解回溯型正则引擎如何工作，以及为什么会灾难性回溯。
- 需要掌握：NFA、DFA、回溯、贪婪、惰性、占有量词、原子组、ReDoS。
- 例子：`^(a+)+$` 遇到 `aaaaaaaaab` 为什么危险。
- 练习：重写一个灾难性回溯表达式。
- 验收：能识别嵌套量词和重叠分支风险。
- 重点：能匹配不代表能安全运行。

## 阶段 4.5：深度机制和复杂度审计

- 目标：能画出正则执行轨迹，并从路径数量判断复杂度风险。
- 需要掌握：选择点、回溯栈、捕获组状态恢复、输入拆分组合、慢失败样本构造。
- 例子：手动推演 `a.*b`、`(a|ab)c`、`^(a+)+$` 的执行过程。
- 练习：为一个高风险正则构造攻击输入，并写出改写方案。
- 验收：能解释 ReDoS 风险的三个条件：可控输入、大量可选路径、靠后失败点。
- 难点：从“表达式含义”切换到“引擎执行路径”。

## 阶段 5：工程实践

- 目标：把正则用于日志解析、表单校验、文本清洗、替换和分割。
- 需要掌握：命名组、替换引用、测试用例、可读性、跨语言差异。
- 例子：解析 Nginx 日志中的 IP、时间、方法、路径、状态码。
- 练习：构建一个文本清洗脚本。
- 验收：正则有正例、反例、边界例和性能测试。

## 阶段 6：生产评审和跨语言迁移

- 目标：能评审生产正则，并在 Java、JavaScript、Python、PCRE、RE2 之间迁移时识别风险。
- 需要掌握：源码转义、引擎特性、API 语义、Unicode、替换引用、路径和 URL 安全边界。
- 例子：把 JavaScript 命名组日期正则迁移到 Java，并说明转义和 API 差异。
- 练习：评审一个路径白名单正则，列出安全绕过风险和替代方案。
- 验收：能写出生产正则设计说明和测试样本集。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 字符类、量词、分组 | 常用匹配练习 |
| 第 2 周 | 锚点、断言、替换 | 文本提取和清洗 |
| 第 3 周 | 引擎机制和性能 | ReDoS 风险识别 |
| 第 4 周 | 方言差异和实战项目 | 日志解析和测试集 |
| 第 5 周 | 执行轨迹和复杂度审计 | 慢失败样本和改写报告 |
| 第 6 周 | 生产评审和跨语言迁移 | 正则设计说明和案例评审 |

## 最终能力清单

- 能写常用匹配、提取、替换正则。
- 能解释贪婪、惰性、占有量词。
- 能说明捕获组、非捕获组、命名组和反向引用。
- 能识别灾难性回溯和 ReDoS。
- 能处理 Java、JavaScript、Python 等方言差异。
- 能为正则编写测试和性能保护。
- 能画出复杂正则的执行轨迹。
- 能构造 ReDoS 慢失败样本并提出修复。
- 能评审安全场景中的正则边界。
- 能处理跨语言迁移时的源码、引擎和 API 差异。
