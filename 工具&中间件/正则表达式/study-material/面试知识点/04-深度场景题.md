## 一、深度场景题

### 1. 给你 `^(a+)+$`，你会如何向面试官解释它的风险？

**参考答案：**

这个表达式的风险在于嵌套量词。内层 `a+` 可以匹配一段或多段 `a`，外层 `(...)+` 又允许这段重复。对一串 `aaaa...` 来说，同一段输入可以被拆成很多组合，例如 `aaaa` 可以拆成 `aaaa`、`aaa+a`、`aa+aa`、`a+a+aa` 等。

如果输入是 `aaaaaaaa!`，前面的 `a` 看起来都能匹配，直到最后的 `!` 让 `$` 失败。回溯型引擎会尝试大量拆分方式，输入长度增长时尝试次数可能指数增长。这就是灾难性回溯，也是 ReDoS 的典型来源。

修复方式要看真实需求。如果只是匹配全是 `a` 的字符串，改成：

```regex
^a+$
```

如果真实需求是重复某个更复杂 token，应让 token 的边界明确，避免内外层量词竞争同一输入。

> **重点：** 风险来自“同一输入片段有大量等价拆分方式”，不是来自 `+` 这个符号本身。
>
> **易错：** 只说“嵌套量词不好”，但讲不出为什么慢。

### 2. 线上接口因为正则导致 CPU 飙高，你如何排查？

**参考答案：**

我会先从请求路径和输入样本入手，确认是否有用户可控的长输入触发某个正则。然后定位最近变更的正则或高频执行的正则，检查是否有嵌套量词、重叠分支、多个 `.*`、复杂断言、反向引用和靠后失败点。

排查步骤：

1. 从 CPU profile 或慢日志中找到调用点。
2. 记录触发输入长度和样本形态。
3. 在隔离环境复现匹配耗时。
4. 构造更短的最小慢失败样本。
5. 画出量词、分支和失败点。
6. 优先限制输入长度或临时下线规则。
7. 再改写表达式或替换为 parser/字符串 API。
8. 补充性能回归测试。

> **重点：** 线上止血优先于优雅改写，先限制输入长度、加超时或关闭高风险规则。

### 3. 如何评审一个用于权限白名单的路径正则？

**参考答案：**

权限白名单不能只看正则是否匹配几个样本。路径安全要考虑 URL 解码、路径标准化、大小写、重复斜杠、反斜杠、`..`、编码绕过和路由系统实际行为。

评审重点：

- 正则运行在解码前还是解码后。
- 是否已经做路径标准化。
- 是否有 `^` 和 `$` 或等价 fullmatch。
- 是否允许 `.*` 这种过宽片段。
- 是否和真实路由匹配规则一致。
- 是否有反例测试，例如 `/admin2`、`/%61dmin`、`/admin/../public`。

更稳妥的方案是：先用标准 URL/path 工具解析和规范化，再用路由或权限模型判断。正则最多作为辅助条件。

> **难点：** 安全边界不应建立在未规范化字符串的正则匹配上。

### 4. 为什么 `".*?"` 不一定是提取字符串的好方案？

**参考答案：**

`".*?"` 的意思是从一个双引号开始，尽量少地匹配任意字符，再遇到下一个双引号结束。它的问题是规则过于模糊：

- `.` 是否匹配换行依赖 dotAll。
- 它不能正确处理转义引号。
- 它没有定义字符串内部允许什么。
- 在复杂输入中仍可能回溯。

如果要匹配支持转义的双引号字符串，更明确的写法是：

```regex
"(?:\\.|[^"\\])*"
```

这个表达式把字符串内部拆成两类互斥 token：转义字符 `\\.`，或不是双引号也不是反斜杠的普通字符 `[^"\\]`。

> **重点：** 好的正则应描述合法 token，而不是用 `.*?` 模糊扫到下一个边界。

### 5. 你会如何把一个复杂正则变得可维护？

**参考答案：**

我会先确认它是否真的应该用正则。如果是复杂嵌套格式或标准格式，优先换 parser。如果确实适合正则，会做这些处理：

- 使用命名组代替编号组。
- 不需要捕获时改成非捕获组。
- 拆分成多个命名片段，在代码中组合。
- 使用 verbose/comment 模式，如果目标语言支持。
- 为正例、反例、边界例和性能例写测试。
- 在文档里写清楚输入来源、最大长度、目标引擎、已知不处理场景。
- 避免 `.*`、重叠分支和嵌套量词。

> **重点：** 可维护正则不是更短，而是意图、边界和测试更明确。

### 6. 如果一个正则在 JavaScript 能用，迁移到 Python/Java 时要检查什么？

**参考答案：**

要检查三层：

源码层：字符串转义是否变化。Java 字符串里 `\d` 要写成 `"\\d"`；Python 推荐原始字符串 `r"\d"`。

引擎层：目标语言是否支持原来的特性，例如 lookbehind、命名组、Unicode 属性、原子组、占有量词。

API 层：调用方法是否等价。Python `match()` 从开头匹配，不等于 JavaScript 的 `test()` 任意位置查找；Java `matches()` 是整体匹配，`find()` 才是查找。

还要检查替换引用语法和 Unicode 行为。

> **易错：** 只迁移表达式文本，不迁移调用语义。
