## 一、基础概念

### 1. 正则表达式是什么？适合解决什么问题？

**参考答案：**

正则表达式是一种描述字符串模式的语言，适合做规则文本的查找、匹配、提取、替换、分割和简单格式校验。它适合局部、线性、结构较稳定的文本，例如日志字段、日期形态、标识符、简单表单格式。

它不适合完整解析复杂嵌套结构，也不适合承担全部业务语义校验。例如完整 HTML、JSON、CSV 应使用 parser；日期是否合法应交给日期库；邮箱是否真实存在应交给业务流程验证。

> **重点：** 正则负责“形态和模式”，复杂语义应由专用解析器或代码处理。
>
> **易错：** 把正则当成万能解析器。

### 2. 匹配、查找、提取、替换和校验有什么区别？

**参考答案：**

匹配通常指判断某段文本是否符合模式；查找是在文本中找到符合模式的片段；提取是把匹配结果中的字段取出来；替换是把匹配片段改写成新内容；校验通常要求整个输入满足规则。

例如 `\d{4}-\d{2}-\d{2}` 可以在 `abc2026-06-06xyz` 中查找日期片段；而 `^\d{4}-\d{2}-\d{2}$` 更适合全量校验日期形态。

> **重点：** 表单校验通常要整体匹配，日志处理通常是局部查找或提取。
>
> **易错：** 用查找式 API 做全量校验。

### 3. 为什么正则不能完整校验所有业务规则？

**参考答案：**

正则擅长表达字符层面的模式，不擅长表达复杂语义、跨字段依赖、外部状态和无限嵌套结构。比如 `2026-99-99` 符合日期形态，但不是合法日期；邮箱形态合法也不代表邮箱存在；优惠码是否有效需要查数据库。

工程上通常让正则做第一层形态过滤，再由业务代码、数据库、日期库或 parser 做语义校验。

> **难点：** “格式像”不等于“语义真”。

### 4. 什么是正则方言？

**参考答案：**

正则方言指不同语言、库和引擎对正则语法和行为的不同实现。Java、JavaScript、Python、PCRE、.NET、RE2 的特性不完全一样，例如 lookbehind、命名组、占有量词、原子组、Unicode 字符类、替换引用语法都可能不同。

迁移正则时要检查三层：源码字符串转义、正则引擎语法、匹配/替换 API 行为。

> **重点：** 正则不是完全跨语言通用。
>
> **易错：** 从在线测试器复制后不在目标语言里验证。
