# 正则表达式学习资料：不同语言和正则方言差异

[返回索引](../正则表达式学习资料.md)

## 学习目标

- 理解正则语法不是完全跨语言通用。
- 掌握 Java、JavaScript、Python、PCRE、.NET、RE2 的常见差异。
- 能迁移正则时检查特性兼容性。
- 能避免字符串字面量、Unicode、替换引用导致的迁移问题。

> 深入阅读：跨语言迁移和线上排查时，继续读 [17-正则方言深度对照.md](17-正则方言深度对照.md)，按源码层、引擎层、API 层拆解差异。

## 理论导读

“这个正则在网站上能跑”不等于在项目中能跑。正则方言差异来自三层：语言字符串字面量、正则引擎语法、正则 API 行为。迁移时需要同时检查这三层，否则最常见的问题是转义错、模式修饰符错、组引用错、Unicode 语义错。

## 核心心智模型

正则落地前经过三道门：

```text
源码字符串 -> 正则编译器 -> 匹配/替换 API
```

任何一道门的规则不同，最终行为都会不同。

## 知识点详解

### 字符串字面量

Java：

```java
Pattern.compile("\\d+");
```

JavaScript 字面量：

```javascript
/\d+/
```

JavaScript 构造器：

```javascript
new RegExp("\\d+")
```

Python 原始字符串：

```python
re.compile(r"\d+")
```

### 特性支持差异

| 特性 | Java | JavaScript | Python re | PCRE | .NET | RE2 |
| --- | --- | --- | --- | --- | --- | --- |
| 命名组 | 支持 | 现代环境支持 | 支持 | 支持 | 支持 | 支持程度依实现 |
| 反向引用 | 支持 | 支持 | 支持 | 支持 | 支持 | 通常不支持 |
| lookahead | 支持 | 支持 | 支持 | 支持 | 支持 | 通常不支持 |
| lookbehind | 支持 | 现代环境支持 | 固定长度为主 | 支持较强 | 支持较强 | 通常不支持 |
| 占有量词 | 支持 | 不常规支持 | 标准 re 不支持 | 支持 | 部分方式支持 | 不支持 |
| 原子组 | 支持 | 不常规支持 | 新版本支持情况需确认 | 支持 | 支持 | 不支持 |

实际项目中应以当前运行时文档和测试为准。

### API 行为差异

同样是“匹配”，API 可能不同：

- Java `matches()` 要求整体匹配。
- Java `find()` 查找下一个片段。
- Python `re.match()` 从开头匹配，`re.search()` 查找任意位置，`fullmatch()` 整体匹配。
- JavaScript `test()` 查找是否存在匹配；带 `g` 标志时还会受 `lastIndex` 状态影响。

### Unicode 差异

`\w`、`\b`、大小写折叠、字符属性类在不同引擎中差异明显。处理多语言文本时，应优先使用明确的 Unicode 属性类或专用分词/校验逻辑。

### 替换引用差异

| 场景 | 常见写法 |
| --- | --- |
| JavaScript | `$1`、`$<name>` |
| Java | `$1`、`${name}` |
| Python | `\1`、`\g<name>` |
| .NET | `$1`、`${name}` |

## 例子

Python 中：

```python
re.match(r"\d+", "abc123")   # 失败
re.search(r"\d+", "abc123")  # 成功
```

JavaScript 中：

```javascript
/\d+/.test("abc123") // true
```

## 练习

1. 把 JavaScript 正则 `/^\d+$/` 迁移到 Java 字符串。
2. 解释 Python `match`、`search`、`fullmatch` 的区别。
3. 找出一个使用 lookbehind 的表达式，判断 RE2 是否能支持。
4. 比较 `$1` 和 `\1` 在匹配与替换中的语义。

## 验收

- 能说出源码字符串和正则语法的区别。
- 能列出至少 5 个常见方言差异。
- 能迁移一个简单正则到 Java、JavaScript、Python。
- 能解释 API 对匹配范围的影响。

## 重点

- 正则跨语言迁移必须重新测试。
- API 行为和正则语法同等重要。
- Unicode 文本处理不要默认相信 `\w` 和 `\b`。

## 难点

- 同一个标志位在不同语言中名字或行为可能不同。
- 全局匹配 API 可能有隐藏状态，例如 JavaScript `lastIndex`。

## 易错

> **易错：** 直接复制在线正则测试器里的表达式到 Java 代码。
>
> 正确做法：根据目标语言处理字符串转义、标志位、API 语义和测试样本。
