# 正则表达式学习资料：字符、转义和字符类

[返回索引](../正则表达式学习资料.md)

## 学习目标

- 掌握普通字符、特殊字符、转义、点号和字符类。
- 理解字符类中的范围、取反和转义规则。
- 能正确使用 `\d`、`\w`、`\s`，并理解 Unicode 和 ASCII 差异。
- 避免在字符串字面量和正则语法中双重转义出错。

## 理论导读

正则最基本的单位是字符。普通字符匹配自身，特殊字符表达规则。字符类 `[]` 表示“当前位置允许出现这些字符之一”。掌握字符层面的规则，是理解后续量词、分组和断言的基础。

转义是正则初学者最容易出错的地方：正则本身需要转义，编程语言字符串也需要转义。例如 Java 字符串里写一个正则 `\d`，实际要写 `"\\d"`。

## 普通字符和特殊字符

常见特殊字符：

```text
. ^ $ * + ? { } [ ] \ | ( )
```

要匹配它们本身，通常需要转义：

```regex
\.
\*
\(
\)
```

## 点号 `.`

`.` 通常匹配任意单个字符，但默认不匹配换行。开启 dotAll/s 模式后才匹配换行。

```regex
a.b
```

可匹配：

```text
acb
a-b
a b
```

不一定匹配：

```text
a
b
```

是否匹配换行取决于引擎和模式。

## 字符类 `[]`

```regex
[abc]
```

表示当前位置可以是 `a`、`b`、`c` 之一。

范围：

```regex
[a-z]
[0-9]
[A-Fa-f0-9]
```

取反：

```regex
[^0-9]
```

表示不是数字的单个字符。

## 字符类中的特殊规则

在 `[]` 内，很多字符的意义和外面不同：

- `.` 通常就是普通点。
- `^` 在开头表示取反，不在开头通常是普通字符。
- `-` 在中间表示范围，要匹配字面量可放开头/结尾或转义。
- `]` 要匹配字面量需要转义或特殊放置。

例子：

```regex
[-a-z]
[a-z-]
```

都可以包含连字符。

## 预定义字符类

| 写法 | 常见含义 |
| --- | --- |
| `\d` | 数字 |
| `\D` | 非数字 |
| `\w` | 单词字符 |
| `\W` | 非单词字符 |
| `\s` | 空白字符 |
| `\S` | 非空白字符 |

注意：`\w` 是否包含 Unicode 字母，取决于语言、引擎和模式。很多情况下它只等价于 `[A-Za-z0-9_]`，不要默认以为能匹配中文。

## 双重转义

Java：

```java
Pattern.compile("\\d+");
```

JavaScript：

```javascript
const re1 = /\d+/;
const re2 = new RegExp("\\d+");
```

Python：

```python
re.compile(r"\d+")
```

## 例子

匹配十六进制颜色：

```regex
^#[A-Fa-f0-9]{6}$
```

匹配简单标识符：

```regex
^[A-Za-z_][A-Za-z0-9_]*$
```

## 练习

1. 匹配 6 位数字验证码。
2. 匹配 Windows 路径中的反斜杠。
3. 匹配十六进制颜色 `#12AF90`。
4. 写一个只允许字母、数字、下划线且不能数字开头的正则。

## 验收

- 能解释点号是否匹配换行。
- 能写字符类范围和取反。
- 能说明字符串字面量和正则语法的双重转义。
- 能判断 `\w` 在不同语言中的差异风险。

## 重点

- 字符类匹配的是单个字符。
- `[]` 内外特殊字符规则不同。
- 编程语言字符串会先处理转义，正则引擎再处理正则转义。

## 难点

- Unicode、locale、ASCII 模式会影响预定义字符类。
- `-`、`^`、`]` 在字符类中位置敏感。

## 易错

> **易错：** 在 Java 中写 `"\d+"`。
>
> 正确做法：Java 字符串里要写 `"\\d+"`，或者在支持原始字符串的语言中使用原始字符串。
