# 正则表达式学习资料：分组、捕获、引用和命名组

[返回索引](../正则表达式学习资料.md)

## 学习目标

- 理解分组在结构、量词、选择和提取中的作用。
- 区分捕获组、非捕获组和命名捕获组。
- 掌握反向引用的含义和风险。
- 能在替换和提取中合理使用组。

## 理论导读

分组 `()` 是正则表达式的结构化工具。它可以把多个 token 合成一个单元，让量词作用于整体；也可以创建捕获结果，供代码读取、替换引用或后续正则反向引用。

捕获不是免费的。捕获组会改变分组编号，复杂表达式中容易因为插入一个括号而破坏下游代码。因此，不需要提取时应优先使用非捕获组 `(?:...)`。

## 核心心智模型

把分组想成“给一段模式加括号”：

- 结构分组：控制优先级和量词作用范围。
- 捕获分组：把匹配到的文本装进编号槽位。
- 命名分组：给槽位起名字，减少编号脆弱性。
- 反向引用：要求当前位置再次出现前面捕获到的同一段文本。

## 知识点详解

### 捕获组

```regex
(\d{4})-(\d{2})-(\d{2})
```

匹配 `2026-06-06` 时：

| 组 | 内容 |
| --- | --- |
| 0 | `2026-06-06` |
| 1 | `2026` |
| 2 | `06` |
| 3 | `06` |

组 0 通常表示整体匹配，不是显式括号。

### 非捕获组

```regex
(?:https?|ftp)://
```

这里分组只用于选择，不需要提取协议内部结构。使用非捕获组可以避免无意义编号。

### 命名捕获组

不同语言写法略有差异：

| 方言 | 写法 |
| --- | --- |
| Python、.NET、现代 JavaScript | `(?<name>...)` 或 `(?P<name>...)` |
| Java | `(?<name>...)` |
| PCRE | 多种写法，常见 `(?<name>...)` |

例子：

```regex
(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})
```

命名组适合日志解析、配置解析、数据清洗等需要稳定字段名的场景。

### 选择和优先级

```regex
cat|dog
```

表示匹配 `cat` 或 `dog`。

```regex
gr(?:a|e)y
```

匹配 `gray` 或 `grey`。

选择分支通常从左到右尝试。若分支有重叠，顺序会影响结果。

### 反向引用

```regex
<([a-z]+)>.*?</\1>
```

`\1` 表示必须和第 1 组捕获到的标签名相同。它可匹配 `<h1>title</h1>`。

反向引用让正则具备“记忆能力”，但也会让引擎更难优化，部分线性引擎不支持。

## 例子

提取 URL：

```regex
^(?<scheme>https?)://(?<host>[^/:?#]+)(?<path>/[^?#]*)?
```

替换日期格式：

```text
输入：2026-06-06
查找：(\d{4})-(\d{2})-(\d{2})
替换：$2/$3/$1
输出：06/06/2026
```

注意替换引用语法在不同语言中可能是 `$1`、`\1`、`${name}` 或 `\g<name>`。

## 练习

1. 用捕获组提取日期中的年、月、日。
2. 用非捕获组匹配 `http` 或 `https` URL。
3. 用命名组解析 `INFO 2026-06-06 start job`。
4. 写一个匹配重复单词的正则，例如 `hello hello`。

## 验收

- 能解释捕获组编号从哪里来。
- 能说出何时使用 `(?:...)`。
- 能使用命名组提高可读性。
- 能解释反向引用和普通重复量词的区别。

## 重点

- 分组既能改变结构，也能产生提取结果。
- 不需要捕获时使用非捕获组。
- 命名组能降低表达式演进时的维护风险。

## 难点

- 分支顺序会影响匹配结果和性能。
- 反向引用依赖运行时捕获内容，不等同于重复同一个模式。

## 易错

> **易错：** 在已有表达式中随意添加捕获括号，导致代码读取的组编号变化。
>
> 正确做法：结构分组默认用 `(?:...)`，需要输出字段时才捕获，并优先命名。
