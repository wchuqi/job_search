# 正则表达式学习资料：综合练习项目

[返回索引](../正则表达式学习资料.md)

## 项目目标

构建一个“日志清洗与结构化提取”练习，用正则完成字段提取、格式校验、脱敏、异常样本记录和测试覆盖。

## 环境假设

- 可使用任意熟悉语言完成，推荐 Python 或 JavaScript。
- 不要求连接数据库。
- 输入为普通文本文件，每行一条日志。

## 输入样本

```text
2026-06-06 21:30:01 INFO ip=10.0.0.1 user=alice action=login cost=32ms
2026-06-06 21:31:05 WARN ip=10.0.0.2 user=bob action=query cost=120ms
2026-06-06 21:32:10 ERROR ip=10.0.0.3 user=carol@example.com action=pay cost=900ms
bad line
```

## 任务拆分

### 任务 1：提取字段

提取：

- 日期
- 时间
- 日志级别
- IP
- 用户
- 动作
- 耗时

建议表达式：

```regex
^(?<date>\d{4}-\d{2}-\d{2}) (?<time>\d{2}:\d{2}:\d{2}) (?<level>INFO|WARN|ERROR) ip=(?<ip>\d{1,3}(?:\.\d{1,3}){3}) user=(?<user>\S+) action=(?<action>\w+) cost=(?<cost>\d+)ms$
```

### 任务 2：字段转换

- `cost` 转成整数。
- `level` 保持枚举。
- `user` 如果是邮箱，输出时脱敏。

邮箱脱敏示例：

```regex
^([^@\s])[^@\s]*(@[^@\s]+)$
```

替换为：

```text
$1***$2
```

### 任务 3：异常样本

无法匹配的行不要丢弃，记录到 `invalid_lines`，并附带行号。

### 任务 4：测试

测试至少包含：

- 正常 INFO 行。
- 正常 WARN 行。
- 正常 ERROR 行。
- 不合法日期形态。
- 缺少字段。
- 超长输入。
- 末尾多余字段。

## 参考实现方向

Python：

```python
import re

line_re = re.compile(
    r"^(?P<date>\d{4}-\d{2}-\d{2}) "
    r"(?P<time>\d{2}:\d{2}:\d{2}) "
    r"(?P<level>INFO|WARN|ERROR) "
    r"ip=(?P<ip>\d{1,3}(?:\.\d{1,3}){3}) "
    r"user=(?P<user>\S+) "
    r"action=(?P<action>\w+) "
    r"cost=(?P<cost>\d+)ms$"
)

def parse_line(line):
    match = line_re.fullmatch(line)
    if not match:
        return None
    data = match.groupdict()
    data["cost"] = int(data["cost"])
    return data
```

## 进阶任务

1. 增加 IP 0-255 校验，比较用正则和代码校验的可读性。
2. 添加性能测试，输入一行 10000 字符的异常日志。
3. 把正则拆成命名片段，提升可维护性。
4. 增加配置项：允许的日志级别列表。

## 验收

- 能把有效日志转成结构化对象。
- 无效日志能被记录，不会静默丢失。
- 用户邮箱能脱敏。
- 有完整测试样本。
- 能说明哪些校验没有放在正则里以及原因。

## 重点

- 正则负责结构化提取，业务校验可以放到代码中。
- 异常样本要可追踪。
- 测试要覆盖失败路径。

## 难点

- 日志格式变更时要保持兼容或明确失败。
- 字段值允许范围可能比字段形态更复杂。

## 易错

> **易错：** 匹配失败时返回空对象。
>
> 正确做法：返回明确错误或记录异常样本，避免下游把空字段当成真实数据。
