# 正则表达式学习资料：提取、替换、分割和清洗

[返回索引](../正则表达式学习资料.md)

## 学习目标

- 掌握正则在查找、提取、替换、分割和清洗中的常见用法。
- 能设计稳定的捕获组和命名组。
- 理解替换引用语法的方言差异。
- 能避免清洗文本时误删业务信息。

## 理论导读

工程中使用正则，更多不是“判断是否匹配”，而是从大量文本中提取字段、替换格式、分割数据和清理噪声。此时表达式要服务于下游数据结构：字段名要稳定、边界要明确、失败时要可诊断。

## 核心心智模型

把正则处理文本看成一条流水线：

```text
原始文本 -> 定位候选片段 -> 捕获字段 -> 校验字段 -> 转成结构化数据 -> 处理异常样本
```

正则只负责其中一部分，不应承担全部业务判断。

## 知识点详解

### 提取

日志：

```text
2026-06-06 10:30:20 INFO user=alice action=login
```

正则：

```regex
^(?<date>\d{4}-\d{2}-\d{2}) (?<time>\d{2}:\d{2}:\d{2}) (?<level>[A-Z]+) user=(?<user>\w+) action=(?<action>\w+)$
```

### 替换

把日期从 `YYYY-MM-DD` 改成 `MM/DD/YYYY`：

```regex
(\d{4})-(\d{2})-(\d{2})
```

替换模板：

```text
$2/$3/$1
```

不同语言中替换组引用可能不同，写跨语言脚本时必须查清楚。

### 分割

按逗号和周围空白分割：

```regex
\s*,\s*
```

适合简单 CSV 风格文本，但不适合完整 CSV，因为 CSV 有引号、转义、换行字段等复杂规则，应使用 CSV parser。

### 清洗

合并连续空白：

```regex
\s+
```

替换成一个空格。

删除首尾空白通常应使用语言自带 `trim`，不必强行用正则。

### 失败处理

正则提取失败时，不要静默返回空字段。工程上应记录原始样本、表达式版本、失败原因和输入长度。

## 例子

Python 提取日志：

```python
import re

pattern = re.compile(
    r"^(?P<date>\d{4}-\d{2}-\d{2}) "
    r"(?P<time>\d{2}:\d{2}:\d{2}) "
    r"(?P<level>[A-Z]+) user=(?P<user>\w+) action=(?P<action>\w+)$"
)

line = "2026-06-06 10:30:20 INFO user=alice action=login"
match = pattern.match(line)
if match:
    print(match.groupdict())
```

JavaScript 替换：

```javascript
const text = "date=2026-06-06";
const result = text.replace(/(\d{4})-(\d{2})-(\d{2})/, "$2/$3/$1");
console.log(result);
```

## 练习

1. 从 `ip=10.0.0.1 status=200 cost=32ms` 中提取 IP、状态码和耗时。
2. 把手机号中间四位替换成 `****`。
3. 按一个或多个空白分割英文句子。
4. 写出清洗用户输入前后的测试样本。

## 验收

- 能用命名组输出结构化字段。
- 能说明替换引用的语言差异。
- 能判断何时应使用 parser 而不是正则。
- 能为提取失败设计日志和兜底。

## 重点

- 提取类正则要面向字段设计。
- 替换要确认组引用语法。
- 清洗操作应可回溯，避免不可逆误删。

## 难点

- 分割格式看似简单，实际可能有转义、引号和嵌套。
- 清洗规则过宽会破坏原始语义。

## 易错

> **易错：** 用 `split(",")` 或 `\s*,\s*` 解析完整 CSV。
>
> 正确做法：简单内部格式可以用正则，标准 CSV 使用成熟 parser。
