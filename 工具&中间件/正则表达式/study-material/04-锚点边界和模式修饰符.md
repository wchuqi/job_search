# 正则表达式学习资料：锚点、边界和模式修饰符

[返回索引](../正则表达式学习资料.md)

## 学习目标

- 掌握 `^`、`$`、`\A`、`\z`、`\b` 等位置匹配。
- 理解“匹配字符”和“匹配位置”的区别。
- 掌握常见模式修饰符：忽略大小写、多行、dotAll、扩展模式。
- 能判断全量校验和局部查找应使用不同边界。

## 理论导读

锚点和边界不消耗字符，它们匹配的是位置。没有边界的正则通常表示“在字符串中找一个片段”；有起止锚点的正则通常表示“整个字符串必须符合规则”。很多校验漏洞都来自忘记加完整边界。

模式修饰符会改变 token 的解释方式。例如多行模式会改变 `^` 和 `$` 的含义，dotAll 会改变 `.` 是否匹配换行。

## 核心心智模型

字符串不仅有字符，还有字符之间的位置：

```text
| a | b | c |
0   1   2   3
```

锚点、边界和断言检查这些位置是否满足条件，而不是吃掉某个字符。

## 知识点详解

### 起止锚点

| 写法 | 常见含义 |
| --- | --- |
| `^` | 字符串开头；多行模式下也可表示行开头 |
| `$` | 字符串结尾或结尾换行前；多行模式下也可表示行结尾 |
| `\A` | 整个字符串开头，部分方言支持 |
| `\z` | 整个字符串绝对结尾，部分方言支持 |

全量校验日期形态：

```regex
^\d{4}-\d{2}-\d{2}$
```

如果不加锚点，`abc2026-06-06xyz` 中也能找到日期片段。

### 单词边界

```regex
\bcat\b
```

用于匹配独立单词 `cat`，避免匹配 `catalog` 中的 `cat`。

`\b` 的行为依赖 `\w` 的定义，因此在 Unicode、ASCII、locale 模式下可能不同。处理中文分词时不要依赖 `\b`。

### 常见模式修饰符

| 修饰符 | 常见名称 | 作用 |
| --- | --- | --- |
| `i` | ignore case | 忽略大小写 |
| `m` | multiline | `^`、`$` 按行工作 |
| `s` | dotAll | `.` 匹配换行 |
| `x` | extended / verbose | 允许空白和注释，便于维护 |
| `u` | Unicode | 启用 Unicode 语义，具体行为依方言而定 |

JavaScript：

```javascript
/^error:/im
```

Python：

```python
re.compile(r"^error:", re.IGNORECASE | re.MULTILINE)
```

### 局部查找和全量校验

局部查找：

```regex
\d{4}-\d{2}-\d{2}
```

全量校验：

```regex
^\d{4}-\d{2}-\d{2}$
```

两者意图不同。表单校验通常需要全量校验，日志提取通常需要局部查找。

## 例子

匹配每一行开头的错误日志：

```regex
^ERROR\b.*
```

需要开启多行模式。

匹配整个字符串是变量名：

```regex
^[A-Za-z_][A-Za-z0-9_]*$
```

## 练习

1. 写一个全量匹配 6 位验证码的正则。
2. 用多行模式匹配每行开头的 `WARN`。
3. 解释 `^abc$` 在多行和非多行模式下的区别。
4. 判断 `\b你好\b` 是否可靠，并说明原因。

## 验收

- 能说明锚点不消耗字符。
- 能区分局部查找和全量校验。
- 能解释 `m` 和 `s` 修饰符的差异。
- 能指出 `\b` 在 Unicode 文本中的风险。

## 重点

- 校验常常需要起止锚点。
- `^`、`$` 会被多行模式改变语义。
- `.` 是否匹配换行取决于 dotAll。

## 难点

- `$` 在部分引擎中可匹配结尾换行前的位置。
- Unicode 语义会影响大小写、单词边界和字符类。

## 易错

> **易错：** 用 `\d{6}` 校验短信验证码。
>
> 正确做法：使用 `^\d{6}$`，否则 `abc123456xyz` 也可能通过“查找式”匹配。
