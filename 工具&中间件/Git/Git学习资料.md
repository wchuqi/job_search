# Git 学习资料

这份资料已经按知识点拆分。总索引保留学习顺序和入口，详细内容放在 `study-material/` 目录下，避免所有内容堆在一个 Markdown 文件里。

## 阅读原则：先理解 Git 在移动什么，再执行命令

Git 命令不是一组孤立口诀。多数命令本质上只在做三类事：移动引用指针、改变工作区/暂存区/提交历史里的内容、和远程仓库交换对象。学习时不要只记 `add`、`commit`、`merge`、`reset` 的命令格式，而要先想清楚“当前改动在哪里”“HEAD 指向哪里”“分支指针会不会移动”“历史是否会被改写”“这个操作是否会影响别人”。

本资料的主章节已加入理论导读。建议每章先读导读，把 Git 想象成“对象数据库 + 引用指针 + 三个工作区域”的系统，再去运行命令。命令是验证模型的实验，不是替代理解的捷径。

## 使用建议

- 初学时按顺序阅读 0 到 7。
- 工作前查 8 和 11。
- 出错排障时优先查 4、7、9。
- 做综合练习时使用 10，并用 12 自测。
- 面试前重点查 13，并结合 11 快速复习命令差异。
- 想确认知识点是否遗漏时先查 14，再按 15 到 22 补齐高级专题。
- 想从“会用 Git”提升到“理解 Git”时重点学习 15、16、17、18。
- 团队协作、发布和安全治理重点学习 19、20、21、22。
- 每学一个命令，都要能说出它影响工作区、暂存区、本地提交、远程引用中的哪一部分。
