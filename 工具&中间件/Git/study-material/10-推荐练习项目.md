# Git 学习资料：推荐练习项目

[返回索引](../Git学习资料.md)

## 理论导读：练习项目要模拟真实状态变化，而不是只跑命令

Git 练习的价值不在于把命令敲一遍，而在于制造真实项目里会遇到的状态变化：文件新增、修改、删除；提交拆分和合并；分支并行开发；冲突解决；远程同步；错误撤销；发布打标签。只有让仓库状态真的变化，才能训练对工作区、暂存区、提交历史和远程引用的判断。

一个好的练习项目应该小到能快速修改，大到能覆盖多文件、多分支和多人协作场景。命令行计算器适合练习，因为它既有代码文件、测试说明、文档，也容易制造功能分支、bugfix 分支和冲突。

做练习时不要追求一次成功。刻意制造冲突、误提交、错误分支和需要恢复的场景，反而能建立稳定的排障能力。

## 推荐练习项目

创建一个小型命令行计算器项目，用它练习所有 Git 操作。

目录结构：

```text
git-calculator/
  README.md
  package.json
  src/
    calculator.js
  test/
    calculator.test.js
  docs/
    test-plan.md
```

`package.json`：

```json
{
  "name": "git-calculator",
  "version": "1.0.0",
  "scripts": {
    "test": "node test/calculator.test.js"
  }
}
```

`src/calculator.js`：

```javascript
function add(a, b) {
  return a + b;
}

function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

function divide(a, b) {
  if (b === 0) {
    throw new Error("Cannot divide by zero");
  }
  return a / b;
}

module.exports = {
  add,
  subtract,
  multiply,
  divide,
};
```

`test/calculator.test.js`：

```javascript
const assert = require("assert");
const { add, subtract, multiply, divide } = require("../src/calculator");

assert.strictEqual(add(1, 2), 3);
assert.strictEqual(subtract(5, 2), 3);
assert.strictEqual(multiply(3, 4), 12);
assert.strictEqual(divide(8, 2), 4);

assert.throws(() => divide(1, 0), /Cannot divide by zero/);

console.log("All tests passed");
```

可以用这个项目练习：

- 阶段 1：初始化、提交、查看日志。
- 阶段 2：每个函数用一个分支开发并合并。
- 阶段 3：推送到远程，模拟多人修改。
- 阶段 4：故意写错函数后撤销和 revert。
- 阶段 5：把多个零散提交 squash。
- 阶段 6：添加 `.gitignore` 和 `v1.0.0` tag。
- 阶段 7：故意引入 bug，用 `git bisect run npm test` 定位。
