# Git 学习资料：命令速查

[返回索引](../Git学习资料.md)

## 理论导读：速查表是按状态选命令，不是按记忆背命令

同一个“撤销”场景，可能对应完全不同的命令。未暂存修改、已暂存修改、本地提交、远程共享提交，恢复方式都不同。速查表的正确用法，是先判断状态，再定位命令，而不是看到某个熟悉命令就直接执行。

命令可以按影响对象来分类：有些看状态，如 `status`、`log`、`diff`；有些改变工作区或暂存区，如 `add`、`restore`；有些生成或移动提交和引用，如 `commit`、`merge`、`rebase`、`reset`；有些与远程交换对象和引用，如 `fetch`、`pull`、`push`。知道分类，就能降低误用风险。

## 使用方式

这份速查表按场景组织，不建议死背。遇到问题时先判断当前修改位于哪里，再选择命令。

> **重点：** 先 `git status`，再操作。看不清状态时不要执行破坏性命令。

## 一、状态和差异

```bash
git status
git status --short
git diff
git diff --staged
git diff HEAD
git diff --name-only
git diff --stat
git diff --check
```

## 二、暂存和提交

```bash
git add <file>
git add .
git add -p
git restore --staged <file>
git commit -m "message"
git commit --amend
git commit --allow-empty -m "Trigger CI"
```

## 三、历史查看

```bash
git log
git log --oneline
git log --oneline --graph --decorate --all
git log --stat
git log --patch
git log -- <file>
git log --follow -- <file>
git show HEAD
git show <commit>
git shortlog -sn
```

## 四、搜索和定位

```bash
git grep -n "keyword"
git log -S "keyword" --patch
git log -G "regex" --patch
git blame <file>
git blame -L 10,30 <file>
git bisect start
git bisect run <test-command>
```

## 五、分支

```bash
git branch
git branch -vv
git switch <branch>
git switch -c <branch>
git switch --detach <commit>
git branch -d <branch>
git branch -D <branch>
git branch -m <new-name>
```

## 六、合并和冲突

```bash
git merge <branch>
git merge --no-ff <branch>
git merge --abort
git mergetool
git rebase --continue
git rebase --abort
git rebase --skip
```

## 七、远程

```bash
git remote -v
git remote add origin <url>
git remote set-url origin <url>
git fetch origin
git fetch --all --prune
git pull
git pull --rebase
git push
git push -u origin <branch>
git push --force-with-lease
```

## 八、撤销和恢复

```bash
git restore <file>
git restore --source=<commit> <file>
git restore --staged <file>
git reset --soft HEAD~1
git reset --mixed HEAD~1
git reset --hard HEAD~1
git revert <commit>
git reflog
git clean -nd
git clean -fd
```

> **易错：** `reset --hard` 和 `clean -fd` 都可能丢失本地内容。先确认，再执行。

## 九、变基和历史整理

```bash
git rebase <base>
git rebase -i HEAD~3
git rebase --onto <new-base> <old-base> <branch>
git commit --fixup <commit>
git rebase -i --autosquash <base>
git cherry-pick <commit>
git cherry-pick --continue
git cherry-pick --abort
```

## 十、标签

```bash
git tag
git tag v1.0.0
git tag -a v1.0.0 -m "Release v1.0.0"
git tag -s v1.0.0 -m "Release v1.0.0"
git show v1.0.0
git push origin v1.0.0
git push origin --tags
git tag -d v1.0.0
git push origin :refs/tags/v1.0.0
```

## 十一、忽略、属性和文件处理

```bash
git check-ignore -v <file>
git rm --cached <file>
git mv <old> <new>
git update-index --chmod=+x <file>
git update-index --assume-unchanged <file>
git update-index --no-assume-unchanged <file>
git update-index --skip-worktree <file>
git update-index --no-skip-worktree <file>
```

## 十二、配置

```bash
git config --list
git config --list --show-origin
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
git config --global pull.rebase true
git config --global alias.lg "log --oneline --graph --decorate --all"
```

## 十三、对象和底层观察

```bash
git rev-parse HEAD
git rev-parse --show-toplevel
git cat-file -t HEAD
git cat-file -p HEAD
git cat-file -p HEAD^{tree}
git ls-tree HEAD
git ls-tree -r HEAD
git hash-object <file>
git show-ref
```

## 十四、stash

```bash
git stash push -m "message"
git stash push -u -m "include untracked"
git stash list
git stash show -p stash@{0}
git stash apply stash@{0}
git stash pop
git stash drop stash@{0}
git stash clear
```

## 十五、worktree、submodule、LFS

```bash
git worktree add ../project-hotfix main
git worktree list
git worktree remove ../project-hotfix
git submodule add <repo-url> libs/common
git submodule update --init --recursive
git lfs install
git lfs track "*.zip"
git lfs ls-files
git lfs pull
```

## 十六、稀疏和浅克隆

```bash
git clone --depth 1 <repo-url>
git fetch --unshallow
git clone --filter=blob:none <repo-url>
git sparse-checkout init --cone
git sparse-checkout set <path>
git sparse-checkout disable
```

## 十七、维护和安全

```bash
git count-objects -vH
git fsck
git gc
git gc --aggressive
git reflog expire --expire=now --all
git repack -ad
git clone --mirror <repo-url>
git push --mirror <new-url>
git log --show-signature
git verify-commit <commit>
git verify-tag <tag>
```

## 十八、高风险命令清单

执行前必须确认影响范围：

```bash
git reset --hard
git clean -fd
git push --force
git push --mirror
git filter-repo
git reflog expire --expire=now --all
git gc --prune=now
```

更安全的习惯：

```bash
git status
git branch backup/before-risky-operation
git clean -nd
git push --force-with-lease
git revert <commit>
```
