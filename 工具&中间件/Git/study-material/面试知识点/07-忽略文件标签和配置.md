## 七、忽略文件、标签和配置

### 34. `.gitignore` 的作用是什么？

**参考答案：**

`.gitignore` 用来忽略不应该提交的未跟踪文件，例如依赖目录、构建产物、日志、本地配置。

```gitignore
node_modules/
dist/
*.log
.env
!.env.example
```

> **易错：** `.gitignore` 只影响未跟踪文件，已经被 Git 跟踪的文件不会自动停止跟踪。

### 35. 已经提交过的 `.env` 怎么从 Git 跟踪中移除？

**参考答案：**

```bash
git rm --cached .env
git commit -m "Stop tracking env file"
```

然后把 `.env` 写入 `.gitignore`。

> **易错：** `git rm --cached` 不删除本地文件，只从 Git 跟踪中移除；`git rm` 会删除本地文件。

### 36. tag 有什么作用？

**参考答案：**

tag 用于给某个提交打固定标记，常见场景是标记发布版本。

```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

> **重点：** tag 默认只在本地创建，不会自动推送到远程。
