## 一、基础概念

### 1. Git 和 GitHub / GitLab / Gitee 有什么区别？

**参考答案：**

Git 是分布式版本控制工具，负责在本地和远程记录代码历史。GitHub、GitLab、Gitee 是代码托管平台，提供远程仓库、Pull Request / Merge Request、代码评审、CI/CD、权限管理等能力。

可以简单理解：

- Git：版本控制工具。
- GitHub / GitLab / Gitee：基于 Git 的协作平台。

> **易错：** 不要说“GitHub 是 Git”。GitHub 只是远程托管平台之一。

### 2. Git 为什么是分布式版本控制？

**参考答案：**

因为每个开发者本地都有完整仓库历史，而不是只保存最新代码。即使没有网络，也可以在本地提交、查看日志、创建分支、回退版本。网络恢复后再和远程仓库同步。

> **重点：** 分布式的核心是“每个本地仓库都保存完整历史”，不是“可以多人协作”这么简单。

### 3. 工作区、暂存区、本地仓库、远程仓库分别是什么？

**参考答案：**

| 区域 | 作用 | 常见命令 |
| --- | --- | --- |
| 工作区 | 当前真实文件 | `git status`, `git diff`, `git restore` |
| 暂存区 | 下一次提交要包含的内容 | `git add`, `git restore --staged` |
| 本地仓库 | 已经提交的历史 | `git commit`, `git log`, `git reset` |
| 远程仓库 | 团队共享仓库 | `git fetch`, `git pull`, `git push` |

> **难点：** 暂存区允许你把工作区中的多个修改拆成不同提交，这就是 Git 能精细组织提交的重要原因。

### 4. `git add` 和 `git commit` 的区别是什么？

**参考答案：**

`git add` 把工作区中的修改加入暂存区，表示这些内容准备进入下一次提交。`git commit` 把暂存区内容保存成本地仓库中的一个提交。

```bash
git add README.md
git commit -m "Update README"
```

> **易错：** `git add` 后如果继续修改同一个文件，新修改不会自动进入暂存区，需要再次 `git add`。

### 5. HEAD 是什么？

**参考答案：**

`HEAD` 表示当前所在位置，通常指向当前分支，而当前分支再指向某个提交。

```text
A --- B --- C  main
              ^
             HEAD
```

如果处于 detached HEAD 状态，`HEAD` 会直接指向某个提交，而不是分支。

> **重点：** 分支是指针，`HEAD` 表示当前检出的指针或提交。

### 6. 提交哈希有什么作用？

**参考答案：**

提交哈希是 Git 根据提交内容、父提交、作者、时间等信息计算出的唯一标识。可以用它查看、回退、比较、cherry-pick 或 revert 某个提交。

```bash
git show <commit-hash>
git revert <commit-hash>
git cherry-pick <commit-hash>
```

> **易错：** 提交内容或历史被 amend / rebase 改写后，提交哈希会变化。
