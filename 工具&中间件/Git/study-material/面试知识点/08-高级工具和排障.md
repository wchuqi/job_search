## 八、高级工具和排障

### 37. stash 的作用是什么？

**参考答案：**

`stash` 用于临时保存当前未提交修改，方便切换分支处理其他任务。

```bash
git stash push -m "WIP user profile"
git stash list
git stash pop
```

保存未跟踪文件：

```bash
git stash push -u -m "WIP including new files"
```

> **易错：** 默认 stash 不保存未跟踪文件，新文件要加 `-u`。

### 38. cherry-pick 的作用是什么？

**参考答案：**

`cherry-pick` 可以把某个提交的改动复制到当前分支，适合只迁移某个修复，而不是合并整个分支。

```bash
git cherry-pick <commit>
```

> **重点：** cherry-pick 会生成新提交，新提交哈希和原提交不同。

### 39. bisect 是什么？什么时候用？

**参考答案：**

`git bisect` 用二分法定位哪个提交引入了 bug。适合历史较长、无法直接判断问题来源，但可以通过测试判断某个提交好坏的场景。

```bash
git bisect start
git bisect bad
git bisect good v1.0.0
git bisect run npm test
git bisect reset
```

> **难点：** bisect 的前提是你能判断某个提交是 good 还是 bad，最好有自动化测试。

### 40. blame 的作用是什么？

**参考答案：**

`git blame` 用来查看文件每一行最后由哪个提交修改，可用于追踪需求背景和历史原因。

```bash
git blame src/calculator.js
git blame -L 1,40 src/calculator.js
```

> **易错：** blame 是用来理解历史，不是用来指责个人。
