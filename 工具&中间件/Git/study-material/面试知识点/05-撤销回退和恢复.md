## 五、撤销、回退和恢复

### 23. `git restore`、`git reset`、`git revert` 的区别是什么？

**参考答案：**

| 命令 | 主要作用 | 是否改写历史 | 常见场景 |
| --- | --- | --- | --- |
| `git restore` | 恢复文件内容或取消暂存 | 否 | 撤销工作区修改 |
| `git reset` | 移动分支指针 | 是 | 撤回本地提交 |
| `git revert` | 新增反向提交 | 否 | 撤销已推送提交 |

> **重点：** 共享分支上优先用 `revert`，不要随意 `reset`。

### 24. 如何撤销工作区修改？

**参考答案：**

```bash
git restore <file>
```

这会把文件恢复到暂存区或最近提交中的版本。

> **易错：** 这个操作会丢弃工作区未保存到 Git 的修改，执行前要确认不需要这些内容。

### 25. 如何取消已经 `git add` 的文件？

**参考答案：**

```bash
git restore --staged <file>
```

它只会从暂存区撤回，不会删除工作区修改。

### 26. 如何修改最近一次提交？

**参考答案：**

修改提交信息：

```bash
git commit --amend -m "New commit message"
```

补充漏掉的文件：

```bash
git add <file>
git commit --amend
```

> **易错：** `commit --amend` 会生成新提交哈希。已经推送并被别人使用的提交不要随意 amend。

### 27. `git reset --soft`、`--mixed`、`--hard` 有什么区别？

**参考答案：**

| 模式 | 分支指针 | 暂存区 | 工作区 |
| --- | --- | --- | --- |
| `--soft` | 移动 | 保留修改 | 保留修改 |
| `--mixed` | 移动 | 清空暂存 | 保留修改 |
| `--hard` | 移动 | 清空暂存 | 丢弃修改 |

```bash
git reset --soft HEAD~1
git reset --mixed HEAD~1
git reset --hard HEAD~1
```

> **易错：** `reset --hard` 会丢弃工作区修改，属于高风险命令。

### 28. 已经推送的错误提交怎么撤销？

**参考答案：**

使用 `git revert`：

```bash
git revert <commit>
git push
```

它会创建一个新的反向提交，不会改写远程历史，适合共享分支。

> **重点：** 面试中回答“已推送错误提交”时，优先说 `revert`。

### 29. 误删分支或误 reset 后怎么找回？

**参考答案：**

使用 `git reflog` 找到之前 HEAD 或分支指针所在的提交：

```bash
git reflog
git branch recover/old-work <commit>
```

如果确认要回到那个位置：

```bash
git reset --hard <commit>
```

> **重点：** 更保守的做法是先创建恢复分支，而不是直接 hard reset。
