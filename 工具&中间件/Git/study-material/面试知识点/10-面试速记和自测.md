## 十、面试速记清单

- `git add`：工作区到暂存区。
- `git commit`：暂存区到本地仓库。
- `git fetch`：只更新远程跟踪分支，不改工作区。
- `git pull`：拉取并 merge / rebase，会影响当前分支。
- `git reset`：移动分支指针，可能改写历史。
- `git revert`：新增反向提交，适合共享分支。
- `git rebase`：重放提交，历史更线性，但会改写哈希。
- `git reflog`：找回误操作前的位置。
- `git stash`：临时保存现场。
- `git cherry-pick`：复制某个提交的改动到当前分支。
- `git bisect`：二分定位问题提交。
- `git cat-file`：查看 Git 对象类型或内容。
- `git rev-parse`：解析引用或输出仓库路径。
- `git check-ignore -v`：排查忽略规则来源。
- `git worktree`：同一仓库创建多个工作区。
- `git lfs`：管理大文件指针和内容。
- `git gc`：执行对象压缩和垃圾回收。
