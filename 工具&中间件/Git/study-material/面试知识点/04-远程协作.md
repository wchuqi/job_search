## 四、远程协作

### 18. `git clone`、`git fetch`、`git pull`、`git push` 的区别是什么？

**参考答案：**

| 命令 | 作用 | 是否修改工作区 |
| --- | --- | --- |
| `git clone` | 下载完整远程仓库 | 是 |
| `git fetch` | 拉取远程更新到本地远程跟踪分支 | 否 |
| `git pull` | 拉取并合并或变基 | 是 |
| `git push` | 推送本地提交到远程 | 否，对远程有影响 |

> **重点：** `fetch` 比 `pull` 更安全，因为它不会自动改变当前工作区。

### 19. `git pull` 等价于什么？

**参考答案：**

默认情况下大致等价于：

```bash
git fetch
git merge
```

如果使用 rebase 模式：

```bash
git pull --rebase
```

大致等价于：

```bash
git fetch
git rebase
```

> **易错：** 团队要统一 pull 策略，否则提交历史风格会混乱。

### 20. `origin/main` 是什么？

**参考答案：**

`origin/main` 是本地保存的远程跟踪分支，表示上一次 fetch 后远程 `origin` 上 `main` 分支的位置。它不是远程服务器上的实时状态，只有执行 `git fetch origin` 后才会更新。

> **难点：** `main`、`origin/main`、远程服务器上的 `main` 是三个不同概念。

### 21. push 被拒绝怎么办？

**参考答案：**

常见原因是远程分支有本地没有的新提交。推荐先同步远程变化：

```bash
git fetch origin
git rebase origin/main
git push
```

如果团队使用 merge 风格：

```bash
git pull
git push
```

> **易错：** push 被拒绝后不要马上 `git push --force`，这可能覆盖别人提交。

### 22. 第一次推送新分支为什么常用 `-u`？

**参考答案：**

`-u` 会建立本地分支和远程分支的追踪关系，之后可以直接使用 `git push` 和 `git pull`。

```bash
git push -u origin feature/login
```

查看追踪关系：

```bash
git branch -vv
```
