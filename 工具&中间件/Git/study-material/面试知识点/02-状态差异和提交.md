## 二、状态、差异和提交

### 7. 如何查看当前仓库状态？

**参考答案：**

```bash
git status
git status --short
```

`git status` 适合看详细说明，`git status --short` 适合日常快速确认哪些文件被修改、暂存或未跟踪。

> **重点：** 多数 Git 操作前都应该先执行 `git status`。

### 8. `git diff` 和 `git diff --staged` 有什么区别？

**参考答案：**

`git diff` 查看工作区和暂存区之间的差异，也就是还没暂存的修改。`git diff --staged` 查看暂存区和最近一次提交之间的差异，也就是下一次提交会包含的内容。

```bash
git diff
git diff --staged
```

> **易错：** 文件已经 `git add` 后，普通 `git diff` 可能没有输出，因为差异已经进入暂存区。

### 9. 如何把一次大修改拆成多个提交？

**参考答案：**

可以分批 `git add` 不同文件，也可以使用交互式暂存：

```bash
git add src/user.js
git commit -m "Add user model"

git add src/order.js
git commit -m "Add order model"
```

按块选择暂存：

```bash
git add -p
```

> **重点：** 好的提交应该小而完整，一次提交只表达一个明确意图。

### 10. 提交信息应该怎么写？

**参考答案：**

提交信息应说明本次提交做了什么，最好使用动词开头。

好的例子：

```text
Add user profile page
Fix login timeout
Refactor payment validation
Document Git workflow
```

较差的例子：

```text
update
fix
change code
```

> **易错：** 只写 `update` 不能帮助评审者和未来的自己理解历史。
