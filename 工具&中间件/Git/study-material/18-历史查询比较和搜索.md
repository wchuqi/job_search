# Git 学习资料：历史查询、比较和搜索

[返回索引](../Git学习资料.md)

## 理论导读：历史查询是在对象图里追踪变化原因

`git log`、`git show`、`git diff`、`git blame`、`git grep`、`git log -S` 不是简单查看命令，它们分别回答不同问题：历史有哪些提交，某次提交改了什么，两个版本差异是什么，某行是谁改的，仓库里哪里出现了某段内容，某个字符串何时被加入或删除。

Git 历史查询的关键是先明确问题类型。想知道“版本之间差什么”用 diff，想知道“谁改了这一行”用 blame，想知道“某段代码何时出现”用 pickaxe，想知道“某文件历史”要限制路径。问题问清楚，命令组合才会清楚。

历史查询也是排障工具。线上 bug、回归问题、接口变更、配置变化，往往都能通过提交图和差异逐步缩小范围。

## 学习目标

完成本章后，你应该能：

- 用 `log` 按条件查询提交历史。
- 用 `diff` 正确比较工作区、暂存区、分支和提交。
- 用 `grep`、`blame`、`bisect` 定位代码和问题来源。
- 理解双点、三点和路径限制在查询中的作用。

## 一、查看历史：git log

常用命令：

```bash
git log
git log --oneline
git log --oneline --graph --decorate --all
git log --stat
git log --patch
```

按文件查看历史：

```bash
git log -- README.md
git log --follow -- src/old-name.js
```

按作者：

```bash
git log --author="Alice"
```

按时间：

```bash
git log --since="2 weeks ago"
git log --until="2026-01-01"
```

按提交信息：

```bash
git log --grep="login"
```

> **重点：** `git log -- <path>` 是排查某个文件历史的常用入口。

## 二、查看提交详情：git show

查看当前提交：

```bash
git show HEAD
```

查看指定提交：

```bash
git show <commit>
```

只看某个文件在该提交中的内容：

```bash
git show HEAD~1:src/app.js
```

查看 tag：

```bash
git show v1.0.0
```

## 三、比较差异：git diff

| 命令 | 比较对象 |
| --- | --- |
| `git diff` | 工作区 vs 暂存区 |
| `git diff --staged` | 暂存区 vs HEAD |
| `git diff HEAD` | 工作区整体 vs HEAD |
| `git diff A B` | A 和 B 的差异 |
| `git diff A..B` | 基本等价于 A 和 B 的差异 |
| `git diff A...B` | merge base 到 B 的差异 |

例子：

```bash
git diff
git diff --staged
git diff origin/main...HEAD
git diff --name-only main...feature
```

> **易错：** PR 页面常用三点 diff 视角，因此本地排查 PR 差异时常用 `git diff origin/main...HEAD`。

## 四、查找代码：git grep

在当前版本中搜索：

```bash
git grep "function login"
```

在指定提交中搜索：

```bash
git grep "timeout" HEAD~3
```

带行号：

```bash
git grep -n "TODO"
```

> **重点：** `git grep` 只搜索 Git 跟踪文件，速度通常很快。

## 五、查找引入某段代码的提交：pickaxe

查找某个字符串在哪些提交中被添加或删除：

```bash
git log -S "validateToken"
```

用正则查找代码变化：

```bash
git log -G "function .*Login"
```

查看 patch：

```bash
git log -S "validateToken" --patch
```

> **难点：** `-S` 关注字符串出现次数变化，`-G` 关注 patch 中匹配正则的行。

## 六、追踪某行代码：git blame

查看每行最后修改来源：

```bash
git blame src/app.js
```

指定行范围：

```bash
git blame -L 20,60 src/app.js
```

忽略空白变化：

```bash
git blame -w src/app.js
```

> **易错：** blame 只是理解历史的工具，不是追责工具。真实排查还要看提交上下文和需求背景。

## 七、定位引入问题的提交：git bisect

手动二分：

```bash
git bisect start
git bisect bad
git bisect good v1.0.0

# Git 切到某个提交后运行测试
git bisect good
# 或
git bisect bad

git bisect reset
```

自动二分：

```bash
git bisect start
git bisect bad
git bisect good v1.0.0
git bisect run npm test
git bisect reset
```

> **重点：** bisect 的价值来自可重复判断 good/bad，最好配合自动化测试。

## 八、统计和汇总

查看贡献摘要：

```bash
git shortlog -sn
```

查看某段时间提交：

```bash
git log --since="1 month ago" --oneline
```

查看文件变化统计：

```bash
git log --stat
git diff --stat main...feature
```

查看谁改过某文件：

```bash
git shortlog -sn -- src/app.js
```

## 九、排查常用组合

### 1. 我这个 PR 到底改了什么？

```bash
git fetch origin
git diff --name-only origin/main...HEAD
git diff origin/main...HEAD
```

### 2. 某行代码是谁改的？

```bash
git blame -L 30,40 src/app.js
git show <commit>
```

### 3. 某个函数什么时候被引入？

```bash
git log -S "function targetName" --patch
```

### 4. 最近谁改了这个文件？

```bash
git log --oneline --decorate -- src/app.js
```

## 练习

在一个有多次提交的仓库中完成：

```bash
git log --oneline --graph --decorate --all
git log --since="1 week ago"
git diff --staged
git grep -n "TODO"
git log -S "关键函数名" --patch
git blame -L 1,80 <file>
```

## 验收

- 能解释 `git diff`、`git diff --staged`、`git diff A...B` 的差异。
- 能用 `log -- <file>` 查某文件历史。
- 能用 `-S` 或 `-G` 查某段代码历史。
- 能用 `bisect` 定位引入问题的提交。
