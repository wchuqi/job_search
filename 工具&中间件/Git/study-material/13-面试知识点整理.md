## 理论导读：Git 面试考的是状态判断和风险意识

Git 面试题表面上会问命令区别，实际是在考你是否理解对象模型、引用移动、工作区和暂存区、历史改写、冲突原因、远程协作风险。能背 `reset` 和 `revert` 的区别不够，还要能说明为什么共享分支上优先 revert，为什么 amend 会改变哈希，为什么 rebase 后需要谨慎推送。

回答 Git 面试题时，先给直接结论，再解释它影响哪个区域或指针，最后补充适用场景和风险。这样的回答比单纯罗列命令更可靠，也更接近真实工作。

这份资料用于 Git 面试前复习。面试问题已按知识点拆分到 `面试知识点/` 目录，避免所有问答堆在一个 Markdown 文件里。

## 使用方式

- 先看问题，自己口述答案，再对照参考答案补齐概念、命令和场景细节。
- 面试前优先复习 01、03、04、05、06、09、11、12。
- 如果时间很短，直接看 10 的速记清单和自测问题，再补 11 的对象模型和引用问题。

> **重点：** Git 面试很少只考命令本身，更多考你是否理解工作区、暂存区、提交历史、分支指针、远程同步和风险控制。

> **易错：** 回答 Git 问题时只背命令、不说适用场景和风险，会显得没有真实项目经验。
