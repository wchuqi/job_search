# Git 学习资料：阶段 6：标签、忽略文件和常用配置

[返回索引](../Git学习资料.md)

## 理论导读：标签、忽略和配置是在给仓库加规则和名字

标签是给某个提交起一个稳定名字，常用于版本发布。分支会随着提交前进，标签通常固定不动，所以 `v1.0.0` 表示某个明确发布点。轻量标签像简单书签，附注标签则是带作者、时间、说明甚至签名的正式发布记录。

`.gitignore` 解决的是“哪些文件不应进入版本控制”。构建产物、日志、缓存、密钥和本地 IDE 配置通常属于环境副产物，不应该污染提交历史。忽略规则只影响未跟踪文件，已经被 Git 跟踪的文件不会因为写进 `.gitignore` 就自动停止跟踪。

配置则决定 Git 在当前用户、当前仓库或系统层面的默认行为。用户名邮箱、默认分支名、换行处理、编辑器、别名、pull 策略都属于配置。理解配置层级，才能解释为什么同一条命令在不同仓库或不同机器上表现不同。

## 阶段 6：标签、忽略文件和常用配置

### 学习目标

完成本阶段后，你应该能：

- 写出常用 `.gitignore`。
- 处理已经被跟踪的忽略文件。
- 配置常用 Git alias。
- 使用 tag 标记发布版本。

### `.gitignore` 基础

`.gitignore` 用来告诉 Git 哪些未跟踪文件不需要纳入版本控制。

示例：

```gitignore
# Dependencies
node_modules/
vendor/

# Build outputs
dist/
build/
target/

# Logs
*.log
npm-debug.log*

# Environment files
.env
.env.local
!.env.example

# IDE
.idea/
.vscode/

# OS
.DS_Store
Thumbs.db
```

规则说明：

| 写法 | 含义 |
| --- | --- |
| `node_modules/` | 忽略目录 |
| `*.log` | 忽略所有 `.log` 文件 |
| `.env` | 忽略根目录或子目录中的 `.env` |
| `!.env.example` | 取消忽略 `.env.example` |
| `/dist/` | 只忽略仓库根目录的 `dist` |

> **重点：** `.gitignore` 只影响未被 Git 跟踪的文件。

### 已经提交的文件，后来想忽略

例如 `.env` 已经被提交过，后来加入 `.gitignore` 不会自动停止跟踪。

正确做法：

```bash
git rm --cached .env
git commit -m "Stop tracking local env file"
```

文件仍保留在本地，只是不再被 Git 跟踪。

> **易错：** `git rm .env` 会删除本地文件；`git rm --cached .env` 只从 Git 跟踪中移除。

### Git 配置层级

| 层级 | 命令 | 作用范围 |
| --- | --- | --- |
| system | `git config --system` | 整台机器 |
| global | `git config --global` | 当前用户 |
| local | `git config --local` | 当前仓库 |

查看配置来源：

```bash
git config --list --show-origin
```

常用配置：

```bash
git config --global init.defaultBranch main
git config --global pull.rebase true
git config --global core.editor "code --wait"
```

> **易错：** `pull.rebase true` 会改变 `git pull` 行为。团队成员最好保持一致，否则历史风格容易混乱。

### 常用 alias

```bash
git config --global alias.st "status --short"
git config --global alias.co "switch"
git config --global alias.br "branch"
git config --global alias.cm "commit -m"
git config --global alias.lg "log --oneline --graph --decorate --all"
```

使用：

```bash
git st
git lg
```

### 标签 tag

标签常用于标记发布版本。

轻量标签：

```bash
git tag v1.0.0
```

附注标签：

```bash
git tag -a v1.0.0 -m "Release v1.0.0"
```

查看标签：

```bash
git tag
git show v1.0.0
```

推送单个标签：

```bash
git push origin v1.0.0
```

推送所有标签：

```bash
git push origin --tags
```

删除本地标签：

```bash
git tag -d v1.0.0
```

删除远程标签：

```bash
git push origin :refs/tags/v1.0.0
```

> **重点：** 发布版本建议使用附注标签，因为它包含创建者、时间和说明。

> **易错：** tag 默认只在本地创建，不会自动推送到远程。

### 版本号建议

常见语义化版本格式：

```text
v1.2.3
主版本.次版本.修订版本
```

- 主版本：不兼容变化。
- 次版本：向后兼容的新功能。
- 修订版本：向后兼容的问题修复。

### 阶段 6 要点

> **重点：** `.gitignore` 是仓库规则，`git config` 是用户或仓库配置，tag 是历史上的固定标记。

> **难点：** 被 Git 跟踪过的文件不会因为加入 `.gitignore` 就自动消失。

> **易错：** 不要提交 `.env`、私钥、访问令牌、真实生产配置。
