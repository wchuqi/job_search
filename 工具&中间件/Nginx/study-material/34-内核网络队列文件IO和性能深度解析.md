# Nginx学习资料：内核网络队列、文件 IO 和性能深度解析

[返回索引](../Nginx学习资料.md)

## 学习目标

- 理解 Nginx 性能与内核网络、文件 IO、FD、队列的关系。
- 掌握 sendfile、tcp_nopush、keepalive、backlog、ulimit。
- 能做容量和瓶颈分析。

## 理论导读

Nginx 性能瓶颈可能在用户态配置，也可能在内核：accept 队列、syn 队列、文件描述符、端口耗尽、磁盘 IO、page cache、网络带宽、CPU 中断。静态文件服务和反向代理的瓶颈不同。

## 文件描述符

```bash
ulimit -n
systemctl show nginx -p LimitNOFILE
```

Nginx worker_connections 不能超过系统允许的 FD 能力。

## sendfile

sendfile 让内核在文件和 socket 之间传输数据，减少用户态复制。适合静态文件，不适合所有动态或加密场景，需要实测。

## 队列

- SYN queue。
- accept queue。
- upstream 连接等待。
- 应用队列。

队列满时可能出现连接失败或延迟尖刺。

## 验收

- 能解释 FD 上限。
- 能区分静态文件和反向代理瓶颈。
- 能从 ss、top、日志和系统指标分析性能。

## 易错

> **易错：** Nginx 502 就只看 Nginx 配置。
>
> 正确做法：同时看内核队列、upstream 连接、DNS、TLS、网络和应用日志。
