# Nginx学习资料：故障排查

[返回索引](../Nginx学习资料.md)

## 学习目标

- 掌握 Nginx 常见状态码排查。
- 使用日志、curl、nginx -T、ss、tcpdump 定位问题。
- 建立生产排障路径。

## 状态码

- 301/302：重定向配置。
- 400：请求非法、header 太大、协议错误。
- 403：权限、deny、目录无 index。
- 404：路径、root/alias、try_files。
- 413：body 超过限制。
- 499：客户端主动断开。
- 502：upstream 连接失败或响应非法。
- 504：upstream 超时。

## 通用命令

```bash
nginx -t
nginx -T
tail -f /var/log/nginx/error.log
curl -v -H 'Host: example.com' http://127.0.0.1/path
ss -lntp
```

## 排查路径

1. 请求是否到 Nginx。
2. 命中了哪个 server。
3. 命中了哪个 location。
4. 静态文件路径或 proxy_pass 是否正确。
5. upstream 是否可连接。
6. 超时和 buffer 是否合适。
7. 客户端是否断开。

## 练习

1. 制造 404 root/alias 错误。
2. 制造 502 后端关闭。
3. 制造 504 后端 sleep。

## 验收

- 能根据状态码选择排查路径。
- 能用 `nginx -T` 找真实配置。
- 能用 curl 指定 Host 验证。

## 易错

> **易错：** 502 一律认为后端挂了。
>
> 正确做法：检查 DNS、连接、TLS、upstream 响应、header、超时和日志。
