# Nginx学习资料：深度实验手册和能力验收

[返回索引](../Nginx学习资料.md)

## 实验 1：server 和 Host 匹配

配置两个 server，用 curl 指定 Host，验证命中。

```bash
curl -H 'Host: a.example.com' http://127.0.0.1/
```

## 实验 2：location 匹配

配置精确、前缀、`^~`、正则 location，记录不同 URI 的命中结果。

## 实验 3：root、alias、try_files

分别配置 root 和 alias，验证实际文件路径。配置 SPA fallback，确保 `/api/` 不被 fallback。

## 实验 4：proxy_pass URI 替换

后端打印收到的 URI，对比 `proxy_pass http://backend/` 和 `proxy_pass http://backend`。

## 实验 5：upstream 重试

两个后端，一个返回 502，一个正常，观察重试和日志。

## 实验 6：TLS 和 SNI

配置两个 HTTPS 域名，用 `openssl s_client -servername` 验证证书。

## 实验 7：缓存和限流

配置 proxy_cache、cache lock、limit_req，观察 HIT/MISS 和限流响应。

## 实验 8：故障注入

制造：

- 后端关闭：502。
- 后端慢：504。
- 客户端超时：499。
- alias 错误：404。
- 证书过期或链错误：TLS 失败。

## 最终能力验收

- 能推导 server/location。
- 能解释 proxy_pass 路径。
- 能排查 404/499/502/504。
- 能配置 TLS、缓存、限流和日志。
- 能写生产发布 checklist 和事故复盘。
