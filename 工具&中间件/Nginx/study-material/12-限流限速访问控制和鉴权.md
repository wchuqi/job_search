# Nginx学习资料：限流、限速、访问控制和鉴权

[返回索引](../Nginx学习资料.md)

## 学习目标

- 掌握 limit_req、limit_conn、allow/deny、auth_basic。
- 理解限流 key、突发、延迟和拒绝。
- 能设计入口保护策略。

## 理论导读

Nginx 限流用于保护后端和入口资源。`limit_req` 控制请求速率，`limit_conn` 控制并发连接数。限流必须选对 key，如 IP、用户、API key。过于粗糙的 key 会误伤，过于细粒度又无法保护全局资源。

## limit_req

```nginx
limit_req_zone $binary_remote_addr zone=perip:10m rate=10r/s;

location /api/ {
    limit_req zone=perip burst=20 nodelay;
    proxy_pass http://backend;
}
```

## limit_conn

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;

location /download/ {
    limit_conn addr 5;
}
```

## 访问控制

```nginx
allow 10.0.0.0/8;
deny all;
```

## Basic Auth

```nginx
auth_basic "restricted";
auth_basic_user_file /etc/nginx/.htpasswd;
```

## 练习

1. 配置 IP 级限流。
2. 用压测工具触发 503/429 类响应。
3. 配置内网 allow/deny。

## 验收

- 能解释 rate、burst、nodelay。
- 能区分限流和限并发。
- 能选择合适限流 key。

## 重点

- 限流 key 决定保护效果和误伤范围。
- Nginx 限流是入口保护，不替代应用鉴权。
- 对真实客户端 IP 要结合代理链路。

## 易错

> **易错：** 在多层代理后直接用 `$remote_addr` 限流，结果按上一层代理 IP 限制。
>
> 正确做法：正确配置 real_ip 后再选择限流 key。
