# Nginx学习资料：静态资源、索引、目录和缓存头

[返回索引](../Nginx学习资料.md)

## 学习目标

- 配置静态文件服务。
- 掌握 index、autoindex、expires、etag、cache-control。
- 避免目录暴露和缓存错误。

## 理论导读

Nginx 静态文件性能高，但安全和缓存配置必须谨慎。静态资源通常适合长缓存，HTML 入口通常不适合长缓存。目录列表默认不应开启，文件路径要避免 alias/root 配错导致泄漏。

## 静态文件

```nginx
server {
    listen 80;
    root /var/www/site;
    index index.html;

    location /assets/ {
        expires 30d;
        add_header Cache-Control "public";
    }
}
```

## HTML 和 assets 缓存

```nginx
location = /index.html {
    expires -1;
    add_header Cache-Control "no-cache";
}

location /assets/ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

前提是 assets 文件名带 hash。

## autoindex

```nginx
autoindex off;
```

除非明确需要下载目录，否则不要开启目录列表。

## 练习

1. 配置静态站点。
2. 为带 hash 的 JS/CSS 加长缓存。
3. 禁止访问隐藏文件。

## 验收

- 能区分 HTML 和静态 assets 缓存策略。
- 能避免目录列表暴露。
- 能用 curl 查看缓存头。

## 重点

- HTML 入口和带 hash 静态资源缓存策略不同。
- 静态文件路径必须确认真实映射。
- 不要暴露 `.git`、配置文件和备份文件。

## 易错

> **易错：** 给 `index.html` 配很长缓存，导致前端发布后用户看旧页面。
>
> 正确做法：HTML 短缓存或 no-cache，带 hash assets 长缓存。
