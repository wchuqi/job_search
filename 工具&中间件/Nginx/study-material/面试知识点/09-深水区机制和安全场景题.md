## 一、深水区机制

### 1. reload 后旧连接会怎样？

**参考答案：**

reload 成功后 master 启动新 worker 处理新连接，旧 worker 不再接新连接，但会继续处理已有连接直到结束或超时。因此长连接可能继续使用旧配置一段时间。

### 2. `$uri` 和 `$request_uri` 有什么区别？

**参考答案：**

`$request_uri` 是原始请求 URI，包含查询字符串；`$uri` 是规范化后的 URI，不含查询字符串，可能被 rewrite 改变。缓存 key、文件路径和代理转发使用哪个变量会影响行为。

### 3. 如何正确获取真实客户端 IP？

**参考答案：**

只信任明确的上游代理 IP 段，配置 `set_real_ip_from` 和 `real_ip_header`，必要时使用 PROXY protocol。不能无条件信任客户端传来的 `X-Forwarded-For`。

### 4. Nginx 中 DNS 解析有什么坑？

**参考答案：**

静态 upstream 域名可能在启动或 reload 时解析，后端 IP 变化后不一定自动更新。容器和 Kubernetes 环境应使用稳定 Service、resolver 或平台控制的配置更新。

### 5. 大文件上传为什么可能打满磁盘？

**参考答案：**

请求体超过 client_body_buffer_size 会写入 client body 临时文件；proxy buffering 也可能把上游响应写入临时文件。必须规划 temp_path 磁盘和监控。

## 二、安全场景

### 6. Host header injection 如何防？

**参考答案：**

配置明确 server_name，默认 server 拒绝未知 Host；应用不要盲目信任 Host 生成敏感链接；日志记录 Host；代理到后端时传递经过验证的 Host。

### 7. request smuggling 本质是什么？

**参考答案：**

前后端对请求边界解析不一致，尤其 Content-Length 和 Transfer-Encoding 处理不同，导致一个请求被拆成不同边界。要统一代理和后端协议处理，避免危险 header 透传。

### 8. 如何防止变量 proxy_pass 造成 SSRF？

**参考答案：**

上游地址必须来自白名单 map 或固定 upstream，不能直接使用用户参数拼接 host。还要限制 resolver 和内网访问边界。

### 9. 499 增加如何分析？

**参考答案：**

检查客户端超时、CDN/LB 超时、Nginx request_time、upstream_response_time、响应体大小、网络质量和后端慢请求。499 表示客户端先断开，不等于 Nginx 一定故障。

### 10. 如何做端到端入口架构评审？

**参考答案：**

画出 Client、DNS、CDN、LB、Nginx、应用链路；明确 TLS、真实 IP、缓存、限流、超时、重试、trace id、错误页、灰度、回滚和告警在每层的职责。
