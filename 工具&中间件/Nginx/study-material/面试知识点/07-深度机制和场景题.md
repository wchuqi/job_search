## 一、深度机制

### 1. Nginx 配置为什么不是简单顺序执行？

**参考答案：**

Nginx 配置在启动或 reload 时解析成模块配置结构，请求到来后按 server/location 匹配和请求处理阶段执行。rewrite、access、content、log 等阶段决定模块执行顺序，而不是配置文件文本顺序。

### 2. 正则 location 和前缀 location 谁优先？

**参考答案：**

先找最长前缀；若最长前缀带 `^~`，直接使用；否则按配置顺序尝试正则，正则命中则使用正则；未命中才使用最长前缀。

### 3. `add_header` 为什么在某些 location 不生效？

**参考答案：**

`add_header` 有状态码生效范围，常需 `always`；此外在子级重新声明时可能影响父级继承。要结合具体 context 和最终配置分析。

### 4. 为什么慢客户端会影响 upstream？

**参考答案：**

如果 Nginx 不缓冲 upstream 响应，慢客户端会让 upstream 连接长期占用。开启 proxy buffering 可让 Nginx 先读完响应，再慢慢发给客户端，保护 upstream。

### 5. 如何解释一次 502？

**参考答案：**

检查 upstream 是否可连接、DNS 是否正确、连接超时、TLS 握手、后端是否关闭连接、响应头是否合法、buffer 是否不足，并结合 error_log 具体错误。
