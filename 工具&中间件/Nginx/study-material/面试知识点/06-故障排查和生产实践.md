## 一、故障排查

### 1. 404 怎么排查？

**参考答案：**

确认请求命中的 server 和 location，检查 root/alias 映射路径，检查 try_files，查看 error_log 中 open file 路径，用 `nginx -T` 确认最终配置。

### 2. 499 是什么？

**参考答案：**

499 是 Nginx 记录的客户端主动断开连接。可能是客户端超时、用户取消、Nginx 或 upstream 响应太慢导致客户端放弃。

### 3. 修改配置后如何安全发布？

**参考答案：**

先 `nginx -t`，再保存 `nginx -T` 快照，reload，执行健康检查，观察日志和指标，保留回滚方案。

### 4. 如何判断请求命中了哪个 server/location？

**参考答案：**

用 curl 指定 Host，结合 `nginx -T` 推导；也可临时在 location 加响应头或不同日志字段，在实验环境验证。
