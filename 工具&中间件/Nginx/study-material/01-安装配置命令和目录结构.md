# Nginx学习资料：安装、配置、命令和目录结构

[返回索引](../Nginx学习资料.md)

## 学习目标

- 掌握 Nginx 常用命令和配置检查。
- 理解主配置、conf.d、sites-enabled 的组织差异。
- 知道 reload 和 restart 的区别。

## 理论导读

Nginx 配置错误会直接影响入口流量。任何变更前都应先 `nginx -t`，必要时 `nginx -T` 查看完整展开配置。reload 会让 master 重新加载配置并平滑替换 worker，通常比 restart 更适合生产变更。

## 常用命令

```bash
nginx -v
nginx -V
nginx -t
nginx -T
nginx -s reload
systemctl status nginx
systemctl reload nginx
journalctl -u nginx
```

## 目录结构

常见位置：

```text
/etc/nginx/nginx.conf
/etc/nginx/conf.d/*.conf
/etc/nginx/sites-enabled/*
/var/log/nginx/access.log
/var/log/nginx/error.log
/usr/share/nginx/html
```

不同发行版目录不同，生产要以 `nginx -T` 输出为准。

## reload 和 restart

- reload：重新加载配置，启动新 worker，旧 worker 处理完已有连接后退出。
- restart：停止再启动，可能中断连接。

## 练习

1. 执行 `nginx -T` 找到完整配置。
2. 故意写错配置，执行 `nginx -t`。
3. 比较 reload 和 restart 对连接的影响。

## 验收

- 能检查配置语法。
- 能查看编译模块。
- 能说明 reload 的平滑过程。

## 重点

- 生产变更先 `nginx -t`。
- `nginx -T` 是排查配置真实状态的关键。
- 不同发行版配置目录不同。

## 易错

> **易错：** 修改了未被 include 的配置文件。
>
> 正确做法：用 `nginx -T` 确认文件是否进入最终配置。
