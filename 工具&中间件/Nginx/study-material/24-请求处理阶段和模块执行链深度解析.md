# Nginx学习资料：请求处理阶段和模块执行链深度解析

[返回索引](../Nginx学习资料.md)

## 学习目标

- 理解 Nginx HTTP 请求处理阶段。
- 掌握 rewrite、access、content、log 等阶段职责。
- 能解释指令为什么在某些位置生效或不生效。

## 理论导读

Nginx 是模块化服务器。HTTP 请求经过多个阶段，不同模块在不同阶段注册 handler。rewrite 阶段修改 URI，access 阶段做访问控制，content 阶段生成响应或代理到上游，log 阶段记录日志。理解阶段有助于解释 rewrite、auth、proxy、try_files 的行为。

## 常见阶段心智模型

```text
post read
server rewrite
find config
rewrite
post rewrite
preaccess
access
post access
try files
content
log
```

实际阶段更细，学习时重点看“匹配配置”和“模块在哪个阶段工作”。

## rewrite 和 content

`rewrite` 改 URI，可能触发重新匹配 location。`proxy_pass`、静态文件、return 等通常在生成响应阶段发挥作用。

## 练习

1. 配置 rewrite last，观察重新匹配。
2. 配置 access deny，观察 content 不执行。
3. 用 error_log debug 观察阶段，实验环境使用。

## 验收

- 能解释 rewrite 和 content 阶段。
- 能说明 access 控制发生在代理前。
- 能推导内部重定向。

## 易错

> **易错：** 以为写在配置文件前面的指令一定先执行。
>
> 正确做法：按请求阶段和 location 匹配推导。
