# Codex学习资料：IDE和Cloud工作流

[返回索引](../Codex学习资料.md)

## 学习目标

- 理解 IDE 与 Cloud Codex 的使用差异。
- 掌握适合远程异步任务的拆分方式。
- 能把 Codex 输出接入 PR 和代码审查流程。

## 理论导读

IDE 中的 Codex 更贴近你正在看的代码，适合局部解释、编辑建议和交互式修改。Cloud 中的 Codex 更适合异步执行较完整的任务，例如探索方案、修复 issue、生成 PR 或运行较长验证。

## IDE 工作流

适合：

- 解释当前文件或函数。
- 快速生成小片段。
- 对选中代码做重构建议。
- 审查当前 diff。
- 结合人工导航做局部修复。

注意：

- 当前打开文件不是完整上下文。
- 复杂问题仍要让 Codex 搜索调用链。
- IDE 建议要经过测试和 code review。

## Cloud 工作流

适合：

- 异步处理 issue。
- 多方案并行探索。
- 自动生成分支或 PR。
- 运行云端可用的测试和构建。

关键准备：

- 明确基准分支。
- 确认依赖安装方式。
- 明确环境变量和密钥边界。
- 指定期望输出，例如 PR 描述、测试结果、风险清单。

## 任务拆分示例

大型目标：

```text
重构支付模块。
```

拆分后：

1. 只读分析支付模块调用链和风险。
2. 提出 2-3 个重构方案，不修改代码。
3. 选择一个低风险子目标，例如抽离金额格式化逻辑。
4. 修改代码并补测试。
5. 生成 PR 描述和回滚说明。

## 练习

用 Cloud 或 IDE 完成一个小型 PR：

- 给出 issue 背景。
- 要求 Codex 先输出计划。
- 允许它修改代码。
- 要求它运行测试。
- 最后审查 PR 描述是否包含风险。

## 验收

- 能说明 IDE 和 Cloud 的上下文差异。
- 能为云端任务提供分支、依赖、测试和输出要求。
- 能审查 Codex 生成的 PR 是否满足团队标准。

## 重点

- IDE 适合即时协作，Cloud 适合异步执行。
- Cloud 任务必须显式说明环境和验收，否则容易出现“云端完成但本地不可复现”。

## 难点

- 云端任务并行时要防止多个分支解决同一问题但修改方向冲突。

## 易错

> **易错：** 把 Cloud 任务当成无需复核的自动外包。
>
> 正确做法：Cloud 输出必须进入正常 PR 审查、测试和合并流程。
