## 高频考察点

- Codex 与代码补全工具的区别。
- 如何给 Codex 一个可执行、可验收的任务。
- `AGENTS.md` 的价值和内容边界。
- CLI 权限、沙箱、审批的安全意义。
- 如何审查 Codex 生成的代码。
- 什么时候应该停止 Codex 继续修改。
- 如何把 Codex 接入团队 PR、CI 和安全流程。
- Codex 一次任务的执行生命周期。
- sandbox、approval、network、auto review 的关系。
- MCP 工具设计中的权限放大和 prompt injection。
- 长上下文、模型选择和成本治理。
- 团队审计、遥测、PR 标注和事故复盘。

## 面试回答标准

好的回答应包含：

- 直接结论。
- 适用场景。
- 风险边界。
- 具体例子。
- 验证方式。

差的回答通常只有：

- “Codex 能自动写代码。”
- “提示词写详细点就行。”
- “生成后跑一下测试。”

这些说法过于粗糙，不能体现工程判断。
