# Codex学习资料：综合练习和验收

[返回索引](../Codex学习资料.md)

## 练习 1：只读代码审查

目标：让 Codex 审查一个仓库，不修改任何文件。

任务提示词：

```text
请只读分析当前仓库的认证模块。
不要修改文件，不要运行会改变状态的命令。
请输出：
1. 模块边界。
2. 关键调用链。
3. 潜在风险。
4. 建议补充的测试。
```

验收：

- 输出包含文件路径和证据。
- 没有修改工作区。
- 风险建议具体，不是泛泛而谈。

## 练习 2：最小 bug 修复

目标：让 Codex 修复一个可复现 bug。

要求：

- 提供复现步骤和错误日志。
- 限制修改范围。
- 要求先说明根因。
- 要求补回归测试。
- 要求运行最小相关测试。

验收：

- 修复前测试失败，修复后测试通过。
- diff 集中。
- 没有无关重构。
- 最终答复包含测试命令和结果。

## 练习 3：编写 AGENTS.md

目标：为仓库建立稳定的 Codex 使用规范。

内容必须包括：

- 项目概览。
- 构建和测试命令。
- 代码风格。
- 目录说明。
- 安全边界。
- 最终输出要求。

验收：

- 新成员和 Codex 都能根据文件运行正确命令。
- 文件不包含一次性任务。
- 安全规则明确可执行。

## 练习 4：权限策略设计

目标：为三类任务设计权限。

任务：

1. 只读审查。
2. 修复单元测试。
3. 生成云端 PR。

验收：

- 每类任务都有允许操作和审批操作。
- 能说明为什么不能默认最高权限。
- 能说明密钥和网络访问策略。

## 练习 5：失败任务恢复

目标：训练 Codex 偏离目标后的收敛能力。

步骤：

1. 给 Codex 一个模糊任务。
2. 观察它产生过大 diff。
3. 要求它停止修改并解释已做假设。
4. 重新给出最小目标。
5. 让它只改必要文件。

验收：

- 能识别偏离原因。
- 能恢复到可审查变更。
- 能说明哪些修改应丢弃。

## 最终能力验收

完成学习后，你应能独立做到：

- 选择合适的 Codex 入口。
- 写出可执行任务说明。
- 编写和维护 `AGENTS.md`。
- 使用 CLI 完成本地修改和验证。
- 配置合理权限和审批策略。
- 审查 Codex 生成代码。
- 处理任务失败和环境问题。
- 把 Codex 接入团队 PR/CI 流程。
