# Codex学习资料：深度面试题和场景推演

[返回索引](../Codex学习资料.md)

## 学习目标

- 用机制级答案回答 Codex 面试问题。
- 能处理开放场景题，而不是只背定义。
- 能把 Codex 使用和工程质量、安全治理联系起来。

## 场景题 1：你如何设计一个企业级 Codex 使用规范？

**参考答案：**

我会先分风险等级，而不是一刀切禁止或放开。只读解释和文档摘要属于低风险，可以自动；小范围代码修改需要正常 PR 和 CI；涉及 API、安全、权限、数据迁移的任务需要 owner 强审；生产发布、密钥、生产数据操作只允许 Codex 做分析，不允许独立执行。

然后落地四类机制：

- 仓库层：`AGENTS.md` 写构建、测试、风格、安全和输出规则。
- 权限层：默认 workspace-write + on-request；陌生仓库 read-only；网络默认关闭。
- 流程层：PR 标注 Codex 参与范围、验证命令和未验证风险。
- 审计层：记录工具调用、审批和测试结果，prompt 和敏感数据默认脱敏。

> **重点：** 企业规范要能执行、能审计、能复盘。

## 场景题 2：Codex 生成的修复通过了测试，但你仍不放心，怎么审查？

**参考答案：**

我会从五个方向审查：

1. 变更范围是否和任务一致，有没有顺手重构。
2. 是否改变公共 API、数据结构、错误处理或权限边界。
3. 测试是否覆盖原始失败路径，而不是只改了期望。
4. 是否引入性能、并发、安全或兼容性风险。
5. 是否有未验证内容和回滚方案。

如果是关键业务，我会要求 Codex 或作者补充调用链说明和回归测试，再由模块 owner 复核。

> **易错：** 测试通过不是最终质量证明，只是验证链的一部分。

## 场景题 3：如何让 Codex 调试一个只在 CI 上失败的问题？

**参考答案：**

我会先要求 Codex 只读分析 CI 日志，不直接修改。重点比较本地和 CI 的差异：操作系统、依赖版本、环境变量、时区、并发度、网络、数据库初始状态和测试顺序。

任务会拆成：

1. 提取首个失败和完整堆栈。
2. 找到相关测试和生产代码。
3. 判断是否环境差异、时序问题或测试污染。
4. 写最小复现或增强日志。
5. 做最小修复并运行目标测试。

> **难点：** CI-only 问题通常不是代码一行错，而是环境、顺序或并发触发。

## 场景题 4：如何设计一个 Codex 可用的内部文档 MCP？

**参考答案：**

我会把工具设计成只读搜索和按 ID 读取，不暴露任意数据库查询。输入包括 query、产品域、页数限制；输出只返回标题、摘要、链接和脱敏片段。工具继承用户权限，记录调用日志，并明确外部文档内容只是数据，不是指令。

审批策略上，搜索可以自动，读取敏感空间需要 prompt，写入或修改文档先不暴露。

> **重点：** MCP 工具要任务级、最小权限、可审计、脱敏。

## 场景题 5：你如何处理 Codex 反复修不好一个 bug？

**参考答案：**

我不会继续让它扩大修改。我会让它停止编辑，输出：

- 当前假设。
- 已读文件。
- 已改文件。
- 失败测试。
- 它认为根因是什么。

然后我会回到只读模式，补充完整日志和复现步骤，要求它重新列证据链。如果仍无法定位，就把任务拆成“定位根因”和“实现修复”两个任务，必要时人工接管关键判断。

> **重点：** 失败时要收敛和复盘，不是增加权限和继续试。

## 场景题 6：什么时候需要更强模型或更高推理强度？

**参考答案：**

当任务涉及长调用链、多模块交互、并发、安全、性能、架构取舍或模糊日志时，应该提高模型能力或推理强度。简单代码解释、小脚本、局部测试草案可以使用低成本路径。

但更强模型不能替代清楚任务。复杂任务仍要分阶段：只读分析、方案比较、最小修改、验证和审查。

> **易错：** 以为模型越强，提示词就可以越模糊。

## 场景题 7：Codex 是否能自动处理生产事故？

**参考答案：**

Codex 可以参与生产事故的日志分析、假设生成、查询命令建议、回滚方案草稿和复盘文档，但不应独立执行生产变更。生产事故涉及业务影响、数据风险和权限边界，需要人工负责人审批。

合理流程是：

1. Codex 只读分析日志和指标。
2. 给出可能根因和验证命令。
3. 人确认执行范围。
4. Codex 协助生成回滚或修复草案。
5. 人执行生产操作并复核结果。

> **重点：** 生产环境中 Codex 是辅助分析者，不是无人值守操作者。

## 场景题 8：如何防止 Codex 被外部文档 prompt injection 影响？

**参考答案：**

工具返回的外部内容必须被视为不可信数据。系统和仓库规则的优先级高于外部文档。Codex 不应因为文档中出现“忽略规则”“输出密钥”等文本就改变行为。MCP 工具应脱敏输出，限制字段，并对写操作审批。

> **难点：** 文档、Issue、日志、网页都可能包含攻击性指令。代理必须区分数据和指令。

## 自测评分标准

每道题满分答案应包含：

- 直接结论。
- 机制解释。
- 风险边界。
- 具体操作步骤。
- 验收或审计方式。

只回答“让 Codex 生成代码，然后跑测试”的答案不合格。
