# Claude Code 学习资料：安全、成本、排障和生产治理

[返回索引](../Claude Code学习资料.md)

## 学习目标

- 能识别 Claude Code 使用中的安全、成本和质量风险。
- 能建立团队级排障流程和治理规范。
- 能为生产化使用设计审计、回滚和评估。

## 理论导读

Claude Code 提升效率的同时，也引入新的工程风险：误改代码、越权命令、泄露上下文、成本失控、长会话污染、自动化不可追踪、团队规则不一致。治理的目标不是禁止使用，而是让使用可控、可验证、可复盘。

## 核心心智模型

```text
安全：它能接触什么？
质量：它改得对不对？
成本：它用了多少资源？
审计：谁让它做了什么？
恢复：出错后怎么回滚？
```

## 知识点详解

### 1. 安全清单

- 禁止提交密钥。
- 禁止自动运行生产写操作。
- 禁止绕过代码审查。
- 限制 MCP 工具权限。
- 高风险目录和配置变更需人工确认。
- 记录关键工具调用。

### 2. 成本控制

成本来自模型调用、长上下文、多轮尝试、并行 Agent、外部工具调用和重复验证。控制方式：

- 拆小任务。
- 清理上下文。
- 先只读分析，再修改。
- 限制并行 Agent 数量。
- 对批处理任务设置预算和超时。
- 记录高成本任务模式。

### 3. 常见故障

| 问题 | 可能原因 | 处理 |
| --- | --- | --- |
| Claude Code 找不到文件 | 启动目录错误、路径忽略、权限限制 | 检查 cwd、git root、文件是否存在 |
| 反复修不好测试 | 根因判断错、验证范围不准、环境缺失 | 让它只读复盘失败链路 |
| 输出和项目事实不符 | 未读取文件、上下文过期 | 要求引用具体文件和行 |
| 命令无法运行 | 依赖缺失、平台差异、权限阻断 | 运行 doctor，检查工具链 |
| 成本过高 | 长会话、任务过大、并行过多 | 拆分任务，重开会话 |

### 4. 团队治理

团队应建立：

- 标准 `CLAUDE.md` 模板。
- 权限策略和禁止命令。
- PR 模板和 AI 使用说明。
- 敏感信息检查。
- 评估样例和回归任务。
- 事故复盘流程。

## 例子

### 失败任务复盘提示

```text
刚才连续两次修复都失败。
请停止修改，只读复盘：
1. 我们的目标是什么
2. 做过哪些改动
3. 哪些假设被验证为错误
4. 当前最小复现是什么
5. 下一步应先收集什么证据
```

### 成本控制提示

```text
请用最小上下文完成任务：
- 只读取与错误日志直接相关的文件
- 修改前先给出文件清单
- 不启动并行 Agent
- 验证只运行相关测试
```

## 练习

1. 写一份 Claude Code 安全使用 checklist。
2. 设计一次“AI 误改导致测试失败”的事故复盘模板。
3. 为团队建立 10 个回归评估任务，验证 Claude Code 是否遵守项目规则。

## 验收

- 能定位 Claude Code 任务失败属于上下文、权限、环境、推理还是验证问题。
- 能设计成本和权限控制方式。
- 能写出团队治理规范。

## 重点

- 治理目标是可控使用，不是禁止使用。
- 没有验证和审计的自动化不可进入团队关键流程。
- 成本控制靠任务拆分和上下文治理。

## 易错

- **易错：** 失败后继续让 Claude Code“再试试”。
  正确做法：停止修改，先复盘证据链和错误假设。
- **易错：** 只统计调用成本，不统计人工复核和返工成本。
  正确做法：评估总成本，包括误报、漏报、返工和等待时间。
