# Claude Code 学习资料：Agent SDK 和程序化集成

[返回索引](../Claude Code学习资料.md)

## 学习目标

- 理解 Claude Agent SDK 适合解决什么问题。
- 能设计程序化调用 Claude Code 能力的边界。
- 能区分 CLI 自动化、SDK 集成和 MCP 工具开发。

## 理论导读

Agent SDK 用于把 Claude Code/Agent 能力嵌入到应用、工具链或内部平台中。与手动 CLI 相比，SDK 更适合需要统一输入输出、权限、日志、评估和产品化体验的场景。

不要为了“自动化”过早上 SDK。先用 CLI 验证流程，再把稳定流程固化成脚本、命令、MCP 或 SDK。

## 核心心智模型

```text
CLI：人驱动的开发协作
脚本：重复任务批处理
MCP：给 Agent 增加外部工具
SDK：把 Agent 能力嵌入自己的系统
```

## 知识点详解

### 1. SDK 适用场景

- 内部代码审查平台。
- 自动生成迁移计划。
- 批量仓库升级建议。
- 与 Issue/PR/CI 系统深度集成。
- 构建企业内部 AI 开发助手。

### 2. SDK 设计要点

- 输入限制：明确任务、仓库、文件范围、权限。
- 输出结构：结构化 JSON 或固定 Markdown。
- 工具边界：哪些命令、文件、MCP 工具可用。
- 审计：记录 prompt、工具调用、输出和人类确认。
- 评估：用历史任务回放，统计误报、漏报和成功率。
- 成本：设置预算、超时和最大上下文。

### 3. 与 MCP 的关系

MCP 是工具协议，SDK 是构建 Agent 应用的编程接口。你可以在 SDK 应用中使用 MCP 工具，也可以让 Claude Code CLI 使用 MCP 工具。两者不是替代关系。

## 例子

### 代码审查平台设计

```text
输入：PR diff、相关文件、测试结果、Issue 背景
Agent 任务：只读审查 bug、安全、回归和缺失测试
工具：读取仓库文件、检索规范、查询 CI
输出：结构化 findings
权限：禁止修改文件、禁止执行部署命令
审计：保存输入摘要、模型版本、工具调用和最终评论
```

### SDK 集成前的验证路径

```text
1. 手动 CLI 跑 20 个真实 PR 审查
2. 总结稳定提示和输出格式
3. 用非交互 CLI 批处理小规模验证
4. 建立评估集
5. 再进入 SDK 产品化
```

## 练习

1. 设计一个“AI PR Reviewer”的输入、输出、权限和失败处理。
2. 设计 20 条评估样例，覆盖真实 bug、误报、无问题 diff。
3. 说明为什么这个场景用 SDK，而不是只用 CLI。

## 验收

- 能区分 CLI、脚本、MCP 和 SDK。
- 能设计 SDK 应用的权限、审计和评估。
- 能避免把一次性 prompt 直接产品化。

## 重点

- SDK 适合平台化，不适合一开始就过度工程化。
- 程序化集成必须有结构化输出和评估集。
- 自动化越深，审计和权限越重要。

## 易错

- **易错：** 只验证一个成功 demo 就上线 SDK。
  正确做法：用历史任务和失败样例做回归评估。
- **易错：** SDK 应用拥有完整仓库和生产系统权限。
  正确做法：按任务最小授权，默认只读。
