# Claude Code 学习资料：项目记忆、CLAUDE.md 和上下文治理

[返回索引](../Claude Code学习资料.md)

## 学习目标

- 会用 `CLAUDE.md` 固化项目规则、测试命令、约束和协作习惯。
- 理解用户级、项目级、子目录级上下文的职责边界。
- 能防止上下文污染、规则冲突和敏感信息泄漏。

## 理论导读

Claude Code 的质量很大程度取决于上下文。`CLAUDE.md` 是给 Agent 的项目说明书，常放在仓库根目录，也可以按子目录补充局部规则。它应该写“可执行的工程事实”，而不是长篇背景介绍。

好的 `CLAUDE.md` 能回答：

- 这个项目如何启动、测试、构建？
- 哪些目录可以改，哪些目录不要改？
- 代码风格、提交规范、测试策略是什么？
- 遇到不确定或高风险操作时应该怎么做？
- 哪些外部服务、环境变量、生成文件需要特殊处理？

## 核心心智模型

`CLAUDE.md` 不是 README 的替代品，而是“Agent 操作手册”。README 面向人类理解项目，`CLAUDE.md` 面向 AI 在项目中行动。

| 内容 | 适合放入 `CLAUDE.md` | 不适合放入 `CLAUDE.md` |
| --- | --- | --- |
| 命令 | 测试、构建、lint、格式化 | 临时本机路径 |
| 约束 | 禁止修改生成目录、迁移需确认 | 含糊的价值观口号 |
| 安全 | 密钥处理、外部请求限制 | 明文 token、账号密码 |
| 流程 | 修改前先读文件、改后跑测试 | 过期会议记录 |

## 知识点详解

### 1. 推荐模板

```markdown
# Claude Code 项目指南

## 项目概况

- 技术栈：
- 主要模块：
- 包管理器：

## 常用命令

- 安装依赖：
- 运行测试：
- 运行单个测试：
- 构建：
- Lint/格式化：

## 修改规则

- 修改前先读取相关文件和测试。
- 优先最小改动，避免无关重构。
- 不要修改生成目录：
- 不要提交密钥、日志、缓存、构建产物。

## 验证要求

- 代码变更后优先运行：
- 无法运行时说明原因和替代验证。

## 高风险操作

- 数据库迁移必须先说明风险和回滚方案。
- 删除文件、批量重命名、修改 CI/CD 配置必须先确认。
```

### 2. 规则冲突处理

当用户消息、项目记忆、子目录规则和工具输出冲突时，应优先遵守更具体、更近、更高优先级的指令，并显式说明冲突。例如用户本轮要求“只读分析”时，即使项目记忆里写着“修复后运行测试”，也不能修改文件。

> **难点：** 规则不是越多越好。过多、过期、互相矛盾的规则会降低 Claude Code 的判断质量。

### 3. 上下文窗口和压缩

长任务中，Claude Code 会逐渐积累对话、文件片段、命令输出和中间计划。上下文太长时，需要压缩或重开。压缩时应保留：

- 当前目标。
- 已完成工作。
- 已修改文件。
- 已知失败和验证结果。
- 剩余风险和下一步。

### 4. 敏感信息治理

永远不要把密钥、个人信息、内部 token 写进 `CLAUDE.md`。如果 Agent 需要使用外部服务，应写变量名、权限边界和只读方式，而不是凭据本身。

## 例子

### 好的规则

```markdown
- 修改 Java 代码后运行 `mvn -q test`；若只改一个模块，优先运行该模块测试。
- `target/`、`dist/`、`coverage/` 是生成目录，不要手工修改。
- 数据库迁移文件必须先生成说明，等待人工确认后再编辑。
```

### 不好的规则

```markdown
- 写出高质量代码。
- 尽量不要犯错。
- 密钥是 sk-xxxx，必要时可以用。
```

## 练习

1. 为当前仓库写一份不超过 80 行的 `CLAUDE.md`。
2. 让 Claude Code 读取 `CLAUDE.md` 后执行一个小任务，检查它是否遵守命令和限制。
3. 故意加入一条过期命令，让 Claude Code 通过读取项目文件纠正它。

## 验收

- `CLAUDE.md` 包含命令、约束、验证、风险规则。
- 没有密钥、账号、内部 token。
- 规则具体、可执行、可验证。
- 子目录特殊规则不会和根目录规则冲突。

## 重点

- `CLAUDE.md` 是 Agent 操作手册。
- 上下文要准确、短、稳定、可验证。
- 敏感信息只写引用方式，不写值。

## 易错

- **易错：** 把所有历史问题都塞进 `CLAUDE.md`。
  正确做法：长期规则放记忆，临时问题放当前会话。
- **易错：** 让 Claude Code 自己长期维护一大段规则但不审查。
  正确做法：规则变更也走代码审查，尤其是权限和命令。
