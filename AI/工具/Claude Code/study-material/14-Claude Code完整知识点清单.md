[返回索引](../Claude Code学习资料.md)

## 一、基础和术语

- Claude Code 定位：终端/开发环境中的 AI 编码 Agent。
- Agentic Coding：模型能读上下文、调用工具、执行步骤、根据反馈迭代。
- 会话：一次持续任务上下文。
- 工具调用：读文件、写文件、运行命令、调用外部工具。
- 权限：限制 Agent 能做什么、何时需要确认。
- 上下文：对话、文件、命令输出、记忆、MCP 返回数据。

## 十五、深度机制清单

- Agent 执行循环：Intake、ContextBuild、Plan、PermissionCheck、Act、Observe、Reflect、Finalize。
- 状态机异常分支：权限拒绝、工具失败、验证失败、上下文漂移。
- 上下文装载：入口文件、关键词搜索、调用链扩展、测试和日志读取。
- 记忆治理：根目录 `CLAUDE.md`、子目录规则、用户偏好、当前会话约束。
- 压缩风险：禁止事项丢失、错误假设残留、失败日志关键行丢失。
- 权限决策算法：动作识别、对象识别、风险分级、配置匹配、任务约束、执行决策。
- 工具路由：读文件、运行命令、调用 MCP、请求用户确认的选择规则。
- 文件编辑安全：Git 状态检查、用户改动保护、最小修改、结构化 diff。
- 验证闭环：最小验证、邻近验证、全量验证、假通过、假失败。
- MCP 深度设计：窄工具、schema、参数校验、脱敏、分页、审计、提示注入隔离。
- Hooks 生命周期：前置风险检查、后置验证、文件修改检查、会话摘要、审计日志。
- Hook 失败策略：fail-open、fail-closed、degrade。
- Subagents 编排：角色拆分、只读并行、结构化输出、冲突消解、串行修改。
- 插件治理：版本、权限声明、兼容性、安全审查、变更记录。
- SDK 平台化：Task API、Context Builder、Policy Engine、Agent Runtime、Result Validator、Human Review、Audit Store。
- 评估体系：离线样例、在线采纳率、误报率、引用准确率、成本和延迟。
- 团队落地：个人试点、小组规范、团队治理、平台化四阶段。
