# Claude Code 学习资料：综合练习项目和能力验收

[返回索引](../Claude Code学习资料.md)

## 项目目标

用 Claude Code 完成一个真实小项目的 AI 辅助开发闭环，并沉淀团队可复用规范。

## 环境假设

- 任选一个已有项目，最好包含测试。
- 使用 Git 管理。
- 可以运行至少一种验证命令。
- 不连接生产环境。

## 任务 1：项目侦察

### 要求

让 Claude Code 只读完成：

- 技术栈识别。
- 目录结构说明。
- 启动、测试、构建命令。
- 风险目录。
- 是否已有 `CLAUDE.md`。

### 验收

- 输出引用实际文件。
- 没有修改文件。
- 能指出至少 3 个后续开发注意事项。

## 任务 2：编写 `CLAUDE.md`

### 要求

创建或更新 `CLAUDE.md`，包含：

- 项目概况。
- 常用命令。
- 修改规则。
- 验证要求。
- 高风险操作。

### 验收

- 不包含密钥。
- 规则具体可执行。
- 与 README 和项目配置一致。

## 任务 3：修复一个小问题

### 要求

选择一个 bug、lint 问题或测试失败。

流程：

1. 先只读定位。
2. 修改前给计划。
3. 最小改动。
4. 运行相关验证。
5. 汇报结果。

### 验收

- Git diff 范围合理。
- 测试通过或说明无法验证原因。
- 没有无关重构。

## 任务 4：设计扩展能力

### 要求

设计以下任意两项：

- MCP 文档检索工具。
- Hook 高风险命令检查。
- Slash Command 代码审查。
- Skill 生成 PR 摘要。
- Subagent 安全审查。

### 验收

- 写明输入、输出、权限、失败处理。
- 默认只读或最小权限。
- 有审计或日志设计。

## 任务 5：团队治理文档

### 要求

写一页团队 Claude Code 使用规范，覆盖：

- 允许和禁止操作。
- 代码评审要求。
- 测试要求。
- 敏感信息处理。
- 失败复盘。

### 验收

- 可以直接放进仓库。
- 与现有研发流程兼容。
- 明确人工确认点。

## 最终自测问题

1. Claude Code 为什么需要 `CLAUDE.md`？
2. 什么时候应该重开会话？
3. 如何防止 Claude Code 误删文件？
4. MCP 和 Hooks 的边界是什么？
5. 如何设计只读数据库 MCP？
6. Subagents 并行时如何避免冲突？
7. Claude Code 修复失败时，下一步该做什么？
8. 如何把 Claude Code 放入 PR 流程？
9. SDK 产品化前要验证什么？
10. 团队如何统计 Claude Code 的实际收益和风险？

## 最终能力验收

- 能独立使用 Claude Code 完成一个开发闭环。
- 能写出项目级上下文和权限规则。
- 能设计至少两种扩展机制。
- 能解释安全、成本、排障和团队治理方案。
- 能在面试中用真实场景回答 Claude Code 相关问题。
