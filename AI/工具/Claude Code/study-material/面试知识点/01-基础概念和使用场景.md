## 一、基础概念

### 1. Claude Code 是什么？

**参考答案：**

Claude Code 是运行在开发环境中的 AI 编码 Agent。它可以在终端或相关集成环境里读取代码库上下文、调用工具、编辑文件、运行命令，并根据测试或命令结果继续迭代。它和普通聊天机器人的区别在于，普通聊天主要生成建议或代码片段，Claude Code 可以参与完整开发闭环。

> **重点：** 回答时要突出“代码库上下文 + 工具调用 + 权限控制 + 验证闭环”。

### 2. Claude Code 和 GitHub Copilot 这类补全工具有什么区别？

**参考答案：**

补全工具通常在编辑器中根据当前文件或局部上下文补全代码；Claude Code 更偏 Agent 工作流，可以跨文件搜索、理解项目结构、执行命令、修改多个文件、运行测试、解释结果。两者可以互补：补全工具提升局部编码速度，Claude Code 更适合任务级开发和排障。

> **易错：** 只说“Claude Code 更智能”。面试中要说清楚上下文范围和执行能力差异。

### 3. 哪些任务最适合 Claude Code？

**参考答案：**

适合需要阅读项目、定位问题、修改代码和验证结果的任务，例如 bug 修复、测试补全、重构、依赖升级、CI 失败排查、PR 摘要、代码审查、文档更新、迁移影响分析。尤其适合已有工程约束明确、验证命令可运行的场景。

> **重点：** 适合“可验证的工程任务”，不适合“高风险且不可恢复的生产操作”。

### 4. 哪些任务不应该直接交给 Claude Code？

**参考答案：**

不应直接交给它执行生产数据库写入、生产部署、密钥轮换、批量删除、强制 push、权限修改、处理未脱敏隐私数据等高风险任务。可以让它做只读分析、生成计划、写回滚方案，但真正执行需要人工审批和受控流水线。

> **易错：** 为了自动化效率，让 Agent 直接操作生产环境。

### 5. 如何给 Claude Code 一个高质量任务？

**参考答案：**

高质量任务要包含目标、上下文、约束、文件范围、验证方式和输出要求。例如“修复这个测试失败，先定位根因，修改前列计划，做最小改动，运行相关测试，最后说明改动文件和剩余风险”。这比“帮我修一下”稳定得多。

## 二、使用场景题

### 6. 你接手一个陌生项目，如何用 Claude Code 快速上手？

**参考答案：**

先在项目根目录启动，要求只读分析 README、依赖配置、目录结构、测试命令和入口文件。然后让它总结主要模块职责、开发命令和风险区域。接着创建或完善 `CLAUDE.md`，把项目规则和验证命令固化。整个过程先不修改文件，避免在不了解项目时误改。

> **重点：** 陌生项目先只读侦察，再做修改。

### 7. 你如何判断 Claude Code 的回答是否可信？

**参考答案：**

不能只看回答本身，要看它是否引用真实文件、是否运行了验证、是否和 Git diff 一致、是否说明未验证原因。对于设计判断，要要求它列证据和替代方案；对于代码变更，要跑测试、lint 或构建。

> **难点：** AI 输出看起来合理不代表项目中正确。
