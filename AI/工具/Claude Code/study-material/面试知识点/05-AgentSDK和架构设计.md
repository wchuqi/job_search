## 一、Agent SDK

### 1. 什么时候需要 Agent SDK，而不是只用 CLI？

**参考答案：**

当需求从个人交互式使用变成平台化、可重复、可审计的产品能力时适合 SDK。例如内部 AI PR Reviewer、批量仓库升级评估、Issue 自动分析平台。CLI 适合人工驱动任务，SDK 适合嵌入系统并统一权限、日志、输出和评估。

### 2. SDK 应用上线前要验证什么？

**参考答案：**

要验证任务成功率、误报漏报、输出格式稳定性、工具权限、失败恢复、成本、延迟、审计日志、敏感信息处理和人工确认流程。最好用历史真实样例建立评估集，而不是只看 demo。

### 3. SDK、MCP、Hooks、Skills 的关系是什么？

**参考答案：**

SDK 是构建 Agent 应用的编程接口；MCP 是外部工具和数据源协议；Hooks 是 Claude Code 事件自动化；Skills 是可复用领域工作法。一个平台化应用可以用 SDK 编排 Agent，用 MCP 接外部系统，用 Hooks 做审计，用 Skills 固化流程。

## 二、架构设计题

### 4. 设计一个 AI PR Reviewer，你会怎么做？

**参考答案：**

输入包括 PR diff、相关文件、CI 结果、Issue 背景和项目规范；任务只读审查 bug、安全、回归和缺失测试；输出结构化 findings，包含文件、原因、严重度、建议；权限禁止修改文件和执行部署；审计保存输入摘要、工具调用、模型版本和最终评论；评估用历史 PR 样例统计误报和漏报。

> **重点：** 这是只读审查系统，不是自动改代码系统。

### 5. 设计一个 Claude Code 团队治理方案。

**参考答案：**

建立仓库级 `CLAUDE.md`、统一权限策略、允许/禁止命令清单、PR 模板、敏感信息扫描、MCP 权限边界、Hook 审计、AI 使用记录、评估集和事故复盘流程。高风险变更必须人工确认，AI 生成代码必须走普通 review 和 CI。

### 6. 如何评估 Claude Code 是否真的提升团队效率？

**参考答案：**

不能只看生成代码量。应统计任务完成时间、缺陷率、review 返工、CI 失败率、测试覆盖、PR 周期、人工介入次数、误改事故、成本和开发者满意度。还要按任务类型分组：bug 修复、文档、测试、审查、迁移的收益不同。

### 7. 如果要让 Claude Code 访问内部文档和 Issue，你如何设计权限？

**参考答案：**

通过 MCP 提供只读检索接口，按用户身份授权，限制返回字段和数量，脱敏敏感信息，记录审计日志。不要让 Claude Code 直接拿到全量 API token。回答时要求它引用文档片段，资料不足时明确说明缺口。

### 8. 你如何设计 Claude Code 的失败恢复机制？

**参考答案：**

失败恢复包括 Git diff 检查、局部回滚、任务复盘、上下文重置、验证重跑、权限规则修正和事故记录。对自动化系统，还要有超时、重试上限、人工接管和安全停止策略。

> **难点：** 恢复机制要覆盖代码状态、上下文状态和外部系统状态。
