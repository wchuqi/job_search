# OpenCode 学习资料：面试知识点整理

[返回索引](../OpenCode学习资料.md)

这份资料用于面试前复习。面试问题已按知识点拆分到 `面试知识点/` 目录。

## 使用方式

- 先读 [01-基础概念和工作流.md](面试知识点/01-基础概念和工作流.md)，确保能准确描述 OpenCode。
- 再读 [02-配置权限和安全.md](面试知识点/02-配置权限和安全.md)，准备工程落地问题。
- 然后读 [03-Agent命令MCP和扩展.md](面试知识点/03-Agent命令MCP和扩展.md)，准备中高级场景题。
- 最后读 [04-团队治理和排障.md](面试知识点/04-团队治理和排障.md)，准备生产实践追问。
- 准备中高级岗位时继续读 [05-深度机制和生产场景题.md](面试知识点/05-深度机制和生产场景题.md)。

## 高频考察点

- OpenCode 与普通 AI 聊天、IDE 补全有什么区别。
- 为什么 `/init` 和 `AGENTS.md` 重要。
- Plan 和 Build 的边界是什么。
- `permission` 如何控制工具调用。
- 配置优先级和合并规则如何影响排障。
- 如何设计安全的 bash 和 edit 权限。
- 自定义命令如何提高团队复用。
- Agent 和 subagent 有什么区别。
- MCP、自定义工具、插件分别适合什么。
- 如何防止密钥泄露、误删文件和 AI 绕过审查。
- 如何把 OpenCode 纳入 PR、测试、CI 和审计流程。
- OpenCode 从用户输入到工具调用的执行生命周期是什么。
- 配置为什么会合并，如何定位最终生效配置。
- 权限通配和最后匹配规则如何造成误放行。
- 子 Agent 输出如何避免证据丢失和上下文污染。
- MCP、插件和自定义工具如何做供应链安全评审。
- 如何设计 OpenCode 团队度量指标，证明它真正提效。
