# OpenCode 学习资料：综合练习项目

[返回索引](../OpenCode学习资料.md)

## 项目目标

用 OpenCode 为一个已有 Web 项目增加“软删除回收站”功能，并建立团队级 OpenCode 工作流。

最终产出：

- `AGENTS.md`
- `opencode.jsonc`
- `.opencode/commands/review-changes.md`
- `.opencode/commands/test-failures.md`
- 一个只读安全审查 Agent
- 功能实现计划
- 最小功能代码改动
- 测试和风险总结

## 阶段 1：项目初始化

### 任务

1. 进入项目根目录。
2. 运行 OpenCode。
3. 使用 `/init` 生成 `AGENTS.md`。
4. 人工编辑 `AGENTS.md`，补充目录、命令、安全规则。

### 验收

- `AGENTS.md` 中包含开发命令、测试命令、禁止修改范围。
- OpenCode 能解释项目结构。

## 阶段 2：权限配置

### 任务

创建项目级 `opencode.jsonc`：

```jsonc
{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "*": "ask",
    "read": "allow",
    "grep": "allow",
    "glob": "allow",
    "bash": {
      "*": "ask",
      "git status*": "allow",
      "git diff*": "allow",
      "pnpm test*": "allow",
      "npm test*": "allow",
      "rm *": "deny",
      "del *": "deny"
    }
  }
}
```

### 验收

- `opencode debug config` 能看到项目配置生效。
- 危险命令会被拒绝或要求审批。

## 阶段 3：计划功能

### Prompt 示例

```text
请进入 Plan 模式，为“软删除回收站”设计实现方案。

需求：
- 删除记录时只标记 deletedAt，不物理删除。
- 新增回收站列表。
- 支持恢复。
- 支持永久删除，但永久删除必须二次确认。

要求：
- 不要修改文件。
- 输出涉及的后端、前端、测试文件。
- 标出数据迁移风险。
- 给出分阶段实现计划。
```

### 验收

- 计划包含数据模型、接口、前端、测试和风险。
- 高风险操作被拆成单独步骤。

## 阶段 4：执行最小改动

### Prompt 示例

```text
按计划只实现第一步：后端软删除字段和查询过滤。
不要实现前端页面。
完成后运行相关单元测试。
```

### 验收

- diff 范围与计划一致。
- 测试通过或失败原因明确。
- OpenCode 输出变更总结。

## 阶段 5：自定义命令

### `review-changes.md`

```markdown
---
description: Review current git changes
agent: plan
---

Review current changes using git diff.

Focus on:
- correctness
- tests
- security
- backward compatibility
- accidental unrelated changes

Return findings first.
```

### `test-failures.md`

```markdown
---
description: Analyze test failures
agent: plan
---

Analyze the latest test failure.
Explain the likely cause, related files, and minimal fix.
Do not modify files.
```

## 阶段 6：安全审查 Agent

创建 `.opencode/agents/security-auditor.md`，只允许只读和 `git diff`。

### 验收

- 安全审查 Agent 不能改文件。
- 能指出数据删除、权限和敏感信息风险。

## 最终验收

- 能完整演示一次 Plan -> Build -> Test -> Review -> Summary。
- 能回滚一次错误修改。
- 能解释项目权限为什么这样设计。
- 能把 OpenCode 修改纳入 PR 模板。

## 重点

- 综合练习的目标不是功能大小，而是完整流程。
- 每一步都要有明确边界和验收。

## 易错

> **易错：** 项目练习只看 OpenCode 是否写出代码。
>
> 正确做法：同时评估计划质量、diff 范围、测试、回滚、安全和团队复用。
