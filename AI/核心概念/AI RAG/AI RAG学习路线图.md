# AI RAG 学习路线图

RAG, Retrieval-Augmented Generation, 是把外部知识检索结果作为证据注入大模型上下文，再让模型基于证据回答的问题解决范式。它的核心不是“接一个向量库”，而是建立一条可评估、可引用、可治理的证据流水线。

## 阶段 1：基础认知

- 目标：理解 RAG 解决什么问题，以及它与微调、长上下文、搜索引擎、知识图谱的边界。
- 需要掌握：检索、生成、证据、chunk、embedding、向量索引、top-k、rerank、citation、faithfulness。
- 例子：公司内部制度问答不能只靠模型参数记忆，需要检索最新版制度原文并在回答中引用条款。
- 练习：为“员工报销制度问答”画出 `用户问题 -> 检索 -> 证据 -> 生成 -> 引用` 数据流。
- 验收：能解释为什么 RAG 可以降低幻觉，但不能天然消除幻觉。
- 重点：RAG 是证据工程，不是向量数据库工程。
- 易错：认为只要把文档丢进向量库，答案就一定准确。

## 阶段 2：知识库建设

- 目标：掌握文档解析、清洗、切分、元数据、版本、权限和增量更新。
- 需要掌握：PDF/HTML/Markdown 解析，结构化切分，父子 chunk，metadata schema，去重，过期文档治理。
- 例子：API 文档应该按标题层级、代码块和版本切分，而不是每 500 字强切。
- 练习：把一份技术文档切成 chunk，并为每个 chunk 标注 `source/version/section/permission/updated_at`。
- 验收：能说明 chunk 过大、过小、断句错误分别会造成什么检索失败。
- 重点：知识库质量决定 RAG 上限。
- 易错：忽略文档版本和权限，导致模型引用旧知识或越权内容。

## 阶段 3：Embedding、索引和召回

- 目标：理解语义向量如何表示文本，以及 ANN 索引如何近似查找相似内容。
- 需要掌握：embedding 模型选择，归一化，余弦相似度，内积，HNSW/IVF/PQ，top-k，阈值，metadata filter。
- 例子：错误码、函数名、订单号这类精确词，纯向量检索可能不如关键词检索。
- 练习：用同一问题分别做关键词召回、向量召回和混合召回，比较命中文档。
- 验收：能解释为什么“语义相似”不等于“足以回答”。
- 重点：召回阶段宁可稍宽，再交给重排和过滤收敛。
- 易错：只调 top-k，不分析召回漏掉的是数据问题、query 问题还是索引问题。

## 阶段 4：查询规划、混合检索和重排

- 目标：能针对复杂问题设计 query rewrite、多路召回、重排和证据覆盖检查。
- 需要掌握：query expansion，multi-query，HyDE，BM25+dense hybrid，reranker，multi-hop retrieval，parent-child retrieval。
- 例子：用户问“v1 到 v2 的认证方式有什么变化”，需要分别检索 v1、v2、迁移说明和兼容性说明。
- 练习：为 10 个复杂问题设计子查询和必须命中的证据类型。
- 验收：能用评测集衡量 recall@k、MRR、nDCG 和证据覆盖率。
- 重点：复杂问题先规划再检索。
- 易错：把 top-5 检索结果直接塞给模型，不做去重、冲突检查和重排。

## 阶段 5：上下文打包、生成和引用

- 目标：掌握如何把证据安全、清晰、低噪声地交给模型，并约束模型基于证据回答。
- 需要掌握：context packing，evidence id，引用格式，拒答策略，答案结构，冲突证据处理，上下文压缩。
- 例子：把证据包装成带来源、时间、权限和可信等级的片段，而不是把网页原文裸拼接。
- 练习：设计一个提示模板，要求模型只引用 `E1/E2/E3` 中的证据回答，证据不足时说明缺口。
- 验收：能区分“回答看起来对”和“回答被证据支持”。
- 重点：生成层要忠实于证据，而不是用模型常识补全缺口。
- 易错：让模型引用不存在的来源，或把来源 URL 当成可信性证明。

## 阶段 6：评估、排障和生产化

- 目标：能建立离线评估、线上监控、成本延迟控制、安全边界和回滚机制。
- 需要掌握：golden set，检索评估，答案忠实性，引用准确率，拒答率，trace，缓存，A/B 测试，权限隔离。
- 例子：一次失败答案要能定位到数据缺失、切分错误、召回漏检、重排错排、上下文污染还是生成不忠实。
- 练习：为一个内部知识库问答系统设计 30 条测试问题和失败归因表。
- 验收：能用日志复现一次回答所使用的 query、候选文档、rerank 分数、最终证据和模型输出。
- 重点：不能回放和评估的 RAG 系统无法稳定迭代。
- 易错：只看最终答案满意度，不记录中间检索链路。

## 阶段 7：进阶 RAG

- 目标：掌握 GraphRAG、Agentic RAG、多模态 RAG、结构化数据 RAG 和权限感知 RAG。
- 需要掌握：实体关系抽取，知识图谱，SQL/搜索/向量多工具路由，递归检索，多跳推理，表格和图片证据。
- 例子：企业风控问答既要查制度文本，也要查用户权限、交易表、审计日志和知识图谱关系。
- 练习：设计一个检索路由器，根据问题类型选择 `vector_search / keyword_search / sql / graph / web`。
- 验收：能说明什么时候 GraphRAG 值得引入，什么时候会增加复杂度而没有收益。
- 重点：进阶 RAG 是为了解决证据结构和任务复杂度，不是堆技术名词。
- 易错：在没有评测集和基线的情况下直接上 Agentic RAG。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | RAG 基础、适用边界、心智模型 | 一张 RAG 数据流图 |
| 第 2 周 | 文档解析、切分、元数据、Embedding | 一个可检索的知识库样例 |
| 第 3 周 | 混合检索、重排、证据打包 | 一个可引用的问答链路 |
| 第 4 周 | 评估、排障、安全、生产化 | 一份 RAG 评测集和失败归因表 |
| 第 5 周 | GraphRAG、Agentic RAG、多源检索 | 一个复杂问题查询规划器 |

## 最终能力清单

- 能判断一个场景是否适合 RAG，而不是默认使用 RAG。
- 能设计从文档摄取到引用回答的完整链路。
- 能解释 chunking、embedding、索引、召回、重排、上下文打包的关键决策。
- 能建立检索和生成两层评估指标。
- 能排查“查不到、查错、排错、答错、引错、越权、过期”七类失败。
- 能处理生产中的权限、隐私、安全、成本、延迟和版本治理问题。
