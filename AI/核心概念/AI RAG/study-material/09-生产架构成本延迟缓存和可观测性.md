# AI RAG 学习资料：生产架构、成本、延迟、缓存和可观测性

[返回索引](../AI RAG学习资料.md)

## 学习目标

- 掌握生产 RAG 的常见架构。
- 理解成本、延迟、缓存、扩展和可观测性。
- 能设计可回放、可监控、可回滚的 RAG 系统。

## 理论导读

生产 RAG 不是 notebook demo。它需要处理文档更新、索引构建、并发查询、权限、缓存、模型限流、日志审计、版本灰度和故障回滚。

## 参考架构

```text
                 +----------------+
文档源 ----------> 摄取服务         |
                 +-------+--------+
                         |
                  解析/切分/元数据
                         |
                 +-------v--------+
                 | 向量库/搜索引擎 |
                 +-------+--------+
                         |
用户请求 -> API -> 查询规划 -> 多路召回 -> 重排 -> 打包 -> LLM -> 后处理
                         |
                 trace / metrics / eval / audit
```

## 成本和延迟来源

| 环节 | 成本/延迟 | 优化 |
| --- | --- | --- |
| Embedding | 摄取时模型调用 | 批处理、增量、缓存 |
| 检索 | 向量库和搜索引擎查询 | 索引调参、过滤前置 |
| Rerank | cross-encoder 或 LLM | 候选裁剪、轻量模型 |
| LLM 生成 | token 成本和延迟 | 压缩上下文、缓存答案 |
| 评估 | 离线和在线打分 | 抽样、异步 |

## 缓存策略

| 缓存 | 适用 |
| --- | --- |
| Query embedding cache | 重复问题和 FAQ |
| Retrieval result cache | 知识库更新不频繁 |
| Rerank result cache | 候选集稳定 |
| Prompt/response cache | 完全相同上下文 |
| Document parse cache | 文档重复摄取 |

缓存必须考虑版本。知识库更新后，旧缓存可能失效。

## 练习

为生产 RAG 写一份上线 checklist：

- 索引版本。
- Prompt 版本。
- 评测集通过标准。
- 监控指标。
- 回滚方案。
- 权限测试。
- 成本预算。

## 验收

- 能画出生产 RAG 架构。
- 能定位延迟和成本瓶颈。
- 能设计 trace 字段复现一次回答。
- 能制定上线、灰度和回滚策略。

## 重点

生产 RAG 的可靠性来自版本化、可观测性和评估，而不是单次 demo 效果。

## 难点

难点是变更管理。chunk 策略、embedding 模型、索引参数、rerank 模型、prompt 和 LLM 任一变化都可能影响答案。

## 易错

> **易错：** 改了知识库或 prompt 后直接上线。
>
> 正确做法：用固定评测集回放，对比检索和生成指标，再灰度发布。
