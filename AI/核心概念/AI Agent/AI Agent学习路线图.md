# AI Agent学习路线图

## 阶段 1：基础认知

- 目标：理解 AI Agent 与普通 LLM 调用、Chatbot、RPA、工作流系统的区别。
- 需要掌握：Agent、工具、环境、状态、目标、规划、执行、观察、反馈。
- 例子：普通 Chatbot 回答“怎么查天气”，天气 Agent 会调用天气工具并返回结果。
- 练习：画出一个“查询订单状态 Agent”的输入、工具、状态和输出。
- 验收：能说明 Agent 的核心不是会聊天，而是能在环境中多步行动。
- 重点：Agent = 模型 + 工具 + 状态 + 控制循环 + 约束。
- 易错：把任意 LLM 应用都叫 Agent。没有工具、状态或多步控制的应用通常只是 LLM workflow。

## 阶段 2：核心机制

- 目标：掌握 Agent 如何计划、调用工具、观察结果、修正行为。
- 需要掌握：ReAct、Plan-and-Execute、Router、Tool Calling、MCP、状态机、错误恢复。
- 例子：客服 Agent 先识别意图，再查订单，再判断退款规则，再给出建议。
- 练习：为“自动生成周报”设计 5 步 Agent 控制循环。
- 验收：能写出 Agent loop 伪代码，并说明停止条件和失败重试策略。
- 重点：Agent 系统必须有外部控制点，不能只靠模型自觉。
- 易错：让模型无限反思和重试。正确做法是设置预算、状态和转人工条件。

## 阶段 3：工程实现

- 目标：能实现一个最小可用 Agent。
- 需要掌握：工具 schema、函数调用、上下文构造、RAG、短期记忆、长期记忆、权限、日志。
- 例子：知识库问答 Agent 检索文档、引用来源、拒答无依据问题。
- 练习：用伪代码实现一个“文档问答 + 工单创建” Agent。
- 验收：Agent 能选择工具、处理工具失败、输出可验证结果。
- 重点：工具调用必须结构化，工具输出必须可验证。
- 易错：让 Agent 直接拼接命令或 SQL。正确做法是通过受控工具接口。

## 阶段 4：生产落地

- 目标：能设计可靠、安全、可观测的 Agent 系统。
- 需要掌握：Guardrails、权限、安全、人类介入、Evals、trace、成本、延迟、并发、缓存。
- 例子：财务 Agent 可以生成付款建议，但真实付款必须人工审批。
- 练习：为“HR 简历筛选 Agent”设计安全和评估方案。
- 验收：能说明哪些任务可自动化，哪些任务必须审批。
- 重点：生产 Agent 的难点是控制风险和证明质量。
- 易错：只在 demo 中验证成功案例。正确做法是建立失败集、对抗样例和回归评估。

## 阶段 5：高级架构

- 目标：掌握多 Agent、长任务、任务调度、记忆治理和平台化。
- 需要掌握：Supervisor、Planner、Worker、Critic、黑板模式、事件驱动、任务队列、checkpoint、human-in-the-loop。
- 例子：软件开发 Agent 系统由需求分析、代码修改、测试验证、评审总结多个角色协作。
- 练习：设计一个多 Agent “市场调研报告生成系统”。
- 验收：能解释为什么要拆 Agent，以及拆分后如何避免协调成本失控。
- 重点：多 Agent 不是越多越好，复杂度必须换来真实收益。
- 易错：为角色感而拆分角色。正确做法是按能力边界、工具权限和验证责任拆分。

## 阶段 6：深度机制和生产治理

- 目标：能从平台和生产系统角度设计 AI Agent，而不是只做 demo。
- 需要掌握：控制平面、数据平面、Tool Router、MCP 工具安全、沙箱、RAG 排序、记忆一致性、多 Agent 冲突、红队测试、线上监控、事故复盘。
- 例子：企业知识库 Agent 出现错误引用时，能定位是检索召回、权威性排序、引用绑定、记忆污染还是 Eval 缺失。
- 练习：为“自动报销 Agent”写一份生产治理方案，包括工具风险、审批、版本、Eval、监控、事故复盘和回滚。
- 验收：能回答“这个 Agent 为什么可以上线、出了事故怎么定位、如何防止复发”。
- 重点：深度来自可执行的控制、评估和治理规则。
- 易错：只讲框架和模型，不讲状态、权限、证据、预算和事故闭环。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | 概念、控制循环、工具调用 | 一个最小 Agent 设计 |
| 第 2 周 | RAG、记忆、MCP、权限 | 一个文档问答 Agent 方案 |
| 第 3 周 | Evals、观测、安全 | 一套评估和 Guardrails |
| 第 4 周 | 多 Agent 和生产架构 | 一个完整系统设计 |
| 第 5 周 | 深度机制和治理 | 控制平面、红队 Eval、事故复盘方案 |

## 最终能力清单

- 能区分 LLM 应用、Workflow 和 Agent。
- 能设计 Agent 控制循环和状态机。
- 能定义工具 schema、权限和错误处理。
- 能设计上下文、RAG、记忆和检索策略。
- 能设计 Guardrails、人类介入和安全策略。
- 能建立 Evals、trace 和质量指标。
- 能为真实业务设计单 Agent 或多 Agent 架构。
- 能设计控制平面、工具沙箱、检索排序、记忆一致性和生产治理闭环。
