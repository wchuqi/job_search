# AI Agent学习资料：生产工程部署成本和性能

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 掌握 Agent 生产部署的工程问题。
- 理解成本、延迟、并发、缓存、降级和可靠性。
- 能设计可运行的 Agent 服务架构。

## 理论导读

Demo Agent 可以只关心能不能跑通，生产 Agent 必须关心稳定性、成本、延迟、安全、观测、恢复和版本管理。Agent 往往会多次调用模型和工具，成本和延迟比普通 LLM 应用更难控制。

## 生产架构

```text
Client -> API Gateway -> Agent Orchestrator -> Model Gateway
                             |
                             +-> Tool Services
                             +-> Memory Store
                             +-> Vector Store
                             +-> Policy Engine
                             +-> Trace Store
                             +-> Eval/Monitor
```

## 成本来源

- 模型输入 token。
- 模型输出 token。
- 多轮调用。
- 工具调用。
- 向量检索。
- 评估模型。
- 日志和存储。
- 人工审批。

## 性能优化

| 问题 | 优化 |
| --- | --- |
| 上下文过长 | 检索压缩、摘要、分阶段上下文 |
| 模型调用多 | 固定流程前置、批量工具调用 |
| 工具慢 | 缓存、并行、超时 |
| 重试多 | 更好错误摘要和工具 schema |
| 输出慢 | 流式返回、分段执行 |
| 成本高 | 小模型路由、缓存、任务拆分 |

## 可靠性设计

- 超时控制。
- 重试预算。
- 幂等工具。
- checkpoint。
- 降级路径。
- 人工接管。
- 灰度发布。
- 版本回滚。

## 部署注意点

- 模型版本要记录。
- Prompt 和工具 schema 要版本化。
- Eval 通过后再上线。
- 高风险功能灰度。
- Trace 不能泄露敏感信息。
- 工具服务要有鉴权。
- 记忆和向量库要支持删除。

## 例子

订单客服 Agent 降级策略：

```text
模型不可用 -> 使用规则模板回复。
退款工具失败 -> 创建人工工单。
检索服务失败 -> 回答“需要人工确认”，不编造政策。
高风险退款 -> 只生成建议，不自动执行。
```

## 练习

设计一个客服 Agent 生产架构，包含：

- API。
- 模型网关。
- 工具服务。
- 权限服务。
- 记忆存储。
- trace。
- eval。
- 降级策略。

## 验收

- 能解释 Agent 成本来源。
- 能提出延迟优化策略。
- 能设计可靠性和降级。
- 能说明版本和回滚。

## 重点

Agent 的生产成本不是单次模型调用，而是多轮模型、工具、检索、验证和人工流程的总成本。

## 难点

难点是稳定性。Agent 路径动态变化，传统接口测试不足以覆盖全部执行路径。

## 易错

> **易错：** Demo 跑通就上线。
>
> 正确做法：上线前必须有 Eval、trace、安全策略、降级和回滚。
