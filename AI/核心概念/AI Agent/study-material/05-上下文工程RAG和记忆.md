# AI Agent学习资料：上下文工程RAG和记忆

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 理解上下文工程、RAG 和记忆的关系。
- 掌握短期记忆、长期记忆、情景记忆、语义记忆。
- 能设计检索、压缩、更新和遗忘策略。

## 理论导读

Agent 的决策质量高度依赖上下文。上下文不是越多越好，而是要相关、权威、及时、低噪声。RAG 负责把外部知识检索进上下文；记忆负责跨步骤或跨会话保存有用信息。

## 上下文来源

- 用户当前输入。
- 系统指令。
- 工具返回结果。
- 文档检索结果。
- 数据库查询结果。
- 历史对话。
- 用户偏好。
- 任务状态。
- 评估和错误反馈。

## RAG在Agent中的作用

RAG = Retrieval-Augmented Generation。它让模型在回答或行动前先检索外部知识。

Agent 使用 RAG 的典型流程：

```text
理解任务 -> 生成检索查询 -> 检索候选文档 -> 重排 -> 压缩 -> 放入上下文 -> 决策或回答
```

## 记忆类型

| 类型 | 内容 | 例子 |
| --- | --- | --- |
| 短期记忆 | 当前任务上下文 | 已经查过订单 |
| 长期记忆 | 跨会话信息 | 用户偏好中文报告 |
| 语义记忆 | 稳定知识 | 公司退款政策 |
| 情景记忆 | 过去事件 | 上次该用户投诉过物流 |
| 程序记忆 | 做事方法 | 生成报告的标准步骤 |

## 记忆写入规则

不应该把所有内容都写入长期记忆。写入前要判断：

- 是否对未来有用。
- 是否经过用户确认或工具验证。
- 是否涉及隐私。
- 是否有过期时间。
- 是否允许被删除。

## 检索排序

建议排序因素：

```text
相关性 + 权威性 + 新鲜度 + 具体性 - 冲突风险 - 隐私风险
```

## 例子

错误记忆：

```text
用户可能喜欢便宜酒店。
```

更好的记忆：

```json
{
  "type": "preference",
  "content": "用户在 2026-06-07 明确表示商务出行优先选择可开发票酒店",
  "source": "user_confirmed",
  "expires_at": "2027-06-07"
}
```

## 练习

为“个人助理 Agent”设计记忆系统：

- 哪些信息可记。
- 哪些信息不能记。
- 记忆如何过期。
- 用户如何查看和删除记忆。
- 检索时如何排序。

## 验收

- 能区分上下文、RAG 和记忆。
- 能说明记忆写入和检索规则。
- 能设计隐私和遗忘机制。

## 重点

记忆不是聊天记录堆积，而是经过筛选、验证、授权和治理的状态。

## 难点

难点是记忆污染。错误记忆会长期影响 Agent 行为，比一次错误回答更危险。

## 易错

> **易错：** 把模型猜测写入长期记忆。
>
> 正确做法：长期记忆必须来自用户确认、工具事实或高可信来源，并能删除和过期。
