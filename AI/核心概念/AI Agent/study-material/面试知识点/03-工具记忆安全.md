## 一、工具、记忆和安全

### 1. Function calling 和 MCP 有什么区别？

**参考答案：**

Function calling 关注模型如何选择并调用应用内定义的函数；MCP 更像连接模型应用与外部工具、资源、提示模板的协议。前者偏 API 调用机制，后者偏工具和上下文生态标准化。

> **重点：** MCP 不是替代所有 function calling，而是提供跨工具集成方式。

### 2. 如何设计 Agent 的记忆？

**参考答案：**

记忆要区分短期、长期、语义、情景和程序记忆。长期记忆不能直接写入模型猜测，必须来自用户确认、工具事实或可信来源，并支持过期、删除和权限控制。检索时要按相关性、权威性、新鲜度和隐私风险排序。

> **易错：** 把聊天记录全部当长期记忆。

### 3. 如何防止 prompt injection？

**参考答案：**

要区分系统指令、用户指令和外部不可信内容。外部网页、邮件、文档只能作为数据，不能覆盖系统规则。高风险工具调用必须经过权限校验和人工确认，敏感信息默认不放入模型上下文。

> **重点：** prompt injection 防护主要靠系统边界，不靠提醒模型。
