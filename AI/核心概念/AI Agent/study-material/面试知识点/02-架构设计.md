## 一、架构设计

### 1. 如何设计一个 AI Agent 系统？

**参考答案：**

我会先明确任务目标、风险和验收标准，然后设计 Agent Orchestrator、Model Gateway、Tool Router、Context Builder、Memory Store、Policy Engine、Verifier、Trace Store 和 Human Approval。核心是让模型负责推理，让系统负责权限、状态、工具执行和验证。

> **重点：** 架构题要讲组件职责和数据流。

### 2. Agent 控制循环如何设计？

**参考答案：**

典型循环是 observe、plan、act、verify、reflect。系统先构造上下文，模型决定动作，工具执行动作，系统更新状态并验证结果。如果成功则结束，如果失败则有限重试，遇到高风险或连续失败则转人工。

> **难点：** 停止条件和重试预算必须外部控制。

### 3. 多 Agent 什么时候有必要？

**参考答案：**

当任务有清晰不同职责、不同工具权限、可独立验证的子任务时，多 Agent 有价值。例如软件开发可拆成需求、实现、测试、评审。若只是为了角色感而拆，会增加协调成本和错误来源。

> **易错：** Agent 越多越强。正确做法是职责清晰才拆。
