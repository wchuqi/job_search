# AI Agent学习资料：总览与心智模型

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 建立 AI Agent 的整体认知。
- 理解 Agent 与普通 LLM 应用的边界。
- 形成从需求到生产架构的学习地图。

## 理论导读

AI Agent 是能围绕目标进行多步行动的 AI 系统。它通常使用大语言模型进行推理和决策，通过工具与外部世界交互，通过状态记录任务进展，通过评估和约束控制风险。

一个生产 Agent 不是“一个很长的提示词”，而是一个系统：

```text
目标 -> 上下文 -> 计划 -> 工具调用 -> 观察结果 -> 更新状态 -> 验证 -> 输出或继续
```

## 核心心智模型

可以把 Agent 想象成一个“有工位、有工具、有权限、有记录、有主管检查”的数字员工：

- LLM 是它的大脑。
- 工具是它的手。
- 上下文是它看到的材料。
- 状态是它的任务记录。
- Guardrails 是制度和权限。
- Evals 是绩效考核。
- Trace 是工作日志。

没有工具，它只能说；没有状态，它无法做长任务；没有 Guardrails，它容易越界；没有 Evals，你无法知道它是否可靠。

## 例子

普通 LLM 应用：

```text
用户：帮我写一封请假邮件。
模型：生成邮件文本。
```

Agent 应用：

```text
用户：帮我安排下周三和张三开会。
Agent：
1. 查询日历。
2. 检查张三可用时间。
3. 找到双方空档。
4. 生成会议邀请。
5. 请求用户确认。
6. 调用日历工具创建会议。
```

## 练习

选择一个场景：客服、运维、招聘、代码开发、数据分析。写出：

- Agent 的目标。
- 它需要哪些工具。
- 它需要哪些上下文。
- 哪些操作必须人工确认。
- 如何判断任务完成。

## 验收

- 能画出 Agent 的数据流。
- 能指出 Agent 与 Chatbot 的区别。
- 能说明为什么工具、状态和验证是 Agent 的核心。

## 重点

Agent 的本质不是“智能人设”，而是“能在受控环境中多步行动的系统”。

## 难点

难点是控制。模型可以生成下一步动作，但系统必须决定哪些动作允许、如何验证、失败如何恢复。

## 易错

> **易错：** 认为模型越强，Agent 系统越简单。
>
> 正确做法：模型越能行动，越需要权限、观测和评估来约束风险。
