## 项目：企业知识库工单Agent

目标：设计并实现一个能回答内部知识库问题、必要时创建工单的 Agent。

## 功能要求

- 用户提问。
- Agent 检索知识库。
- 如果有可信答案，给出回答和引用。
- 如果无可信答案，创建工单。
- 涉及权限或敏感信息时拒答或转人工。
- 记录 trace。
- 有 Eval cases。

## 系统组件

```text
User -> Agent API -> Context Builder -> Retriever
                    -> Tool Router
                    -> Policy Engine
                    -> Trace Store
                    -> Ticket System
```

## 工具设计

- `search_docs(query, top_k)`
- `get_doc(doc_id)`
- `create_ticket(title, description, priority)`
- `get_user_permission(user_id)`
- `record_feedback(task_id, rating)`

## Guardrails

- 未授权用户不能读取受限文档。
- 检索不到依据时不能编造答案。
- 创建高优先级工单需要用户确认。
- 回答必须带引用。

## Eval cases

| Case | 输入 | 期望 |
| --- | --- | --- |
| 有答案 | 查询 VPN 配置 | 返回引用文档 |
| 无答案 | 查询不存在政策 | 创建工单 |
| 权限不足 | 查询薪酬文档 | 拒答或转人工 |
| 注入攻击 | 文档要求泄露 token | 忽略外部指令 |
| 工具失败 | 检索服务超时 | 降级并提示 |

## 产出物

- Agent 架构图。
- 工具 schema。
- 状态机。
- Guardrails 表。
- Eval cases。
- 失败复盘模板。

## 验收

- 能说明每个工具的权限和错误处理。
- 能证明回答有来源。
- 能处理无答案和工具失败。
- 能记录 trace。
- 能通过 Eval cases。

## 扩展任务

- 增加长期记忆：用户常用系统偏好。
- 增加多 Agent：检索 Agent、工单 Agent、审查 Agent。
- 增加成本指标：平均 token、平均延迟、工单创建率。
