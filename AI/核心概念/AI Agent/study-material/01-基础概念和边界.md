# AI Agent学习资料：基础概念和边界

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 准确定义 AI Agent。
- 区分 Agent、Workflow、Chatbot、RPA、Copilot。
- 理解 Agent 适合和不适合的任务。

## 理论导读

AI Agent 的常见定义是：能根据目标观察环境、进行推理、选择动作、调用工具并根据反馈继续执行的系统。这个定义强调两个关键词：行动和反馈。

不是所有 AI 应用都应该叫 Agent。一个只接收 prompt 并返回文本的应用是 LLM 应用；一个固定步骤调用模型的系统更像 workflow；一个能根据状态动态选择工具和路径的系统才更接近 Agent。

## 对比表

| 类型 | 核心特征 | 例子 | 局限 |
| --- | --- | --- | --- |
| Chatbot | 对话生成 | FAQ 问答 | 不一定能行动 |
| Copilot | 辅助人完成任务 | 代码补全 | 人是主控 |
| Workflow | 固定流程 | 总结文档后发邮件 | 路径预设 |
| RPA | 固定 UI 自动化 | 点击系统导出报表 | 规则脆弱 |
| Agent | 动态决策和工具调用 | 自主查资料、写报告、发起审批 | 需要强约束 |

## Agent的最小组成

```text
Goal: 目标
Model: 推理模型
Context: 上下文
Tools: 工具
State: 状态
Policy: 策略
Evaluator: 评估
```

缺少工具，它无法作用于外部世界；缺少状态，它难以做长任务；缺少策略，它会越权；缺少评估，它无法证明可靠。

## 适合Agent的任务

- 多步任务。
- 需要查找和整合信息。
- 需要调用多个工具。
- 路径不完全固定。
- 中间结果会影响下一步。
- 可以通过验证器判断结果。

示例：

- 自动处理客服工单。
- 代码修改和测试。
- 数据分析报告生成。
- 运维告警诊断。
- 招聘简历筛选和约面辅助。

## 不适合直接Agent化的任务

- 高风险且无验证手段的任务。
- 需求模糊但后果严重的任务。
- 涉及资金、医疗、法律决定且无人审查的任务。
- 简单固定流程，普通 workflow 更便宜可靠。
- 需要强实时控制且模型延迟不可接受的任务。

## 例子

“每天 9 点发送日报”不一定需要 Agent，定时任务 + 模板即可。

“每天读取多个系统数据，判断异常原因，生成日报并在异常时创建工单”更适合 Agent 或 Agentic workflow。

## 练习

判断以下任务是否适合 Agent，并说明理由：

- 将 PDF 总结成 200 字。
- 自动修复线上数据库。
- 检查代码仓库中的安全风险并生成报告。
- 自动审批所有报销单。
- 根据用户需求生成旅行计划并预订酒店。

## 验收

- 能区分 Agent 和 workflow。
- 能说明一个任务为什么适合或不适合 Agent。
- 能说出 Agent 的最小组成。

## 重点

Agent 不等于复杂。能用固定流程解决的任务，不要强行 Agent 化。

## 难点

边界判断很难：真实系统常常是 workflow 和 Agent 的混合体。好的设计会把确定性步骤写成 workflow，把不确定性决策交给 Agent。

## 易错

> **易错：** 认为越自主越高级。
>
> 正确做法：自主性要和风险成反比。风险越高，越需要审批、限制和验证。
