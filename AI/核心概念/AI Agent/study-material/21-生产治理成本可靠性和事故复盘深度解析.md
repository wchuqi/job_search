# AI Agent学习资料：生产治理、成本、可靠性和事故复盘深度解析

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 掌握 Agent 生产治理的完整体系。
- 理解成本、延迟、可靠性、安全和合规如何共同约束 Agent。
- 能做生产事故复盘，并转化为系统改进。

## 理论导读

生产 Agent 是一个在线决策系统。它不仅要回答正确，还要在预算内、权限内、时限内、审计内完成任务。Agent 的不确定性要求系统具备更强的治理能力：版本、灰度、回滚、监控、事故响应和持续评估。

## 生产治理对象

- 模型版本。
- Prompt 版本。
- 工具 schema 版本。
- 检索索引版本。
- 记忆策略版本。
- Policy 版本。
- Eval 版本。
- 发布批次。

每次线上问题都要能回答：

```text
当时用的哪个模型？
哪个 prompt？
哪些工具版本？
检索到了哪些文档？
调用了哪些工具？
为什么允许这个动作？
哪个 Eval 没覆盖？
```

## 成本控制

成本公式：

```text
total_cost =
  model_input_cost
+ model_output_cost
+ tool_cost
+ retrieval_cost
+ eval_cost
+ storage_cost
+ human_review_cost
```

优化策略：

- 小模型路由简单任务。
- 高价值上下文优先，减少无效 token。
- 工具批量调用。
- 检索缓存。
- 中间结果缓存。
- 重试预算。
- 对低价值任务降级为 workflow。

## 可靠性策略

| 风险 | 策略 |
| --- | --- |
| 模型不可用 | 降级到模板或排队 |
| 工具超时 | 超时、重试、替代工具 |
| 检索失败 | 拒答或转人工，不编造 |
| 状态丢失 | checkpoint 和恢复 |
| 高风险误操作 | 审批、双人确认、回滚 |
| 成本异常 | 熔断和预算限制 |

## 案例：Agent错误发送客户邮件

现象：销售 Agent 根据错误记忆，把报价邮件发给了错误客户联系人。

根因：

- 长期记忆没有来源和过期时间。
- 发送邮件工具没有二次确认。
- Trace 记录了工具调用，但没有记录联系人来源。
- Eval 没覆盖“相似客户名”场景。

改进：

- 联系人信息必须来自 CRM 实时查询。
- 发送外部邮件必须用户确认。
- 记忆加入 source 和 expires_at。
- 新增相似实体混淆 Eval。

## 上线门禁

上线前必须满足：

- 核心 Eval 通过。
- 高风险 Eval 零越权。
- Trace 可回放。
- Policy 版本冻结。
- 回滚方案存在。
- 监控指标接入。
- 人工接管路径可用。

## 练习

为“自动报销 Agent”设计生产治理方案：

- 版本管理。
- 成本预算。
- 审批策略。
- 监控指标。
- 事故复盘模板。
- 回滚方案。
- 高风险 Eval。

## 验收

- 能说明生产 Agent 需要版本化哪些对象。
- 能拆解成本结构。
- 能设计可靠性降级。
- 能写事故复盘并产出防复发动作。

## 重点

生产 Agent 的核心不是“让模型更聪明”，而是让每个动作可解释、可审计、可回滚、可评估。

## 难点

难点是线上反馈闭环。事故复盘如果不能转成 Eval、Guardrail、工具或审批改进，同类问题会重复发生。

## 易错

> **易错：** 事故后只换模型或改 prompt。
>
> 正确做法：同时检查版本、上下文、工具、权限、Eval、监控和人类介入。
