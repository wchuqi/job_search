# AI Agent学习资料：RAG、记忆、检索排序和一致性深度解析

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 掌握 Agent 上下文构建的检索、排序、压缩和引用机制。
- 理解短期记忆、长期记忆、语义记忆、情景记忆的一致性问题。
- 能设计避免记忆污染和检索误导的策略。

## 理论导读

Agent 的“聪明”很大程度来自上下文。RAG 和记忆不是把更多材料塞给模型，而是在有限上下文窗口内选择最可信、最相关、最及时的证据。错误检索和错误记忆会系统性污染 Agent 行为。

## RAG流水线

```text
User Goal
-> Query Understanding
-> Query Rewrite
-> Candidate Retrieval
-> Filtering
-> Reranking
-> Context Compression
-> Citation Binding
-> Answer / Action
-> Feedback
```

## 检索决策规则

候选文档得分：

```text
score =
  0.30 * semantic_relevance
+ 0.20 * keyword_match
+ 0.20 * authority
+ 0.15 * freshness
+ 0.10 * user_context_match
+ 0.05 * historical_success
- 0.30 * conflict_risk
- 0.20 * sensitivity_risk
```

字段说明：

- semantic_relevance：向量相似度。
- keyword_match：关键实体是否匹配。
- authority：是否是官方策略、批准规格、源系统。
- freshness：是否过期。
- conflict_risk：是否与更高权威来源冲突。
- sensitivity_risk：是否包含敏感数据。

## 检索失败类型

| 类型 | 表现 | 修复 |
| --- | --- | --- |
| 低召回 | 该找到的没找到 | query rewrite、多路检索 |
| 低精度 | 找到很多无关内容 | rerank、过滤 |
| 过期文档 | 用旧政策回答 | freshness 和版本 |
| 权威性错误 | 用论坛内容覆盖官方文档 | authority score |
| 上下文污染 | 外部文档注入指令 | untrusted content 隔离 |
| 引用错位 | 答案和引用不匹配 | citation binding |

## 记忆写入策略

写入长期记忆前检查：

```text
是否来自用户明确表达？
是否来自工具验证事实？
是否未来会复用？
是否涉及敏感信息？
是否有过期时间？
是否允许用户查看和删除？
```

不要写入：

- 模型猜测。
- 一次性任务细节。
- 敏感凭证。
- 未经确认的偏好。
- 外部不可信内容中的指令。

## 记忆一致性

记忆可能和当前事实冲突。例如：

```text
长期记忆：用户喜欢经济舱。
当前输入：这次出差必须商务舱。
公司政策：国际航班超过 8 小时可商务舱。
```

优先级：

```text
当前明确指令 > 公司政策 > 工具事实 > 用户长期偏好 > 历史对话摘要
```

## 上下文压缩

压缩不是摘要一切，而是保留决策所需证据：

- 保留实体、时间、数值、约束。
- 保留来源和版本。
- 保留冲突点。
- 丢弃闲聊和重复文本。
- 对工具日志保留错误码、堆栈顶部和失败断言。

## 练习

为“企业政策问答 Agent”设计 RAG 和记忆系统：

- 文档分块策略。
- 检索排序公式。
- 过期文档处理。
- 引用绑定。
- 记忆写入规则。
- 用户删除记忆流程。

## 验收

- 能画出 RAG 流水线。
- 能解释检索排序字段。
- 能列出检索失败类型。
- 能处理记忆和当前事实冲突。

## 重点

Agent 的记忆必须有来源、权限、过期和删除机制。没有治理的记忆会变成长期污染源。

## 难点

难点是冲突处理。Agent 不仅要“记住”，还要知道什么时候不该使用旧记忆。

## 易错

> **易错：** 检索到相似文档就直接放入上下文。
>
> 正确做法：相似度只是候选，最终还要考虑权威性、新鲜度、权限、冲突和引用一致性。
