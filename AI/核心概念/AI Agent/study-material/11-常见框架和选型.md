# AI Agent学习资料：常见框架和选型

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 了解常见 Agent 框架和协议。
- 能根据项目需求选择框架。
- 理解不要被框架掩盖核心设计。

## 常见选择

| 名称 | 关注点 | 适合 |
| --- | --- | --- |
| OpenAI Agents SDK | Agent、tools、handoff、guardrails、tracing | 构建基于 OpenAI 生态的 Agent |
| LangChain Agents | 模型、工具、Agent 抽象 | 快速集成工具和模型 |
| LangGraph | 状态图、长任务、多 Agent、可恢复流程 | 复杂 Agent 工作流 |
| MCP | 工具和资源连接协议 | 工具生态标准化 |
| CrewAI | 角色、任务、团队协作 | 多角色任务编排 |
| AutoGen | 多 Agent 对话和协作 | 研究和多 Agent 原型 |

## 选型维度

- 是否需要复杂状态机。
- 是否需要多 Agent。
- 是否需要生产级 tracing。
- 是否需要 human-in-the-loop。
- 是否需要跨模型支持。
- 是否需要 MCP 工具生态。
- 团队熟悉的语言和部署环境。
- 是否容易测试和回滚。

## 决策建议

```text
简单工具调用 -> 原生 API + 少量封装
复杂状态和长任务 -> LangGraph 类状态图
OpenAI 生态应用 -> OpenAI Agents SDK
多角色协作原型 -> CrewAI / AutoGen
跨工具生态 -> MCP
高风险生产系统 -> 框架 + 自研 Policy/Eval/Trace
```

## 框架不能替你解决的问题

- 业务规格。
- 工具权限。
- 数据安全。
- Eval 标准。
- 成本控制。
- 失败复盘。
- 人工审批。
- 组织流程。

## 练习

为以下场景选择技术方案：

- 文档问答 Agent。
- 代码修复 Agent。
- 多 Agent 市场调研。
- 企业内部工具助手。
- 高风险财务审批 Agent。

写出选择理由和不选其他方案的原因。

## 验收

- 能列出常见框架。
- 能按需求选型。
- 能说明框架和系统设计的边界。

## 重点

框架是实现工具，不是架构答案。Agent 的可靠性主要来自规格、权限、验证和观测。

## 难点

难点是不要过早框架化。简单任务用复杂框架，会增加调试和维护成本。

## 易错

> **易错：** 选了 Agent 框架就认为有了生产 Agent。
>
> 正确做法：框架之外还要设计安全、评估、权限、监控和业务流程。
