# AI Agent学习资料：任务分解路由调度和状态机

[返回索引](../AI Agent学习资料.md)

## 学习目标

- 掌握 Agent 的任务分解、路由和调度。
- 理解状态机在 Agent 系统中的作用。
- 能设计任务队列、优先级和冲突处理。

## 理论导读

复杂任务不能直接丢给模型“一步完成”。Agent 系统需要把任务拆成子任务，决定执行顺序，选择工具或子 Agent，记录状态，并处理失败和并发冲突。

## 任务分解

任务分解方式：

- 按目标拆：调研、分析、写作、审查。
- 按数据源拆：订单、支付、库存。
- 按风险拆：低风险自动，高风险审批。
- 按验证拆：每个子任务都有验收标准。

## 路由机制

Router 决定任务交给哪个工具、流程或 Agent。

路由规则示例：

```text
if task.type == "refund" and amount > 1000:
    route_to("human_approval")
elif task.type == "faq":
    route_to("rag_answer")
elif task.type == "bug_fix":
    route_to("code_agent")
```

## 调度策略

| 策略 | 适用 | 风险 |
| --- | --- | --- |
| 顺序执行 | 强依赖任务 | 慢 |
| 并行执行 | 独立检索或分析 | 合并冲突 |
| 优先级队列 | 多任务系统 | 饥饿问题 |
| 事件驱动 | 外部状态变化 | 复杂 |
| 人工门禁 | 高风险任务 | 延迟 |

## 状态机

Agent 任务状态示例：

```text
NEW -> PLANNING -> WAITING_TOOL -> RUNNING -> VERIFYING -> DONE
                         |              |
                         v              v
                   WAITING_HUMAN      FAILED
```

状态机的作用：

- 防止重复执行。
- 防止非法跳转。
- 支持恢复。
- 方便监控。
- 支持人类介入。

## 冲突处理

常见冲突：

- 两个 Agent 修改同一资源。
- 工具返回状态与记忆不一致。
- 用户中途改变目标。
- 子任务结果互相矛盾。

处理方式：

- 资源锁。
- 乐观并发控制。
- 版本号。
- 人工确认。
- 重新规划。

## 例子

招聘 Agent：

```text
1. 解析岗位要求。
2. 路由到简历筛选 Agent。
3. 并行打分候选人。
4. 对高分候选人生成面试问题。
5. 高风险拒绝理由进入人工审核。
```

## 练习

为“代码审查 Agent”设计状态机和路由规则，覆盖：

- 安全问题。
- 性能问题。
- 风格问题。
- 架构问题。
- 需要人工确认的问题。

## 验收

- 能设计任务分解规则。
- 能设计路由条件。
- 能画出任务状态机。
- 能说明并发和冲突处理。

## 重点

Agent 系统需要显式任务管理。复杂任务如果没有状态机，会很难恢复、监控和排障。

## 难点

难点是任务粒度。太粗，Agent 难以控制；太细，调度和上下文开销变大。

## 易错

> **易错：** 把所有子任务都交给一个模型自由决定。
>
> 正确做法：用路由、状态机和策略控制任务路径，把关键决策显式化。
