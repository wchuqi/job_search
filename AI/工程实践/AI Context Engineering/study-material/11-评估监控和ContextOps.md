# AI Context Engineering 学习资料：评估、监控和 ContextOps

[返回索引](../AI Context Engineering学习资料.md)

## 学习目标

- 建立上下文策略的离线评估和线上监控意识。
- 掌握 RAG、Agent、工具调用和压缩的评估指标。
- 能把上下文模板、检索配置和记忆 schema 版本化。

## 理论导读

生产问题不能只看最终回答。必须同时观察模型看到的上下文：检索到了什么、排序是否正确、压缩是否丢信息、工具结果是否被正确摘要、历史状态是否过期、敏感内容是否被过滤。

## 指标体系

| 类型 | 指标 |
| --- | --- |
| 检索 | recall@k、MRR、nDCG、命中文档覆盖率 |
| 上下文质量 | 相关性、完整性、冲突率、来源可信度 |
| 压缩 | 保真度、关键事实保留率、token 降低率 |
| 生成 | 忠实度、引用准确率、拒答正确率 |
| 工具 | 工具选择准确率、参数正确率、失败恢复率 |
| 安全 | 注入拦截率、敏感信息泄漏率、越权调用率 |
| 成本 | token、延迟、缓存命中率 |

## ContextOps 对象

需要版本化的内容：

- prompt 模板。
- 上下文槽位定义。
- 检索 query rewrite 规则。
- chunk 和 embedding 配置。
- rerank 模型和阈值。
- 压缩策略。
- 记忆 schema。
- 工具 schema。
- 安全策略。
- 评测集和评分 rubrics。

## 回放测试

```text
保存真实失败样本 -> 固定输入和外部资料版本 -> 重放上下文流水线
-> 对比最终上下文 -> 对比模型输出 -> 标注失败归因
```

## 练习

为一个 RAG 客服系统设计 30 条评测样本，覆盖：命中单文档、多文档合成、资料缺失、版本冲突、恶意注入、用户问题模糊。

## 验收

- 能从失败案例定位上下文环节。
- 能设计离线和线上指标。
- 能说明一次上下文策略变更如何灰度和回滚。

## 重点

ContextOps 的核心是让上下文策略可重复、可比较、可回滚。

## 难点

难点是指标之间会冲突：更高召回可能带来更多噪声，更强压缩可能损失细节，更严格安全策略可能提高拒答率。

## 易错

> **易错：** 只评估最终答案准确率。
>
> 正确做法：同时评估检索结果、上下文包、工具调用、压缩结果和安全决策。
