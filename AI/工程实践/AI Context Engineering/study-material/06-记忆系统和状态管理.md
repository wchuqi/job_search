# AI Context Engineering 学习资料：记忆系统和状态管理

[返回索引](../AI Context Engineering学习资料.md)

## 学习目标

- 区分短期状态和长期记忆。
- 掌握记忆写入、读取、更新、删除和冲突处理。
- 能设计 Agent 的状态 schema。

## 理论导读

记忆不是把所有历史永久保存。工程上要区分：当前任务的工作状态、用户确认的长期偏好、过去交互的事件记录、系统操作流程知识。记忆进入上下文前需要筛选，否则旧记忆会污染新任务。

## 记忆类型

| 类型 | 内容 | 例子 |
| --- | --- | --- |
| Working memory | 当前任务状态 | 当前步骤、待办、最近工具结果 |
| Episodic memory | 过去事件 | 上次模拟面试暴露了 JVM 弱项 |
| Semantic memory | 稳定事实 | 用户目标岗位是 Java 后端 |
| Procedural memory | 做事方法 | 简历修改按 STAR 法展开 |
| Preference memory | 用户偏好 | 喜欢中文、要高密度资料 |

## 记忆写入规则

记忆应该满足至少一条：

- 用户明确确认。
- 多次行为稳定出现。
- 对未来任务有明显帮助。
- 与当前目标强相关。
- 不包含敏感或短期无效信息。

## 状态 schema 示例

```json
{
  "task_id": "interview-prep-001",
  "goal": "生成 AI Context Engineering 学习资料",
  "constraints": ["中文", "拆分文件", "含面试题"],
  "facts": [],
  "decisions": [],
  "actions_done": [],
  "pending": [],
  "risks": [],
  "last_updated": "2026-06-07"
}
```

## 冲突和遗忘

记忆必须允许：

- 新事实覆盖旧事实。
- 用户删除或修改偏好。
- 按时间过期。
- 按任务范围隔离。
- 对敏感内容不写入长期存储。

## 练习

为“长期求职助手”设计长期记忆表，字段至少包含：事实、偏好、技能短板、已投公司、面试反馈、隐私等级、更新时间、确认状态。

## 验收

- 能区分历史记录、任务状态和长期记忆。
- 能说明何时写入记忆、何时只保留在当前会话。
- 能处理用户偏好变更。

## 重点

记忆进入模型上下文前必须经过检索和过滤。长期保存不等于每次都注入。

## 难点

难点是“记忆可信度”：AI 总结出来的记忆可能误解用户，最好带确认状态和来源。

## 易错

> **易错：** 把模型自己推断出的用户画像直接长期保存。
>
> 正确做法：重要记忆应来自用户明确表达或经过确认，并允许查看、修改和删除。
