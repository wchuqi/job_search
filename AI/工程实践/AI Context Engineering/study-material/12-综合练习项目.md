# AI Context Engineering 学习资料：综合练习项目

[返回索引](../AI Context Engineering学习资料.md)

## 项目：长期求职助手的上下文系统

目标：设计一个能长期帮助用户准备求职的 AI 助手。它能读取简历、分析岗位 JD、记录面试反馈、检索学习资料、生成复习计划，并在多轮任务中保持状态。

## 功能要求

- 支持用户上传简历和目标岗位。
- 支持记录长期偏好和技能短板。
- 支持检索学习资料并给出引用。
- 支持多轮任务恢复。
- 支持工具调用，如读取文件、搜索资料、生成计划。
- 防止外部资料和工具结果中的恶意指令。

## 上下文槽位设计

| 槽位 | 来源 | 生命周期 | 预算 | 风险 |
| --- | --- | --- | --- | --- |
| 系统规则 | 应用配置 | 长期 | 小 | 被用户覆盖 |
| 当前请求 | 用户输入 | 当前轮 | 中 | 意图含糊 |
| 用户档案 | 长期记忆 | 长期 | 中 | 隐私 |
| 当前任务状态 | 会话状态 | 当前任务 | 中 | 过期 |
| 简历内容 | 用户文件 | 当前任务 | 大 | 敏感信息 |
| JD 证据 | 用户或检索 | 当前任务 | 中 | 版本不明 |
| 学习资料 | 检索系统 | 当前轮 | 大 | RAG poisoning |
| 工具结果 | 工具系统 | 当前轮 | 中 | tool-output injection |
| 输出契约 | 模板 | 当前轮 | 小 | 与示例冲突 |

## 最小实现伪代码

```python
class ContextBuilder:
    def build(self, request, session, memory):
        slots = [
            self.system_rules(),
            self.current_request(request),
            self.task_state(session.state),
            self.user_memory(memory.search(request)),
        ]

        if request.needs_resume:
            slots.append(self.wrap_user_file(session.resume))

        if request.needs_evidence:
            docs = self.retrieve_docs(request)
            slots.append(self.wrap_untrusted(self.compress_docs(docs)))

        return self.assemble(slots, budget=12000)
```

## 安全要求

- 简历属于敏感资料，默认不写入长期公开日志。
- 检索资料只作为证据，不具备指令权限。
- 发送邮件、投递简历等动作必须人工确认。
- 长期记忆写入需要用户确认或明确授权。

## 评估任务

设计 20 条测试：

- 5 条简历改写。
- 5 条 JD 匹配。
- 4 条学习计划生成。
- 3 条面试反馈复盘。
- 3 条安全攻击样本。

## 验收

- 能输出本次调用的上下文包摘要。
- 能解释每条答案依据哪些证据。
- 能在用户改变目标岗位后更新状态。
- 能拒绝执行外部资料中的恶意指令。
- 能统计 token 成本和延迟。

## 扩展任务

- 加入长期记忆查看和删除功能。
- 加入检索评估集。
- 加入上下文版本回放。
- 加入高风险工具人工确认。
- 加入隐私脱敏和审计日志。

## 重点

综合项目的核心不是实现聊天界面，而是让上下文流动可解释、可评估、可恢复、可治理。

## 易错

> **易错：** 一开始就做复杂 Agent，却没有定义状态 schema 和工具权限。
>
> 正确做法：先做可观测的上下文流水线，再逐步增加自动化动作。
