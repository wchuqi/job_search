# AI Context Engineering 学习资料：上下文选择、排序、预算和冲突算法深度解析

[返回索引](../AI Context Engineering学习资料.md)

## 学习目标

- 掌握上下文选择的评分模型。
- 理解预算分配、排序、去重和冲突处理。
- 能把“凭感觉拼 prompt”改造成可测试算法。

## 理论导读

上下文选择是一个受约束的排序和打包问题：在有限 token 预算内，选出最能支持当前任务的信息，同时控制风险和冲突。它类似搜索结果排序、背包问题和权限过滤的组合。

## 基础评分模型

```text
score(item, task) =
  w1 * relevance
+ w2 * authority
+ w3 * recency
+ w4 * utility
+ w5 * diversity
- w6 * token_cost
- w7 * risk
- w8 * conflict_penalty
```

解释：

- relevance：和当前问题是否相关。
- authority：来源是否可信。
- recency：是否符合当前版本和时间。
- utility：是否能直接支持答案或动作。
- diversity：是否补充不同角度证据。
- token_cost：占用预算。
- risk：是否包含注入、隐私、越权风险。
- conflict_penalty：是否与更高权威信息冲突。

## 决策顺序

不要先排序再过滤。推荐顺序：

```text
权限过滤 -> 安全过滤 -> 任务相关性筛选 -> 冲突标注
-> 分组去重 -> 预算分配 -> 排序打包 -> 最终校验
```

权限过滤必须早于相关性排序。越权资料即使高度相关，也不能进入上下文。

## 预算分配算法

### 静态预算

适合任务类型稳定的系统：

```yaml
system_rules: 8%
current_request: 10%
state: 15%
evidence: 45%
tool_results: 12%
schema: 5%
reserved: 5%
```

### 动态预算

根据任务类型调整：

| 任务 | 预算重点 |
| --- | --- |
| 文档问答 | evidence |
| 多轮规划 | state 和 memory |
| 工具执行 | tool_results 和 permission |
| 结构化抽取 | schema 和 examples |
| 代码修复 | files、tests、stack traces |

## 去重和多样性

RAG 中常见问题是 top-k 全部来自同一段重复内容。需要按文档、章节、版本、观点分组，保留最强证据，同时补充覆盖不同子问题的片段。

```python
def diversify(items, group_key, max_per_group=2):
    selected = []
    groups = {}
    for item in sorted(items, key=lambda x: x.score, reverse=True):
        key = group_key(item)
        if groups.get(key, 0) < max_per_group:
            selected.append(item)
            groups[key] = groups.get(key, 0) + 1
    return selected
```

## 冲突处理

冲突不是简单删除。系统应保留冲突信息，给模型或上层逻辑明确处理规则。

| 冲突 | 优先规则 |
| --- | --- |
| 官方文档 vs 博客 | 官方文档优先 |
| 新版本 vs 旧版本 | 当前目标版本优先 |
| 当前请求 vs 历史偏好 | 当前请求优先 |
| 工具结果 vs 模型推测 | 工具结果优先 |
| 安全规则 vs 用户要求 | 安全规则优先 |

## 最终打包检查

- 是否包含当前任务目标。
- 是否包含硬约束。
- 是否包含足够证据。
- 是否有来源和时间。
- 是否有不可信内容包装。
- 是否留出输出预算。
- 是否存在未解释冲突。

## 练习

给 12 条候选上下文片段打分，字段包括 relevance、authority、recency、risk、token_cost。要求在 4000 token 内选出最终上下文，并解释每个被舍弃片段的原因。

## 验收

- 能写出上下文评分公式。
- 能说明权限过滤为什么在相关性排序之前。
- 能处理重复证据和冲突证据。

## 重点

上下文选择要可解释。生产系统中“为什么模型看到了这段内容”必须有答案。

## 难点

难点是权重调节：不同任务对 relevance、authority、recency 和 risk 的权重不同，需要通过评测集调参。

## 易错

> **易错：** 用单一相似度分数决定所有上下文。
>
> 正确做法：把权限、风险、权威度、时效、成本和多样性纳入选择逻辑。
