# AI Context Engineering 学习资料：多模态和代码上下文工程

[返回索引](../AI Context Engineering学习资料.md)

## 学习目标

- 理解图片、表格、音频、视频和代码仓库作为上下文时的特殊问题。
- 掌握代码上下文选择、文件摘要和错误信息压缩。
- 能为多模态任务设计证据链。

## 理论导读

多模态上下文不只是“上传图片”。图片需要关注区域、OCR、视觉证据和不确定性；表格需要 schema、样本和统计；代码仓库需要目录结构、相关文件、依赖、测试输出和修改历史。模型看到的上下文必须能支持具体判断。

## 多模态上下文

| 类型 | 关键上下文 | 风险 |
| --- | --- | --- |
| 图片 | 原图、裁剪区域、OCR、视觉描述 | 看错细节、忽略小字 |
| 表格 | schema、字段含义、样本、统计 | 把样本当全量 |
| 音频 | 转写、时间戳、说话人 | 转写错误 |
| 视频 | 关键帧、时间段、事件摘要 | 时间顺序丢失 |
| PDF | 页面、段落、表格、脚注 | 版式解析错 |

## 代码上下文

代码任务的上下文通常包括：

- 用户目标。
- 相关文件路径。
- 目录结构。
- 相关函数或类。
- 配置和依赖。
- 测试命令和结果。
- 错误堆栈。
- 最近 diff。
- 代码风格和约束。

## 文件选择思路

```text
入口文件 -> 调用链 -> 失败堆栈 -> 测试文件 -> 配置文件 -> 相关文档
```

不要把整个仓库一次性塞给模型。应该先检索符号、路径和错误堆栈，再选出最小相关文件集。

## 例子

```json
{
  "code_context": {
    "task": "修复登录接口测试失败",
    "files": [
      {"path": "src/auth/service.py", "reason": "失败函数所在文件"},
      {"path": "tests/test_auth.py", "reason": "失败测试"},
      {"path": "config/settings.py", "reason": "认证配置"}
    ],
    "test_failure": "test_login_empty_password expects 400 but got 500",
    "constraints": ["不要改公开 API", "保持已有错误码语义"]
  }
}
```

## 练习

给一个“修复编译错误”的任务列出应进入上下文的文件和不应进入上下文的文件，并说明理由。

## 验收

- 能解释多模态证据的不确定性。
- 能为代码任务选择最小相关上下文。
- 能把错误堆栈压缩成可操作摘要。

## 重点

代码上下文工程的核心是相关性和可验证性：模型要看到足够的调用链和测试信息，但不需要看到所有文件。

## 难点

难点是跨文件依赖：一个错误可能由配置、依赖版本、测试数据和运行环境共同导致。

## 易错

> **易错：** 只给模型报错最后一行。
>
> 正确做法：同时给失败测试、相关堆栈、目标函数、输入数据和预期行为。
