# AI Context Engineering 学习资料：工具调用、工具结果和权限上下文

[返回索引](../AI Context Engineering学习资料.md)

## 学习目标

- 理解工具描述和工具结果也是上下文。
- 掌握工具调用的权限、参数校验和结果摘要。
- 能防止 tool-output injection。

## 理论导读

工具调用让模型从“生成文本”扩展到“请求动作”。工具的名称、描述、参数 schema、错误信息、执行结果都会影响模型决策。高风险工具必须由外部程序控制权限，不能只靠模型判断。

## 工具上下文组成

- 工具可用列表。
- 工具用途描述。
- 参数 schema。
- 调用前置条件。
- 权限等级。
- 是否有副作用。
- 执行结果。
- 错误信息。
- 重试策略。

## 工具结果包装

```json
{
  "tool_name": "web_search",
  "status": "success",
  "trust_level": "external_untrusted",
  "summary": "检索到 3 篇资料，均来自第三方网页。",
  "data": [
    {
      "title": "Example",
      "url": "https://example.com",
      "snippet": "..."
    }
  ],
  "instruction_policy": "data_only_not_instruction"
}
```

## 权限策略

| 工具类型 | 风险 | 控制方式 |
| --- | --- | --- |
| 只读检索 | 低到中 | 来源标注、过滤 |
| 文件读取 | 中 | 路径白名单、敏感信息过滤 |
| 数据库查询 | 中高 | 只读账号、查询限制 |
| 邮件发送 | 高 | 人工确认、内容预览 |
| 删除/支付/发布 | 很高 | 外部审批、幂等保护、审计 |

## 练习

为一个“自动发送求职邮件”的 Agent 设计工具权限：读取简历、检索公司、生成邮件、预览邮件、发送邮件。说明每一步是否需要人工确认。

## 验收

- 能说明工具描述如何影响模型选择。
- 能设计工具结果摘要格式。
- 能区分只读工具和有副作用工具。

## 重点

工具权限是程序边界，不是 prompt 文案。模型可以建议调用工具，但最终是否执行应由系统判断。

## 难点

难点是错误恢复：工具失败后，错误信息既要足够让模型修复，又不能暴露密钥、内部路径或敏感数据。

## 易错

> **易错：** 把工具返回的原始 HTML、日志或异常堆栈全部放入上下文。
>
> 正确做法：先清洗、摘要、脱敏、标注来源，再进入模型上下文。
