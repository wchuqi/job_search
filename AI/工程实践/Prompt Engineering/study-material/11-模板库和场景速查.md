# Prompt Engineering 学习资料：模板库和场景速查

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 掌握常见任务的 prompt 骨架。
- 能根据具体场景改写模板。
- 避免把模板当作万能答案。

## 通用模板

```text
你是{角色}。

任务：{要完成的事情}

目标：{成功标准}

输入：
<{input_tag}>
{用户材料}
</{input_tag}>

规则：
- {必须遵守的规则}
- {禁止事项}
- 信息不足时输出：{固定拒答/追问}

输出格式：
{字段或结构}
```

## 分类任务

```text
请把工单分类到以下枚举之一：
- billing
- technical
- account
- other

只输出 JSON：
{
  "category": "...",
  "confidence": 0.0,
  "evidence": ["..."]
}
```

## 摘要任务

```text
请总结 <meeting_notes>。

输出：
1. 决策事项
2. 待办事项：负责人、截止时间、动作
3. 风险
4. 未决问题

不要添加原文没有的信息。
```

## 改写任务

```text
请把文本改写为{目标风格}。

保留：
- 事实
- 数字
- 专有名词

禁止：
- 添加新承诺
- 改变语义
```

## RAG 问答任务

```text
只依据 <context> 回答。
每个结论引用资料 id。
资料不足时输出“资料中没有足够信息”。
```

## 评审任务

```text
请按以下维度评审：
- 正确性
- 完整性
- 风险
- 可执行性
- 测试缺口

每个问题都要包含：问题、影响、证据、建议。
```

## 练习

从本页选 3 个模板，分别改写成：

- 招聘场景。
- 代码审查场景。
- 客服知识库场景。

## 验收

- 能根据任务改写模板，而不是原样套用。
- 能补充业务边界、输出格式和失败处理。
- 能识别模板和实际需求冲突的地方。

## 易错

> **易错：** 从网上复制“万能提示词”。
>
> 正确做法：先定义任务和评估标准，再选择模板骨架。
