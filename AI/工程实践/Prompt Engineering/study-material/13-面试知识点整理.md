# Prompt Engineering 学习资料：面试知识点整理

## 使用方式

- 先读 [01-基础概念.md](面试知识点/01-基础概念.md)，确保术语表达准确。
- 再读 [02-提示设计和输出控制.md](面试知识点/02-提示设计和输出控制.md)，准备常见设计题。
- 然后读 [03-RAG工具调用和Agent.md](面试知识点/03-RAG工具调用和Agent.md)，准备工程场景。
- 最后读 [04-安全评估和生产实践.md](面试知识点/04-安全评估和生产实践.md)，这是中高级岗位的重点。
- 如果准备中高级或 AI 应用工程岗位，继续读 [05-深度机制和生产场景题.md](面试知识点/05-深度机制和生产场景题.md)。

## 高频考察点

- Prompt Engineering 和普通聊天的区别。
- 如何设计一个稳定的 prompt。
- few-shot、zero-shot、反例的作用。
- 如何控制 JSON 输出。
- RAG 和微调的区别。
- 工具调用如何设计权限和失败处理。
- Agent 为什么容易失控。
- 如何评估 prompt 改动。
- 提示注入是什么，如何防护。
- 生产 prompt 如何版本管理和回滚。
- 长上下文为什么不能直接替代 RAG。
- 工具调用的参数幻觉、权限校验和审计。
- Agent 状态机、循环防护和失败恢复。
- PromptOps 的灰度、监控、成本和治理。
