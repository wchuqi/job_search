# Prompt Engineering 学习资料：安全威胁建模、OWASP 和权限隔离深度解析

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 从威胁建模角度理解 LLM 应用安全。
- 掌握 prompt injection、sensitive information disclosure、insecure output handling、excessive agency 等风险。
- 能为 AI 应用设计权限隔离和安全评审。

## 威胁建模视角

不要只问“prompt 怎么防攻击”，要问：

```text
资产是什么？
攻击者是谁？
入口在哪里？
模型能看到什么？
模型能调用什么？
输出会进入哪里？
失败后影响多大？
```

## 资产

- 系统提示词。
- 用户隐私数据。
- API key 和 token。
- 工具权限。
- 数据库内容。
- 内部文档。
- 模型输出进入的下游系统。

## 攻击面

| 攻击面 | 例子 |
| --- | --- |
| 用户输入 | 直接提示注入 |
| 外部网页 | 间接提示注入 |
| 文档/RAG | 恶意文档片段 |
| 工具结果 | 返回文本中嵌入指令 |
| 多轮历史 | 逐步诱导越权 |
| 输出通道 | XSS、SQL、命令注入 |

## 关键风险

### Prompt Injection

攻击者试图改变模型行为。防护核心是把不可信内容降级为数据，并在系统层做权限控制。

### Sensitive Information Disclosure

模型泄露敏感信息。防护核心是敏感信息最小化进入上下文，输出前做 DLP 或安全分类。

### Insecure Output Handling

模型输出被下游当作可信代码、SQL、HTML 或命令执行。防护核心是下游转义、校验、沙箱执行。

### Excessive Agency

模型拥有过大行动权限。防护核心是工具最小权限、写操作确认、最大步数和审计。

## 权限隔离架构

```text
用户 -> 应用后端鉴权 -> LLM 编排层 -> 工具代理 -> 受限工具/API
                         |             |
                         v             v
                    安全策略       审计日志
```

原则：

- 模型不直接拿密钥。
- 工具代理按用户身份执行。
- 每个工具有 scope。
- 高风险操作需要 human approval。
- 工具结果脱敏后再给模型。

## 安全评审清单

- 模型上下文中是否含敏感数据？
- 外部资料是否标为不可信？
- 工具权限是否最小化？
- 写操作是否确认？
- 输出是否进入浏览器、数据库、shell 或邮件？
- 是否有红队评估集？
- 是否记录审计日志？
- 是否有事故回滚和封禁策略？

## 练习

为“AI 邮件助手”做威胁建模：

- 资产。
- 攻击入口。
- 高风险工具。
- 提示注入样例。
- 权限隔离方案。
- 审计字段。

## 验收

- 能用威胁建模分析 LLM 应用。
- 能解释 OWASP LLM 风险和 prompt 的关系。
- 能设计权限隔离而不是只靠提示词。

## 难点

LLM 安全的核心是系统安全，不是 prompt 文案安全。Prompt 是策略表达，后端权限和数据隔离才是强控制。
