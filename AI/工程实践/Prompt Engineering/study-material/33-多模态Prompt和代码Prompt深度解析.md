# Prompt Engineering 学习资料：多模态 Prompt 和代码 Prompt 深度解析

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 掌握图像、表格、截图、代码等非纯文本输入的 prompt 设计。
- 理解多模态任务的观察、定位、引用和不确定性表达。
- 能设计代码生成、代码审查和调试 prompt。

## 多模态 Prompt

多模态输入不只是“看图回答”。要定义模型应该观察什么、忽略什么、如何引用位置、如何表达不确定性。

### 图片/截图分析模板

```text
请分析图片中的 UI 问题。

要求：
- 先描述可见元素。
- 再指出布局、可读性、交互状态问题。
- 不要推测图片外的信息。
- 如果文字看不清，明确说明。
- 输出问题、位置、影响、建议。
```

### 多模态风险

- OCR 识别错误。
- 小字不可见。
- 图片裁剪遗漏上下文。
- 模型误判空间关系。
- 对图中人物或敏感属性做不当推断。

## 表格 Prompt

表格任务要明确：

- 行列含义。
- 缺失值。
- 单位。
- 统计口径。
- 是否允许计算。
- 是否需要保留原始值。

## 代码 Prompt

代码任务常见类型：

- 生成代码。
- 解释代码。
- 找 bug。
- 重构。
- 写测试。
- 性能分析。
- 安全审计。

### 代码审查 Prompt

```text
你是代码审查助手。

优先级：
1. 明确 bug。
2. 安全风险。
3. 并发/边界问题。
4. 测试缺口。
5. 可维护性。

每个问题必须包含：
- 文件/位置。
- 触发条件。
- 影响。
- 修复建议。

不要提出无关风格建议。
```

### 代码生成 Prompt

关键要素：

- 语言和版本。
- 依赖限制。
- 输入输出。
- 错误处理。
- 边界条件。
- 测试用例。
- 性能要求。

## 代码 Prompt 失败模式

| 失败 | 防护 |
| --- | --- |
| 编造 API | 提供官方文档或要求不确定时说明 |
| 忽略边界 | 明确边界测试 |
| 生成不可运行代码 | 要求最小可运行示例和测试 |
| 安全漏洞 | 加安全约束和审计 |
| 过度重构 | 限制修改范围 |

## 练习

设计 3 个 prompt：

- UI 截图问题分析。
- Java 代码审查。
- SQL 性能优化建议。

每个 prompt 都要包含输出结构和拒绝推测规则。

## 验收

- 能说明多模态 prompt 的观察边界。
- 能写代码审查 prompt。
- 能识别代码生成中的 API 幻觉和安全风险。

## 易错

> **易错：** 让模型“看图猜业务”或“按常识补代码依赖”。
>
> 正确做法：限定可见信息、给出环境约束、要求不确定时说明。
