# Prompt Engineering 学习资料：工具调用和函数调用

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 理解工具调用和普通文本生成的区别。
- 能设计工具 schema、调用边界和失败处理。
- 能避免模型越权操作和参数幻觉。

## 理论导读

工具调用让模型不只生成文本，还能选择调用外部函数，例如查数据库、搜索、计算、发邮件、创建工单。Prompt 的职责是告诉模型什么时候调用、调用哪个工具、参数如何填、什么时候不调用，以及调用失败后怎么处理。

## 工具设计原则

| 原则 | 说明 |
| --- | --- |
| 单一职责 | 一个工具完成一类清晰动作 |
| 参数明确 | 使用类型、枚举、必填字段 |
| 权限最小化 | 不把危险能力暴露给模型 |
| 可回滚 | 写操作前确认，保留审计 |
| 可校验 | 工具返回结构化结果 |
| 失败可处理 | 超时、无结果、权限不足都要定义 |

## Prompt 示例

```text
你可以使用 `search_jobs` 查询职位。

调用规则：
- 当用户要求匹配职位时，必须先调用 `search_jobs`。
- 如果用户没有提供城市或岗位方向，先追问，不要调用工具。
- 不要编造工具没有返回的职位。
- 工具返回空列表时，输出没有匹配职位，并给出调整建议。
```

## 工具 Schema 思维

```json
{
  "name": "search_jobs",
  "description": "Search job postings by role, city and seniority.",
  "parameters": {
    "role": "string",
    "city": "string",
    "seniority": "junior | mid | senior"
  }
}
```

## 失败处理

| 失败 | Prompt/系统处理 |
| --- | --- |
| 参数不足 | 追问用户 |
| 工具超时 | 提示稍后重试或降级 |
| 无结果 | 说明没有匹配结果 |
| 权限不足 | 拒绝操作并说明需要授权 |
| 写操作高风险 | 要求用户确认 |

## 练习

为“候选人求职助手”设计 3 个工具：

- 搜索职位。
- 分析简历。
- 生成投递邮件。

写出每个工具的输入、输出、调用条件和禁止条件。

## 验收

- 能区分模型决策和工具执行。
- 能设计工具调用条件。
- 能说明为什么写操作需要确认和审计。

## 易错

> **易错：** 把工具结果和模型推测混在一起。
>
> 正确做法：明确要求“只根据工具返回的数据生成结论”。
