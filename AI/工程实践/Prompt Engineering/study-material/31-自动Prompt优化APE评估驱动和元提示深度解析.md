# Prompt Engineering 学习资料：自动 Prompt 优化、APE、评估驱动和元提示深度解析

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 理解自动 prompt 优化的流程和风险。
- 掌握评估驱动迭代、候选生成、选择和防过拟合。
- 能设计元提示，让模型帮助改进 prompt。

## 自动 Prompt 优化是什么

自动 Prompt 优化是用模型或搜索算法生成多个 prompt 候选，再用评估集选择表现更好的版本。

简化流程：

```text
初始 prompt -> 生成候选 prompt -> 在评估集运行 -> 打分 -> 选择/组合 -> 回归测试
```

## 候选生成方式

| 方法 | 说明 |
| --- | --- |
| 人工改写 | 质量高，速度慢 |
| LLM 改写 | 快速生成候选 |
| 元提示 | 让模型根据失败案例改 prompt |
| 随机搜索 | 改变措辞、顺序、示例 |
| 网格搜索 | 对模型、温度、top-k 组合试验 |

## 元提示示例

```text
你是 prompt 优化专家。

给定：
1. 当前 prompt
2. 失败样例
3. 评分规则

任务：
- 找出失败模式。
- 提出不超过 3 个 prompt 修改。
- 每个修改说明预期改善的指标。
- 不要改变输出 schema。
```

## 失败案例驱动

自动优化不能只看成功样例，要围绕失败归因：

- 格式失败。
- 事实错误。
- 引用错误。
- 拒答错误。
- 安全违规。
- 过度冗长。
- 遗漏关键字段。

## 防过拟合

Prompt 也会过拟合评估集。表现是：benchmark 分数上升，线上失败增多。

防护：

- train/dev/test 分离。
- 历史失败集只作为回归，不直接优化到死。
- 保留盲测集。
- 按场景分层看指标。
- 人工审查 prompt 是否引入狭隘规则。

## Prompt 变更评审

每个自动生成候选都要问：

- 是否改变任务边界？
- 是否破坏安全规则？
- 是否改变 schema？
- 是否增加成本过多？
- 是否只针对少数样例打补丁？

## 练习

为“客服工单分类 prompt”设计自动优化流程：

- 失败样例输入。
- 元提示。
- 候选 prompt 数量。
- 评估指标。
- 选择规则。
- 防过拟合策略。

## 验收

- 能描述自动 prompt 优化全流程。
- 能写元提示。
- 能说明为什么自动优化必须配合评估集和人工审查。

## 易错

> **易错：** 让模型直接“把 prompt 优化到最好”。
>
> 正确做法：给失败样例、评分标准和不可改变的约束，再用评估选择。
