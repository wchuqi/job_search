# Prompt Engineering 学习资料：评估、迭代和版本管理

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 建立 prompt 评估闭环。
- 掌握测试集、指标、评分表和版本管理。
- 能把 prompt 改动纳入工程发布流程。

## 理论导读

Prompt Engineering 的专业分水岭是评估。没有评估时，优化只是主观感觉；有评估时，prompt 才能像代码一样被测试、审查、发布和回滚。

## 评估维度

| 维度 | 指标 |
| --- | --- |
| 正确性 | 准确率、召回率、事实错误率 |
| 稳定性 | 多次运行一致性、格式合规率 |
| 完整性 | 遗漏率、覆盖率 |
| 安全性 | 越权率、泄露率、注入成功率 |
| 体验 | 可读性、帮助度、人工评分 |
| 成本 | token 数、延迟、工具调用次数 |

## 评估集设计

评估集应覆盖：

- 常规样例。
- 边界样例。
- 反例和恶意输入。
- 缺失信息。
- 多语言或格式异常。
- 真实线上失败案例。

## 版本管理

推荐把生产 prompt 放在代码或配置中，并配套：

- prompt 版本号。
- 修改原因。
- 关联评估结果。
- 发布范围。
- 回滚方式。
- 模型版本和参数。

## 示例评分表

| 评分项 | 0 分 | 1 分 | 2 分 |
| --- | --- | --- | --- |
| 准确性 | 明显错误 | 部分正确 | 完全符合资料 |
| 格式 | 无法解析 | 字段不完整 | 字段完整合法 |
| 引用 | 无引用 | 引用不全 | 关键结论有引用 |
| 安全 | 有越权/泄露 | 有轻微风险 | 符合安全规则 |

## 练习

为一个“简历评分 prompt”建立 20 条评估样例，并记录：

- 输入。
- 期望输出。
- 评分规则。
- 当前 prompt 得分。
- 失败原因。
- 改进方案。

## 验收

- 能解释为什么单次试跑不足以证明 prompt 有效。
- 能设计覆盖正常、边界、恶意输入的评估集。
- 能把 prompt 改动纳入版本管理。

## 易错

> **易错：** 改 prompt 后只挑成功案例展示。
>
> 正确做法：固定评估集，每次改动都跑同一批样例。
