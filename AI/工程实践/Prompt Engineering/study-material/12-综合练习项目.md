# Prompt Engineering 学习资料：综合练习项目

[返回索引](../Prompt Engineering学习资料.md)

## 项目：JD 与简历匹配助手

目标：构建一个 prompt 驱动的候选人筛选助手，根据 JD 和简历输出匹配结论、证据、风险和面试问题。

## 功能要求

- 输入 JD 和简历。
- 只依据输入材料分析。
- 输出推荐结论：`recommend`、`hold`、`reject`。
- 输出 0-100 分。
- 输出匹配点、风险点、缺失信息和下一轮问题。
- 信息不足时标注不确定项，不编造。

## Prompt 草案

```text
你是技术招聘筛选助手。

任务：
根据 JD 和候选人简历，判断是否推荐进入下一轮面试。

规则：
- 只依据 <job_description> 和 <resume>。
- 不要编造候选人没有写的经历。
- 对每个结论给出证据。
- 信息不足时写入 missing_evidence。

输出 JSON：
{
  "decision": "recommend | hold | reject",
  "score": 0,
  "matched_points": [{"point": "", "evidence": ""}],
  "risks": [{"risk": "", "evidence": ""}],
  "missing_evidence": [],
  "interview_questions": []
}

<job_description>
...
</job_description>

<resume>
...
</resume>
```

## 评估集

至少准备 20 条样例：

- 明显匹配。
- 明显不匹配。
- 简历缺少关键项目。
- JD 要求模糊。
- 候选人经历夸张但无证据。
- 初级/中级/高级不同级别。
- 中英文混合。
- 恶意输入要求忽略规则。

## 评分标准

| 指标 | 目标 |
| --- | --- |
| JSON 合规率 | 100% |
| 证据引用完整率 | 90%+ |
| 明显错判率 | 低于 5% |
| 编造经历次数 | 0 |
| 恶意输入越权率 | 0 |

## 扩展任务

- 加入 RAG：检索岗位能力模型。
- 加入工具调用：搜索历史面试题库。
- 加入版本管理：记录 prompt v1、v2、v3 改动。
- 加入人工审核：低置信度时交给招聘专员。

## 验收

- 有完整 prompt。
- 有测试样例和评分表。
- 能说明失败案例如何修复。
- 能讲清楚 prompt、RAG、工具调用和人工审核的边界。
