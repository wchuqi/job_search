## 一、机制追问

### 1. 为什么 prompt 改了几个词，输出可能明显变化？

**参考答案：**

模型是在上下文条件下逐 token 生成，prompt 改动会改变任务语义、约束权重、示例模式和后续 token 概率。特别是输出格式、否定规则、示例、上下文顺序和关键术语变化，会影响模型对任务的理解。生产中应通过固定评估集验证，而不是凭单次输出判断。

### 2. 长上下文窗口是否能替代 RAG？

**参考答案：**

不能简单替代。长上下文能放更多资料，但仍有成本高、注意力稀释、引用不准、资料冲突和更新困难的问题。RAG 的价值是检索相关资料、控制来源、降低上下文噪声并支持引用评估。长上下文和 RAG 可以组合使用。

## 二、RAG 场景题

### 3. 知识库问答答错了，你怎么排查？

**参考答案：**

先分层定位：检索是否召回正确文档，重排是否把正确文档排前，上下文是否包含足够证据，prompt 是否要求引用和拒答，生成答案的引用是否支持结论。不能只改 prompt，要同时看检索日志、上下文拼装和输出校验。

> **重点：** RAG 错误要区分 retrieval failure 和 generation failure。

### 4. 如何评估 RAG 引用是否可靠？

**参考答案：**

检查引用是否存在、是否来自允许来源、是否支持对应结论、是否遗漏冲突资料、是否引用过期资料。可以建立人工标注集，也可以先用 LLM-as-judge 初筛，再人工抽检关键样例。

## 三、工具调用和 Agent 场景题

### 5. 工具调用中模型填错参数怎么办？

**参考答案：**

prompt 中要定义缺参追问、枚举范围和参数格式；工具执行前后端必须做 schema 校验、权限校验和业务校验。非法参数拒绝执行，必要时让模型根据错误信息修复，但不能绕过后端校验。

### 6. Agent 一直循环调用工具，怎么防？

**参考答案：**

设计状态机和停止条件，限制最大步数和最大工具调用次数；记录已尝试动作；工具失败时进入恢复或降级；低置信度时追问或交给人工。Agent 不应自由行动，而应在受控 workflow 中运行。

## 四、安全和生产题

### 7. 为什么 prompt 安全不能只靠系统提示词？

**参考答案：**

系统提示词能提供行为约束，但模型仍可能受不可信输入、外部网页、历史状态和工具结果影响。真实安全需要多层防护：输入隔离、敏感信息不进上下文、后端权限校验、工具最小权限、写操作确认、输出审计和红队测试。

### 8. 如何上线一个生产 prompt？

**参考答案：**

像代码一样上线：建立 prompt 版本，固定模型和参数，准备离线评估集，做安全评审，灰度发布，监控格式合规率、错误率、拒答率、成本、延迟和安全违规，出现问题可回滚。

### 9. prompt、微调、RAG、工具调用怎么取舍？

**参考答案：**

prompt 适合任务规格和输出控制；RAG 适合外部知识和引用；工具调用适合查询、计算和操作；微调适合稳定风格、领域标签和固定模式。生产系统通常组合使用，并用评估验证取舍。

### 10. 如果 prompt 评估集得分提高，但线上投诉变多，可能是什么原因？

**参考答案：**

可能评估集不代表线上分布，边界样例不足，优化了平均分但伤害了关键场景，线上输入含恶意或脏数据，模型版本/参数不一致，RAG 检索分布不同，或指标没有覆盖用户体验。应收集线上失败样例加入回归集，并分场景看指标。
