## 一、RAG

### 1. RAG 和 Prompt Engineering 是什么关系？

**参考答案：**

RAG 提供外部知识，Prompt Engineering 规定模型如何使用这些知识。RAG prompt 要说明只依据检索资料回答、如何引用、资料不足时如何拒答、资料冲突时如何处理。

### 2. RAG 和微调怎么选择？

**参考答案：**

知识经常变化、需要引用来源、依赖私有文档时优先 RAG。需要固定风格、稳定标签或领域表达习惯时可以考虑微调。很多生产系统会同时使用 prompt、RAG、工具和少量微调。

## 二、工具调用

### 3. 工具调用 prompt 需要写哪些规则？

**参考答案：**

需要写工具可用场景、参数要求、禁止条件、失败处理、权限边界和写操作确认。工具结果要和模型推测区分，不能让模型编造工具没有返回的数据。

### 4. 如何防止工具调用越权？

**参考答案：**

不能只靠 prompt。后端必须按用户身份做权限校验，工具能力要最小化，写操作要二次确认，敏感操作要审计，模型不能直接接触密钥。

## 三、Agent

### 5. Agent 和普通聊天机器人有什么区别？

**参考答案：**

普通聊天机器人主要回答问题；Agent 会围绕目标进行多步骤计划、调用工具、观察结果、更新状态并继续执行。Agent 需要状态管理、权限控制、停止条件和失败恢复。

### 6. Agent 为什么容易失控？

**参考答案：**

因为它能多轮行动和调用工具。如果目标不清、工具权限过大、状态污染、缺少确认机制或没有停止条件，就可能循环、越权、误操作或泄露数据。

> **重点：** Agent 设计的核心是可控行动，不是让模型无限自主。
