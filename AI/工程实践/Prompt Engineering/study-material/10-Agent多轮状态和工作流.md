# Prompt Engineering 学习资料：Agent、多轮状态和工作流

[返回索引](../Prompt Engineering学习资料.md)

## 学习目标

- 理解 Agent 与单次问答的区别。
- 掌握计划、工具、状态、记忆和检查点。
- 能设计可控的多轮任务流程。

## 理论导读

Agent 是能在目标驱动下进行多步骤操作的 AI 系统。它通常包含模型、工具、状态、记忆、权限、规划和评估。Prompt 在 Agent 中负责定义角色、目标、行动边界、工具选择、停止条件和用户确认规则。

## Agent 工作流

```text
用户目标 -> 澄清需求 -> 制定计划 -> 调用工具 -> 观察结果 -> 修正计划 -> 输出结果 -> 自检
```

## 关键设计点

| 设计点 | 问题 |
| --- | --- |
| 目标 | Agent 最终要完成什么 |
| 状态 | 哪些信息跨轮保留 |
| 工具 | 可以做什么，不能做什么 |
| 权限 | 哪些操作需要确认 |
| 停止条件 | 何时结束，何时追问 |
| 失败恢复 | 工具失败、信息不足、冲突时怎么办 |
| 审计 | 记录哪些决策和调用 |

## 示例 Prompt

```text
你是求职流程 Agent。

目标：
帮助用户完成职位筛选、简历匹配和投递准备。

规则：
- 没有用户确认，不要发送邮件或提交申请。
- 每次工具调用后，基于返回结果更新计划。
- 如果缺少城市、岗位方向或工作年限，先追问。
- 每轮结束时给出当前状态和下一步。
```

## 多轮状态

状态应结构化保存，而不是依赖模型自然记忆：

```json
{
  "target_roles": ["Backend Engineer"],
  "city": "Shanghai",
  "seniority": "mid",
  "resume_version": "v3",
  "selected_jobs": [],
  "pending_confirmations": []
}
```

## 练习

设计一个“学习计划 Agent”，要求：

- 先收集目标、基础、时间。
- 生成计划。
- 每周复盘。
- 根据测验结果调整计划。
- 高风险建议需要人工确认。

## 验收

- 能说明 Agent 的组成。
- 能定义状态字段和停止条件。
- 能把工具调用、用户确认和失败恢复写进 prompt。

## 易错

> **易错：** 让 Agent 自由行动，没有权限边界。
>
> 正确做法：每个外部动作都要有工具权限、确认策略和审计记录。
