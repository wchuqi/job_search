## 一、基础概念

### 1. SDD 是什么？

**参考答案：**

SDD 是 Specification-Driven Development，也就是规格驱动开发。它强调先把需求、约束、验收标准和任务拆解写成可审查、可验证的规格，再基于规格实现、测试和评审。在 AI 辅助开发中，SDD 的价值更明显，因为 Agent 需要明确意图、边界和完成标准。

> **重点：** SDD 的核心是“规格成为开发中心工件”，不是“多写文档”。

> **易错：** 把 SDD 解释成系统设计文档。系统设计文档可以是 SDD 的一部分，但 SDD 是开发方法。

### 2. SDD 和 TDD 有什么区别？

**参考答案：**

TDD 的中心是测试，流程是先写失败测试，再写实现让测试通过。SDD 的中心是规格，规格可以包含业务场景、约束、验收标准、设计边界和任务。TDD 可以作为 SDD 的验证手段：把规格中的验收标准转成测试。

> **重点：** SDD 定义“应该做什么”，TDD 验证“实现是否满足预期”。

### 3. 为什么 AI 编码时代更需要 SDD？

**参考答案：**

AI Agent 生成代码很快，但容易误解需求、忽略边界、违反架构或过度修改。SDD 把隐性需求显性化，让 Agent 有明确输入，也让人可以审查 Agent 是否按规格完成。没有 SDD，AI 编码容易变成一次性 prompt 和事后补救。

> **难点：** 关键不是让 Agent 更快写代码，而是让 Agent 写对、改对、验对。

### 4. Harness Engineering 是什么？

**参考答案：**

Harness Engineering 是为 AI Agent 设计运行外壳的工程实践，包括上下文装配、工具、权限、沙箱、验证、反馈、日志、恢复和治理。目标是让 Agent 在可控、可观测、可验证的环境中工作。

> **重点：** Harness 是系统，不是单条提示词。

### 5. Harness Engineering 和 Prompt Engineering 有什么区别？

**参考答案：**

Prompt Engineering 关注如何写好指令；Harness Engineering 关注 Agent 运行所需的完整工程环境。Prompt 可以告诉 Agent “不要改支付模块”，但 Harness 可以通过权限和检查器真正阻止或发现这类修改。

> **易错：** 认为写更长 prompt 就等于 Harness。正确理解是：prompt 是 Harness 的一个输入，不是 Harness 本身。

### 6. SDD 和 Harness Engineering 的关系是什么？

**参考答案：**

SDD 定义意图和验收，Harness 执行和验证这些意图。可以说 SDD 是 Agent 的任务契约，Harness 是 Agent 的执行环境。二者组合后，AI 编码才有机会从“生成代码”变成“工程交付”。

> **重点：** SDD 解决“目标清不清楚”，Harness 解决“执行可不可控”。
