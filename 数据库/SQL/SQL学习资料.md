# SQL 学习资料

这是一份跨数据库通用 SQL 专题索引。详细内容按知识点拆分到 `study-material/` 目录，覆盖查询、连接、聚合、窗口函数、数据建模、事务、索引、执行计划、SQL 方言差异、性能优化、生产排障和面试复习。

本资料以标准 SQL 心智模型为主，示例尽量使用 PostgreSQL、MySQL、SQL Server、Oracle 都容易理解的写法；涉及方言差异时会单独标注。它不替代某个数据库的运维资料，而是用于建立“会写、会读、会优化、会解释”的 SQL 能力。

## 使用建议

- 零基础：按 0 到 9 学习，先能稳定写出正确查询。
- 后端开发：重点读 3、4、5、6、8、10、11、12。
- 数据分析：重点读 5、7、17、20。
- 性能优化：重点读 11、12、15、16、21。
- 深水区机制：重点读 22 到 28，补齐关系代数、优化器、索引访问路径、事务异常、递归 CTE 和计划稳定性。
- 生产治理：重点读 12、19、21、29，建立 SQL 评审、发布、回滚和观测能力。
- 面试复习：先读 14 查漏，再读 13 和 `面试知识点/`。
- 跨数据库迁移：重点读 18，并结合目标数据库官方文档核对方言。

## 环境假设

- 推荐练习数据库：PostgreSQL 16+、MySQL 8.0+ 或 SQLite 3.40+。
- 本资料示例业务：招聘系统，包括候选人、岗位、投递、面试、Offer、薪资和部门。
- SQL 方言：默认写标准 SQL 风格；特殊语法会标注数据库。

## 学习验收

完成后你应该能够：

- 解释 SQL 逻辑执行顺序和优化器物理执行顺序的区别。
- 写出正确的过滤、连接、聚合、窗口函数和分页查询。
- 处理 NULL、重复行、外连接过滤、`NOT IN`、聚合粒度等高频坑。
- 设计带约束的业务表结构。
- 分析事务隔离和并发异常。
- 读执行计划，判断索引是否有效。
- 重构慢查询，解释性能改善原因。
- 回答 SQL 后端、数据分析和数据库岗位面试题。
