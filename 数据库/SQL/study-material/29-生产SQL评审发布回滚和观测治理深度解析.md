# SQL 学习资料：生产SQL评审发布回滚和观测治理深度解析

[返回索引](../SQL学习资料.md)

## 学习目标

- 建立生产 SQL 从开发、评审、发布、观测到回滚的闭环。
- 能评审 DDL、DML、查询、报表、迁移脚本的风险。
- 能设计慢查询监控、执行计划基线和 SQL 变更回滚方案。

## 理论导读

生产 SQL 不是写完能跑就结束。一个 SQL 可能带来锁表、慢查询、主从延迟、缓存击穿、连接池耗尽、磁盘临时文件暴涨、报表口径错误或敏感数据泄露。SQL 变更必须像代码一样评审和发布，并且比普通代码更重视数据可逆性。错误代码可以回滚版本，错误 DELETE 不能靠重启恢复。

## SQL 变更分类

| 类型 | 风险 |
| --- | --- |
| 查询 | 慢查询、锁等待、返回过多数据 |
| DML | 误更新、误删除、长事务、复制延迟 |
| DDL | 锁表、重写表、索引构建压力 |
| 报表 | 口径错误、资源争用、敏感数据 |
| 迁移 | 方言差异、数据质量、回滚困难 |
| 权限 | 越权访问、导出泄露 |

## 上线前评审清单

### 正确性

- 是否有边界数据测试：NULL、重复、无匹配、多匹配。
- JOIN 粒度是否正确。
- 聚合口径是否明确。
- 时间范围是否半开区间。
- 分页排序是否稳定。

### 性能

- 是否有执行计划。
- 是否使用合适索引。
- 估算行数和实际行数是否接近。
- 是否有大排序、大哈希、临时表。
- 是否限制返回列和行数。
- 是否验证热门和冷门参数。

### 并发和锁

- DML 是否分批。
- WHERE 是否命中索引。
- 事务是否足够短。
- 是否可能锁表或阻塞核心写入。
- 是否有重试和幂等。

### 安全

- 是否参数化。
- 动态结构是否白名单。
- 是否最小权限。
- 是否涉及敏感字段。
- 导出是否审计和脱敏。

## DML 发布规范

高危：

```sql
UPDATE candidates SET city = 'Shanghai';
```

安全流程：

```sql
-- 1. 先 SELECT 确认
SELECT candidate_id, city
FROM candidates
WHERE city IS NULL
LIMIT 100;

-- 2. 小批量 UPDATE
UPDATE candidates
SET city = 'Unknown'
WHERE city IS NULL
  AND candidate_id BETWEEN 1 AND 1000;

-- 3. 记录影响行数
-- 4. 分批执行
-- 5. 验证结果
```

## DDL 发布规范

- 大表加字段优先允许 NULL 或默认值策略要看数据库是否重写表。
- 建索引考虑在线能力和锁级别。
- 加 NOT NULL 前先清洗历史数据。
- 加唯一约束前先查重复。
- DDL 变更要有回滚或兼容方案。

## 观测指标

- 慢查询次数和 P95/P99。
- 查询总耗时和总调用次数。
- 扫描行数和返回行数。
- 临时文件或排序落盘。
- 锁等待和死锁。
- 连接池活跃数和等待数。
- 主从复制延迟。
- SQL 错误码和超时。

## 执行计划基线

关键 SQL 应保存：

```text
SQL 标识：
业务接口：
典型参数：
执行计划：
索引依赖：
平均/P95/P99：
返回行数：
扫描行数：
更新时间：
```

统计信息、索引、版本升级、数据量变化都可能导致计划变化。核心 SQL 应在发布前后对比计划。

## 回滚策略

| 变更 | 回滚 |
| --- | --- |
| 查询 SQL | 回滚应用版本或开关 |
| 索引新增 | 可删除，但注意影响依赖查询 |
| 字段新增 | 通常保留，应用回滚兼容 |
| 数据修复 | 需要备份旧值或反向脚本 |
| 删除数据 | 依赖备份恢复，风险最高 |
| 权限变更 | 保留授权记录和撤销脚本 |

## 生产事故剧本

### 慢 SQL 打满数据库

1. 确认 SQL 指纹和调用来源。
2. 临时限流或关闭功能开关。
3. 查看执行计划和锁等待。
4. 杀长查询需确认是否会导致回滚压力。
5. 加索引或改 SQL 前评估 DDL 风险。
6. 复盘缺失的评审和监控。

### 误 UPDATE

1. 立即停止继续写入。
2. 保留 binlog/WAL/审计。
3. 评估影响范围。
4. 从备份或日志恢复旧值。
5. 补充 WHERE 保护、权限和审批。

## 练习

1. 为一条 UPDATE 脚本写上线方案和回滚方案。
2. 为一个核心查询建立执行计划基线。
3. 写一份 SQL 评审意见，覆盖正确性、性能、并发、安全。
4. 设计慢查询告警指标。

## 验收

- 能评审生产 SQL 风险。
- 能写 DML/DDL 发布和回滚方案。
- 能设计 SQL 观测指标和执行计划基线。

## 重点

- 数据变更的回滚成本远高于普通代码变更。

## 难点

- SQL 风险跨越正确性、性能、锁、安全和数据可恢复性。

## 易错

> **易错：** 在生产直接执行一次性脚本，不保存影响行和旧值。
>
> 正确做法：先 SELECT 验证，小批量执行，记录影响，准备回滚。
