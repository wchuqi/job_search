## 一、聚合、窗口和分析查询

### 1. GROUP BY 的本质是什么？

**参考答案：**

GROUP BY 按分组列把多行压成每组一行，结果粒度由分组列决定。SELECT 中非聚合列必须属于分组粒度，否则语义不明确。

> **重点：** 聚合前先问“结果一行代表什么业务粒度”。

### 2. `DISTINCT` 和 `GROUP BY` 有什么区别？

**参考答案：**

`DISTINCT` 对结果集去重；`GROUP BY` 按分组列形成组，并可以计算聚合。只去重时二者有时结果类似，但语义不同。不要用 DISTINCT 掩盖错误 JOIN。

### 3. 条件聚合怎么写？

**参考答案：**

通用写法是 `SUM(CASE WHEN ... THEN 1 ELSE 0 END)`。

```sql
SELECT
    job_id,
    COUNT(*) AS total_count,
    SUM(CASE WHEN status = 'offered' THEN 1 ELSE 0 END) AS offered_count
FROM applications
GROUP BY job_id;
```

### 4. 窗口函数和 GROUP BY 有什么区别？

**参考答案：**

GROUP BY 通常减少行数，把多行聚合成每组一行；窗口函数不减少行数，而是在每一行上基于窗口计算一个值。窗口函数适合排名、累计、相邻行比较和每组 TopN。

### 5. 每组最新一条记录怎么写？

**参考答案：**

使用 `ROW_NUMBER()` 排名后过滤。

```sql
WITH ranked AS (
    SELECT
        a.*,
        ROW_NUMBER() OVER (
            PARTITION BY job_id
            ORDER BY applied_at DESC, application_id DESC
        ) AS rn
    FROM applications a
)
SELECT *
FROM ranked
WHERE rn = 1;
```

### 6. `ROW_NUMBER`、`RANK`、`DENSE_RANK` 区别是什么？

**参考答案：**

`ROW_NUMBER` 不处理并列，强行编号；`RANK` 并列后跳号；`DENSE_RANK` 并列后不跳号。是否保留并列取决于业务需求。

### 7. `LAST_VALUE` 为什么容易错？

**参考答案：**

因为有窗口 `ORDER BY` 时，默认窗口帧通常截止到当前行，`LAST_VALUE` 取到的可能是当前行而不是整个分区最后一行。应显式指定完整帧。

```sql
LAST_VALUE(status) OVER (
    PARTITION BY candidate_id
    ORDER BY applied_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
)
```
