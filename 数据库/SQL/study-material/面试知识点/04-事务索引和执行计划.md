## 一、事务、索引和执行计划

### 1. ACID 是什么？

**参考答案：**

ACID 分别是原子性、一致性、隔离性、持久性。原子性保证事务内操作要么都成功要么都失败；一致性保证约束和业务规则不被破坏；隔离性定义并发事务互相影响程度；持久性保证提交后数据不会丢失。

### 2. 常见隔离级别有哪些？

**参考答案：**

常见隔离级别包括 READ UNCOMMITTED、READ COMMITTED、REPEATABLE READ、SERIALIZABLE。隔离越强，并发异常越少，但潜在锁冲突或重试成本越高。不同数据库同名隔离级别实现可能不同。

### 3. 丢失更新怎么解决？

**参考答案：**

可以用原子更新、条件更新、版本号乐观锁或更高隔离级别解决。

```sql
UPDATE counters
SET value = value + 1
WHERE counter_id = 1;
```

或：

```sql
UPDATE applications
SET status = 'interviewing', version = version + 1
WHERE application_id = 1003
  AND version = 7;
```

### 4. 如何防止重复投递？

**参考答案：**

在数据库建立唯一约束 `UNIQUE(candidate_id, job_id)`，再用 UPSERT 或捕获唯一冲突。不能只依赖应用层“先查再插”，因为并发请求可能同时通过检查。

### 5. 什么是 SARGability？

**参考答案：**

SARGability 表示谓词能否有效利用索引查找。函数包列、隐式类型转换、前导通配 LIKE 等常让条件不可 SARGable。

```sql
-- 更好
WHERE applied_at >= '2026-03-01'
  AND applied_at < '2026-03-02'
```

### 6. 组合索引列顺序怎么设计？

**参考答案：**

结合查询模式设计。通常把等值过滤列放前面，再考虑范围列、排序列和覆盖列。例如查询某岗位最新投递：

```sql
CREATE INDEX idx_app_job_time
ON applications (job_id, applied_at DESC, application_id DESC);
```

### 7. 执行计划重点看什么？

**参考答案：**

看扫描方式、估算行数和实际行数、JOIN 算法、是否额外排序、是否回表、过滤是否发生在扫描后、是否有 hash 或 sort 溢出。不同数据库计划格式不同，但分析目标一致。
