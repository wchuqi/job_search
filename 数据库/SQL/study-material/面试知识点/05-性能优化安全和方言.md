## 一、性能优化、安全和方言

### 1. 慢查询如何排查？

**参考答案：**

先确认 SQL、参数、频率和耗时分布，再看执行计划，判断扫描行数、返回行数、JOIN 算法、排序、聚合、索引使用、统计信息和锁等待。优化后要对比执行计划和实际耗时，并验证结果正确性。

### 2. 深分页为什么慢，怎么优化？

**参考答案：**

`OFFSET` 深分页需要跳过大量行，越往后越慢。可以使用 keyset pagination，通过上一次最后一条记录的排序键继续查询。

```sql
WHERE applied_at < :last_applied_at
   OR (applied_at = :last_applied_at AND application_id < :last_id)
ORDER BY applied_at DESC, application_id DESC
LIMIT 20;
```

### 3. `SELECT *` 有什么问题？

**参考答案：**

会读取和传输不必要列，增加 IO、网络、内存和反序列化成本；也可能让覆盖索引失效。生产查询应只取需要的列。

### 4. SQL 注入的本质是什么？

**参考答案：**

应用把用户输入拼进 SQL 结构，让攻击者改变 SQL 语义。防护核心是参数化查询，把 SQL 结构和数据值分离。

### 5. 动态 ORDER BY 如何防注入？

**参考答案：**

列名和排序方向通常不能用普通参数占位符，需要白名单映射。

```java
Map<String, String> allowed = Map.of("createdAt", "created_at", "salary", "max_salary");
String column = allowed.getOrDefault(input, "created_at");
```

### 6. 常见 SQL 方言差异有哪些？

**参考答案：**

分页、UPSERT、字符串拼接、日期函数、布尔类型、JSON、标识符引号、NULL 排序、空字符串处理、事务隔离实现、CTE 优化策略都可能不同。

### 7. SQL 迁移怎么验证？

**参考答案：**

不仅要验证语法，还要验证结果集、NULL/空字符串/时间边界、执行计划、性能、并发行为和权限。迁移前应收集 SQL 分类，标记方言函数，构造边界数据并做对比测试。
