# SQL 学习资料：深度实验手册和能力验收

[返回索引](../SQL学习资料.md)

## 学习目标

- 用实验验证 SQL 核心机制。
- 建立正确性、性能、安全和方言意识。
- 完成最终能力验收。

## 实验 1：NULL 和三值逻辑

### 任务

构造包含 NULL 的数据，测试：

```sql
col = NULL
col <> 'x'
col IS NULL
NOT IN (包含 NULL)
NOT EXISTS
```

### 验收

- 能解释 UNKNOWN 如何影响 WHERE。
- 能说明为什么反连接更推荐 `NOT EXISTS`。

## 实验 2：外连接过滤

### 任务

写两条 SQL：

- 条件放 `ON`。
- 条件放 `WHERE`。

比较 LEFT JOIN 结果。

### 验收

- 能说明补 NULL 发生在 WHERE 之前。

## 实验 3：JOIN 行数放大

### 任务

构造一个投递多轮面试的场景，统计投递数和面试数，观察 JOIN 后 `COUNT(*)` 差异。

### 验收

- 能指出统计粒度。
- 能修复为先聚合或 `COUNT(DISTINCT ...)`，并说明取舍。

## 实验 4：窗口函数帧

### 任务

用重复时间数据测试：

- `ROWS`
- `RANGE`
- `LAST_VALUE` 默认帧
- 显式完整帧

### 验收

- 能说明窗口帧如何影响结果。

## 实验 5：索引和 SARGability

### 任务

对比：

```sql
WHERE DATE(applied_at) = '2026-03-01'
```

和：

```sql
WHERE applied_at >= '2026-03-01'
  AND applied_at < '2026-03-02'
```

### 验收

- 能通过执行计划说明索引使用差异。

## 实验 6：深分页

### 任务

构造大量投递数据，对比 OFFSET 和 keyset pagination。

### 验收

- 能说明 OFFSET 为什么越深越慢。
- 能设计稳定游标。

## 实验 7：并发重复写入

### 任务

模拟两个事务同时投递同一岗位：

- 无唯一约束。
- 有唯一约束。
- UPSERT 或捕获唯一冲突。

### 验收

- 能解释数据库约束如何保护并发正确性。

## 实验 8：SQL 注入

### 任务

构造字符串拼接查询，输入注入 payload，再改为参数化。

### 验收

- 能说明 SQL 结构和数据值分离。

## 自测问题

1. SQL 的逻辑执行顺序是什么？
2. `COUNT(*)` 和 `COUNT(col)` 有什么区别？
3. LEFT JOIN 后 WHERE 右表条件为什么可能丢行？
4. `NOT IN` 为什么怕 NULL？
5. 窗口函数和 GROUP BY 有什么区别？
6. 每组最新一条记录怎么写？
7. 如何防止并发重复插入？
8. 什么是 SARGability？
9. 组合索引列顺序如何设计？
10. SQL 注入如何防？

## 重点

- SQL 能力要通过实验和执行计划验证。

## 易错

> **易错：** 只背语法，不验证边界数据。
>
> 正确做法：每个知识点都用 NULL、重复行、无匹配行、并发写入等边界数据测试。
