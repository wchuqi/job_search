# SQL 学习资料：综合练习项目

[返回索引](../SQL学习资料.md)

## 项目目标

围绕招聘系统完成一套 SQL 综合练习，覆盖建模、查询、聚合、窗口、事务、索引、慢查询和安全。

## 项目结构

```text
sql-practice/
  01-schema.sql
  02-seed-data.sql
  03-basic-queries.sql
  04-join-aggregate.sql
  05-window-functions.sql
  06-transactions.sql
  07-index-and-explain.sql
  08-security-review.md
  09-optimization-report.md
```

## 任务 1：建模

- 创建部门、岗位、候选人、投递、面试、Offer 表。
- 设置主键、外键、唯一约束、检查约束。
- 为高频查询设计初始索引。

### 验收

- 重复投递会被唯一约束阻止。
- 非法状态会被 CHECK 阻止，若目标数据库支持。
- 投递不能引用不存在的候选人和岗位。

## 任务 2：基础查询

- 查询开放岗位。
- 查询上海候选人。
- 查询最近 30 天投递。
- 实现稳定分页。

## 任务 3：JOIN 和聚合

- 查询候选人投递岗位明细。
- 查询没有投递的候选人。
- 统计每个岗位投递数。
- 统计每个部门 Offer 数。
- 修复一个 JOIN 后统计翻倍的问题。

## 任务 4：窗口函数

- 每个岗位最新 3 条投递。
- 每个候选人最近一次投递。
- 每个岗位按时间累计投递数。
- 每个部门薪资 Top3 岗位。

## 任务 5：事务和幂等

- 实现同一候选人同一岗位只能投递一次。
- 写 UPSERT 或捕获唯一冲突方案。
- 用条件 UPDATE 实现状态流转。
- 说明并发下为什么不会重复。

## 任务 6：索引和执行计划

- 为岗位投递列表设计组合索引。
- 对比函数包列和范围查询。
- 对比 OFFSET 深分页和 keyset 分页。
- 写一份优化报告。

## 任务 7：安全评审

- 找出动态 SQL 注入风险。
- 改成参数化和白名单。
- 设计应用账号最小权限。
- 说明导出敏感字段如何脱敏。

## 最终验收

- 提交所有 SQL 文件。
- 每个查询说明业务目标。
- 每个优化说明执行计划变化。
- 每个写操作说明并发安全依据。
- 安全评审覆盖参数化、白名单、权限和脱敏。

## 重点

- 综合项目要求你把 SQL 当成工程能力，而不是语法题。

## 易错

> **易错：** 只写查询，不验证结果和执行计划。
>
> 正确做法：每条关键 SQL 都给出预期结果、边界数据和性能证据。
