# SQL 学习路线图

## 阶段 1：关系模型和查询基础

- 目标：理解 SQL 查询的是“关系表”和“结果集”，不是逐行脚本。
- 需要掌握：表、行、列、主键、外键、结果集、投影、选择、排序、分页。
- 例子：查询最近 30 天投递记录，按投递时间倒序分页。
- 练习：创建候选人、岗位、投递三张表，写 10 条基础查询。
- 验收：能不用提示写出 `SELECT ... FROM ... WHERE ... ORDER BY ... LIMIT/OFFSET`。
- 重点：SQL 是声明式语言，描述“要什么”，不直接描述“怎么逐行执行”。
- 易错：把 SQL 当循环脚本，忽略结果集和集合语义。

## 阶段 2：NULL、连接和聚合

- 目标：掌握 SQL 最容易出错的三块：NULL、JOIN、GROUP BY。
- 需要掌握：三值逻辑、内连接、外连接、半连接、反连接、聚合粒度、HAVING。
- 例子：查询“没有面试记录的候选人”、统计每个岗位投递数。
- 练习：分别用 `NOT EXISTS` 和 `LEFT JOIN ... IS NULL` 写反连接。
- 验收：能解释 `WHERE right_table.col = ...` 为什么可能把左连接变成内连接效果。
- 重点：连接会改变行数，聚合会改变粒度，NULL 会改变布尔判断。
- 难点：外连接过滤位置、`COUNT(*)` 和 `COUNT(col)`、`NOT IN` 遇到 NULL。

## 阶段 3：子查询、CTE和窗口函数

- 目标：能写分层查询、排名、分组 TopN、累计统计和同比环比。
- 需要掌握：标量子查询、相关子查询、派生表、CTE、窗口分区、排序、窗口帧。
- 例子：查询每个部门薪资最高的 3 个岗位；计算候选人连续投递间隔。
- 练习：用 `ROW_NUMBER()` 实现每组最新一条记录。
- 验收：能说明窗口函数和 GROUP BY 的区别。
- 重点：窗口函数不减少行数，聚合通常减少行数。
- 易错：忘记窗口帧默认规则，导致 `LAST_VALUE` 结果不符合预期。

## 阶段 4：写操作、建模和事务

- 目标：能安全地设计表、写入数据并处理并发。
- 需要掌握：INSERT、UPDATE、DELETE、MERGE/UPSERT、约束、范式、事务隔离、锁。
- 例子：实现“同一候选人不能重复投递同一岗位”的唯一约束和幂等写入。
- 练习：设计招聘系统核心表，并写并发投递的事务方案。
- 验收：能解释脏读、不可重复读、幻读、丢失更新和写偏斜。
- 重点：数据正确性优先依赖约束和事务，不应只依赖应用层 if 判断。
- 易错：没有唯一约束却以为应用层查重可以防重复。

## 阶段 5：索引、执行计划和慢查询

- 目标：从“能写 SQL”升级到“能解释为什么快或慢”。
- 需要掌握：B+Tree 索引、组合索引、选择性、覆盖索引、SARGability、执行计划、基数估算。
- 例子：优化职位搜索、候选人筛选、投递统计报表。
- 练习：对比函数包列、隐式类型转换、前导通配 LIKE 对索引的影响。
- 验收：能读懂 `EXPLAIN` 中扫描方式、连接方式、排序和估算行数。
- 难点：优化器可能重写 SQL，物理执行顺序不等于书写顺序。

## 阶段 6：深度机制、方言和生产安全

- 目标：具备跨数据库迁移、生产排障和高级面试能力。
- 需要掌握：优化器重写、JOIN 算法、窗口帧、SQL 方言、权限、SQL 注入、慢查询治理。
- 例子：把 MySQL 查询迁移到 PostgreSQL，检查分页、UPSERT、日期函数、字符串函数差异。
- 练习：为慢查询写一份诊断报告：现象、SQL、执行计划、索引、改写、验证。
- 验收：能给出“正确性、性能、安全、兼容性”四维 SQL 评审意见。

## 推荐学习节奏

| 周期 | 学习重点 | 产出 |
| --- | --- | --- |
| 第 1 周 | SELECT、WHERE、ORDER BY、分页 | 30 条基础查询 |
| 第 2 周 | NULL、JOIN、GROUP BY | 连接和聚合错题集 |
| 第 3 周 | 子查询、CTE、窗口函数 | TopN、排名、累计统计 |
| 第 4 周 | DML、DDL、事务 | 招聘系统表结构和幂等写入 |
| 第 5 周 | 索引和执行计划 | 5 个慢查询优化报告 |
| 第 6 周 | 方言、安全、面试 | SQL 完整清单和面试复述 |
| 第 7 周 | 深水区机制 | 关系代数改写、优化器估算、B+Tree 访问路径笔记 |
| 第 8 周 | 并发和生产治理 | 事务异常实验、生产 SQL 评审和回滚方案 |

## 最终能力清单

- 能写出正确、清晰、可维护的 SQL。
- 能解释逻辑执行顺序和物理执行计划的区别。
- 能处理 NULL、连接、聚合、窗口函数的边界问题。
- 能设计基础业务表和约束。
- 能理解事务隔离和并发异常。
- 能通过执行计划定位慢查询。
- 能识别 SQL 注入和权限风险。
- 能回答 SQL 面试中的机制题和场景题。
- 能解释优化器为什么选错计划，以及如何用统计信息和参数样本验证。
- 能把复杂报表拆成可校验的分层 SQL。
- 能为生产 SQL 变更写评审、发布、观测和回滚方案。
