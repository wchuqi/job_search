## 一、基础概念

### 1. PostgreSQL 中 cluster、database、schema 有什么区别？

**参考答案：**

cluster 是一个 PostgreSQL 实例的数据目录和运行环境，由 `initdb` 初始化，里面可以包含多个 database。database 是连接和隔离边界，同一个连接只能连接到一个 database。schema 是 database 内的命名空间，用来组织表、视图、函数等对象。

```sql
SELECT current_database(), current_schema();
SHOW data_directory;
```

> **重点：** cluster 大于 database，database 大于 schema。
>
> **易错：** 把 schema 当成 database，导致误以为可以跨 database 直接 JOIN。

### 2. PostgreSQL 13 为什么仍然值得学？生产使用有什么风险？

**参考答案：**

PostgreSQL 13 的核心机制和现代 PostgreSQL 一脉相承，包括 MVCC、WAL、优化器、索引、复制和权限模型。很多存量系统仍在使用它，所以学习有实际价值。

风险是 PostgreSQL 13 已进入 EOL，安全补丁和 bugfix 停止后，新漏洞和兼容问题无法依赖官方修复。生产环境应规划升级到受支持版本，并验证扩展、SQL、复制、备份和应用兼容性。

> **重点：** 学机制仍有价值，但新项目不应优先选 EOL 版本。

### 3. `psql` 中常用的元命令有哪些？

**参考答案：**

常用元命令包括：

```text
\l              -- 列出数据库
\c dbname       -- 切换数据库
\dn             -- 列出 schema
\dt             -- 列出表
\d+ table_name  -- 查看表详情
\df             -- 查看函数
\x              -- 扩展显示
\timing         -- 显示执行耗时
\copy           -- 客户端导入导出
```

> **重点：** `\copy` 在客户端读写文件，SQL `COPY` 在服务端读写文件。

### 4. `postgresql.conf` 和 `pg_hba.conf` 分别负责什么？

**参考答案：**

`postgresql.conf` 负责数据库运行参数，例如监听地址、端口、内存、WAL、日志、autovacuum 等。`pg_hba.conf` 负责客户端认证规则，决定哪些来源、哪些用户、哪些数据库可以用什么认证方式连接。

`pg_hba.conf` 从上到下匹配，第一条命中后停止。

> **重点：** 连接问题要同时检查监听、网络、`pg_hba.conf` 和账号权限。
>
> **易错：** 修改 `listen_addresses='*'` 后以为外部一定能连。

### 5. PostgreSQL 的后台进程有哪些？各自有什么作用？

**参考答案：**

常见后台进程包括：

- checkpointer：周期性触发 checkpoint，把脏页刷盘。
- background writer：提前写出脏页，降低后端进程刷盘压力。
- WAL writer：负责 WAL 写出。
- autovacuum launcher/worker：自动清理 dead tuple 和更新统计信息。
- wal sender：向备库或复制客户端发送 WAL。
- 后端进程：每个客户端连接通常对应一个服务进程。

> **重点：** 很多生产问题和后台进程有关，例如 autovacuum 跟不上、checkpoint 过频、复制发送滞后。

### 6. PostgreSQL 中 extension 是什么？

**参考答案：**

extension 是扩展包，可以向数据库增加对象和能力，例如函数、类型、操作符、索引方法或统计能力。常见扩展有 `pg_stat_statements`、`pgcrypto`、PostGIS 等。

```sql
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;
```

扩展通常在 database 内安装，不是所有 database 自动可用。生产安装要验证权限、版本兼容和备份恢复。

> **易错：** 以为在一个库安装扩展后另一个库也自动可用。
